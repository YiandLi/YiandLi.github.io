现状篇

  • 使用完整的评测集来进行检验，而不是单纯的一两个样本。
  • 需要一个具有统计意义上的数据集：
    • 数量上能支撑特定指标的计算足够稳定。
    • 抽样情况和实际使用的场景尽可能一致。
    • 质量可靠，足够支撑当前方案的数据集。
  • 指标
  • 结论

分析篇

准确率分错的case，看漏召回的case

粗看

笼统地大体上去看，错误的样本都有什么特点，带有那些特征，例如长度、句式等是否有什么特点，又或者是问句等等。
这个只要通篇看下来，就会有比较明显的特点了，一下子能总结出比较突出的问题。
快速看快速找到比较大的问题，优点就是快，但问题是你要能粗看的出来才行，所以缺点就是细节问题比较难发现，而且有些问题是比较深层次的，例如同样是问句，但是为什么问句就会出错

追溯法

对一个case，重现整个训练和预测过程的方法。给定一个bad case，准备好日志，分析预测的每个阶段的分析结果，查看是否符合预期，不符合预期的，大概率这个部分有bug、词典词汇等什么明显的问题了。

比如，很多问句会被误召回，这时候我们可以查查训练集中带有问句关键词的正负样本的比例是不是有问题，例如正样本远多于负样本，此时模型很可能就会认为只要带有疑问词的就是正类，我们需要加一些带疑问词的负样本进去。

解决法

例如短句预测不正确，而且一般都比较模糊，所以我们考虑的是过滤所有短句，然后再通过白名单的方式补召回，那我们就可以把query归类为短句。

问题标记

case是需要我们逐个分析的，后续我们需要对重点问题进行归纳总结，所以case分析往往是需要我们做一些关键的记录。

query 原因分析
关闭 短句
今天天气怎么样 疑问词，关键词拒绝
你为什么 疑问词，模糊

标记的时候注意词汇和问题的统一，方便我们后续统计各个问题的占比，值得注意的是，一个问题是可能有多个解决方法的，我们记得要标记上，统计分析的时候都是要关注的。

问题统计和归纳

有了上面的标记，我们能很轻易地看拿到各个问题的分布情况，甚至可以总结出来。

解决篇

样本的误导

对于一些数据内很浅层直白的信息，模型也会预测的非常直白，例如正类里面有大量含有“怎么”的样本，而负类里面没有，则模型很有可能会把所有带有“怎么”的句子都预测为正类。这个时候就需要负采样了。

阈值的确定和权衡

阈值本质是个准入条件，是和模型一起出现的标配，毕竟模型大部分时候只提供意见，要不要采纳还是要通过阈值（参考sigmoid，一般阀值为0.5）。
阀值变小，召回率变高，精确率下降，我们需要确定一个最佳阈值，对于搜索和对话场景，更多的是需要我们保准确，一般是0.85+，召回率另说。

softmax 呢？相当于双重条件？max+大于阀值

前后处理和预处理

不同的场景和数据的预处理方式会很不同，哪怕是同样的NLP问题，到了细分场景也会有很多不同的操作，共同的可能是对标点的处理，而不同的例如客服场景要把“你好”之类的删除，语音助手要把“小爱同学”、“小布”之类的称呼删除等，重点不在于做了什么，而是要把一些对预测有影响干扰项、不利于预测的因素给去掉，至于哪些是干扰项，请回去看bad case，这些错误样本会告诉你。

需要把一些肯定确定能快速预测出来的东西给过滤掉，从而提升性能也降低模型的负担，最简单的例如黑白名单，复杂的可以有一些针对业务的规则，例如超短句或者超长句直接拒绝等等，能用规则的尽量用规则，能不用模型的尽量不用模型，模型的不可控性众所周知，我们知道修复bad case的困难性的情况下，没必要太往依赖模型了。

后处理是指在模型预测以后的一些调整，最直接能想到的就是阈值过滤，但不仅是这些，有的时候需要结合模型的预测打分进行调整，例如关键词抽取领域，需要对预测的重要性分数进行加减分，如句首词和句尾词要提权，对于百科的一些问题，语义相似度的定义可能会和实际应用不同，于是我们也要进行一些过滤，例如“五一节的由来”和“六一节的由来”从语义相似度上来说确实很接近，但是我们不能然你给他们相似，因为“五一节”和“六一节”很不同，所以我们通过“实体约束”来过滤掉他们，当然背后就需要一批实体词典了。

超详细的Yolo检测框预测分析
目标检测|YOLO原理与实现

原本目标分割采用最简单的分类方法，遍历边框（位置，大小），然后分类是否对应某一目标类别（其实还分两步，是前景还是背景，如果是前景的话属于哪一个类别）。

这里 yolo 将其转化为回归问题，让模型直接预测出向量 (x,y,w,h,c)(x,y,w,h,c)(x,y,w,h,c) 表示一个被检测出来的对象，c 为置信度 confidence。

现将图片切分成不同的 patch 区域，逐 patch 检测对象。为什么分 patch：因为conv操作是位置强相关的，就是原来的目标在哪里，你conv之后的feature map上还在哪里，所以图片划分为 n 个区域，结果也应该分布在 n 个区域上 。

如果每个区域让模型预测一个向量 (x,y,w,h,c)(x,y,w,h,c)(x,y,w,h,c) ，则有 n 个可能的结果。

问题：

  1. 同一个目标可能被切分到不同的区域中了，针对这样的情况：

    • 训练时的 label：我们就看这个物体的中心落在了哪个grid里面，哪个 grid 的 c 就为 1 。
    • 在 inference 时，也会出现相邻的两个 grid 预测出来同一个物体，但是两个被预测出来的窗口大小不一定相同，这时候需要一个指标来消歧： NMS(非极大值抑制)
      重合度的计算方法：交并比IoU = 两个框的交集面积 / 两个框的并集面积 。 于是可以使用这个指标来得到最终的检测答案

  2. 同一个区域如何检测到多个对象呢？
    刚才区域是 4×4 ，现在变成 40×40 ，或者更大，使区域更密集，就可以缓解多个目标的问题，但无法从根本上去解决。

  3. 多物体类别检测，怎么办？
    上面是单分类所以直接使用置信度表示概率，没有表示类别的量，而这里如果是两个类别，可以加上两个维度，即 (c,x,y,w,h,p1,p2)(c,x,y,w,h,p1,p2)(c,x,y,w,h,p1,p2) ，其中 (p1,p2)(p1,p2)(p1,p2) 表示两个类别的 one-hot 编码。

  4. 如果想在同一区域预测多个不同类别的 token，如何做呢？
    这个问题和 问题2 一样，还是无法很好的解决。

  5. 小目标难以拟合：小目标总是检测不佳，所以我们专门设计神经元去拟合小目标。
    小目标和大目标需要分开去预测，效果比较好，所以这里设置了两个 5 元组，被称为 bbox 或者 cell，分别负责两种目标，而标签还是 one-hot ，所以一共长度为 12 。

  6. 存在样本不均匀问题，能预测出来的 grid 还是占了少数的

得到更多目标

yolo-v1 一次只能得到 49 个目标，即一个框内部只能有一个目标，同时每一个 patch 的两个框对应一个类别，可以是大目标也可以是小目标。

这就导致当目标比较多时，效果很差。

基于此，作者将原始的 7×77\times 77×7 进行拓展，变成了 13×1313 \times 1313×13 个 patch，每个区域有 5 个 anchor ，同时每个 anchor 对应一个 1个类别。
这样每个 patch 中就能预测出来 5 个潜在的类别，总体可以预测 13∗13∗5=84513*13*5 = 84513∗13∗5=845 个目标，进而最后出来的 logistic元素 个数为 13∗13∗5∗(5+20)=2112513 * 13 * 5 * (5 + 20) = 2112513∗13∗5∗(5+20)=21125 。

这样拆分后，每一个框和一组 xywh 绑定，同时需要多预测出来一组 class label，也就是一个 cell 最多预测 num of anchor 个 object 。即，object class num 不变，但是 num of object 变多了，等于 num of anchor 了。

分析

为什么需要 anchor ？ -> 注入先验知识
作者观察了很多图片的所有Ground Truth，发现：比如车，GT都是矮胖的长方形，再比如行人，GT都是瘦高的长方形，且宽高比具有相似性。
其实可以看成是并行地降低预测头预测的难度，因为相当于变相进行归一了预测的空间范围。
同时如果每一个 anchor 都可以预测一个 object，则也可以得到更多目标。

缺点：增加了推理最后一层的复杂度

怎么得到 anchor ？
对于任意数据集，将标签的 w,h 进行聚类，聚成几类呢？( Faster-RCNN 中的 Anchor 的宽高和大小是手动挑选的)。

作者进行了实验发现 anchor 数量越多，效果确实越好，但是复杂度也变差。

最终选择5类，衡量了 recall vs. complexity 。

小目标分割检测

目标检测发展很快，但对于小目标的检测还是有一定的瓶颈，特别是大分辨率图像小目标检测。比如 7920∗21607920*21607920∗2160 ，甚至 16000∗1600016000*1600016000∗16000 的图像。

最后一层变成了 19∗19,38∗38,76∗7619*19, 38*38, 76*7619∗19,38∗38,76∗76 ，其中最细粒度的 76∗7676*7676∗76 负责检测小目标 ，对应到 7920∗21607920*21607920∗2160 上，那么最终每一个像素表示了原始的 (7920//76)∗(2160//76)=104∗28(7920//76) * (2160//76) = 104 * 28(7920//76)∗(2160//76)=104∗28 ，然后去进行回归。

那么也带来一个困难，即如果原始图像中目标的宽或高小于 104∗28104 * 28104∗28 像素，网络很难学习到目标的特征信息。

高分辨率 , 显卡爆炸

很多图像分辨率很大，如果简单的进行下采样，下采样的倍数太大，容易丢失数据信息。

但是倍数太小，网络前向传播需要在内存中保存大量的特征图，极大耗尽GPU资源,很容易发生显存爆炸，无法正常的训练及推理。

-> 2018 年 YOLT 算法，大分辨率图片先进行分割，变成一张张小图，再进行检测，最后将监测目标组合起来 。

针对 Overlap 问题
为了避免两张小图之间，一些目标正好被分割截断，所以两个小图之间设置overlap重叠区域，比如分割的小图是 960*960 像素大小，则overlap可以设置为 960*20%=192 像素。

个人理解：虽然模型可以拟合到边边角角，但是仍然需要特征，如果没有重合的话，单个子图就看不到调整了，即目标就是被切开了。

每个小图检测完成后，再将所有的框放到大图上（ 按照在大图上裁剪的位置，直接回归到大图 ）。
对大图整体做一次nms操作，将重叠区域的很多重复框去除。

这样操作，可以将很多小目标检出，比如16000*16000像素的遥感图像。

优点：

（1）准确性

分割后的小图，再输入目标检测网络中，对于最小目标像素的下限会大大降低。

比如分割成608608大小，送入输入图像大小608608的网络中，按照上面的计算方式，原始图片上，长宽大于8个像素的小目标都可以学习到特征。

（2）检测方式

在大分辨率图像，比如遥感图像，或者无人机图像，如果无需考虑实时性的检测，且对小目标检测也有需求的项目，可以尝试此种方式。

缺点：

（1）增加计算量

比如原本7680*2160的图像，如果使用直接大图检测的方式，一次即可检测完。

但采用分割的方式，切分成N张608*608大小的图像，再进行N次检测，会大大增加检测时间。

借鉴Yolov5的四种网络方式，我们可以采用尽量轻的网络，比如Yolov5s网络结构或者更轻的网络。

当然Yolov4和Yolov5的网络各有优势，我们也可以借鉴Yolov5的设计方式，对Yolov4进行轻量化改造，或者进行剪枝。

这两个 biases 让 CNN 比 transformer 学的更快，即在相对少的数据上训练的结果更好。

动机：

  • 卷积是一个基于划窗的算法，它和其它嵌入（位置嵌入等）的融合比较困难。
  • 信息密度不同：
    • 文本数据比较密集，仅仅预测文本中的几个被掩码掉的单词就能很好的捕捉文本的语义特征。即 mask 率不高也能学习到整体的语言。
    • 图像数据是一个信息密度非常小的矩阵，其中包含着大量的冗余信息，而且像素和它周围的像素仅仅在纹理上就有非常大的相似性，恢复被掩码的像素并不需要太多的语义信息。
      基于此，是需要将 mask ratio 调高到。
  • 解码器（ maps the latent representation back to the input）的作用不同（reconstruct text and images）：
    • 文本数据的解码器需要了解文本的语义信息。预测一个 token 需要考虑全局的语义信息，输出语义级别很丰富（contain rich semantic information）。
    • 在计算机视觉的掩码预测任务中，预测被掩码的像素（reconstructs pixels）往往对图像的语义信息依赖的并不严重，输出语义级别很低（of a lower semantic level）。
    • 在 CV 领域，Decoder 的作用是重建 image pixels，所以 Decoder 的输出语义级别很低。在 NLP 领域，Decoder 的作用是重建 sentence words ，所以 Decoder 的输出语义级别很丰富。

火的一个原因：igpt 是生成模型，vit是判别模型，一半生成模型都比不上判别模型。但是 mae 其实算是生成模型，但是效果比判别模型要好很多。

核心：通过75%的高掩码率来对图像添加噪音，这样图像便很难通过周围的像素来对被掩码的像素进行重建，迫使编码器去学习图像中的语义信息。

模型：非对称的Encoder-Decoder架构的模型

  • 结构上：Encoder 参考 VIT，Decoder是一个轻量级的结构，它在深度和宽度上都比Encoder小很多。
  • 输入上：Encoder仅将未被掩码的部分作为输入，而Decoder将整个图像的Patch（掩码标志和Encoder编码后的未被掩码patch的图像特征）作为输入。

模型

an asymmetric encoder-decoder design -> a large reduction in computation & memory consumption
Encoder 只输入 observed signal / unmasked patch ； Decoder 比较轻量（lightweight）

Encoder : maps the observed signal to a latent representation

这种做法可以大幅度降低原本图像的冗余度 largely eliminates redundancy ， creating a task that cannot be easily solved by extrapolation from visible neighboring patches。
将输入减少了75%，因此训练速度也提高了75%。

模型部分就是 ViT ：

  • 输入不是整张图，而是只输入了未被掩码的部分 ; operates on a small subset (e.g., 25%) of the full set; Masked patches are removed
  • Encoder会为每一个未被掩码Patch计算一个特征向量，也要加 positional embedding 。
    encoder embeds patches by a linear projection with added positional embeddings
  • processes the resulting set via a series of Transformer blocks

Decoder : reconstructs the original signal from the latent representation
输入为 the full set of tokens ，包含两部分：

  1. encoded visible patches： 一个是图像嵌入，它由 Encoder编码之后的特征
  2. mask tokens： Decoder的被掩码的标志是一个共享的且可以学习的嵌入 [mask] ，只是位置编码不一样。
  3. 都需要加上位置编码（ 整个图像的1-D位置编码 ）

模型结果上，和 encoder 结构是相同的，不是自回归 token by token；参数是不同的。

MAE Decoder 仅用于预训练期间执行图像重建任务。做任务只用到了 Encoder 来得到 image representations 。
自监督学习的特点就是只用最后预训练好的 Encoder 完成分类任务。

Pre-train Loss
在 decoder 最后拼接一个 linear project，将输出的 patch 重构为 pixel values。所以 Decoder 的输出会进一步 reshape 成图像的形状 。
损失函数就是 pixel-level MSE Loss，这里和 bert 一样，只去计算masked patches 上的 loss 。

作者还尝试了另外一种方案，计算 loss 前归一化一下：

  • 先计算出每个 patch 的像素值的 mean 和 deviation（the mean and standard deviation of all pixels in a patch），
  • 使用它们去归一化这个 patch 的每个像素值。最后再使用归一化的像素值进行 MSE Loss 计算。发现这样做的效果比直接 MSE Loss 好 。

两种范式：

  1. full fine-tuning（更新所有模型参数）
  2. linear probing （只更新最后一个linear layer参数）
    预训练模型的表征层的特征固定，参数固化后未发生改变，只通过监督数据去训练分类器（通常是Softmax分类器或者SVM分类器等等）。

does not require any specialized sparse operations

  1. generate a token for every input patch (by linear projection with an added positional embedding)
  2. randomly shuffle the list of tokens and remove the last portion of the list, based on the masking ratio -》 a small subset of tokens for the encoder
  3. After encoding, we append a list of mask tokens to the list of encoded patches, and un-shuffle this full list
  4. The decoder is applied to this full list (with positional embeddings added).

Abstract

当前实体识别和关系的联合抽取模型 （joint） 严重依赖于人工标注的特征（hand-crafted features），比如 POS（词性标记器）和依赖关系（dependency parsers）。

这篇论文的着力点是研究一个联合模型，该模型同时执行实体识别和关系抽取任务，而且同时处理多个关系。
同时，还有三个优点：不依赖现有的自然语言处理工具； 同时提取一个句子中的实体和关系；一个实体可以涉及多种关系。

模型


将CRF用于实体识别，并且将关系抽取视为多头选择问题，为每一个实体识别潜在的多种关系。
多头的意思为，任何实体都有可能存在于不同的关系中。

  • 模型架构
    • Embedding layer
    • BiLSTM layer
    • CRF layer
    • Sigmoid scoring layer
  • 过程：
    • 输入的 Token Embedding 放入 BiLSTM layer，得到隐藏层表征。
    • CRF层和Sigmoid层可以同时输出两个任务
    • 每个 token 的输出为两部分：
      1. 实体识别标签（e.g., I-PER）
      2. 元组集合，每个元组包含**实体头标记(head tokens of the entity)**和两个实体之间的关系（eg.{(Center, Works for), (Atlanta, Lives in)}）。
    • 为了消除冗余，实体头标记为实体的最后一个token。同时使用标签 N 表示该 Token 没有关系。
