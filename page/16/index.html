RandomWalk 是一种可重复访问已访问节点的深度优先遍历算法 DFS，用来采样得到节点的共现关系。
RandomWalk 跳转时是概率一致的，即相同的概率跳到下个邻居结点。

目标函数为最大化 skip-gram 的似然概率，使用单词来预测context：

minimize⁡Φ−log⁡Pr⁡({vi−w,⋯ ,vi+w}\vi∣Φ(vi))\underset{\Phi}{\operatorname{minimize}} \quad-\log \operatorname{Pr}\left(\left\{v_{i-w}, \cdots, v_{i+w}\right\} \backslash v_{i} \mid \Phi\left(v_{i}\right)\right) Φminimize​−logPr({vi−w​,⋯,vi+w​}\vi​∣Φ(vi​))

context是给定节点单词的左右两边 w窗口 内的单词组成；
不考虑句子中context单词出现的顺序，直接最大化所有出现在上下文中单词的概率（和 word2vec 一致）。

调整节点移动到下一个邻居的概率为：跳转概率 = 当前出边权重/所有出边的权重和，作者称其为 biased random walks。
这种 biased random walks 使 node embedding 的结果在网络的 同质性（homophily） 和 结构性（structural equivalence） 中进行权衡。

  • 同质性（homophily）：距离相近节点的embedding应该尽量近似；
    深度优先搜索（DFS）反映同质性
  • 结构性（structural equivalence）：结构上相似的节点的embedding应该尽量接近。
    比如各个社群中心点的 embedding 表达也应该近似。
    宽度优先搜索（BFS） 反映结构性；

Biased random walks：在考虑跳转概率时，考虑二跳关系，记当前节点为 vvv ，上一个节点为 ttt ，则跳往下一个节点 xxx 的概率为：

αpq(t,x)={1p if dtx=01 if dtx=11q if dtx=2\alpha_{p q}(t, x)= \begin{cases}\frac{1}{p} & \text { if } d_{t x}=0 \\ 1 & \text { if } d_{t x}=1 \\ \frac{1}{q} & \text { if } d_{t x}=2\end{cases} αpq​(t,x)=⎩⎪⎪⎨⎪⎪⎧​p1​1q1​​ if dtx​=0 if dtx​=1 if dtx​=2​

dtxd_{tx}dtx​ 指的是节点 ttt 到节点 xxx 的距离，参数 ppp 和 qqq 共同控制着随机游走的倾向性。

  • 参数 ppp 被称为返回参数（return parameter）。
    ppp 越小，越容易维持在中心点附近，则越近似宽度优先搜索，node2vec就更注重表达网络的结构性。
    ppp 越小，越容易返回，一直在起始点周边震荡，是一种在社区中的微观搜索，能够发现节点处于中心（频率高）还是边缘（频率低），结果越反映结构性。

  • 参数 qqq 被称为进出参数（in-out parameter）。
    qqq 越小，则随机游走到远方节点的可能性越大，近似深度优先搜索，node2vec就更注重表达网络的同质性。
    qqq 越小，则越容易往远处走，进行更广范围的搜索，才能发现不同“群体”的节点之间的相似性，才能更加体现宏观的同质性。

边的权重 wij\mathrm{w}_{\mathrm{ij}}wij​ 有大有小，反向传播时是直接乘到梯度上的。
如果图中的边权方差很大，则很难选择一个合适的学习率。若使用较大的学习率那么对于较大的边权可能会引起梯度爆炸，较小的学习率对于较小的边权则会导致梯度过小。

如果所有边权相同，那么选择一个合适的学习率会变得容易。

方法1： 这里采用了将带权边拆分为等权边的一种方法，假如一个权重为 www 的边，则拆分后为 www 个权重为1的边。即多复制了 w−1w-1w−1 个相同的节点，梯度从原始的乘性变成了加性。
这样可以解决学习率选择的问题，但是由于边数的增长，存储的需求也会增加。

方法2： 另一种方法则是按 wijw_{ij}wij​ 来对边（其实就是相应的邻居节点）进行采样计算，不对所有边进行计算。
每条边/邻居节点被采样的概率正比于原始图中边的权重，这样既解决了学习率的问题，又没有带来过多的存储开销。
这也是作者的做法。

显示指定的目录或文件所占用的磁盘空间。
注意区分磁盘空间和文件大小 ，ls -l 显示的是真实子目录大小，du显示的是 占用磁盘空间 ，即占用 block 的总大小。
一般来说，存储文件的时候很少刚好把block填满，因此du -sh *一般都要比ls -lh大。

比如文件系统的block是 4K，一个15K的文件占用的空间是 15k/4k=3.75 个block，一个block只能被一个文件占用，因此实际占用空间就是4个block，就是16K。

  • -h或--human-readable：以K，M，G为单位，提高信息的可读性。`
  • -s或--summarize 仅显示该目录的和总。
  • -c或--total 除了显示个别目录或文件的大小外，同时也显示所有目录或文件的总和。
  • -d N参数或者--max-depth=N来控制du命令递归输出的目录层次数。N表示递归输出到第几级目录。

  2. ps -ef | grep xxx （功能描述：可以查看子父进程之间的关系），用于查看进程的 父进程 ID 。

PID：进程的 ID 号
%CPU：该进程占用 CPU 资源的百分比，占用越高，进程越耗费资源；
%MEM：该进程占用物理内存的百分比，占用越高，进程越耗费资源
STAT：进程状态。
常见的状态有：R：运行状态、S：睡眠状态、T：暂停状态、 Z：僵尸状态、s：包含子进程、l：多线程、+：前台显示
START：该进程的启动时间
TIME：该进程占用 CPU 的运算时间，注意不是系统时间
COMMAND：产生此进程的命令名


UID：用户 ID
PID：进程 ID
PPID：父进程 ID
C：CPU 用于计算执行优先级的因子。
数值越大，表明进程是 CPU 密集型运算，执行优先级会降低；数值越小，表明进程是 I/O 密集型运算，执行优先级会提高。

kill

kill [选项] 进程号 （功能描述：通过进程号杀死进程）
killall 进程名称 （功能描述：通过进程名称杀死进程，也支持通配符，在系统因负载过大而变得很慢时很有用）

top

实时监控系统进程状态，显示当前系统正在执行的进程的相关信息，包括进程ID、内存占用率、CPU占用率等。

前五行是当前系统情况整体的统计信息区。详见：每天一个linux命令（44）：top命令。
第一行信息为任务队列信息

PID — 进程id
USER — 进程所有者
PR — 进程优先级
NI — nice值。负值表示高优先级，正值表示低优先级
VIRT — 进程使用的虚拟内存总量，单位kb。VIRT=SWAP+RES
RES — 进程使用的、未被换出的物理内存大小，单位kb。RES=CODE+DATA
SHR — 共享内存大小，单位kb
S — 进程状态。D=不可中断的睡眠状态 R=运行 S=睡眠 T=跟踪/停止 Z=僵尸进程
%CPU — 上次更新到现在的CPU时间占用百分比
%MEM — 进程使用的物理内存百分比
TIME+ — 进程使用的CPU时间总计，单位1/100秒
COMMAND — 进程名称（命令名/命令行）

命令参数：
-p [进程PID]：指定监控进程 ID 来仅仅监控某个进程的状态。
-i 使 top 不显示任何闲置或者僵死进程。
-d 秒数 指定 top 命令每隔几秒更新。默认是 3 秒在 top 命令的交互模式当 中可以执行的命令
-o [key] 指定按照什么排序

操作说明：
P 以 CPU 使用率排序，默认就是此项
M 以内存的使用率排序
N 以 PID 排序
