badcase

Posted on 2022-03-14 In NLP
Symbols count in article: 1.5k Reading time ≈ 7 mins.

source: 心法利器[37] | bad case治疗术：认知篇

现状篇

  • 使用完整的评测集来进行检验，而不是单纯的一两个样本。
  • 需要一个具有统计意义上的数据集：
    • 数量上能支撑特定指标的计算足够稳定。
    • 抽样情况和实际使用的场景尽可能一致。
    • 质量可靠，足够支撑当前方案的数据集。
  • 指标
  • 结论

分析篇

准确率分错的case，看漏召回的case

粗看

笼统地大体上去看，错误的样本都有什么特点，带有那些特征，例如长度、句式等是否有什么特点，又或者是问句等等。
这个只要通篇看下来，就会有比较明显的特点了，一下子能总结出比较突出的问题。
快速看快速找到比较大的问题，优点就是快，但问题是你要能粗看的出来才行，所以缺点就是细节问题比较难发现，而且有些问题是比较深层次的，例如同样是问句，但是为什么问句就会出错

追溯法

对一个case，重现整个训练和预测过程的方法。给定一个bad case，准备好日志，分析预测的每个阶段的分析结果，查看是否符合预期，不符合预期的，大概率这个部分有bug、词典词汇等什么明显的问题了。

比如，很多问句会被误召回，这时候我们可以查查训练集中带有问句关键词的正负样本的比例是不是有问题，例如正样本远多于负样本，此时模型很可能就会认为只要带有疑问词的就是正类，我们需要加一些带疑问词的负样本进去。

解决法

例如短句预测不正确，而且一般都比较模糊，所以我们考虑的是过滤所有短句，然后再通过白名单的方式补召回，那我们就可以把query归类为短句。

问题标记

case是需要我们逐个分析的，后续我们需要对重点问题进行归纳总结，所以case分析往往是需要我们做一些关键的记录。

query 原因分析
关闭 短句
今天天气怎么样 疑问词，关键词拒绝
你为什么 疑问词，模糊

标记的时候注意词汇和问题的统一，方便我们后续统计各个问题的占比，值得注意的是，一个问题是可能有多个解决方法的，我们记得要标记上，统计分析的时候都是要关注的。

问题统计和归纳

有了上面的标记，我们能很轻易地看拿到各个问题的分布情况，甚至可以总结出来。

解决篇

样本的误导

对于一些数据内很浅层直白的信息，模型也会预测的非常直白，例如正类里面有大量含有“怎么”的样本，而负类里面没有，则模型很有可能会把所有带有“怎么”的句子都预测为正类。这个时候就需要负采样了。

阈值的确定和权衡

阈值本质是个准入条件，是和模型一起出现的标配，毕竟模型大部分时候只提供意见，要不要采纳还是要通过阈值（参考sigmoid，一般阀值为0.5）。
阀值变小，召回率变高，精确率下降，我们需要确定一个最佳阈值，对于搜索和对话场景，更多的是需要我们保准确，一般是0.85+，召回率另说。

softmax 呢？相当于双重条件？max+大于阀值

前后处理和预处理

不同的场景和数据的预处理方式会很不同，哪怕是同样的NLP问题，到了细分场景也会有很多不同的操作，共同的可能是对标点的处理，而不同的例如客服场景要把“你好”之类的删除，语音助手要把“小爱同学”、“小布”之类的称呼删除等，重点不在于做了什么，而是要把一些对预测有影响干扰项、不利于预测的因素给去掉，至于哪些是干扰项，请回去看bad case，这些错误样本会告诉你。

需要把一些肯定确定能快速预测出来的东西给过滤掉，从而提升性能也降低模型的负担，最简单的例如黑白名单，复杂的可以有一些针对业务的规则，例如超短句或者超长句直接拒绝等等，能用规则的尽量用规则，能不用模型的尽量不用模型，模型的不可控性众所周知，我们知道修复bad case的困难性的情况下，没必要太往依赖模型了。

后处理是指在模型预测以后的一些调整，最直接能想到的就是阈值过滤，但不仅是这些，有的时候需要结合模型的预测打分进行调整，例如关键词抽取领域，需要对预测的重要性分数进行加减分，如句首词和句尾词要提权，对于百科的一些问题，语义相似度的定义可能会和实际应用不同，于是我们也要进行一些过滤，例如“五一节的由来”和“六一节的由来”从语义相似度上来说确实很接近，但是我们不能然你给他们相似，因为“五一节”和“六一节”很不同，所以我们通过“实体约束”来过滤掉他们，当然背后就需要一批实体词典了。

解决问题的验证和迭代

解决后继续跑分，然后继续分析优化，往复。

转置卷积层（Transpose Convolution Layer），在一些文献中也称之为反卷积(De-convolution)和部分跨越卷积(Fractionally-strided Convolution) 。

从实现上来说，卷积本身是不可逆的，这里的反卷积仅仅是正常卷积 尺寸 上的逆向，内容不一定是逆向关系。所以除了提高分辨率，还可以进行风格迁移，理论上只要是 domain 2 domain 的任务，都可以进行。

一个卷积操作表示一个 多对一(many-to-one) 的映射关系。
而转置卷积层我们表示一个 一对多(one-to-many) 的映射关系。

从普通卷积讲起

假设我们有一个 4×44\times 44×4 的图片，同时也有一个 $3\times 3 $ 的卷积核 （不考虑通道数），步长为 1 ，则最后会得到一个 2×22 \times 22×2 的输出。

如果我们把每一步的卷积核对应原始图片位置进行 flatten ， 则会得到 1×161 \times 161×16 的矩阵：
比如在第一步骤中：

更一般地：
使用 n2×1n^2 \times 1n2×1 表示图像 ；
使用 $\text{time\ step}\ \times\ n^2 $ 的矩阵表示卷积核 ，n 为图片height / weight ；
最终输出结果为 time step×1\text{time\ step}\times 1time step×1

所以，如果可以改变 time step\text{time\ step}time step 那么，就可以改变输出的尺寸。
而同时如果反过来， output 变成了 input 2×22\times 22×2 ，「卷积相对矩阵」被转置了 16×416 \times 416×4 ，那么就可以得到原始图像 (16×4)⋅(2×2∗1)=(16×1)(16 \times 4)\cdot(2\times 2*1) = (16\times 1)(16×4)⋅(2×2∗1)=(16×1)

所以本质上，我们需要学习一个矩阵，大小为 outputsize2×input2output size ^ 2 \times input ^ 2outputsize2×input2 ，即转置矩阵。

在具体实现上，大多数情况下转置矩阵是用不到卷积矩阵的参数的，只是根据卷积矩阵推导出 一个shape,让网络能够上采样得到之前大小图片。

超详细的Yolo检测框预测分析
目标检测|YOLO原理与实现

原本目标分割采用最简单的分类方法，遍历边框（位置，大小），然后分类是否对应某一目标类别（其实还分两步，是前景还是背景，如果是前景的话属于哪一个类别）。

这里 yolo 将其转化为回归问题，让模型直接预测出向量 (x,y,w,h,c)(x,y,w,h,c)(x,y,w,h,c) 表示一个被检测出来的对象，c 为置信度 confidence。

现将图片切分成不同的 patch 区域，逐 patch 检测对象。为什么分 patch：因为conv操作是位置强相关的，就是原来的目标在哪里，你conv之后的feature map上还在哪里，所以图片划分为 n 个区域，结果也应该分布在 n 个区域上 。

如果每个区域让模型预测一个向量 (x,y,w,h,c)(x,y,w,h,c)(x,y,w,h,c) ，则有 n 个可能的结果。

问题：

  1. 同一个目标可能被切分到不同的区域中了，针对这样的情况：

    • 训练时的 label：我们就看这个物体的中心落在了哪个grid里面，哪个 grid 的 c 就为 1 。
    • 在 inference 时，也会出现相邻的两个 grid 预测出来同一个物体，但是两个被预测出来的窗口大小不一定相同，这时候需要一个指标来消歧： NMS(非极大值抑制)
      重合度的计算方法：交并比IoU = 两个框的交集面积 / 两个框的并集面积 。 于是可以使用这个指标来得到最终的检测答案

  2. 同一个区域如何检测到多个对象呢？
    刚才区域是 4×4 ，现在变成 40×40 ，或者更大，使区域更密集，就可以缓解多个目标的问题，但无法从根本上去解决。

  3. 多物体类别检测，怎么办？
    上面是单分类所以直接使用置信度表示概率，没有表示类别的量，而这里如果是两个类别，可以加上两个维度，即 (c,x,y,w,h,p1,p2)(c,x,y,w,h,p1,p2)(c,x,y,w,h,p1,p2) ，其中 (p1,p2)(p1,p2)(p1,p2) 表示两个类别的 one-hot 编码。

  4. 如果想在同一区域预测多个不同类别的 token，如何做呢？
    这个问题和 问题2 一样，还是无法很好的解决。

  5. 小目标难以拟合：小目标总是检测不佳，所以我们专门设计神经元去拟合小目标。
    小目标和大目标需要分开去预测，效果比较好，所以这里设置了两个 5 元组，被称为 bbox 或者 cell，分别负责两种目标，而标签还是 one-hot ，所以一共长度为 12 。

  6. 存在样本不均匀问题，能预测出来的 grid 还是占了少数的

YOLO v1

  • 7 * 7 grids：一共最多只能检测 49 个物体
  • 2 bbox / cell：对应大小两种目标 -> 2×[P(Object)+P(x)+P(y)+P(w)+P(h)]2 \times [ P(Object)+P(x)+P(y)+P(w)+P(h)]2×[P(Object)+P(x)+P(y)+P(w)+P(h)]

得到更多目标

yolo-v1 一次只能得到 49 个目标，即一个框内部只能有一个目标，同时每一个 patch 的两个框对应一个类别，可以是大目标也可以是小目标。

这就导致当目标比较多时，效果很差。

基于此，作者将原始的 7×77\times 77×7 进行拓展，变成了 13×1313 \times 1313×13 个 patch，每个区域有 5 个 anchor ，同时每个 anchor 对应一个 1个类别。
这样每个 patch 中就能预测出来 5 个潜在的类别，总体可以预测 13∗13∗5=84513*13*5 = 84513∗13∗5=845 个目标，进而最后出来的 logistic元素 个数为 13∗13∗5∗(5+20)=2112513 * 13 * 5 * (5 + 20) = 2112513∗13∗5∗(5+20)=21125 。

这样拆分后，每一个框和一组 xywh 绑定，同时需要多预测出来一组 class label，也就是一个 cell 最多预测 num of anchor 个 object 。即，object class num 不变，但是 num of object 变多了，等于 num of anchor 了。

分析

为什么需要 anchor ？ -> 注入先验知识
作者观察了很多图片的所有Ground Truth，发现：比如车，GT都是矮胖的长方形，再比如行人，GT都是瘦高的长方形，且宽高比具有相似性。
其实可以看成是并行地降低预测头预测的难度，因为相当于变相进行归一了预测的空间范围。
同时如果每一个 anchor 都可以预测一个 object，则也可以得到更多目标。

缺点：增加了推理最后一层的复杂度

怎么得到 anchor ？
对于任意数据集，将标签的 w,h 进行聚类，聚成几类呢？( Faster-RCNN 中的 Anchor 的宽高和大小是手动挑选的)。

作者进行了实验发现 anchor 数量越多，效果确实越好，但是复杂度也变差。

最终选择5类，衡量了 recall vs. complexity 。

YOLO v3

全网呕血整理：关于YOLO v3原理分析

v2 的检测头 grid cell 数量为 $ 13 * 13 $


YOLO v3检测头分叉了，分成了3部分：

  • 32倍下采样: 13∗13∗3∗(4+1+80)13*13*3*(4+1+80)13∗13∗3∗(4+1+80) -> 大目标
  • 16倍下采样: 26∗26∗3∗(4+1+80)26*26*3*(4+1+80)26∗26∗3∗(4+1+80) -> 中目标
  • 8倍下采样: 52∗52∗3∗(4+1+80)52*52*3*(4+1+80)52∗52∗3∗(4+1+80) -> 小目标

当然，再加采样倍数也可以，但是作者遵循recall vs. complexity的trade off 。

原本每个 grid cell 可以对应 num of anchor = 9 个先验框，这里让上面分叉的三个头分别对应 3个大的，3个中的，3个小的先验框，但是预测出来的 object 数量还是不变的。

每个分支预测3个框，每个框预测5元组+80个one-hot vector类别，所以一共size是：3∗(4+1+80)3*(4+1+80)3∗(4+1+80) 。

算上 grid cell ，每个分支的输出size为：

[13,13,3∗(4+1+80)][26,26,3∗(4+1+80)][52,52,3∗(4+1+80)][13,13,3*(4+1+80)] \\ [26,26,3*(4+1+80)] \\ [52,52,3*(4+1+80)] [13,13,3∗(4+1+80)][26,26,3∗(4+1+80)][52,52,3∗(4+1+80)]

最多预测 (13∗13+26∗26+52∗52)∗3( 13*13 + 26*26 + 52*52 ) * 3(13∗13+26∗26+52∗52)∗3 个 object bounding box (bbox) 。

选择先验框：还是聚类

模型

DBL: Darknet CNN + BatchNormalization（BN）+ LeakyReLU

  1. 经过六个DBL结构和最后一层卷积层得到第一个特征图谱 (13*13)，在这个特征图谱上做第一次预测 。
  2. 上面的预测前结果经过一个 DBL 和 一次（2,2）上采样，将上采样特征与第2个Res8结构输出的卷积特征张量连接，经过六个DBL结构和最后一层卷积层得到第二个特征图谱 。
  3. 和上面的一样，一次（2,2）上采样，将上采样特征与第1个Res8结构输出的卷积特征张量连接，经过六个DBL结构和最后一层卷积层得到第三个特征图谱 。

类型预测

Yolo v2 每一个 anchor 最多只能预测出来一个 object，当对应的 certification 大于某一个阀值时，就在 20 个 logistic 上 softmax 取最大，得到 object class。

但是在一些复杂场景下，一个目标可能属于多个类（有重叠的类别标签），因此 Yolo v3 用多个独立的 Logistic二分类器 替代 Softmax 多标签分类器，只要最后 sigmoid > 0.5，就都预测出来。

所以具体预测输出时：

  1. 首先将 certification score 低的的框删除
  2. 当多个框重合度高且都检测同一个物体时只选择一个框（NMS）。

训练

build target：

  1. 确定 object center 落在了哪一个 grid cell 上，然后计算这个 cell 的每个 anchor 与 ground truth bbox 的 IOU 值，计算该值时不考虑中心点坐标，只考虑形状，即先将anchor与ground truth的中心点都移动到同一位置（原点），然后计算出对应的IOU值。 IOU 最大的 anchor 最终被选定为 labeled anchor 。

anchor 类型： 正例（positive）、负例（negative）、忽略样例（ignore）

  • 正例（positive）(与GT的IOU最大的框)：对于一个 object，使用其 GT bbox 用上面的方法计算 IOU 得到一个对应的正例 anchor。 注意，一个预测框, 只能分配给一个ground truth object。 那么下一个ground truth, 就在余下的检测框中, 寻找IOU最大的检测框作为正例。
  • 忽略样例（ignore）(与GT的IOU>0.5 但不是最大的框)：除了 positive anchor，还存在其他 anchor 与当前样本 bbox IOU 大于阀值，我们将这部分 anchor 设定为忽略样本，不计算损失。
  • 负例（negative）(与GT的IOU<0.5 的框)：与全部的 gt bbox 的 iou 都小于阀值的，设置为 negative anchor ，仅仅提供 certification loss 。

CIoU-loss

以前的版本在计算 xywh 的损失时，都是用 MSE loss，而后面人们开始使用 IoU loss：

LIoU=1−∣B∩Bgt∣∣B∪Bgt∣L_{I o U}=1-\frac{\left|B \cap B_{g t}\right|}{\left|B \cup B_{g t}\right|} LIoU​=1−∣B∪Bgt​∣∣B∩Bgt​∣​

使用「检测框和真实框的之间的重合面积」反应监测效果。

问题：

  1. 只考虑了重合面积，不能反映中心点的匹配情况 (如下图)。
  2. 当GT和bounding box不挨着时，没有梯度回传 。

GIoU

于是作者做出了改进，提出 GIoU：

LGIoU=1−IoU+∣C−B∪Bgt∣∣C∣L_{G I o U}=1-I o U+\frac{\left|C-B \cup B_{g t}\right|}{|C|} LGIoU​=1−IoU+∣C∣∣C−B∪Bgt​∣​

CCC 为同时包含了预测框和真实框的最小框的面积。
GIoU Loss 增加了一项，中点偏移的越多，后一项越大，所以可以解决上面IoU Loss对距离不敏感的问题。 但是GIoU Loss存在训练过程中发散等问题。

DIoU

LDIoU=1−IoU+ρ2(b,bgt)c2L_{D I o U}=1-I o U+\frac{\rho^2\left(b, b^{g t}\right)}{c^2} LDIoU​=1−IoU+c2ρ2(b,bgt)​

ccc 代表的是能够同时包含预测框和真实框的最小闭包区域的对角线距离。

这里直接使用中心点的距离来建模，可以直接最小化两个目标框的距离，因此比GIoU loss收敛快得多 。


同时对于上面三张图的情况， IoU 和 GIoU loss 三者都相同，但是 DIoU 从左到右，依次变好。

所以好处为：收敛快 + 缓解了Bounding box全包含GT问题。

但Yolov5中将此功能嵌入到代码中，每次训练时，自适应的计算不同训练集中的最佳锚框值。当然，如果觉得计算的锚框效果不是很好，也可以在代码中将自动计算锚框功能关闭

自适应图片缩放:
在常用的目标检测算法中，不同的图片长宽都不相同，因此常用的方式是将原始图片统一缩放到一个标准尺寸，再送入检测网络中。

但是在缩放时，如果两张图片的长宽比不一致，并且不拉伸，会导致一边有黑边，比如 800∗600→416∗416800*600 \rightarrow 416*416800∗600→416∗416 ，就会导致上下侧有黑边。

如果黑边过多，则在推理时这部分信息其实可以直接忽略掉的，即存在信息冗余现象。

于是作者让模型在推理时候自适应的添加最少的黑边。

训练时没有采用缩减黑边的方式，还是采用传统填充的方式，即缩放到416*416大小。只是在测试，使用模型推理时，才采用缩减黑边的方式，提高目标检测，推理的速度。

小目标分割检测

目标检测发展很快，但对于小目标的检测还是有一定的瓶颈，特别是大分辨率图像小目标检测。比如 7920∗21607920*21607920∗2160 ，甚至 16000∗1600016000*1600016000∗16000 的图像。

最后一层变成了 19∗19,38∗38,76∗7619*19, 38*38, 76*7619∗19,38∗38,76∗76 ，其中最细粒度的 76∗7676*7676∗76 负责检测小目标 ，对应到 7920∗21607920*21607920∗2160 上，那么最终每一个像素表示了原始的 (7920//76)∗(2160//76)=104∗28(7920//76) * (2160//76) = 104 * 28(7920//76)∗(2160//76)=104∗28 ，然后去进行回归。

那么也带来一个困难，即如果原始图像中目标的宽或高小于 104∗28104 * 28104∗28 像素，网络很难学习到目标的特征信息。

高分辨率 , 显卡爆炸

很多图像分辨率很大，如果简单的进行下采样，下采样的倍数太大，容易丢失数据信息。

但是倍数太小，网络前向传播需要在内存中保存大量的特征图，极大耗尽GPU资源,很容易发生显存爆炸，无法正常的训练及推理。

-> 2018 年 YOLT 算法，大分辨率图片先进行分割，变成一张张小图，再进行检测，最后将监测目标组合起来 。

针对 Overlap 问题
为了避免两张小图之间，一些目标正好被分割截断，所以两个小图之间设置overlap重叠区域，比如分割的小图是 960*960 像素大小，则overlap可以设置为 960*20%=192 像素。

个人理解：虽然模型可以拟合到边边角角，但是仍然需要特征，如果没有重合的话，单个子图就看不到调整了，即目标就是被切开了。

每个小图检测完成后，再将所有的框放到大图上（ 按照在大图上裁剪的位置，直接回归到大图 ）。
对大图整体做一次nms操作，将重叠区域的很多重复框去除。

这样操作，可以将很多小目标检出，比如16000*16000像素的遥感图像。

优点：

（1）准确性

分割后的小图，再输入目标检测网络中，对于最小目标像素的下限会大大降低。

比如分割成608608大小，送入输入图像大小608608的网络中，按照上面的计算方式，原始图片上，长宽大于8个像素的小目标都可以学习到特征。

（2）检测方式

在大分辨率图像，比如遥感图像，或者无人机图像，如果无需考虑实时性的检测，且对小目标检测也有需求的项目，可以尝试此种方式。

缺点：

（1）增加计算量

比如原本7680*2160的图像，如果使用直接大图检测的方式，一次即可检测完。

但采用分割的方式，切分成N张608*608大小的图像，再进行N次检测，会大大增加检测时间。

借鉴Yolov5的四种网络方式，我们可以采用尽量轻的网络，比如Yolov5s网络结构或者更轻的网络。

当然Yolov4和Yolov5的网络各有优势，我们也可以借鉴Yolov5的设计方式，对Yolov4进行轻量化改造，或者进行剪枝。
