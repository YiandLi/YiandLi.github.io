介绍

GlobalPointer

  • Global即利用全局归一化的思路来进行NER任务，无差别地识别嵌套实体和非嵌套实体。

常规的Pointer Network的设计在做实体识别或者阅读理解时，一般是用两个模块分别识别实体的首和尾，这会带来训练和预测时的不一致。而GlobalPointer就是针对这个不一致而设计的，它将首尾视为一个整体去进行判别，所以它更有“全局观”（更Global）。

  • Pointer即与CRF不同，使用指针的形式（start pointer & end pointer）预测实体。

NER中未标注实体问题分析和解决

Posted on 2022-02-04 In NLP , Named entity recognition
Symbols count in article: 2.3k Reading time ≈ 11 mins.

论文：EMPIRICAL ANALYSIS OF UNLABELED ENTITY PROBLEM IN NAMED ENTITY RECOGNITION

Introduction

NER任务中存在未标注实体问题（unlabeled entity problem），即部分命名实体少标，漏标。
作者通过随机去除一些标注实体数据，用 性能下降的分数（degradation rates） 表示缺少标注带来的影响。

根据实验，作者提出了两点性能下降（performance degradation）的原因：

  1. 标注实体数量的减少（ the reduction of annotated entities ）导致训练不充分
  2. 未标记实体被采样为负样本（ treating unlabeled entities as negative instances ）

注意区分未标注实体和负样本：未标注实体通常是人工标注数据集中漏标的实体数据；负样本则为非实体数据，比如BIO中的O。

前者导致的性能下降可以通过使用大规模预训练模型作为sentence encoder弥补；而后者会严重误导模型，从而导致性能大大下降。

针对第二个问题，作者提出了一个通用的解决方式。简单讲即负采样（negative sampling），从而将训练中未标注实体被采样到的概率保持在很低的数值。
这种做法可以几乎消除未标注实体带来的误导，提升模型鲁棒性，并且提升模型性能。

Training Via Negative Sampling

从上文实验可知，最影响性能的原因是未标注实体作为负样本，从而误导模型。
本文使用负采样的方式解决这个问题，即随机采样部分未标注span，而不是像传统mlp输出bio标注一样将所有的位置都作为样本。

Step 1

给定标注样本集合 y\mathbf yy ，首先得到所有的候选负样本：

{(i,j,O)∣(i,j,l)∉y,1≤i≤j≤n,l∈L}\{(i, j, \mathrm{O}) \mid(i, j, l) \notin \mathbf{y}, 1 \leq i \leq j \leq n, l \in \mathcal{L}\} {(i,j,O)∣(i,j,l)∈/y,1≤i≤j≤n,l∈L}

L\mathcal{L}L 为标签空间
O\mathrm{O}O 为负样本的标签，即bio标注方式下非实体的标签。

Step 2

然后随机从中采样，得到大小为 ⌈λ∗n⌉,0<λ<1\lceil\lambda * n\rceil, 0<\lambda<1⌈λ∗n⌉,0<λ<1 的负样本集合 y^\mathbf{\widehat y}y​ ， ⌈⌉\lceil \rceil⌈⌉ 为ceiling function，向上取整。

Step 3

(∑(i,j,l)∈y−log⁡(oi,j[l]))+(∑(i′,j′,l′)∈y^−log⁡(oi′,j′[l′]))\left(\sum_{(i, j, l) \in \mathbf{y}}-\log \left(\mathbf{o}_{i, j}[l]\right)\right)+\left(\sum_{\left(i^{\prime}, j^{\prime}, l^{\prime}\right) \in \widehat{\mathbf{y}}}-\log \left(\mathbf{o}_{i^{\prime}, j^{\prime}}\left[l^{\prime}\right]\right)\right) ⎝⎛​(i,j,l)∈y∑​−log(oi,j​[l])⎠⎞​+⎝⎛​(i′,j′,l′)∈y​∑​−log(oi′,j′​[l′])⎠⎞​

同时对于一个长度为 nnn 的句子，单个未标注实体没有被采样到作为负样本的概率为：

不考虑边界冲突问题，只 −i-i−i ，因为训练时即使存在选择的边界冲突，被选择的两个冲突span中，最少有一个也会被当成负样本。

MRC for NER

Posted on 2022-01-29 In NLP , Named entity recognition
Symbols count in article: 2.1k Reading time ≈ 11 mins.

论文：A Unified MRC Framework for Named Entity Recognition
代码：ShannonAI/mrc-for-flat-nested-ner

Abstract

NER任务分为 flat NER 和 nested NER 。
序列标注模型只能针对一个token分配一个label，这不适用于嵌套模型，因为嵌套模型中的一个token可能被分配多个label。

作者提出一个统一处理nested NER与flat NER的框架，
即将NER任务视为一个 阅读理解任务（a machine reading comprehension (MRC) task） 。

在这个任务中，如果想要提取不同类别的实体，只需要分别回答不同的问题即可，这就解决了嵌套NER中实体覆盖的问题。
此外，查询语句编码了丰富的先验知识，加速了实体提取的速度，在嵌套NER上以及非嵌套NER上都产生了更好的性能。

nested ner

序列标注框架

  1. token的多标签分类问题：学习难度大，label依赖缺失
  2. CRF多个标签层，即每层只有一个label，然后合并所有标签层：指数级增加标签
  3. token-level的多标签分类，即将softmax改为多个sigmoid：标签之间的依赖关系会有所缺失

指针标注框架

  1. C个指针网络（bert-span
  2. MRC + 指针 + span概率矩阵「单类别标注」

Span标注框架/多头标注框架

需要注意数据稀疏问题
构建span概率矩阵有多种方式：

  1. biaffine：Span概率矩阵的「多类别标注」。
  2. GlobalPointer：Span概率矩阵的「单类别标注」，但是target为整个Span概率矩阵，是一个「多标签分类问题」。
  3. EMPIRICAL ANALYSIS OF UNLABELED ENTITY PROBLEM：Span概率矩阵的「多类别标注」。

问题

真实的工业界场景中，通常面临

  1. 标注成本昂贵
  2. 泛化迁移能力不足
  3. 可解释性不强
  4. 计算资源受限

在医疗领域中

  1. 泛化能力差
    不同医院、不同疾病、不同科室的文本描述形式不一致，而标注成本又很昂贵。导致模型的泛化迁移能力很差。
    所以当前的NER技术在医疗领域并不适合做成泛化的工具，应该具体问题具体模型。
  2. 可解释性
    医疗领域的严肃性要求模型的强可解释性。
    处理过程应该随着处理对象的「层次」和「深度」而逐步叠加模块，下级模块使用上级结果，方便进行迭代优化、并具备可解释性。
  3. 多特征
    除了模型外，医疗领域相关的「实体词典」和「特征挖掘」也对NER性能起着关键作用。
  4. 计算资源受限
    医院数据不可出院，需要在院内部署NER系统。而通常医院内部的GPU计算资源又不是很充足，所以机器学习模型需要又轻又快，同时要更充分的利用显存。

关于Bert

问题

不受计算资源限制、通用领域、小样本的场景下，BERT表现会更好。但是对于工业场景中的绝大部分NLP问题（特别是垂直领域），都没有必要堆资源。

  • 直接BERT+CRF的性能不一定会好，推断速度也非常堪忧。
  • BERT和CRF层的学习率也不要设成一样，让CRF层学习率要更大一些（一般是BERT的5～10倍），要让CRF层快速学习。
  • 不要试图对BERT进⾏蒸馏压缩，很可能吃⼒不讨好。
  • NER任务是⼀个重底层的任务，上层模型/网络结构再深、性能提升往往也是有限的。
  • 不同的解码方式（CRF/指针网络/Biaffine）之间的差异其实也是有限的，不要过分拘泥于解码⽅式。
  • QA阅读理解的方式进行NER任务，效果也许会提升，但计算复杂度也会大大提升。
  • 设计NER任务时，尽量不要引入嵌套实体，不好做，这往往是一个长尾问题。
  • Transformer不合适做NER。

使用

  • 在低耗时场景中，BERT可以作为一个“对标竞品”，我们可以采取轻量化的多种策略组合去逼近甚至超越BERT的性能。
  • 在垂直领域应用BERT时，我们首先确认领域内的语料与BERT原始的预训练语料之间是否存在gap，如果这个gap越大，那么我们就不要停止预训练：继续进行领域预训练、任务预训练。
  • 在小样本条件下，利用BERT可以更好帮助我们解决低资源问题。
    低资源NLP问题的解决方法通常都针对分类任务，往往在NER中提升并不明显。NER本质是基于token的分类任务，其对噪声极其敏感的。如果盲目应用弱监督方法去解决低资源NER问题，可能会导致全局性的性能下降，甚至还不如直接基于词典的NER。
    • 使用BERT进行数据蒸馏（半监督学习+置信度选择），同时利用实体词典辅助标注。
    • 实体词典+BERT相结合，进行半监督自训练
  • 在竞赛任务中，可以选取不同的预训练语⾔模型在底层进行特征拼接。语⾔模型的差异越⼤，效果越好。
    如果需要对语言模型finetune，需要设置不同的学习率。

如何快速有效地提升NER性能

规则 + 领域词典

垂直领域：不断积累、不断完善的实体词典
通⽤领域：多种分词工具和多种句法短语⼯具进行融合来提取候选实体，并结合词典进行NER。

在模型层面提升NER性能

在embedding层下功夫，引入丰富的特征：比如char、bigram、词典特征、词性特征、elmo等等。
底层的特征越丰富、差异化越大越好（构造不同视角下的特征）。

构建引入词汇信息（词向量）的NER

中文NER通常是基于字符进行标注的，这是由于基于词汇标注存在分词误差问题。
但词汇边界对于实体边界很有效，一种行之有效的方法就是信息无损的、引入词汇信息的NER方法，比如FLAT和Simple-Lexicon。

解决NER实体span过长的问题

如果NER任务中某一类实体span比较长（⽐如医疗NER中的⼿术名称是很长的），直接采取CRF解码可能会导致很多连续的实体span断裂。

  • 加入规则进行修正
  • 指针网络+CRF构建多任务学习（指针网络会更容易捕捉较长的span，不过指针网络的收敛是较慢的，可以试着调节学习率）

Abstract and Introduction

传统的ner任务大多为Flat NER，没有考虑到Nested NER（嵌套实体：一个token可能分别拥有两个不同的类型）的情况。
比如北京大学中的北同时属于「B-Location」，也属于「B-Organization」；而京也拥有「I-Location」与「I-Organization」两个标签。
本文通过使用biaffine模型解决Nested NER。并且这种方法在「flat ner」和「nested ner」任务都有着不错的表现。

在句法分析（dependency parsing）任务中，biaffine模型对每个token预测出一个「head」，然后对「head-child pairs」指定关系。
在NER任务中，就是把实体抽取任务看成为识别「start」与「end」索引的问题，同时对这个「start」与「end」形成的span赋予实体类型。
具体地：使用多层Bi—LSTM和biaffine得到每一个span的分数，然后rank这些candidate span的分数，返回符合任务要求的top-ranked spans。

在ner中引入词汇信息

Posted on 2022-01-25 In NLP , Named entity recognition
Symbols count in article: 298 Reading time ≈ 1 mins.

为什么中文NER任务需要词汇增强？

NER任务中，基于字符的方法通常好于基于单词的方法。
但基于字符的NER没有利用词汇信息，而词汇边界对于实体边界通常起着至关重要的作用。

如何在基于字符的NER系统中引入词汇信息，是近年来NER的一个研究重点。
这种引入词汇的方法称之为「词汇增强」，以表达引入词汇信息可以增强NER性能。

词汇增强的方法

基于词汇增强的中文NER主要分为2条主线[9]：

  1. Dynamic Architecture：设计一个动态框架，能够兼容词汇输入；
    比如 Lattice-LSTM，FLAT。
    FLAT不去设计或改变原生编码结构，设计巧妙的位置向量就融合了词汇信息
  2. Adaptive Embedding：基于词汇信息，构建自适应Embedding；
    比如 Simple-Lexicon，该方法n对于词汇信息的引入更加简单有效，采取静态加权的方法可以提前离线计算。

使用一种无参数的（但是有超参数）后处理方法 (a non-parametric post-processing) 应用于分类问题。
具体地，在验证集上，使用那些高置信度样本的预测概率调整低置信度样本的预测概率。

步骤

  1. 针对样本预测概率输出，用一个指标来划分 （简单样本）高置信度样本 和 （难样本）低置信度样本 。
    use ambiguity level to identify high-confidence validation examples and challenging examples
  2. 迭代进行 交替归一化 alternating normalization
    每一次迭代过程中先进行 列归一化/行间归一化，然后进行 行归一化/行内归一化。
    normalizing across rows (examples) and columns (target labels)
  3. 输出：上一步迭代过程中，高置信样本和低置信样本的概率分布都会改变。我们只保留最终的低置信度样本的概率分布，作为输出；高置信样本的概率分布还是迭代前的概率分布。

Identify Hard Examples

一共有 mmm 个类别
最简单的划分指标肯定是信息熵:

H(b0)=−∑ib0,ilog⁡m(b0,i)\mathcal{H}\left(\mathbf{b}_{0}\right)=-\sum_{i} \mathbf{b}_{0, i} \log _{m}\left(\mathbf{b}_{0, i}\right) H(b0​)=−i∑​b0,i​logm​(b0,i​)

概率分布的信息熵越大，置信度越低。但是有时，信息熵并不符合我们分类的目标，比如两个样本:

b01=[0.5  0.25  0.25]b02=[0.5  0.5  0]H(b01)  >  H(b02)\begin{aligned} b^1_0 &= [0.5\; 0.25\; 0.25]\\ b^2_0 &= [0.5\; 0.5\; 0]\\ \mathcal{H}\left(b^1_0\right)\; &\gt\; \mathcal{H}\left(b^2_0\right) \end{aligned} b01​b02​H(b01​)​=[0.50.250.25]=[0.50.50]>H(b02​)​

可以看到 b01b^1_0b01​ 的信息熵大于 b02b^2_0b02​ 的信息熵，所以按理说 b01b^1_0b01​ 的置信度会小于 b02b^2_0b02​ ，但是对于分类问题，明显是 b01b^1_0b01​ 更容易做出预测。

实验

  1. 低置信样本的不确定性越高/信息熵越大，并且用于纠正的高置信样本的确定性越高/信息熵越低，最终的提升效果越好。
  2. 对高置信样本 A0A_0A0​ 具有良好的鲁棒性。
  3. 分类任务的类别数量越多，提升越明显。
  4. 超参数 (τ,α,d)(τ,\alpha,d)(τ,α,d) ，一般 d=1d=1d=1 效果最好，实验中 α>10\alpha>10α>10 ， τ=0.75τ=0.75τ=0.75 效果最好（但是论文的归一结果明显小于1，就不一样了呢）。
    基本 alphaalphaalpha 越大，效果越好。并且随着 α\alphaα 的增加，τττ 和 ddd 的影响越来越小。

苏神博客中指出 α\alphaα 纯粹是为了调参而引入的（参数多了，总能把效果调到有所提升），没有太多实际意义。同时， α=1\alpha=1α=1 时，基本已经是最优选择了。
