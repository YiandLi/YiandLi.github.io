如何训练

  • 传统Gradient descent从每一个样本的梯度下降中得到 δw\delta wδw 并且累加所有样本的梯度，最后一起对w做一次更新。但是这样拟合速度很慢，并且不一定会找到全剧最优解。
    一个随机的版本，为Incremental gradient descent：在每个样本梯度下降得到梯度 δw\delta wδw 后，直接对权重进行更新，也就是多少个样本，就更新多少次。

  • Revision

    • on-line（incremental） learning: revision by example
      每个样本都会直接更新梯度
    • batch learning: revision by epoch
      将每个样本的梯度累积起来，每个epoch结束一次行更新权重

  • Presentation order：random order 优于 sequential order

  • Random Initialization：

    • 权重：[-0.5, 0.5] 之间的随机数
    • 输入：归一化到 (0,1)

  • 两层网络结构已经足够拟合任意的表示，所以增加一层不会额外增加模型的表示能力。

  • 隐藏层节点数量通常设置为

n=input_neurons+output_neurons2n=\dfrac{input\_neurons+output\_neurons}2 n=2input_neurons+output_neurons​

  • 停止条件
    • 损失下降到阀值以下
    • 最大epochs
    • early stop，使用开发集检验
  • 学习率：
    • 太小导致拟合缓慢
    • 太大导致在最优解附近来回震动 oscillation
    • 通常设置在 [0.05, 0.75] 之间

过拟合

  • 原因：参数数量>训练样本数量
  • 措施：
    • 早停
    • 网络减枝
    • 正则

动量 Momentum

动量梯度下降法 就是在梯度下降法中，迭代点的更新方向由当前的负梯度方向和上一次的迭代更新方向加权组合形成。

△w(t)=−η∂E∂w(t)+α△w(t−1)\triangle w(t) = -\eta\frac{\partial E}{\partial w(t)}+\alpha\triangle w(t-1) △w(t)=−η∂w(t)∂E​+α△w(t−1)

α\alphaα 是一个动量系数，通常令 η=1−α\eta=1-\alphaη=1−α ， α∈(0.6,0.9)\alpha\in (0.6,0.9)α∈(0.6,0.9) 。

  • 类似于物理中，一个小球在滚动时，运动方向受当前受力方向和原速度方向共同影响。
  • 目的：
    • 一方面是加快下降，因为在下降的方向上不断累积动量，使得下降的速度变快了。
    • 另一方面，这样的方式可以使迭代跳出 驻点，鞍点和部分较浅的局部极小值点 ，使更有可能找到全局极小值。
    • smooth the weight changes, cancel side-to-side oscillations across the error valley：

      每个参数在各方向上的移动幅度不仅取决于当前的梯度，还取决于过去各个梯度在各个方向上是否一致，如果一个梯度一直沿着当前方向进行更新（水平方向），那么每次更新的幅度就越来越大，如果一个梯度不在同一个方向上变化（竖直方向），那么其更新幅度就会被衰减，这样我们就可以使用一个较大的学习率，使得收敛更快，同时梯度比较大的方向就会因为动量的关系每次更新的幅度减少。

讨论

  • 适用于线性拟合问题和分类问题。
  • 理论上，可以模拟任何的线性映射和非线形映射。
  • 但是容易陷入局部最优
  • 效果对初始权重很敏感
  • 对学习率很敏感
    • 太小：拟合很慢
    • 太大：不稳定或者模型效果很差
  • 对网络层数和神经元数量很敏感
    • 太少：欠拟合
    • 太多：过拟合，学习速度慢

深度学习框架的优势

  1. 简化计算图的搭建
  • 计算图是一种描述函数的语言，其中节点代表输入值，边代表函数操作。
  • 计算图可以用于描述表达式，可以更加精确描述网络中的参数传播过程。

  1. 简化偏导数的计算
    计算图以模块化的方式完整表达了函数体的内部逻辑关系，所以对复杂函数求导的过程就变成了简单地从输入到输出进行遍历的过程。绝大多数深度学习框架都选择基于计算图的声明式求解。

  2. 高效运行
    移植性：可以将代码部署到GPU或CPU上

  • 中间层的激活函数： 数据的分布绝大多数是非线性的，而一般神经网络的计算是线性的，引入激活函数，是在神经网络中引入非线性，强化网络的学习能力。
  • 输出层激活函数：

    二分类中，需要使用sigmoid激活函数，压到[0,1]之间得到logistic值，后加ce求损失。

不同的激活函数

  • Sigmoid激活函数：σ(x)=11+e−x\sigma(x) = \frac{1}{1+e^{-x}}σ(x)=1+e−x1​
    • 取值范围：(0, 1)
    • 缺点：计算量大，反向传播求误差梯度时，求导涉及除法，
    • Sigmoid导数取值范围是[0, 0.25]，由于神经网络反向传播时的“链式反应”，很容易就会出现梯度消失的情况。例如对于一个10层的网络， 第10层的误差相对第一层卷积的参数的梯度将是一个非常小的值，这就是所谓的 “梯度消失”。
    • Sigmoid的输出不是0均值（即zero-centered）；这会导致后一层的神经元将得到上一层输出的非0均值的信号作为输入，随着网络的加深，会改变数据的原始分布。
  • tanh激活函数：tanh(x)=ex−e−xex+e−xtanh(x) = \frac{e^{x}-e^{-x}}{e^{x}+e^{-x}}tanh(x)=ex+e−xex−e−x​
    • tanh的输出范围时(-1, 1)，解决了Sigmoid函数的不是zero-centered输出问题
    • tanh导数范围在(0, 1)之间，相比sigmoid的(0, 0.25)，梯度消失（gradient vanishing）问题会得到缓解，但仍然还会存在。
  • ReLu激活函数：relu=max(0,x)relu=max(0, x)relu=max(0,x)

    https://zhuanlan.zhihu.com/p/172254089
    • ReLU的有效导数是常数1，解决了深层网络中出现的梯度消失问题，也就使得深层网络可训练。

性质

  • f(x)≈xf(x)≈xf(x)≈x ：
    当激活函数满足这个性质的时候，如果参数的初始化是random的很小的值，那么神经网络的训练将会很高效；如果不满足这个性质，那么就需要很用心的去设置初始值
  • 输出值的范围：
    当激活函数输出值是有限的时候，基于梯度的优化方法会更加稳定，因为特征的表示，当激活函数的输出是无限的时候，模型训练会更加的高效，不过在这种情况很小，一般需要一个更小的学习率

batch-size 和 epoch

  • batch_size：每批数据量的大小，一次训练batchsize个样本，计算它们的平均损失函数值，来更新参数。
  • epoch：1个epoch指用训练集中的全部样本训练一次。

batch_size的大小决定了是否可以表示全体样本的特征，梯度越准，手链速度越快。
batch_size过大：计算量太大，内存占用太多，更新速度过慢。并且缺乏随机性，容易使梯度始终向单一方向下降，陷入局部最优。
batch_size过小：每次训练的速度快，但是因为样本随机性较高，收敛速度很慢，产生局部震荡，有限迭代次数内很可能无法收敛。

梯度爆炸 / 消失

原因：

  1. 反向传播时，偏导数逐渐相乘，导致值很小/很大 。
  2. 初始化参数过小 / 大
  3. 激活层的输入过大/过小，导致梯度很小。比如 Sigmoid 导数范围 [0, 0.25] ，Relu 会出现 0 ，都容易导致梯度消失
  4. Softmax 层输出分数逼近 1，导致梯度消失

解决方法：

  1. 残差连接
  2. Norm
  3. 梯度裁剪 ，防止梯度爆炸
  4. softmax 除以 sqrt(d)

Multi-Task Learning 如何设置不同 loss 的权重 ？

  1. Multi-Task Learning Using Uncertainty to Weigh Losses for Scene Geometry and Semantics ： 每个loss除以不确定度/样本方差，如果不确定度大，大体上相当于自动把loss的权重变小。
  2. 为每个任务分配一个可学习或固定的权重wi
  3. 对每个损失进行归一化，比如除以其均值、标准差或其他统计量，使得所有任务的损失大致处于同一数量级。但这并不能解决任务间固有的复杂性差异，即某些任务可能天生就比其他任务更容易或更难优化。
  4. 基于梯度范数调整：这种情况下提到的“除以根号dk”，这里的dk通常代表任务k的某种梯度规模指标，如梯度的L2范数（Euclidean norm）。
    • 缓解梯度爆炸/消失：通过除以根号dk，可以对较大梯度进行抑制，对较小梯度进行放大。
    • 保持梯度方向不变：这样的操作仅改变了梯度的幅度而不改变其方向。
    • 促进任务间的相对平衡：理想情况下，经过这样的调整后，所有任务的梯度会更加接近。

优化算法的常用 tricks
（1）首先，各大算法孰优孰劣并无定论。如果是刚入门，优先考虑SGD+Nesterov Momentum或者Adam。
（2）选择你熟悉的算法——这样你可以更加熟练地利用你的经验进行调参。
（3）充分了解你的数据——如果模型是非常稀疏的，那么优先考虑自适应学习率的算法。
（4）根据你的需求来选择——在模型设计实验过程中，要快速验证新模型的效果，可以先用Adam进行快速实验优化；在模型上线或者结果发布前，可以用精调的SGD进行模型的极致优化。
（5）先用小数据集进行实验。有论文研究指出，随机梯度下降算法的收敛速度和数据集的大小的关系不大。因此可以先用一个具有代表性的小数据集进行实验，测试一下最好的优化算法，并通过参数搜索来寻找最优的训练参数。
（6）考虑不同算法的组合。先用Adam进行快速下降，而后再换到SGD进行充分的调优。
（7）数据集一定要充分的打散（shuffle）。这样在使用自适应学习率算法的时候，可以避免某些特征集中出现，而导致的有时学习过度、有时学习不足，使得下降方向出现偏差的问题。
（8）训练过程中持续监控训练数据和验证数据上的目标函数值以及精度或者AUC等指标的变化情况。对训练数据的监控是要保证模型进行了充分的训练——下降方向正确，且学习率足够高；对验证数据的监控是为了避免出现过拟合。
（9）制定一个合适的学习率衰减策略。可以使用定期衰减策略，比如每过多少个epoch就衰减一次；或者利用精度或者AUC等性能指标来监控，当测试集上的指标不变或者下跌时，就降低学习率。

为什么 AdamW > SGD
① 自适应学习率：AdamW 结合了 Adam 优化器和权重衰减（Weight Decay）的概念。Adam 优化器具有自适应学习率的特性，可以根据参数的不同情况自动调整每个参数的学习率。
② 动量优化：AdamW 还使用了动量优化的方法，可以加速模型的收敛过程。动量优化有助于处理非凸优化问题，并帮助模型跳出局部最小值，更好地搜索全局最优解。
③ 权重衰减：AdamW 在 Adam 的基础上增加了权重衰减（L2 正则化）的项，用于控制模型的复杂度并防止过拟合。
④ 平稳性：相对于 SGD，AdamW 通常具有更平稳的训练过程。AdamW 通过计算梯度的一阶和二阶矩估计来调整学习率，可以更精确地更新参数，从而更稳定地优化模型。
① 适应文本数据特点：NLP任务中的文本数据通常具有大量的稀疏特征和高维度。AdamW 可以根据参数的梯度情况自适应地调整学习率，对于稀疏特征更加敏感，从而提高训练的效率和收敛速度。
② 处理复杂模型结构：在 NLP 中，常见的模型，如循环神经网络（RNN）、长短期记忆网络（LSTM）和变压器（Transformer），具有复杂的结构和大量的参数。AdamW 的动量更新策略能够帮助模型更好地处理这种复杂性，加速收敛过程并减少震荡。
③ 控制模型复杂度：NLP 模型往往面临过拟合问题，即模型在训练数据上表现良好，但在测试数据上表现较差。AdamW 的权重衰减机制有助于控制模型的复杂度，通过给予较大的权重衰减项（L2 正则化），在优化过程中防止模型过度拟合。
一般实测，小 lr + 大 wd，效果比较好，并且更快。实测AdamW lr0.001 + wd0.009 在我这个小数据集里第30个epoch就超过 SGD lr0.1+wd0.0005 或者 AdamW lr0.00035+wd0.0005。
④平稳性和稳定性：AdamW 结合了 Adam 优化器的特点，通过计算梯度的一阶和二阶矩估计来自适应地调整学习率。这种特性使得优化过程更加平稳，减少了极值点附近的震荡，有助于模型更好地捕捉数据中的模式和结构。

AdamW 缺点：

  • 对前期出现的特征过拟合，后期才出现的特征很难纠正前期的拟合效果。
  • 最终收敛的结果并没有SGD好，主要是后期Adam的学习率太低，影响了有效的收敛。他们试着对Adam的学习率的下界进行控制，发现效果好了很多。（ ImproVing Generalization Performance by Switching from Adam to SGD ），因此他们提出了一个用来改进Adam的方法：前期用Adam，享受Adam快速收敛的优势；后期切换到SGD 。
  • 自适应学习率训练到后期，学习率出现极端情况，更新参数时有些维度上学习率特别大，有些维度学习率特别小。

表文件的关系

  • [Coding, MySQL]通过表文件的数据关系来描述现实生活中的隶属关系。

  • 隶属关系角色划分：

    1. 一方：拥有资源的一方
    2. 多方：依附于一方的资源

  • 三种关系

    • 一对一

      • e.g. 用户和用户详情
        常用于单表拆分，将一个列数很多的用户信息表，拆分称为一个只包含基础信息的小表（列数小）和一个包含其他信息的大表（列数大），来提升操作效率。
      • 实现：
        在任意的一个表中加一个 UNIQUE 外键，关联另一个表的主键。
        UNIQUE 保证了一对一的关系。

    • 多对多

      • e.g. 学生-课程：一个学生可以选修多门课，一门课可以被多个学生选择。
      • 实现：建立第三个中间表（选课表），中间表的一个外键关联学生的主键，一个外键关联课程的主键。

    • 一对多 / 多对一

      • e.g. 部门-员工：一个部门对应多个员工，一个员工只对应一个部门。
      • 实现：多方表（员工）建立外键，指向一方表（部门）的主键。

表文件字段的分类

  • 主键字段：

  • 非主键字段：描述主键编号。
  • 外键字段：
    1. 只存在多方表，描述多方数据与一方数据的依赖关系。
    2. 外键字段的值应该来源于一方表中的主键值。

关系型数据库

关系型数据库(rational database) 中的关系是 relation，定义为：

给定 nnn 个集合 S1,S2,S3,...,SnS1, S2, S3, ..., SnS1,S2,S3,...,Sn 。同时 RRR 是一个 nnn 元数组 (n-tuples)，它的第一个元素取自集合 S1S1S1 ，第二个元素取自集合 S2S2S2 ，以此类推。我们将 RRR 称之为基于该 n 个集合的一个 Relation， SjSjSj 为 RRR 的第 jjj 个域 (Domain)。

这里集合可以看成是 column 属性集合，由其元素组成的元组 RRR 可以看成每一行，即每一个实例都是一个 relation 。

所以在定义每个表时候，会先定义 “值域”（即上面所说的集合S1, S2, S3, …, Sn），然后再定义 “表”，即使用哪些集合。这种建模思想能 促进重用，例如 “性别”这个值域也可用在“教师”这个表里。这点在建立大型信息系统是极有价值的。

关系型数据库的关系模式就是二维表模型。

对象型数据库，比如 MongoDB，是 用面向对象的思想去设计数据存储模型。

非关系型数据库 NoSQL

参考博客：Sql Or NoSql，看完这一篇你就懂了

NoSQL 是随着数据越来越多，SQL的优化已经不能满足数据膨胀式的增长，而且SQL在分布式方面表现不是特别好，NoSQL就是在这种情况下产生的。

Not Only SQL，泛指非关系型数据库，是对关系型数据库的一种补充，不是替换。
NoSql与关系型数据库并不是对立关系，二者各有优劣，在合适的场景下选择合适的存储引擎才是正确的做法。

数据以键值对（kv）来存储，没有固定的(二维)结构，这样就不用局限于固定的结构。用于存储那些类型不固定的，也没有什么规律的数据。

在查询时，根据 key 就可以取出对应的 value。

优点/为什么还需要 NoSQL：

  1. 非关系型数据库是基于键值对的，数据处理过程不需要经过SQL层的解析，效率相对很高。
  2. 传统的关系型数据库很难实现水平扩展，扩容成本很大。
    而对于非关系型数据库是基于键值对，数据之间没有耦合性，能够非常容易水平扩展。

文档数据库（MongoDB）

这里说的“文档”其实是一个数据记录，这个记录能够对包含的数据类型和 内容进行 “自我描述”。
文档数据库可以包含非常复杂的数据结构，比如嵌套对象并且不需要使用特定的数据模式，每个文档可以具有 完全不同的结构。

DML (Data Manipulation Language) 数据操作语言
数据操纵分成 数据查询 和 数据更新 两类。数据更新又分成 插入、删除、和修改 三种操作。
mysql一共有7个查询命令：FROM，SELECT，WHERE，GROUP BY，HAVING，ORDER BY，LIMIT

查询命令和临时表

  1. 每一个查询命令在执行时，都是在操作上一个命令生成的临时表。
  2. 每一个命令执行完毕后，Mysql服务器自动在内存中销毁上一个查询命令生成的临时表。所以，用户只能看到最后一个查询命令生成的临时表。
  3. 只有FROM命令不需要操作临时表。
  4. 只有GROUP BY命令在执行完后有机会生成多个临时表。
  5. 只有HAVING命令在执行完毕后，不会生成新的临时表。负责将GROUP BY命令生成的临时表中不满足的临时表从内存中删除。

FROM命令

  1. 永远是第一个执行的查询命令，负责将硬盘上的表文件加载到内存中生成临时表供后续查询命令使用。所以，执行查询命令时，是不会修改文件原来的数据的。
  2. 临时表的名称与硬盘上的表文件名称保持一致。

      1. 以".frm"结尾的一种文件
      2. 存在于服务端计算机硬盘上
      3. 以数据形式进行数据存储的文件
    • 结构：

  • 数据库：存放表文件的文件夹

  • 数据库服务器：

    1. 一种专门对表文件进行调用和管理的软件
    2. 用户通过数据库服务器对数据库进行操作

SQL命令

  • 全称：Structured Query Language 结构化查询编程语言
  • 作用：用户通过SQL命令向数据库服务器发送请求，用于对表文件进行调用管理
  • 特点：包含主谓宾

数据库服务器分类

  1. 关系型数据库服务器
    a. 管理的表文件有隶属关系，可以完整地描述一段数据
    b. 在查询时，所涉及的数据比较多，因此查询的速度有所下降
    c. 分类：甲骨文公司的Oracle和Mysql，微软公司的SqlServer
  2. 非关系型数据库服务器
    a. 表文件都是独立的，无法描述一段完整的数据
    b. 查询的大数据比较少，查询速度快
