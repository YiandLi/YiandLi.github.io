UniRel

Paper：UniRel: Unified Representation and Interaction for Joint Relational Triple Extraction

将三元组学习目标拆分三种交互关系预测：

  1. 实体之间的关系（s，o）
  2. 实体跟关系类型的关系（s|o，r）
  3. 关系类型跟实体的关系（r，s|o）

encoding

所有关系类型的名称的都用一个核心词来代表，如关系“/business/company/founders”用“founders”来表示，“is captial of”用“captial”来表示。
然后将预测文本跟所有关系类别的核心词进行拼接，形成新的序列，输入到 encoder 中进行编码：

REBEL

Paper：EMNLP2021 - REBEL: Relation Extraction By End-to-end Language generation

现存问题：

  1. Pipeline / table filling 会误差传递，并且对于一个 entity pair，通常仅仅考虑一种关系进行训练，不同关系类别之间没有交互。
  2. 不能直接迁移到别的数据集，因为需要针对不同的关系类型做出训练。
  3. 当前的自回归方式则训练很多轮(epoch)才能很好地拟合，而且同样有暴露偏差的问题

作者：

  1. 根据维基百科，提出一个新的开放性关系抽取数据集（more than 200 different relation types
  2. 使用自回归生成模型解决关系抽取和关系分类问题，设计一种新的解码模版（a novel triplet linearization
  3. 使用 BART-large 在该数据集上进行预训练，然后只需要在目标数据集上微调几轮即可有很好的迁移能力

REBEL Model

base model：BART-large

解码模版

其中<triplet>，<subj>，<obj> 是新引入的 marker
对于相同的头实体，则只生成一次，可以降低解码复杂度和时间

如果需要同时预测实体类型，则将 <subj>，<obj> 替换为实体类型标签 marker 即可，比如 <per>，<org> 。

训练数据预处理

为了维护一个一致的序列，作者对关系进行处理，来生成训练序列

  1. 将所有实体根据在输入数据中的频率排序（by their order of appearance in the input text）
  2. 依次拿出频率最高的 head entity
  3. 针对每个 head entity，依次拿出频率最高的 tail entity

即首先是按照头实体的频率顺序sample，然后按照尾实体的频率顺序sample 。

REBEL dataset

问题：

  1. 自回归模型需要更多的训练数据
  2. 现存的 end-to-end RE dataset 很少

作者根据维基百科构建了新的数据集

  1. 输入数据是维基百科中每个词条的 abstract ，使用 wikiextractor工具 获得
  2. 将其中出现的超链接作为实体，使用 wikimapper工具 将每个超链接转换为真正的 entity ，并抽取出关系
  3. 尽管可以抽取出实体间的关系，但是这并不表示 abstract 一定表达了这种关系，所以作者使用 xlm-RoBERTa-large 在结果上进行 entailment 打分，取得分大于 0.75 的关系组。

DIRECT

Paper: ACL2021 - Adjacency List Oriented Relational Fact Extraction via Adaptive Multi-task Learning

Model: aDjacency lIst oRiented rElational faCT (DIRECT)
使用 multi-task learning 训练方式，作者使用了一个 动态的子任务损失均衡问题 (a dynamic sub-task loss balancing)

RE：a directed graph construction task

作者认为关系抽取其实是一个有向图构建任务，并且目前有三种方式来构建这个关系图，即三种解决RE的范式：

  1. Edge List：使用端到端生成模型直接生成三元组。
    但是存在问题：

    • 难以应对相同 entity pair 之间的多标签现象（multi-relation between entities
    • 对于上述现象，模型需要对同一对 entity pair 重复性生成多个三元组，增加了推理成本
    • 多个生成 triplet 之间存在顺序关系，并且顺序会影响模型性能

  2. Adjacency Matrices：使用矩阵表示关系
    模型首先抽取出所有实体，然后对不同实体对进行分类
    存在问题：每个关系各有一张表，矩阵比较稀疏，空间复杂度比较高。

  3. Adjacency List：使用 array of linked lists 表示关系路径
    复杂度介于 edge list 和 adj matrix，可以平衡空间和算法复杂度。
    通常依次抽取出 subj, obj, relation
    存在问题：错误传递，各个任务都是独立的，无法共享监督信号

Model: DIRECT

使用 fully adjacency list 的思路，subj，obj，rel 依次进行抽取

1 shared BERT encoder
3 output layers: subject extraction, object extraction, relation classification.

  1. 编码 sentence [[cls],x,[sep]][[\mathrm{cls}], \mathbf{x},[\mathrm{sep}]][[cls],x,[sep]] ，使用 subject extraction module 抽取出所有的 subj

  2. 编码 subj+sentence [[cls],s,[sep],x,[sep]][[\mathrm{cls}], s,[\mathrm{sep}], \mathbf{x},[\mathrm{sep}]][[cls],s,[sep],x,[sep]]，使用 object extraction module 抽取出所有的 obj

  3. 编码 subj+obj+sentence [[cls],s,[sep],o,[sep],x,[sep]][[\mathrm{cls}], s,[\mathrm{sep}], o,[\mathrm{sep}], \mathbf{x},[\mathrm{sep}]][[cls],s,[sep],o,[sep],x,[sep]] ，使用 relation classification module 对 [cls] 进行关系分类

SPN

Paper: 2020 - Joint Entity and Relation Extraction with Set Prediction Networks

生成式抽取的问题：句子中关系三元组之间是无序的，但是以前的工作需要将不同的关系三元组转换为文本序列，并且使用 ce loss 强制模型学习了三元组之间的顺序。

为了避免生成顺序的问题 ，作者

  1. 使用 非自回归的平行解码方案 non-autoregressive parallel decoding 来生成答案。

  2. 设计了 a set-based bipartite matching loss ，可以忽略三元组的位置关系

Model：set prediction networks (SPN)

  1. a sentence encoder - BERT
  2. a set generator - the transformer-based non-autoregressive decoder : generate a set of triples in a single pass
  3. a set based loss function：bipartite matching loss function

首先将输入句子编码成 He∈Rl×d\mathbf{H}_e \in \mathbb{R}^{l \times d}He​∈Rl×d

decoder architecture

包含 N 层 transformer 层，每层为：

  1. multi-head self attention：model the relationship between triples,
  2. multi- head inter attention mechanism：fuse the information of the given sentence
  3. feed-forward network： decode every output embedding into relation type and entities

没有必要使用 beam search 来预防早停 。

实验：

RIFRE

Paper: Representation iterative fusion based on heterogeneous graph neural
network for joint entity and relation extraction

将句子中每个 token 和数据集中需要判别的 relation 看作为两种不同的 node，使用 GAT 去编码 node 的信息。

优点：

  1. 将关系也看作是节点，容易解决 overlapping 的问题。
  2. 通过图的信息传递，不同节点可以融合相互的信息（token <-> relation）。

Word node

使用 bert 分词，然后编码，得到 token representation：

[h1,h2,…,hN]=B([w1,w2,…,wN])\left[\mathbf{h}_1, \mathbf{h}_2, \ldots, \mathbf{h}_N\right]=\mathbf{B}\left(\left[w_1, w_2, \ldots, w_N\right]\right) [h1​,h2​,…,hN​]=B([w1​,w2​,…,wN​])

Relation node

将每个关系单独看作节点，然后随机初始化 embedding：

[r1,r2,…,rM]=WrE([r1,r2,…,rM])+br\left[\mathbf{r}_1, \mathbf{r}_2, \ldots, \mathbf{r}_M\right]=\mathbf{W}_r \mathbf{E}\left(\left[r_1, r_2, \ldots, r_M\right]\right)+\mathbf{b}_r [r1​,r2​,…,rM​]=Wr​E([r1​,r2​,…,rM​])+br​

rir_iri​ 为 one-hot 编码。

Heterogeneous graph layer

Word node 和 Relation node 是不同类型的节点，节点的邻居仅仅为另一个节点类型的其他节点。

类似 LSTM，使用一种自回归的模式，

  • Partition：每个时间步将嵌入切分为三个部分：entity partition, relation partition and shared partition 。
  • Filter：给不同任务生成不同的的特征。
