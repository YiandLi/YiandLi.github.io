Plug-and-Play VQA: Zero-shot VQA by Conjoining Large Pre-trained Models with Zero Training

From the perspective of general-purpose AI, the ability to perform new tasks by simply recombining large-scale pre-trained models, or foundation models without architectural changes or extra training would be highly desirable.

However, to obtain high performance without some form of end-to-end training would seem difficult.

Plug-and-Play VQA (PNP-VQA) -> zero-shot visual question answering:

  1. 使用 a network interpretability technique：Grad-CAM 检测问题相关的 image-patches
  2. 使用 Blip 为 Image 生成 textual caption
  3. 使用 UnifiedQA-v2 生成答案

Related Works

  • Large-scale image-text pre-training : Image-conditional Text Generation , Image-conditional MLM , Prefix Language Model (SimVLM) , Image-Text Match, Image-Text contrastive learning .

  • Low Resource : Zero-shot image-text retrieval , image captioning , VQA .

  • Natural language as an intermediate representation:

    • Few-shot image classification by generating natural language rules.
    • Natural Language Deduction through Search over Statement Compositions.
    • Least-to-most prompting enables complex reasoning in large language models.

Method

核心为，找到一个 PLM 和 PVLM 之间的接口 （an interface between a pre-trained language model and a pre-trained vision-language model）。

本文中，这个接口为 natural language image captions: cover information that is present in the image and be relevant to the question

模型包含了：

  1. an image-question matching module：找到问题相关 image patches
  2. an image captioning module：根据上面找到的 patch 去生成 caption
  3. a question answering module：根据 caption 回答问题

Informative Image Captioning

more than one way to describe these regions

generate maximally diverse captions to provide coverage of possible answers

使用 BLIP 中的 image captioning network ，并且在生成时使用了 top-k sample ，而不是 beam search，因为这样可以生成重复冗余的 caption 。

输入就前面采样到的 K’ 个 patch（每次采样到的 patch 不同），输出的前缀为 " a picture of " ，一共生成 N 个 caption 。

同时为了去重，这里用最简单的「非包含关系才保留」的原则。

Answering the Question

Pass the question and the generated captions as input to the model.

因为存在很多 caption， 这里使用了 Fusion-in-Decoder (FiD) 策略 。


Fusion-in-Encoder (FiE) ：concatenates the question and all captions into a long paragraph as input

Fusion-in-Decoder (FiD) ： encodes each caption with the question separately and concatenates the encoded representations of all tokens from all captions. The result is fed as input to the decoder and is processed through the cross-attention mechanism.
- reduce time complexity due to Attention Mechanism
- no maximum input length constraint

ICLR 2023 | UNIVERSAL VISION-LANGUAGE DENSE RETRIEVAL: LEARNING A UNIFIED REPRESENTATION SPACE FOR MULTI-MODAL RETRIEVAL

github: /OpenMatch/UniVL-DR

图 1(a) a divide-and-conquer approach :
为了实现多模态检索过程，当前的多媒体搜索系统通常采用“分而治之”( a divide-and-conquer approach )的方法。这些方法首先在单个模态中进行搜索，包括文本、图像、视频等 ，然后将各个模态的检索结果融合在一起，例如，在这些单/交叉模态检索器之上构建另一个排序模块来进行模态融合。

图 1(b) universal multi-modal retrieval -> a Universal Vision-Language Dense Retrieval (UniVL-DR) :
构建一个 end2end model ，同时检索多模态的字段 。首先将所有模态的字段映射到同一个表征空间（one universal embedding space），然后使用 KNN 来检索多模态的候选字段。

相关性建模（Relevance Modeling） 和 检索结果融合（Retrieval Result Fusion） 二者的建模过程通常交织在一起，以实现更准确的多模态检索结果。但是在多模态检索的场景下，不同的模态之间的建模和结果融合存在 gap 。

最终，本文将相关性建模（Relevance Modeling）、跨模态匹配（Cross-Modality Matching）和检索结果融合（Retrieval Result Fusion）进行统一的建模。

具体地：

  1. 使用 contrastive leaning + hard negatives + balanced-modality sampling，来对齐 vision-language embedding space，并且减轻 the modality preference，即检索时对某一种模态的偏好性。
  2. 同时使用 an image verbalization method ，使用 image caption 来减轻模态之间的 gap 。
    这种方法首先将 caption 和 原始像素块对齐（ aligns the semantics of image captions and figure pixels ），然后对 caption 进行润色，得到最终的 caption。

关于数据集，作者使用 a multi-modal question answering (QA) benchmark Web QA ，并将其转化为：给定一个 query，检索出对应的 candidate text & image collection 。

MULTI-MODAL RETRIEVAL TASK


Single Modality Retrieval
文本-文本检索 & 图像-图像检索，难点在于 「相关性搜索 search relevance」。

Cross Modality Retrieval
「text-image retrieval 文本-图像检索」 & 「image-text retrieval 图像-文本检索」
除了「相关性搜索 search relevance」之外，难点更加侧重于 「模态对齐 align semantics between captions and images」 。

这种跨模态匹配更加注重“浅层”的语义匹配，对于深层的文档理解能力要求不高。

Multi-Modal Retrieval
给定 query qqq ，在不同模态的字段数据集中进行查询，找出候选文本/图片字段，检索结果可能由文本文档、图像文档或文本文档与图像文档的混合序列组成。

多模态检索更加关注查询和文档之间的关联建模，且检索过程中涉及查询与文本文档的单模态匹配、查询与图像文档的跨模态匹配以及不同模态文档与查询的相似度之间的比较。
focuses more on relevance modeling between queries and documents, single/cross modality matching, and modality routing.

同时，我们可以将其他模态的字段转换为某个特定模态的数据（pre-route queries to a single modality），将任务简化为 text-text retrieval 和 text-image retrieval 。

Method

Univ-Search By Learning a Unified Embedding Space

query qqq
multi-modality documents D={dText 1,…,dText n,dImage 1,…,dImage m}\mathcal{D}=\left\{d_{\text {Text }}^1, \ldots, d_{\text {Text }}^n, d_{\text {Image }}^1, \ldots, d_{\text {Image }}^m\right\}D={dText 1​,…,dText n​,dImage 1​,…,dImage m​}

Paper：ITERATIVE ZERO-SHOT LLM PROMPTING FOR KNOWLEDGE GRAPH CONSTRUCTION

Proposes an innovative knowledge graph generation pipeline approach:

  • Iteratively prompting large language models to extract relevant components of the final graph
  • Zero Shot Prompt
  • Due to LLM, a scalable solution w/o labeling

技术： Named Entity Recognition, Relation Extraction, and Entity Resolution techniques

影响知识图谱构建工作的因素：
the right tradeoff between providing high data quality, scalability and automation

  • Data availability and acquisition：多个数据源，并且是异构的；数据是动态变化的，所以通过数据构造图谱的方式也要不断更新（对标监督学习）。
  • Data quality：数据质量应该是 accurate, complete, consistent, and reliable ，体现在标注上，但是成本很高。
  • Scalability：由于数据维度与数据源的异质性，需要可拓展的计算资源和成本。
  • Subjectivity and contextual knowledge：schema 需要上下文知识和外部知识。
  • Semantic disambiguation：语义消歧
  • Domain-specific expertise
  • External resources：entity linking，high risk of filtering out many relevant entities or concepts not included in the reference KB，还是泛化性不高。可以使用 Open Information Extraction (OpenIE)，但是会错误标注，precision 很低。
  • Evaluation
  • Pipeline definition

作者：
(i) 使用 LLM 进行抽取工作：Relies on adopting an LLM that is iteratively queried with a sequence of adequately well-formed prompts to perform the main stages of the KG generation
(ii) 不需要外部知识库和标注，自动化进行任务：Can effectively perform an automated entity/predicate resolution without the need of external knowledge-bases or any human effort.

Methodology

LLM Prompting

There are four supported roles, but we are only focused on the following:

  • system: the role which aims to tell the model how to behave in its responses; 提供指令
  • user: the role of the user messages/requests; 提供 input
  • assistant: the role of the model responses. 模型响应

Methodology Overview

  1. Candidate Triplet Extraction
  2. Entity/Predicate Resolution
  3. Schema Inference

Candidate Triplet Extraction

关键：设计合适的 prompt 来抽取三元组 ，作者将三元组的信息进行拓展 ，加入到 prompt 中。拓展内容为：

A proper entity characterization ， 除了需要抽取出 entity mention 外，还需要：

  1. a representative entity label
  2. a proper entity description
  3. a list of types or hypernyms that denote the entity beyond the mention

An appropriate characterization of triplets and predicates ，即关系标签：

  1. a suitable label
  2. a general description of the relationship existing between subject and object

总之，对单一的 label text 进行了拓展，加入了 description 和 hypernyms 。

作者认为 the generated description + the entity type list 能够提供更好的 entity context information，引导我们进行实际的语义化（semantification）。

Entity/Predicate Resolution

抽取出实体和关系后，需要进行消歧工作：

  • 实体消歧：identification and merging of mentions which refers to the same underlying entity 。
  • 关系消歧：identifying and resolving different expressions or textual forms that convey the identical relation between entities

难点：

  • 不借助于外部知识库和人工知识
  • 获取实体或谓词的完整语义知识仍然很复杂，尤其是在处理特定领域或复杂概念时。一词多义

作者使用 clustering + prompt LLM 的方式，构造一个解析模型，将抽取出来的不同的语义概念进行聚合（combines a semantic aggregation of concepts / identify and aggregate similar semantic concepts） 。

Schema Inference

the types of entities，their attributes and relationships ，traditionally requires domain expertise and feedback by human knowledge

作者仍然使用 prompt LLM 的方式自动化构建

  2. Description similarity：use embedding model

Concept shrinkage

  2. Hierarchical Agglomeration

Selective Context

压缩你的Prompt，让LLMs处理多达2倍的Context

背景：

  1. prompt/instruction 过长时，输入部分就需要进行 padding
  2. 连续对话时，压缩历史上下文记录能够提升效果
  3. LLMs在训练的过程中很大概率已经见到过类似内容的文本，所以这些内容并没有带来新的信息。

Prompt压缩的首要问题在于，确定需要压缩的内容。换句话说，判断哪些文本并不能为LLMs带来新的信息。

我们在这里采用self-information来对这一点进行衡量。

很简单，就是将log probability取负得到self-information ，表示信息量 I(x)=−log⁡P(x)I(x)=-\log P(x)I(x)=−logP(x)

里的P可以是小号的LLMs，不需要是大模型本体。例如 1.3b 的OPT和OpenAI-Curie(6.3b).

过程：
首先计算文本中每个短语的 self-information I(x)=−log⁡P(x)I(x)=-\log P(x)I(x)=−logP(x) 。
再根据想要的压缩比设置百分位数（类似 outliers），得到阈值。
删去低于阈值的短语。

RECOMP

RECOMP：通过压缩和选择性增强来改进检索增强型语言模型

将检索到的文本进行摘要

提出两种压缩器/摘要器：

  1. an extractive compressor：从检索到的文档中选择有用的句子
    Model： a dual encoder model，用 sentence embeddings 的点积表示 helpfulness 。

  2. an abstractive compressor：综合多个文档中的信息来生成摘要
    Model： an encoder-decoder model，对 LLM 进行蒸馏 。

两个压缩器都经过训练，以提高 LM 在最终任务上的性能，同时保持摘要的简洁。
如果检索到的文档与输入无关或没有向 LM 提供附加信息，我们的压缩器可以返回一个空字符串，实现选择性增强。
