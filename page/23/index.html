介绍

动机

图提供了数据的通用表示，很多问题可以通过图上的计算任务解决。

  • 问题：但是图上的节点是连通的，不是独立的。然而传统的机器学习的方法假设数据是独立同分布的，因此不适合直接用来解决图上的计算任务。
  • 措施：
    • 建立一种特定于图的新机制：基于图的分类问题被称为集体分类问题。对于某一个节点来说，集体分类不仅考虑其特征与标签的映射，同时也考虑其邻域之间的映射。
    • 构建一组特征来表示节点（图上的信息），然后在这个特征表示的基础上使用机器学习的算法。关键是如何为节点构造一组特征。
  • 为了找到最合适的特征表示方式，可以有两种方法：特征工程和特征学习。
    特征工程为人工设计的特征。
    特征学习则为自动学习节点的特征。根据下游任务自动学习，不需要大量的人工干预，可以轻松地适应新任务。

特征学习

  • 图特征选择：移除节点上的无关和冗余的特征。比如原有的图节点有100个特征，我们通过一种度量算法，找到最符合任务的几个特征。因为还是原来的特征，所以有很强的解释性。
  • 图表示学习：生成一组新的节点表示。通过原本的特征，变换为一组新的特征，解释性不强，但是表示能力更强，对后续的学习能力更强。

图特征选择

现实世界中的数据往往是高维的，存在噪音的。
特征选择的目标是自动选择一小部分特征，使之有最小的冗余度，但是与学习目标有最大的相关性。

图表示学习

  • 方式：
    • 谱聚类
    • 基于图的降维
    • 矩阵分解：将节点嵌入到低维空间中，再利用新的节点表示来重建邻接矩阵。

介绍

  • 我们需要在训练的每一步中调整模型的参数。优化算法 Optimization algorithm定义了这个过程。该算法被封装在了optimizer对象中。
  • 优化器的初始化需要传入模型中待优化的参数和学习率。
  • 在训练的loop中，我们经常需要3个步骤：
    • optimizer.zero_grad()：梯度清空
    • loss.backwards()：传播梯度
    • optimizer.step()：根据梯度调整参数

torch.optim 库

torch.optim是一个实现了多种优化算法的库，可以保存当前状态并且根据计算的梯度来更新参数。

例子

创建

  • 需要传入一个包含Parameter参数的可迭代对象，最简单的是model.parameters()，然后设置超参数(defaults)选项，比如学习率，权重衰减等。
  • 此外，用于分层学习率：optimizer也支持为每个参数单独设置选项，传入一个元素为字典的可迭代对象。
    每一个字典都分别定义了一组参数：必须包含一个param键及其超参数(defaults)选项（注意这里params键的值为Parameter类型元素的列表），还可以加上针对这些参数的参数选项（比如weight_decay,lr,momentum）。
  • 还可以使用分层学习率。后面传递的总参数选项，用于没有定义这些选项的parameter。这在只想改变一个参数组选项时是很有用的。
权值衰减/l2正则

首先正则化主要是为了防止过拟合，而过拟合一般表现为模型对于输入的微小改变产生了输出的较大差异。
这主要是由于有些参数w过大的关系，通过对||w||进行惩罚，可以缓解这种问题。
参数b对于输入的改变是不敏感的，不管输入改变是大还是小，参数b的贡献就只是加个偏置而已。
举个例子，如果你在训练集中，w和b都表现得很好，但是在测试集上发生了过拟合，b是不背这个锅的，因为它对于所有的数据都是一视同仁的（都只是给它们加个偏置），要背锅的是w，因为它会对不同的数据产生不一样的加权。

更新参数

所有的优化器都有step()方法来更新参数。有两种调用方式：

  • 方法一
    optimizer.step()：用在参数已经存好梯度的情况下，比如在调用loss.backward()方法之后。
  • 方法二
    有些优化算法需要多次重新计算函数，因此需要传递一个闭包，使它们能够重新计算您的模型。闭包应该清除梯度，计算损失，然后返回。

StepLR

StepLR(optimizer, step_size, gamma=0.1, last_epoch=-1, verbose=False)

    • gamma：调整系数，也就是每次调整学习率，都将之前的学习率乘以这个系数，具体调整方式：lr=lr∗gammalr=lr*gammalr=lr∗gamma

“为什么warmup有效 ？” 这个问题目前还没有被充分证明。

  • 直觉：
    • 深层网络随机初始化差异较大，如果一开始以较大的学习率，初始学习带来的偏差在后续学习过程中难以纠正。
      若一开始以较大的学习率学习，易对数据造成过拟合，需要经过之后更多轮的训练进行修正。
    • 而浅层神经网络一开始使用幅度大的更新并不会导致方向错误。
  • 理论：
    • 缓解模型在初期对 mini-batch 过拟合的现象
    • 保持模型深层的稳定性。在训练的过程中，如果有mini-batch内的数据分布方差特别大，这会导致模型学习剧烈波动，使其学得的权重很不稳定，这在训练初期最为明显，最后期较为缓解。
      初期在预热的小学习率下，模型可以慢慢趋于稳定,等模型相对稳定后再选择预先设置的学习率进行训练,使得模型收敛速度变得更快，模型效果更佳。
  • 实现：warmup_lr的初始值是跟训练预料的大小成反比的，也就是说训练预料越大，那么warmup_lr 初值越小
    • 使得学习率可以适应不同的训练集合size
    • 即使不幸学习率设置得很大，那么也能通过warmup机制看到合适的学习率区间（即训练误差先降后升的关键位置附近），以便后续验证。
  • 不同的策略和代码实现：Transformers之自定义学习率动态调整

Method torch.nn.functional.log_softmax

  • input (Tensor) : 输入
  • dim (int) : log_softmax 的维度
  • dtype (torch.dtype, optional) : 返回张量的 dtype。 如果设置了，则会先将 input 转为目标 dtype 然后进行 log_softmax 操作。可以有效防止数值 overflow 。

面试问题：

因为softmax会进行指数操作，也就是 exp() 的输入比较大的时候，指数形式会变的很大，可能就会产生 overflow 。python2中它的上限为 709。

所以考虑用 x-max(x) 替代原始的 x 输入到 exp() 中。值域变为了 (-∞,0]，即可有效防止上溢。
但是这里仍然可能存在下溢的可能性：

  1. 分母会不会下溢：不会！因为减去最大值后，分母一定有一项为 0，exp 后值为 1。(对应 max(x) 那一项 exp(0)=1）
  2. 分子上的下溢，仍然可能导致表达式估值为零。但是在预测时，为零不是什么问题，因为取的是最大值。
    如果先做 softmax ，然后再做 log，会导致下溢的情况产生。但是可以在前面加一个 log：log(exp(分子)/exp(分母)) = 分子-log(exp(分母))， 这样分子就不会被计算 exp()了（可参考下方推导）。

所以这里，一般会通过分子分母同时除以一个 eM{e^{M}}eM 的方法进行归一：

M=max⁡(x1,x2,...,xn)f(xi)=exiex1+ex2+…+exn=exieMex1eM+ex2eM+…+exneM=e(xi−M)∑jne(xj−M)\begin{aligned} & M = \max (x_1,x_2,...,x_n) \\ & f\left(x_{i}\right) = \frac{e^{x_{i}}}{e^{x_{1}}+e^{x_{2}}+\ldots+e^{x_{n}}} = \dfrac{\dfrac{e^{x_{i}}}{e^{M}}}{\dfrac{e^{x_{1}}}{e^{M}}+\dfrac{e^{x_{2}}}{e^{M}}+\ldots+\dfrac{e^{x_{n}}}{e^{M}}} =\frac{e^{\left(x_{i}-M\right)}}{\sum_{j}^{n} e^{\left(x_{j}-M\right)}} \\ \end{aligned} ​M=max(x1​,x2​,...,xn​)f(xi​)=ex1​+ex2​+…+exn​exi​​=eMex1​​+eMex2​​+…+eMexn​​eMexi​​​=∑jn​e(xj​−M)e(xi​−M)​​

所以 log 并不是防止上溢出的原因，但是可以起到防止分子下溢的功能。
加上 log 之后变为：

为什么需要加 log ：
使用log可以将乘法转换为加法，或许（不确定）可以起到加快运算的效果。
同时 求导更加方便，可以加快反向传播的速度。

CLASS torch.nn.MultiheadAttention

首先是公式：

MultiHead⁡(Q,K,V)=Concat⁡( head 1,…, head h)WOwhere headi=Attention(QWiQ,KWiK,VWiV)\begin{aligned} &\operatorname{MultiHead}(Q, K, V)=\operatorname{Concat}\left(\text { head }_{1}, \ldots, \text { head }_{h}\right) W^{O} \\ &\text{where head}_i= Attention \left(Q W_{i}^{Q}, K W_{i}^{K}, V W_{i}^{V}\right) \end{aligned} ​MultiHead(Q,K,V)=Concat( head 1​,…, head h​)WOwhere headi​=Attention(QWiQ​,KWiK​,VWiV​)​

可以看到跟，V 的维度并不是输出的维度，还会使用一个 WO∈RV∗EW^O\in \mathbb{R}^{V*E}WO∈RV∗E 矩阵进行维度转换。


同时注意输入序列（source sequence）和输出/目标序列（target sequence）：

  • encoder 中使用 self attention，所以两个序列长度相同，对应位置表示的 token 相同。
  • 但是 decoder中，K 和 Q 的序列长度是不相同的。即 K 包含了encoder 中的所有 token，表示需要被 attention 的 token；而 Q 序列只表示需要被预测的 token，同时并行计算时还需要在 forward() 方法中加 mask，防止看到后面的 token。

__init__()

  • embed_dim：输出的 embedding 维度
  • num_heads：并行的注意力头的数量，每个头对应 embed_dim//num_heads 维度，同时存在约束 embed_dim // num_heads==0。
  • dropout=0.0：Dropout概率值
  • bias=True：初始化 Q，K，V 时是否使用 bias。
  • add_bias_kv：是否在初始化K，V时，使用 bias。
    具体实现为：在维度 0（序列长度）上，增加 bias，即计算K，V之后增加 bias。
  • add_zero_attn：在维度 1 上，给 key 和 value batch 增加一个新的全 0 的 序列，即增加一个全 0 的 source sequence length。
  • kdim=None：默认为 embed_dim
  • vdim=None：默认为 embed_dim， 公式中的 WOW^OWO 矩阵的尺寸就是 vdim * embed_dim。
  • batch_first=False

forward()

  • query
    • unbatched input：(L,E_q)
    • batched input：(L,N,E_q) 或 (N,L,E_q)(batch_first=True)
      L为 target sequence length；N为 batch size；E_q为 query embedding dim edim，需要和模型的 embed_dim 一致。
  • key
    • unbatched input：(S,E_k)
    • batched input：(S,N,E_k) 或 (N,S,E_k)(batch_first=True)
      S为 source sequence length；N为 batch size；E_k为 key embedding dim kdim，和模型的 kdim 一致。
  • value
    • unbatched input：(S,E_v)
      • batched input：(S,N,E_v) 或 (N,S,E_v)(batch_first=True)
        S为 source sequence length；N为 batch size；E_v为 value embedding dim，和模型的 vdim 一致。
  • key_padding_mask：boolean型tensor，标识哪些输入不被query，即不被计算注意力分数。
    具体实现为，将其注意力分数设置为 -inf，这样对应的输出概率为 0。
    • unbatched query：S
    • batched query：(N,S)
  • need_weights=True：会同时输出最后的注意力权重张量，否则为 None。
  • attn_mask：给定一个和注意力分数张量相同的张量
    • 尺寸：
      • 2D：(L,S)：targeted sequence len * source sequence length，会被广播为 3D 张量。
      • 3D：(N * num_heads,L,S)
        N 为 batch size；L为 target sequence length；S为 source sequence length。
    • 数值类型：
      • boolean：True表示不做 attention
      • byte：非 0 值表示不做 attention
      • float：值会被增加到 attention 分数上去

定义模型

nn.Module是PyTorch体系下所有神经网络模块的基类。

通过继承nn.Module来定义神经网络，并且通过其_init__()方法初始化神经网络中的module，parameter和buffer。
每一个nn.Module子类，即模型，都需要实现forward()方法来操作输入数据。

  • 模型进行前向传播时，除了forward()方法之外，还会有一些底层的运算。
    所以在得到输出时，不要直接调用模型的forward()方法，而是使用output = model(input)的形式。

__dict__ 实例变量

model.__dict__保存了模型的所有实例变量的键值对。

属性添加

  • add_module(name, module)：增加子神经网络模块，更新 self._modules。
    初始化类型为nn.Module。
  • register_parameter(name, param)：增加通过BP更新的 parameters ，更新 self._parameters。
    初始化类型为nn.Parameter。
  • register_buffer(name, tensor, persistent=True)：增加不通过 BP 更新的 buffer，更新 self._buffers
    初始化类型为torch.Tensor或者 None。
    默认persistent=True时，会将该buffer添加到模型的state_dict中。
    当persistent=False时，会更新到self._non_persistent_buffers_set中。

这三个方法中的name属性需要非空并且不包含'.'。
这 3 个函数都会:

  • 先检查 self.dict 中是否包含对应的属性字典以确保 nn.Module 被正确初始化。
  • 然后检查属性的 name 是否合法
  • 同时还会检查他们是否已经存在于要修改的属性字典中

不过更常见的是使用self.xxx = xxx添加module和parameter。这种方式本质上会调用 nn.Module 重载的函数__setattr__。
在调用在增加 self._parameters，self._modules 的时候，会预先调用 remove_from 函数从其余私有属性中删除对应的 name。
这说明 self.dict，self._buffers，self._parameters，self._modules 中的属性名是互斥的 。

  • buffer是内存中的一个常量，在后向传播的过程中不会被更新，在模型保存和加载时，可以被写入和读出。
    例子：批次归一化层中的平均值和方差。
    对于buffer而言：register_buffer()用于新添加Buffer：self.xxx = xxx只能将self._buffers中已有的 buffer 重新赋值为 None 或者 tensor。

self.xxxx = torch.Tensor() 是一种不被推荐的行为，因为这样新增的 attribute 既不属于 self._parameters，也不属于 self._buffers，而会被视为普通的 attribute ，在将模块进行状态转换的时候，self.xxxx 会被遗漏进而导致 device 或者 type 不一样的 bug。

  • parameters()：返回模型的参数（张量形式）迭代器，可以看作named_parameters()返回元组的参数部分。

  • named_modules(memo=None, prefix='')

训练状态的转换

nn.Module 通过 self.training 来区分训练和测试两种状态，使得模块可以在训练和测试时有不同的 forward 行为。
nn.Module 通过 self.train() 和 self.eval() 来修改训练和测试状态，其中 self.eval 直接调用了 self.train(False)，而 self.train() 会修改 self.training 并通过 self.children() 来调整所有子模块的状态。

  • train((mode=True) ：只对Dropout和BatchNorm有效。
  • eval() == train(False)：只对Dropout和BatchNorm有效。

__apply(fn)与apply(fn)方法

nn.Module 实现了如下 8 个常用函数将模块转变成不同的数据类型、并且转移到CPU/ GPU上。

  1. CPU：将所有 parameters 和 buffer 转移到 CPU 上
  2. type：将所有 parameters 和 buffer 转变成另一个类型
  3. CUDA：将所有 parameters 和 buffer 转移到 GPU 上
  4. float：将所有浮点类型的 parameters 和 buffer 转变成 float32 类型
  5. double：将所有浮点类型的 parameters 和 buffer 转变成 double 类型
  6. half：将所有浮点类型的 parameters 和 buffer 转变成 float16 类型
  7. bfloat16：将所有浮点类型的 parameters 和 buffer 转变成 bfloat16 类型
  8. to：移动模块或/和改变模块的类型

这些转换参数的数据类型(dtype)，转移数据位置(cpu/gpu)等操作最终都是通过self._apply(function)来实现的。function 一般是 lambda 表达式或其他自定义函数。

  • self._apply(function)函数做了三件事：
    1. 对self.children()中的子module进行递归调用：module._apply(fn)，最后处理到到下面两步。
    2. 对self._parameters中的param.data及其param.gradient通过 function 进行处理：
      param.data=fn(param)和param.grad.data=fn(param.grad)。
      在更改这两个值时，会使用compute_should_use_set_data( param, fn(param) )函数来判断是否原地修改。
    3. 对self._buffers中的 buffer 逐个通过 function 来进行处理：self._buffers[key]=fn(buf)
  • self.apply(fn)：只是简单地递归调用了 self.children() 去处理自己以及子模块：module.apply(fn)

Dataset

torch.utils.data.Dataset对象有两种：「Map-style dataset」和 「Iterable-style dataset」。
每个样本有对应的索引，通过输入具体的索引，就能够得到自定义的样本信息。

Map-style dataset

自定义的Dataset对象必须实现__init__(), __len__()和__getitem__(self, idx)方法，作为一种存储和读取数据的协议。

「Map-style dataset」并不是直接读取的数据集，而是读取每个样本的索引。
即后续取样本时，通过借助索引，即__getitem__()协议的方式读取样本。大致流程为：

参数：

  • __init__()：在创建dataset时被调用一次
  • __len__()：返回样本数量
  • __getitem__(self, idx)：加载并且返回一个下标为idx的样本。
    对应的操作符就是索引符号[]，即dataset.__getitem__(idx)和dataset[idx]操作一致。

Iterable-style dataset

并不需要实现__getitem__()方法和__len__()方法。Dataset本身更像是一个迭代器，使用__iter()__协议。
不同于「map-style dataset」，该类型数据集下样本的索引相互独立，注意使用 DataLoader 载入数据时，如果 num_worker 数量大于1，则需要独立分配索引。
