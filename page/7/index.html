MixMatch

Posted on 2021-11-27 Edited on 2022-05-22

论文地址：MixMatch: A Holistic Approach to Semi-Supervised Learning
代码地址：https://github.com/google-research/mixmatch

介绍

传统半监督学习（Semi-supervised learning /SSL）都是通过在损失函数中添加与未标记数据相关的损失项，使模型可以泛化到未标注数据。
这个损失项一般为三种：
- entropy minimization：模型预测出低熵分布。
- consistency regularization：对扰动输入保持相同的概率分布。
- generic regularization：增强泛化能力，减少过拟合。
为未标记的数据引入统一的损失项，该项减少了熵，同时保持一致性并与传统正则化技术保持兼容。

相关工作

一个根据输入x和参数 θ\thetaθ 生成标签分布y的通用模型： pmodel⁡(y∣x;θ)\mathrm{p}_{\operatorname{model}}(y \mid x ; \theta)pmodel​(y∣x;θ)

Consistency Regularization 自洽正则化

对于一个无标注数据，原始的样本数据和增强后的样本数据经过模型，得到的预测值应该一致。
即新的损失项为：

∥pmodel (y∣ Augment (x);θ)−pmodel (y∣Augment⁡(x);θ)∥22\| \mathrm{p}_{\text {model }}(y \mid \text { Augment }(x) ; \theta)-\mathrm{p}_{\text {model }}(y \mid \operatorname{Augment}(x) ; \theta) \|_{2}^{2} ∥pmodel ​(y∣ Augment (x);θ)−pmodel ​(y∣Augment(x);θ)∥22​

注意数据增广是随机操作，两个 Augment(x)Augment(x)Augment(x) 的输出不同。
本文使用了对图像的随机左右翻转和剪切（Crop）。

Entropy Minimization 最小化熵

许多半监督学习方法都基于一个共识，即分类器的分类边界不应该穿过边际分布的高密度区域 (high-density regions of the marginal data distribution)。
传统做法就是，分类器给未标注数据输出低熵预测。实现方法为增加一个新的损失项：pmodel⁡(y∣x;θ)\mathrm{p}_{\operatorname{model}}(y \mid x ; \theta)pmodel​(y∣x;θ)
本文用了一种名为 “sharpening” 的方法来保证无标准数据的低熵预测。

Traditional Regularization 传统正则化

正则：对模型施加约束，避免其记忆训练样本从而只达到统计效果，从而让模型泛化性更高，即预测没有见过的样本。
本文使用了Adam实现权值衰减，并且将 Mixup 同时用在了标记数据和未标记数据中。

mixup

介绍

  • 大型深度神经网络的问题：记忆能力有局限性 (memorization)，对对抗样本太敏感 (sensitivity to adversarial examples)。
    说到底就是 只能对训练集的样本有很好的拟合效果，轻微的扰动和训练集之外的样本效果都不佳。
  • 做法：在 成对样本及其标签 的 凸组合（convex combinations） 上训练网络，增强了训练样本之间的线性表达。
  • 目的：增强泛化性，增加对抗样本的鲁棒性，稳定生成对抗网络的训练过程。

背景

ERM

  • 大型的深层神经网络有两个共性：
  1. 经验风险最小化原则/ Empirical Risk Minimization Rule（ERM）：
    训练的目的是为了减小训练集上的平均错误。
  2. 当前的sota模型的大小size正比于训练集样本的数量。
  • 但是传统机器学习理论指出：为了保证平均错误ERM收敛，模型大小不应该随着训练集数量增加而增加。
  • 这两个矛盾指出：ERM不适合被当作指标，因为会导致模型只是统计而不是学习，模型泛化能力太差，无法应对一些轻微的扰动攻击。

VRM

Vicinal Risk Minimization 临近风险最小化
使用数据增强技术得到相似但是不同的样本，得到样本的临域。

  • 常用的数据增强的弊端：
    • 数据增强可以提高泛化能力，但这一过程依赖于数据集，而且需要专门知识。
    • 数据增强没有对不同类不同样本之间领域关系进行建模。

mixup

基于上述问题，作者提出了一种新的数据无关的数据增强方式来构建样本：

(xi,yi)\left(x_{i}, y_{i}\right)(xi​,yi​) 和 (xj,yj)\left(x_{j}, y_{j}\right)(xj​,yj​) 为训练数据中随机抽取的两个样本。

From EMP to mixup

真实损失

在监督学习中，将预期损失（expected risk）/真实损失，定义为损失函数在数据的真实分布P上的平均值 ：

R(f)=∫ℓ(f(x),y)dP(x,y)R(f)=\int \ell(f(x), y) \mathrm{d} P(x, y) R(f)=∫ℓ(f(x),y)dP(x,y)

ℓ\ellℓ 表示损失函数。
P(X,Y)P(X,Y)P(X,Y) 表示特征集和标签集的联合分布 joint distribution。
(x,y)∼P(x, y) \sim P(x,y)∼P 表示P中的样本。

EMP/ Empirical Risk Minimization

但是，多数情况下我们不知道真实分布情况，即不知道特征和标签的全部关系，只有一个训练集合 D={(xi,yi)}i=1n\mathcal{D}=\left\{\left(x_{i}, y_{i}\right)\right\}_{i=1}^{n}D={(xi​,yi​)}i=1n​ 。

尽管易于求解并且广泛使用，但是只限定于训练集的样本，所以不适用于训练集之外的样本。

VRM

经验分布(empirical distribution) PδP_{\delta}Pδ​ 只是一种逼近 真实分布 PPP 的做法。

文本中的Mixup

论文：https://arxiv.org/pdf/1905.08941.pdf

  • WordMixup：用zero padding的方式将所有句子padding到相同的长度，然后将两个训练样本的word embedding序列与label用上面公式进行插值得到增强样本，发生在模型的embedding层。
  • SenMixup：在句子经过LSTM、CNN或者Transfomer处理后会得到整句话的sentence encoding，然后对两个样本的sentence encoding与label进行插值，最后将插值后的结果送到网络最末端的softmax层去处理。这种插值发生在网络的末端，可以复用之前encoding的结果，所以计算效率较高。

    同时，可以看到，传统的正则做法（dropout+L2）会导致很快过拟合，导致模型在test集上错误率增加。
    但是添加了Mixup后traning loss会下降到一个“平台”开始缓慢震荡下行，进入这个过程后，test loss并没有反弹，而是逐渐稳定下降，说明模型还是可以学到新的知识。这是因为Mixup增加了样本空间，使得模型不那么容易过拟合。
    同时，senMixup可以使得training loss维持在一个相对更高的位置更加剧烈的震荡，而test loss则更快速平稳地下行，所以senMixup地正则化效果更好。

实验

  • 实验步骤：
    • 在初步实验中，我们发现三个或更多示例的 凸组合（ convex combinations） 与从 Dirichlet分布 带权重采样（weights sampled） 同样有效（就是三个融合也有效），但是不会提供进一步的增益，而且会增加混合的计算成本。
    • 现在的做法：单个DataLoader得到的一个minibatch，然后shuffle做mixup。这种策略同样有效，同时减少了I/O需求。
    • 在相同label的样本中做mixup不会有增益效果。猜测可能是两者本来就相同，混过来混过去没效果。

mixup临域分布可以看成一种数据增强的方式。
它令模型学习到样本和样本之间的线性关系。我们认为，这种线性建模减少了在预测训练样本以外的数据时的不确定性。

讨论

论文中

  • 在作者的实验中，有以下一贯的趋势：随着α的增加，在真实数据中的训练错误增加，同时泛化差距 (Generalization Gap)减小（泛化能力增强）。
    • 泛化误差 Generalization error：模型在测试数据上测试，会得到一个误差，叫测试误差。但其实我们真正想要的是模型在总体上的误差，这就是泛化误差。而测试误差因为采样的关系，与泛化误差是有偏差的。
      Generalization error = training error + generalization gap
      泛化误差 = 训练集上的错误 + 训练集和测试集/总体上的差距
  • mixup可以隐式地控制模型的复杂度。
  • 增大网络容量，可以使训练误差对大的α值敏感性降低，这给mixup 带来更多的优势。

网上的讨论

  • https://www.zhihu.com/question/67472285
    • data augmentation -> 控制模型复杂度
      让 NN 在“空白区域”学到一个简单的线性插值函数，大大降低了无数据覆盖空间的复杂度。否则在空白区间可能会自己搞出来一个复杂度很高的巨型函数。
      一般的data augmentation生成的样本，并不是来自于data distribution，更不是它的i.i.d.抽样。但是一般的假设是训练集和测试集都是data distribution的i.i.d.抽样，所以这并不是经典意义上的增加training data。
      这些合成的training data的作用，是“增强模型对某种变换的确定性 invariance”，就是机器学习里经常提到的“减少模型估计的不确定性 variance”，也就是控制了模型的复杂度。
    • label的线性加权
      • label用的是one-hot vector编码，可以理解为对k个类别的每个类给出样本属于该类的概率。加权以后就变成了"two-hot"，也就是认为样本同时属于混合前的两个类别。
      • 另一种视角是不混合label，而是用加权的输入在两个label上分别计算cross-entropy loss，最后把两个loss加权作为最终的loss。由于cross-entropy loss的性质，这种做法和把label线性加权是等价的，大家可以自行思考一下。
    • 数学上：离散样本空间进行连续化，提高邻域内的平滑性。

    • eq=True：开启后仍然不支持比大小。
    • frozen=True后，所有属性变为final。

dataclasses.field

有些数据类功能需要额外的每字段信息。可以通过调用提供的field()函数来替换默认字段值。

  • 参数
    • default：该字段的默认值。
    • default_factory：必须是一个零参数可调用对象，当该字段需要一个默认值时，它将被调用。
      同时指定 default 和 default_factory 将产生错误。
    • init = true：开启时，该字段作为参数包含在生成的 __init__() 方法中。
    • repr = true：开启时，该字段包含在生成的 __repr__() 方法返回的字符串中。

模型


char embedding，word embedding，character squence representation，word squence representation

  • 三层架构：
    • character squence representation 字粒度序列表征
    • word squence representation 词粒度序列表征
    • inference layer 推理层

Character Sequence Representations

  • 可以大幅提高正确率
  • 两种结构：
    • Character CNN
    • Character LSTM
      Liu et al. (2018) 提出全文的bi-lstm表征模型，而不是以单词为单位。两者正确率差不多，但是后者可以并行计算。

Word Sequence Representations

  • 大多数情况下，bi-LSTM比TextCNN更好。
  • 两种结构：
    • Word CNN
      [window size = 3 + ReLu ] * 4 + Batch Normalization + dropout
    • Word LSTM

Inference Layer

CRF在NER和词性标注(POS-Tagging)效果更好，组块分析(Chunking)则表现不佳
