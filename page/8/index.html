Post Normalization

原版Transformer和BERT使用了 Post Normalization
为什么使用 Post Normalization ：残差会进一步放大方差，新的方差变成 σx2+σF(x)2\sigma^2_{x}+\sigma^2_{F(x)}σx2​+σF(x)2​ 所以我们也要想相应的策略缩小其方差。

一个朴素的想法是在残差后面加一个 Layer Normalization 。

但是这种方法会削弱 残差的恒等分支 ，即削弱本来流向 x 的梯度，从而弱化了残差 易于训练 的优点。

证明：
假设输入 xxx 和 F(x)F(x)F(x) 的方差均为 1，则进行 normalization 前残差结果 x+F(x)x+F(x)x+F(x) 的方差变为了 2 。
然后我们进行 Normalization Norm⁡(x+F(x))\operatorname{Norm}(x+F(x))Norm(x+F(x)) ，让方差从原始的 2 变成了 1 ，可以看成是除以了标准差 2\sqrt 22​ ，即有：

xt+1=Norm⁡(xt+Ft(xt))=xt+Ft(xt)2\quad \boldsymbol{x}_{t+1}=\operatorname{Norm}\left(\boldsymbol{x}_t+F_t\left(\boldsymbol{x}_t\right)\right) \\ =\frac {\boldsymbol{x}_t+F_t\left(\boldsymbol{x}_t\right)}{\sqrt 2} xt+1​=Norm(xt​+Ft​(xt​))=2​xt​+Ft​(xt​)​

对于一个 l 层的网络，我们可以得到输出 xlx_lxl​ 就等于：

xl=xl−1+Fl−1(xl−1)2=xl−12+Fl−1(xl−1)2=xl−222+Fl−2(xl−2)22+Fl−1(xl−1)2=xl−323+Fl−3(xl−3)23+Fl−2(xl−2)22+Fl−1(xl−1)2=...=x02l+F0(x0)2l+F1(x1)2l−1+...+Fl−2(x2)2l−2+Fl−1(xl−1)2\begin{aligned} \quad x_l &= \frac {\boldsymbol{x}_{l-1}+F_{l-1}\left(\boldsymbol{x}_{l-1}\right)}{\sqrt 2} = \frac {\boldsymbol{x}_{l-1}}{\sqrt 2}+\frac {F_{l-1}\left(\boldsymbol{x}_{l-1}\right)}{\sqrt 2} \\ & = \frac {\boldsymbol{x}_{l-2}}{ {\sqrt 2}^2 }+\frac {F_{l-2}\left(\boldsymbol{x}_{l-2}\right)}{ {\sqrt 2}^2 } + \frac {F_{l-1}\left(\boldsymbol{x}_{l-1}\right)}{\sqrt 2} \\ & = \frac {\boldsymbol{x}_{l-3}}{ {\sqrt 2}^3}+\frac {F_{l-3}\left(\boldsymbol{x}_{l-3}\right)}{ {\sqrt 2}^3} +\frac {F_{l-2}\left(\boldsymbol{x}_{l-2}\right)}{ {\sqrt 2}^2} + \frac {F_{l-1}\left(\boldsymbol{x}_{l-1}\right)}{\sqrt 2} \\ & = ... \\ & = \frac {\boldsymbol{x}_{0}}{ {\sqrt 2}^l} + \frac {F_0\left(\boldsymbol{x}_{0}\right)}{ {\sqrt 2}^l} + \frac {F_1\left(\boldsymbol{x}_{1}\right)}{ {\sqrt 2}^{l-1}} + ... +\frac {F_{l-2}\left(\boldsymbol{x}_{2}\right)}{ {\sqrt 2}^{l-2}} + \frac {F_{l-1}\left(\boldsymbol{x}_{l-1}\right)}{\sqrt 2} \end{aligned} xl​​=2​xl−1​+Fl−1​(xl−1​)​=2​xl−1​​+2​Fl−1​(xl−1​)​=2​2xl−2​​+2​2Fl−2​(xl−2​)​+2​Fl−1​(xl−1​)​=2​3xl−3​​+2​3Fl−3​(xl−3​)​+2​2Fl−2​(xl−2​)​+2​Fl−1​(xl−1​)​=...=2​lx0​​+2​lF0​(x0​)​+2​l−1F1​(x1​)​+...+2​l−2Fl−2​(x2​)​+2​Fl−1​(xl−1​)​​

梯度经过回传的时候，传递给残差项时需要乘以一个 2l{\sqrt 2}^l2​l ，即削弱了原始的梯度，同时层数越高，削弱程度越高。

Pre Normalization

xt+1=xt+Ft(Norm⁡(xt))\quad \boldsymbol{x}_{t+1}=\boldsymbol{x}_t+F_t\left(\operatorname{Norm}\left(\boldsymbol{x}_t\right)\right) xt+1​=xt​+Ft​(Norm(xt​))

类似地，进行推导：

xt=x0+F0(Norm⁡(x0))+F1(Norm⁡(x1))+...+Ft−1(Norm⁡(xt−1))x_t = x_0 + F_0\left(\operatorname{Norm}\left( x_0\right)\right)+F_1\left(\operatorname{Norm}\left(x_1\right)\right)+...+F_{t-1}\left(\operatorname{Norm}\left(x_{t-1}\right)\right) xt​=x0​+F0​(Norm(x0​))+F1​(Norm(x1​))+...+Ft−1​(Norm(xt−1​))

一共有 t+1{t+1}t+1 项，假设每项方差为 1 ，则 Normalization 操作可以近似为除以标准差 t+1\sqrt {t+1}t+1​ ，即：

Norm⁡(xt)=xt/t+1\operatorname{Norm}(x_t) = x_t / \sqrt {t+1} Norm(xt​)=xt​/t+1​

原式

x1=x0+F0(Norm⁡(x0))≃x0+F0(x0)x2=x1+F1(Norm⁡(x1))≃x0+F0(x0)+F1(x1/2)x3≃x0+F0(x0)+F1(x1/2)+F2(x2/3))...xl=x0+F0(x0)+F1(x1/2)+F2(x2/3)+⋯+Fl−1(xl−1/l)\begin{aligned} & \quad \boldsymbol{x}_{1}=\boldsymbol{x}_0+F_0\left(\operatorname{Norm}\left(\boldsymbol{x}_0\right)\right) \simeq \boldsymbol{x}_0+F_0 \left(\boldsymbol{x}_0\right)\\ & \quad \boldsymbol{x}_{2}=\boldsymbol{x}_1+F_1\left(\operatorname{Norm}(\boldsymbol{x}_1)\right) \simeq \boldsymbol{x}_0+F_0\left(\boldsymbol{x}_0\right) + F_1( {\boldsymbol{x}_1 } / {\sqrt 2}) \\ & \quad \boldsymbol{x}_{3} \simeq \boldsymbol{x}_0+F_0\left(\boldsymbol{x}_0\right) + F_1( {\boldsymbol{x}_1 } / {\sqrt 2}) + F_2(\boldsymbol x_2 / {\sqrt 3}))\\ & \quad ... \\ & \quad \boldsymbol x_l=\boldsymbol x_0+F_0\left(\boldsymbol x_0\right)+F_1\left(\boldsymbol x_1 / \sqrt{2}\right)+F_2\left(\boldsymbol x_2 / \sqrt{3}\right)+\cdots+F_{l-1}\left(\boldsymbol x_{l-1} / \sqrt{l}\right) \end{aligned} ​x1​=x0​+F0​(Norm(x0​))≃x0​+F0​(x0​)x2​=x1​+F1​(Norm(x1​))≃x0​+F0​(x0​)+F1​(x1​/2​)x3​≃x0​+F0​(x0​)+F1​(x1​/2​)+F2​(x2​/3​))...xl​=x0​+F0​(x0​)+F1​(x1​/2​)+F2​(x2​/3​)+⋯+Fl−1​(xl−1​/l​)​

可以看到，无论多少层，「恒等分支项」 的损失是不被缩减的。

当然，这样最后的 xl\boldsymbol x_lxl​ 方差将会很大，所以在接预测层之前 xl\boldsymbol x_lxl​ 也还要加个Normalization，这就是 GPT-2 的做法。

对比

  • 训练： 同一设置之下，Pre Norm结构往往更容易训练，甚至可以不加 warmup，因为它的「恒等路径」更突出；
    而 Post Norm 难以训练，容易崩溃，必须要加warmup，并且对超参数很敏感。

  • 梯度： Pre Norm 各层的梯度近似相同，但是底层的梯度还是要大于高层的梯度的。
    Post Norm 则是从输出层到输入层，梯度不断减小。

  • 最终效果（pre-train model/微调后）： 在 Pre-training 中，Pre-Norm 和 Post-Norm 都能做到大致相同的结果。
    Post-Norm 的结构迁移性能更加好，也就是说，Post-Norm 的 Fine-tune 效果明显更好。

Pre-Norm


Pre Norm: xt+1=xt+Ft(Norm⁡(xt))\quad \boldsymbol{x}_{t+1}=\boldsymbol{x}_t+F_t\left(\operatorname{Norm}\left(\boldsymbol{x}_t\right)\right)xt+1​=xt​+Ft​(Norm(xt​))
Post Norm: xt+1=Norm⁡(xt+Ft(xt))\quad \boldsymbol{x}_{t+1}=\operatorname{Norm}\left(\boldsymbol{x}_t+F_t\left(\boldsymbol{x}_t\right)\right)xt+1​=Norm(xt​+Ft​(xt​))

首先「主干」为灰色部分

方差累计：
从图中进行理解，pre-normalization 中的 Ft(Norm⁡(xt))F_t\left(\operatorname{Norm}\left(\boldsymbol{x}_t\right)\right)Ft​(Norm(xt​)) 输入为 Norm⁡(xt)\operatorname{Norm}\left(\boldsymbol{x}_t\right)Norm(xt​) 都是经过归一化的。

FFN / MHA 层的输出会引发「方差」/「对输入的映射」，从而方差聚集到「主干」上，而主干上是没有 LN 的，所以每一层 FFN/MTH 的「方差」/「对输入的映射」被保存在残差中的「恒等项」中的，不断累加到了主干网络上。

网络退化现象：

xl=x0+F0(LN(x0))+F1(LN(x1))+F2(LN(x2))+⋯+Fl−1(LN(xl−1))\quad \boldsymbol x_l=\boldsymbol x_0+F_0\left(\boldsymbol {LN}(x_0)\right)+F_1\left(\boldsymbol {LN}(x_1)\right)+F_2\left(\boldsymbol {LN}(x_2)\right)+\cdots+F_{l-1}\left(\boldsymbol {LN}(x_{l-1})\right) xl​=x0​+F0​(LN(x0​))+F1​(LN(x1​))+F2​(LN(x2​))+⋯+Fl−1​(LN(xl−1​))

而网路加深后，层数不断变大。
所有层 FFF 都是平权的，所以「单层带来的微小方差/效果」对于「主干上的总方差/效果」的影响可以视为小量，（同时输入都是 norm 过的，这一点不是决定因素），导致相邻层带来的计算效果差不多 / 不同层的 f 统计上是相似的，于是有：

f(norm⁡(xn))≈f(norm⁡(xn+1))f\left(\operatorname{norm}\left(x_n\right)\right) \approx f\left(\operatorname{norm}\left(x_{n+1}\right)\right) f(norm(xn​))≈f(norm(xn+1​))

于是有：

xn+2=xn+1+f(norm⁡(xn+1))=xn+f(norm⁡(xn))+f(norm⁡(xn+1))≈xn+2f(norm⁡(xn))\begin{aligned} x_{n+2} & =x_{n+1}+f\left(\operatorname{norm}\left(x_{n+1}\right)\right) \\ & =x_n+f\left(\operatorname{norm}\left(x_n\right)\right)+f\left(\operatorname{norm}\left(x_{n+1}\right)\right) \\ & \approx x_n +2 f\left(\operatorname{norm}\left(x_n\right)\right) \end{aligned} xn+2​​=xn+1​+f(norm(xn+1​))=xn​+f(norm(xn​))+f(norm(xn+1​))≈xn​+2f(norm(xn​))​

深层部分实际上更像扩展了模型宽度，所以相对好训练，但某种意义上并不是真正的 deep 。

上面每一层都是平权的，也可以在论文 《On Layer Normalization in the Transformer Architecture》中看到，各层梯度都差不多。

Post-Norm


Pre Norm: xt+1=xt+Ft(Norm⁡(xt))\quad \boldsymbol{x}_{t+1}=\boldsymbol{x}_t+F_t\left(\operatorname{Norm}\left(\boldsymbol{x}_t\right)\right)xt+1​=xt​+Ft​(Norm(xt​))
Post Norm: xt+1=Norm⁡(xt+Ft(xt))\quad \boldsymbol{x}_{t+1}=\operatorname{Norm}\left(\boldsymbol{x}_t+F_t\left(\boldsymbol{x}_t\right)\right)xt+1​=Norm(xt​+Ft​(xt​))

保证了主干方差恒定，每层对 x 都可能有较大影响。
代价则是模型结构中没有从头到尾的恒等路径，梯度难以控制。通常认为会更难收敛，但训练出来的效果更好。

Post Norm 和 Pre Norm 相反，它每 Norm 一次就削弱一次恒等项 xt\boldsymbol{x}_txt​ 的权重，变相更突出残差项 FtF_tFt​ 的（但是梯度还是递减的，只不过总体看更加偏向于残差项）。

所以其实 Post-N 和传统的深度网络都一样有梯度消失的问题，而 Pre-N 避免了这种问题，让训练变简单，但是带了新的问题，即网络退化问题。

RMS Norm

区别

RMS 没有减去均值，也没有 bias。可以在梯度下降时令损失更加平滑。

RMSNorm 比 LayerNorm 好在哪里?

  1. 虽然二者的时间复杂度一致，但 RMSNorm 确实减少了减去均值和加上bias的计算，这在目前大模型预训练的计算量下就能够体现出训练速度上的优势。
  2. RMSNorm在模型效果上的表现并不弱于LayerNorm(LN取得成功的原因可能是缩放不变性，而不是平移不变性，也就是减去均值可能没有作用)。

SwiGLU activation function

原始的公式中，在 FFN 层中间使用到了 ReLu 激活函数。

FFN(x,W1,W2,b1,b2)=max⁡(0,xW1+b1)W2+b2F F N\left(x, W_{1}, W_{2}, b_{1}, b_{2}\right)=\max \left(0, x W_{1}+b 1\right) W_{2}+b 2 FFN(x,W1​,W2​,b1​,b2​)=max(0,xW1​+b1)W2​+b2

在随后的实现中，它被进一步完善为没有偏见的版本，没有性能损失：

FFN(x,W1,W2)=max⁡(0,xW1)W2F F N\left(x, W_{1}, W_{2}\right)=\max \left(0, x W_{1}\right) W_{2} FFN(x,W1​,W2​)=max(0,xW1​)W2​

这里采用 SwiGLU激活函数 取代 ReLU激活函数，以提高性能。

GeLu 激活函数

The functional form of a Gaussian Error Linear Units is:

xP(X≤x)=xΦ(x)x P(X \leq x)=x \Phi(x) xP(X≤x)=xΦ(x)

XXX 是具有零均值和单位方差的高斯随机变量，
P(X<=x)P(X<=x)P(X<=x) 是 XXX 小于或等于给定值x的概率，即上面随机变量的累积概率分布。
这么做的原因是因为神经元的输入 xxx 往往遵循正态分布，前一层的输出如果越大，则越不容易被置０，越小则越容易被置０。

有的版本是将 P(X≤x)P(X \leq x)P(X≤x) 作为伯努利分布的概率参数，随机采样出0或1，0则输出被mask，否则不被mask；而上面的的乘法方式是一个近似。

定义上讲，P(X≤x)P(X \leq x)P(X≤x) 体现了随机变量 X 有多大概率是小于输入 x 的。e.g. 如果输入 x=0→P(X≤0)=0.5x=0\rightarrow P(X \leq 0)=0.5x=0→P(X≤0)=0.5 ；
效果上讲， P(X≤x)P(X \leq x)P(X≤x) 充当了系数，也决定了输入 xxx 被“丢弃”的可能性更高。

Swish 激活函数

下面是 Swish⁡β\operatorname{Swish}_\betaSwishβ​ 激活函数:

Swish⁡β(x)=xσ(βx)σ(z)∼sigmoid function\operatorname{Swish}_\beta(x)=x \sigma(\beta x) \\ \sigma(z) \sim \text{sigmoid function} Swishβ​(x)=xσ(βx)σ(z)∼sigmoid function

像GELU一样，Swish在零时也是可微的。
Swish 可以被视为线性函数和ReLU函数之间非线性插值的光滑函数。如果将 β\betaβ 设置为可训练参数，则模型可以控制插值程度。

  • 模型细节

    1. Tokenizer：使用的 byte- pair encoding (BPE)

    2. Pre-Norm + RMSNorm : 为了提高训练稳定性，对每个Transformer子层的输入进行归一化，而不是对输出进行归一化。

      • 在 Attention 和 MLP 前面加一个 Pre-RMS Norm ； 对比 pre-norm 和 post-norm

      • 同时使用的 RMS Norm，代码公式对比

        左边为 LN ， 右边为 RMS

        在 layer norm 的基础上进行了修改，发现 LN 的减去均值没有什么作用，所以这里使用：

        y=xMean(x2)+ϵ∗Wy = \dfrac{x}{\sqrt {\text{Mean}(x^2)+\epsilon}} * Wy=Mean(x2)+ϵ​x​∗W

        注意

        分子：没有减均值

        分母：求的「RMS ：均方根（Root Mean Square）」，而不是方差。

        外部：仅仅乘以一个可训练 gamma，没有加偏置

    3. Attention 中的 Q，K 使用了 旋转位置嵌入（RoPE）

      优点是这种方法不仅可以处理位置信息，还可以处理距离信息，因为旋转操作可以很好地反映出元素之间的相对位置关系；缺点是它需要更多的计算资源，因为旋转操作比简单的向量加法更复杂。

    4. Casual Mask : 确保模型只能看到前面的tokens

    5. MLP 用 SwiGLU 来替代 ReLU 激活函数

      从 relu->SwiGLU，因为涉及到多一维度的线性表示，所以会面临着带来 4LH24LH^24LH2 的参数量新增和计算量增加。而llama中怎么克服这一点的呢？llama是把SwiGLU中的W，V，W2的矩阵维度从(dim， dim)变成(dim, 2/3 dim)，从而打平参数量和计算量。

    6. AdamW 优化器： β1 = 0.9, β2 = 0.95。

  • 高效实现

    1. Causal Multi-Head Attention (基于 x-formers)，不存储注意力权重，且不计算 mask 掉的 query 和 key 的值。
    2. 手动实现了Transformer的激活函数，而没有用pytorch库的autograd，以得到更优的训练速度。同时使用了并行化技术提高训练速度。
    3. 重计算激活值 。手动实现 transformer 层的反向传递函数，保存了计算成本高的 activations，例如线性层的输出。
    4. 通过使用 model parallelism 和 sequence parallelism 来减少显存的使用量。
    5. 尽可能地将 activations 的计算和GPU之间的通讯进行并行。

    lamma2

    与Llama相同的是采用RMSNorm归一化、SwiGLU激活函数、RoPE位置嵌入、词表的构建与大小 。

    增加GQA（分组查询注意力），扩增了模型输入最大长度，语料库增加了40% 。

    GQA - 34B, 70B

    MHA（Multi-head Attention ： 标准的多头注意力机制，h个Query、Key 和 Value 矩阵。

    MQA（multi-query attention）让所有的 head 共享 1 个 KV projection 矩阵；

    GQA（grouped-query attention ）使用 8 个 KV projections（选择8是因为A100 8GPUs） 来减少内存占用。

    首先是缓存机制，一旦模型规模很大长度很长时，KV 根本就存不进缓存。

    比如 Llama 7B 模型，hidden size 是 4096，那么每个 timestep 需缓存参数量为 4096232=262144，假设半精度保存就是 512KB，1024 长度那就要 512MB 。

    于是退一步，放不进缓存可以放 DRAM 上去，而 DRAM 内存也就是我们常说的 GPU 显存。
    但 DRAM 读取到计算芯片和 SRAM 到计算芯片的速度，差了一个量级的，这会让计算芯片一直在等待。

    → Flash Attention，Paged Attention ; Flash Attention 就是减少了计算 SoftMax 时从 DRAM 内存读取数据次数，从而提高了效率。

    同样，MQA 也是一个软件层面上翻墙的一个方法。

    MQA 让 Q 仍然保持原来的头数，但 K 和 V 只有一个头(mean)，不同head 之间共享同一个key矩阵和value矩阵。

    即：预测一个 token，原本需要 8 头 query token 对 8 头 key / value token ；现在是 8 头 query token 对 1 头 key / value token 。

    这样只需要传输 一个 head 的 token 就好，相当于减到 1/8 的传输量。实验发现，吞吐量提升 30 - 40 %

    实际上 MQA 运算量和 MHA 是差不多的，可理解为读取一组 KV 头之后，给所有 Q 头用；但是传输量减少很多，同时也可以增加 batch - size 。

    而 GQA 呢，是 MHA 和 MQA 的折衷方案，既不想损失性能太多，又想获得 MQA 带来的推理加速好处。

    具体思想是，不是所有 Q 头共享一组 KV，而是分组一定头数 Q 共享一组 KV，比如上面图片就是两组 Q 共享一组 KV。

    如果要用 MQA 和 GQA，可以是从头训练的时候就加上，也可以像 GQA 论文里面一样，用已有的开源模型，挑一些头取个 mean 用来初始化 MQA 或 GQA 继续训练一段时间。

    LLaMA-2-chat

    Meta 在 LLaMA 2 基础上，通过监督微调创建 LLaMA-2-chat 的初始版本。
    接下来，LLaMA-2-chat 使用人类反馈强化学习（RLHF）进行迭代细化，其中包括拒绝采样和近端策略优化（PPO）。
    LLaMA-2 都有对应的 chat 版本 ，注意没有开源 34 B 版本，算作没有。

    细节：

    一个常见的问题是模型可能会忘记给定的指令。例如，您可能要求她以特定的格式生成回复，但经过一段时间后，LLM 会忘记它。为了解决这个问题，本文的作者提出了一种新的方法 Ghost Attention（GAtt），它帮助模型在多个回合中控制对话流程。

    SFT：

    为了提高模型训练效率，将多组数据进行拼接，尽量填满4096，每条数据直接用停止符隔开，计算loss时仅计算每条样本target内容的loss。

    RM：

    要求标注者写一个提示，模型输出多个回复。（为了保证多样性，使用 2 个 model response，并使用不同的 temperature。

    然后根据提供的标准在两个采样的模型回应之间进行选择。

    两个 Reward Model：Helpfulness Reward Model 用于评估回应的帮助程度；Safety Reward Model 用于评估回应的安全性

    有用性上：要求标注者标记他们对所选回应相对于另一种回应的偏好程度：要么他们的选择：明显更好（significantly better），更好（better），稍微更好（slightly better），或者微不足道地更好/不确定（negligibly better/unsure）。

    在安全性上，对两个结果会标记都符合安全性、只有一个复合安全性、都不符合安全性，以此收集安全性数据。

    RLHF IFT（迭代微调）：

    训练了五个连续版本的RLHF模型

    在每一轮迭代时，让模型重新生成回复。助于保持 Reward Model 在分布上，并为最新模型维持准确奖励。

    拒绝采样微调：模型输出时采样K个结果，选择奖励值最高的一个，在强化学习阶段进行梯度更新。

    只有70B模型进行了拒绝采样微调，而其他小模型的微调数据来自于大模型拒绝采样数据，相当于用大模型蒸馏小模型。
