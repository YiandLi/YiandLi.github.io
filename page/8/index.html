介绍

传统半监督学习（Semi-supervised learning /SSL）都是通过在损失函数中添加与未标记数据相关的损失项，使模型可以泛化到未标注数据。
这个损失项一般为三种：
- entropy minimization：模型预测出低熵分布。
- consistency regularization：对扰动输入保持相同的概率分布。
- generic regularization：增强泛化能力，减少过拟合。
为未标记的数据引入统一的损失项，该项减少了熵，同时保持一致性并与传统正则化技术保持兼容。

Entropy Minimization 最小化熵

许多半监督学习方法都基于一个共识，即分类器的分类边界不应该穿过边际分布的高密度区域 (high-density regions of the marginal data distribution)。
传统做法就是，分类器给未标注数据输出低熵预测。实现方法为增加一个新的损失项：pmodel⁡(y∣x;θ)\mathrm{p}_{\operatorname{model}}(y \mid x ; \theta)pmodel​(y∣x;θ)

正则：对模型施加约束，避免其记忆训练样本从而只达到统计效果，从而让模型泛化性更高，即预测没有见过的样本。

mixup

介绍

  • 大型深度神经网络的问题：记忆能力有局限性 (memorization)，对对抗样本太敏感 (sensitivity to adversarial examples)。
    说到底就是 只能对训练集的样本有很好的拟合效果，轻微的扰动和训练集之外的样本效果都不佳。
  • 做法：在 成对样本及其标签 的 凸组合（convex combinations） 上训练网络，增强了训练样本之间的线性表达。
  • 目的：增强泛化性，增加对抗样本的鲁棒性，稳定生成对抗网络的训练过程。

背景

ERM

  • 大型的深层神经网络有两个共性：
  1. 经验风险最小化原则/ Empirical Risk Minimization Rule（ERM）：
    训练的目的是为了减小训练集上的平均错误。
  2. 当前的sota模型的大小size正比于训练集样本的数量。
  • 但是传统机器学习理论指出：为了保证平均错误ERM收敛，模型大小不应该随着训练集数量增加而增加。
  • 这两个矛盾指出：ERM不适合被当作指标，因为会导致模型只是统计而不是学习，模型泛化能力太差，无法应对一些轻微的扰动攻击。

VRM

Vicinal Risk Minimization 临近风险最小化
使用数据增强技术得到相似但是不同的样本，得到样本的临域。

  • 常用的数据增强的弊端：
    • 数据增强可以提高泛化能力，但这一过程依赖于数据集，而且需要专门知识。
    • 数据增强没有对不同类不同样本之间领域关系进行建模。

  • WordMixup：用zero padding的方式将所有句子padding到相同的长度，然后将两个训练样本的word embedding序列与label用上面公式进行插值得到增强样本，发生在模型的embedding层。
  • SenMixup：在句子经过LSTM、CNN或者Transfomer处理后会得到整句话的sentence encoding，然后对两个样本的sentence encoding与label进行插值，最后将插值后的结果送到网络最末端的softmax层去处理。这种插值发生在网络的末端，可以复用之前encoding的结果，所以计算效率较高。

    同时，可以看到，传统的正则做法（dropout+L2）会导致很快过拟合，导致模型在test集上错误率增加。
    但是添加了Mixup后traning loss会下降到一个“平台”开始缓慢震荡下行，进入这个过程后，test loss并没有反弹，而是逐渐稳定下降，说明模型还是可以学到新的知识。这是因为Mixup增加了样本空间，使得模型不那么容易过拟合。
    同时，senMixup可以使得training loss维持在一个相对更高的位置更加剧烈的震荡，而test loss则更快速平稳地下行，所以senMixup地正则化效果更好。

讨论

论文中

  • 在作者的实验中，有以下一贯的趋势：随着α的增加，在真实数据中的训练错误增加，同时泛化差距 (Generalization Gap)减小（泛化能力增强）。
    • 泛化误差 Generalization error：模型在测试数据上测试，会得到一个误差，叫测试误差。但其实我们真正想要的是模型在总体上的误差，这就是泛化误差。而测试误差因为采样的关系，与泛化误差是有偏差的。
      Generalization error = training error + generalization gap
      泛化误差 = 训练集上的错误 + 训练集和测试集/总体上的差距
  • mixup可以隐式地控制模型的复杂度。
  • 增大网络容量，可以使训练误差对大的α值敏感性降低，这给mixup 带来更多的优势。

动机

图提供了数据的通用表示，很多问题可以通过图上的计算任务解决。

  • 问题：但是图上的节点是连通的，不是独立的。然而传统的机器学习的方法假设数据是独立同分布的，因此不适合直接用来解决图上的计算任务。
  • 措施：
    • 建立一种特定于图的新机制：基于图的分类问题被称为集体分类问题。对于某一个节点来说，集体分类不仅考虑其特征与标签的映射，同时也考虑其邻域之间的映射。
    • 构建一组特征来表示节点（图上的信息），然后在这个特征表示的基础上使用机器学习的算法。关键是如何为节点构造一组特征。
  • 为了找到最合适的特征表示方式，可以有两种方法：特征工程和特征学习。
    特征工程为人工设计的特征。
    特征学习则为自动学习节点的特征。根据下游任务自动学习，不需要大量的人工干预，可以轻松地适应新任务。

特征学习

  • 图特征选择：移除节点上的无关和冗余的特征。比如原有的图节点有100个特征，我们通过一种度量算法，找到最符合任务的几个特征。因为还是原来的特征，所以有很强的解释性。
  • 图表示学习：生成一组新的节点表示。通过原本的特征，变换为一组新的特征，解释性不强，但是表示能力更强，对后续的学习能力更强。

6. 优化器optimizer和调度器scheduler

Posted on 2021-09-23 Edited on 2022-05-25

知乎 ： https://zhuanlan.zhihu.com/p/346205754?utm_medium=social&utm_oi=73844937195520

介绍

  • 我们需要在训练的每一步中调整模型的参数。优化算法 Optimization algorithm定义了这个过程。该算法被封装在了optimizer对象中。
  • 优化器的初始化需要传入模型中待优化的参数和学习率。
  • 在训练的loop中，我们经常需要3个步骤：
    • optimizer.zero_grad()：梯度清空
    • loss.backwards()：传播梯度
    • optimizer.step()：根据梯度调整参数

torch.optim 库

torch.optim是一个实现了多种优化算法的库，可以保存当前状态并且根据计算的梯度来更新参数。

例子

创建

  • 需要传入一个包含Parameter参数的可迭代对象，最简单的是model.parameters()，然后设置超参数(defaults)选项，比如学习率，权重衰减等。
  • 此外，用于分层学习率：optimizer也支持为每个参数单独设置选项，传入一个元素为字典的可迭代对象。
    每一个字典都分别定义了一组参数：必须包含一个param键及其超参数(defaults)选项（注意这里params键的值为Parameter类型元素的列表），还可以加上针对这些参数的参数选项（比如weight_decay,lr,momentum）。
  • 还可以使用分层学习率。后面传递的总参数选项，用于没有定义这些选项的parameter。这在只想改变一个参数组选项时是很有用的。
权值衰减/l2正则

首先正则化主要是为了防止过拟合，而过拟合一般表现为模型对于输入的微小改变产生了输出的较大差异。
这主要是由于有些参数w过大的关系，通过对||w||进行惩罚，可以缓解这种问题。
参数b对于输入的改变是不敏感的，不管输入改变是大还是小，参数b的贡献就只是加个偏置而已。
举个例子，如果你在训练集中，w和b都表现得很好，但是在测试集上发生了过拟合，b是不背这个锅的，因为它对于所有的数据都是一视同仁的（都只是给它们加个偏置），要背锅的是w，因为它会对不同的数据产生不一样的加权。

更新参数

所有的优化器都有step()方法来更新参数。有两种调用方式：

  • 方法一
    optimizer.step()：用在参数已经存好梯度的情况下，比如在调用loss.backward()方法之后。
  • 方法二
    有些优化算法需要多次重新计算函数，因此需要传递一个闭包，使它们能够重新计算您的模型。闭包应该清除梯度，计算损失，然后返回。

“为什么warmup有效 ？” 这个问题目前还没有被充分证明。

  • 直觉：
    • 深层网络随机初始化差异较大，如果一开始以较大的学习率，初始学习带来的偏差在后续学习过程中难以纠正。
      若一开始以较大的学习率学习，易对数据造成过拟合，需要经过之后更多轮的训练进行修正。
    • 而浅层神经网络一开始使用幅度大的更新并不会导致方向错误。
  • 理论：
    • 缓解模型在初期对 mini-batch 过拟合的现象
    • 保持模型深层的稳定性。在训练的过程中，如果有mini-batch内的数据分布方差特别大，这会导致模型学习剧烈波动，使其学得的权重很不稳定，这在训练初期最为明显，最后期较为缓解。
      初期在预热的小学习率下，模型可以慢慢趋于稳定,等模型相对稳定后再选择预先设置的学习率进行训练,使得模型收敛速度变得更快，模型效果更佳。
  • 实现：warmup_lr的初始值是跟训练预料的大小成反比的，也就是说训练预料越大，那么warmup_lr 初值越小
    • 使得学习率可以适应不同的训练集合size
    • 即使不幸学习率设置得很大，那么也能通过warmup机制看到合适的学习率区间（即训练误差先降后升的关键位置附近），以便后续验证。
  • 不同的策略和代码实现：Transformers之自定义学习率动态调整
