为什么现在都是 decoder only 架构 ？

  1. Encoder 的低秩问题

    双向 Attention 的注意力矩阵容易退化为低秩状态，而 causal attention 的注意力矩阵是下三角矩阵，必然是满秩的，建模能力更强，表达能力更强；

    Encoder-Decoder架构之所以能够在某些场景下表现更好，大概只是因为它多了一倍参数。所以，在同等参数量、同等推理成本下，Decoder-only架构就是最优选择了。

  2. 预训练任务难度

    纯粹的decoder-only架构 + 「next token prediction 预训练」，每个位置所能接触的信息比其他架构少，要预测下一个token难度更高，当模型足够大，数据足够多的时候，「decoder-only模型」 学习通用表征的上限更高。

    Bert 的双向模型可以看到前向和后向，这在预测的时候是天然的优势，但在训练的时候其实反而降低了学习难度，bert的双向提高了下限也拉低了上限。

    相同规模的 GPT 和 BERT 模型，GPT 的效果要比 BERT 差。但是从另一个角度去想，如果能够把 「预测未来」 这个事情做好的话，它最终所能达到的效果的天花板一定是更高的。

  3. 泛化性质

    In-context learning 对 LLM 有 few-shot fine tune 的作用，few shot / zero shot 效果更强。

    decoder-only 模型在没有任何 tuning 数据的情况下、zero-shot 表现最好，而 encoder-decoder 则需要在一定量的标注数据上做 multitask finetuning 才能激发最佳性能。

  4. 大数据训练+大参数模型的涌现能力替代了multitask fine-tuning

    5B参数量+170B token 数据量时，在做 multitask fine-tuning 后 encoder-decoder 相比 decoder-only 反而在新任务（zero-shot）上会有一定的优势。

    但是，随着参数量进一步增加，encoder-decoder 在 multitask fine-tuning 上的优势在大参数量时被LLM的推理能力给拉平了。也就是 LLM 可以自动地从大数据里面做 self multitask fine-tuning 。

  5. causal attention （就是decoder-only的单向attention）具有隐式的位置编码功能 。

    3. Attention 中的 Q，K 使用了 旋转位置嵌入（RoPE）

      优点是这种方法不仅可以处理位置信息，还可以处理距离信息，因为旋转操作可以很好地反映出元素之间的相对位置关系；缺点是它需要更多的计算资源，因为旋转操作比简单的向量加法更复杂。

    4. Casual Mask : 确保模型只能看到前面的tokens

    5. MLP 用 SwiGLU 来替代 ReLU 激活函数

      从 relu->SwiGLU，因为涉及到多一维度的线性表示，所以会面临着带来 4LH24LH^24LH2 的参数量新增和计算量增加。而llama中怎么克服这一点的呢？llama是把SwiGLU中的W，V，W2的矩阵维度从(dim， dim)变成(dim, 2/3 dim)，从而打平参数量和计算量。

    6. AdamW 优化器： β1 = 0.9, β2 = 0.95。

  • 高效实现

    1. Causal Multi-Head Attention (基于 x-formers)，不存储注意力权重，且不计算 mask 掉的 query 和 key 的值。
    2. 手动实现了Transformer的激活函数，而没有用pytorch库的autograd，以得到更优的训练速度。同时使用了并行化技术提高训练速度。
    3. 重计算激活值 。手动实现 transformer 层的反向传递函数，保存了计算成本高的 activations，例如线性层的输出。
    4. 通过使用 model parallelism 和 sequence parallelism 来减少显存的使用量。
    5. 尽可能地将 activations 的计算和GPU之间的通讯进行并行。

    增加GQA（分组查询注意力），扩增了模型输入最大长度，语料库增加了40% 。

GQA - 34B, 70B

MHA（Multi-head Attention ： 标准的多头注意力机制，h个Query、Key 和 Value 矩阵。

MQA（multi-query attention）让所有的 head 共享 1 个 KV projection 矩阵；

GQA（grouped-query attention ）使用 8 个 KV projections（选择8是因为A100 8GPUs） 来减少内存占用。

首先是缓存机制，一旦模型规模很大长度很长时，KV 根本就存不进缓存。

比如 Llama 7B 模型，hidden size 是 4096，那么每个 timestep 需缓存参数量为 4096232=262144，假设半精度保存就是 512KB，1024 长度那就要 512MB 。

于是退一步，放不进缓存可以放 DRAM 上去，而 DRAM 内存也就是我们常说的 GPU 显存。
但 DRAM 读取到计算芯片和 SRAM 到计算芯片的速度，差了一个量级的，这会让计算芯片一直在等待。

→ Flash Attention，Paged Attention ; Flash Attention 就是减少了计算 SoftMax 时从 DRAM 内存读取数据次数，从而提高了效率。

同样，MQA 也是一个软件层面上翻墙的一个方法。

MQA 让 Q 仍然保持原来的头数，但 K 和 V 只有一个头(mean)，不同head 之间共享同一个key矩阵和value矩阵。

即：预测一个 token，原本需要 8 头 query token 对 8 头 key / value token ；现在是 8 头 query token 对 1 头 key / value token 。

这样只需要传输 一个 head 的 token 就好，相当于减到 1/8 的传输量。实验发现，吞吐量提升 30 - 40 %

实际上 MQA 运算量和 MHA 是差不多的，可理解为读取一组 KV 头之后，给所有 Q 头用；但是传输量减少很多，同时也可以增加 batch - size 。

而 GQA 呢，是 MHA 和 MQA 的折衷方案，既不想损失性能太多，又想获得 MQA 带来的推理加速好处。

具体思想是，不是所有 Q 头共享一组 KV，而是分组一定头数 Q 共享一组 KV，比如上面图片就是两组 Q 共享一组 KV。

如果要用 MQA 和 GQA，可以是从头训练的时候就加上，也可以像 GQA 论文里面一样，用已有的开源模型，挑一些头取个 mean 用来初始化 MQA 或 GQA 继续训练一段时间。

LLaMA-2-chat

Meta 在 LLaMA 2 基础上，通过监督微调创建 LLaMA-2-chat 的初始版本。
接下来，LLaMA-2-chat 使用人类反馈强化学习（RLHF）进行迭代细化，其中包括拒绝采样和近端策略优化（PPO）。
LLaMA-2 都有对应的 chat 版本 ，注意没有开源 34 B 版本，算作没有。

细节：

一个常见的问题是模型可能会忘记给定的指令。例如，您可能要求她以特定的格式生成回复，但经过一段时间后，LLM 会忘记它。为了解决这个问题，本文的作者提出了一种新的方法 Ghost Attention（GAtt），它帮助模型在多个回合中控制对话流程。

SFT：

为了提高模型训练效率，将多组数据进行拼接，尽量填满4096，每条数据直接用停止符隔开，计算loss时仅计算每条样本target内容的loss。

RM：

要求标注者写一个提示，模型输出多个回复。（为了保证多样性，使用 2 个 model response，并使用不同的 temperature。

然后根据提供的标准在两个采样的模型回应之间进行选择。

两个 Reward Model：Helpfulness Reward Model 用于评估回应的帮助程度；Safety Reward Model 用于评估回应的安全性

有用性上：要求标注者标记他们对所选回应相对于另一种回应的偏好程度：要么他们的选择：明显更好（significantly better），更好（better），稍微更好（slightly better），或者微不足道地更好/不确定（negligibly better/unsure）。

在安全性上，对两个结果会标记都符合安全性、只有一个复合安全性、都不符合安全性，以此收集安全性数据。

RLHF IFT（迭代微调）：

训练了五个连续版本的RLHF模型

在每一轮迭代时，让模型重新生成回复。助于保持 Reward Model 在分布上，并为最新模型维持准确奖励。

拒绝采样微调：模型输出时采样K个结果，选择奖励值最高的一个，在强化学习阶段进行梯度更新。

只有70B模型进行了拒绝采样微调，而其他小模型的微调数据来自于大模型拒绝采样数据，相当于用大模型蒸馏小模型。

  • SFT：数据合成，主题分类(Topic Classification)，质量分类(Quality Scoring), 难度分类(Difficulty Scoring)以及语义去重(Semantic deduplication)。
  • reward model + DPO，没有 PPO 的部分 。（ Qwen 也只用了 DPO，所以 Reward Model 重要，而 dpo 还是 ppo 可能没有那么重要）
  • DPO的重点是: Masking out formatting tokens in DPO loss以及Regularization with NLL loss
  • reward model: We train a reward model (RM) covering different capabilities on top of the pre-trained checkpoint。 直接用预训练后的模型做reward model。用它来筛数据搞Rejection Sampling和DPO。
  • 整个post training需要迭代六轮，每轮重标一次preference data训练reward model，然后RS，PPO。
  • 每次的流程相同，不同的地方在于拒绝采样阶段用来对给定Prompt产生回答的LLM模型，会从上一轮流程最后产生的若干不同DPO模型（不同超参等）里选择最好的那个在下一轮拒绝采样阶段给Prompt生成答案。很明显，随着迭代的增加DPO模型越来越好，所以拒绝采样里能选出的最佳答案质量越来越高，SFT模型就越好，如此形成正反馈循环。
  • 可以看出，尽管RLHF 和DPO两种模式都包含RM，但是用的地方不一样，RLHF是把RM打分用在PPO强化学习阶段，而LLaMA 3则用RM来筛选高质量SFT数据。而且因为拒绝采样的回答是由LLM产生的，可知这里大量采用了合成数据来训练SFT模型。

双线性建模三元组

RESCAL 2011

属于使用双线性模型建模三元组的一种

核心思想：将整个知识图谱编码为一个三维张量，由这个张量分解出一个核心张量和一个因子矩阵，核心张量中每个二维矩阵切片代表一种关系，因子矩阵中每一行代表一个实体。

如果两个实体之间存在着关系，则三维张量 X\mathcal XX 对应的点为1，否则为0，就像概率矩阵。

属于**「基于三元组+旋转距离 建模」，是「基于三元组+平移距离 建模 」**（Trans 系列）的升级版。

首先介绍两节点之间关系的几种类型：

  1. symmetric：对称关系，比如婚姻，j-婚姻→i ；i-婚姻→j
  2. antisymmetric：非对称关系
  3. inversion：相反关系，比如位词和下位词之间的两个关系
  4. composition：连续组合关系，比如 (母亲+丈夫) → 父亲

TransE 模型将 relation 表示为translation，旨在对 inversion 和 composition  这两种模式进行建模；

DisMult 模型用三种方法对头实体–关系–尾实体建模，旨在对 symmetry 模式建模。

但是，现有模型都无法建模和推断所有上述模式。

TransE模型可以对除了对称关系外的所有其他关系建模，主要原因在于TransE模型中所有的对称关系都会映射到零translation向量，这样就使得所有的具有对称关系的实体在目标空间中很接近。

文章提出了RotateE模型 ，来建模所有的上述关系。

损失函数部分

MSE 对比 MAE

MSE： l=(y−f(x))2l = (y-f(x))^2l=(y−f(x))2
MAE: l=∣y−f(x)∣l = |y-f(x)|l=∣y−f(x)∣

异常点：

  • mse对异常点更敏感，原因在于mse的计算中，异常点会因为平方而进一步放大，导致了异常点会对训练过程造成很大的影响。
  • 在计算回归树叶节点预测值时，mae 的最优解是中位数形式，所以会取得中位数作为叶子节点的预测值，而 mae 是取绝对值，影响不如mse的大。

训练速度：

  • mae的梯度是恒定的（不考虑不可导点），故在损失值大或小的情况下梯度均一致，显然不利于收敛。
  • mse会在损失值大的情况下赋予大的梯度，小的情况下赋予小的梯度，更利于收敛。

有一个超参数：阀值 δ\deltaδ

  • 当预测的 绝对值误差 小于 δ\deltaδ 时，使用平方误差：正常点使用 MSE，「下一个模型需要拟合的对象/梯度」为残差。
  • 当预测的 绝对值误差 当大于 δ\deltaδ 时使用线性误差：outlier 使用 MAE，「下一个模型需要拟合的对象」为 ±1\pm 1±1 。

当 δ→∞\delta\rightarrow \inftyδ→∞ 时， huber 趋近于MSE；
当 δ→0\delta\rightarrow 0δ→0 时， huber 趋近于 MAE 。
在每一轮训练中，GDBT 动态地确定 δ\deltaδ ，首先计算每个样本的 mae，然后从小到大排序，选择一个 α−\alpha-α− 分位点，默认 0.9 ，即认为 90% 的样本都是正常样本，而 10% 的数据是 outlier 样本。

相比于 MSE ，Huber Loss 降低了对异常点的惩罚程度 ，是一种常用的robust regression的损失函数。

Huber损失函数结合了MAE和MSE的鲁棒性，两者损失函数的收敛目标分别对应着label的中值和均值。均值受异常值的影响较大，而中值则相对稳定。如果数据中存在较多异常点，MSE受到的影响较大。但尽管MAE比较稳定，它的梯度大小始终不变（-1和1），在实际中，预估与label值相近的时候，梯度应该变得更小。Huber Loss结合了两者的优点，是很优秀的Robust Loss function。

牛顿迭代法

又称为牛顿-拉弗森方法，五次及以上多项式方程没有根式解，但是可以使用牛顿法近似求根。

思想：对于某一点A，在该点上的切线是函数的线性逼近，并且离A点距离越近，这种逼近的效果也就越好。

步骤：对于函数 f(x)f(x)f(x) ，在点 xnx_nxn​ 处做切线，得到切线方程 f(xn)+f′(xn)(x−xn)f(x_n)+f'(x_n)(x-x_n)f(xn​)+f′(xn​)(x−xn​) ，然后得到该切线和x轴的 交点 xn+1=xn−f(xn)f′(xn)x_{n+1}=x_n-\frac{f\left(x_n\right)}{f^{\prime}\left(x_n\right)}xn+1​=xn​−f′(xn​)f(xn​)​ ，然后沿着 x 轴作垂线，相交于新的点 xn+1,f(xn+1)x_{n+1}, f(x_{n+1})xn+1​,f(xn+1​) 。

不断迭代下去，最终和 x 轴的交点会不断接近根，这就是我们称的 收敛 。

不收敛的情况：

  1. 切线和 x 轴平行，于是和 x 轴没有交点
  2. 计算出和 x 轴的交点 xnx_{n}xn​ 不断远离根 -> f(x)=x1/3f(x)=x^{1/3}f(x)=x1/3
  3. 不断的循环震荡，交点呈现循环状态 -> f(x)=∣x∣1/2f(x)=|x|^{1/2}f(x)=∣x∣1/2
