知识蒸馏

Posted on 2021-08-11 In Machine Learning , Deep Learning
Symbols count in article: 1.7k Reading time ≈ 11 mins.

介绍

  • 论文：Distilling the Knowledge in a Neural Network
  • api：https://intellabs.github.io/distiller/install.html

问题：训练和部署的矛盾

  • 模型的需求是精度，所以导致：**1.**模型很复杂，计算资源占用大。**2.**实验时效果好的模型规模很大，甚至多个模型融合。
  • 部署的需求是：速度快，对资源要求低。

因此，模型压缩（在保证性能的前提下减少模型的参数量）成为了一个重要的问题。而”模型蒸馏“属于模型压缩的一种方法。

思想歧路

  • 一个误解: 人们在直觉上会觉得，要保留相近的效果，必须保留相近规模的模型。也就是说，一个模型的参数量基本决定其效果极限。
  • 但是
    • 模型的参数量和其所能捕获的“知识“量之间并非稳定的线性关系。
    • 完全相同的模型架构和模型参数量，使用完全相同的训练数据，模型的效果并不一定完全相同，另一个关键因素是训练的方法。

原理

Teacher—Student模型

知识蒸馏使用的是Teacher—Student模型，其中teacher是“知识”的输出者，student是“知识”的接受者。知识蒸馏的过程分为2个阶段:

  1. 原始模型训练:
    训练"Teacher模型", 简称为Net-T，它的特点是模型相对复杂，也可以由多个分别训练的模型集成而成。
    唯一的要求就是，对于输入X, 其都能输出Y，其中Y经过softmax的映射，输出值对应相应类别的概率值。
  2. 精简模型训练:
    训练"Student模型", 简称为Net-S，它是参数量较小、模型结构相对简单的单模型。
    同样的，对于输入X，其都能输出Y，Y经过softmax映射后同样能输出对应相应类别的概率值。

关键思想

  • 机器学习最根本的目的在于训练出在某类问题上泛化能力强的模型。
    因为我们不可能得到某问题的所有数据，所以训练目标变成在已有的训练数据集上建模输入和输出之间的关系。
    由于训练数据集是对真实数据分布情况的采样，训练数据集上的最优解往往会多少偏离真正的最优解。
  • 而在知识蒸馏时，由于我们已经有了一个泛化能力较强的Net-T，我们在利用Net-T来蒸馏训练Net-S时，可以直接让Net-S去学习Net-T的泛化能力。
    由此，如果让新模型Net-S近似原模型Net-T，因为原模型的函数是已知的，我们可以直接使用很多非训练集内的伪数据来训练新模型。
  • 在本论文中，作者将问题限定在分类问题下，即最后会有一个softmax层，其输出值对应了相应类别的概率值。
    所以可以直接使用softmax层输出的类别的概率来作为“soft target”，即让新模型与原模型在给定输入下的softmax分布匹配。

训练方式

  • 这里的训练过程并不单纯的是使用最大似然去让某个类别对应的概率最大，而是让整个softmax分布都相同。
    因为负标签也带有大量的信息，比如某些负标签对应的概率远远大于其他负标签。
    也就是说，KD的训练方式使得每个样本给Net-S带来的信息量大于传统的训练方式。
  • 在一般的softmax函数中，自然指数eee先拉大logits之间的差距，然后作归一化，其输出是一个接近one-hot的向量，其中一个值很大，其他的都很小。
    这样会导致一些负样本的值接近于0，就会对损失函数的贡献非常小，小到可以忽略不计。
    所以相较类似one-hot这样的硬性输出，我们更希望输出更「软」一些。
  • 于是文献提出一个新的softmax函数：qi=exp(zi/T)∑jexp(zj/T)q_i = \dfrac{exp(z_i/T)}{\sum_j exp(z_j/T)}qi​=∑j​exp(zj​/T)exp(zi​/T)​
    T表示温度。T越高，softmax的输出概率分布越平滑。如此一来，负标签携带的信息就能被放大，模型训练将更加关注负标签。

训练过程

  • 理论上：
    • 第一步：训练Net-T
    • 第二步：Net-T的softmax结果蒸馏到Net-S
  • 实际上：可以直接将两个模型都放到训练Net-T用到的training set中去，然后根据两者的输出构建损失函数（Net-T的结果为sotf target）。

Attention是否必须要有Softmax？

对于长度为n的序列，Attention的时间和空间复杂度都是 O(n2)O(n^2)O(n2) ，即在序列长度比较大时，会出现问题。
为此，可以通过修改Attention结构来减少复杂度。

剥离Softmax

  • 当前的自注意力机制形如 Attention(Q,K,V)=softmax(QKT)V Q,K,V∈Rn∗dAttention(Q,K,V) = softmax(QK^T)V\ Q,K,V \in R^{n*d}Attention(Q,K,V)=softmax(QKT)V Q,K,V∈Rn∗d 。
    n为序列长度，d为QKV矩阵维度，一般 n>dn>dn>d ，甚至 n>>dn>>dn>>d 。
    主要影响attention性能的就是sotfmax函数中的QKTQK^TQKT，这一步计算的复杂度为 O(n2)O(n^2)O(n2) 。
  • 而如果没有softmax层，则公式变为 Attention(Q,K,V)=QKTVAttention(Q,K,V) = QK^TVAttention(Q,K,V)=QKTV 。
    于是可以先计算 KTVK^TVKTV ，得到 d∗dd*dd∗d 的矩阵，然后左乘Q，这是一个 [n⋅d]⋅[d⋅d][n \cdot d]\cdot[d \cdot d][n⋅d]⋅[d⋅d] 的过程。
    时间复杂度为O(nd)O(nd)O(nd)，又因为n>>dn>>dn>>d，所以时间复杂度简化为 O(n)O(n)O(n) 。
  • softmax的作用：softmax的作用就是以eqiTkje^{q_i^Tk_j}eqiT​kj​为权重对 vjv_jvj​ 做平均加权。

增大Key_size

Attention的瓶颈

  • 问题：
    Attention中的权重分布 P=softmax(QKTdk)P=softmax(\frac{QK^T}{\sqrt{d_k}})P=softmax(dk​​QKT​) 表示Q与K的两两向量之间的联系。
    对长度为n的序列，在抽象空间中，P是一个二元联合分布/n个一元分布，有 n2n^2n2 个值。但是在表示空间中，在多头注意力机制中，每一个token的Q与K的参数量都为 d/hd/hd/h ，所以总的参数量为 2nd/h2nd/h2nd/h 。
    所以就是使用这 2nd/h2nd/h2nd/h 个参数去表示/逼近n2n^2n2个值。
    但是通常 2nd/h << n22nd/h\ <<\ n^22nd/h << n2 ，尤其是h比较大时，这就是论文[1]中描述的低秩瓶颈Low-Rank Bottleneck）。
  • 解法：
    • 增大 2nd/h2nd/h2nd/h ：减少h，会破坏模型的表达能力；增大d，会导致计算量加剧，所以不是很理想。
    • 增大 keysizekey_sizekeys​ize ：我们通常称Q,K的维度为key_size，V的维度才叫head_size，改变key_size的大小而不改变head_size的话，也不影响模型的hidden_size。
      所以论文[1]中提到增大模型的key_size，可以增加模型的表达能力，同时不改变模型整体的hidden_size。原论文其实是同时增大key_size和head_size，然后另外拼接一个降维矩阵，这样看来其实增加head_size就没有什么意义。

联系head

孤立的head运算

  • 问题：
    前文指出，因为key_size太小所以 QiKiTQ^i{K^i}^TQiKiT 表达能力不足，因此softmax之后无法很好地建议完整的二元分布。为了缓解这个问题，除了增大key_size之外，还有没有其他方法呢？
    论文[1]指出，当前的Multi-Head Attention每个head的运算是相互孤立的，而通过将它们联系（Talking）起来，则可以得到更强的Attention设计，即标题的“Talking-Heads Attention”。
  • 解法：
    这篇文章提到了混合分布：就是多个简单分布的叠加（比如加权平均），它能极大地增强原分布的表达能力。
    想要增加Attention中分布的表达能力，又不想增加key_size，那么可以考虑叠加多个低秩分布。
    即在 QKTQK^TQKT 之后、softmax之前”用一个参数矩阵λ将各个 QKTQK^TQKT 的结果叠加一下。
  • 实验：
    同时论文中大量实验表明，hidden_size不变的情况下，head数量越多（key_size和head_size都越小），使用混合分布后的效果越好。

Hierarchical Attention-based Recurrent Neural Network (HARNN)

论文：Hierarchical Multi-label Text Classification: An Attention-based Recurrent Network Approach
下载地址：https://base.ustc.edu.cn/pdf/Wei-Huang-CIKM2019.pdf
源码：https://github.com/RandolphVI/Hierarchical-Multi-Label-Text-Classification

介绍

  • 研究的现状：
    对于层级分类任务 Hierarchical multi-label text classification (HMTC)
    先前的研究：1. 当作单标签来做 2. 逐层训练分类器
    因而忽视了：1. 文本和层级标签的关系 2. 层级间的依赖关系（传统模式的弊端）

  • 任务的挑战：

    • 对于长文本，不同的句子可能被分为不同的标签，如图红句子和蓝句子。这也表示文档与层级结构是有关系的，即从文档得到层级结构。
    • 对于分类层次，上层标签决定下层标签。
    • 对于样本，在考虑具体标签同时，也要考虑全局标签层级结构。

  • 本文的创新：
    作者引入了一个 文档表示层 a documentation representing layer(DRL)，来整合文本和标签层次结果信息。
    然后作者引入一个基于层次注意力的递归层 an hierarchical attention-based recurrent layer (HARL)
    以一种自上而下的模式模拟标签不同层级之间的依赖关系 model the dependencies among different levels of the hierarchical structure in a top-down fashion。

    综上，作者创建了一种混合的方法，能够预测每个层次的标签，同时能够得到自上而下的标签层次。简而言之就是既能得到最终结果，也能得到完整的标签树。

HFT-CNN层次标签分类

Posted on 2021-08-01 In NLP , Text Classification
Symbols count in article: 832 Reading time ≈ 6 mins.

Paper：HFT-CNN: Learning Hierarchical Category Structure for Multi-label Short Text Categorization

介绍

  • 当前的短文本分类问题忽略了层次信息（non-hierarchical），带来了数据稀疏性问题，即低层次的标签信息忽略了其父类的样本信息。
  • 所以作者提出了一种方式，可以将高层的信息用于底层分类任务。
  • 作用：
    • 最大化标签信息的影响
    • 微调CNN，来学习层级信息。
    • 这个模型在短文本，多标签数据下效果显著。

Hierarchical Fine-Tuning based CNN / 基于层次微调的CNN

CNN architecture / CNN的结构

  • 第一层：word emebdding layer
    首先使用fasttext结构：
    单词向量长度为k
    使用 xi ∈ Rkx_i\ \in\ R^kxi​ ∈ Rk 表示句子中的第i个单词向量
    则整个长为n的句子被表示为 x1:n=[x1,x2,..xn] ∈ Rnkx_{1:n}=[x_1,x_2,..x_n]\ \in\ R^{nk}x1:n​=[x1​,x2​,..xn​] ∈ Rnk

  • 第二层：convolution layer
    使用卷积层 w ∈ Rhkw\ \in\ R^{hk}w ∈ Rhk （窗口大小为h个单词），得到一个新的特征cic_ici​：
    ci = f(w⋅xi:i+h−1+b) where b∈Rc_i\ =\ f(w \cdot x_{i:i+h-1}+b)\ where\ b\in Rci​ = f(w⋅xi:i+h−1​+b) where b∈R，f表示一个非线性激活函数，b表示一个偏秩。
    通过改变窗口大小，可以得到多个长度不同的特征结果。

  • 第三层：a max pooling layer
    将前面不同窗口大小得到的特征结果分别最大池化，然后作拼接得到本层特征向量。

  • 第四层：a fully connected layer with dropout

  • 第五层：一个二元分类器

  • 损失函数
    通过随机梯度下降法（SGD）最小化交叉熵损失函数（BCE），

Hierarchical structure learning / 层次结构学习

  • 关键思想：在CNN中使用微调技术来解决数据稀疏性问题，尤其是低层次的信息。
    即 将上层训练的CNN的参数转移到下层，然后对下层CNN的参数进行微调。
  • 这种方法可以有效地利用上层的数据来帮助下层的分类。

We note that fine-tuning the last few layers are usually suffificient for transfer learning as the last few layers become more specifific features.

  • 微调最后几层通常足以进行迁移学习，因为最后几层表示了更加具体的特征。然而，由深层组成的HS也需要对早期层进行微调，因为类别的上下层之间的距离非常大。
    这个不是很懂
  • 作者将前两层的参数，即embedding和卷积层迁移到新的模型上，来学习下层标签。
  • 在相邻层间循环这个过程。

Multi-label categorization / 多标签分类

  • 每一个样本都会被打分函数分配给一个标签。打分函数分为两种：

    • Boolean Scoring Function (BSF)：只有在文本被分到一级分类的情况下才会认为分类的二级分类是正确的。
    • Multiplicative Scoring Function (MSF)

  • 每一个打分函数同时有一个阀值设置，文本在某个分类的得分超过阈值则认为是该分类的。

思考

若以父类的特征进行初始化，理论上是保存了父类特征，但那些影响子类标签的差异性特征能学到吗，可能在父类标签学习的过程中就过滤掉或者弱化掉了这些差异性特征。不过这些仅是我的个人思考，读者可以结合自己的理解和实践去判断。

介绍

  • 目前的语言模型主流都是非监督预训练+监督微调，但是作者想构建一个泛化能力更强的，适用于多种任务的，甚至不需要去打标签做微调的模型。
  • 从作者的实验中可见：数据量够大和参数较多的情况下模型的效果是有一定的提升的（废话）。
  • 作者希望通过多任务学习 Multitask learning，提高语言模型的泛化能力。
  • 这篇论文会把多任务学习和非监督学习联系起来，展示的语言模型能在没有训练样本（没有任何参数和结构的修改）的情况下执行一些下游任务，并且在零样本的情况下泛化性能更强，在一些任务上我们也取得了业界最佳的效果。

方法

  • 语言建模，将某句话x的联合概率分解为其单词的条件概率的乘积：

p(x) = ∏i=1np(sn∣s1,...,sn−1)p(x)\ =\ \prod_{i=1}^{n} p(s_n|s_1,...,s_{n-1}) p(x) = i=1∏n​p(sn​∣s1​,...,sn−1​)

  • 学习某项单一的任务可以用概率学中的条件概率p(output∣input)p(output|input)p(output∣input)来表示，而对于多任务学习模型来说，同一个input可以根据任务不同输出多个答案，即p(output∣input,θtask)p(output|input,\theta_{task})p(output∣input,θtask​)
  • ?? 监督学习和非监督学习的目标是相同的，但是监督学习只能在子集上进行评估，非监督学习的全局最优解也是监督学习的全局最优解.
    但是非监督学习的训练速度慢很多。

数据

数据集合

输入表示

预处理，例如转小写，分词，处理oov问题，而将Unicode字符串处理为UTF-8字节序列可以很好地解决这一问题。

这种输入表示允许我们将word级语言模型的经验优势与char级方法的通用性结合起来。

模型

一种自回归模型，一次只输出一个单词。

  • 结构
    和GPT 1.0类似，使用带掩模的自注意力模型。
    输入一个单词序列，模型输出下一个可能的单词。
    GPT-2有一个top-k参数，从输出概率前k大的单词中抽样选取下一个单词，默认为1.
    移除了lay normation，在最后的self-attention模块中添加了additional layer normalization。
    考虑到模型深度对残差路径累加的影响，在初始化权重时将剩余层的权值乘以 1/N1/\sqrt{N}1/N​，其中N是残缺层的数量。
    词典被扩展到了50257，context的维度从512提高到了1024并且batchsize采用了512。

简介

  1. Unsupervised pre-training：在大量的无标注数据集合上进行无监督预训练
  2. Supervised fine-tuning：然后在标注数据集上进行监督微调

adapt the model to a discriminative task with labeled data.

这种使用监督学习和无监督学习的方法也称为半监督学习 Semi-supervised learning。

模型结构

Unsupervised pre-training

  • 使用：
    使用类似Transformer中的decoder的结构作为特征抽取器，使用单向的模型结构，多层结构（12层）。
  1. 特征抽取器

    GPT Decoder包含一层Masked Multi-head Attention和一层前馈神经网络。并且在每层后面都进行了残差连接和归一化操作（norm）。
    使用decoder的原因是因为需要根据上文来预测下文，同时避免了多层双向传播的数据泄露问题。
  2. 单向的模型结构
    设置了窗口大小k，即在预测单词i时，仅仅根据i前面k个单词进行预测。
  3. 多层结构：12层decoder
  • 训练：使用最大似然的思想和随机梯度下降的方法来训练模型
    对于输入文本序列U=[u1,u2,...un]U=[u_1,u_2,...u_n]U=[u1​,u2​,...un​]，根据最大似然思想，基于过去窗口中的单词来预测当前单词。整个文本序列的损失函数为：

L1(U) = ∑i=k+1nlogP(ui∣ui−k,...,ui−1;θ)L_1(U)\ =\ \sum_{i=k+1}^{n} logP(u_i|u_{i-k},...,u_{i-1};\theta) L1​(U) = i=k+1∑n​logP(ui​∣ui−k​,...,ui−1​;θ)

k：窗口大小
θ\thetaθ：网络参数

  • 训练目标：每个位置输出的向量和正确答案向量（原词词向量）的相似度最大
    对于每个单词，其词向量由embedding和位置编码组成。

Supervised fine-tuning

在训练好模型后，我们增加一些标签对模型进行微调来适应下游任务。
使用C^\hat CC^表示样本库，其中每个样本都包含一个序列X=[x1,x2,..,x3]X=[x_1,x_2,..,x_3]X=[x1​,x2​,..,x3​]，即输入的文本，同时对应一个标签yyy。

Task-specific input transformations

不同的任务需要不同结构的输入，

Previous work proposed learning task specific architectures on top of transferred representations.
Such an approach reintroduces a significant amount of task-specific customization and does not use transfer learning for these additional architectural components.

因为我们预先训练的模型是在连续的文本序列上训练的，所以我们需要一些转换来将它应用到这些任务中。
所有的转换都包括添加随机初始化的开始和结束标记(<s><s><s>，<e><e><e>)

  1. 对于分类问题，不用怎么动，加上一个起始和终结符号即可。
  2. 对于句子关系判断问题，比如 Entailment，两个句子中间再加个分隔符即可。
  3. 对文本相似性判断问题，把两个句子顺序颠倒下做出两个输入即可，这是为了告诉模型句子顺序不重要。
  4. 对于多项选择问题，则多路输入，每一路把文章和答案选项拼接作为输入即可。

Deep Context

Posted on 2021-06-30 In NLP
Symbols count in article: 1.6k Reading time ≈ 10 mins.

论文：Deep Context Model for Grammatical Error Correction

简介

使用RNN进行文本纠错，将纠错任务（error correction task）当作一个分类任务。
这个模型不需要复杂的特征工程，不需要预先得到全部的上下文信息。

  • GEC（grammatical error correction）语法纠错任务常用到三种方法：
    • 预定义规则 pre-defined rules
      弊端：无法涵盖所有的语法错误
    • 分类 classification
      将GEC看成一个多分类问题，一个混淆集代表一个特定的错误，样本特征包含文本的表层表示和语言抽象表示（比如词性标签和依存语法）。
      可以使用最大熵分类，平均感知机，朴素贝叶斯等分类器先确定错误，再修改错误。
      弊端：样本特征需要被人工设计，但是难以考虑到所有的情况。
    • 机器翻译/转换 machine translation（MT）
      • statistical machine translation
        将错误的文本转换为正确的文本。
        优点：
        直接利用大规模语料
        可以处理复杂的错误
        缺点：基于短语的机器转换方法存在字的离散性、线性映射和对全局上下文不敏感等问题。
      • neural machine translation (NMT)
        优点：
        应对分类方法无法处理的冗余和非惯用语法错误
        处理 符合多种错误类型结合的复杂错误
        1 encoder-decoder框架
        2 bidirectional LSTMs
        用于语法错误检测（ grammatical error detection tasks）任务
  • 为什么使用分类器：
    • 不需要标记数据，而MT需要。

    does not rely on annotated learner data which are expensive but required by most MT approaches.

    • 更方便组合文本深层和浅层信息
  • 这篇论文介绍一种基于双向GRU（bidirectional Gated Recurrent Units）的深层文本模型（a deep context model）。
    与传统的分类起相比：
    • 直接使用文本的深层上下文特征。
      深度上下文特征以端到端的方式从大型纯文本语料库中学习。
    • 并且不需要对每种错误逐一设计特征。

模型

对于一个特定的错误类别，模型通过目标单词的上下文进行预测，如果预测词和原词不同则判定为原词错误，预测词即为纠错结果。
DeepContext模型使用了双向GRU，并且基于context2vec结构构建。
上下文表示可以从句子首尾开始训练，或者在指定的一个窗口中进行训练。

预测

  1. 给定一个上下文文本表示W=[w1,w2,..,wn]W=[w_1,w_2,..,w_n]W=[w1​,w2​,..,wn​]，设待判断的单词位置为i，则该词的上下文词向量wiw_iwi​为：

biGRU(wi:n,i)=lGRU(w1:i−1)⊕rGRU(wi+1:n)biGRU(w_{i:n},i)=lGRU(w_{1:i-1}) \oplus rGRU(w_{i+1:n}) biGRU(wi:n​,i)=lGRU(w1:i−1​)⊕rGRU(wi+1:n​)

即只考虑左侧信息得到的预测和只考虑右侧信息得到的预测的拼接。

  1. 然后将这个拼接词向量交给一个多层感知机 multi-layer perceptron (MLP)

feed the concatenated vector to the multi-layer perceptron (MLP) to capture the inter-dependencies of the two sides

多层感知机的输出为估计词词向量，使用估计词词向量做softmax得到预测词。
注意：在放进softmax之前可以使用Relu函数激活一下，这样可以有效减小计算量。

MLP(x)=softmax(ReLU(L(x)))MLP(x) = softmax(ReLU(L(x))) MLP(x)=softmax(ReLU(L(x)))

MLP：多层感知机
L(x)=Wx+bL(x)=Wx+bL(x)=Wx+b：全连接线性操作
softmax：表示一个softmax层，这里个人理解为输出的也是一个估计词词向量，和输入向量尺寸相同，然后使用softmax找到概率最大的单词，输出这个单词的词向量表示。

  • 所以模型最终输出为：y=MLP(biGRU(w1:n,i))y=MLP(biGRU(w_{1:n}, i))y=MLP(biGRU(w1:n​,i))
    y可以是目标单词的预测单词词向量或者是预测单词状态向量，
    预测词：如果y和对应位置输入相同，则判定为正确，否则则为错误，并且将y看作纠错结果。
    预测词状态/形式：y放入分类器中，得到预测类别/词形式，和原词作对比。

  • 在得到y之后，将其输入到一个分类器中，输出y^\hat yy^​，不同错误类型的分类器不同。

the probability is larger than the predefined threshold

这篇论文主要包含5个错误类型：
- 冠词纠错模型：0-a;1-an;2-the/非冠词
- 介词纠错模型：y表示每个介词（in , from , to，at …）的索引位置
- 动词纠错模型：0-动词原形；1-动名词或现在分词；2-动词过去式
- 主谓一致（人称）纠错模型：0-非第三人称单数，1-第三人称单数

“Efficiently Learning an Encoder that Classifies Token Replacements Accurately”

概述

  • BERT中的MLM（mask language model）使用[mask]标签来替换一些（15%）单词，然后训练一个模型去预测/重新生成这些单词。
    因为每次都只学习了15%的特征，所以训练代价很大。

  • 所以这里引入一个高效的方法RTD（replaced token detection）

    • 使用小型的生成模型生成单词，替换掉一些输入单词.
    • 然后使用一个判别模型判断句子中的每个单词是生成词还是原词。

    predict for every token whether it is an original or a replacement

  • 微调时只训练判别器

  • 这种判别的方式同时也可以解决BERT中mismatch的问题：训练时使用了[mask]，但是在微调时不会使用mask。

  • 在transformer的结果上继续训练。？？是否可以看作transformer的微调？？

we apply it to pre-train Transformer text encoders that can be fine-tuned on downstream tasks.

结构

Generator

  • 用于生成/预测一个单词，通常使用一个小型的神经网络。

    E暂时没搞懂
    • 和GAN不同，生成器的训练目标是努力生成一个real样本，而不是努力生成一个fake样本来骗过识别器。

联合损失函数：

minθG,θD = ∑x∈TLMLM(x,θG)+λLDisc(x,θD)min_{\theta_G,\theta_D}\ =\ \sum_{x\in T} L_{MLM}(x,\theta_G)+\lambda L_{Disc}(x,\theta_D) minθG​,θD​​ = x∈T∑​LMLM​(x,θG​)+λLDisc​(x,θD​)

T表示原始文本：在原始文本上选择单词序列/句子来进行训练。
最终是最小化联合损失（minimize the combined loss）。

  • 这两个神经网络是分别训练的，即判别器的损失不会传给生成器。
  • 生成器更新所有的token embedding，而判别器只更新被sample的embedding。
  • 预训练之后，继续单独对判别器进行微调。

模型拓展选择

  • small generator：
    • 生成器不能太厉害，否则一定会预测出正确的单词，这样判别器只需要判别正确即可。
      所以需要使用一个小型的生成器，当生成器的大小为判别器大小的1/2-1/4时，模型效果最好。

训练算法

上述的训练方式是训练联合损失。此外，论文中提到了两种其他的训练方式，但是都不如原本的训练方式好：

  • two stage training:
    这种方式只适用于生成器和判别器大小相同的情况。
    • 首先只训练MLM模型n次
    • 然后使用MLM的参数初始化判别器，冻结MLM，单独训练判别器n次。
      如果不初始化，则会导致出题人太强而答题人训练失败的情况。
  • 借鉴GAN的思想，生成器的目标改为生成一个最有可能的赝品。
