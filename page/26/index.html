Hierarchical Attention-based Recurrent Neural Network (HARNN)

论文：Hierarchical Multi-label Text Classification: An Attention-based Recurrent Network Approach
下载地址：https://base.ustc.edu.cn/pdf/Wei-Huang-CIKM2019.pdf
源码：https://github.com/RandolphVI/Hierarchical-Multi-Label-Text-Classification

介绍

  • 研究的现状：
    对于层级分类任务 Hierarchical multi-label text classification (HMTC)
    先前的研究：1. 当作单标签来做 2. 逐层训练分类器
    因而忽视了：1. 文本和层级标签的关系 2. 层级间的依赖关系（传统模式的弊端）

  • 任务的挑战：

    • 对于长文本，不同的句子可能被分为不同的标签，如图红句子和蓝句子。这也表示文档与层级结构是有关系的，即从文档得到层级结构。
    • 对于分类层次，上层标签决定下层标签。
    • 对于样本，在考虑具体标签同时，也要考虑全局标签层级结构。

  • 本文的创新：
    作者引入了一个 文档表示层 a documentation representing layer(DRL)，来整合文本和标签层次结果信息。
    然后作者引入一个基于层次注意力的递归层 an hierarchical attention-based recurrent layer (HARL)
    以一种自上而下的模式模拟标签不同层级之间的依赖关系 model the dependencies among different levels of the hierarchical structure in a top-down fashion。

    综上，作者创建了一种混合的方法，能够预测每个层次的标签，同时能够得到自上而下的标签层次。简而言之就是既能得到最终结果，也能得到完整的标签树。

HFT-CNN层次标签分类

Posted on 2021-08-01 In NLP , Text Classification
Symbols count in article: 832 Reading time ≈ 4 mins.

Paper：HFT-CNN: Learning Hierarchical Category Structure for Multi-label Short Text Categorization

介绍

  • 当前的短文本分类问题忽略了层次信息（non-hierarchical），带来了数据稀疏性问题，即低层次的标签信息忽略了其父类的样本信息。
  • 所以作者提出了一种方式，可以将高层的信息用于底层分类任务。
  • 作用：
    • 最大化标签信息的影响
    • 微调CNN，来学习层级信息。
    • 这个模型在短文本，多标签数据下效果显著。

Hierarchical structure learning / 层次结构学习

  • 关键思想：在CNN中使用微调技术来解决数据稀疏性问题，尤其是低层次的信息。
    即 将上层训练的CNN的参数转移到下层，然后对下层CNN的参数进行微调。
  • 这种方法可以有效地利用上层的数据来帮助下层的分类。

We note that fine-tuning the last few layers are usually suffificient for transfer learning as the last few layers become more specifific features.

  • 微调最后几层通常足以进行迁移学习，因为最后几层表示了更加具体的特征。然而，由深层组成的HS也需要对早期层进行微调，因为类别的上下层之间的距离非常大。
    这个不是很懂
  • 作者将前两层的参数，即embedding和卷积层迁移到新的模型上，来学习下层标签。
  • 在相邻层间循环这个过程。

Multi-label categorization / 多标签分类

  • 每一个样本都会被打分函数分配给一个标签。打分函数分为两种：

    • Boolean Scoring Function (BSF)：只有在文本被分到一级分类的情况下才会认为分类的二级分类是正确的。
    • Multiplicative Scoring Function (MSF)

  • 每一个打分函数同时有一个阀值设置，文本在某个分类的得分超过阈值则认为是该分类的。

思考

若以父类的特征进行初始化，理论上是保存了父类特征，但那些影响子类标签的差异性特征能学到吗，可能在父类标签学习的过程中就过滤掉或者弱化掉了这些差异性特征。不过这些仅是我的个人思考，读者可以结合自己的理解和实践去判断。

GPT 2.0

Posted on 2021-07-03 In NLP , Language Model
Symbols count in article: 891 Reading time ≈ 4 mins.

介绍

  • 目前的语言模型主流都是非监督预训练+监督微调，但是作者想构建一个泛化能力更强的，适用于多种任务的，甚至不需要去打标签做微调的模型。
  • 从作者的实验中可见：数据量够大和参数较多的情况下模型的效果是有一定的提升的（废话）。
  • 作者希望通过多任务学习 Multitask learning，提高语言模型的泛化能力。
  • 这篇论文会把多任务学习和非监督学习联系起来，展示的语言模型能在没有训练样本（没有任何参数和结构的修改）的情况下执行一些下游任务，并且在零样本的情况下泛化性能更强，在一些任务上我们也取得了业界最佳的效果。

方法

  • 语言建模，将某句话x的联合概率分解为其单词的条件概率的乘积：

p(x) = ∏i=1np(sn∣s1,...,sn−1)p(x)\ =\ \prod_{i=1}^{n} p(s_n|s_1,...,s_{n-1}) p(x) = i=1∏n​p(sn​∣s1​,...,sn−1​)

  • 学习某项单一的任务可以用概率学中的条件概率p(output∣input)p(output|input)p(output∣input)来表示，而对于多任务学习模型来说，同一个input可以根据任务不同输出多个答案，即p(output∣input,θtask)p(output|input,\theta_{task})p(output∣input,θtask​)
  • ?? 监督学习和非监督学习的目标是相同的，但是监督学习只能在子集上进行评估，非监督学习的全局最优解也是监督学习的全局最优解.
    但是非监督学习的训练速度慢很多。

输入表示

预处理，例如转小写，分词，处理oov问题，而将Unicode字符串处理为UTF-8字节序列可以很好地解决这一问题。

模型

一种自回归模型，一次只输出一个单词。

  • 结构
    和GPT 1.0类似，使用带掩模的自注意力模型。
    输入一个单词序列，模型输出下一个可能的单词。
    GPT-2又一个top-k参数，从输出概率前k大的单词中抽样选取下一个单词，默认为1.
    移除了lay normation，在最后的self-attention模块中添加了additional layer normalization。
    考虑到模型深度对残差路径累加的影响，在初始化权重时将剩余层的权值乘以 1/N1/\sqrt{N}1/N​，其中N是残缺层的数量。
    词典被扩展到了50257，context的维度从512提高到了1024并且batchsize采用了512。

Unsupervised pre-training

  • 使用：
    使用类似Transformer中的decoder的结构作为特征抽取器，使用单向的模型结构，多层结构（12层）。
  1. 特征抽取器

    GPT Decoder包含一层Masked Multi-head Attention和一层前馈神经网络。并且在每层后面都进行了残差连接和归一化操作（norm）。

    设置了窗口大小k，即在预测单词i时，仅仅根据i前面k个单词进行预测。

Task-specific input transformations

不同的任务需要不同结构的输入，

Previous work proposed learning task specific architectures on top of transferred representations.
Such an approach reintroduces a significant amount of task-specific customization and does not use transfer learning for these additional architectural components.

因为我们预先训练的模型是在连续的文本序列上训练的，所以我们需要一些转换来将它应用到这些任务中。
所有的转换都包括添加随机初始化的开始和结束标记(<s><s><s>，<e><e><e>)

  1. 对于分类问题，不用怎么动，加上一个起始和终结符号即可。
  2. 对于句子关系判断问题，比如 Entailment，两个句子中间再加个分隔符即可。
  3. 对文本相似性判断问题，把两个句子顺序颠倒下做出两个输入即可，这是为了告诉模型句子顺序不重要。
  4. 对于多项选择问题，则多路输入，每一路把文章和答案选项拼接作为输入即可。

论文：Deep Context Model for Grammatical Error Correction

简介

使用RNN进行文本纠错，将纠错任务（error correction task）当作一个分类任务。
这个模型不需要复杂的特征工程，不需要预先得到全部的上下文信息。

  • GEC（grammatical error correction）语法纠错任务常用到三种方法：
    • 预定义规则 pre-defined rules
      弊端：无法涵盖所有的语法错误
    • 分类 classification
      将GEC看成一个多分类问题，一个混淆集代表一个特定的错误，样本特征包含文本的表层表示和语言抽象表示（比如词性标签和依存语法）。
      可以使用最大熵分类，平均感知机，朴素贝叶斯等分类器先确定错误，再修改错误。
      弊端：样本特征需要被人工设计，但是难以考虑到所有的情况。
    • 机器翻译/转换 machine translation（MT）
      • statistical machine translation
        将错误的文本转换为正确的文本。
        优点：
        直接利用大规模语料
        可以处理复杂的错误
        缺点：基于短语的机器转换方法存在字的离散性、线性映射和对全局上下文不敏感等问题。
      • neural machine translation (NMT)
        优点：
        应对分类方法无法处理的冗余和非惯用语法错误
        处理 符合多种错误类型结合的复杂错误
        1 encoder-decoder框架
        2 bidirectional LSTMs
        用于语法错误检测（ grammatical error detection tasks）任务
  • 为什么使用分类器：
    • 不需要标记数据，而MT需要。

    does not rely on annotated learner data which are expensive but required by most MT approaches.

    • 更方便组合文本深层和浅层信息
  • 这篇论文介绍一种基于双向GRU（bidirectional Gated Recurrent Units）的深层文本模型（a deep context model）。
    与传统的分类起相比：
    • 直接使用文本的深层上下文特征。
      深度上下文特征以端到端的方式从大型纯文本语料库中学习。
    • 并且不需要对每种错误逐一设计特征。

模型

对于一个特定的错误类别，模型通过目标单词的上下文进行预测，如果预测词和原词不同则判定为原词错误，预测词即为纠错结果。
DeepContext模型使用了双向GRU，并且基于context2vec结构构建。
上下文表示可以从句子首尾开始训练，或者在指定的一个窗口中进行训练。

biGRU(wi:n,i)=lGRU(w1:i−1)⊕rGRU(wi+1:n)biGRU(w_{i:n},i)=lGRU(w_{1:i-1}) \oplus rGRU(w_{i+1:n}) biGRU(wi:n​,i)=lGRU(w1:i−1​)⊕rGRU(wi+1:n​)

概述

  • BERT中的MLM（mask language model）使用[mask]标签来替换一些（15%）单词，然后训练一个模型去预测/重新生成这些单词。
    因为每次都只学习了15%的特征，所以训练代价很大。

  • 这种判别的方式同时也可以解决BERT中mismatch的问题：训练时使用了[mask]，但是在微调时不会使用mask。

  • 在transformer的结果上继续训练。？？是否可以看作transformer的微调？？

FastText是从cBow（continuous bag of words）演化而来的：

  • 区别：

    • fasttext输出的是文本的标签概率向量，输入的是待分类的文本特征列表。这个特征有多种表示方法：embedding,n-gram,tf-idf…
      无论是输入one-hot然后通过embedding得到词向量，还是直接输入随机初始化的embedding，本质上都是一样的，都是随机初始化。
    • cbow和fasttext的词向量都是随机初始化的。

结构：


和cbow一样，也是三层设计：输入层，隐含层和输出层。
输入多个特征向量，然后叠加取平均，成为文本向量进入隐藏层，然后输入到一个线性分类器中，结果通过softmax得到分类概率向量。

Model architecture of fastText for a sentence with N ngram features x1, . . . , xN.
The features are embedded and averaged to form the hidden variable.
The word representations are then averaged into a text representation, which is in turn fed to a linear classifier.

n-gram

n-gram是锦上添花的东西，产生的仅仅是文本特征的候选集，不是fasttext必须的。

  • 几个概念/分类：

    • Bow：bag of words 词袋模型
    • CBow：continuous Bow，考虑了连续的分布式特征。
    • (word) n-gram：可以保留小范围的部分词序

    use a bag of n-grams as additional features to capture some partial information about the local word order.

    英文 2-gram：apple pie，pie is，is good
    中文 2-gram：苹果派｜是，是｜好吃，好吃｜的

    • character-level n-gram：解决oov和一些单词形态学的概念，低频词的词向量效果也会更好。
      英文 ‘<ap’,‘app’,‘ppl’…
      中文 苹果派，果派是，派是好 …

  • 在输入到fastText时，通常可以将字符级别的n-gram向量作为额外的输入。
    训练时，可以顺便得到字符级别的n-gram的向量，然后字符级向量叠加得到单词的向量。

  • 优点：(补充上面的优点）char n-gram 提高了句法词类比任务的性能
    比如单复数，不同的形式，内部多数字母/汉字还是没有改变的。

  • 缺点：char n-gram 降低了语义类比任务的性能
    不同的单词会有相同的内部字母/汉字，

Hierarchical Softmax

简介：

二分类问题是一种特殊的多分类问题，所以可以将多分类问题拆解成多个嵌套的二分类问题简化计算。
- 当需要预测的文本类别过多时，计算量会特别大：类别数量为K，文本向量维度为h，则时间复杂度为O(Kh)O(Kh)O(Kh)
线性分类器就是多个特征/类别文本表示向量的矩阵，本文向量逐个和特征文本向量做点积求相似度，然后softmax找到相似度最高的得到本文类别。
文本向量长度为h，所以点积操作的时间复杂度为O(h)O(h)O(h)，共有k个特征文本向量，所以作k次点积，softmax的时间复杂度为O(k)。
- 使用层次softmax来减少训练时间

表示：


- 根节点：输入的待分类的向量
- 叶节点：目标类
- 内部节点：表示一个参数向量，看作子树的根节点。
同样可以看作一个感知机/神经元，根据相似度输出走向左叶节点的概率。(哈夫曼树的思想：频率->概率大的放在左边, append 0)
- 根节点到叶节点的一条路径：表示叶节点表示的类别的编码，即路径上所有二分类问题的结果。
是用概率来得到编码，即路径，所以可以用编码反推的概率连乘得到叶节点的概率。
- 边：{0,1}，向左走用0表示，向右走用1表示。
- 目标类别的概率：从root结点开始，走到目标类别（叶节点）的概率。
因此在途中路过非叶子结点（包括root）时，需要分别知道往左走和往右走的概率。
