混合精度训练

  • 在训练过程中的每一个step，为模型的所有权重维护一个FP32（单精度浮点格式）的副本，称为主权重 Master Weights；
    在做前向和后向传播过程中，Master Weights会转换成FP16（半精度浮点数）格式，其中权重、激活函数和梯度都是用FP16进行表示，最后梯度会转换成FP32格式去更新主权重。
  • 提高训练速度，计算float16比float32快，使训练速度提高2-3倍，同时也减少模型的空间占用，从而可以利用较大的batch size。

LAMB优化器

Batch size 存在一定的隐式上限，一旦突破这个上限，梯度更新极端的取值会导致自适应学习率调整收敛困难，从而无法享受增加的 batch size 带来的提速增益。
LAMB优化器通过一个自适应式的方式为每个参数调整learning rate，能够在Batch Size很大的情况下不损失模型的效果，使得模型训练能够采用很大的Batch Size，进而极大提高训练速度。

{L+=(1−p)γlog⁡(p)L−=pγlog⁡(1−p)\left\{\begin{array}{l} L_{+}=(1-p)^\gamma \log (p) \\ L_{-}=p^\gamma \log (1-p) \end{array}\right. {L+​=(1−p)γlog(p)L−​=pγlog(1−p)​

对抗学习

  • 对抗训练有两个作用，一是提高模型对恶意攻击的鲁棒性，二是提高模型的泛化能力。
    在CV任务，根据经验性的结论，对抗训练往往会使得模型在非对抗样本上的表现变差，然而神奇的是，在NLP任务中，模型的泛化能力反而变强了。
    因此，在NLP任务中，对抗训练的角色不再是为了防御基于梯度的恶意攻击，反而更多的是作为一种regularization，提高模型的泛化能力。

  • 过程：
    给样本增加一些干扰，生成一个新的扰动样本，然后放给模型训练，这样模型就有了识别对抗样本的能力，即增强了模型的识别性。

  • NLP中的应用算法：

    • 基本单步算法FGM，“一阶扰动”最强多步算法PGD，以及针对时耗等改进的FreeAT，YOPO和FreeLB，其中FreeLB成为了目前刷榜的SOTA。
    • 用一句话形容对抗训练的思路，就是在输入上进行梯度上升(增大loss)，在参数上进行梯度下降(减小loss)。由于输入会进行embedding lookup，所以实际的做法是在embedding table上进行梯度上升。
    • 后续方法优化的主要方向有两点：得到更优的扰动 & 提升训练速度

AR模型和AE模型

AR模型

AR是自回归模型（AutoRegressive LM），对语料的分布进行建模。是一种使用上下文词来预测下一个词的模型。但是在这里，只能选择一个方向。

  • 例子：
    ELMo也是一个AR模型，不过是两个AR模型拼接得到；而GPT则是最典型的的AR模型。
  • 公式：
    对于给定序列：X=(x1,x2,...xn)X=(x_1,x_2,...x_n)X=(x1​,x2​,...xn​)
    序列的似然值为：P(X)=∑(t=1)Tp(xt∣x<t)P(X)=\sum_{(t=1)}^T p(x_t|x_{< t})P(X)=∑(t=1)T​p(xt​∣x<t​) 或 P(X)=∑(t=1)Tp(xt∣x>t)P(X)=\sum_{(t=1)}^T p(x_t|x_{>t})P(X)=∑(t=1)T​p(xt​∣x>t​)
  • 缺点：只能利用上文或者下文的信息，不能同时利用上文和下文的信息。
  • 优点：符合下游NLP任务语言环境，比如生成类NLP任务，比如文本摘要，机器翻译等，就是从左向右的，自回归语言模型天然匹配这个过程。

AE模型

AE是自编码语言模型（AutoEncoder LM），从被破坏的输入中恢复原始数据，所以能看到被预测单词的上下文。

  • Denoising Autoencoder（DAE）：Bert为例，被Mask掉的单词就是在输入侧加入的噪音。
  • 缺点：引入人为误差[mask]，从而导致mismatch问题。
  • 优点：比较自然地融入双向语言模型，同时看到被预测单词的上下文。

排列语言建模（Permutation Language Modeling）

流程仍然是Pre-Train和Fine-Tunning

  • 目标：
    依然是从左到右输入，在预测一个单词时，输入的是其上文Context_before，不过在上文Context_before中同时也包含了下文的信息。

知识蒸馏

Posted on 2021-08-11 In Machine Learning , Deep Learning
Symbols count in article: 1.7k Reading time ≈ 8 mins.

介绍

  • 论文：Distilling the Knowledge in a Neural Network
  • api：https://intellabs.github.io/distiller/install.html

问题：训练和部署的矛盾

  • 模型的需求是精度，所以导致：**1.**模型很复杂，计算资源占用大。**2.**实验时效果好的模型规模很大，甚至多个模型融合。
  • 部署的需求是：速度快，对资源要求低。

因此，模型压缩（在保证性能的前提下减少模型的参数量）成为了一个重要的问题。而”模型蒸馏“属于模型压缩的一种方法。

思想歧路

  • 一个误解: 人们在直觉上会觉得，要保留相近的效果，必须保留相近规模的模型。也就是说，一个模型的参数量基本决定其效果极限。
  • 但是
    • 模型的参数量和其所能捕获的“知识“量之间并非稳定的线性关系。
    • 完全相同的模型架构和模型参数量，使用完全相同的训练数据，模型的效果并不一定完全相同，另一个关键因素是训练的方法。

原理

Teacher—Student模型

知识蒸馏使用的是Teacher—Student模型，其中teacher是“知识”的输出者，student是“知识”的接受者。知识蒸馏的过程分为2个阶段:

  1. 原始模型训练:
    训练"Teacher模型", 简称为Net-T，它的特点是模型相对复杂，也可以由多个分别训练的模型集成而成。
    唯一的要求就是，对于输入X, 其都能输出Y，其中Y经过softmax的映射，输出值对应相应类别的概率值。
  2. 精简模型训练:
    训练"Student模型", 简称为Net-S，它是参数量较小、模型结构相对简单的单模型。
    同样的，对于输入X，其都能输出Y，Y经过softmax映射后同样能输出对应相应类别的概率值。

关键思想

  • 机器学习最根本的目的在于训练出在某类问题上泛化能力强的模型。
    因为我们不可能得到某问题的所有数据，所以训练目标变成在已有的训练数据集上建模输入和输出之间的关系。
    由于训练数据集是对真实数据分布情况的采样，训练数据集上的最优解往往会多少偏离真正的最优解。
  • 而在知识蒸馏时，由于我们已经有了一个泛化能力较强的Net-T，我们在利用Net-T来蒸馏训练Net-S时，可以直接让Net-S去学习Net-T的泛化能力。
    由此，如果让新模型Net-S近似原模型Net-T，因为原模型的函数是已知的，我们可以直接使用很多非训练集内的伪数据来训练新模型。
  • 在本论文中，作者将问题限定在分类问题下，即最后会有一个softmax层，其输出值对应了相应类别的概率值。
    所以可以直接使用softmax层输出的类别的概率来作为“soft target”，即让新模型与原模型在给定输入下的softmax分布匹配。

训练方式

  • 这里的训练过程并不单纯的是使用最大似然去让某个类别对应的概率最大，而是让整个softmax分布都相同。
    因为负标签也带有大量的信息，比如某些负标签对应的概率远远大于其他负标签。
    也就是说，KD的训练方式使得每个样本给Net-S带来的信息量大于传统的训练方式。
  • 在一般的softmax函数中，自然指数eee先拉大logits之间的差距，然后作归一化，其输出是一个接近one-hot的向量，其中一个值很大，其他的都很小。
    这样会导致一些负样本的值接近于0，就会对损失函数的贡献非常小，小到可以忽略不计。
    所以相较类似one-hot这样的硬性输出，我们更希望输出更「软」一些。
  • 于是文献提出一个新的softmax函数：qi=exp(zi/T)∑jexp(zj/T)q_i = \dfrac{exp(z_i/T)}{\sum_j exp(z_j/T)}qi​=∑j​exp(zj​/T)exp(zi​/T)​
    T表示温度。T越高，softmax的输出概率分布越平滑。如此一来，负标签携带的信息就能被放大，模型训练将更加关注负标签。

训练过程

  • 理论上：
    • 第一步：训练Net-T
    • 第二步：Net-T的softmax结果蒸馏到Net-S
  • 实际上：可以直接将两个模型都放到训练Net-T用到的training set中去，然后根据两者的输出构建损失函数（Net-T的结果为sotf target）。

增大Key_size

Attention的瓶颈

  • 问题：
    Attention中的权重分布 P=softmax(QKTdk)P=softmax(\frac{QK^T}{\sqrt{d_k}})P=softmax(dk​​QKT​) 表示Q与K的两两向量之间的联系。

联系head

孤立的head运算

  • 问题：
    前文指出，因为key_size太小所以 QiKiTQ^i{K^i}^TQiKiT 表达能力不足，因此softmax之后无法很好地建议完整的二元分布。为了缓解这个问题，除了增大key_size之外，还有没有其他方法呢？
    论文[1]指出，当前的Multi-Head Attention每个head的运算是相互孤立的，而通过将它们联系（Talking）起来，则可以得到更强的Attention设计，即标题的“Talking-Heads Attention”。
  • 解法：
    这篇文章提到了混合分布：就是多个简单分布的叠加（比如加权平均），它能极大地增强原分布的表达能力。
    想要增加Attention中分布的表达能力，又不想增加key_size，那么可以考虑叠加多个低秩分布。
    即在 QKTQK^TQKT 之后、softmax之前”用一个参数矩阵λ将各个 QKTQK^TQKT 的结果叠加一下。
  • 实验：
    同时论文中大量实验表明，hidden_size不变的情况下，head数量越多（key_size和head_size都越小），使用混合分布后的效果越好。
