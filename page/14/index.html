官方文档：https://keras.io/zh/layers/core/#dropout

keras.layers.Dropout(rate, noise_shape=None, seed=None)

  • 作用：在训练中每次更新时， 将输入单元的按比率随机设置为 0， 这有助于防止过拟合。
  • 参数：
    • rate: 在 0 和 1 之间浮动。需要丢弃的输入比例。
    • noise_shape: 1D 整数张量， 表示将与输入相乘的二进制 dropout 掩层的形状。
      例如，如果你的输入尺寸为 (batch_size, timesteps, features)，然后希望dropout在所有时间步都是一样的，可以使用 noise_shape=(batch_size, 1, features)。
    • seed: 一个作为随机种子的 Python 整数。

首先来看一个基本的正向的基于RNN网络的语言模型

RNN-based language models：

  • 首先讲解前向语言模型，即根据前文来预测下一个单词，是一个自回归语言模型。
  • 目标公式：p(t1,t2,...,tN)=∑k=1Np(tk∣t1,t2,...,tk−1)p(t_1, t_2,...,t_N) = \sum_{k=1}^N p(t_k|t_1,t_2,...,t_{k-1})p(t1​,t2​,...,tN​)=∑k=1N​p(tk​∣t1​,t2​,...,tk−1​)
  1. 首先生成一个[n*1]的词向量，可以使用word2vec，在论文中作者使用了cnn-big-lstm。这个词向量对应于lstm中的输入值。
  2. 将上一时刻的隐藏态ht−1h_{t-1}ht−1​和第一步中的word embedding一并输入到lstm中，得到输出和隐状态[m*1]的hth_tht​。

ELMo：

  • 目标：一个预训练的词表示应该能够包含丰富的句法和语义信息，并且能够对多义词进行建模。

结构

由encoder和decoder组成

Encoder

  • 在论文中，Encoder由6个相同的layer组成，即左边的部分。
  • 每个layer由两个sub-layer组成，分别为多头注意力机制multi-head self-attention mechanism和全连接前馈网络fully connected feed-forward network。

multi-head self-attention

multi-head self-attention中，通过 h组(h头) 不同的线性变化（Q，K，V）得到h组解，然后拼接起来。这里采用scaled dot-product方法计算attention的值：
词向量与相同的QKV矩阵乘得到对应的三个矩阵。

Attention(Q,K,V)=softmax(QKTdkV)Attention(Q,K,V)=softmax(\frac{QK^T}{\sqrt{d_k}}V) Attention(Q,K,V)=softmax(dk​​QKT​V)

Q表示所需计算的注意力项，K为被计算注意力项，V为被注意力项的取值。

可以看到，Q和K的尺寸是相同的，得出结果为相似度矩阵，矩阵的第[i,j]项表示Q的第i行和K的第j行的点积相似度（投影相似度），Query和key的关联强度。
scorei,jscore_{i,j}scorei,j​表示第i个单词和第j个单词的相似度得分。
然后除以一个标量，然后对权重矩阵进行mask处理！！。这里为什么要用mask？尺寸可以对得上啊！
（标量的作用：1. qk相乘值很大时，softmax的梯度就会很小，然后消失。2.方差控制为1。3.softmax中指数函数会将数值的差距拉大，所以在softmax前进行等比例缩小。）

再通过softmax进行映射，得到的就是相似度得分。
最后用相似度得分乘注意力项的取值得到最终的attention结果。
WO∈Rhdv∗dmodelW^O\in R^{hd_v*d_{model}}WO∈Rhdv​∗dmodel​矩阵是为了将输出值转换为和词向量相同大小的向量。

这里再次理解下多头注意力机制，论文中原本的词向量为512维，所以原本得到的QKV均为512维度。
QK的维度只是为了去计算相似度，应该是相同的，而V的维度则决定了最终一次输出的向量维度，拼接次数h乘dVd_VdV​则为最终的输出长度。
现在重新设置QKV的大小：设置成8组，维度均为64的矩阵，这样会生成8个64维的词向量。然后concat成为512维度的向量。然后再乘一个WOW^OWO矩阵，得到最终的多头注意力值。
多头的优点：多个独立的attention计算，作为一个集成的作用，防止过拟合。并且可以在不同的表示子空间里学习到相关的信息

  • 训练细节：
    如果输入序列n小于表示维度d的话，每一层的时间复杂度self-attention是比较有优势的。
    当n比较大时，作者也给出了一种解决方案self-attention（restricted）即每个词不是和所有词计算attention，而是设置窗口，只与限制的r个词去计算attention。

Decoder

  1. 同样也是6个layer组成
  2. 每个layer由3个sub-layer组成，分别为一个隐模的多头自注意力层，多头注意力层，全连接神经网络层组成。
    • 输入：encoder的输出，decoder的第i位的输出
    • 输出：对应i+1位置的输出词的概率分布
    • 训练和预测是不一样的：
      解码需要使用 sequence mask，让解码时看不到未来的token，来支持并行操作。而如果解码器被设置为串行操作，则无需sequence mask。
      在实际预测过程中，由于模型在训练过程中是按照前述方式训练的，所以继续使用seq mask会和训练方式匹配，得到更好的预测结果。

Position Encode/Embedding （PE）

  • RNN的优点为考虑了单词之间的位置关系，但是Transformer剥离了RNN，就不能捕捉位置信息的缺陷，所以需要位置编码来记录位置信息。
  • 要求：
    • It should output a unique encoding for each time-step (word’s position in a sentence).
    • Distance between any two time-steps should be consistent across sentences with different lengths.
    • Our model should generalize to longer sentences without any efforts. Its values should be bounded.
    • It must be deterministic.
  • 两种方法
    • 1. 绝对位置编码：
      • 学习得到的 Positional Embedding
        0表示第一个词的position embedding，1表示最后一个词的position embedding，其余部分按照分数表示。
        但是这样操作，当句子长度不一样时，time-step delta就会不一样。
        eg: 1. 我爱学习 2. 我爱学习，他也爱学习。“我”“爱”之间的距离表示差值是不同的。
      • 用不同频率的sin和cos表达：任意位置之间都可以进行计算。

      PE(pos,2i)=sin(pos/100002i/dmodel)PE(pos,2i+1)=cos(pos/100002i/dmodel)\begin{aligned} PE(pos, 2i) &= sin(pos/10000^{2i/d_{model}})\\ PE(pos, 2i+1) &= cos(pos/10000^{2i/d_{model}}) \end{aligned} PE(pos,2i)PE(pos,2i+1)​=sin(pos/100002i/dmodel​)=cos(pos/100002i/dmodel​)​

      PE函数输出一个和word embedding维度一致的矩阵，行表示词语，列表示词向量
      i表示词向量的位置
      pos 表示词语在句子中的位置
      dmodeld_{model}dmodel​表示词向量的维度
      然后和原来的词向量相加输入到encoder中。
      可以使用三角函数性质来得到一个特定位置的向量，所以该方法蕴含了相对位置信息，但是这种位置信息会在注意力机制那里消失。
      代码实现：https://blog.csdn.net/qq_34771726/article/details/102918440

Mask

  • mask 表示掩码，它对某些值进行掩盖，使其在参数更新时不产生效果。
  • Transformer中涉及到两种mask：padding mask 和 sequence mask。
    • padding mask 在所有的 scaled dot-product attention 里面都需要用到
    • sequence mask 只有在 decoder 的 self-attention 里面用到。

所以在decoder的masked self-attention需要同时用到两者，具体实现是将两者相加。其余都使用padding mask。

padding mask

每个批次输入序列长度是不一样的也就是说，我们要对输入序列进行对齐padding。

  • 序列较短：在后面填充占位符号，考虑到softmax函数最后要压一下，我们需要填充负无穷的数，这样softmax之后就会变成0.
    但是如果输入的序列太长，则是截取左边的内容，把多余的直接舍弃。

sequence mask

为了使得 decoder 不能看见未来的信息，即在t的时刻，我们的解码输出应该只能依赖于t时刻之前的输出，而不能依赖t之后的输出。
方法：使用一个上三角矩阵，使softmax后的上三角全为0。

优点

  1. RNN 固有的顺序属性阻碍了训练样本间的并行化，对于长序列，内存限制将阻碍对训练样本的批量处理。
    transformer中的并行是指矩阵运算吗？？
  2. 自注意力可以产生更具可解释性的模型。我们可以从模型中检查注意力分布。
  3. 用自注意力层构建 Transformer：传统的网络之间，两点的依赖程度/相似程度是与计算步骤/距离有关系的，但是attention则不会出现这个问题。

介绍

预训练 pre-train：根据大量的数据，训练出一个"通用"的模型。
然后针对不同的具体任务，微调 fine-tune 一下，就可以了。

预训练

  • 预训练的优点：

    • 在大规模语料上通过预训练学习通用语言表示对下游任务很有帮助；
    • 预训练提供了更好的模型初始化参数，使得在目标任务上有更好的泛化性能和更快的收敛速度；
    • 预训练是一种有效的正则化方法，能够避免在小数据集上过拟合。

  • 分类：上下文编码器的使用

    • 非上下文嵌入：是静态且与上下文无关的，无法处理多义词； 难以解决未登陆词 oov。
      • word2vec
      • Glove
    • 上下文嵌入：考虑上下文语义，将词在不同上下文的语义做区分。通过对词的 token 加一层编码器得到词的上下文嵌入。
      正确预测 --》 清楚关系

微调

在model的输出上叠加一层特定层 Task-specific Layer，使其适应下游任务。

  • NLP的任务分类
    • Input：
      • one sentence
      • multiple sentences：+分割符号[sep]
    • Output:
      • one class
      • class for each token
      • copy from input
      • general sequence
  • How:
    1. 只调整task-specific层参数
    2. 调整pre-trained和task-specific层的部分，效果更好。

  • 问题：
    不同的task，会得到不同的预处理层，但是参数太多，会浪费。
    解决：引入adapter layer。冻结前面的层，fine tune后面的层。

飞桨训练营

自监督表示学习

自监督学习的监督信息不是人工标注的，而是算法在大规模无监督数据中自动构造监督信息，来进行监督学习或训练。

  • one-hot
  • embedding：one-hot的线性映射
  • word2vec：分布式假设，考虑了语义。
    • cbow：效率更高。
    • skip-gram：生僻词更加准确，耗时更加少。

自回归语言模型

  1. 马尔可夫语言模型：计算后验概率，得到的是词之间的依赖概率。
  2. RNN：可以建模一个句子，或者某个单词的考虑前面单词的表示。
    • 梯度爆炸/消失：选择其他激活函数

命名实体识别

Posted on 2021-06-16 Edited on 2022-05-22

命名实体识别（Named Entity Recognition， NER）：识别出句子中命名实体的边界和类别的任务。

  • 对于规则较强的命名实体，比如网址，email，isbn等，可以通过正则表达式匹配。
  • 对于较短的命名实体，比如人名，可以通过分词确定边界，通过词性标注模块确定类别。
    这样就不需要专门的命名实体识别模块和语料库，只需要普通的分词词性标注模块。
    微软系列的语料库，以颗粒大著称，命名实体被标注为了一个词语。

基于规则的命名实体识别

基于规则的音译人名识别

在一段待识别的文本中，如果音译字符连续出现，则很有可能来自于一个音译人名。
词典中会有音译任命nrf这个词性，但是音译实体是无穷多的，不可能全部被收录，基于此设定一个规则：

  1. 若粗分结果中，某单词的被选词性含有nrf，则
  2. 向右扫描，遇到音译人命词库中的词语则合并，否则终止扫描。

这个规则只负责将分词模块切碎的已知音译人名合并起来，并不负责识别未知的音译人名片段。

基于规则的日本人名识别

日本人名长度一般为3-5位，且含明确的姓氏，用规则识别起来更加简单。

  • HanLp中规则如下：
    • 匹配日本人名的姓氏和名字，分别计作x和m。
    • 将连续的xm合并为日本人名，标注为nrj。
    • 考虑到日本人经常以居住地附近的山水作为姓名，比如龟山公园，可能存在姓氏与名字组成了地名的情况，这些情况被标注为A。

基于规则的数词英文识别

数词和英文单词同样是常见的命名实体，适合利用规则进行识别。
正则表达式匹配开销较大，HanLp中采用将相同类型的连续字符分别合并。

  • 过程：
    • 首先获得每个字符的类型。（单字节，分隔符，中文字符，字母，数字，序号等）
    • 然后扫描字符串的类型数组，将类型相同的字符合并，利用其类型确定词性（数词m / 英文nx）。

基于统计的命名实体识别

基于层叠隐马尔可夫模型的角色标注框架

为构成命名实体的短词语打标签，标签序列满足某种模式则识别为某种命名实体。
对于如何打标签，基于规则的方式是根据词典匹配规则，基于统计的方式是根据隐马尔可夫模型的预测。
该框架中，首先进行分词，然后进行标注。所以标注模块的输入是分词模块的输出，并且两个模块都由隐马尔可夫模型驱动，所以称为层叠隐马尔可夫模型。

最大似然

Posted on 2021-05-10 Edited on 2022-05-22

概率与似然

概率：已知一些参数的情况下，预测接下来的观测所得到的结果。
似然：已知某些观测所得到的结果时，对有关事物的性质的参数/概率进行估计。可以将似然看作为参数的靠谱程度。
在这种意义上，似然函数可以理解为条件概率的逆反。

似然函数

似然函数这个概念是建立在频率学派之下，参数是一个真值，并不是随机变量。所以参数没有分布，参数也没有概率。

  • L(θ∣x)L(\theta | x)L(θ∣x)，表示的是在给定样本 x 的时候，哪个参数 θ\thetaθ 使得 x 出现的可能性最大。
    它在数值上等于给定参数 θ\thetaθ 后变量 X 取某个值的概率：p( X=x∣ θ )p(\ X=x|\ \theta\ )p( X=x∣ θ )。
  • 这个值越大，说明给定参数θ\thetaθ，数据值x出现的概率就越大。同样也表示数据值x出现时，参数为θ\thetaθ的可能性就越大，就越靠谱。
    似然值越大，表示在该参数下，事件越有可能发生。似然值越小，在该参数下事件越不可能发生。
    从这个角度来看，MLE也很好理解了：找到一个参数的估计，使得在该参数估计下，事件发生的可能性最大。
  • 从定义上，似然函数 和 概率密度函数是完全不同的两个数学对象：前者是关于X的函数，后者是关于θ\thetaθ的函数。
    所以这里的等号 = 理解为函数值形式的相等。它们不是同一个函数，但是具有相同的函数形式。

极大似然估计

利用已知的样本结果信息，反推最具有可能（最大概率）导致这些样本结果出现的模型参数值！
极大似然估计中采样需满足一个重要的假设，就是所有的采样都是独立同分布的。

通过encoder-decoder，根据给定的序列，通过特定的方法生成另一个序列的方法。

原理

  • 首先，Seq2Seq的输入序列和输出序列长度是不可变的。

  • 最基础的Seq2Seq模型包含了三个部分，即Encoder、Decoder以及连接两者的中间状态向量C：

    在Seq2Seq结构中，编码器Encoder把所有的输入序列都编码成一个统一的语义向量Context，然后再由解码器Decoder解码。在解码器Decoder解码的过程中，不断地将前一个时刻的输出作为后一个时刻的输入，循环解码，直到输出停止符为止。

    • encoder: 获取语义向量最简单的方式就是直接将最后一个输入的隐状态作为语义向量C，
      也可以对最后一个隐含状态做一个变换得到语义向量，
      还可以将输入序列的所有隐含状态做一个变换得到语义变量。
    • decoder: 最简单的方式是将encoder得到的语义变量作为初始状态输入到decoder的RNN中，得到输出序列。
      可以看到上一时刻的输出会作为当前时刻的输入，而且其中语义向量C只作为初始状态参与运算，后面的运算都与语义向量C无关。

      decoder处理方式还有另外一种，就是语义向量C参与了序列所有时刻的运算，上一时刻的输出仍然作为当前时刻的输入，但语义向量C会参与所有时刻的运算。

弊端和改进

  1. 在seq2seq结构中将上一时刻输出值输入进网络时，那么某一时刻输出值错误就会导致后面全错，这种蝴蝶效应格外明显。
    为了解决这个问题，Google提出了大名鼎鼎的Scheduled Sampling
    即在训练中xtx_txt​按照一定概率选择输入yt−1y_{t-1}yt−1​或t−1t-1t−1时刻对应的真实值，即标签，不仅能加快训练速度，也能提高训练精度。
  2. LSTM模型虽然具有记忆性，但是当Encoder阶段输入序列过长时，解码阶段的LSTM也无法很好地针对最早的输入序列解码。
    除此之外，如果按照上述方式实现，只用到了编码器的最后一个隐藏层状态，信息利用率低下。
    Attention注意力分配的机制被提出，就是为了解决这个问题。

  • 剥离RNN，搭建Attention：

多头注意力机制

组件

  • 多头Self-attention：
    上面讲的剥离了RNN的self-attention被称为单头注意力机制，每个单头注意力机制有3个参数矩阵，l头则有3l个参数矩阵。
    所有不同的单头自注意力机制的输入都相同，但是参数矩阵各不相同，所以输出的c序列是不同的。
    设一共输入了m个单词，将所有不同的输出都堆叠起来。如果单个输出d*m的矩阵，则堆叠后得到ld*m的矩阵。
  • 多头Attention也一样。
