ls (list)

ls [-alrtAFR] [name...]：列出目前工作目录所含的文件及子目录

  • -a：显示所有文件及目录，用于显示隐藏文件
    Command+Shift+.：在finder中显示隐藏文件
  • -l：除文件名称外，同时输出文件形态、权限、拥有者、文件大小等。
    • 指定文件的权限一般就用 ls -l 文件名称
    • -lh，-ll
  • -S -> -lhS：希望以文件大小排序。
  • -p：增加/ (斜线) 标记目录
  • -R：递归列出子目录

ls 命令显示结果图解

ls命令显示结果图解

  • 每个文件的属性由左边第一部分的 10 个字符来确定.

    • 0位字符：表示这个文件是目录、文件或链接文件等等。
      d为目录；-为文件；l为链接文档(link file) …
    • 123位字符：表示「属主权限」（该文件的所有者），以三个为一组，且均为 rwx 的三个参数的组合，分别表示：读，写，可执行，无权限。
    • 456位字符：表示「属组权限」（所有者的同组用户），以三个为一组，同理…
    • 789位字符：表示「其他用户权限」

  • 第三列为 owner；

  • 第四列为 group name；

  • 第五列为 apparent sizes(文件长度,file数据结构中定义的文件长度字段)。

    • 当使用ls -ll，会显示成字节大小；
    • 而ls -lh会以KB、MB等为单位进行显示，这样比较直观一些。

  • 倒数第二列为 last modification time

  • 最后一列为 file name

移动设置

mv ( move file )

用来为文件或者目录改名，或者将文件或者目录移到其他位置。

[options] 参数说明:

  • -b: 当目标文件或目录存在时，在执行覆盖前，会为其创建一个备份。
  • -i: 如果指定移动的源目录或文件与目标的目录或文件同名，则会先询问是否覆盖旧文件，输入 y 表示直接覆盖，输入 n 表示取消该操作。
  • -f: 如果指定移动的源目录或文件与目标的目录或文件同名，不会询问，直接覆盖旧文件。
  • -n: 不要覆盖任何已存在的文件或目录。
  • -u：当源文件比目标文件新或者目标文件不存在时，才执行移动操作。
source dest 运行结果
file file 给文件改名
file directory 将文件移动到指定目录下
directory directory dest_directory 存在：把 source_directory 移动到 dest_directory 下面
dest_directory 不存在：将 source_directory 改名为 dest_directory

同样支持「移动」+「改名」：mv /home/ffxhd/a.txt /home/ffxhd/test/c.txt。
同样支持批量移动文件：mv /home/ffxhd/testThinkPHP5/tp5/* /home/ffxhd/testThinkPHP5

cp ( copy )

复制文件或目录

-r：若给出的源文件是一个目录文件，此时将复制该目录下所有的子目录和文件。
-a：此选项通常在复制目录时使用，它保留链接、文件属性，并复制目录下的所有内容。其作用等于dpR参数组合。
-f：覆盖已经存在的目标文件而不给出提示。相当于在命令前加一个反斜杠 \cp 。反斜杠表示直接使用原生命令。

-d：复制时保留链接。这里所说的链接相当于 Windows 系统中的快捷方式。
-i：默认操作，与 -f 选项相反，在覆盖目标文件之前给出提示，要求用户确认是否覆盖，回答 y 时目标文件将被覆盖。
-p：除复制文件的内容外，还把修改时间和访问权限也复制到新文件中。
-l：不复制文件，只是生成链接文件。

rm ( remove )

删除一个文件或者目录

-i 删除前逐一询问确认。
-f 不逐一确认，强行删除。即使原档案属性设为只读，也会被直接删除，
-r 删除的对象是目录，递归删除目录中所有内容。

mkdir

mkdir [-p] dirName

-p： 确保目录名称存在，不存在的就建一个，用于创建多层目录。

比如mkdir -p runoob2/test，如果当前路径下没有父文件夹runoob2，则创建父文件夹。
本例若不加 -p 参数，且原本 runoob2 目录不存在，则产生错误。

rmdir

rmdir [-p] dirName

命令删除空的目录，只能删除空的，删除非空的话需要使用 rm -r dir_name

rmdir -p BBB/Test：-p 表示在工作目录下的 BBB 目录中，删除名为 Test 的子目录。若 Test 删除后，BBB 目录成为空目录，则 BBB 亦予删除。

cat ( concatenate )

读取和合并文件，并将其内容写入到标准输出。

最常用于：

  • 打印 / 查看一个或多个文本文件的内容
  • 将一个文件的内容追加到另一个文件的末尾来合并文件
  • 创建文件

查看文件内容

读取文件内容，写入到标准输出中，这里标准输出的结果就是打印在命令行中。

重定向文件的内容

默认情况下，cat 命令将内容写入标准输出。
但可以使用标准输出的重定向操作符 > 或者 >> 将标准输出重定向到文件，而不是终端的屏幕。

>：类似 w，不存在则创建 ，存在则覆盖
>>：类似 a，不存在则创建 ，存在则追加

当使用重定向操作符时，可以实现 cp 的功能：cat file1.txt > file2.txt 。

打印参数

-n 或 --number：显示行号，由 1 开始对所有输出的行数编号。

-b 或 --number-nonblank：和 -n 相似，只不过对于空白行不编号。
-s 或 --squeeze-blank：当遇到有连续两行以上的空白行，就代换为一行的空白行。
-v 或 --show-nonprinting：使用 ^ 和 M- 符号，除了 LFD 和 TAB 之外。
-E 或 --show-ends : 在每行结束处显示 $。
-T 或 --show-tabs: 将 TAB 字符显示为 ^I。
-A, --show-all：等价于 -vET。
-e：等价于"-vE"选项；
-t：等价于"-vT"选项；

合并文件用法

将两个或多个文件名作为参数传递给 cat 命令时，这会合并文件的内容。
cat 命令将按其参数给出的顺序读取文件，并以相同的顺序合并打印文件内容。

这也就可以进行文件操作了

创建文件

在编写脚本时你可能会想让用户输入非常的长的内容或者让用户可以输入任何字符然后保存到文件。

最简单的解决方式就是使用 cat 命令作为编辑器，让用户输入任何内容。要做到这一点请在运行命令 cat > file1.txt。

当用户的完成内容的出入后，可按快捷键 CRTL+D 保存文件。如果用户不知道如何保存内容时，可在运行 cat 之前告诉用户保存的快捷键是 CRTL+D。

链接

ln (link files)

为某一个文件在另外一个位置建立一个同步的链接。

需要在不同的目录，用到相同的文件时，我们不需要在每一个需要的目录下都放一个必须相同的文件，我们只要在某个固定的目录，放上该文件，然后在 其它的目录下用ln命令链接（link）它就可以，不必重复的占用磁盘空间。

链接又可分为两种 : 硬链接(hard link)与软链接(symbolic link)

  1. 软链接：链接文件指向原始文件

    1. 以路径的形式存在。类似于Windows操作系统中的快捷方式
    2. 软链接可以 跨文件系统 ，硬链接不可以
    3. 软链接可以对一个不存在的文件名进行链接
      如果原始文件丢失，则链接文件将失效：
    4. 软链接可以对目录进行链接
    5. 删除/移动原始文件/原始文件改名 后，软连接失效。copy 不影响。
      can be seen as a static link to the last known location of the original file.
      work even if you replace the original file with a different file with the same name.

  2. 硬链接：链接文件和原始文件一同指向原始文件的 inode

    1. 硬链接，以文件副本的形式存在。但不占用实际空间。
    2. 不允许给目录创建硬链接
    3. 硬链接只有在同一个文件系统中才能创建
    4. 原始文件被删除/移动后，硬链接文件依然可以访问

修改软硬链接文件内容均会修改原始文件。

inode 存储了文件的 metadata 和 物理文件地址，系统根据 inode 获取文件基本信息和磁盘上的地址，来读取文件。

参数的格式为:

-s 表示使用软链接

软连接文件头为 l ，并且有指针表示。硬链接则没有。
注意：如果使用 rm -rf 软链接名/ 删除，会把软链接对应的真实目录下内容删掉，所以不要加/。

du (disk usage)

du 目录/文件
显示目录下每个子目录的磁盘使用情况
默认情况下，du的输出是递归输出所有文件夹/子目录的磁盘大小，不会输出目录下文件的磁盘大小。

显示指定的目录或文件所占用的磁盘空间。
注意区分磁盘空间和文件大小 ，ls -l 显示的是真实子目录大小，du显示的是 占用磁盘空间 ，即占用 block 的总大小。
一般来说，存储文件的时候很少刚好把block填满，因此du -sh *一般都要比ls -lh大。

比如文件系统的block是 4K，一个15K的文件占用的空间是 15k/4k=3.75 个block，一个block只能被一个文件占用，因此实际占用空间就是4个block，就是16K。

常用参数 ：

  • -a或-all： 递归式写计数的所有文件，不只是目录，即也显示所有内部文件。如果指定-a，则也也递归查询所有文件大小
  • -h或--human-readable：以K，M，G为单位，提高信息的可读性。`
  • -s或--summarize 仅显示该目录的和总。
  • -c或--total 除了显示个别目录或文件的大小外，同时也显示所有目录或文件的总和。
  • -d N参数或者--max-depth=N来控制du命令递归输出的目录层次数。N表示递归输出到第几级目录。

可见会默认输出当前文件夹下的总和，即3.9M .。

用于显示目前在 Linux 系统上的文件系统 磁盘使用情况统计。

Linux 进程管理

Posted on 2022-06-28 In Coding , Linux
Symbols count in article: 1k Reading time ≈ 5 mins.

进程是正在执行的一个程序或命令，每一个进程都是一个运行的实体，都有自己的地 址空间，并占用一定的系统资源。

ps (process status)

查看当前系统进程状态

参数：
a 列出带有终端的所有用户的进程
x 列出当前用户的所有进程，包括没有终端的进程
u 面向用户友好的显示风格
-e 列出所有进程
-u 列出某个用户关联的所有进程
-f 显示完整格式的进程列表

基本语法：

  1. ps aux | grep xxx （功能描述：查看系统中所有进程），用于查看进程的 CPU 占用率 和 内存占用率 。
  2. ps -ef | grep xxx （功能描述：可以查看子父进程之间的关系），用于查看进程的 父进程 ID 。

PID：进程的 ID 号
%CPU：该进程占用 CPU 资源的百分比，占用越高，进程越耗费资源；
%MEM：该进程占用物理内存的百分比，占用越高，进程越耗费资源
STAT：进程状态。
常见的状态有：R：运行状态、S：睡眠状态、T：暂停状态、 Z：僵尸状态、s：包含子进程、l：多线程、+：前台显示
START：该进程的启动时间
TIME：该进程占用 CPU 的运算时间，注意不是系统时间
COMMAND：产生此进程的命令名


UID：用户 ID
PID：进程 ID
PPID：父进程 ID
C：CPU 用于计算执行优先级的因子。
数值越大，表明进程是 CPU 密集型运算，执行优先级会降低；数值越小，表明进程是 I/O 密集型运算，执行优先级会提高。

kill

kill [选项] 进程号 （功能描述：通过进程号杀死进程）
killall 进程名称 （功能描述：通过进程名称杀死进程，也支持通配符，在系统因负载过大而变得很慢时很有用）

-9 表示强迫进程立即停止

top

实时监控系统进程状态，显示当前系统正在执行的进程的相关信息，包括进程ID、内存占用率、CPU占用率等。

前五行是当前系统情况整体的统计信息区。详见：每天一个linux命令（44）：top命令。
第一行信息为任务队列信息

PID — 进程id
USER — 进程所有者
PR — 进程优先级
NI — nice值。负值表示高优先级，正值表示低优先级
VIRT — 进程使用的虚拟内存总量，单位kb。VIRT=SWAP+RES
RES — 进程使用的、未被换出的物理内存大小，单位kb。RES=CODE+DATA
SHR — 共享内存大小，单位kb
S — 进程状态。D=不可中断的睡眠状态 R=运行 S=睡眠 T=跟踪/停止 Z=僵尸进程
%CPU — 上次更新到现在的CPU时间占用百分比
%MEM — 进程使用的物理内存百分比
TIME+ — 进程使用的CPU时间总计，单位1/100秒
COMMAND — 进程名称（命令名/命令行）

命令参数：
-p [进程PID]：指定监控进程 ID 来仅仅监控某个进程的状态。
-i 使 top 不显示任何闲置或者僵死进程。
-d 秒数 指定 top 命令每隔几秒更新。默认是 3 秒在 top 命令的交互模式当 中可以执行的命令
-o [key] 指定按照什么排序

操作说明：
P 以 CPU 使用率排序，默认就是此项
M 以内存的使用率排序
N 以 PID 排序
