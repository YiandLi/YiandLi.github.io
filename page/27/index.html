FastText

Posted on 2021-06-27 In NLP , Text Classification
Symbols count in article: 2k Reading time ≈ 10 mins.
  • 优点：
    1、fastText在保持高精度的情况下加快了训练速度和测试速度
    2、fastText不需要预训练好的词向量，fastText会自己训练词向量
    3、fastText两个重要的优化：Hierarchical Softmax、N-gram

简介

n-gram只是锦上添花的东西，不是必须的。
FastText是从cBow（continuous bag of words）演化而来的：

  • 区别：
    • cbow输出目标单词的概率向量，输入的是目标单词的上下文单词列表。
    • fasttext输出的是文本的标签概率向量，输入的是待分类的文本特征列表。这个特征有多种表示方法：embedding,n-gram,tf-idf…
      无论是输入one-hot然后通过embedding得到词向量，还是直接输入随机初始化的embedding，本质上都是一样的，都是随机初始化。
    • cbow和fasttext的词向量都是随机初始化的。

结构：


和cbow一样，也是三层设计：输入层，隐含层和输出层。
输入多个特征向量，然后叠加取平均，成为文本向量进入隐藏层，然后输入到一个线性分类器中，结果通过softmax得到分类概率向量。

Model architecture of fastText for a sentence with N ngram features x1, . . . , xN.
The features are embedded and averaged to form the hidden variable.
The word representations are then averaged into a text representation, which is in turn fed to a linear classifier.

n-gram

n-gram是锦上添花的东西，产生的仅仅是文本特征的候选集，不是fasttext必须的。

  • 几个概念/分类：

    • Bow：bag of words 词袋模型
    • CBow：continuous Bow，考虑了连续的分布式特征。
    • (word) n-gram：可以保留小范围的部分词序

    use a bag of n-grams as additional features to capture some partial information about the local word order.

    英文 2-gram：apple pie，pie is，is good
    中文 2-gram：苹果派｜是，是｜好吃，好吃｜的

    • character-level n-gram：解决oov和一些单词形态学的概念，低频词的词向量效果也会更好。
      英文 ‘<ap’,‘app’,‘ppl’…
      中文 苹果派，果派是，派是好 …

  • 在输入到fastText时，通常可以将字符级别的n-gram向量作为额外的输入。
    训练时，可以顺便得到字符级别的n-gram的向量，然后字符级向量叠加得到单词的向量。

  • 优点：(补充上面的优点）char n-gram 提高了句法词类比任务的性能
    比如单复数，不同的形式，内部多数字母/汉字还是没有改变的。

  • 缺点：char n-gram 降低了语义类比任务的性能
    不同的单词会有相同的内部字母/汉字，

Hierarchical Softmax

简介：

二分类问题是一种特殊的多分类问题，所以可以将多分类问题拆解成多个嵌套的二分类问题简化计算。
- 当需要预测的文本类别过多时，计算量会特别大：类别数量为K，文本向量维度为h，则时间复杂度为O(Kh)O(Kh)O(Kh)
线性分类器就是多个特征/类别文本表示向量的矩阵，本文向量逐个和特征文本向量做点积求相似度，然后softmax找到相似度最高的得到本文类别。
文本向量长度为h，所以点积操作的时间复杂度为O(h)O(h)O(h)，共有k个特征文本向量，所以作k次点积，softmax的时间复杂度为O(k)。
- 使用层次softmax来减少训练时间

表示：


- 根节点：输入的待分类的向量
- 叶节点：目标类
- 内部节点：表示一个参数向量，看作子树的根节点。
同样可以看作一个感知机/神经元，根据相似度输出走向左叶节点的概率。(哈夫曼树的思想：频率->概率大的放在左边, append 0)
- 根节点到叶节点的一条路径：表示叶节点表示的类别的编码，即路径上所有二分类问题的结果。
是用概率来得到编码，即路径，所以可以用编码反推的概率连乘得到叶节点的概率。
- 边：{0,1}，向左走用0表示，向右走用1表示。
- 目标类别的概率：从root结点开始，走到目标类别（叶节点）的概率。
因此在途中路过非叶子结点（包括root）时，需要分别知道往左走和往右走的概率。

TF-IDF

Term frequency–inverse document frequency 词频-逆向文件频率

  • 思想：词的重要性随着它在文件中出现的次数成正比增加，但同时会随着它在语料库中出现的频率成反比下降。
  • 定义：
    • TF（Term Frequency）
      指词频，表示词条在文本中出现的频率。值等于某词出现的次数/文件中的总词数。

    tfi,j = ni,j∑knk,jtf_{i,j}\ =\ \dfrac{n_{i,j}}{\sum_k{n_{k,j}}} tfi,j​ = ∑k​nk,j​ni,j​​

    • IDF（Inverse Document Frequency）
      指逆向文件频率，表示单词的特色程度。
      由总文件数目除以包含该词语的文件的数目，再将得到的商取对数得到。

    idfi = log∣D∣ ∣{j:ti∈dj}∣idf_{i}\ =\ log \frac{|D|}{\ |\{j:t_i\in d_j\}|} idfi​ = log ∣{j:ti​∈dj​}∣∣D∣​

    • TF-IDF = TF * IDF
  • 特点：
    • 单纯地认为文本频率小的单词就越重要，文本频率大的单词就越无用。
    • 没有考虑词之间的语义联系
    • 没有考虑分布情况：在同类语料库中，往往一些同类文本的关键词被忽略。
    • 改进：https://pdf.hanspub.org/CSA20130100000_81882762.pdf

介绍

  • LSTM很适合用于序列标注问题，但是LSTM并没有考虑上一时刻的输出，yt−1y_{t-1}yt−1​。
  • 这样会导致一个问题，以“我 喜欢 跑步”为例，LSTM输出“喜欢”的标签是“动词”，而“跑步”的标签可能也是“动词”。所以，使用LSTM无法对标签转移关系进行建模。
  • 为了引入标签转移关系，我们可以引入一个标签转移矩阵作为参数，即CRF中的特征矩阵。

Bidirectional Encoder Representatons from Transformers
ELMo和GPT都是单向语言模型，不适合应用于依靠上下文的任务。

结构

Bert的基础架构是Transformer的 encoder 部分（embedding+attention+feed forward）。
是由这样的encoder堆叠，形成了一个多层双向Transformer的encoder（a multi-layer bi-directional Transformer encoder）。
官方给出了两个版本：L表示的是 encoder 的层数，H表示输出的维度，A表示multi-head attention的个数。

输入Encoder之前的处理

输出的序列既可以表示单句也可以表示一对句子。
输入的词向量和Transformer相同，可以是w2v，也可以是随机初始化的向量。
但是位置编码部分则不同，变为：Input = token emb + segment emb + position emb。

  1. Token embedding
    除了正常词汇之外，还有两种特殊符号，和下文提到的NSP任务有关。
  • 特殊符号：
    • [CLS]：表示句子分类问题的结果，这个向量后续会接一个二分类器来进行二分类任务。
      注意：第一个字符总是[CLS]，[CLS]的输出向量不代表整个句子的语义信息。
      但是分类时使用[cls]。无偏性：不偏向任何词，更加公平融合所有词的信息。
    • [SEP]：分割句子。
  • 过程：对所有的输入词汇和标签做正常的embedding。

  1. Segment embedding
    因为处理的是两个句子，所以我们需要对两个句子进行区分。第一个句子全部用0表示，第二个句子都用1来表示。

  2. position embedding
    和 Transformer 不同，Bert 使用随机初始化的方式，然后让模型不断学习更新出来。
    两种位置编码的方式：

    • position Embedding：学习式，不必刻意设计，直接让模型去学就好。缺点是不可扩展，不能处理超长的句子。
    • position Encoding：固定式，通常通过三角函数公式计算出来，通常函数内部使用周期函数表示位置。

使用下游任务中的有标注的数据。

  3. 问答任务 Question Answering Tasks
  4. 序列标注任务 Single Sentence Tagging Tasks

提升BERT的下游任务表现

  • 常见做法：

    • 首先得到公开的bert数据集
    • 基于任务数据进行微调

  • 优化方法：拿微博文本情感分析为例

    1. Pre-train：得到训练好的bert模型（中文谷歌）
    2. Domain transfer：在相同领域上继续训练LM，即在大量微博文本上继续训练。
    3. Task transfer：在任务相关的小数据上继续训练LM，即在微博情感文本上继续训练。
    4. Fine-tune：在具体任务相关的数据上做具体任务。

即：先领域，在任务，最后微调。

  • 参数推荐：
    • Batch size：16， 32
    • Lr：5e-5, 3e-5, 2e-5 小一点，避免遗忘太多
    • Number of epochs：3，4
    • Weighted decay修改后的adam，使用warmup，搭配线性衰减。

BERT训练时使用的学习率 warm-up 策略是怎样的？为什么要这么做？

warmup 需要在 训练最初使用较小的学习率 来启动，并很快切换到 大学习率 而后进行常见的 decay。

  1. 刚开始模型对数据的“分布”理解为零，或者是说“均匀分布”（当然这取决于你的初始化）；训练初期，模型会很快地进行数据分布修正，如果这时候学习率就很大，极有可能导致开始的时候就对该数据“过拟合”，后面要通过多轮训练才能拉回来，浪费时间。
  2. 当训练了一段时间（比如两轮、三轮）后，模型已经对每个数据点看过几遍了，或者说对当前的batch而言有了一些 正确的先验 ，较大的学习率就不那么容易会使模型学偏，所以可以适当调大学习率。
  3. 这个过程就可以看做是warmup。那么为什么之后还要decay呢？当模型训到一定阶段后（比如十个epoch），模型的分布就已经比较固定了，或者说能学到的新东西就比较少了。如果还沿用较大的学习率，就会破坏这种稳定性，用我们通常的话说，就是已经接近loss的local optimal了，为了靠近这个point，我们就要慢慢来。

为什么 BERT 的三个 Embedding 可以进行相加？

假设 token Embedding 矩阵维度是 [4,768]；position Embedding 矩阵维度是 [3,768]；segment Embedding 矩阵维度是 [2,768]。对于一个字，假设它的 token one-hot 是[1,0,0,0]；它的 position one-hot 是[1,0,0]；它的 segment one-hot 是[1,0]。那这个字最后的 word Embedding，就是上面三种 Embedding 的加和。如此得到的 word Embedding，和concat后的特征：[1,0,0,0,1,0,0,1,0]，再过维度为 [4+3+2,768] = [9, 768] 的全连接层，得到的向量其实就是一样的。

再换一个角度理解：直接将三个one-hot 特征 concat 起来得到的 [1,0,0,0,1,0,0,1,0] 不再是one-hot了，但可以把它映射到三个one-hot 组成的特征空间，空间维度是 432=24 ，那在新的特征空间，这个字的one-hot就是[1,0,0,0,0…] (23个0)。此时，Embedding 矩阵维度就是 [24,768]，最后得到的 word Embedding 依然是和上面的等效，但是三个小 Embedding 矩阵的大小会远小于新特征空间对应的 Embedding 矩阵大小。当然，在相同初始化方法前提下，两种方式得到的 word Embedding 可能方差会有差别，但是，BERT还有Layer Norm，会把 Embedding 结果统一到相同的分布。

Mask

  • mask 表示掩码，它对某些值进行掩盖，使其在参数更新时不产生效果。
  • Transformer中涉及到两种mask：padding mask 和 sequence mask。
    • padding mask 在所有的 scaled dot-product attention 里面都需要用到
    • sequence mask 只有在 decoder 的 self-attention 里面用到。

所以在decoder的masked self-attention需要同时用到两者，具体实现是将两者相加。其余都使用padding mask。

padding mask

每个批次输入序列长度是不一样的也就是说，我们要对输入序列进行对齐padding。

  • 序列较短：在后面填充占位符号，考虑到softmax函数最后要压一下，我们需要填充负无穷的数，这样softmax之后就会变成0.
    但是如果输入的序列太长，则是截取左边的内容，把多余的直接舍弃。

sequence mask

为了使得 decoder 不能看见未来的信息，即在t的时刻，我们的解码输出应该只能依赖于t时刻之前的输出，而不能依赖t之后的输出。
方法：使用一个上三角矩阵，使softmax后的上三角全为0。

命名实体识别（Named Entity Recognition， NER）：识别出句子中命名实体的边界和类别的任务。

  • 对于规则较强的命名实体，比如网址，email，isbn等，可以通过正则表达式匹配。

  2. 向右扫描，遇到音译人命词库中的词语则合并，否则终止扫描。

概率与似然

概率：已知一些参数的情况下，预测接下来的观测所得到的结果。
似然：已知某些观测所得到的结果时，对有关事物的性质的参数/概率进行估计。可以将似然看作为参数的靠谱程度。
在这种意义上，似然函数可以理解为条件概率的逆反。

似然函数

似然函数这个概念是建立在频率学派之下，参数是一个真值，并不是随机变量。所以参数没有分布，参数也没有概率。

  • L(θ∣x)L(\theta | x)L(θ∣x)，表示的是在给定样本 x 的时候，哪个参数 θ\thetaθ 使得 x 出现的可能性最大。
    它在数值上等于给定参数 θ\thetaθ 后变量 X 取某个值的概率：p( X=x∣ θ )p(\ X=x|\ \theta\ )p( X=x∣ θ )。
  • 这个值越大，说明给定参数θ\thetaθ，数据值x出现的概率就越大。同样也表示数据值x出现时，参数为θ\thetaθ的可能性就越大，就越靠谱。
    似然值越大，表示在该参数下，事件越有可能发生。似然值越小，在该参数下事件越不可能发生。
    从这个角度来看，MLE也很好理解了：找到一个参数的估计，使得在该参数估计下，事件发生的可能性最大。
  • 从定义上，似然函数 和 概率密度函数是完全不同的两个数学对象：前者是关于X的函数，后者是关于θ\thetaθ的函数。
