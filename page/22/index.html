Lecture 14 竞争学习1

生物学中，神经元相互存在竞争关系。

竞争学习 Competitive Learning：
属于无监督学习。
在 Hebbian Learning 中，所有的神经元都可以被激活。
但是在竞争学习中，每次训练时，每一组/层只有一个神经元会激活。
输出单元和其他单元竞争 competitve，最出色的一个我们称之为赢家 Winner-takes-all（WTA）unit，赢家通吃。

无监督竞争学习

  • 层与层之间神经元相互刺激 excitatory
  • 层内神经元相互阻碍的 inhibitory
    • 层内会有多个神经元簇 cluster，聚类内部神经元相互抑制
    • 在每个神经元簇中，只有一个神经元可以被激活

竞争学习中的聚类问题

  1. 随机初始化K个原型聚类中心（prototype vector
  2. 随机找到一个样本点（a single example
  3. 找到距离这个样本点最近的聚类中心（closest prototype），就是我们所说的赢家。
  4. 使这个聚类中心（赢家）靠近样本点

更新权重规则

在聚类问题中，权重 www 可以理解为聚类点的向量（我们使用点积表示相似度时，点积就是相乘，所以也可以理解为权重）。

训练方式

输入：m个样本点，随机K个聚类中心，学习率 η\etaη
每次训练[a. 随机抽取一个/ b. 按顺序遍历各个]样本点

  1. 求样本点到各个聚类中心的距离/相似度
  2. 找到最近的聚类中心 j∗j^*j∗
  3. 带入公式 △wj∗i=ηyj(xi−wj∗i)\triangle w_{j^*i} = \eta y_j (x_i -w_{j^*i})△wj∗i​=ηyj​(xi​−wj∗i​) 求得权重变化量
  4. 权重变化量即该聚类中心 j∗j^*j∗ 位置变化量，使用 wj∗−△wj∗iw_{j^*}-\triangle w_{j^*i}wj∗​−△wj∗i​ 得到该聚类中心的新坐标

这种方式生成的结果，对随机初始化的权重/聚类中心很敏感。

加强竞争公平性

  • 初始化的聚类中心建议在样本点的区域内部 (in region with few patterns
  • 一些聚类中心/单位永远不可能成为赢家，进而成为了死亡神经元（dead unit
  • 为了更有效地保证竞争公平性，每个单位应该有平等的机会，最终都能代表训练数据。

Maxnet

  • Maxnet 定义为一个赢家通吃（Winner Take All）竞争网络，并且竞争者之间侧抑制。
  • 侧抑制层（Lateral inhibition）：同一层的神经元也会连接，但是相互抑制（feed to others through inhibitory connections 。

yi(t+1)=f[∑j=1nωijyj(t)]=f[θyi(t)−ε∑j≠iyj(t)]\begin{aligned} y_{i}(t+1)&=f\left[\sum_{j=1}^{n} \omega_{i j} y_{j}(t)\right]\\ &= f\left[\theta y_i(t) - \varepsilon\sum_{j\ne i}y_j(t) \right] \end{aligned} yi​(t+1)​=f[j=1∑n​ωij​yj​(t)]=f⎣⎢⎡​θyi​(t)−εj=i∑​yj​(t)⎦⎥⎤​​

上图反映了侧抑制机制，具体为同层不同神经元相互影响，表现为权重改变，可以看成一个横截面去理解。
输出层神经元之间相互抑制，每个神经元都倾向于抑制跟他同一层的其他神经元，即其他神经元的权值会减小。
通过抑制作用，Maxnet经过若干次迭代，使除了最大输入外的所有输入值变为负值，并使得对应输出为0。最终剩下的一个输出节点恰好是输入最大的节点。

抑制权值 0<ε<1m0\lt\varepsilon\lt\frac1m0<ε<m1​ ，mmm 为竞争者数量。
如果 ε\varepsilonε 太小，则拟合时间过久；如果 ε\varepsilonε 太大，则会抑制整个网络从而导致没有赢家出现。

以上只讨论了一个赢家出现的情况，但是多个赢家都胜出的情况也是存在的。
此外，侧连接产生的抑制性权值通常满足一定的分布关系，如距离远的抑制作用弱，距离近的抑制作用强。

Mexican Hat Network

根据到赢家神经元的距离，侧连接会产生不同的积极作用(excitatory effect)或者抑制作用(inhibitory effect)。
墨西哥草帽图模拟了输出层神经元之间的突触权值的分布。

  • close neighbours: cooperative (mutually excitatory)
  • distant neighbours: competitive (mutually inhibitory)
  • too far away neighbours: irrelevant

获胜神经元对其邻近神经元的影响是由近及远，由兴奋逐渐转变为抑制，因此其学习算法中不仅获胜神经元本身要调整权向量，它周围的神经元在其影响下也要程度不同地调整权向量。

原型向量m也被称之为 codeword/codevector/centroid 。
任务是给新向量 xix_ixi​ 分类，需要找到一个最优原型向量(best matched)，取这个最优原型向量代表的类别。即：

在这里，使用两点之间的欧式距离作为指标，q为最近临函数。在这个情况下，k-means可以用来构建codebook。

Learning Vector Quantizer(LVD)

监督学习方法
使用类别信息，不断移动Voronoi encoding vectors，来提高分类决策边界的准确度。

方法

  2. 找到最近的encoding vector，如果其和样本x的类别标签一致，则w向x的方向移动；否则向远离x的方向上移动。

自组织映射(Self-organizing map, SOM)

生物动机

Biological Motivation
功能相近的神经元分布在一起。
人脑在处理信息时，会将外部的 高维 信号(external multidimensional representation)映射到脑内的二维空间(2-dimensional internal representation)。
即 大脑的自组织机制self-organization ：大脑会生成线性或者平面拓扑的特征 linear or planar topology。

网络特征

  • 无监督学习，竞争学习
  • 通过学习输入空间中的数据，生成一个 低维、离散的映射(Map) ，是一种矢量量化算法，从某种程度上也可看成一种降维算法。
  • 融合了竞争机制和协作机制 (incorporation of competitive and cooperative mechanism)。
  • 使用 近邻关系函数(neighborhood function) 来维持输入空间的拓扑结构：二维映射包含了数据点之间的相对距离。
    输入空间中相邻的样本会被映射到相邻的输出神经元。
    输出空间维持了输入空间的近邻关系，输出空间中近邻的神经元表示了相似的输入。

方差 & 偏差

偏差

【背下来】偏差是指由所有采样得到的大小为m的训练数据集，训练出的模型的输出平均值和真实模型输出之间的偏差。

表现在训练误差不收敛。
描述模型输出结果的期望与样本真实结果的差距，模型表达能力有限导致的系统性错误。

方差

【背下来】是指有所有采样得到的大小为m的训练数据集，训练出的所有模型的输出的方差
描述模型对于给定值的输出稳定性。
分类器对样本分布过于敏感，到指在训练样本较少的时候，出现过拟合

基分类器的错误，是偏差和方差之和
boosting方法通过逐步聚焦分类器分错的样本，减少集成分类器的偏差
Bagging采用分而治之的策略，通过对样本多次采样，分别训练多个模型，减少方差

为什么决策树是常用的基分类器？

  1. 可以方便地将样本权重整合到训练过程中，不需要使用过采样的方法来调整样本权重，每次采样需要耗费时间
  2. 决策树的表达能力和泛化能力，可以通过调节树的层数来做折中
  3. 数据样本扰动对决策树影响较大，因此不同子样本集生成的基分类器随机性就较大。
    这样的不稳定学习器更适合作为基分类器。

Boosting

  • 集成学习大致可分为两大类：Bagging和Boosting

    • Bagging一般使用强学习器，其个体学习器之间不存在强依赖关系，容易并行。
    • Boosting则使用弱学习器，其个体学习器之间存在强依赖关系，是一种序列化方法。
    • Bagging主要关注降低方差，而Boosting主要关注降低偏差。

  • Boosting是一族算法，其主要目标为将弱学习器“提升”为强学习器，大部分Boosting算法都是根据前一系列弱学习器的不足(shortcoming)对样本分布进行调整，再根据新的样本分布训练下一弱学习器，如此迭代M次，最后将一系列弱学习器组合成一个强学习器。

  • boosting算法中，损失函数改变，给不同的样本赋予不同的权重：

∑n=1N1NI[j(xn≠tn)]becomes      ∑n=1NwnI[j(xn≠tn)]where      wn>0  and  ∑n=1Nwn=1\begin{aligned} \sum_{n=1}^N \frac1N \mathbb I\left[j(x^n \ne t^n)\right]\\ becomes\;\;\; \sum_{n=1}^N w^n \mathbb I\left[j(x^n \ne t^n)\right]\\ where\;\;\; w^n>0\; and \;\sum_{n=1}^Nw^n=1 \end{aligned} n=1∑N​N1​I[j(xn=tn)]becomesn=1∑N​wnI[j(xn=tn)]wherewn>0andn=1∑N​wn=1​

GridSearchCV

GridSearch和CV，即网格搜索和交叉验证。

缺点：在面对大数据集和多参数的情况下，非常耗时

交叉验证：

  • 将训练数据集划分为K份，K一般为10。
  • 依次取其中一份为验证集，其余为训练集训练分类器，测试分类器在验证集上的精度。
  • 取K次实验的平均精度为该分类器的平均精度。

三种 ： KFold、StratifiedKFold、GroupKFold

KFold

直接随机的将数据划分为k折，不考虑 train set 和 eval set 的 class 和 group 分布情况。
只需要传入数据X即可，模型会动态返回两个数组train, test，分别对应 train example 和 test example 的索引。

模型会递归输出 n_splits 次，其实每次相当于单拎出来一个 fold 作为 test 集合，将其他的 fold 拼接起来，这样就看不到具体是如何划分 fold 的了，即每次只能显示输出一个 fold 的索引，即 test。

更常用的是动态根据每次生成的单个 test 索引集合设置 fold_id ：

这样后续只需要 topic_df_non_source[topic_df_non_source["fold"]=1] 即可多次拿出对应的fold。

StratifiedKFold

根据数据集的分布来划分，使得划分后的数据集的目标比例和原始数据集近似，即 train set 和 eval set 的类别分布都和原始数据集分布近似。
这里需要同时传入标签列表 y ，

GroupKFold

保证同一个group的数据不会同时出现在训练集和测试集上。
因为如果训练集中包含了每个group的几个样例，可能训练得到的模型能够足够灵活地从这些样例中学习到特征，在测试集上也会表现很好。但一旦遇到一个新的group它就会表现很差。


可以将 group 理解成 domain，这样让同一个 domain 的数据只出现在 train set 或者 dev set，是想评价模型的迁移能力。
需要同时传入数据集合的 group信息。
