特征提取

TF-IDF

Term frequency–inverse document frequency 词频-逆向文件频率

  • 思想：词的重要性随着它在文件中出现的次数成正比增加，但同时会随着它在语料库中出现的频率成反比下降。
  • 定义：
    • TF（Term Frequency）
      指词频，表示词条在文本中出现的频率。值等于某词出现的次数/文件中的总词数。

    tfi,j = ni,j∑knk,jtf_{i,j}\ =\ \dfrac{n_{i,j}}{\sum_k{n_{k,j}}} tfi,j​ = ∑k​nk,j​ni,j​​

    • IDF（Inverse Document Frequency）
      指逆向文件频率，表示单词的特色程度。
      由总文件数目除以包含该词语的文件的数目，再将得到的商取对数得到。

    idfi = log∣D∣ ∣{j:ti∈dj}∣idf_{i}\ =\ log \frac{|D|}{\ |\{j:t_i\in d_j\}|} idfi​ = log ∣{j:ti​∈dj​}∣∣D∣​

    • TF-IDF = TF * IDF
  • 特点：
    • 单纯地认为文本频率小的单词就越重要，文本频率大的单词就越无用。
    • 没有考虑词之间的语义联系
    • 没有考虑分布情况：在同类语料库中，往往一些同类文本的关键词被忽略。
    • 改进：https://pdf.hanspub.org/CSA20130100000_81882762.pdf

LSTM+CRF

Posted on 2021-06-23 Edited on 2022-05-22

介绍

  • LSTM很适合用于序列标注问题，但是LSTM并没有考虑上一时刻的输出，yt−1y_{t-1}yt−1​。
  • 这样会导致一个问题，以“我 喜欢 跑步”为例，LSTM输出“喜欢”的标签是“动词”，而“跑步”的标签可能也是“动词”。所以，使用LSTM无法对标签转移关系进行建模。
  • 为了引入标签转移关系，我们可以引入一个标签转移矩阵作为参数，即CRF中的特征矩阵。

LSTM-CRF

初始的隐藏层输出为0。

LSTM-CRF


在网络输出的分数矩阵中

  • network score： [fθ]i,t\left[f_{\theta}\right]_{i , t}[fθ​]i,t​ ：网络参数 θ\thetaθ 时，第t个单词标签为i的分数。
    可以看做发射得分矩阵。这个发射的过程可以看做lstm直接分类的结果，所以发射得分是由网路后面的分类器决定的。
    网络参数（包括最后一层分类器）需要被训练。
  • transition score： [A]i,j[\mathrm{A}]_{\mathrm{i} , \mathrm{j}}[A]i,j​ ：一对连续时间步骤从第i标签到第j标签的分数。
    可以看做转移得分矩阵。这个转移矩阵与位置t无关！
    这个转移得分矩阵也要被训练。
  • 将两者参数融合：θ~=θ∪{[A]i,j∀i,j}\tilde{\theta}=\theta \cup\left\{[A]_{i, j} \forall i, j\right\}θ~=θ∪{[A]i,j​∀i,j}

  • 输入句子序列 [x]1T[x]_{1}^{T}[x]1T​ ，标签序列 [i]1T[i]_{1}^{T}[i]1T​ 的分数为：
    下标1：一个句子，一个标注

s([x]1T,[i]1T,θ~)=∑t=1T([A][i]t−1,[i]t+[fθ][i]t,t)s\left([x]_{1}^{T},[i]_{1}^{T}, \tilde{\theta}\right)=\sum_{t=1}^{T}\left([A]_{[i]_{t-1},[i]_{t}}+\left[f_{\theta}\right]_{[i]_{t}, t}\right) s([x]1T​,[i]1T​,θ~)=t=1∑T​([A][i]t−1​,[i]t​​+[fθ​][i]t​,t​)

Bi-LSTM CRF + Feature


额外的箭头表示feature

  • spelling features and context features
    • Spelling features
      除了小写的单词特征外，我们还提取给定单词的下列特征。
      是否以大写字母开头
      是否有所有大写字母
      是否有所有小写字母
      是否有非首字母大写
      是否字母和数字混合
      是否有标点符号
      字母前缀和后缀(窗口大小为2到5)
      是否有撇号结尾(’ s)
      非字母，例如 A. T. &T. to …&
      单词模式特征，大写字母、小写字母和数字分别映射到“A”、“A”和“0”，例如，d56 -3到A00a-0
      词型总结特征，类似于词型特征，但去掉了连续的相同字符。例如，d56 -3到A0a-0
    • Context features
      uni-gram features and bi-grams features (and tri-gram features).
  • features connection tricks
    这些特征可以拼接到word embdding后一起进入网络训练；也可以直接concat到LSTM的输出层，作为线性全连接层的输入，计算发射得分。

原理

  • CRF中有两类特征： a. xtx_txt​和yty_tyt​之间的特征 b. yty_tyt​ 和 yt−1y_{t-1}yt−1​ 之间的特征。
    a. 输入xtx_txt​, LSTM的输出yty_tyt​，对应CRF中xtx_txt​和yty_tyt​之间的特征
    b. 根据特征转移矩阵得到 [yt−1,yt][y_{t-1}, y_t][yt−1​,yt​] 对应的得分，对应CRF中 $y_t $ 和 yt−1y_{t-1}yt−1​ 之间的特征。
  • CRF层接受lstm层输出的发射分数，并且计算时会加上转移分数。
    在训练时可以自动学到标注约束。
  • 假设输入n个单词，每个的标注都有m个可能，则总序列有mnm^nmn种标注方法。设每个句子的得分为score(y)，然后类似CRF，使用softmax求出其条件概率：

p(y∣x) = escore(y)ZZ = ∑yescore(y)\begin{aligned} &p(y|x)\ =\ \dfrac{e^{score(y)}}{Z}\\ &Z\ =\ \sum_y{e^{score(y)}} \end{aligned} ​p(y∣x) = Zescore(y)​Z = y∑​escore(y)​

给定输入x，如何计算某个输出序列y的概率？

  1. 首先对单个标注结果
    发射得分：LSTM的每个节点会输出一个m维的向量，将时刻i时的输出得分矩阵记作eie_iei​。
    输出结果yiy_iyi​，对应的得分记作ei[yi]e_i[y_i]ei​[yi​]。
    转换得分：根据前面的标签转移矩阵T，可以得到时刻i时的分数T[i−1][i]T[i-1][i]T[i−1][i]。
    然后我们可以得到某个句子的总的得分：score(y)=∑i=1n(ei)+∑i=2n(T[i−1][i])score(y) = \sum_{i=1}^n (e_i) + \sum_{i=2}^n(T[i-1][i])score(y)=∑i=1n​(ei​)+∑i=2n​(T[i−1][i])。
    （因为是得分，所以这里使用和的形式，即总得分。）

  2. 然后计算所有标注结果，求出Z：Z = ∑yscore(y)Z\ =\ \sum_y score(y)Z = ∑y​score(y)，这里使用动态规划来计算，降低计算量。

  3. 于是就可以求出概率p(y∣x) = escore(y)Zp(y|x)\ =\ \dfrac{ e^{score(y)} }{Z}p(y∣x) = Zescore(y)​，这里Z=∑yescore(y){Z}=\sum_y{e^{score(y)}}Z=∑y​escore(y)。
    为了简化计算，取对数：logp(y∣x) = score(y)−log(∑yescore(y))logp(y|x)\ =\ score(y) - log(\sum_y{e^{score(y)}})logp(y∣x) = score(y)−log(∑y​escore(y))

如何训练模型？

直接定义损失函数为负的目标似然函数：loss = −logp(y∣x)loss\ =\ -logp(y|x)loss = −logp(y∣x)，然后求导更新令loss最小即可。

如何使用模型预测？

使用动态规划求解即可。

为什么是LSTM-CRF而不是LSTM-HMM

  1. hmm的状态转移是概率，crf是分数，再全局做的归一化，这是一点点区别。
  2. lstm的输出考虑了整个观测，但是hmm做了观测独立假设，所以这个lstm+crf更符合crf的定义，而不是hmm。

pad_sequences

keras.preprocessing.sequence.pad_sequences(sequences, maxlen=None, dtype='int32', padding='pre', truncating='pre', value=0.0)
将多个序列截断或补齐为相同长度

  • 简介：
    该函数将一个 num_samples 的序列（整数列表）转化为一个2维的Numpy矩阵，其尺寸为 (num_samples, num_timesteps)。
    每一行为一个单独的序列/句子，长度为num_timesteps。
    num_timesteps 要么是给定的 maxlen 参数，要么是最长序列的长度。
    比 num_timesteps 短的序列将在末端以 value 值补齐。
    比 num_timesteps 长的序列将会被截断以满足所需要的长度。
    补齐或截断发生的位置分别由参数 pading 和 truncating 决定。
  • 参数：
    • sequences：必选参数，列表的列表，通常为sentences。
    • maxlen: 整数，所有序列的最大长度。
    • dtype: 输出序列的类型。 要使用可变长度字符串填充序列，可以使用 object。
    • padding: 字符串，设置补齐的方式。‘pre’ 或 ‘post’，在序列的前端补齐还是在后端补齐。
    • value: 浮点数，表示用来补齐的值。
    • truncating: 字符串，设置截断的方式。‘pre’ 或 ‘post’ ，移除长度大于 maxlen 的序列的值，要么在序列前端截断，要么在后端。
  • 返回：一个Numpy矩阵，尺寸为 (len(sequences), maxlen)。

Bert

Posted on 2021-06-20 Edited on 2022-05-22

Bidirectional Encoder Representatons from Transformers
ELMo和GPT都是单向语言模型，不适合应用于依靠上下文的任务。

结构

Bert的基础架构是TRM的encoder部分（embedding+attention+feed forward）。
是由这样的encoder堆叠，形成了一个多层双向TRM的encoder（a multi-layer bidirectionnal Transformer encoder）。
官方给出了两个版本：L表示的是encoder的层数，H表示输出的维度，A表示mutil-head attention的个数。

输入Encoder之前的处理

输出的序列既可以表示单句也可以表示一对句子。
输入的词向量和TRM相同，可以是w2v，也可以是随机初始化的向量。
但是位置编码部分则不同，变为：Input = token emb + segment emb + position emb。

  1. Token embedding
    除了正常词汇之外，还有两种特殊符号，和下文提到的NSP任务有关。
  • 特殊符号：
    • [CLS]：表示句子分类问题的结果，这个向量后续会接一个二分类器来进行二分类任务。
      注意：第一个字符总是[CLS]，[CLS]的输出向量不代表整个句子的语义信息。
      但是分类时使用[cls]。无偏性：不偏向任何词，更加公平融合所有词的信息。
    • [SEP]：分割句子。
  • 过程：对所有的输入词汇和标签做正常的embedding。

  1. Segment embedding
    因为处理的是两个句子，所以我们需要对两个句子进行区分。第一个句子全部用0表示，第二个句子都用1来表示。

  2. position embedding
    和TRM不同，Bert使用随机初始化的方式，然后让模型不断学习更新出来。

预训练

两个任务 MLM + NSP
Bert选择的是大量的无标注的语料，所以属于无监督/自监督学习。

  • 采用了自编码模型 auto encoding（AE）：使用上下文的信息从损坏的输入数据中重建原始数据。
    即最大化P(我爱吃饭｜我爱mask饭)=P(mask=吃｜我爱饭)的概率。
    模型缺陷：在存在多个mask时，模型会认为这些mask是相互独立的，其实不是。eg.p(我爱mask mask)=P(吃｜我爱)p(饭｜mask)

MLM：masked language model

  • 问题：因为fine-tuning和实际任务中没有[mask]这个单词，所以可能存在**误匹配(mismatching)**的问题。
  • 解决：随机选择 15%的单词，对于这部分单词
    1. 10% 替换成为其他单词
    2. 10% 原封不动
    3. 80% 替换成为mask
      根据paper中实验可见，有利于各个实验的平衡。
      这里的1：1：8是利用随机数来实现了，也就是说这个百分比不是严格不变的。

为什么用15%，word2vec里面的文本窗口大小一般为7，1/7就是15%。

  • 细节：动态mask，n-gram mask
  • 最后使用预测值和实际值的交叉熵损失函数来训练模型。

NSP：next sentence prediction

使用[cls]标签标识表示是否为上下句关系

  • 目的：描述两个句子之间的关系。
  • 进步：先前的技术中，都只能利用句向量来判断句子之间的关系，而Bert可以利用词向量。
  • 不同的样本：
    • 从训练预料库中取出两个连续的段落作为正样本
    • 从不同的文档中随机抽取一对段落作为负样本
  • 缺点：将主题预测（来源相同）和连贯性预测（连续的两个句子）合并成为了一个单项任务。通常主题预测是简单很多的，非常容易去学习，所以导致没有效果。

微调

使用下游任务中的有标注的数据。
针对不同的任务，需要根据相同的预训练模型，通过相同的结构来训练得到不同的微调模型。
分为四种情况：

  1. 句对分类任务 Sentence Pair Classification Tasks
  2. 单句分类任务 Single Sentence Classification Tasks
  3. 问答任务 Question Answering Tasks
  4. 序列标注任务 Single Sentence Tagging Tasks

提升BERT的下游任务表现

  • 常见做法：

    • 首先得到公开的bert数据集
    • 基于任务数据进行微调

  • 优化方法：拿微博文本情感分析为例

    1. Pre-train：得到训练好的bert模型（中文谷歌）
    2. Domain transfer：在相同领域上继续训练LM，即在大量微博文本上继续训练。
    3. Task transfer：在任务相关的小数据上继续训练LM，即在微博情感文本上继续训练。
    4. Fine-tune：在具体任务相关的数据上做具体任务。

即：先领域，在任务，最后微调。

  • 参数推荐：
    • Batch size：16， 32
    • Lr：5e-5, 3e-5, 2e-5 小一点，避免遗忘太多
    • Number of epochs：3，4
    • Weighted decay修改后的adam，使用warmup，搭配线性衰减。

keras.layers.Dropout(rate, noise_shape=None, seed=None)

  • 作用：在训练中每次更新时， 将输入单元的按比率随机设置为 0， 这有助于防止过拟合。
  • 参数：
    • rate: 在 0 和 1 之间浮动。需要丢弃的输入比例。
    • noise_shape: 1D 整数张量， 表示将与输入相乘的二进制 dropout 掩层的形状。
      例如，如果你的输入尺寸为 (batch_size, timesteps, features)，然后希望dropout在所有时间步都是一样的，可以使用 noise_shape=(batch_size, 1, features)。
    • seed: 一个作为随机种子的 Python 整数。

Embedding Layer

文档：https://keras.io/zh/layers/embeddings/

简介

嵌入层 keras.layers.Embedding
将正整数表示的稀疏向量转换为固定尺寸的稠密向量。
只能用于模型的第一层。

参数：

  • input_dim：大于0的整数，表示词汇表的大小。
  • output_dim：大于0的整数，词向量的维度。
  • embeddings_initializer：矩阵初始化的方法。
  • embeddings_regularizer：矩阵的正则化方法。
  • embeddings_constraint
  • mask_zeros

由encoder和decoder组成

Encoder

  • 在论文中，Encoder由6个相同的layer组成，即左边的部分。
  • 每个layer由两个sub-layer组成，分别为多头注意力机制multi-head self-attention mechanism和全连接前馈网络fully connected feed-forward network。

multi-head self-attention

multi-head self-attention中，通过 h组(h头) 不同的线性变化（Q，K，V）得到h组解，然后拼接起来。这里采用scaled dot-product方法计算attention的值：
词向量与相同的QKV矩阵乘得到对应的三个矩阵。

Attention(Q,K,V)=softmax(QKTdkV)Attention(Q,K,V)=softmax(\frac{QK^T}{\sqrt{d_k}}V) Attention(Q,K,V)=softmax(dk​​QKT​V)

Q表示所需计算的注意力项，K为被计算注意力项，V为被注意力项的取值。

可以看到，Q和K的尺寸是相同的，得出结果为相似度矩阵，矩阵的第[i,j]项表示Q的第i行和K的第j行的点积相似度（投影相似度），Query和key的关联强度。
scorei,jscore_{i,j}scorei,j​表示第i个单词和第j个单词的相似度得分。
然后除以一个标量，然后对权重矩阵进行mask处理！！。这里为什么要用mask？尺寸可以对得上啊！
（标量的作用：1. qk相乘值很大时，softmax的梯度就会很小，然后消失。2.方差控制为1。3.softmax中指数函数会将数值的差距拉大，所以在softmax前进行等比例缩小。）

再通过softmax进行映射，得到的就是相似度得分。
最后用相似度得分乘注意力项的取值得到最终的attention结果。
WO∈Rhdv∗dmodelW^O\in R^{hd_v*d_{model}}WO∈Rhdv​∗dmodel​矩阵是为了将输出值转换为和词向量相同大小的向量。

这里再次理解下多头注意力机制，论文中原本的词向量为512维，所以原本得到的QKV均为512维度。
QK的维度只是为了去计算相似度，应该是相同的，而V的维度则决定了最终一次输出的向量维度，拼接次数h乘dVd_VdV​则为最终的输出长度。
现在重新设置QKV的大小：设置成8组，维度均为64的矩阵，这样会生成8个64维的词向量。然后concat成为512维度的向量。然后再乘一个WOW^OWO矩阵，得到最终的多头注意力值。
多头的优点：多个独立的attention计算，作为一个集成的作用，防止过拟合。并且可以在不同的表示子空间里学习到相关的信息

  • 训练细节：
    如果输入序列n小于表示维度d的话，每一层的时间复杂度self-attention是比较有优势的。
    当n比较大时，作者也给出了一种解决方案self-attention（restricted）即每个词不是和所有词计算attention，而是设置窗口，只与限制的r个词去计算attention。

Position Encode/Embedding （PE）

  • RNN的优点为考虑了单词之间的位置关系，但是Transformer剥离了RNN，就不能捕捉位置信息的缺陷，所以需要位置编码来记录位置信息。
  • 要求：
    • It should output a unique encoding for each time-step (word’s position in a sentence).
    • Distance between any two time-steps should be consistent across sentences with different lengths.
    • Our model should generalize to longer sentences without any efforts. Its values should be bounded.
    • It must be deterministic.
  • 两种方法
    • 1. 绝对位置编码：
      • 学习得到的 Positional Embedding
        0表示第一个词的position embedding，1表示最后一个词的position embedding，其余部分按照分数表示。
        但是这样操作，当句子长度不一样时，time-step delta就会不一样。
        eg: 1. 我爱学习 2. 我爱学习，他也爱学习。“我”“爱”之间的距离表示差值是不同的。
      • 用不同频率的sin和cos表达：任意位置之间都可以进行计算。

      PE(pos,2i)=sin(pos/100002i/dmodel)PE(pos,2i+1)=cos(pos/100002i/dmodel)\begin{aligned} PE(pos, 2i) &= sin(pos/10000^{2i/d_{model}})\\ PE(pos, 2i+1) &= cos(pos/10000^{2i/d_{model}}) \end{aligned} PE(pos,2i)PE(pos,2i+1)​=sin(pos/100002i/dmodel​)=cos(pos/100002i/dmodel​)​

      PE函数输出一个和word embedding维度一致的矩阵，行表示词语，列表示词向量
      i表示词向量的位置
      pos 表示词语在句子中的位置
      dmodeld_{model}dmodel​表示词向量的维度
      然后和原来的词向量相加输入到encoder中。
      可以使用三角函数性质来得到一个特定位置的向量，所以该方法蕴含了相对位置信息，但是这种位置信息会在注意力机制那里消失。

Mask

  • mask 表示掩码，它对某些值进行掩盖，使其在参数更新时不产生效果。
  • Transformer中涉及到两种mask：padding mask 和 sequence mask。
    • padding mask 在所有的 scaled dot-product attention 里面都需要用到
    • sequence mask 只有在 decoder 的 self-attention 里面用到。

所以在decoder的masked self-attention需要同时用到两者，具体实现是将两者相加。其余都使用padding mask。

padding mask

每个批次输入序列长度是不一样的也就是说，我们要对输入序列进行对齐padding。

  • 序列较短：在后面填充占位符号，考虑到softmax函数最后要压一下，我们需要填充负无穷的数，这样softmax之后就会变成0.
    但是如果输入的序列太长，则是截取左边的内容，把多余的直接舍弃。

sequence mask

为了使得 decoder 不能看见未来的信息，即在t的时刻，我们的解码输出应该只能依赖于t时刻之前的输出，而不能依赖t之后的输出。
方法：使用一个上三角矩阵，使softmax后的上三角全为0。

优点

  1. RNN 固有的顺序属性阻碍了训练样本间的并行化，对于长序列，内存限制将阻碍对训练样本的批量处理。
    transformer中的并行是指矩阵运算吗？？
  2. 自注意力可以产生更具可解释性的模型。我们可以从模型中检查注意力分布。
  3. 用自注意力层构建 Transformer：传统的网络之间，两点的依赖程度/相似程度是与计算步骤/距离有关系的，但是attention则不会出现这个问题。

预训练 pre-train：根据大量的数据，训练出一个"通用"的模型。

预训练

  • 预训练的优点：

    • 在大规模语料上通过预训练学习通用语言表示对下游任务很有帮助；
    • 预训练提供了更好的模型初始化参数，使得在目标任务上有更好的泛化性能和更快的收敛速度；
    • 预训练是一种有效的正则化方法，能够避免在小数据集上过拟合。

  • 分类：上下文编码器的使用

    • 非上下文嵌入：是静态且与上下文无关的，无法处理多义词； 难以解决未登陆词 oov。
      • word2vec
      • Glove
    • 上下文嵌入：考虑上下文语义，将词在不同上下文的语义做区分。通过对词的 token 加一层编码器得到词的上下文嵌入。

  • NLP的任务分类

      • one sentence

    • Output:
      • one class
      • class for each token
      • copy from input
      • general sequence
  • How:
    1. 只调整task-specific层参数

    • 多义词有歧义：因为词向量是静态的有歧义的。
    • 过于关注词本身的解释性和效果，不关注下游任务。
    • 缺乏对于文本全局信息的建模。
  • 解决：预先训练和微调的机制
    • Pre-training：在大规模数据集上学习尽可能好的通用表示
      因为低层特征具有高可复用性
    • Fine-tuning：利用学习好的通用表示初始化下游的任务网络
      因为高层的特征和任务有强相关性

语义理解模型

  1. ELMo：Feature base
  • Pre-train：纯粹的表示抽取器
    使用双向双层LSTM对语言模型进行建模
    一种无监督训练
    词向量是基于上下文，动态输出的。
    建模不同层次的语义信息。
  • Fine-tuning：
    提取不同的语义信息，加权相加交给下游任务的网络。
    加权相加的参数是可以学习的。
  • 优点：解决一词多义
  • 缺点：
    • 不完全的双向预训练
    • fine-tuning阶段，需要搭建特定的网络。
    • 仅仅学习了语言模型，没有针对句子层面。

问题：是否为自回归模型

  1. GPT：
  • Model-based：预训练模型作为任务网络的一部分参与学习，减少下游任务的新增参数，网络结构修改少。
  • 缺点：
    • 单项语言模型
    • 仅有词向量，没有句向量建模。
  • GPT2，GPT3
  1. Bert：
  • 特点：
    • 沿用了GBT网络结构：12层Transformer
    • 交互式双向语言模型建模
    • 引入了sentence-level预训练任务
    • 更大规模地训练数据集
  • 预训练策略：
    • Token-level：mask-LM任务，交互式双向语言模型。
    • Sentence-level：Next-sentence预测任务，建模句向量。
  • 缺点：
    难以学到词，短语，实体的完整语义。
  1. ERNIE
  • 区别：
    • mask：token-》word
    • 百度语料库
  1. ERNIE 2.0
    持续学习语义理解模型，保证在学习新知识时，不遗忘旧知识。
