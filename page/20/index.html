自组织映射(Self-organizing map, SOM)

生物动机

Biological Motivation
功能相近的神经元分布在一起。
人脑在处理信息时，会将外部的 高维 信号(external multidimensional representation)映射到脑内的二维空间(2-dimensional internal representation)。
即 大脑的自组织机制self-organization ：大脑会生成线性或者平面拓扑的特征 linear or planar topology。

网络特征

  • 无监督学习，竞争学习
  • 通过学习输入空间中的数据，生成一个 低维、离散的映射(Map) ，是一种矢量量化算法，从某种程度上也可看成一种降维算法。
  • 融合了竞争机制和协作机制 (incorporation of competitive and cooperative mechanism)。
  • 使用 近邻关系函数(neighborhood function) 来维持输入空间的拓扑结构：二维映射包含了数据点之间的相对距离。
    输入空间中相邻的样本会被映射到相邻的输出神经元。
    输出空间维持了输入空间的近邻关系，输出空间中近邻的神经元表示了相似的输入。

方差 & 偏差

偏差

【背下来】偏差是指由所有采样得到的大小为m的训练数据集，训练出的模型的输出平均值和真实模型输出之间的偏差。

表现在训练误差不收敛。
描述模型输出结果的期望与样本真实结果的差距，模型表达能力有限导致的系统性错误。

方差

【背下来】是指有所有采样得到的大小为m的训练数据集，训练出的所有模型的输出的方差
描述模型对于给定值的输出稳定性。
分类器对样本分布过于敏感，到指在训练样本较少的时候，出现过拟合

基分类器的错误，是偏差和方差之和
boosting方法通过逐步聚焦分类器分错的样本，减少集成分类器的偏差
Bagging采用分而治之的策略，通过对样本多次采样，分别训练多个模型，减少方差

为什么决策树是常用的基分类器？

  1. 可以方便地将样本权重整合到训练过程中，不需要使用过采样的方法来调整样本权重，每次采样需要耗费时间
  2. 决策树的表达能力和泛化能力，可以通过调节树的层数来做折中
  3. 数据样本扰动对决策树影响较大，因此不同子样本集生成的基分类器随机性就较大。
    这样的不稳定学习器更适合作为基分类器。

Boosting

  • 集成学习大致可分为两大类：Bagging和Boosting

    • Bagging一般使用强学习器，其个体学习器之间不存在强依赖关系，容易并行。
    • Boosting则使用弱学习器，其个体学习器之间存在强依赖关系，是一种序列化方法。
    • Bagging主要关注降低方差，而Boosting主要关注降低偏差。

  • Boosting是一族算法，其主要目标为将弱学习器“提升”为强学习器，大部分Boosting算法都是根据前一系列弱学习器的不足(shortcoming)对样本分布进行调整，再根据新的样本分布训练下一弱学习器，如此迭代M次，最后将一系列弱学习器组合成一个强学习器。

  • boosting算法中，损失函数改变，给不同的样本赋予不同的权重：

当使用一些大规模集成的算法时，参数的问题就出来了，比如说 XGBoost的参数大概在20个左右，GBDT的参数个数也在同一个级别，这种时候，会调参和不会调参在同样的数据集上效果可能就是两码事了。

GridSearchCV

GridSearch和CV，即网格搜索和交叉验证。

缺点：在面对大数据集和多参数的情况下，非常耗时

交叉验证：

  • 将训练数据集划分为K份，K一般为10。
  • 依次取其中一份为验证集，其余为训练集训练分类器，测试分类器在验证集上的精度。
  • 取K次实验的平均精度为该分类器的平均精度。

RandomizedSearchCV

以随机在参数空间中采样的方式代替了 GridSearchCV
对于参数的网格搜索，在对于有连续变量的参数时，RandomizedSearchCV 会将其当作一个分布进行采样，这是网格搜索做不到的。
它的搜索能力取决于设定的n_iter参数，即随机搜索几次

三种 ： KFold、StratifiedKFold、GroupKFold

KFold

直接随机的将数据划分为k折，不考虑 train set 和 eval set 的 class 和 group 分布情况。
只需要传入数据X即可，模型会动态返回两个数组train, test，分别对应 train example 和 test example 的索引。

模型会递归输出 n_splits 次，其实每次相当于单拎出来一个 fold 作为 test 集合，将其他的 fold 拼接起来，这样就看不到具体是如何划分 fold 的了，即每次只能显示输出一个 fold 的索引，即 test。

更常用的是动态根据每次生成的单个 test 索引集合设置 fold_id ：

这样后续只需要 topic_df_non_source[topic_df_non_source["fold"]=1] 即可多次拿出对应的fold。

StratifiedKFold

根据数据集的分布来划分，使得划分后的数据集的目标比例和原始数据集近似，即 train set 和 eval set 的类别分布都和原始数据集分布近似。
这里需要同时传入标签列表 y ，

GroupKFold

保证同一个group的数据不会同时出现在训练集和测试集上。
因为如果训练集中包含了每个group的几个样例，可能训练得到的模型能够足够灵活地从这些样例中学习到特征，在测试集上也会表现很好。但一旦遇到一个新的group它就会表现很差。


可以将 group 理解成 domain，这样让同一个 domain 的数据只出现在 train set 或者 dev set，是想评价模型的迁移能力。
需要同时传入数据集合的 group信息。
