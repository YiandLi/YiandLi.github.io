为什么 BERT 的三个 Embedding 可以进行相加？

假设 token Embedding 矩阵维度是 [4,768]；position Embedding 矩阵维度是 [3,768]；segment Embedding 矩阵维度是 [2,768]。对于一个字，假设它的 token one-hot 是[1,0,0,0]；它的 position one-hot 是[1,0,0]；它的 segment one-hot 是[1,0]。那这个字最后的 word Embedding，就是上面三种 Embedding 的加和。如此得到的 word Embedding，和concat后的特征：[1,0,0,0,1,0,0,1,0]，再过维度为 [4+3+2,768] = [9, 768] 的全连接层，得到的向量其实就是一样的。

再换一个角度理解：直接将三个one-hot 特征 concat 起来得到的 [1,0,0,0,1,0,0,1,0] 不再是one-hot了，但可以把它映射到三个one-hot 组成的特征空间，空间维度是 432=24 ，那在新的特征空间，这个字的one-hot就是[1,0,0,0,0…] (23个0)。此时，Embedding 矩阵维度就是 [24,768]，最后得到的 word Embedding 依然是和上面的等效，但是三个小 Embedding 矩阵的大小会远小于新特征空间对应的 Embedding 矩阵大小。当然，在相同初始化方法前提下，两种方式得到的 word Embedding 可能方差会有差别，但是，BERT还有Layer Norm，会把 Embedding 结果统一到相同的分布。

为什么 Transformer 的 Embedding 最后要乘 dmodel\sqrt{d_{model}}dmodel​​
如果使用 Xavier 初始化，Embedding 的方差为 1/d_model，当d_model非常大时，矩阵中的每一个值都会减小。通过乘一个 dmodel\sqrt{d_{model}}dmodel​​ ，可以将方差恢复到 1。

因为 Bert 使用的是学习式的Embedding，所以 Bert 这里就不需要放大。

Position Encode/Embedding （PE）

  • RNN的优点为考虑了单词之间的位置关系，但是Transformer剥离了RNN，就不能捕捉位置信息的缺陷，所以需要位置编码来记录位置信息。
  • 要求：
    • It should output a unique encoding for each time-step (word’s position in a sentence).
    • Distance between any two time-steps should be consistent across sentences with different lengths.
    • Our model should generalize to longer sentences without any efforts. Its values should be bounded.
    • It must be deterministic.
  • 两种方法

    • 1. 绝对位置编码：

      • 学习得到的 Positional Embedding
        初始化一个 512×768 的矩阵作为位置向量，让它随着训练过程更新。
        缺点是不可扩展，即如果预训练最大长度为 512 的话，那么最多就只能处理长度为 512 的句子，再长就处理不了了
      • 用不同频率的sin和cos表达：任意位置之间都可以进行计算。

      PE(pos,2i)=sin(pos/100002i/dmodel)PE(pos,2i+1)=cos(pos/100002i/dmodel)\begin{aligned} PE(pos, 2i) &= sin(pos/10000^{2i/d_{model}})\\ PE(pos, 2i+1) &= cos(pos/10000^{2i/d_{model}}) \end{aligned} PE(pos,2i)PE(pos,2i+1)​=sin(pos/100002i/dmodel​)=cos(pos/100002i/dmodel​)​

      PE函数输出一个和word embedding维度一致的矩阵，行表示词语，列表示词向量
      i表示词向量的位置
      pos 表示词语在句子中的位置
      dmodeld_{model}dmodel​表示词向量的维度
      然后和原来的词向量相加输入到encoder中。
      可以使用三角函数性质来得到一个特定位置的向量，所以该方法蕴含了相对位置信息，但是这种位置信息会在注意力机制那里消失。
      代码实现：https://blog.csdn.net/qq_34771726/article/details/102918440

      • 为什么公式中有一个魔法值 10000？
        表示不同周期，PE 在不同位置上的变化。使用 10000 很可能就是确保循环周期足够大，以便编码足够长的文本。如果为 1，则周期很短，即相邻位置的位置编码也差不多。

Mask

  • mask 表示掩码，它对某些值进行掩盖，使其在参数更新时不产生效果。
  • Transformer中涉及到两种mask：padding mask 和 sequence mask。
    • padding mask 在所有的 scaled dot-product attention 里面都需要用到
    • sequence mask 只有在 decoder 的 self-attention 里面用到。

所以在decoder的masked self-attention需要同时用到两者，具体实现是将两者相加。其余都使用padding mask。

padding mask

每个批次输入序列长度是不一样的也就是说，我们要对输入序列进行对齐padding。

  • 序列较短：在后面填充占位符号，考虑到softmax函数最后要压一下，我们需要填充负无穷的数，这样softmax之后就会变成0.
    但是如果输入的序列太长，则是截取左边的内容，把多余的直接舍弃。

sequence mask

为了使得 decoder 不能看见未来的信息，即在t的时刻，我们的解码输出应该只能依赖于t时刻之前的输出，而不能依赖t之后的输出。
方法：使用一个上三角矩阵，使softmax后的上三角全为0。

优点

  1. RNN 固有的顺序属性阻碍了训练样本间的并行化，对于长序列，内存限制将阻碍对训练样本的批量处理。

  3. 用自注意力层构建 Transformer：传统的网络之间，两点的依赖程度/相似程度是与计算步骤/距离有关系的，但是attention则不会出现这个问题。

命名实体识别（Named Entity Recognition， NER）：识别出句子中命名实体的边界和类别的任务。

  • 对于规则较强的命名实体，比如网址，email，isbn等，可以通过正则表达式匹配。
  • 对于较短的命名实体，比如人名，可以通过分词确定边界，通过词性标注模块确定类别。
    这样就不需要专门的命名实体识别模块和语料库，只需要普通的分词词性标注模块。
    微软系列的语料库，以颗粒大著称，命名实体被标注为了一个词语。

基于规则的命名实体识别

基于规则的音译人名识别

在一段待识别的文本中，如果音译字符连续出现，则很有可能来自于一个音译人名。
词典中会有音译任命nrf这个词性，但是音译实体是无穷多的，不可能全部被收录，基于此设定一个规则：

  1. 若粗分结果中，某单词的被选词性含有nrf，则
  2. 向右扫描，遇到音译人命词库中的词语则合并，否则终止扫描。

这个规则只负责将分词模块切碎的已知音译人名合并起来，并不负责识别未知的音译人名片段。

基于规则的日本人名识别

日本人名长度一般为3-5位，且含明确的姓氏，用规则识别起来更加简单。

  • HanLp中规则如下：
    • 匹配日本人名的姓氏和名字，分别计作x和m。
    • 将连续的xm合并为日本人名，标注为nrj。
    • 考虑到日本人经常以居住地附近的山水作为姓名，比如龟山公园，可能存在姓氏与名字组成了地名的情况，这些情况被标注为A。

基于规则的数词英文识别

数词和英文单词同样是常见的命名实体，适合利用规则进行识别。
正则表达式匹配开销较大，HanLp中采用将相同类型的连续字符分别合并。

  • 过程：
    • 首先获得每个字符的类型。（单字节，分隔符，中文字符，字母，数字，序号等）
    • 然后扫描字符串的类型数组，将类型相同的字符合并，利用其类型确定词性（数词m / 英文nx）。

基于统计的命名实体识别

基于层叠隐马尔可夫模型的角色标注框架

为构成命名实体的短词语打标签，标签序列满足某种模式则识别为某种命名实体。
对于如何打标签，基于规则的方式是根据词典匹配规则，基于统计的方式是根据隐马尔可夫模型的预测。
该框架中，首先进行分词，然后进行标注。所以标注模块的输入是分词模块的输出，并且两个模块都由隐马尔可夫模型驱动，所以称为层叠隐马尔可夫模型。

概率与似然

概率：已知一些参数的情况下，预测接下来的观测所得到的结果。
似然：已知某些观测所得到的结果时，对有关事物的性质的参数/概率进行估计。可以将似然看作为参数的靠谱程度。
在这种意义上，似然函数可以理解为条件概率的逆反。

似然函数

似然函数这个概念是建立在频率学派之下，参数是一个真值，并不是随机变量。所以参数没有分布，参数也没有概率。

  • L(θ∣x)L(\theta | x)L(θ∣x)，表示的是在给定样本 x 的时候，哪个参数 θ\thetaθ 使得 x 出现的可能性最大。
    它在数值上等于给定参数 θ\thetaθ 后变量 X 取某个值的概率：p( X=x∣ θ )p(\ X=x|\ \theta\ )p( X=x∣ θ )。
  • 这个值越大，说明给定参数θ\thetaθ，数据值x出现的概率就越大。同样也表示数据值x出现时，参数为θ\thetaθ的可能性就越大，就越靠谱。
    似然值越大，表示在该参数下，事件越有可能发生。似然值越小，在该参数下事件越不可能发生。
    从这个角度来看，MLE也很好理解了：找到一个参数的估计，使得在该参数估计下，事件发生的可能性最大。
  • 从定义上，似然函数 和 概率密度函数是完全不同的两个数学对象：前者是关于X的函数，后者是关于θ\thetaθ的函数。
    所以这里的等号 = 理解为函数值形式的相等。它们不是同一个函数，但是具有相同的函数形式。

极大似然估计

利用已知的样本结果信息，反推最具有可能（最大概率）导致这些样本结果出现的模型参数值！
极大似然估计中采样需满足一个重要的假设，就是所有的采样都是独立同分布的。

Seq2Seq

介绍

通过encoder-decoder，根据给定的序列，通过特定的方法生成另一个序列的方法。

原理

  • 首先，Seq2Seq的输入序列和输出序列长度是不可变的。

  • 最基础的Seq2Seq模型包含了三个部分，即Encoder、Decoder以及连接两者的中间状态向量C：

    在Seq2Seq结构中，编码器Encoder把所有的输入序列都编码成一个统一的语义向量Context，然后再由解码器Decoder解码。在解码器Decoder解码的过程中，不断地将前一个时刻的输出作为后一个时刻的输入，循环解码，直到输出停止符为止。

    • encoder: 获取语义向量最简单的方式就是直接将最后一个输入的隐状态作为语义向量C，
      也可以对最后一个隐含状态做一个变换得到语义向量，
      还可以将输入序列的所有隐含状态做一个变换得到语义变量。
    • decoder: 最简单的方式是将encoder得到的语义变量作为初始状态输入到decoder的RNN中，得到输出序列。
      可以看到上一时刻的输出会作为当前时刻的输入，而且其中语义向量C只作为初始状态参与运算，后面的运算都与语义向量C无关。

      decoder处理方式还有另外一种，就是语义向量C参与了序列所有时刻的运算，上一时刻的输出仍然作为当前时刻的输入，但语义向量C会参与所有时刻的运算。

弊端和改进

  1. 在seq2seq结构中将上一时刻输出值输入进网络时，那么某一时刻输出值错误就会导致后面全错，这种蝴蝶效应格外明显。
    为了解决这个问题，Google提出了大名鼎鼎的Scheduled Sampling
    即在训练中xtx_txt​按照一定概率选择输入yt−1y_{t-1}yt−1​或t−1t-1t−1时刻对应的真实值，即标签，不仅能加快训练速度，也能提高训练精度。
  2. LSTM模型虽然具有记忆性，但是当Encoder阶段输入序列过长时，解码阶段的LSTM也无法很好地针对最早的输入序列解码。
    除此之外，如果按照上述方式实现，只用到了编码器的最后一个隐藏层状态，信息利用率低下。
    Attention注意力分配的机制被提出，就是为了解决这个问题。

one-hot编码的弊端

首先one-hot的特点：

  1. 纬度太高 high dimension
  2. 每个单词太过于离散 discrete for each element
  3. 单独表示 local representation：只有一位表示其向量
  • 这些特点带来的弊端：无法表现两个词向量间的相似度

分布式表示 distributed representation

  • 特点：
    1. 纬度相对较小 low dimension
    2. 每一位都是实数 real number
    3. 每个元素联合表示 distributed
  • 常见的分布式表示方式为Word2Vec表示方式

生成式模型和判别式模型

在拥有已经训练好数据集后，预测一个新的输入样本时，按照模型的不同，可以划分为两种模型。

生成式模型

  • 已知y的知识，所以可以生成样本结果，其预测的根据是联合概率P(x,y)。
  • 没有一个明确的分类边界
  • 首先生成所有可能的模型，然后找到概率最大的哪一个作为预测结果。
  • 例子：隐马尔可夫模型（HMM），K近邻KNN，朴素贝叶斯等

判别式模型

  • 有明确的分类边界，输入属性X可以直接得到Y，或者score，然后根据区间得到分类结果。
  • 根据训练数据得到分类函数和分界面，然后计算条件概率P(y|x)，找出新样本的分类。
  • 例子：神经网络，逻辑回归，线性回归
  • CRF（Conditional Random Field）中，Conditional表示条件概率 Conditional probability，可知是一个判别模型。
