具体

  1. 只传入input_sequence，或者past_key_values=None。
    output 回传两个属性参数，logits 和 past_key_values：
    1. logits 为每个 token 的输出值和字典每个token embedding 的点乘相似度，维度为[ batch_size, seq_len, vocab_size ] 。
    2. past_key_values 为GPT2模型中12层Block模块计算后得到的存储12个present张量 的presents元组，这里用到了缓存机制。

表示每一层上，每一个 token 的 key 和 value 值，他们是不会变的，因为都仅仅是和自己前面token进行attention计算。而当前时间步也仅仅需要前面token的key和value进行计算。注意 GTP 只有 masked attention，没有 self attention，所以每一个 token 的前向传播和它后面的 token 没有关系。
大小为：length config.n_layers with each tensor of shape (2, batch_size, num_heads, sequence_length, embed_size_per_head)

而 T5的类 transformer 结构中，这里引用 Transformer里面的缓存机制： 我们首先来考察decoder的self_attention， mask为两部分tgt_mask和self_attention_mask。其中，tgt_mask和tgt的长度相关， self_attention_mask为三角矩阵的形式(对角线及下三角为0， 上三角为很大的负数，如-1e9)，可以保证在计算某一个位置的token时，这一位置之后的token对该位置的输出结果不产生影响。

  1. 传入input_sequence和past_key_values：
    这里相当于将input_sequence拼接到past_key_values对应的输入后面，所以一般input_sequence为单个token（可见下面代码 ），这个input_sequence 其实就是上一步生成的 token，在下一步需要把它作为 q，对缓存的 past_key_values 进行 attention。
    因为有缓存机制，所以可以指定past_key_values，来减少运行成本。
    输出还是logits 和 past_key_values。

  2. 额外传入参数 labels，output 会多生成一个 loss，用来进行自回归预训练。

    • 该参数中-100 的位置不计入 loss，相当于 mask 。
      同时需要和 inputs 进行区分， inputs 使用 pad_token_id 进行 padding，因为需要过一层 embedding layer ，而 label 在相同位置上为 -100 。
    • 注意，这种方法传入时需要在序列前加一个 <sos>（原因见下方LMHeadModel 源码解读）。output会多出来一个loss属性，对应自回归损失。

  3. 模型还可以输出每一层每个token的隐状态，在初始化时候传入参数 output_hidden_states=True，或者通过model.config.output_hidden_states=True来设置。
    然后就能够得到一个Tuple of torch.FloatTensor，元组长度为layer_num+1，对应 embedding 层和后面的decoder层。元祖每一个元素都是一个 tensor，尺寸为batch_size, seq_len, hidden_state 。

单句生成

  • 区分下不同时间步骤，model的forward方法的参数：

    • past_key_values：
      第一次调用时past_key_values=None，第一次之后的调用都是模型内部自己更新的值。
    • inputs：
      第一次调用时就是整个序列([batch_size, seq_len])，随后遍历都是后面新加上的单个 token([batch_size, 1])。

  • 其实模型的 input 序列就是 [past_key_values, inputs]。

  • generated 汇总了生成的所有 token id

LMHeadModel 源码

初始化方法

一个GPT2编码器 self.transformer，和一个用于预测的token分类器self.lm_head。

  • label存在则输出loss
    shift_logits维度为 [batch_size, seq_len-1, vocab_size]，是删除了最后一个 token的，即生成的 token 的。
    shift_labels维度为 [batch_size, seq_len-1]，是删除了第一个token 的。
    两者长度相同，但是有个坑，即shift_logits和shift_labels是错位的 。
    比如labels/input_id为[0,1,2,3]， 则模型输出的logits对应了[1,2,3,4]。
    所以 shift 后，shift_logits对应了 [1,2,3]，shift_labels对应了[1,2,3]，正好匹配，一个删了头，一个删了尾巴。
    所以在训练自回归模型时，这里 label/input_id首个token应该是 <sos> 占位符号。
  • lm_logits为 最后一层所有token的词典 token相似度，所以 维度为batch_size, seq_len, vocab_size.
  • past_key_values则是 GPT2Model内部进行生成/更新的，但是我们还是需要在代码中先获取，再传入。
