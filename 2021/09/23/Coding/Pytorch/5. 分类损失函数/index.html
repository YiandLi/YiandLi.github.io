5. 分类损失函数

Posted on 2021-09-23 In Coding , PyTorch
Symbols count in article: 1.6k Reading time ≈ 8 mins.

Pytorch 中的损失函数分为两种形式：

  1. torch.nn.functional库中的 函数形式，通过传入神经网络预测值和目标值来计算损失函数。
  2. torch.nn库中的 模块形式，通过调用模块的方法来计算最终的损失。

这里主要介绍分类问题的损失

二分类

CLASS torch.nn.BCELoss

L={l1,…,lN}⊤ln=−wn[yn⋅log⁡xn+(1−yn)⋅log⁡(1−xn)]\begin{aligned} L &= \left\{l_{1}, \ldots, l_{N}\right\}^{\top} \\ l_{n} &= -w_{n}\left[y_{n} \cdot \log x_{n}+\left(1-y_{n}\right) \cdot \log \left(1-x_{n}\right)\right] \end{aligned} Lln​​={l1​,…,lN​}⊤=−wn​[yn​⋅logxn​+(1−yn​)⋅log(1−xn​)]​

LLL 为 batch_size
wnw_{n}wn​ 是一个超参数，对于单标签二分类，设不设置 wnw_{n}wn​ ，没有影响。

构造参数

  • weight=None (Tensor, optional)：放缩权重，形状和 batch_size 一致，tensor 类型。
  • reduction='mean' (string, optional)：可选 ‘none’，‘mean’，‘sum’ 。
    如果设置了 size_average 和 reduce 参数，则 reduction 参数会被忽略。
  • size_average=None (bool, optional)：默认为True
  • reduce=None (bool, optional)：默认为True

基本用不到 size_average 和 reduce 两个参数，直接使用reduction可以得到所有效果。

调用参数

  • Input (∗)：正类标签的概率值，必须要是 sigmoid 后的概率结果。
  • Target (∗)：标签数据值。形状和 Input 一致，必须是torch.float类型。

CLASS torch.nn.BCEWithLogitsLoss

和 BCELoss 类似，不过不需要提前使用 sigmoid 得到概率。

多分类

Method torch.nn.functional.log_softmax

先 softmax，然后加 log :

LogSoftmax⁡(xi)=log⁡(exp⁡(xi)∑jexp⁡(xj))\operatorname{LogSoftmax}\left(x_{i}\right)=\log \left(\frac{\exp \left(x_{i}\right)}{\sum_{j} \exp \left(x_{j}\right)}\right) LogSoftmax(xi​)=log(∑j​exp(xj​)exp(xi​)​)

参数：

  • input (Tensor) : 输入
  • dim (int) : log_softmax 的维度
  • dtype (torch.dtype, optional) : 返回张量的 dtype。 如果设置了，则会先将 input 转为目标 dtype 然后进行 log_softmax 操作。可以有效防止数值 overflow 。

面试问题：

因为softmax会进行指数操作，也就是 exp() 的输入比较大的时候，指数形式会变的很大，可能就会产生 overflow 。python2中它的上限为 709。

所以考虑用 x-max(x) 替代原始的 x 输入到 exp() 中。值域变为了 (-∞,0]，即可有效防止上溢。
但是这里仍然可能存在下溢的可能性：

  1. 分母会不会下溢：不会！因为减去最大值后，分母一定有一项为 0，exp 后值为 1。(对应 max(x) 那一项 exp(0)=1）
  2. 分子上的下溢，仍然可能导致表达式估值为零。但是在预测时，为零不是什么问题，因为取的是最大值。
    如果先做 softmax ，然后再做 log，会导致下溢的情况产生。但是可以在前面加一个 log：log(exp(分子)/exp(分母)) = 分子-log(exp(分母))， 这样分子就不会被计算 exp()了（可参考下方推导）。

所以这里，一般会通过分子分母同时除以一个 eM{e^{M}}eM 的方法进行归一：

M=max⁡(x1,x2,...,xn)f(xi)=exiex1+ex2+…+exn=exieMex1eM+ex2eM+…+exneM=e(xi−M)∑jne(xj−M)\begin{aligned} & M = \max (x_1,x_2,...,x_n) \\ & f\left(x_{i}\right) = \frac{e^{x_{i}}}{e^{x_{1}}+e^{x_{2}}+\ldots+e^{x_{n}}} = \dfrac{\dfrac{e^{x_{i}}}{e^{M}}}{\dfrac{e^{x_{1}}}{e^{M}}+\dfrac{e^{x_{2}}}{e^{M}}+\ldots+\dfrac{e^{x_{n}}}{e^{M}}} =\frac{e^{\left(x_{i}-M\right)}}{\sum_{j}^{n} e^{\left(x_{j}-M\right)}} \\ \end{aligned} ​M=max(x1​,x2​,...,xn​)f(xi​)=ex1​+ex2​+…+exn​exi​​=eMex1​​+eMex2​​+…+eMexn​​eMexi​​​=∑jn​e(xj​−M)e(xi​−M)​​

所以 log 并不是防止上溢出的原因，但是可以起到防止分子下溢的功能。
加上 log 之后变为：

log⁡[f(xi)]=log⁡(exiex1+ex2+…+exn)=log⁡(exieMex1eM+ex2eM+…+exneM)=log⁡(e(xi−M)∑jne(xj−M))=log⁡(e(xi−M))−log⁡(∑jne(xj−M))=(xi−M)−log⁡(∑jne(xj−M))\begin{aligned} \log \left[f\left(x_{i}\right)\right] &=\log \left(\frac{e^{x_{i}}}{e^{x_{1}}+e^{x_{2}}+\ldots+e^{x_{n}}}\right) \\ &=\log \left(\frac{\frac{e^{x_{i}}}{e^{M}}}{\frac{e^{x_{1}}}{e^{M}}+\frac{e^{x_{2}}}{e^{M}}+\ldots+\frac{e^{x_{n}}}{e^{M}}}\right)\\ &=\log \left(\frac{e^{\left(x_{i}-M\right)}}{\sum_{j}^{n} e^{\left(x_{j}-M\right)}}\right) \\ &=\log \left(e^{\left(x_{i}-M\right)}\right)-\log \left(\sum_{j}^{n} e^{\left(x_{j}-M\right)}\right)\\ &=\left(x_{i}-M\right)-\log \left(\sum_{j}^{n} e^{\left(x_{j}-M\right)}\right) \end{aligned} log[f(xi​)]​=log(ex1​+ex2​+…+exn​exi​​)=log(eMex1​​+eMex2​​+…+eMexn​​eMexi​​​)=log(∑jn​e(xj​−M)e(xi​−M)​)=log(e(xi​−M))−log(j∑n​e(xj​−M))=(xi​−M)−log(j∑n​e(xj​−M))​

为什么需要加 log ：
使用log可以将乘法转换为加法，或许（不确定）可以起到加快运算的效果。
同时 求导更加方便，可以加快反向传播的速度。

∂∂xilog⁡softmax⁡=∂∂xi[xi−log⁡∑jnexp⁡(xj)]=1−softmax⁡(xi)\begin{aligned} \frac{\partial}{\partial \mathrm{x}_{\mathrm{i}}} \log \operatorname{softmax} & =\frac{\partial}{\partial \mathrm{x}_{\mathrm{i}}}\left[\mathrm{x}_{\mathrm{i}}-\log \sum_{\mathrm{j}}^{\mathrm{n}} \exp \left(\mathrm{x}_{\mathrm{j}}\right)\right] \\ & =1-\operatorname{softmax}\left(\mathrm{x}_{\mathrm{i}}\right) \end{aligned} ∂xi​∂​logsoftmax​=∂xi​∂​⎣⎢⎡​xi​−logj∑n​exp(xj​)⎦⎥⎤​=1−softmax(xi​)​

CLASS torch.nn.NLLLoss

负对数似然损失函数（negative log likelihood loss）。
输入为 log_softmax() 之后的结果。

构造参数

  • weight=None (Tensor, optional)：类别的权重，是一维tensor，长度等于类别数量。
  • ignore_index=-100, (int, optional)：
    第ignore_index个类别的样本不构成损失（reduction='sum' or 'mean'），或者返回的损失为 0（reduction='none'）。
    • 可以使用 CrossEntropyLoss().ignore_index 得到该 id
  • reduction='mean' (string, optional)：'none' | 'mean' | 'sum'

调用参数

  • Input：形状为 N×CN\times CN×C ， CCC 为类别数量。
    元素为不同类别的 对数似然值，需要使用torch.nn.functional.log_softmax(output, -1) 得到。
  • Target：形状为 NNN ，元素为真实样本值，索引从0开始。必须是torch.long类型。

CLASS torch.nn.CrossEntropyLoss

直接对模型输出的 logit 使用，内部相当于 log_softmax + NLLLoss。

构造参数

  • weight=None (Tensor, optional)
  • ignore_index=-100 (int, optional): 不会计算 target 中 label=-100 位置的损失。
    在做序列标注时，将[cls]和[sep]的label设置为-100。
  • reduction='mean' (string, optional)
  • label_smoothing=0.0 (float, optional)： 属于 [0.0,1.0][0.0, 1.0][0.0,1.0] 。torch@1.11.0 新功能。
    将原始的 hard-label 改为 smooth-label，避免过拟合，缓解错误标签带来的影响。

ti={1−εif    i=trueε/(class_num−1)otherwise  t_i=\left \{\begin{array}{lc}1-\varepsilon&\text{if}\;\;i=\text{true}\\\varepsilon/(\text{class\_num}-1)&\mathrm{otherwise}\;\end{array}\right. ti​={1−εε/(class_num−1)​ifi=trueotherwise​

调用参数

  • Input：形状为 (N,C)(N,C)(N,C)
  • Target：必须是torch.long类型
    • 如果表示one-hot索引，形状为 (N)(N)(N) ，元素属于 [0,C−1][0,C-1][0,C−1] 。
    • 如果表示类别的概率值，则形状和Input一致，元素属于 [0,1)[0,1)[0,1) 。
