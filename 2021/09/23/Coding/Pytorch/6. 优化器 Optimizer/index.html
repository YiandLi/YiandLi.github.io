6. 优化器optimizer和调度器scheduler

Posted on 2021-09-23 In Coding , PyTorch
Symbols count in article: 2.4k Reading time ≈ 12 mins.

知乎 ： https://zhuanlan.zhihu.com/p/346205754?utm_medium=social&utm_oi=73844937195520

介绍

  • 我们需要在训练的每一步中调整模型的参数。优化算法 Optimization algorithm定义了这个过程。该算法被封装在了optimizer对象中。
  • 优化器的初始化需要传入模型中待优化的参数和学习率。
  • 在训练的loop中，我们经常需要3个步骤：
    • optimizer.zero_grad()：梯度清空
    • loss.backwards()：传播梯度
    • optimizer.step()：根据梯度调整参数

torch.optim 库

torch.optim是一个实现了多种优化算法的库，可以保存当前状态并且根据计算的梯度来更新参数。

例子

创建

  • 需要传入一个包含Parameter参数的可迭代对象，最简单的是model.parameters()，然后设置超参数(defaults)选项，比如学习率，权重衰减等。
  • 此外，用于分层学习率：optimizer也支持为每个参数单独设置选项，传入一个元素为字典的可迭代对象。
    每一个字典都分别定义了一组参数：必须包含一个param键及其超参数(defaults)选项（注意这里params键的值为Parameter类型元素的列表），还可以加上针对这些参数的参数选项（比如weight_decay,lr,momentum）。
  • 还可以使用分层学习率。后面传递的总参数选项，用于没有定义这些选项的parameter。这在只想改变一个参数组选项时是很有用的。
权值衰减/l2正则

首先正则化主要是为了防止过拟合，而过拟合一般表现为模型对于输入的微小改变产生了输出的较大差异。
这主要是由于有些参数w过大的关系，通过对||w||进行惩罚，可以缓解这种问题。
参数b对于输入的改变是不敏感的，不管输入改变是大还是小，参数b的贡献就只是加个偏置而已。
举个例子，如果你在训练集中，w和b都表现得很好，但是在测试集上发生了过拟合，b是不背这个锅的，因为它对于所有的数据都是一视同仁的（都只是给它们加个偏置），要背锅的是w，因为它会对不同的数据产生不一样的加权。

更新参数

所有的优化器都有step()方法来更新参数。有两种调用方式：

  • 方法一
    optimizer.step()：用在参数已经存好梯度的情况下，比如在调用loss.backward()方法之后。
  • 方法二
    有些优化算法需要多次重新计算函数，因此需要传递一个闭包，使它们能够重新计算您的模型。闭包应该清除梯度，计算损失，然后返回。

基类

torch.optim.Optimizer(params, defaults)

初始化函数：

__init__(self, params, defaults)：

  • params：parameter的迭代对象，或者是一个字典。
  • 核心为：

可以看到，如果params传入非字典形式（model.parameters()），则构造函数会自动创建字典，键为'params'，值为传入的parameter。如果是字典形式，则不用转换。
强调一下，__init__方法生成了两个实例变量和一个字典列表：
- self.param_groups = []
- self.defaults = defaults
- param_groups = [{'params': list, ('lr': -, ..)}]
然后对self.param_groups中每个字典元素使用self.add_param_group()方法进行处理。

其他函数：

  • Optimizer.add_param_group(param_group)：param_group为字典形式，向self.param_groups中增加参数组。
    源码实现过程：

    • 将param_group['params']转化为list形式。
    • 使用self.defaults中设置的超参数给这个param_group赋值。
    • 将赋值后的param_group字典放入self.param_groups。

  • Optimizer.step()：完成一次模型参数的更新。基层Optimizer定义了 step 方法接口，不同的子类优化器进行了不同的实现。

  • Optimizer.zero_grad()：将Tensor的梯度清空

  • Optimizer.state_dict()：返回优化器的字典

  • Optimizer.load_state_dict(dict)：根据字典加载优化器\

上面两个方法可用来实现模型训练中断后继续训练功能

torch.optim.lr_scheduler 库

用于根据epoch来动态调整lr。
原理：在每个epoch中，计算参数组的lr，然后更新optimizer对应参数中的lr。

基类

Class：torch.optim.lr_scheduler._LRScheduler

主要attribute：

  • optimizer：关联的优化器
  • last_epoch：记录epoch数
  • base_lrs：记录初始学习率

初始化函数

__init__(self, optimizer, last_epoch=-1, verbose=False)

  • 参数：
    optimizer
    last_epoch = -1：最后一次epoch的index。
  • 源码实现：
    • 根据last_epoch是否为-1初始化optimizer.param_groups中每个字典的initial_lr键值对，并将所有的initial_lr放入self.base_lrs。
      · 如果last_epoch==-1，则代表初次训练模型，此时会向optimizer里的各参数字典中添加初始学习率：group.setdefault('initial_lr', group['lr'])
      · 若last_epoch传入值大于-1，则代表从某个epoch开始继续上次训练，此时要求optimizer的参数组中有initial_lr初始学习率信息，否则报错。
    • 初始化内部with_counter函数：确保lr_scheduler.step()是在optimizer.step()之后调用的。
    • self.optimizer._step_count = 0
      self._step_count = 0
      self.last_epoch = last_epoch
      self.verbose = verbose
    • 执行一次self.step()

其他函数：

  • step(epoch=None): 子类公用。

    • 对last_epoch自增，然后调整学习率。
    • 调用子类的get_lr()方法得到学习率，并且更新到optimizer的para_groups属性中。
    • 将所有的学习率更新到self._last_lr中。

  • get_lr(): 子类需要实现，不同子类继承后会有不同的实现.其返回值是[lr1, lr2, ...]结构。

  • get_last_lr(): 子类公用，记录的最后一次optimizer各参数组里更新后的学习率信息。

  • print_lr(is_verbose, group, lr, epoch=None): 显示 lr 调整信息

  • state_dict(): 子类可能会重写

  • load_state_dict(state_dict): 子类可能会重写

常用的学习率调整方法

StepLR

StepLR(optimizer, step_size, gamma=0.1, last_epoch=-1, verbose=False)

  • 功能：等间隔调整学习率，也就是每隔一段时间去调整学习率，最终学习率的图形是阶梯形状逐渐下降（gamma小于1）或者上升（gamma大于1，估计没有人会这么设置吧）
  • 参数：
    • step_size：调整学习率的间隔数
    • gamma：调整系数，也就是每次调整学习率，都将之前的学习率乘以这个系数，具体调整方式：lr=lr∗gammalr=lr*gammalr=lr∗gamma

MultiStepLR

MultiStepLR(optimizer, milestones, gamma=0.1, last_epoch=-1, verbose=False)

  • 功能：按给定间隔调整学习率
  • 参数：
    • milestones：设定调整时刻数，这个参数是一个列表，其中每一项都是一个整数，代表所需要调整学习率的epoch时刻。
      例如：[50, 125, 180]表示分别在epoch为 50，125，180时进行调整。
      具体图像和StepLR类似。

ExponentialLR

ExponentialLR(optimizer, gamma, last_epoch=-1, verbose=False)

  • 功能：按指数衰减调整学习率
  • 参数：
    • gamma：指数的底，通常设置为接近1的数（0.9），调整方式：lr=lr∗gamma∗∗epochlr=lr*gamma**epochlr=lr∗gamma∗∗epoch

更加常用的warmup等策略可以使用transformers库中的调度器

“为什么warmup有效 ？” 这个问题目前还没有被充分证明。

  • 直觉：
    • 深层网络随机初始化差异较大，如果一开始以较大的学习率，初始学习带来的偏差在后续学习过程中难以纠正。
      若一开始以较大的学习率学习，易对数据造成过拟合，需要经过之后更多轮的训练进行修正。
    • 而浅层神经网络一开始使用幅度大的更新并不会导致方向错误。
  • 理论：
    • 缓解模型在初期对 mini-batch 过拟合的现象
    • 保持模型深层的稳定性。在训练的过程中，如果有mini-batch内的数据分布方差特别大，这会导致模型学习剧烈波动，使其学得的权重很不稳定，这在训练初期最为明显，最后期较为缓解。
      初期在预热的小学习率下，模型可以慢慢趋于稳定,等模型相对稳定后再选择预先设置的学习率进行训练,使得模型收敛速度变得更快，模型效果更佳。
  • 实现：warmup_lr的初始值是跟训练预料的大小成反比的，也就是说训练预料越大，那么warmup_lr 初值越小
    • 使得学习率可以适应不同的训练集合size
    • 即使不幸学习率设置得很大，那么也能通过warmup机制看到合适的学习率区间（即训练误差先降后升的关键位置附近），以便后续验证。
  • 不同的策略和代码实现：Transformers之自定义学习率动态调整
