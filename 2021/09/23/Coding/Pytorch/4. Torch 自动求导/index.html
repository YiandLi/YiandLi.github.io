4. Torch autograd 自动求导

Posted on 2021-09-23 In Coding , PyTorch
Symbols count in article: 1.3k Reading time ≈ 9 mins.

训练神经网络时，最常用的方法就是后向传播 back propagation：从loss中得到参数的的梯度，然后根据梯度和学习率更新参数。
PyTorch有一个内置的微分引擎 torch.autograd，提供自动计算梯度。

计算图

autograd将所有的数据和运行操作保存在一个有向无环的计算图图 directed acyclic graph (DAG) 中。在这个图中，叶节点为输入张量，跟节点为输出值。跟踪计算图中根节点到叶节点的路径可以帮助通过链式求导法则得到输出对每个输入的导数。
DAG是动态的，即在每次.backward之后，autograd都会开始填充新的图形，所以可以在每次迭代中更改形状、大小和操作。

  • 前向传播：
    • 计算输出张量
    • 维度计算图中的梯度公式
  • 当根结点调用了backward()方法后，开始后向传播：
    • 根据.grad_fn()函数求梯度
    • 将梯度累加到张量的.grad属性中
    • 使用链式法则，传播梯度

设置可导张量

  • 我们将参数设定为Variable数据类型，即在参数中增加requires_grad = True。
    或者在参数后面指定x.requires_grad_(True)。
    张量的grad_fn属性保存了后向传播函数的信息。

在平常使用中，不需要这些操作，因为nn.Moudle模块已经自定义了可导变量。

  • Function类对象用于将张量放入计算图中，并且跟踪前向计算，和后向传播过程中的梯度。

计算梯度

在使用损失函数得到loss之后，可以使用loss张量的backword()函数得到参数的梯度。

冻结参数

  • 默认情况下，所有requires_grad = True的参数都会跟踪自身的计算历史，并且提供梯度计算。
  • 但是在我们不需要后向计算时，比如在dev集合上，可以使用torch.no_grad()模块环绕对参数的计算过程。
  • 或者，针对某一个张量，我们可以调用其自身的detach()方法来冻结其参数。
  • required_grad=false会计算误差，不计算wb的梯度（原因在于一个网路如果是ABC层这样的连接方式，B不求梯度，但是A还是会获得梯度，这就需要计算B的误差，从而传播到A，而计算B中参数的梯度就没有必要了）
    而detach会切断反向传播，不计算误差也不计算梯度，即前面层的参数也无法计算误差和梯度了。
  • 好处：
    • 冻结参数
    • 加速前向传播

detach(), to(), detach_() 方法对比

https://www.cnblogs.com/wanghui-garcia/p/10677071.html

  1. detach()，返回一个新的 Variable 变量，但是其required_grad=false，不会被计算梯度。
  • 同时调用 backword()时候，反向计算在此被 截断，导致计算图上该变量前面参数的梯度无法计算。
    函数对变量进行的操作为 .grad_fn() = None，required_grad=False ，如果.grad_fn() = None的变量调用 backward()方法则会报错。
  • 新老张量共享同一块内存，修改其中一个tensor的值，另一个也会改变。如果对其中一个tensor执行某些内置操作，则会报错，例如resize_、resize_as_、set_、transpose_。
    对返回的变量使用in-place operation后(见后文)，这个更改会被autograd追踪，所以原始张量就不能再backward()了。
  1. .data()，区别在于对其进行的修改不会被 autograd 追踪，但是仍然会改变原始变量，导致backward()返回了错误的梯度。
  2. .detach_()，将一个变量从计算图中分离出来，设置为叶子结点。同样也是设置.grad_fn() = None，required_grad=False，只不过这里是对变量本身进行修改，而不是返回一个新的变量。

In-place

所有的变量都会自动记录其参与的 in-place operations，如果监测到有in-place operation出现在backward()之后，则backward()方法会报错。这种机制保证了只要backward()没有报错，则梯度计算一定是正确的。

底层

  • torch.autograd.function
    我们通常使用 pytorch 所提供的nn.Module （例如nn.Conv2d, nn.ReLU等）作为基本单元。而这些 Module 通常是封装了可以自动求导的方法 autograd function，以其作为真正实现的部分。
    一个模型的运算部分由 autograd functions 组成，这些 autograd functions 内部定义了 forward，backward 用以描述前向和梯度反传的过程，组合后可以实现整个模型的前向和梯度反传。
    以torch.autograd.function中所定义的Function类为基类，我们可以实现自定义的autograd function，所实现的 function 需包含forward及backward两个方法。
  • torch.autograd.functional
    Tensor类的backward方法，实际调用的就是torch.autograd.backward这一接口。
