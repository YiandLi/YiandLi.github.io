LSTM+CRF

Posted on 2021-06-23 In NLP , Named entity recognition
Symbols count in article: 1.2k Reading time ≈ 6 mins.

介绍

  • LSTM很适合用于序列标注问题，但是LSTM并没有考虑上一时刻的输出，yt−1y_{t-1}yt−1​。
  • 这样会导致一个问题，以“我 喜欢 跑步”为例，LSTM输出“喜欢”的标签是“动词”，而“跑步”的标签可能也是“动词”。所以，使用LSTM无法对标签转移关系进行建模。
  • 为了引入标签转移关系，我们可以引入一个标签转移矩阵作为参数，即CRF中的特征矩阵。

LSTM-CRF


在网络输出的分数矩阵中

  • network score： [fθ]i,t\left[f_{\theta}\right]_{i , t}[fθ​]i,t​ ：网络参数 θ\thetaθ 时，第t个单词标签为i的分数。
    可以看做发射得分矩阵。这个发射的过程可以看做lstm直接分类的结果，所以发射得分是由网路后面的分类器决定的。
    网络参数（包括最后一层分类器）需要被训练。
  • transition score： [A]i,j[\mathrm{A}]_{\mathrm{i} , \mathrm{j}}[A]i,j​ ：一对连续时间步骤从第i标签到第j标签的分数。
    可以看做转移得分矩阵。这个转移矩阵与位置t无关！
    这个转移得分矩阵也要被训练。
  • 将两者参数融合：θ~=θ∪{[A]i,j∀i,j}\tilde{\theta}=\theta \cup\left\{[A]_{i, j} \forall i, j\right\}θ~=θ∪{[A]i,j​∀i,j}

  • 输入句子序列 [x]1T[x]_{1}^{T}[x]1T​ ，标签序列 [i]1T[i]_{1}^{T}[i]1T​ 的分数为：
    下标1：一个句子，一个标注

s([x]1T,[i]1T,θ~)=∑t=1T([A][i]t−1,[i]t+[fθ][i]t,t)s\left([x]_{1}^{T},[i]_{1}^{T}, \tilde{\theta}\right)=\sum_{t=1}^{T}\left([A]_{[i]_{t-1},[i]_{t}}+\left[f_{\theta}\right]_{[i]_{t}, t}\right) s([x]1T​,[i]1T​,θ~)=t=1∑T​([A][i]t−1​,[i]t​​+[fθ​][i]t​,t​)

Bi-LSTM CRF + Feature


额外的箭头表示feature

  • spelling features and context features
    • Spelling features
      除了小写的单词特征外，我们还提取给定单词的下列特征。
      是否以大写字母开头
      是否有所有大写字母
      是否有所有小写字母
      是否有非首字母大写
      是否字母和数字混合
      是否有标点符号
      字母前缀和后缀(窗口大小为2到5)
      是否有撇号结尾(’ s)
      非字母，例如 A. T. &T. to …&
      单词模式特征，大写字母、小写字母和数字分别映射到“A”、“A”和“0”，例如，d56 -3到A00a-0
      词型总结特征，类似于词型特征，但去掉了连续的相同字符。例如，d56 -3到A0a-0
    • Context features
      uni-gram features and bi-grams features (and tri-gram features).
  • features connection tricks
    这些特征可以拼接到word embdding后一起进入网络训练；也可以直接concat到LSTM的输出层，作为线性全连接层的输入，计算发射得分。

原理

  • CRF中有两类特征： a. xtx_txt​和yty_tyt​之间的特征 b. yty_tyt​ 和 yt−1y_{t-1}yt−1​ 之间的特征。
    a. 输入xtx_txt​, LSTM的输出yty_tyt​，对应CRF中xtx_txt​和yty_tyt​之间的特征
    b. 根据特征转移矩阵得到 [yt−1,yt][y_{t-1}, y_t][yt−1​,yt​] 对应的得分，对应CRF中 $y_t $ 和 yt−1y_{t-1}yt−1​ 之间的特征。
  • CRF层接受lstm层输出的发射分数，并且计算时会加上转移分数。
    在训练时可以自动学到标注约束。
  • 假设输入n个单词，每个的标注都有m个可能，则总序列有mnm^nmn种标注方法。设每个句子的得分为score(y)，然后类似CRF，使用softmax求出其条件概率：

p(y∣x) = escore(y)ZZ = ∑yescore(y)\begin{aligned} &p(y|x)\ =\ \dfrac{e^{score(y)}}{Z}\\ &Z\ =\ \sum_y{e^{score(y)}} \end{aligned} ​p(y∣x) = Zescore(y)​Z = y∑​escore(y)​

给定输入x，如何计算某个输出序列y的概率？

  1. 首先对单个标注结果
    发射得分：LSTM的每个节点会输出一个m维的向量，将时刻i时的输出得分矩阵记作eie_iei​。
    输出结果yiy_iyi​，对应的得分记作ei[yi]e_i[y_i]ei​[yi​]。
    转换得分：根据前面的标签转移矩阵T，可以得到时刻i时的分数T[i−1][i]T[i-1][i]T[i−1][i]。
    然后我们可以得到某个句子的总的得分：score(y)=∑i=1n(ei)+∑i=2n(T[i−1][i])score(y) = \sum_{i=1}^n (e_i) + \sum_{i=2}^n(T[i-1][i])score(y)=∑i=1n​(ei​)+∑i=2n​(T[i−1][i])。
    （因为是得分，所以这里使用和的形式，即总得分。）

  2. 然后计算所有标注结果，求出Z：Z = ∑yscore(y)Z\ =\ \sum_y score(y)Z = ∑y​score(y)，这里使用动态规划来计算，降低计算量。

  3. 于是就可以求出概率p(y∣x) = escore(y)Zp(y|x)\ =\ \dfrac{ e^{score(y)} }{Z}p(y∣x) = Zescore(y)​，这里Z=∑yescore(y){Z}=\sum_y{e^{score(y)}}Z=∑y​escore(y)。
    为了简化计算，取对数：logp(y∣x) = score(y)−log(∑yescore(y))logp(y|x)\ =\ score(y) - log(\sum_y{e^{score(y)}})logp(y∣x) = score(y)−log(∑y​escore(y))

如何训练模型？

直接定义损失函数为负的目标似然函数：loss = −logp(y∣x)loss\ =\ -logp(y|x)loss = −logp(y∣x)，然后求导更新令loss最小即可。

如何使用模型预测？

使用动态规划求解即可。

为什么是LSTM-CRF而不是LSTM-HMM

  1. hmm的状态转移是概率，crf是分数，再全局做的归一化，这是一点点区别。
  2. lstm的输出考虑了整个观测，但是hmm做了观测独立假设，所以这个lstm+crf更符合crf的定义，而不是hmm。
