NLP预训练模型综述

Posted on 2021-06-17 Edited on 2022-05-22

介绍

预训练 pre-train：根据大量的数据，训练出一个"通用"的模型。
然后针对不同的具体任务，微调 fine-tune 一下，就可以了。

预训练

  • 预训练的优点：

    • 在大规模语料上通过预训练学习通用语言表示对下游任务很有帮助；
    • 预训练提供了更好的模型初始化参数，使得在目标任务上有更好的泛化性能和更快的收敛速度；
    • 预训练是一种有效的正则化方法，能够避免在小数据集上过拟合。

  • 分类：上下文编码器的使用

    • 非上下文嵌入：是静态且与上下文无关的，无法处理多义词； 难以解决未登陆词 oov。
      • word2vec
      • Glove
    • 上下文嵌入：考虑上下文语义，将词在不同上下文的语义做区分。通过对词的 token 加一层编码器得到词的上下文嵌入。
      正确预测 --》 清楚关系

微调

在model的输出上叠加一层特定层 Task-specific Layer，使其适应下游任务。

  • NLP的任务分类
    • Input：
      • one sentence
      • multiple sentences：+分割符号[sep]
    • Output:
      • one class
      • class for each token
      • copy from input
      • general sequence
  • How:
    1. 只调整task-specific层参数
    2. 调整pre-trained和task-specific层的部分，效果更好。

  • 问题：
    不同的task，会得到不同的预处理层，但是参数太多，会浪费。
    解决：引入adapter layer。冻结前面的层，fine tune后面的层。

飞桨训练营

自监督表示学习

自监督学习的监督信息不是人工标注的，而是算法在大规模无监督数据中自动构造监督信息，来进行监督学习或训练。

  • one-hot
  • embedding：one-hot的线性映射
  • word2vec：分布式假设，考虑了语义。
    • cbow：效率更高。
    • skip-gram：生僻词更加准确，耗时更加少。

自回归语言模型

  1. 马尔可夫语言模型：计算后验概率，得到的是词之间的依赖概率。
  2. RNN：可以建模一个句子，或者某个单词的考虑前面单词的表示。
    • 梯度爆炸/消失：选择其他激活函数
    • 无法很好地表达有层次结构的信息
    • 会遗忘，无法并行
  3. Self-Attention：
    剥离了RNN，使用Wq,Wv,WkW_q, W_v, W_kWq​,Wv​,Wk​。
  4. Transformer:
    • 多头注意力机制
    • 残差连接：
      • 避免梯度消失
      • 避免网络退化 ？？？
  5. GPT：基于Transformer，最大化句子的概率。
  6. BERT：最大化中间词的概率，基于CBoW。

语义理解

正确理解语义的例子

  • 挑战：
    • 语言复杂性和多样性
    • 语言背后的环境知识
  • 四个粒度
    • 字的理解
    • 词的理解
    • 句子的理解
    • 篇章的理解
  • 学习通用语义的方法：
    • Word2Vec，从海量无标注语料学习到通用的语言表示。
    • 基于深度学习的序列编码网络，通过wordEmbedding层，生成与任务无关的词向量。
      查表得到词向量矩阵后：
      1. token-level（观点抽取）：后面加一个分类器即可
      2. sentence-level（文本匹配）：pooling + 分类器
  • 缺点：
    • 多义词有歧义：因为词向量是静态的有歧义的。
    • 过于关注词本身的解释性和效果，不关注下游任务。
    • 缺乏对于文本全局信息的建模。
  • 解决：预先训练和微调的机制
    • Pre-training：在大规模数据集上学习尽可能好的通用表示
      因为低层特征具有高可复用性
    • Fine-tuning：利用学习好的通用表示初始化下游的任务网络
      因为高层的特征和任务有强相关性

语义理解模型

  1. ELMo：Feature base
  • Pre-train：纯粹的表示抽取器
    使用双向双层LSTM对语言模型进行建模
    一种无监督训练
    词向量是基于上下文，动态输出的。
    建模不同层次的语义信息。
  • Fine-tuning：
    提取不同的语义信息，加权相加交给下游任务的网络。
    加权相加的参数是可以学习的。
  • 优点：解决一词多义
  • 缺点：
    • 不完全的双向预训练
    • fine-tuning阶段，需要搭建特定的网络。
    • 仅仅学习了语言模型，没有针对句子层面。

问题：是否为自回归模型

  1. GPT：
  • Model-based：预训练模型作为任务网络的一部分参与学习，减少下游任务的新增参数，网络结构修改少。
  • 缺点：
    • 单项语言模型
    • 仅有词向量，没有句向量建模。
  • GPT2，GPT3
  1. Bert：
  • 特点：
    • 沿用了GBT网络结构：12层Transformer
    • 交互式双向语言模型建模
    • 引入了sentence-level预训练任务
    • 更大规模地训练数据集
  • 预训练策略：
    • Token-level：mask-LM任务，交互式双向语言模型。
    • Sentence-level：Next-sentence预测任务，建模句向量。
  • 缺点：
    难以学到词，短语，实体的完整语义。
  1. ERNIE
  • 区别：
    • mask：token-》word
    • 百度语料库
  1. ERNIE 2.0
    持续学习语义理解模型，保证在学习新知识时，不遗忘旧知识。

预训练语言模型在NLP经典任务中的应用

  • 文本分类
    • 单句分类：情感分析
    • 句对分类：自然语言推断
    • 文本匹配：文本语义相似度
  • 命名实体识别
  • 阅读理解

文本匹配

  • NLP基础问题之一
  • 问题：
    1. 多义同义
    2. 组合结构问题
    3. 表达的多样性问题
    4. 匹配的非对称问题
  • 解：基于预训练模型的文本语义匹配
