Input

为什么 BERT 的三个 Embedding 可以进行相加？

假设 token Embedding 矩阵维度是 [4,768]；position Embedding 矩阵维度是 [3,768]；segment Embedding 矩阵维度是 [2,768]。对于一个字，假设它的 token one-hot 是[1,0,0,0]；它的 position one-hot 是[1,0,0]；它的 segment one-hot 是[1,0]。那这个字最后的 word Embedding，就是上面三种 Embedding 的加和。如此得到的 word Embedding，和concat后的特征：[1,0,0,0,1,0,0,1,0]，再过维度为 [4+3+2,768] = [9, 768] 的全连接层，得到的向量其实就是一样的。

再换一个角度理解：直接将三个one-hot 特征 concat 起来得到的 [1,0,0,0,1,0,0,1,0] 不再是one-hot了，但可以把它映射到三个one-hot 组成的特征空间，空间维度是 432=24 ，那在新的特征空间，这个字的one-hot就是[1,0,0,0,0…] (23个0)。此时，Embedding 矩阵维度就是 [24,768]，最后得到的 word Embedding 依然是和上面的等效，但是三个小 Embedding 矩阵的大小会远小于新特征空间对应的 Embedding 矩阵大小。当然，在相同初始化方法前提下，两种方式得到的 word Embedding 可能方差会有差别，但是，BERT还有Layer Norm，会把 Embedding 结果统一到相同的分布。

为什么 Transformer 的 Embedding 最后要乘 dmodel\sqrt{d_{model}}dmodel​​
如果使用 Xavier 初始化，Embedding 的方差为 1/d_model，当d_model非常大时，矩阵中的每一个值都会减小。通过乘一个 dmodel\sqrt{d_{model}}dmodel​​ ，可以将方差恢复到 1。

因为 Bert 使用的是学习式的Embedding，所以 Bert 这里就不需要放大。

为什么 attention 要用三个矩阵 QKV ？$
意义：Q 是 query，用于表示「目标 token 的特征」；K 是被匹配项，用于表示「作为被选择方时公开的信息」；V 时 value，表示「token 本质是什么」。

为什么不能只用一个或者两个？
总体：是为了对 V 进行提纯，多个矩阵可以得到多个空间上的投影，增加表达能力。
单独：

  1. 计算 V=XWVV = XW_VV=XWV​ ，是为了映射到能够和 Softmax 结果相乘的表示空间中。
  2. K / Q 如果不进行区分，则
    1. attention 值变成了对称的，A 对 B 的分数 == B 对 A 的分数，但是这种对称关系是不必要的，降低了表示能力。
      google 的 reformer 提到，W1 和 W2 保持相同也基本不会影响模型的表现
    2. 自己和自己的 attention 分数最高，不利于处理代词。

Encoder

  • 在论文中，Encoder由6个相同的layer组成，即左边的部分。
  • 每个layer由两个sub-layer组成，分别为多头注意力机制multi-head self-attention mechanism和全连接前馈网络fully connected feed-forward network。

multi-head self-attention

multi-head self-attention中，通过 h组(h头) 不同的线性变化（Q，K，V）得到h组解，然后拼接起来。这里采用scaled dot-product方法计算attention的值：
词向量与相同的QKV矩阵乘得到对应的三个矩阵。

Attention(Q,K,V)=softmax(QKTdkV)Attention(Q,K,V)=softmax(\frac{QK^T}{\sqrt{d_k}}V) Attention(Q,K,V)=softmax(dk​​QKT​V)

Q表示所需计算的注意力项，K为被计算注意力项，V为被注意力项的取值。

可以看到，Q和K的尺寸是相同的，得出结果为相似度矩阵，矩阵的第[i,j]项表示Q的第i行和K的第j行的点积相似度（投影相似度），Query和key的关联强度。
scorei,jscore_{i,j}scorei,j​表示第i个单词和第j个单词的相似度得分。

当输入元素的 「数量级」 很高，方差很大时，softmax 的输出接近 ont-hot，会导致梯度消失问题。

所以我们希望的是让 a 的差距不要那么大，方差控制为1 。

假设 q, k 每一个维度的均值 0 方差 1，则 qk 均值 0 方差 dkd_kdk​ ，所以点积后除 dk\sqrt{d_k}dk​​ ，让方差控制在 1 。

除了梯度问题，也可以防止 attention 过于关注某一个 token ，和温度类似。

为什么在其他 softmax 的应用场景，不需要做 scaled ？
比如说分类层：

  1. 分类层的 softmax 不是两个随机变量相乘的情况，不会有方差变成 dkd_kdk​ 的问题 。

  2. 通常和交叉熵联合求导，某个目标类别 iii 上的整体梯度变为 yi′−yiy_i'-y_iyi′​−yi​ ，预测概率 - 真实概率 。

    当出现了极大值 yiy_iyi​ ，如果 yi′y_i'yi′​ 为 1 ，即正确预测，则就不用大幅度更新了；如果 yi′y_i'yi′​ 为 0 ，即错误预测，则会给出最大程度的负反馈。

    transformer中的softmax是在中间，如果是点积很大的时候，就没有梯度回传了； 但是在分类场景里面，最后一层是softmax，这时候如果是预测对了，本就不需要回传。


再通过softmax进行映射，得到的就是相似度得分。
最后用相似度得分乘注意力项的取值得到最终的attention结果。

这里再次理解下多头注意力机制，论文中原本的词向量为512维，所以原本得到的QKV均为512维度。
QK的维度只是为了去计算相似度，应该是相同的，而V的维度则决定了最终一次输出的向量维度，拼接次数h乘dVd_VdV​则为最终的输出长度。
现在重新设置QKV的大小：设置成8组，维度均为64的矩阵，这样会生成8个64维的词向量。然后concat成为512维度的向量。然后再乘一个WOW^OWO矩阵，得到最终的多头注意力值。

多头讨论：https://www.zhihu.com/question/350369171

  • 多头的优点：多个独立的attention计算，作为一个集成的作用，防止过拟合。并且可以在不同的表示子空间里学习到相关的信息（空间维度降低，各个子空间更小，容易聚在一起去学到语义相似的信息）。

  • 多头的计算量分析：张量乘法部分没有引入额外计算量，但是 softmax attention 部分计算量是加倍的，但是基本忽略不计。

  • 训练细节：
    如果输入序列n小于表示维度d的话，每一层的时间复杂度self-attention是比较有优势的。
    当n比较大时，作者也给出了一种解决方案self-attention（restricted）即每个词不是和所有词计算attention，而是设置窗口，只与限制的r个词去计算attention。

fully connected feed-forward network

Paper：《Rethinking the Value of Transformer Components》
可视化了全连接层和self-attention层的作用

上面得到的每个向量都被输入到同一权重的全连接网络中。
作用就是维度变换，该层包含了两层线性层，中间的激活函数为ReLu。
FFN的加入引入了非线性，从而增加了模型的表现能力。把FFN去掉模型也是可以用的，但是效果差了很多。

Paper：《Transformer Feed-Forward Layers Are Key-Value Memories》和《Knowledge Neurons in Pretrained Transformers》
Transformer起到了记忆网络的作用，起到了存储知识的作用。

因此可以将每个sub-layer的输出表示为：sublayeroutput=LayerNorm(x+(SubLayer(x)))sublayer_{output} = LayerNorm(x+(SubLayer(x)))sublayeroutput​=LayerNorm(x+(SubLayer(x)))。

残差连接 residual connection / skip connect

  • 更深层的网络会出现梯度消失的问题，导致误差变大。
    解决方法：让低层的原始信息直接传到后续的高层中，让高层专注于残差的学习。强行将一个输入添加到函数的输出的时候，则新的输出函数G(x)可以明确的拆分为F(x)和X的线性叠加。即将输出表述为输入和输入的一个非线性变换的线性叠加。再次求导时，就会有dGdx=dFdx+1\dfrac{dG}{dx}=\dfrac{dF}{dx}+1dxdG​=dxdF​+1。

  • 同时还可以解决网络退化的问题。

Decoder

  1. 同样也是6个layer组成
  2. 每个layer由3个sub-layer组成，分别为一个隐模的多头自注意力层，多头注意力层，全连接神经网络层组成。
    • 输入：encoder的输出，decoder的第i位的输出
    • 输出：对应i+1位置的输出词的概率分布
    • 训练和预测是不一样的：
      解码需要使用 sequence mask，让解码时看不到未来的token，来支持并行操作。而如果解码器被设置为串行操作，则无需sequence mask。
      在实际预测过程中，由于模型在训练过程中是按照前述方式训练的，所以继续使用seq mask会和训练方式匹配，得到更好的预测结果。

      Masked：后面讲。
      而预测时，因为没有ground truth了，需要一个个预测。

Position Encode/Embedding （PE）

  • RNN的优点为考虑了单词之间的位置关系，但是Transformer剥离了RNN，就不能捕捉位置信息的缺陷，所以需要位置编码来记录位置信息。
  • 要求：
    • It should output a unique encoding for each time-step (word’s position in a sentence).
    • Distance between any two time-steps should be consistent across sentences with different lengths.
    • Our model should generalize to longer sentences without any efforts. Its values should be bounded.
    • It must be deterministic.
  • 两种方法

    • 1. 绝对位置编码：

      • 学习得到的 Positional Embedding
        初始化一个 512×768 的矩阵作为位置向量，让它随着训练过程更新。
        缺点是不可扩展，即如果预训练最大长度为 512 的话，那么最多就只能处理长度为 512 的句子，再长就处理不了了
      • 用不同频率的sin和cos表达：任意位置之间都可以进行计算。

      PE(pos,2i)=sin(pos/100002i/dmodel)PE(pos,2i+1)=cos(pos/100002i/dmodel)\begin{aligned} PE(pos, 2i) &= sin(pos/10000^{2i/d_{model}})\\ PE(pos, 2i+1) &= cos(pos/10000^{2i/d_{model}}) \end{aligned} PE(pos,2i)PE(pos,2i+1)​=sin(pos/100002i/dmodel​)=cos(pos/100002i/dmodel​)​

      PE函数输出一个和word embedding维度一致的矩阵，行表示词语，列表示词向量
      i表示词向量的位置
      pos 表示词语在句子中的位置
      dmodeld_{model}dmodel​表示词向量的维度
      然后和原来的词向量相加输入到encoder中。
      可以使用三角函数性质来得到一个特定位置的向量，所以该方法蕴含了相对位置信息，但是这种位置信息会在注意力机制那里消失。
      代码实现：https://blog.csdn.net/qq_34771726/article/details/102918440

      • 为什么公式中有一个魔法值 10000？
        表示不同周期，PE 在不同位置上的变化。使用 10000 很可能就是确保循环周期足够大，以便编码足够长的文本。如果为 1，则周期很短，即相邻位置的位置编码也差不多。

    • 2. 改进：相对位置编码 https://zhuanlan.zhihu.com/p/105001610

Mask

  • mask 表示掩码，它对某些值进行掩盖，使其在参数更新时不产生效果。
  • Transformer中涉及到两种mask：padding mask 和 sequence mask。
    • padding mask 在所有的 scaled dot-product attention 里面都需要用到
    • sequence mask 只有在 decoder 的 self-attention 里面用到。

所以在decoder的masked self-attention需要同时用到两者，具体实现是将两者相加。其余都使用padding mask。

padding mask

每个批次输入序列长度是不一样的也就是说，我们要对输入序列进行对齐padding。

  • 序列较短：在后面填充占位符号，考虑到softmax函数最后要压一下，我们需要填充负无穷的数，这样softmax之后就会变成0.
    但是如果输入的序列太长，则是截取左边的内容，把多余的直接舍弃。

sequence mask

为了使得 decoder 不能看见未来的信息，即在t的时刻，我们的解码输出应该只能依赖于t时刻之前的输出，而不能依赖t之后的输出。
方法：使用一个上三角矩阵，使softmax后的上三角全为0。

优点

  1. RNN 固有的顺序属性阻碍了训练样本间的并行化，对于长序列，内存限制将阻碍对训练样本的批量处理。
    transformer中的并行是指矩阵运算吗？？
  2. 自注意力可以产生更具可解释性的模型。我们可以从模型中检查注意力分布。
  3. 用自注意力层构建 Transformer：传统的网络之间，两点的依赖程度/相似程度是与计算步骤/距离有关系的，但是attention则不会出现这个问题。
