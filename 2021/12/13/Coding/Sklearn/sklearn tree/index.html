本篇介绍介绍决策树：回归树和分类树。
决策树其实也是对空间进行切分的方式

理论上，区域形状是任意的，但出于模型简化和增强可解释性的考虑，这里将预测变量空间划分成高维矩形，或称盒子（boxes）。

一般关注的问题就是：

  1. 特征选择，即不同的启发式函数
  2. 树的构造
  3. 树的剪枝

回归树 Regressions Tree

  • 叶节点的值为其内部所有样本值的平均
  • 评价指标：MSE，越小越好
    MSE：预测值和真实值误差的平方和的均值

MSE=1m∑i=1m(fi−yi)2MSE=\frac{1}{m} \sum_{i=1}^{m}\left(f_{i}-y_{i}\right)^{2} MSE=m1​i=1∑m​(fi​−yi​)2

构造树

  • 整体流程
  1. 利用递归二叉分裂在训练集中生成一棵大树
  2. 使用不同的 α\alphaα ，对大树进行代价复杂性剪枝，得到一系列最优子树。
  3. 利用 KKK 折交叉验证选择 α\alphaα 和对应的剪枝树。

生成树

使用 递归二叉分裂（ recursive binary splitting） 方法，是一种自上而下的贪婪算法。
自上而下（top-down）：从树顶端开始依次分裂预测变量空间，每个分裂点都产生两个新的分支。
贪婪（greedy）：在建立树的每一步中，最优分裂确定仅限于某一步进程，而不是针对全局去选择那些能够在未来进程中构建更好的树的分裂点。
执行递归二叉分裂时，遍历选择每一对可能的 预测变量/属性 XjX_jXj​和分割点 sss，将预测变量空间分为两个区域{X∣Xj<s},{X∣Xj≥s}\{X|X_j \lt s\}, \{X|X_j\ge s\}{X∣Xj​<s},{X∣Xj​≥s} 。最优分割为能使得MSE最小的那种.
即，对 jjj 和 sss ，定义一对半平面：

R1(j,s)={X∣Xj<s},R2(j,s)={X∣Xj≥s}R_1(j,s)=\{X∣X_j \lt s \}, R_2(j,s)=\{X∣X_j \ge s\} R1​(j,s)={X∣Xj​<s},R2​(j,s)={X∣Xj​≥s}

寻找 jjj 和 sss ，使下式取得最小值：

∑i:xi∈R1(j,s)(yi−y^R1)2+∑i:xi∈R2(j,s)(yi−y^R2)2\sum_{i: x_{i} \in R_{1}(j, s)}\left(y_{i}-\hat{y}_{R_{1}}\right)^{2}+\sum_{i: x_{i} \in R_{2}(j, s)}\left(y_{i}-\hat{y}_{R_{2}}\right)^{2} i:xi​∈R1​(j,s)∑​(yi​−y^​R1​​)2+i:xi​∈R2​(j,s)∑​(yi​−y^​R2​​)2

重复上述步骤，寻找继续分割数据集的最优预测变量和最优分割点，使随之产生的区域中的RSS达到最小。此时被分割的不再是整个预测变量空间，而是之前确定的两个区域之一，如此一来得到3个区域。然后不断重复，直到符合某个停止准则，或者不断加深搜索树然后进行 剪枝prune 。
停止准则：最大深度或者叶节点最少的元素数量


有些时候运用上述方法会造成数据的过拟合，即分的太细了，导致在测试集上效果不佳。 一棵分裂点更少、规模更小（区域更少）的树会有更小的方差和更好的可解释性（以增加微小偏差为代价）。

一种可能的解决方法是：仅当分裂使MES减小量超过某阈值时，才分裂树结点。这种策略能生成较小的树，但可能产生过于短视的问题，一些起初看起来不值得的分裂却可能之后产生非常好的分裂——也就是说在下一步中，MSE大幅减小。
更好的策略是先生成一个大树，然后剪枝。

剪枝

使用代价复杂性剪枝（Cost complexity pruning CCP），也称最弱联系剪枝（weakest link pruning）
全局来看，我们定义一个函数来评估这决策树的表现，希望：

  1. 其 损失最小 ，使用 C(T)C(T)C(T) 表示
  2. 其 分支/叶节点最少 来，复杂度最低，避免过拟合，使用 α∣T∣\alpha|T|α∣T∣ 表示 。
    用到一个 非负调整参数 α\alphaα 对树的复杂度进行惩罚，类似正则化操作。

两项加和为：

Cα(T)=C(T)+α∣T∣=∑m=1∣T∣  ∑i:xi∈Rm(yi−y^RM)2+α∣T∣\begin{aligned} C_{\alpha}(T)=& C(T)+\alpha|T|\\ =& \sum_{m=1}^{|T|}\; \sum_{i: x_{i} \in R_m} \left(y_{i}-\hat{y}_{R_{M}}\right)^{2} + \alpha |T| \end{aligned} Cα​(T)==​C(T)+α∣T∣m=1∑∣T∣​i:xi​∈Rm​∑​(yi​−y^​RM​​)2+α∣T∣​

TTT 表示决策树
∣T∣|T|∣T∣ 表示树的终端 叶子节点 数量
C(T)C(T)C(T) 表示树的预测误差，即所有节点的预测误差的和
RmR_mRm​ 表示第m个终端节点对应的切割矩形
当 α\alphaα 增大时，终端结点数多的树将 为它的复杂付出更多代价 ，所得的 树将会变得更小 。所以， α\alphaα 的取值决定了复杂度和拟合度如何平衡。

找 α\alphaα

那么如何找到合适的 α\alphaα 呢 ？
另 α\alphaα 从0取到无穷大，每一个取值都可以找到令 Cα(T)C_{\alpha}(T)Cα​(T) 最小最优剪枝后决策树。
当 α\alphaα 很小时，原树 T0T_0T0​ 是最优树；当 α\alphaα 最大时，单独的根节点是最优树。
随着 α\alphaα 不断增大，我们可以得到这样的剪枝树序列 T0,T1, ... ,TnT_0, T_1,\ ...\ ,T_nT0​,T1​, ... ,Tn​ ，并且 Ti+1T_{i+1}Ti+1​ 一定是 TiT_iTi​ 剪去某一个内部节点生成的结果。

Breiman证明：将 α\alphaα从小到大排列，0=α0<α1<  ...  <αn<∞0=\alpha_0 \lt \alpha_1 \lt\; ...\; \lt\alpha_n < \infty0=α0​<α1​<...<αn​<∞ ，每个区间 [αi,αi+1)[\alpha_i, \alpha_{i+1})[αi​,αi+1​) 中，对应子树/剪枝后树 TiT_iTi​ 是最优的。

每次剪枝都是针对某一个内部节点/子树，其余非剪枝节点的损失不会变化。
所以我们只需要计算该节点剪枝前后的损失函数即可，将上面的全局损失函数引申到这个单独节点损失：
对于任意内部节点 ttt 作为根节点的子树 TtT_tTt​ ，剪枝前该子树有 ∣Tt∣|T_t|∣Tt​∣ 个叶节点，预测误差为 C(Tt)C(T_t)C(Tt​) ；剪枝后只有本身一个叶节点t，预测误差计为 C(t)C(t)C(t) 。
剪枝前损失函数为 Cα(T)=C(Tt)+α∣T∣C_{\alpha}(T)=C(T_t)+\alpha |T|Cα​(T)=C(Tt​)+α∣T∣
剪枝后损失函数为 Cα(t)=C(t)+α∣t∣=C(t)+αC_{\alpha}(t)=C(t)+\alpha |t|=C(t)+\alphaCα​(t)=C(t)+α∣t∣=C(t)+α
通过Breiman证明，我们知道一定存在一个 α\alphaα ，令 Cα(T)=Cα(t)C_{\alpha}(T)=C_{\alpha}(t)Cα​(T)=Cα​(t) ：

α=C(t)−C(Tt)∣Tt∣−1\alpha = \frac{C(t)-C(T_t)}{|T_t|-1} α=∣Tt​∣−1C(t)−C(Tt​)​

（就算我们仍然使用全局的损失函数计算，这个 α\alphaα 也不会改变。
在 [αi,αi+1)[\alpha_i, \alpha_{i+1})[αi​,αi+1​) 中，子树/剪枝后树 TiT_iTi​ 都是最优的，当 α\alphaα 大于这个值时，应该剪枝。

于是对每个内部节点/子树，求得对应的 α\alphaα 得到 0=α0<α1<  ...  <αn<∞0=\alpha_0 \lt \alpha_1 \lt\; ...\; \lt\alpha_n < \infty0=α0​<α1​<...<αn​<∞ 。
然后又因为上述， α\alphaα 从小到大排列时对应了将原决策树剪枝到只有一个叶节点的过程，所以我们不断将令 α\alphaα 最小的内部节点剪去即可。

代码 DecisionTreeRegressor

sklearn.tree.DecisionTreeRegressor

初始化参数

  • criterion : {“squared_error”, “friedman_mse”, “absolute_error”, “poisson”}, default=”squared_error”：不同的评价指标
  • splitter : {“best”, “random”}, default=”best”
    • 'best' 是在所有特征中找最好切分点，让决策树的目标函数达到最小，即寻找一棵最优决策树，非常耗时。
    • 'random' 是随机在特征中寻找切分点，所得的决策树可能不是最优，但也不会差，适合数据量特别大或要使用随机森林的时候。
  • random_state : int, RandomState instance or None, default=None
    每次建树的时候都是通过选取不同的特征值来形成不同的树。但是每次返回的最优的树都不同。所以可以通过固定一个种子数来固定最优树模型。
  • max_depth : int, default=None 剪枝参数
    限制决策树的最大深度。None为无限制，约束条件受min_samples_split印象。
  • min_samples_split : int or float, default=2剪枝参数
    一个节点必须要包含至少min_samples_split个训练样本，这个节点才允许被分枝，否则分枝就不会发生。
    • int类型表示数量
    • float类型表示一个比例
  • min_samples_leaf : int or float, default=1剪枝参数
    一个节点在分枝后的每个子节点都必须包含至少min_samples_leaf个训练样本，否则分 枝就不会发生。
    • int为绝对数量
    • float为所占样本比例（向下取整）。
      搭配max_depth使模型更平滑，一般从5开始。
  • max_features : int, float or {“auto”, “sqrt”, “log2”}, default=None
    限制分枝时考虑的特征个数，超过限制个数的特征都会被舍弃。
    用来限制高维度数据的过拟合，但其方法比较暴力，是直接限制可以使用的特征数量而强行使决策树停下。可能会导致考虑的特征数量不足导致欠拟合。
    • int类型表示数
    • float类型表示一个比例
    • auto和None都是相当于不设置
    • 其余对总特征数量进行数学函数计算求值
  • min_impurity_decrease : float, default=0.0：限制信息增益的大小

分类树 Classification Tree

叶节点的值为它所属区域内的训练集中最常出现的类（most commonly occurring class）。

评价指标

用来衡量整棵树分类结果的不确定性，来比较如何选择划分指标。注意是以整棵树为单位，去计算全局的评价指标大小。
分类错误率/信息熵/Gini系数 越小越好，IG 越大越好。

分类错误率 (classification error rate)

E=1−max⁡k(p^mk)E=1-\max _{k}\left(\hat{p}_{m k}\right) E=1−kmax​(p^​mk​)

p^mk\hat{p}_{m k}p^​mk​ 表示第 mmm 个区域中第 kkk 类所占的比例，这里限定为最多数量的类。

衡量的是一个节点的 分类错误率，整棵树的分类错误率可以使用叶子节点错误率加权和的方法得到。

这也是这里讲到的唯一一个通过错误率 直接定义 的评价指标。


下面讲到的全都是通过 信息量的概念定义评价指标，出发点是被分到叶子节点处的 叶子样本子集合的纯度。
因为叶子节点的值为频率最大的类，所以样本越纯，方差越小，肯定越好。


基尼系数 (Gini index)

G=∑k=1Kp^mk(1−p^mk)G=\sum_{k=1}^{K} \hat{p}_{m k}\left(1-\hat{p}_{m k}\right) G=k=1∑K​p^​mk​(1−p^​mk​)

它衡量的是 KKK 个类别的总方差。如果所有 p^mk\hat p_{mk}p^​mk​ 的取值都接近0或1，基尼系数会很小。
因此基尼系数被视为衡量结点纯度的指标——如果它较小，就意味着某个结点包含的观测值几乎都来自同一类别。

以上是一个节点的基尼系数，而 划分后的整棵子树 的基尼系数为叶节点基尼系数的加权和，权重为叶节点例子数量/总数量。

信息熵 (entropy)

表示信息量的期望
单位是 bits

H=−∑k=1Kp^mklog⁡p^mkH = -\sum_{k=1}^{K} \hat{p}_{m k} \log \hat{p}_{m k} H=−k=1∑K​p^​mk​logp^​mk​

一颗树原始的信息熵是确定的，不用考虑条件，直接通过标签概率 $ \hat{p}_{m k}$ 计算得到，可以看作是一个大集合。

而根据不同节点，原始大集合被分成了不同的小集合，这时的信息熵就会发生改变。

  • 多变量信息熵

H(X,Y)=−∑x∈X∑y∈Yp(x,y)log2p(x,y)H(Y∣X=x)=−∑y∈Yp(y∣x)log2p(y∣x)H(Y∣X)=∑x∈Xp(x)H(Y∣X=x)=−∑x∈X∑y∈Yp(x,y)log2p(y∣x)\begin{aligned} &H(X,Y) =-\sum_{x\in X}\sum_{y\in Y}p(x,y)log_2 p(x,y)\\ &H(Y|X=x) =-\sum_{y\in Y}p(y|x)log_2 p(y|x)\\ &H(Y|X) =\sum_{x\in X} p(x)H(Y|X=x) =-\sum_{x\in X}\sum_{y\in Y}p(x,y)log_2 p(y|x) \end{aligned} ​H(X,Y)=−x∈X∑​y∈Y∑​p(x,y)log2​p(x,y)H(Y∣X=x)=−y∈Y∑​p(y∣x)log2​p(y∣x)H(Y∣X)=x∈X∑​p(x)H(Y∣X=x)=−x∈X∑​y∈Y∑​p(x,y)log2​p(y∣x)​

当根据一个标准 XXX 对原始大集合进行划分后（即继续向叶子节点走），信息熵变为 H(Y∣X)=∑x∈Xp(x)H(Y∣X=x)H(Y|X) =\sum_{x\in X} p(x)H(Y|X=x)H(Y∣X)=∑x∈X​p(x)H(Y∣X=x) ，即遍历 XXX 的不同取值范围，其实就是划分的两/n个组，然后每个组内计算信息熵，用最终的和得到。

  • 和基尼系数类似，不过用的是对数形式。

信息增益 (Information gain)

在猜测随机变量Y时，在得到其观测属性X后，我们对Y的不确定性会变小。
我们可以使用前后 信息熵的差 来度量减少的不确定性的多少，这就是信息增益。
还可以用来表示变量X和Y之间的互信息 mutual information。

IG(Y∣X)=H(Y)−H(Y∣X)IG(Y|X)=H(Y)-H(Y|X) IG(Y∣X)=H(Y)−H(Y∣X)

在训练时使用基尼系数（比分类错误率效果更好，因为不单单是检测错误率，还看了纯度；计算时比交叉熵快。
在剪枝时使用分类错误率，来追求更高的正确率。

前处理

在分类树中，对于非结构化数据，可以不使用之前提到的构造虚拟变量（get_dummy）的方法。在切分时，会尝试使用同特征的不同类的组合进行切分。
比如对于"ChestPain:bc"表示ChestPain = b or c的对象，都不属于则返回右边分支。


但是，sklearn中不支持这种做法，所以：
二分类特征：map为[0,1]编码，程序会以0.5为界，左边为0类，右边为1类。
多分类有序特征：map为连续数字[1,2,3,…,n]
多分类无序特征：构造虚拟变量get_dummies()，同样使用n-1来表示n个情况。
如果分类数太多，推荐使用连续数字，而不是虚拟变量，来减少模型的尺寸和复杂度。

决策树的优缺点

  • 优点
    • 解释明确，可以被表示为规则，和人脑类似。
    • 没有参数，当数据是非线性的时候，比线性回归好。
  • 缺点
    • 对扰动很敏感
    • 递归二叉分类算法得到的是局部最优解
    • 数据不平衡时，会产生所谓的偏见，生成bias-tree。
