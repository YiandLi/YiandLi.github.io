对比学习 Contrastive Learning

自监督学习的一种，通过样本增强的方式自动构造相似实例和不相似实例，习得一个表示学习模型。
通过这个模型，在投影空间中拉近相似实例的距离，增大不相似实例在投影空间中的距离。

温度系数

当温度系数很小时， 困难样本相似度 sim⁡(hi,hi+)/τ\operatorname{sim}\left(\mathbf{h}_{i}, \mathbf{h}_{i}^{+}\right)/ \tausim(hi​,hi+​)/τ 会增大，在分母叠加项中所占的比重就会越大，对整体loss的影响就会越大，具体的表现就是 对应的负梯度值越大 。

源码是取的 0.05 ，即 sim_score = sim_score * 20 。

温度系数的作用是 调节对困难样本的关注程度 ：越小的温度系数越关注于将本样本和最相似的困难样本分开 ，去得到更均匀的表示。

然而困难样本往往是与本样本相似程度较高的，很多困难负样本其实是潜在的正样本，过分强迫与困难样本分开会破坏学到的潜在语义结构，因此，温度系数不能过小。

考虑两个极端情况，温度系数趋向于0时，对比损失退化为只关注最困难的负样本的损失函数；当温度系数趋向于无穷大时，对比损失对所有负样本都一视同仁，失去了困难样本关注的特性。

Connection to Anisotropy

语言模型生成的语义向量分布存在各向异性的问题，这极大地限制了语义向量的表达能力，缓解这个问题的一种简单方法是加入 后处理步骤 ，比如 BERT-flow 和 BERT-whitening 将向量映射到各向同性的分布上，而本文作者证明了对比学习的训练目标可以隐式地压低分布的奇异值，提高uniformity。
可以看成模型坍塌问题：表示空间应该均匀分布。所以对比学习中的推开负样本对可以解决这个问题。

E-SimCSE ( for unsupervised )

SimCSE 用 dropout 作为一种最小的数据增强方法简单有效，但也存在不足：

  1. BERT 中的 positional embedding 会提示模型让长度相同的文本有着更相似的表征。
    具体地，从同一个句子中 dropout 两次出来的一对正对的长度是相同的，而由两个不同的句子衍生出来的否定对通常会包含不同的长度信息。
    因此，正负对所包含的长度信息是不同的，这可以作为区分正负对的特征。

  2. 更大的batch size会导致SimCSE性能下降（这点确实很困扰）

Token Repetition

对于每个正例对，我们期望在不改变其语义的情况下改变句子的长度。

对于每个正例样本，我们首先通过其长度 NNN 确定复制出来的自序列的长度：

 dup_len ∈[0,max⁡(2,int⁡(dup_rate ∗N))]\text { dup\_len } \in[0, \max (2, \operatorname{int}(\text{dup\_rate } * N))]  dup_len ∈[0,max(2,int(dup_rate ∗N))]

$\text{dup_rate } $ 为最大重复率，是一个超参数。

在确定dup_len后，利用均匀分布从序列中随机选取需要重复的dup_len子词，组成dup子词集合：

 dup_set = uniform ( range =[1,N], num =dup_ len )\text { dup\_set }=\text { uniform }\left(\text { range }=[1, N], \text { num }=\text{dup\_ len } \right)  dup_set = uniform ( range =[1,N], num =dup_ len )

Momentum Contrast

对比学习一个潜在的优化方向是利用更多的负例；然而，更大的批量并不总是更好的选择。
训练SimCSE时，batch_size增大到一定大小时， 负例增多，SimCSE效果却下降:

原因之一可能是随着batch_size增大，数据中可能存在：：一些 与正例相近的文本 被划分到了负例中，这给模型训练带来误差。

在 CV 中，为了缓解扩展 batch size 时 GPU 内存的限制，一个可行的方法是引入动量对比，这也是通过维护一个队列，重用来自 前一个 mini-batch 的编码 来扩展负例对。

作者在使用动量编码器时关闭了dropout，来缩小训练和预测之间的差距。

即：维护一个句子编码序列，将 当前 mini-batch 的句子 Embedding 入队，同时将 “最旧”的句子Embedding 出队。

来分析下为什么两个不同的编码器编码出来的向量能相互做负例，其中一个细节是作者在 CLS 向量后接了一个 MLP 才得到最终的句向量，这点和 MoCo v2 的做法一致，即通过 MLP 转换到相同或者说极其相似的向量空间中，所以才能 work。

Moco：Momentum contrast

Paper：Momentum contrast for unsupervised visual representation learning

对比学习的过程其实是一种查字典的过程，字典中存储了 key 字段的所有表征，可以看作一次查询/训练中所有候选 key 样本的编码组成的编码空间，查询的过程则为使用 query 去空间中查找最接近的 key 。

而作者认为这个 字典/样本嵌入空间 必须：

  1. 足够大：（负）样本越多，训练出的效果越好，当字典的容量足够大时，它可以包含足够多的负样本，这有利于提取到很好的特征。
    也可以看作一个 softmax 问题，所以负样本肯定越多越好，in-batch 的内部其他样本看成是负采样。
  2. 在训练过程中词典不断演化，保持一致性，即编码 key 的编码器和编码 query 的编码器应该一致或者近似

我理解这个一致性是对于字典中的反向传播来说，网络的参数受到每个样本的影响，而字典中包含大量的样本，那么在每一次反向传播的过程中对于字典的编码器网络的变化太大，从而导致提取的特征一致性太低，所以在这里采用的时momentum update的方法。

related work

出了本文提出的 Moco 对比学习框架之外，还有两个传统的方法：

end-to-end：
维护两个encoder 分别编码 query 和 key，每次训练完一个 batch 后，更新这两个 encoder。
这就是最朴素的方法 ，但是字典规模受制于 batch size 和 显存。

memory bank：
Unsupervised feature learning via non-parametric instance discrimination.
将「字典大小」和「batch大小」进行解耦，从而使得字典不受限于batch大小。
作者认为关键的点在于如何 高效的进行损失函数“分母”中的 negative 采样 。

具体地，维护一个负样本集合（不是队列），
1. 在训练开始之前，首先对数据集所有「key 样本」进行编码，得到 memory bank。
2. 每次训练时，从中随机挑选一些「负样本」作为当前 batch 的 negative key embeddings，然后 query encoder 编码得到当前的 query embeddings equerye_{query}equery​ ，使用对比损失训练当前步的 query encoder
3. 更新 bank：使用 equerye_{query}equery​ 去替换 memory bank 中的对应编码（这里使用直接替换，应该可以说明是无监督的对比学习方式，如果是有监督的方式的话是不能替换的，因为两个文本本来就不一样，可能需要再进行编码一次？）

可见，memory bank中一个样本的 key embedding 在它最后一次作为 query 时被更新，所以本质上被采样的 key 来自过去不同 step 的不同的 encoder ，因此非常地不一致。

虽然这个方法满足了“词典词表足够大”的条件，但是“QK编码器一致性”没有保证，因为Key编码总是落后于Query编码的，这个过程中无法保证Query-Key匹配的一致性。

Moco

使用两个方式：维护队列 + 维护key encoder

列表

为了保持大规模的词典词表大小，动态维护一个负样本队列。

每一次将batch_size大小的样本送入字典中，当队列满时，会将队列中最旧的批次淘汰掉。这样子做不仅可以扩展字典的大小，而且还可以很好的保持字典中的一致性。

为什么更新队列：随着训练的过程，网络在变，因此早期的数据可能会和网络不是那么匹配。

和 memory bank 一样，队列中的 key 编码不是来自同一个 key encoder，而是来自先前多个 mini-batch ，即一个逐渐演化的key encoder 。

key encoder

使用 query encoder 和 key encoder 分别进行编码，反向传播更新时只更新 query encoder。

不同的是对Key编码器的更新方式改为了 “同步” 的方式。

即：直接将当前step更新好的 Query编码器 的参数 根据权重 mmm 拷贝到 Key编码器 中。

θk←mθk+(1−m)θq\theta_k \leftarrow m \theta_k+(1-m) \theta_q θk​←mθk​+(1−m)θq​

为什么不直接全部拷贝 query encoder 的权重：会导致Key编码器训练状态的完全紊乱，即Key编码器训练不一致。
使得虽然队列中的key被不同的encoder编码（不同的mini-batch），但是这些encoder之间的不同是很小的。
mmm 通常是一个很大的值 (>0.99>0.99>0.99) 。这表明一个缓慢演化的key encoder对于采用队列的方法很关键。

其实个人有个疑问：
为什么实验部分直接复制，效果就那么差？：应该是 f_q 更新的太快了，导致同一个 queue 中的编码的差异太大，导致训练崩溃。
所以不能直接复制，但是这种动量的方式，虽然 queue 内部样本表征比较 consistent 了，但是 query 和 key 就不consistent 了呀。因为两个编码器其实也不不是很 consistent，超参数设置 0.99，感觉就是聊胜于无。

为了让模型能够学习到多个句子间的 语义相似性顺序 ，作者提出了一个新的预训练任务，建模「三元句子对」的蕴含关系。

给定一个句子 sss ，按不同的 mask ratio 做两次 mask，这里取 mask ratio=20%,40%\text{mask ratio} = 20\%, 40\%mask ratio=20%,40% ，得到 s′ , s′′s' \ ,\ s''s′ , s′′ 组成三元组句子对。

根据 mask ratio， sim⁡(hˉi,hˉi′)\operatorname{sim}\left(\bar{h}_i, \bar{h}_i^{ \prime}\right)sim(hˉi​,hˉi′​) 应该低于 sim⁡(hˉi,hˉi′′)\operatorname{sim}\left(\bar{h}_i, \bar{h}_i^{\prime \prime}\right)sim(hˉi​,hˉi′′​) ，这里使用 hinge loss：

Ltri =max⁡(0,sim⁡(hˉi,hˉi′′)−sim⁡(hˉi,hˉi′)+m)\mathcal{L}_{\text {tri }}=\max \left(0, \operatorname{sim}\left(\bar{h}_i, \bar{h}_i^{\prime \prime}\right)-\operatorname{sim}\left(\bar{h}_i, \bar{h}_i^{\prime}\right)+m\right) Ltri ​=max(0,sim(hˉi​,hˉi′′​)−sim(hˉi​,hˉi′​)+m)

SNCSE

SNCSE: Contrastive Learning for Unsupervised Sentence Embedding with Soft Negative Samples

无监督学习时，正样本需要数据增强，包括增，删，改token。simcse 是用的 dropout。

作者认为获取的正样本均极为相似，导致模型存在特征抑制，即「模型不能区分文本相似度和语义相似度，并更偏向具有相似文本，而不考虑它们之间的实际语义差异」。

这里作者提出了新的构造负例的方法，想要构造 文本相似 但是 语义相反 的负例，这里称为 软负样本 Soft Negative Sample 。
这样的话模型就不会只根据字形来判断了，从而更偏向于语义。

同时针对该软负样本 ，定义了 双向边际损失(bidirectional margin loss,BML) 进行训练。

样本构造

  • 正例生成方式

    • 与SimCSE相同，Dropout产生正例

  • 负例生成方式

    • 纯负例：batch内其他数据
    • 软负例：通过spacy进行语法解析，为动词添加否定词前缀。

软负样本

在SNCSE中，将原始样本的否定内容作为软负样本。利用Spacy工具对句子进行句子解析，获取句子的句法树、词性标签并标记词干，然后据这些信息，将句子转化为句法正确、语义清晰的否定句。

注意：这里的「否定」是通过添加「否定词」的显式否定，而不是词语反义的否定。因为添加否定词的否定，在文本相似度上，与原始样本更相似。

句向量构造

使用 PromptBert 的方法：
原始样本：The sentence: " XXX " means [MASK][\text{MASK}][MASK]
正例样本：The sentence of " X+X^{+}X+" means [MASK][\text{MASK}][MASK]
其他/负样本：The sentence of " X−X^{-}X− " means [MASK][\text{MASK}][MASK]
软负样本：The sentence of " X#X^\#X# " means [MASK][\text{MASK}][MASK]

并在训练过程中，像SimCSE一样，增加了一个带有tanh激活函数的全连接层对 h[MASK]h_{[\text{MASK}]}h[MASK]​ 进行变换，得到最终的句子向量：

h=tanh⁡(MLP(h[MASK]))h=\tanh \left(M L P\left(h_{[\text{MASK}]}\right)\right) h=tanh(MLP(h[MASK]​))

训练

使用 InfoNCE 训练正例样本和负样本
使用 双向边际损失(bidirectional margin loss,BML) 训练正例样本和软负样本

计算原始样本与正例和原始样本与软负例之间的 余弦相似差 ：

△= con_sim (hi,hi#)−con_sim⁡(hi,hi+)\triangle=\text { con\_sim }\left(h_i, h_i^\#\right)-\operatorname{con\_sim}\left(h_i, h_i^{+}\right) △= con_sim (hi​,hi#​)−con_sim(hi​,hi+​)

定义双向边际损失(bidirectional margin loss,BML)来模拟语义相似度差异：

LBML=ReLU⁡(△+α)+ReLU⁡(−△−β)L_{B M L}=\operatorname{ReLU}(\triangle+\alpha)+\operatorname{ReLU}(-\triangle-\beta) LBML​=ReLU(△+α)+ReLU(−△−β)

这样就可以把 余弦相似差 限制在 [−β,−α][-\beta, -\alpha][−β,−α] ，这样可以保证相似度差距并不会太小，也不会太大。

DiffCSE

Paper: DiffCSE: Difference-based Contrastive Learning for Sentence Embeddings

「不变对比学习」：采用不敏感的数据增强(比如 dropout)得到对比损失。
「可变对比学习」：采用敏感的数据增强(如，旋转变换，增删改句子)得到对比损失。
用不变对比学习和可变对比学习相互结合的方法，效果更好 。

这里作者

  1. 使用 dropout 作「不变对比学习」
  2. 使用基于MLM语言模型进行词语替换的方法作为「可变对比学习」，来学习「原始句子与编辑句子」之间的差异

，共同优化句向量表征。

左边就是同一个 batch 作传统的 NCE loss 学习

Lcontrast =−log⁡esim⁡(hi,hi+)/τ∑j=1Nesim⁡(hi,hj+)/τL_{\text {contrast }}=-\log \frac{e^{\operatorname{sim}\left(h_i, h_i^{+}\right) / \tau}}{\sum_{j=1}^N e^{\operatorname{sim}\left(h_i, h_j^{+}\right) / \tau}} Lcontrast ​=−log∑j=1N​esim(hi​,hj+​)/τesim(hi​,hi+​)/τ​

右边是一个 RTD 任务：

  1. 通过一定的 mask 比例对原始句子进行 mask
  2. 然后使用一个固定的 bert 将 mask 的位置还原
  3. 然后拼接左边的句向量 hhh ，送入到一个 discriminator 中，让他逐个 token 判断是否是生成的/mask/替换的 token。

LRTD x=∑t=1T(−1(xt′′=xt)log⁡D(x′′,h,t)−1(xt′′≠xt)log⁡(1−D(x′′,h,t)))LRTD =∑i=1NLRTDxiL_{\text {RTD }} ^x=\sum_{t=1}^T\left(-1\left(x_t^{\prime \prime}=x_t\right) \log D\left(x^{\prime \prime}, h, t\right)-1\left(x_t^{\prime \prime} \neq x_t\right) \log \left(1-D\left(x^{\prime \prime}, h, t\right)\right)\right)\\ L_{\text {RTD }} = \sum_{i=1}^N L_{R T D}^{x_i} LRTD x​=t=1∑T​(−1(xt′′​=xt​)logD(x′′,h,t)−1(xt′′​=xt​)log(1−D(x′′,h,t)))LRTD ​=i=1∑N​LRTDxi​​

一个 batch 的总的损失函数为：

L=Lcontrast +λLRTD L= L_{\text {contrast }} + \lambda L_{\text {RTD }} L=Lcontrast ​+λLRTD ​

对比学习损失为RTD损失200倍时，模型效果最优。


将句子向量 hhh 进行拼接的原因：

  1. 为了让梯度进行传播，RTD 任务也能训练到 Sentence Encoder
  2. 鼓励 Sentence Encoder ，能让 hhh 信息量足够大，从而使判别器能够区分 x′x'x′ 和 x′′x''x′′ 之间的微小差别。

当训练DiffCSE模型时，固定生成器G参数，只有句子编码器f和鉴别器D得到优化。
训练结束后，丢弃鉴别器D，只使用句子编码器f提取句子嵌入对下游任务进行评价。

在 SimCSE 模型中，采用 pooler 层（一个带有tanh激活函数的全连接层）作为句子向量输出。
本文实验发现，采用 带有BN的两层 pooler 效果更好。
