mysql多表操作

Posted on 2021-03-06 In Coding , MySQL

多表操作的结果是一个新的临时表，一般是用于 FROM 后面，然后进行新的 SELECT。

表文件的关系

  • [Coding, MySQL]通过表文件的数据关系来描述现实生活中的隶属关系。

  • 隶属关系角色划分：

    1. 一方：拥有资源的一方
    2. 多方：依附于一方的资源

  • 三种关系

    • 一对一

      • e.g. 用户和用户详情
        常用于单表拆分，将一个列数很多的用户信息表，拆分称为一个只包含基础信息的小表（列数小）和一个包含其他信息的大表（列数大），来提升操作效率。
      • 实现：
        在任意的一个表中加一个 UNIQUE 外键，关联另一个表的主键。
        UNIQUE 保证了一对一的关系。

    • 多对多

      • e.g. 学生-课程：一个学生可以选修多门课，一门课可以被多个学生选择。
      • 实现：建立第三个中间表（选课表），中间表的一个外键关联学生的主键，一个外键关联课程的主键。

    • 一对多 / 多对一

      • e.g. 部门-员工：一个部门对应多个员工，一个员工只对应一个部门。
      • 实现：多方表（员工）建立外键，指向一方表（部门）的主键。

表文件字段的分类

  • 主键字段：
    1. 存放主键编号，每一个表都应该有一个主键字段。
    2. 用来区分表中的唯一的一个元组/对象。
    3. 不为null，同一表中不可重复。
  • 非主键字段：描述主键编号。
  • 外键字段：
    1. 只存在多方表，描述多方数据与一方数据的依赖关系。
    2. 外键字段的值应该来源于一方表中的主键值。

多表查询

  • 本质：将多张表数据合并到同一个临时表中，然后使用六个查询命令，根据需求得到需要的数据。
  • 关键：如何将多张表中的数据合并到同一张表中。
  • 多张表数据合并方案：
    1. 连接查询 合并方案
      a. 首先确保两张表之间存在了隶属关系。
      b. 将两张表的内容水平方向拼接。
      c. 最终得到拥有完整隶属关系的新数据行。
    2. 联合查询 合并方案
      a. 不需要两张表存在隶属关系。
      b. 两张表的数据行沿着垂直方向堆砌。


连接查询合并方案

例子：查询各个部门名称及其部门下的职员名称。

  • 数据来源：Dept.dname 和 Emp.ename
  • 表文件的关系：Emp隶属于Dept
  • 命令：join
  • 运算过程：
  1. JOIN 将两个表水平拼接生成一张新的临时表，字段由两个表相加而成，新的字段名格式为表名.字段名，这样解决了重复字段名的问题。
  2. 临时表行数为原表行数做乘法，是一个笛卡尔积。
  3. 这时候行数很多，而且每一行是直接拼接的，很多数据根本不合法（脏数据）。这时候我们就需要进行过滤操作了 。
    Join 过滤操作会删除不合法的行，即主客键数据冲突的信息（脏数据），然后放置在一个新的临时表中，接受后续操作。

连接查询过滤方案：将合法数据行与脏数据行进行剥离的形为。

  • 命令格式： FROM 一方表 [INNER/LEFT/RIGHT] JOIN 多方表 ON 定位合法数据行条件
  • 合法数据行与脏数据行区分原则：
    1. 表文件存在明显外键字段：ON 一方表.主键字段 = 多方表.外键字段
    2. 表文件没有明显外键字段：ON 根据实际隶属关系进行判断
  • 分类：内连接过滤方案，外连接过滤方案，自连接过滤方案

Null 值现象

首先空值 '' 也是一个值，和普通值一样的，和 null 没关系。
在 MySQL 中，NULL 值与任何其它值的比较（即使是 NULL）永远返回 NULL，即 NULL = NULL 返回 NULL ，而我们筛选是需要返回 True / False 。

left join 时，如果多方表中没有和一方表中匹配的选项，那么一方表中的该 tuple 也会被输出，但是多方字段就是 null 了。

可以看到 |2 |null|null|null| 和 |5 |null|null|null| ，A.NAME=Null，并未关联到B表NAME为NULL的部分 B: |3 |null| ，所以 一方表中的 null，不会匹配多方表中的任何结果，即使多方表中也是 null 。

解决方法：

  1. 让 Null 也去匹配多方表中的 Null 或者 ''
    使用 COALESCE 关键字，COALESCE(A.NAME,'')，如果是 Null 则返回空值 。
    因为左右都将 Null 转换为了 ''，所以一方表中的 Null 可以匹配到多方表中的 Null 和 '' ，并且返回表还会显示原始的元素值。

COALESCE (expression_1, expression_2, ...,expression_n) 依次参考各参数表达式，遇到非 null 值即停止并返回该值。如果所有的表达式都是空值，最终将返回一个空值。

Union

Union (all): {a,b,b,c} U {a,b,b,b,e,f,f} = {a,a,b,b,b,b,b,c,e,f,f}

  • 定义：将两张表 沿着垂直方向进行堆砌 ，即把多次查询结果竖直合并起来，形成一个新的查询结果集。

  • 命令：

    • UNION会自动删除重复行（同一个表和 or 类似）
    • UNION ALL则不会删除重复数据。且效率高于 UNION 。

  • 要求：

    1. 参与合并的临时表字段结构保持一致，即字段个数/列数和字段类型的顺序保持一致。
    2. 联合查询生成临时表的字段只能来自于第一个临时表字段。即表头使用前面的父表。

    • 需求：将部门编号，部门名称，职员编号，职员名称展示出来

    • 临时表：
      1. SELECT deptno, dname FROM dept
      2. SELECT empno, ename FROM emp
      3. 通过UNION命令，将两个临时表沿着垂直方向堆砌。

子查询

  • 定义：在查询时无法从当前临时表得到需要的数据，此时mysql服务器允许开发人员通过完整查询语句从别的表中得到需要的数据，从而保证查询正常执行。
    这个起到帮助作用的完整查询语句相当于外援，在技术上被称之为子查询或者嵌套查询。

  • 适用条件：根据某些限制特征来得到某一目标群体的其他特征

  • 用法：使用 小括号 放置查询命令，令其返回一个值。可以看成调用了一次简单的函数。

  • 命令：SELECT * FROM t1 WHERE column1=(SELECT column1 FROM t2)

  • 根据查询结果，即 小括号() 内部查询命令返回值的不同，分为四种子查询：

    • 标量子查询：返回结果为一个标量
    • 列子查询：返回结果为一列
    • 行子查询：返回结果为一行
    • 表子查询：返回结果为一个临时表

  • 可用范围：

    • SELECT 命令：SELECT ename,sal,(SELECT AVG(sal) FROM emp) AS 'avg'
    • FROM 命令：把子查询的结果(内存里的一张表)当作一张临时表，然后交给外部继续对它进行处理。
    • WHERE 命令：把内层查询的结果当作外层查询的条件。
    • HAVING 命令：通过子查询来得到判断标准。HAVING avg(sal)>(SELECT avg(sal) FROM emp)
    • EXISTS 命令：对外层表进行循环，再对内表进行内层查询。
      和 in() 差不多，但是它们还是有区别的。主要是看两个张表大小差的程度。若子查询表大则用exists（内层索引），子查询表小则用in（外层索引）。

  • 运算符号：

  1. in子查询：内层查询语句仅返回一个数据列，这个数据列的值将供外层查询语句进行比较。
  2. any子查询：只要满足内层子查询中的任意一个比较条件，就返回一个结果作为外层查询条件。
  3. all子查询：内层子查询返回的结果需同时满足所有内层查询条件。
  4. 比较运算符子查询：子查询中可以使用的比较运算符如 >、>=、<=、<、=、 <>
  5. exists子查询：把外层的查询结果（支持多行多列），拿到内层，看内层是否成立，简单来说后面的返回true,外层（也就是前面的语句）才会执行，否则不执行。
  • 子查询的顺序：https://www.cnblogs.com/houqijun/p/6085603.html

标量子查询

子查询返回的是一个标量结果（数字，字符串，日期），最简单的形式，这种子查询被称为 标量子查询。

常见的操作符号为 > < >= <= = <>。

  • 例子：查看工资高于公司平均工资的职员信息

列子查询

子查询返回的是一列（多行），可以看作是一个集合或者列表。

常见操作符为 IN, NOT IN, ANY, SOME, ALL

  • 例子：
    • 查询销售部和市场部所有员工信息：
    • 查询比财务部所有人工资都高的员工信息 --》max
    • 查询比财务部任意一个人工资都高的员工信息 --》 min

行子查询

子查询返回结果为一行，可以看作一个新的实例，有结构关系。

常用操作符为：=, !=, IN, NOT IN

使用小括号进行组合，来实现行与行的比较。

  • 例子：查询和 Sam 薪资和直属领导相同的员工

返回结果为多行，一个临时表，常用操作为 IN。
