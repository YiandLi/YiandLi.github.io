git 相关文件

.git:objects/ ：存放历史修改信息
README
.gitignore: ignore all .bak files

分支

对某一个时刻的版本 copy 一份副本，这就是一个分支，然后不同的开发人员开发不同的分支，最后将所有的分支进行合并。

定义：file / option / Git / branch name
合并：choose a branch to merge into main

底层逻辑 ：
refs/heads/. 文件指向了具体的分支信息，然后分支又指向了提交信息，然后指向文件状态，最后到文件。

其他术语

Fetch ：抓取远程仓库到本地
Pull： 抓取并且合并

三个区域

工作区域 working directory：用户直接看到的是 Git work directory 。
存储区域 repository：git 内部维护了存储区域，存储不同版本和分支内部的文件，同时版本变更时会从中拿出文件覆盖在 git 工作区域。
暂存区域 staging area：用户在工作区域修改了文件，会对比上面两个区域的文件，用户可以选择 add + commit，即更新存储区域，和工作区域内容一致。

Git 命令

git init : 初始化本地的 git 仓库（ .git 存储区域 ）。
git config user.name <user_name> / git config user.email <user.mail> : 配置仓库信息
git log --oneline : 显示提交历史 （ 只显示前面几个 ）

删除：对需要删除的文件、文件夹进行如下操作:

x

提交：
git status : 查看暂存区状态，会对比本地和仓库中的差异，提供对比功能
git add : 工作区修改的文件发布到暂存区域
git add *.py : 添加所有 python 文件
git rm --cached <file> : 删除暂存区域中文件
git commit : 暂存区域文件更新到存储区域
git commit -m "<message>" : 设置本次提交的 message

恢复：
git restore <file> : 撤销误删除文件（只能在 commit 操作之前进行恢复 ）
git reset --hard <version_code> : 恢复历史的版本 （直接从git log 中拷贝版本号即可），但是会删除中间的几次提交记录。注意是直接还原到指定版本提交之后的状态，所以是完成的状态。
git revert <version_code> : 同样是恢复，但是是还原到指定版本提交之前的状态，不会丢失历史提交记录。

分支：git 中的分支是基于 commit 操作的，所以最少需要有一次 commit 。
git branch -v : 查看当前所有分支 ， * 表示当前分支
git branch <branch_name> : 创建分支
git branch -d <branch_name> : 删除分支
git checkout <branch_name> : 切换分支
git checkout -b <branch_name> : 创建并切换分支

合并：
首先 checkout 到主分支 main / master，然后 git merge <branch_name> : 合并目标分支到自己
如果上面的操作存在冲突，则会切换到 MERGING 状态，需要我们手动去修改文件，然后再次 add commit 该文件，就成功了。

远程仓库：URL地址变量 记录在 git config 文件中的 remote 字段
git remote -v : 查看管理的地址
git remote add <远程仓库名称> <git\@url> : 创建关联地址 ，<远程仓库名称> 为一个别名，表示某一个线上的主机/仓库
git remote rm <远程仓库名称> : 删除操作
git push <远程仓库名称> <分支名称>， 将本地的master分支推送到origin主机的master分支。如果master不存在，则会被新建。如果本地分支与远程分支名称相同，并且之前已经进行过关联，可以简化为 git push。
git pull <远程仓库名称> : 将远程仓库的状态拉取下来，同时会更新本地文件
