文件夹作为包

对于一个python项目，里面的每一个文件夹都可以认为是一个 package ，而每一个 .py 文件被认为是一个 module 。
如果一个 python 项目文件夹下含有 __init__.py 文件，那么这个文件夹便是一个 python 库。

当我们从一个 package 里面调用东西的时候，其内部 __init__.py 文件内的代码会被首先执行。
无论是直接 import package_1 这样直接导入整个包，还是通过 from package_1 import module 这样仅仅导入一个方法，都会被执行。

于是通常我们需要导入 package_1 文件夹内 pyfile1 文件中的 method1 方法时，我们需要写下 from package_1.pyfile1 import method1，多个外部文件中我们都需要这样写，比较繁琐。
我们可以在其对应的 __init__.py 文件内写下 from package_1.pyfile1 import method1，然后外部文件直接调用 from packaged_1 import method1 即可，逻辑就是执行了 from package_1.pyfile1 import method1 ，所以现在 __init__ 有 method1，然后我们再通过 __init__ 直接调用 method1。
__init__.py相当于 一个接口，我们在其内部注册一些方法，然后package 外面只能调用这个接口返回的某些方法和类别。

总结：

  1. 标识 package，在导入一个包时，实际上是导入了它的__init__.py文件。
    这样我们可以在__init__.py文件中批量导入我们所需要的模块，而不再需要一个一个的导入，比如导入 re，numpy 等…
  2. 直接定义了这个package可以返回的所有方法，对象。就不需要一个一个文件去翻了（*除外…）
  3. 定义__all__用来模糊导入
  4. 一般情况下，在目录很深时，我们使用 init，同时在 __init__ 中定义导入别名 as 来简化。
