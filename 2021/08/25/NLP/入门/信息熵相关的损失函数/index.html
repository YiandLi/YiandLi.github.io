信息熵相关的损失函数

Posted on 2021-08-25 Edited on 2022-05-22

熵

信息熵

  • 公式：H(x)=−∑i=1np(xi)log(p(xi))H(x) = -\sum_{i=1}^n p(x_i)log(p(x_i))H(x)=−∑i=1n​p(xi​)log(p(xi​))
    p(xi)p(x_i)p(xi​)：事件 xix_ixi​ 发生的概率。
    −logp(xi)-log p(x_i)−logp(xi​)：事件 xix_ixi​ 的信息量/熵，是一个正数。
  • 前提：事件 xix_ixi​ 之间是独立同分布的，所以满足信息量可以叠加，
  • 本质：
    • 编码方案完美时，最短的平均编码长度。
    • 香农信息量 log(1/p)log(1/p)log(1/p) 的期望，即对概率分布的量化。(一个事件结果的出现概率越低，其信息量越高。)
    • 记录随机事件结果的平均编码长度。（一个事件结果的出现概率越低，对其编码的bit长度就越长。）

交叉熵

  • 公式：H(p,q)=−∑i=1np(xi)log(q(xi))=H(p)+DKL(p∣∣q)H(p,q) = -\sum_{i=1}^n p(x_i)log(q(x_i)) = H(p) + D_{KL}(p||q)H(p,q)=−∑i=1n​p(xi​)log(q(xi​))=H(p)+DKL​(p∣∣q)
  • 本质：
    • 编码方案不完美时的平均编码长度。 即对于真实概率p的事件，使用q的编码方案进行编码。
    • p的信息熵 + p和q的相对熵
  • 在ML中：
    • 用来评估当前训练得到的概率分布，与真实分布的差异.我们的目标是求p，所以需要让交叉熵尽可能地小。
    • 若P是固定的分布，与训练无关；Q是估计的分布，应尽量等于 P。

相对熵/KL散度

  • 公式：DKL(P∣∣Q)=H(p,q)−H(p)=∑ip(i)logp(i)−∑ip(i)logq(i)=∑ip(i)logp(i)q(i)D_{KL}(P||Q)=H(p,q)-H(p)=\sum_i p(i)logp(i) - \sum_i p(i)logq(i) = \sum_i p(i)log \dfrac{p(i)}{q(i)}DKL​(P∣∣Q)=H(p,q)−H(p)=∑i​p(i)logp(i)−∑i​p(i)logq(i)=∑i​p(i)logq(i)p(i)​
  • 本质：由于编码方案不一定完美，导致的平均编码长度的增大值
  • 在ML中，真实分布的信息熵H§是常数，所以优化KL散度等价于优化交叉熵。

损失函数

两种交叉熵函数

有两种交叉熵损失函数：

L=−tjlog(yj)L=−∑itilog(yi)+(1−ti)log(1−yi)\begin{aligned} L &= -t_j log(y_j)\\ L &= -\sum_i t_i log(y_i) + (1-t_i)log(1-y_i) \end{aligned} LL​=−tj​log(yj​)=−i∑​ti​log(yi​)+(1−ti​)log(1−yi​)​

tit_iti​ 表示样本实际的类别表示，即one-hot表示， ti∈[1,0]t_i \in [1,0]ti​∈[1,0]。
yiy_iyi​ 表示机器出的预测值。
同为交叉熵损失函数，前者适用于softmax的多分类问题，而后者应用于logistic的二分类问题。这个表达可能不准确，因为多分类问题也可以使用多个逻辑斯蒂来判别。
即前者对应最后一层是softmax，后者对应最后一层是sigmoid（logistic问题）。

最后一层为softmax

L=−tjlog(yj)L = -t_j log(y_j) L=−tj​log(yj​)

因为用了softmax做归一化处理，所以可以将输出yjy_jyj​看作模型的预测分布。
tit_iti​为真实分布，所以正确类别为1，因为其余全为0，只要让正确类别的预测概率最大化即可。
PyTorch官方实现的CrossEntropyLoss()函数就是这种定义形式。

最后一层为sigmoid

L=−∑itilog(yi)+(1−ti)log(1−yi)L = -\sum_i t_i log(y_i) + (1-t_i)log(1-y_i) L=−i∑​ti​log(yi​)+(1−ti​)log(1−yi​)

sigmoid作为最后一层输出的话，那就不能把最后一层的输出看作成一个分布了，因为加起来不为1。
应该将最后一层的每个神经元看作一个分布，对应的target属于二项分布(target的值代表是这个类的概率)。

总结

其实本质还是上面信息论中讲的内容，只不过不同的是两种对应的分布。即使用同一事件的实际分布和预测分布得到loss。

  • L=−tjlog(yj)L = -t_j log(y_j)L=−tj​log(yj​)对应的分布为一个多分类的整体分布（softmax和为1，one-hot和也为1）。
  • L=−∑itilog(yi)+(1−ti)log(1−yi)L = -\sum_i t_i log(y_i) + (1-t_i)log(1-y_i)L=−∑i​ti​log(yi​)+(1−ti​)log(1−yi​)对应的是单独类的二元分布。

Dice Loss

Dice Similariy Coefficient/DSC

  • 定义：对任意两个集合A和B，DSC(A,B)=2∣A∩B∣∣A∣+∣B∣DSC(A,B)=\frac{2\left|A\cap B\right|}{\left|A\right|+\left|B\right|}DSC(A,B)=∣A∣+∣B∣2∣A∩B∣​
  • 推导：以二分类为例
predict_label=1 predict_label=0
real_label=1 True Positive (TP) False Negative (FN)
real_label=0 False Positive (FP) True Negative (TN)

A表示模型预测的正例： TP+FPTP+FPTP+FP
B表示ground truth的正例： TP+FNTP+FNTP+FN
所以 A∩B=TPA\cap B=TPA∩B=TP ， DSC=2TP2TP+FP+FN=...=F1DSC= \dfrac{2TP}{2TP+FP+FN}=...=F1DSC=2TP+FP+FN2TP​=...=F1

F1= 2 recall −1+ precision −1= 2⋅ precision ⋅ recall  precision + recall = tptp+12(fp+fn)DSC= 2TP2TP+FP+FN\begin{aligned} F_{1}=&\ \frac{2}{\text { recall }^{-1}+\text { precision }^{-1}}\\\\ =&\ 2 \cdot \frac{\text { precision } \cdot \text { recall }}{\text { precision }+\text { recall }}\\\\ =&\ \frac{\mathrm{tp}}{\mathrm{tp}+\frac{1}{2}(\mathrm{fp}+\mathrm{fn})} \\\\ DSC=&\ \frac{2 T P}{2 T P+F P+F N} \end{aligned} F1​===DSC=​  recall −1+ precision −12​ 2⋅ precision + recall  precision ⋅ recall ​ tp+21​(fp+fn)tp​ 2TP+FP+FN2TP​​

所以可以看出是直接优化F1的。

  • 概率化DSC：对某个样本xix_ixi​，对待预测某一类，其ground_truth值为 yi∈{0,1}y_i\in \{0,1\}yi​∈{0,1} ，其网络预测值为 pi∈[0,1]p_i\in [0,1]pi​∈[0,1]。
    于是，单个样本的DSC可以写为 DSC(xi)=2piyipi+yiDSC(x_i)=\dfrac{2p_{i}y_{i}}{p_{i}+y_{i}}DSC(xi​)=pi​+yi​2pi​yi​​。
    注意这里的 yiy_iyi​ 和 pip_ipi​ 的主体都是一个样本。
    这里提出问题：从定义来看，主体为一个batch或者所有样本才对，即直接对f1求loss，这里怎么泛化到单个样本？同时这里泛化到单个样本为什么对每个类的概率/softmax的元素都作计算？

    个人理解： 可以用微分的角度，将某一个样本分为1000份，当然这1000份的真实值全都是该样本的类别。将1000份看作一个整体来求F1，概率看作机器预测为1的数量。正概率转化为相应的预测数，即看作TP，反之负概率可以看作FP。
    • 同时考虑多个类别的f1，微分的角度看，p和y可以对照上面的集合A和B，结果是对应类别的f1。Macro f1就是会综合多个类别的f1。
    • TP+FP的总数量即真实值为1的总数量，即微分后的1000，或者概率和为1。
      这样看来CE只关注Precise，又因为真实值的样本总数是确定的，即TP+FN/FP+TN的值是确定的，所以TP上升，FN就下降，所以只关注TP数量上升即可。
      而对于F1来说，分析可知，更新后，正例的precise增大，负例的recall增大。
      但是这里又有一个问题： 上文也说了TP+FP的总数恒定，如果为二分类问题，那么TP增大不就代表着FP减小？又回到CE了。如果单纯地从loss构造来看，无关泛化性，ce只看正概率，而DSC同时想减小负概率，那平常的loss是不是也加上减小负概率效果可以更好？
  • 平滑系数
    如果y=0，则值为0，所以添加一个平滑系数 γ\gammaγ ： DSC(xi)=2piyi+γpi+yi+γDSC(x_i)=\dfrac{2p_{i}y_{i}+\gamma}{p_{i}+y_{i}+\gamma}DSC(xi​)=pi​+yi​+γ2pi​yi​+γ​ 。

Label Smoothing

原始论文：《Rethinking the inception architecture for computer vision》

问题

我们使用传统的 ce loss做训练时，模型输出 逻辑斯蒂值 ziz_izi​ ，softmax 后概率值为：

pi=exp⁡(zi)∑j=1Kexp⁡(zj)p_i = \frac{\exp(z_i)}{\sum_{j=1}^{K}\exp (z_j)} pi​=∑j=1K​exp(zj​)exp(zi​)​

假设真实标签为 yyy ，则 one-hot 概率分布为：

ti=1 if  i=y otherwise 0t_i = 1 \ \text{if}\ \ i=y\ \text{otherwise}\ 0 ti​=1 if  i=y otherwise 0

于是ce损失为：

l(t,p)=−∑i=1Ktilog⁡pil(t,p)=-\sum_{i=1}^K t_i \log p_i l(t,p)=−i=1∑K​ti​logpi​

想让损失函数最小，直接通过正规方程求解：

∂l∂zi=pi−ti=0ptrue=1pfalse=0\begin{aligned} \frac{\partial l}{\partial z_i} &= p_i - t_i = 0\\ p_{true} &= 1\\ p_{false} &= 0 \end{aligned} ∂zi​∂l​ptrue​pfalse​​=pi​−ti​=0=1=0​

即只需要真实标签输出的 softmax 值为1， 其他非真实标签的 softmax 值为0。
即真实标签的逻辑斯蒂值 ztruez_{true}ztrue​ 为任意常数，其他标签的逻辑斯蒂值 zfalsez_{false}zfalse​ 为 −∞-\infty−∞ (保证 pfalsep_{false}pfalse​ 近似为0)。

这种情况一般很难达到，因为有l2正则存在，同时 ztruez_{true}ztrue​ 会远大于 zfalsez_{false}zfalse​ ，导致：

  1. 过拟合，将所有的概率都赋给了真值，泛化能力下降。
  2. 导数 ∂l∂zi=pi−ti∈(−1,1)\dfrac{\partial l}{\partial z_i} = p_i - t_i\in(-1,1)∂zi​∂l​=pi​−ti​∈(−1,1) ，导数最大也才 1，想让 zfalse=−∞z_{false}=-\inftyzfalse​=−∞ ，需要迭代很多次。

从标签角度看，在实际情况下，使用 one hot 作为标签方式也会带来问题：

  1. 单个样本来看，其特征其实也不绝对，肯定多多少少有其他样本的特征。这样直接标1，会导致过拟合。
  2. 模型角度看，可能会对预测过于自信，因为样本答案就是绝对的1，从而忽略一些其他样本的特征和标签。

解决

调整学习的编码形式，其中 ε\varepsilonε 为超参数一般设置为 0.1，KKK 为类别数量。

ti={1−ε if   i= true εK−1 otherwise t_{i}=\left\{\begin{array}{ccc} 1-\varepsilon & \text { if }\ \ i=\text { true } \\ \frac{\varepsilon}{K-1} & \text { otherwise } \end{array}\right. ti​={1−εK−1ε​​ if   i= true  otherwise ​

带入损失后，同样求 ∂l∂zi=0\dfrac{\partial l}{\partial z_i}=0∂zi​∂l​=0 ，可得：

zi∗={log⁡((K−1)(1−ε)/ε)+α if   i= true α otherwise z_i^*=\left\{\begin{array}{ccc} \log\left((K-1)(1-\varepsilon)/\varepsilon\right)+\alpha & \text { if }\ \ i=\text { true } \\ \alpha & \text { otherwise } \end{array}\right. zi∗​={log((K−1)(1−ε)/ε)+αα​ if   i= true  otherwise ​

α\alphaα 可以是任意实数，错误类的 zfalse∗z^*_{false}zfalse∗​ 不会要求是负无穷。且错误类和正确类的logit值之间的距离也不用那么大了。

论文中，定义了 gap 为 ztrue∗−zfalse∗=log⁡((K−1)(1−ε)/ε)z^*_{true}-z^*_{false} = \log\left((K-1)(1-\varepsilon)/\varepsilon\right)ztrue∗​−zfalse∗​=log((K−1)(1−ε)/ε) ，超参数 ε=0\varepsilon =0ε=0 时，gap值为 ∞\infty∞ ；随着 ε\varepsilonε 增大，gap值不断减小；当 ε=K−1K\varepsilon=\frac{K-1}{K}ε=KK−1​ 时，所有的 z∗z^*z∗ 都相同。

label-smooth的编码方式只要正确类和错误类有一定的数值误差即可，这个取决于分类的类别数量 KKK 和超参数 ε\varepsilonε 。网络极使在正则化的情况下也比one-hot容易学习到最优情况。
