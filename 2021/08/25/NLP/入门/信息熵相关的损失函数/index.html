三种损失函数

人脑中有一个模型(量化到训练集)，电脑上也存了一个模型，终极目标是让这两个模型一致。
那么如何衡量两个模型的差异，这就用到损失函数了。

极大似然法和最小二乘法都是定量衡量两个模型的差异。
交叉熵是先把模型换成了熵这个数值，然后再用这个数值来比较模型的差异。

平方是因为绝对值不可导
但是使用这个损失函数进行过分类问题时，很麻烦。

极大似然函数

似然：概率的概率，即首先设定原子事件/随机变量的概率，然后表示观测事件的联合总概率，这个总概率就表示我们假设的概率的似然。即估计的子概率让观测事件发生的可能性。

最大似然估计法，就是找到似然值最高的那个模型。

具体地，就是让模型给出事件发生的概率，即似然，训练目标为让这个似然值不断变大，即不断变成最优模型。

P(x1,x2,x3,x4,…,xn∣W,b)=∏i=1nP(xi∣W,b)\begin{aligned} & P\left(x_1, x_2, x_3, x_4, \ldots, x_n \mid W, b\right) \\ & =\prod_{i=1}^n P\left(x_i \mid W, b\right) \end{aligned} ​P(x1​,x2​,x3​,x4​,…,xn​∣W,b)=i=1∏n​P(xi​∣W,b)​

这里 xxx 对应训练集/观测数据的发生，可以理解成标签=1，即事件发生， W,bW,bW,b 为模型参数。

继续， W,bW,bW,b 表示一组固定的模型参数， 定义 yiy_iyi​ 为模型输出 yi=NNW,b(input)y_i=NN_{W,b}(input)yi​=NNW,b​(input) ，即概率值，然后得到：

∏i=1nP(xi∣yi)\prod_{i=1}^n P\left(x_i \mid y_i\right) i=1∏n​P(xi​∣yi​)

而整体这个式子，表示模型 W,bW,bW,b 让事件发生的概率。

交叉熵

熵是用来定义一个系统的混乱程度，用来统一衡量模型。

信息量 −log⁡2(p)-\log_2(p)−log2​(p) 表示了一个事件从不确定转化到确定的难度有多大。
熵 ∑i−pilog⁡2(pi)\sum_i - p_i \log_2(p_i)∑i​−pi​log2​(pi​) 是衡量整个系统从原始的不确定转化到确定（概率 p=1），的难度有多大。
这里不是直接相加，而是使用概率做作为权重，即是所有可能发生的事件的信息量的期望。

这里使用相对熵来衡量当前模型转化到真实观测数据（概率是 pip_ipi​ ），系统需要的难度/信息量大小，即 ∑ip(i)logp(i)q(i)\sum_i p(i)log \dfrac{p(i)}{q(i)}∑i​p(i)logq(i)p(i)​ ，即表示两个系统相差的信息量。
最好的模型就是让这个相差的信息量最小的模型。相对熵的分子相同，所以可以转化为交叉熵。

负号是写在定义中的。

熵

信息熵

  • 公式：H(x)=−∑i=1np(xi)log(p(xi))H(x) = -\sum_{i=1}^n p(x_i)log(p(x_i))H(x)=−∑i=1n​p(xi​)log(p(xi​))
    p(xi)p(x_i)p(xi​)：事件 xix_ixi​ 发生的概率。
    −logp(xi)-log p(x_i)−logp(xi​)：事件 xix_ixi​ 的信息量/熵，是一个正数。
  • 前提：事件 xix_ixi​ 之间是独立同分布的，所以满足信息量可以叠加，
  • 本质：
    • 编码方案完美时，最短的平均编码长度。
    • 香农信息量 log(1/p)log(1/p)log(1/p) 的期望，即对概率分布的量化。(一个事件结果的出现概率越低，其信息量越高。)
    • 记录随机事件结果的平均编码长度。（一个事件结果的出现概率越低，对其编码的bit长度就越长。）

交叉熵

  • 公式：H(p,q)=−∑i=1np(xi)log(q(xi))=H(p)+DKL(p∣∣q)H(p,q) = -\sum_{i=1}^n p(x_i)log(q(x_i)) = H(p) + D_{KL}(p||q)H(p,q)=−∑i=1n​p(xi​)log(q(xi​))=H(p)+DKL​(p∣∣q)
  • 本质：
    • 编码方案不完美时的平均编码长度。 即对于真实概率p的事件，使用q的编码方案进行编码。
    • p的信息熵 + p和q的相对熵
  • 在ML中：
    • 用来评估当前训练得到的概率分布，与真实分布的差异.我们的目标是求p，所以需要让交叉熵尽可能地小。
    • 若P是固定的分布，与训练无关；Q是估计的分布，应尽量等于 P。

相对熵/KL散度

  • 公式：DKL(P∣∣Q)=H(p,q)−H(p)=∑ip(i)logp(i)−∑ip(i)logq(i)=∑ip(i)logp(i)q(i)D_{KL}(P||Q)=H(p,q)-H(p)=\sum_i p(i)logp(i) - \sum_i p(i)logq(i) = \sum_i p(i)log \dfrac{p(i)}{q(i)}DKL​(P∣∣Q)=H(p,q)−H(p)=∑i​p(i)logp(i)−∑i​p(i)logq(i)=∑i​p(i)logq(i)p(i)​
  • 本质：由于编码方案不一定完美，导致的平均编码长度的增大值
  • 在ML中，真实分布的信息熵H(p)是常数，所以优化KL散度等价于优化交叉熵。

损失函数

两种交叉熵函数

有两种交叉熵损失函数：

L=−tjlog(yj)L=−∑itilog(yi)+(1−ti)log(1−yi)\begin{aligned} L &= -t_j log(y_j)\\ L &= -\sum_i t_i log(y_i) + (1-t_i)log(1-y_i) \end{aligned} LL​=−tj​log(yj​)=−i∑​ti​log(yi​)+(1−ti​)log(1−yi​)​

tit_iti​ 表示样本实际的类别表示，即one-hot表示， ti∈[1,0]t_i \in [1,0]ti​∈[1,0]。
yiy_iyi​ 表示概率。
同为交叉熵损失函数，前者适用于softmax的多分类问题，而后者应用于logistic的二分类问题。这个表达可能不准确，因为多分类问题也可以使用多个逻辑斯蒂来判别。
即前者对应最后一层是softmax，后者对应最后一层是sigmoid（logistic问题）。

最后一层为softmax

L=−tjlog(yj)L = -t_j log(y_j) L=−tj​log(yj​)

因为用了softmax ei/∑jeje^i/\sum_j e^jei/∑j​ej 做归一化处理，所以可以将输出yjy_jyj​看作模型的预测分布。
tit_iti​为真实分布，所以正确类别为1，因为其余全为0，只要让正确类别的预测概率最大化即可。
PyTorch官方实现的CrossEntropyLoss()函数就是这种定义形式。

最后一层为sigmoid

L=−∑itilog(yi)+(1−ti)log(1−yi)L = -\sum_i t_i log(y_i) + (1-t_i)log(1-y_i) L=−i∑​ti​log(yi​)+(1−ti​)log(1−yi​)

sigmoid作为最后一层输出的话，那就不能把最后一层的输出看作成一个分布了，因为加起来不为1。
应该将最后一层的每个神经元看作一个分布，对应的target属于二项分布(target的值代表是这个类的概率)。

总结

其实本质还是上面信息论中讲的内容，只不过不同的是两种对应的分布。即使用同一事件的实际分布和预测分布得到loss。

  • L=−tjlog(yj)L = -t_j log(y_j)L=−tj​log(yj​)对应的分布为一个多分类的整体分布（softmax和为1，one-hot和也为1）。
  • L=−∑itilog(yi)+(1−ti)log(1−yi)L = -\sum_i t_i log(y_i) + (1-t_i)log(1-y_i)L=−∑i​ti​log(yi​)+(1−ti​)log(1−yi​)对应的是单独类的二元分布。

Dice Similariy Coefficient/DSC

  • 定义：对任意两个集合A和B，DSC(A,B)=2∣A∩B∣∣A∣+∣B∣DSC(A,B)=\frac{2\left|A\cap B\right|}{\left|A\right|+\left|B\right|}DSC(A,B)=∣A∣+∣B∣2∣A∩B∣​
  • 推导：以二分类为例
predict_label=1 predict_label=0
real_label=1 True Positive (TP) False Negative (FN)
real_label=0 False Positive (FP) True Negative (TN)

    个人理解： 可以用微分的角度，将某一个样本分为1000份，当然这1000份的真实值全都是该样本的类别。将1000份看作一个整体来求F1，概率看作机器预测为1的数量。正概率转化为相应的预测数，即看作TP，反之负概率可以看作FP。

      而对于F1来说，分析可知，更新后，正例的precise增大，负例的recall增大。
      但是这里又有一个问题： 上文也说了TP+FP的总数恒定，如果为二分类问题，那么TP增大不就代表着FP减小？又回到CE了。如果单纯地从loss构造来看，无关泛化性，ce只看正概率，而DSC同时想减小负概率，那平常的loss是不是也加上减小负概率效果可以更好？

Asymmetric Loss For Multi-Label Classification

ICCV2021 Allibaba : https://arxiv.org/abs/2009.14119

场景为 multi-label classification setting ，将多标签分类任务视为多个二分类任务，每一个类别的分数输入到 sigmoid函数中得到对应概率。

做个复习 ，ASL 是根据 Focal Loss 进行改进的。

Focal Loss 原始公式为：

{L+=(1−p)γlog⁡(p)L−=pγlog⁡(1−p)\left\{\begin{array}{l} L_{+}=(1-p)^\gamma \log (p) \\ L_{-}=p^\gamma \log (1-p) \end{array}\right. {L+​=(1−p)γlog(p)L−​=pγlog(1−p)​

γ\gammaγ 比较高的时候，Focal Loss 会抑制 easy sample 。对于负样本，抑制 easy negative 合理，但是对于正样本，尽管一些 positive sample 过于 easy，但是我们不想抑制它们，尤其是在样本极端不均的情况下，每一个正样本都很珍贵。

Asymmetric Focusing：
设置 γ−>γ+\gamma^{-}>\gamma^{+}γ−>γ+ ，用来解耦对 positive sample 和 negative sample的 decay rate，做到在 easy sample 的范围内，会对 negative 抑制较大、对 positive 抑制相对较小。对比下面的方法，这是 一种 soft threshold 。

Asymmetric Probability Shifting：
设置 shifted probability，即直接丢弃easy negative（预测probability很低的negative），这是一种hard threshold 。

shifted probability:  pm=max⁡(p−m,0)asymmetric probability-shifted focal loss:  L−=(pm)γlog⁡(1−pm)\text{shifted probability: }\ p_m=\max (p-m, 0)\\ \text{asymmetric probability-shifted focal loss: }\ L_{-}=\left(p_m\right)^\gamma \log \left(1-p_m\right) shifted probability:  pm​=max(p−m,0)asymmetric probability-shifted focal loss:  L−​=(pm​)γlog(1−pm​)

probability shifting其实就是将损失函数右移

然后将上面两个方法组合起来，就得到 ASL：

ASL={L+=(1−p)γ+log⁡(p)L−=(pm)γ−log⁡(1−pm)pm=max⁡(p−m,0)\begin{aligned} A S L &=\left\{\begin{array}{l} L_{+}=(1-p)^{\gamma_{+}} \log (p) \\ L_{-}=\left(p_m\right)^{\gamma_{-}} \log \left(1-p_m\right) \end{array}\right.\\ p_m &=\max (p-m, 0) \end{aligned} ASLpm​​={L+​=(1−p)γ+​log(p)L−​=(pm​)γ−​log(1−pm​)​=max(p−m,0)​

Adaptive asymmetric：
用 Adaptive Asymmetry 在每一个 batch 后动态调整指数权重 ，防止「概率差/confidence gap Δp\Delta pΔp 」小于0的情况出现 。

首先在训练时监控网络输出的 probability pˉ\bar ppˉ​ ，得到 positive confidence 和 negative confidence ：

pt={pˉ if y=11−pˉ otherwise p_t= \begin{cases}\bar{p} & \text { if } y=1 \\ 1-\bar{p} & \text { otherwise }\end{cases} pt​={pˉ​1−pˉ​​ if y=1 otherwise ​

然后对正label对应概率取平均，负label对应概率取平均，得到 positive average confidence pt+p_t^+pt+​ 和 negative average confidence pt−p_t^-pt−​ ，如果这个值很低，则说明模型并没有很好地学习到对应的特征。

进而得到 confidence gap Δp\Delta pΔp ：

Δp=pt+−pt−\Delta p = p_t^+ - p_t^- Δp=pt+​−pt−​

可以根据 pt+p_t^+pt+​ 和 pt−p_t^-pt−​ 判断模型的偏向性。A large negative probability gap 说明模型预测时对负样本赋予了过多的bias，也就表明训练时低估了正样本的损失。
虽然是从输出概率出发，这个 gap 同样也是网络输出的特征的表示，是BP不断优化的结果，如果负样本的平均p一直高于正样本的平均p，则说明网络依然没能很好的关注正样本，而是服用了样本不均匀，将关注过多放在负样本的优化上。

于是根据 confidence gap Δp\Delta pΔp 调整负样本的指数权重：

γ−←γ−+λ(Δp−Δptarget )\gamma_{-} \leftarrow \gamma_{-}+\lambda\left(\Delta p-\Delta p_{\text {target }}\right) γ−​←γ−​+λ(Δp−Δptarget ​)

λ\lambdaλ 为 a dedicated step size，步长
Δptarget\Delta p_{target}Δptarget​ 为 a desired probability gap ，用于动态调整 probability margin。这个值越大，会使得训练时更关注正样本，能够在整个训练过程中动态增加不对称水平，迫使优化过程更多地关注正样本的梯度。不过动态的gamma-其实在数据上并没有静态的gamma-效果好，低了0.2%，不过好在可以放心收敛，不必调整超参。

为什么以 e 为底 -》 最大熵

训练时，其实我们已经默认了前提假设：似然值最大的模型，就是最优的模型，因为我们的训练过程就是在让似然值不断变大。即给定一组观测变量（训练集合），最优模型可以让观测变量的可能性最大，那么就体现在我们的损失函数上，让熵（信息量，bit长度）最小，见上文。

此外，只要选择 softmax / sigmoid，那么还有一个假设就被勾选了：最大熵原理。

训练集合样本点有限，那么即使能全部拟合，这样的模型也有无数个。我们就是选择熵最大的那个模型，作为最终模型。

即，既能满足已知的信息（训练集），又能让未知的信息熵最大，换言之，让未知的信息概率比较平均，也可以看成是泛化性强。
