为了解决无模型的马尔可夫决策过程

  • 蒙特卡洛 MC
  • 时间差分 TD

都是不断更新价值，通过价值函数来更新策略。
区别在于如何计算价值，MC 是探索到轨迹结束，得到累计折扣奖励 G；而 TD 只探索一步，使用的是下一个状态的估计量来计算当前的估计量（和上一章动态规划的方法挺像）。

和前面一样，当使用动态规划方式求解马尔可夫决策过程的时候（动态规划体现在价值函数上， vtv_tvt​ 依赖于 vt+1v_{t+1}vt+1​ ，而对这个公式进行求解使用到了迭代法），
可以使用广义上的策略迭代方法，即每一步迭代上：

  1. 策略评估：使用当前的策略计算状态价值函数：遍历 k 次，，类似GNN聚合k次；每次遍历更新每个状态的价值
    牢记：是用状态机制来评估策略

  2. 策略改善：根据最终的状态价值贪婪地更新每个状态的策略函数

其中，状态价值函数的计算方式为（体现了动态规划思想）：

vπ(s)=∑a∈Aπ(a∣s)(Rsa+γ∑s′∈SPssa,vπ(s′))v_\pi(s)=\sum_{a \in A} \pi(a \mid s)\left(R_s^a+\gamma \sum_{s^{\prime} \in S} P_{s s}^a, v_\pi\left(s^{\prime}\right)\right) vπ​(s)=a∈A∑​π(a∣s)(Rsa​+γs′∈S∑​Pssa​,vπ​(s′))

当模型 PssaP_{s s}^aPssa​ 已知时，可以直接利用未来的状态计算得到当前状态的价值。

当模型未知时，需要使用其他方法计算状态价值，即评估当前策略。

蒙特卡洛 MC

vπ(s)=Eπ[Gt∣St=s]=Eπ[∑k=0∞γkRt+k+1∣St=s]qπ(s)=Eπ[∑k=0∞γkRt+k+1∣St=s,At=a]\begin{gathered} v_\pi(s)=E_\pi\left[G_t \mid S_t=s\right]=E_\pi\left[\sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \mid S_t=s\right] \\ q_\pi(s)=E_\pi\left[\sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \mid S_t=s, A_t=a\right] \end{gathered} vπ​(s)=Eπ​[Gt​∣St​=s]=Eπ​[k=0∑∞​γkRt+k+1​∣St​=s]qπ​(s)=Eπ​[k=0∑∞​γkRt+k+1​∣St​=s,At​=a]​

蒙特卡洛的方法：利用随机样本估计期望，即利用 经验平均 来代替随机变量的期望。

实验 ：利用策略 π\piπ 可以产生很多次实验（episode），每次实验都是从 任意的初始状态 开始， 直到最终状态 ，即探索。
对于一次实验 s1,a1,r2,...sTs_1, a_1, r_2, ... s_Ts1​,a1​,r2​,...sT​ ，其内部出现了多次状态 s ，则这时候我们可以用状态 s 后面序列来计算状态 s 的价值回报返回值。对于某一次出现状态 s，可以将其后续的 reward 整合起来作为其状态价值：

Gt(s)=Rt+1+γRt+2+...G_t(s) = R_{t+1}+\gamma R_{t+2} + ... Gt​(s)=Rt+1​+γRt+2​+...


经验 ：利用策略来产生很多实验，决定了正确的价值函数，所以是核心。
平均 ：就是求上面多个实验，得到出现在不同位置的状态 s ，然后利用其后续价值来计算平均状态价值函数。

如下图，两次实验中，状态 s 出现了三次 .

这里求平均时，分为第一次访问蒙特卡洛方法和多次访问蒙塔卡洛方法。
每次实验中某一个状态会出现多次，此时：

  1. 第一次访问蒙特卡罗方法：只利用（某次实验中）第一次访问到状态 s1s_1s1​ 的价值，来计算平均状态价值
  2. 每次访问蒙特卡洛方法：利用所有访问到该状态的价值。

为了保证价值函数的收敛性，需要逐个扫描状态空间中的状态。
无模型方法中充分评估价值函数的前提是：每一个状态都要有机会被访问到，所以必须采用一定的策略。

方法之一： 探索性初始化 ， 即每个状态都有机会作为初始状态。

此外，还需 对策略进行改进 ，并且找到一个 能够进行迭代计算的平均方法 。

蒙特卡洛 - 策略改善

和上一章一样，选择动作价值最大的动作：

π(s)=arg⁡max⁡aq(s,a)\pi(s) = \arg\max_a q(s,a) π(s)=argamax​q(s,a)

因为我们不知道模型，即选定了动作后，下一个状态是模拟得到的，无法保证用了这个 act 就能到达下一个 state，所以不能使用状态价值这个标准来更新策略。
换言之，在推理时，如果你选定了这个动作，下一个状态是其他的比较差的状态，那么不是傻了吗？

所以这里自由探索选定一个动作，得到下一个随机状态，然后让模型随机向后进行探索，使用 （s,a）之后的价值 G 来决定动作价值。

G的意义：在某个路径上，状态S到最终状态的总收获。
V的意义：V是G的平均数。

注意 v 和 策略 π\piπ 相关的 ，因为策略会改变整个路径的走向，从而改变了统计的奖励。

算法

随机初始化：初始状态是随机分配选择的，来保证每一个状态和行为都有机会被选中。

在活动的随机 state + act 后，后续的 act 则是根据 policy 选定当前认定的最优动作，进而生成整个实验序列 。

这里的最小单元是一个 (s,t) 表示当前状态 s 下使用某一个 act a ；使用了这个 act 之后其实下一个状态也是不确定的，进而整个轨迹也是不确定的，然后计算所有可能轨迹给予的反馈 G ，计算期望，作为平均的动作价值。

探索策略

上述算法的前提条件是每一个 (s,t) 对都能够被选中，
首先 初始状态是随机分配的，那么如何保证每个 act 都能有机会被选中呢 ？或者说选择什么 探索策略 呢？

温和策略 的要求：对所有状态 s 和 动作 a，都需要有 π(a∣s)>0\pi(a|s)>0π(a∣s)>0 ，可以使用 ε−greedy\varepsilon-greedyε−greedy 搜索

π(a∣s)←{1−ε+ε∣A(s)∣ if a=argmax⁡aQ(s,a)ε∣A(s)∣ if a≠argmax⁡aQ(s,a)\pi(a \mid s) \leftarrow \begin{cases}1-\varepsilon+\dfrac{\varepsilon}{|A(s)|} & \text { if } a=\operatorname{argmax}_a Q(s, a) \\ \dfrac{\varepsilon}{|A(s)|} & \text { if } a \neq \operatorname{argmax}_a Q(s, a)\end{cases} π(a∣s)←⎩⎪⎪⎨⎪⎪⎧​1−ε+∣A(s)∣ε​∣A(s)∣ε​​ if a=argmaxa​Q(s,a) if a=argmaxa​Q(s,a)​

这里涉及到两个策略：「探索策略（行动策略）」和 「评估和改善策略」

  • 如果两者相同，则为 on-policy

  • 如果两者不同，则为 off-policy：可以保证探索的充分性，即探索时使用 ε−soft\varepsilon-softε−soft 搜索，而用于评估和改善的策略为 贪心搜索 ； 在 off-policy 场景下，需要使用重要性采样。

  • 要遵守前提条件，「探索策略」包含「评估和改善策略」 。

off-policy

重要性采样(Importance Sampling)详细学习笔记

需要更新的目标策略为 π\piπ
实验时采样的策略为 μ\muμ
首先需要满足「覆盖性条件」，即行动策略 μ\muμ 产生的行为覆盖需要包含目标策略 π\piπ 产生的行为 ，即对于任意的 π(a∣s)>0\pi(a|s)>0π(a∣s)>0 都要有 μ(a∣s)>0\mu(a|s)>0μ(a∣s)>0 ，这样可以保证训练过程中可以采样到目标策略的动作，从而进行更新。

但是还是有一个 mismatch 的问题，即 行为策略 和 目标策略 不一致。
我们其实是在使用 训练策略 模拟 目标策略，中间有个 gap，这个时候就需要使用到 重要性采样 。

p(z)p(z)p(z) 可以看作随机变量 zzz 的概率密度函数；f(z)f(z)f(z) 可以看作随机变量 zzz 的价值函数。
应用到RL中， p(z)p(z)p(z) 为策略中动作 z 被选择到的概率，而 f(z)f(z)f(z) 为选择该动作之后实验得到的动作价值函数 GGG 。

而离散状态下：

对于变量 z\mathrm{z}z, 根据 q(z)\mathrm{q}(\mathrm{z})q(z) 这个概率分布, 采样 N\mathrm{N}N 个数据, 每个数据对应有值 f(zn)\mathrm{f}\left(\mathrm{z}^{\mathrm{n}}\right)f(zn), 对他们 求平均, 就近似整个概率分布的期望。


这个概率密度函数 p(z)p(z)p(z) ，即目标策略函数，一般是很复杂的（并不是未知），无法利用解析的方法产生用于逼近期望的样本，这时，我们可以选用一个概率分布很简单，产生样本很容易的概率分布 q(z)q(z)q(z) ，比如正态分布，即使用一个行为策略去模拟目标策略。

原来的期望可变为：
