Markov Decision Process

  • 当前的状态代表了环境

  • Defined by (S,A,R,P,γ)(S, A, R, P, \gamma)(S,A,R,P,γ)

    • S：状态集合
    • A：动作集合
    • R：对(state, action)集合，定义的 agent 采取某个动作后的即时奖励。
    • P：根据(state, action)集合，定义的状态转移概率。
    • γ∈[0,1]\gamma \in [0,1]γ∈[0,1]：折扣率，令往后的状态所反馈回来的 reward 乘上这个 discount 系数，这样意味着当下的 reward 比未来反馈的 reward 更重要。

  • Policy π\piπ 是一个从S到A的规则函数，根据当前 state 来产生 action。

    • 目标:找到 π∗\pi^*π∗ 最大化累计折扣回报(culmulated discounted reward)

    U(s0,s1,.)=∑t≥0γtrtU(s_0, s_1,. )=\sum_{t \ge 0} \gamma^t r_t U(s0​,s1​,.)=t≥0∑​γtrt​

    U 代表所有状态累计的 reward 之和，引入了 discount，可以看到我们把一个无限长度的问题转换成了一个拥有最大值上限的问题。强化学习的目的是最大化长期未来奖励，即寻找最大的 U。

价值函数

价值函数用来衡量某一状态或动作-状态的优劣，即对智能体来说是否值得选择某一状态或在某一状态下执行某一动作。

Value Function 状态价值函数

意义为基于 t 时刻的状态 s0s_0s0​ 能获得的未来回报（return）的期望，所以初始状态为s的，时刻t的打分为：

Vπ(s)=E[∑t≥0γtrt  ∣  s0=s,π]V^\pi (s) = E \left[\sum_{t\geq0}\gamma^tr_t\;\vert\;s_0=s,\pi\right] Vπ(s)=E[t≥0∑​γtrt​∣s0​=s,π]

deep Q-learning

Q-learning的目的是使用 Q(s,a;θ)Q(s,a;\theta)Q(s,a;θ) 近似动作价值函数，去逼近最优选择 Q∗(s,a)Q^*(s,a)Q∗(s,a) 。
而当使用神经网络去近似时， θ\thetaθ 则为权重项/网络参数。

问题

从 连续样本中学习(batches of consecutivee samples) 存在着问题：

  1. 样本高度相关，从而学习效率差
  2. 网络参数控决定了下一次训练样本的行为：如果当前训练的最佳动作是向左移动，则训练样本将被左撇子的样本控制，甚至导致负反馈）

使用 经验回放(experiency replay)：
从以往的状态转移（经验）中随机采样进行训练。
在训练的游戏进行中，不断更新回放记忆列表 (st,at,rt,st+1)(s_t, a_t, r_t, s_{t+1})(st​,at​,rt​,st+1​) 。
随机转换训练的minibatch。
可以增大训练效率。

这些只是数据，并不是策略。（st,at,rt,st+1)这样一组数据，其实跟策略是没有什么关系的。rt是环境返回的，可以当作是个常量，st+1 这个也是环境决定的。 我们的目标学习是（st,at）状态对的 Q value.
所以这个 Experience Buffer是可以一定程度上 增加数据多样性的。因为同一个策略 在st上产生的行为 总是at，而我们的Q-learning是希望能够学习更多的状态对，这样他的泛化性能也会更好。
如果我们要一个语义相似度任务，跟sentence1 和 sentence2 判断他们相似还是不相似，相似的话 输出结果为1 ，不相似的话，输出结果为2
这个时候我们一定是要数据打乱在传入神经网络。如果你尝试把相似句子为一批传进神经网络 不相似数据为一批 在传进神经网络，你会发现网络根本没法训练。网络就不断地交替学习 所有输出数据全部输出1 或者全部输出 0。
