KBQA：by CBR

Posted on 2023-04-06 In NLP

Paper：ICML 2022 - Knowledge Base Question Answering by Case-based Reasoning over Subgraphs

前身：Case-based Reasoning for Natural Language Queries over Knowledge Bases

Abstract and Introduction

reasoning patterns:

  • 给定一个 query，对当前 query 有效的 KG facts 被称为一个 reasoning pattern，比如有固定类型的多节点连通路，或者是固定类型的子图结构。
  • 通常使用一个 weakly-supervised knowledge base question answering (KBQA) 模型，不依赖数据标注，得到每个 query 的 reason pattern 。

现存的基于 reasoning patterns 工作的问题：

  • 一些方法只适用于解码出 individual relational paths ， 即只支持 one-jump reasoning pattern ，但是其实真实场景的 query 需要考虑多个 fact，即需要更复杂的 reasoning pattern。
  • 有些 reasoning pattern 对应的 query 样本很少。可以使用 Case-based Reasoning ，即首先检索出相似的问题样本，然后复用解决方法。
  • 当前许多抽取子图的方法都依赖于 hops of edges ，但是在一个巨大的 KG 中，生成的子图是很大，不利于后续的答案检索。

contributions


作者认为相似 query 有着相似的 reasoning pattern（如 Fig1 red edge），所以提出 CBR-SUBG ：

  1. 使用 a nonparametric component，从训练集中抽取出和 input query 最相近的 k 个 neighbor training queries 。
  2. 提出了一个 adaptive subgraph collection method ，为每个 query 定制一个专属的子图，做到 [ high recall and small size ]。
  3. 使用 a parametric GNN model，根据匹配到的 train queries 对应的 subgraphs 学习 (latent) reasoning patterns，然后在 iinput query 的子图上得到答案。

Retrieval of Similar Cases

使用 Boberta-base + normalized token mean pooling 得到 query representation。

因为我们考虑的是 reasoning pattern type ，所以需要一个 entity-agnostic representation ，于是在编码时使用 [mask] 将 entity span 替换掉。

使用 cosine similarity + normalized vector representations 从 D\mathcal DD 中检索出 top-k training queries kNNq\text {kNN}_qkNNq​ 。

  1. 为 {q}∪kNNq\{q\} \cup \text {kNN}_q{q}∪kNNq​ 中的每一个 query 生成一个子图，得到子图集合 Kqi\mathcal K_{qi}Kqi​ 。

Query-subgraph Selection

在 query 中出现的实体周围选择子图，理想的子图应该是包含了 reasoning pattern 和 answer。

传统的方法是考虑 edges in 2-3 hops around the query entities ， 这种策略导致生成的子图仅和 query 中的实体有关，而和查询无关。同时如果KG很大，生成的子图也会特别大。


作者这里提出一个新的无参数方法，对于每个匹配出来的 train query q′∈kNNqq' \in \text {kNN}_qq′∈kNNq​ ：

  1. 首先使用 DFS 统计内部每个实体节点到答案节点的通路（chains of edges in the graph that connect the entity to its answers）
  2. 然后单独将每个 chain 的 relation sequence 提出来
  3. 对于 input queries 中的每个 entity node，使用上面提出来的的 relation sequence 进行进行拓展（from the query），得到子图。如果该relation chain 不满足情况，则整个 relation chain 都被直接丢弃。

Reasoning over Multiple Subgraphs

一个 input query 检索到了多个 training query，也就对应了多个子图，每个子图中的答案节点都是不一样的。

使用 Gq=(Vq,Eq)\mathcal{G}_q=\left(V_q, E_q\right)Gq​=(Vq​,Eq​) 表示 query qqq 采样到的子图， v∈Vqv\in V_qv∈Vq​ 表示其中的节点。

Input node representations

因为使用预训练的稠密节点不易于拓展，即推理时无法适应新的节点，于是这里使用基于边的稀疏节点表示方法，将一个节点表示为 xv∈{0,1}∣R∣\boldsymbol{x}_v \in\{0,1\}^{|\mathcal{R}|}xv​∈{0,1}∣R∣ ，如果有出边 r ，则对应第 r 个元素为 1，即出边的 one-hot 编码。

这种初始化方式可以很容易第进行拓展，同时还可以表示节点的结构信息。

Relative distance embedding

子图中存在三种节点：

  1. query 中出现的实体
  2. answer 实体
  3. hop 路径上的其他实体

我们将第一种实体看作 ‘center’ entities ，并将额外的 relative distance embedding xd∈{0,1}∣d∣\boldsymbol{x}_d \in\{0,1\}^{|d|}xd​∈{0,1}∣d∣ 拼接到节点后，即距离 center entities 的最短距离（+1）对应的 one-hot embedding ，最短距离为 1，依次递增。

如果按照 n-hop 采子图，则 d=n+1 。
