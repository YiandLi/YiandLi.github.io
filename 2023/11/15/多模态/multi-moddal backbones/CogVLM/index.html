COGVLM: VISUAL EXPERT FOR LARGE LANGUAGE MODELS
CogVLM：深度融合引领视觉语言模型革新，多领域性能创新高
THUDM/CogVLM

deep fusion : CogVLM 通过在 Frozen Pre-trained Language Model 中的 FFN 和 Self-attention 层中插入 a trainable visual expert module ，深度融合不同模态的特征，指导语言模型的输出。

CogVLM-17B 在 10 个传统跨模态任务上取得了 SOTA 效果（ NoCaps, Flicker30k captioning, RefCOCO, RefCOCO+, RefCOCOg, Visual7W, GQA, ScienceQA, VizWiz VQA and TDIUC），并且在另外几个数据集上得到第二名的成绩（ VQAv2, OKVQA, TextVQA, COCO captioning, etc）。

INTRODUCTION

为什么采用 visual encoder + text generator 架构？

  1. 许多跨模态任务可以被建模为 next token prediction 任务，比如 image captioning (Agrawal et al., 2019), visual question answering (Antol et al., 2015), visual grounding (Yu et al., 2016) and even segmentation (Chen et al., 2022a) 。

next token prediction for image segmentation : A generalist framework for panoptic segmentation of images and videos - 2022

  1. 随着 VLM 参数量变大，模型在多个下游任务上出现了 in-context learning 的能力。

但是现在 from scratch 训练一个 Large VLM 还是很困难，所以选择在 P-LLM ，构造并且训练 VLM 。

目前最流行的做法是「浅层对齐方法 shallow alignment」，通过一个 module ( Q-Former , MLP )，将视觉特征映射到语言模型的输入嵌入空间中。
尽管这种方法收敛速度快，但性能（例如BLIP-2 NoCaps CIDEr 121.6）并不如同时训练视觉和语言模块的方法（例如PaLI-X NoCaps CIDEr 126.3）好。
同时，采用浅层对齐方法训练的聊天式VLM在视觉理解能力上表现不足，可能会出现幻觉/hallucination。

为什么不同时训练 LLM ？-- LLM 会产生灾难性遗忘问题，在 text-centered tasks 上的能力大幅度退化 。

所以 CogVLM 在 LLM 的每一层中引入可训练的 visual expert，其包含新的「QKV矩阵」和「MLP层」，用来 attend 视觉信息。
尽管模型参数量变成了原本的两倍，但是 FLOPs 保持相同。同时，由于 visual expert 是可插拔的，并且没有训练 LLM，在纯文本任务上，效果不会下降。

METHOD

Architecture

CogVLM-17B 包含：

  • LLM：Frozen Vicuna-7B-v1.5，此模型在所有的注意力操作中都应用了因果掩码(ausal mask)，包括图像特征之间的注意力。
  • ViT encoder：EVA2-CLIP-E ，负责将图像转化为特征表示。在CogVLM-17B中，移除了ViT编码器的最后一层，因为该层专注于整合 [CLS] 特征以用于对比学习。
  • MLP adapter：a two-layer SwiGLU MLP，用于将ViT的输出映射到与文本特征相同的空间。所有的图像特征在语言模型中共享相同的「位置编码id」。
  • Visual expert module：在 LLM 的每一层中引入可训练的 visual expert，其包含专门处理 image feature 的「QKV矩阵」和「MLP层」，以实现深度的视觉-语言特征对齐。
    QKV矩阵和MLP的形状与预训练语言模型中的相同，并从中进行初始化。
    trainable visual expert 专门用于转换图像特征，功能上和 LLM QKV/MLP 一致，但是只针对 image feature，从而实现模态间的深度融合。

PRETRAINING -> CogVLM-17B

数据：

  • 用了公开可用的图像文本对进行训练，为 LAION-2B 和 COYO-700M
  • 预处理部分，移除了损坏的URL、含有不适当内容的图像、带有噪声字幕的图像、具有政治偏见的图像，以及长宽比大于6或小于1/6的图像。筛选后，大约还剩下15亿个图像文本对用于预训练。
  • 经过筛选后，大约还剩下15亿个图像文本对用于预训练。
  • 作者还构建了一个大小为 40M 的图像的 visual grounding 数据集。
    在这个数据集中，image caption 中的 image noun 都与 bounding boxes 相关联，以指示其在图像中的位置。
    这些图像文本对是从 LAION-400 M的子集 LAION-115M 中抽取的，LAION-115M是由Li等人进行了筛选的。筛选后，保留了4000万张图像的子集，以确保超过75%的图像至少包含两个边界框。

训练：

  • The first stage：
    • Image captioning loss, next token prediction task
    • 1.5B image-text pairs
    • 120,000 iterations with a batch size of 8,192
  • The second stage：
    • a mixture of image captioning and Referring Expression Comprehension (REC) 。在答案的部分，只考虑了下一个标记的预测损失。
    • REC 任务是根据 text description of an object 来预测图像中的 bounding box ，比如 “Question: Where is the [object]?” and “Answer: [x0, y0, x1, y1]” 。其中，x和y坐标的取值范围从000到999，表示在图像中的归一化位置。
    • 60,000 iterations with a batch size of 1,024
    • During the final 30,000 iterations, we change the input resolution from 224 × 224 to 490 × 490

The total number of trainable parameters is 6.5B and the pretraining consumes about 4,096 A100×days

ALIGNMENT -> CogVLM-Chat

为了强化指令跟随能力，作者继续对模型在大量任务上进行微调，得到模型 CogVLM-Chat 。

监督微调数据来源：500,000 VQA pairs

  • LLaVA-Instruct（Liu et al., 2023b）
  • LRV-Instruction（Liu et al., 2023a）
  • LLaVAR (Zhang et al. 2023)
  • 一个内部数据集

数据规模：总共包括约 500,000 个视觉问答（VQA）对。
数据质量：监督微调数据的质量至关重要，但LLaVA-Instruct数据集是通过涉及仅含有语言的GPT-4的流程生成的，因此难免会存在错误。特别地，通过手动检查和标注，已经纠正了LLaVA-Instruct数据集中的错误。

  • 进行了 8,000 个迭代
  • 使用批量大小为 640
  • 学习率为 10^-5
  • 前 50 个迭代进行学习率的预热（warm-up）

在监督微调过程中，为了避免过拟合数据集的文本答案，使用了一个较小的学习率（其他参数的学习率的10%）来更新预训练语言模型。所有参数（除了ViT编码器）在SFT期间都是可训练的。

EXPERIMENTS

三大任务：

  • Image Captioning：NoCaps、COCO、Flickr30K、TextCaps
  • Visual Question Answering：VQAv2、OKVQA、GQA、VizWiz-VQA、OCRVQA、TextVQA、ScienceQA
  • Visual Grounding：Grounded Captioning (GC)、Referring Expression Generation (REG)、Referring Expression Comprehension (REC)、Grounded Visual Question Answering (GroundedVQA)

Instruction Following in Real-World User Behavior：利用TouchStone benchmark对CogVLM-Chat模型进行了评估，展示了相对于其他公开可用的VLMs，CogVLM-Chat在生成和标准答案之间的相似性得分方面取得了显著的优势。
