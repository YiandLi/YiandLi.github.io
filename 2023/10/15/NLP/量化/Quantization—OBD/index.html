OBD (Optimal Brain Damage)

目的是删除一些参数，也就是参数设置为 0 ，来裁剪掉影响较小的参数以降低模型复杂度来提高泛化能力 。

那么问题就转化为如何找到「被删除参数」。作者通过海森矩阵，即参数的二阶导数度量「被删除参数」对「模型结果」的影响，然后选择影响较小的参数进行裁减。

理论

对于全体参数 uuu ，可以建立一个误差函数来模拟轻微扰动 $ \delta u$ 对损失函数的影响 ： δE=L(u+δu)−L(u)\delta E = L(u+\delta u)-L(u)δE=L(u+δu)−L(u) ；然后去找令 δE\delta EδE 最小的参数进行修剪。

首先使用泰勒二阶展开来近似扰动后参数的损失 :

L(u+δu)=L(u)+gTδu+12δuTHδu+O(∥δu∥3)L(u+\delta u)=L(u)+g^T \delta u+\frac{1}{2} \delta u^T H \delta u+O\left(\|\delta u\|^3\right) L(u+δu)=L(u)+gTδu+21​δuTHδu+O(∥δu∥3)

gig_igi​ 表示 Loss 对参数 uuu 的一阶导数 ， gi=σLuig_i = \dfrac{\sigma L}{u_i}gi​=ui​σL​ ；
δu\delta uδu 表示参数的扰动 ；
HHH 表示黑塞矩阵元素，对应二阶导数 ；

进而损失函数的扰动 δE\delta EδE 被表示为 :

δE=L(u+δu)−L(u)=gTδu+12δuTHδu+O(∥δU∥3)\delta E=L(u+\delta u)-L(u)=g^T \delta u + \frac{1}{2} \delta u^T H \delta u + O\left(\|\delta U\|^3\right) δE=L(u+δu)−L(u)=gTδu+21​δuTHδu+O(∥δU∥3)

δui\delta u_iδui​ 也可以理解为 −ui-u_i−ui​ ，即通过泰勒使用当前已知参数去模拟修剪后损失的变化量。

可以将 12δuTHδu\frac{1}{2} \delta u^T H \delta u21​δuTHδu 拆开，公式变为 :

δE=∑igiδui+12∑ihiiδui2+12∑i!=jhijδuiδuj+O(∥δU∥3)\delta E=\sum_i g_i \delta u_i+\dfrac{1}{2} \sum_i h_{i i} \delta u_i^2+\dfrac{1}{2} \sum_{i !=j} h_{i j} \delta u_i \delta u_j+O\left(\|\delta U\|^3\right) δE=i∑​gi​δui​+21​i∑​hii​δui2​+21​i!=j∑​hij​δui​δuj​+O(∥δU∥3)

这里提出假设：

  1. 多个参数引起的 δE\delta EδE 是由删除每个参数单独引起的损失偏差导致的，和交叉项无关，于是省略了 12∑i!=jhijδuiδuj\dfrac{1}{2} \sum_{i !=j} h_{i j} \delta u_i \delta u_j21​∑i!=j​hij​δui​δuj​ 。
    由于这个假设造成OBD算法并没有利用黑塞矩阵带来全局二阶信息，也就是没有考虑参数的相互影响。
  2. 在神经网络参数熟练后，每个参数的一阶导数 gig_igi​ 应该为 0 ， 所以 ∑igiδui\sum_i g_i \delta u_i∑i​gi​δui​ 可以省略 。
  3. 二次假定：忽略公式中的3次误差。
    the “quadratic” approximation assumes that the cost fundion is nearly quadratic so that the last term in the equation can be neglected.

最终得到：

δE=12∑ihiiδui2=12∑ihiiui2\delta E=\frac{1}{2} \sum_i h_{i i} \delta u_i^2 = \frac{1}{2} \sum_i h_{i i} u_i^2 δE=21​i∑​hii​δui2​=21​i∑​hii​ui2​

也就可以使用参数二阶导来度量参数的重要性。

流程

  1. 搭建神经网络
  2. 训练神经网络至损失函数收敛
  3. 计算神经网络每个参数的二阶导数 hiih_{i i}hii​
  4. 计算神经网络每个参数的「显著性 saliency」: si=12hiiui2s_i=\frac{1}{2} h_{i i} u_i^2si​=21​hii​ui2​
  5. 按照显著性对参数进行排序，并删除一些低「显著性的参数 low-saliency parameters」。
    可认为删除参数是将其设置为0并训练时冻结。
  6. 从 步骤2 开始重复

高效计算 hiih_{ii}hii​ : https://blog.csdn.net/epic_Lin/article/details/120735721

特点

  • 需要重复进行训练
  • 修正了原来直接使用参数本身值的大小作为参数显著性的评价方式，使用了二阶导作为评价标准。将一个参数的显著性定义为删除该参数引起的目标函数的变化。

OBS (Optimal Brain Surgeon)

同 OBD 一样用于神经网络模型剪枝，他们对参数变换带来的代价函数扰动都采用泰勒二阶展开近似，并忽略3次项和一阶梯度。
不同点在对海森矩阵的处理：OBD 粗暴的只考虑海森矩阵对角线元素；而 OBS 考虑海森矩阵全局信息，由此也获得参数相互之间的影响。

原理

回到上面 OBD 的推导：

ΔE=gTΔw+12ΔwTHΔw+O(∥Δw∥3)\Delta \mathbf E=g^T \Delta \mathbf{w} + \frac{1}{2} \Delta \mathbf{w}^T \mathbf H \Delta \mathbf{w} + O\left(\|\Delta \mathbf{w}\|^3\right) ΔE=gTΔw+21​ΔwTHΔw+O(∥Δw∥3)

同样忽略了一阶导和三次项：

ΔE=12ΔwTHΔw\Delta \mathbf E=\frac{1}{2} \Delta \mathbf{w}^{\mathrm{T}} \mathbf{H} \Delta \mathbf{w} ΔE=21​ΔwTHΔw

我们的目标是，裁剪某个参数 wq\mathbf{w}_qwq​ ，能够使得 $\Delta E $ 最小 。

删除某个参数 wq\mathbf{w}_qwq​ ， 也就是 Δw=−wq\Delta w=-\mathbf w_qΔw=−wq​ ：

eqT⋅Δw+wq=0\mathbf{e}_{\mathbf{q}}^{\mathbf{T}} \cdot \Delta \mathbf{w}+\mathbf w_q=0 eqT​⋅Δw+wq​=0

eqT\mathbf{e}_{\mathbf{q}}^{\mathbf{T}}eqT​ 是一个和 w\mathbf ww 同纬度的 one-hot 矩阵，只有 q 上为 1 。

我们希望找到最合适的权重 wqw_qwq​, 使得删除它对目标的影响最小。这可以表示为一个最优化问题:

min⁡Δw,q12ΔwTHΔw s.t. eqT⋅Δw+wq=0\min _{\Delta \mathbf{w}, q} \frac{1}{2} \Delta \mathbf{w}^{\mathbf{T}} \mathbf{H} \boldsymbol{\Delta} \mathbf{w} \quad \text { s.t. } \quad \mathbf{e}_{\mathbf{q}}^{\mathbf{T}} \cdot \boldsymbol{\Delta} \mathbf{w}+w_q=0 Δw,qmin​21​ΔwTHΔw s.t. eqT​⋅Δw+wq​=0

用 Lagrange 乘数法求解:

L=12ΔwTHΔw+λ(eqT⋅Δw+wq)L=\frac{1}{2} \boldsymbol{\Delta} \mathbf{w}^{\mathbf{T}} \mathbf{H} \boldsymbol{\Delta} \mathbf{w}+\lambda\left(\mathbf{e}_{\mathbf{q}}^{\mathbf{T}} \cdot \boldsymbol{\Delta} \mathbf{w}+w_q\right) L=21​ΔwTHΔw+λ(eqT​⋅Δw+wq​)

直接求解得到另损失 LLL 最小的 Δw\boldsymbol{\Delta} \mathbf{w}Δw 和最小损失 LLL 。

Δw=−wq[H−1]qqH−1⋅eq and L=12wq2[H−1]qq\Delta \mathbf{w}=-\frac{w_q}{\left[\mathbf{H}^{-1}\right]_{q q}} \mathbf{H}^{-1} \cdot \mathbf{e}_{\mathbf{q}} \quad \text { and } \quad L=\frac{1}{2} \frac{w_q^2}{\left[\mathbf{H}^{-1}\right]_{q q}} Δw=−[H−1]qq​wq​​H−1⋅eq​ and L=21​[H−1]qq​wq2​​

注意在上述计算后，我们是将 Δw\Delta \mathbf{w}Δw 的第 q 维进行了固定，但其他维度的值可变，可以用于减少删除该权重带来的目标偏离。
所以计算结果中 Δw\Delta \mathbf{w}Δw 的其他维度是发生了改变的，也就对应了下面的 「step 3」。

详细推理见 ： GPTQ 模型量化

于是迭代进行：

  1. 对每一个参数求最小损失 $ L=\dfrac{1}{2} \dfrac{w_q2}{\left[\mathbf{H}{-1}\right]_{q q}}$ ，作为「显著性 saliency」。
  2. 按照影响从小到大排序，确定参数剪枝的次序，每次剪枝一个参数。
  3. 其他的参数也按照 Δw\Delta \mathbf{w}Δw 更新一次 (w+=Δw\mathbf{w}+=\Delta \mathbf{w}w+=Δw) 。

也就是对某个 block 内的所有参数逐个量化，每个参数量化后，需要适当调整这个 block 内其他未量化的参数，以弥补量化造成的精度损失。

OBC (OPTIMAL BRAIN COMPRESSION)

OBD 和 OBS 都存在一个缺点，就是剪枝需要计算全参数的海森矩阵（或者它的逆矩阵），如下：

但在动辄上亿参数的神经网络下求解海森矩阵显然不可能。

于是，我们可以假设参数矩阵的「同一行参数」互相之间是相关的，而「不同行之间的参数」互不相关，这样，海森矩阵就只需要在「每一行内」单独计算就行啦，如下：

对于每一行 (row) ，「原始参数和输入的点乘结果」 Wi\mathbf{W}_{\mathbf{i}}Wi​ 和「量化后参数和输入的点乘结果」之间的差距要尽可能小 ：

则每一行 i ，在上式中都是一个标准的最小二乘优化问题，从最小二乘的角度上，海森矩阵为 2XXT2\mathbf{X}\mathbf{X}^T2XXT 。

OBQ

OBQ （和OBC是同一篇文章）指出，剪枝是一种特殊的量化（即剪枝的参数等价于量化到 0 点），我们只需要修改一下 OBC 的约束条件即可：

eqT⋅Δw+wq=quant⁡(wq)\mathbf{e}_{\mathbf{q}}^{\mathbf{T}} \cdot \boldsymbol{\Delta} \mathbf{w}+w_q=\operatorname{quant}\left(w_q\right) eqT​⋅Δw+wq​=quant(wq​)

quant⁡(wq)\operatorname{quant}\left(w_q\right)quant(wq​) 是对参数 wqw_qwq​ 的量化结果 。

也就是：

eqT⋅Δw+(wq−quant⁡(wq))=0\mathbf{e}_{\mathbf{q}}^{\mathbf{T}} \cdot \boldsymbol{\Delta} \mathbf{w} + \left( w_q - \operatorname{quant}\left(w_q\right)\right) = 0 eqT​⋅Δw+(wq​−quant(wq​))=0

于是根据 OBC， 得到：

Δw=−wq−quant⁡(wq)[H−1]qqH−1⋅eq and L=12(wq−quant(wq))2[H−1]qq\Delta \mathbf{w}=-\frac{w_q-\operatorname{quant}\left(w_q\right)}{\left[\mathbf{H}^{-1}\right]_{q q}} \mathbf{H}^{-1} \cdot \mathbf{e}_{\mathbf{q}} \quad \text { and } \quad L=\frac{1}{2} \frac{\left(w_q-\text{quant}\left(w_q\right)\right)^2}{\left[\mathbf{H}^{-1}\right]_{q q}} Δw=−[H−1]qq​wq​−quant(wq​)​H−1⋅eq​ and L=21​[H−1]qq​(wq​−quant(wq​))2​

过程：

Using OBQ, we can quantize the easiest weight first and then adjust all remaining non-quantized weights to compensate for this precision loss. Then we pick the next weight to quantize, and so on.

在更新完未进行量化的权重后 w←...\mathbf w \leftarrow ...w←... ，为了提高性能，这里直接对海森矩阵的逆进行更新 H−1←...\mathbf H^{-1} \leftarrow ...H−1←... 。
by removing the row and column associated with that weight (using Gaussian elimination)

GPT-Q

OBQ 的算法复杂度还是太高了，GPTQ 对 OBQ 做了一些算法和性能上的优化，因而可以实现大模型的高效量化。

创新点：

  • OBS 采用贪心策略，先量化对目标影响最小的参数；但 GPTQ 发现直接按顺序做参数量化，对精度影响也不大，具体实现方法为每次同时对 W:,jW_{:,j}W:,j​ ，即所有行的第 j 列进行量化 ( aims to quantize all weights in the same order for all rows of a matrix )。
    -> 使得参数矩阵每一行的量化可以做并行的矩阵计算 ，即同一行上的每一列的元素并行进行量化，将时间复杂度降低到 O(max⁡{drow∗dcol2,dcol3})O(\max\{ d_{row} * d_{col}^2, d_{col}^3 \})O(max{drow​∗dcol2​,dcol3​})
  • Lazy Batch-Updates，延迟一部分参数的更新，它能够缓解 bandwidth 的压力，避免频繁访问内存。
  • Cholesky Reformulation：
    When the algorithm scales up to very large models, numerical inaccuracies can become a problem. Specifically, repeated applications of a certain operation can accumulate numerical errors.
    用 Cholesky 分解求海森矩阵的逆，在增强数值稳定性的同时，不再需要对海森矩阵做更新计算，进一步减少了计算量。

Lazy Batch-Updates

虽然 GPTQ 降低了时间复杂度，但这个算法的计算/通信比太低，通信带宽成为了瓶颈。

在OBQ 中，每一个权重进行量化后，其他所有「未量化的参数」都要按公式全都要更新一遍：

w←w−H:,p−11[H−1]pp⋅(wp−q(wp))\mathbf{w} \leftarrow \mathbf{w}-\mathbf{H}_{:, p}^{-1} \frac{1}{\left[\mathbf{H}^{-1}\right]_{p p}} \cdot\left(w_p-q\left(w_p\right)\right) w←w−H:,p−1​[H−1]pp​1​⋅(wp​−q(wp​))

那么每次更新过程就需要 read + write 一次参数矩阵。
如果参数矩阵的维度为 k×kk \times kk×k ，那么量化这个参数矩阵就需要读 kkk 次参数（量化的参数的数量，近似 k），总共的 IO 量为 k3k^3k3 个元素。当 kkk 比较大时（>= 4096），需要读写的元素就非常多了，运行时间大都被 IO 占据。

思路：由于参数量化是一列一列按次序进行的，在量化第 i 列之前，传统上需要对这一列更新 i-1 次，这里我们将这些更新量都缓存起来，仅仅在量化第 i 列前才更新一次。

具体实现：将参数矩阵按每 128 列划分为一个个 group，量化某一列时，group 内的参数立即更新，而 group 后面的列只记录更新量，延迟更新。当一个 group 的参数全部量化完成，再统一对后面的所有参数做一次更新。这就是 Lazy Batch-Updates。

Lazy Batch-Updates 不减少实际的计算量，但它能有效解决吞吐的瓶颈问题。
