Baichuan-7B / 13B

结构：

采用了和LLaMA一样的模型设计：

RoPE： 虽然训练过程中最大长度为4096，但是实际测试中模型可以很好的扩展到 5000 tokens 以上。

Pre RMSNorm Layer Norm

SwiGLU FFN：缩小了隐藏层维度，由原来的4倍变成8/3倍，再调整为128的整数。

数据：

开源的中英文数据和自行抓取的中文互联网数据，以及部分高质量知识性数据。

  • 经过不断的调整和多轮测试，最终确认了一个在下游任务上表现最好的中英文配比。
  • 我们使用了一个基于自动学习的数据权重策略，对不同类别的数据进行配比。

分词：BPE

  1. 使用 2000 万条以中英为主的多语言语料训练分词模型，增加 chinese token，显著提升对于中文的压缩率。
  2. 对于数学领域，我们参考了 LLaMA 和 Galactica 中的方案，对数字的每一位单独分开，避免出现数字不一致的问题，对于提升数学能力有重要帮助。
  3. 罕见字词（如特殊符号等），支持 UTF-8 characters 的 byte 编码，因此做到未知字词的全覆盖。

BaiChuan2

https://github.com/baichuan-inc/Baichuan2

数据清洗：

based on clustering and deduplication

借助 LSH-like 和 Embedding 特征对数据进行聚类和去重，主要是对每个聚类的簇给文档、段落、句子进行去重和打分，分值用于用于 pretrain 时的采样。

数据质量：句子级别质量过滤，但未说明明确过滤规则。

Tokenizer

两个因素：压缩率，有助于推理快速 + 词表大小，保证token能够被充分训练到

利用 SentencePiece 中 BPE 方法构建 Tokenizer，从 64_000 扩张到了 125_696，在 BPE 时不对 text 进行 normalization ，同时也不添加 dummy prefix 。

模型部分：

7B 使用 RoPE ； 13B 使用 ALiBi

对 block 的输入进行采用 Layer Norm （pre-norm）：提高warm-up的鲁棒性

NormHead：训练时，对 output embedding 进行归一化，主要解决：（1）稀有标记的embedding在训练过程中变小，干扰了训练的动态；（2）分析发现在输出空间中， token 的语义信息主要受到了余弦相似度的影响，对L2距离不明显，归一化可以减少 Linear Head 通过点积计算logits时，L2距离的影响。

Max-z loss：LLM 输出的 logits 可能会变的非常大，虽然不会对训练过程产生影响（ softmax 操作），但是推理过程中会产生问题：repetition 惩罚会对 logistic 增加一个 scalar（1.1 / 1.2），从而让模型十分敏感。 受 Norm-SoftMax 启发，增加 max-z loss：

Lmax-z =2e−4∗z2\mathcal{L}_{\text {max-z }}=2 e^{-4} * z^2Lmax-z ​=2e−4∗z2 ， zzz 为 the maximum logit value ，起到正则的作用

从而对logits进行归约。主要有助于稳定训练并使推理对超参数更具鲁棒性。

Attention 使用 xFormers 的 memory efficient attention

Chat Model

通过 对齐 ，得到两个版本的 chat 模型

对齐分为两个部分：Supervised Fine-Tuning (SFT) && Reinforcement Learning from Human Feedback (RLHF)

SFT

严格对SFT数据进行把关，最终收集100k数据进行模型有监督微调。

PS：100k数据个人感觉太少了，仅对Pretrain模型进行100k的指令微调真的够吗？其实可以构造更多为什么没有使用？个人分析，此处SFT的数据是真的巨高质量的数据，其他很多SFT数据，可以利用MIP的方式，在Pretrain阶段就使用了。

RM

设计了一个三层分类系统全面覆盖所有类型的用户需求，包括6个主要类别、30个二级类别、200多个三级类别。

在奖励模型训练时，需要保证每个类别内的数据应该有足够的多样性，以确保奖励模型能够有更好地泛化性。并且奖励数据中结果需要由Baichuan2模型生成，以确保数据分布的统一。
