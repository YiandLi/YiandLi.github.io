Create

Operation / Domain Specific Language (DSL) : Context and Expression

Two main operations : context – verbs && expression – nouns

  1. Contexts
    • Definition : determine the specific environment in which an expression is evaluated and executed.
    • Three main contexts:
      • Selection :
        Apply expression over columns , the expression produce Series of the same length or of a length of 1 .
        The 1-length result will be broadcasted to match the height of the DataFrame .
        select drops the original columns ; with_columns retains the original columns .
      • Filtering : filter the existing dataframe based on expression that evaluates to Boolean data type
      • Groupby/aggregation
  2. Expressions :
    • Definition : a map from a series too a series
      computations or transformations performed on data columns , allowing to apply operations to derive new column results.
    • Including : mathematical operations, aggregations, comparisons, string manipulations, etc.
    • Features :
      • Can pipe different expression togethers
      • Doesn’t always run in the specified order. Instead, Polars optimizes the order in which expressions are evaluated in a query.
      • Run in parallel: the work is spread across available cores.

For instance, buildings.select(pl.col("sqft")) : pl.col("sqft") is the expression that passed to the buildings.select() context .

More:

  • select : buildings.select(pl.col("sqft").sort() / 1000)
  • filter + select : buildings.filter(pl.col("year") > 2015).select(pl.col("year").min()): put select() after filter()
  • groupby + agg :

Lazy API

Lazy API : Evaluate sophisticated expressions on large datasets while keeping memory efficiency in mind.

  • The object is LazyFrame
  • Allow to specify a sequence of operations without immediately running them.
  • Operations are saved as a computation graph and run when necessary，allowing Polars optimize queries before execution and catch errors, and perform memory-efficient queries on datasets that don’t fit into memory.

  • create a LazyFrame from buildings : pl.LazyFrame()
  • convert an existing DataFrame to a LazyFrame : .lazy()
  • scan a file, rather than reading the entire file into memory, create a LazyFrame to refer the data

Select

  • select a column : df.select( pl.col( 'column_name' ))
  • select all columns : df.select(pl.col("*")) / df.select(pl.all())
  • select all columns except some : df.select(pl.col("*").exclude("logged_at", "rn"))
  • select some columns : df.select(pl.col("date", "sales"))
  • specify multiple strings : df.select(pl.col("date", "logged_at").dt.to_string("%Y-%h-%d")) , noticed that the original format is suitable.
  • By regular expressions : df.select(pl.col("^.*(as|sa).*$")) 正则表达式
  • By data type : pl.col( pl.Int64, pl.Boolean )
  • Get Unique : pl.col( 'col_num' ).unique()
  • Count unique : pl.col( df.n_unique() ) global table unique ; df.select(pl.col('logged_at', 'date').n_unique()) unique per each columns
  • Selector : select based on columns name, dtype, or other properties. -> import polars.selectors as cs
    • by dtype: df.select( cs.integer(), cs.string() )
    • except the 1st matched column: df.select(cs.numeric() - cs.first())
    • By patterns and substrings: df.select( cs.contains('rn), cs.matches(".*_.*") )

Casting

Goal: convert the DataType of a column to a new one .

cast() function includes a parameter strict=True , when met a value that can not be converted , it will throw an exception ;
otherwise, strick=False , it will convert that value to null .

Rolling

Specific for a Series : return a Series as well , and of the same size / length

The window at a given row will include the row itself and the window_size - 1 elements before it.

  • Series.rolling_max :
    • window_size : int
    • weights=None : list[float]
    • min_periods=None : int: The number of values in the window that should be non-null before computing a result. If None, it will be set equal to window size.
    • center=False : Set the labels at the center of the window
  • Series.rolling_mean
  • Series.rolling_min
  • Series.rolling_median
  • Series.rolling_sum
  • Series.rolling_quantile
  • Series.rolling_apply / Series.rolling_map : bad performance , not recommend

Missing data

  • Difference between np.NaN and None :
    • NaN values are considered to be a type of floating point data and are not considered to be missing data in Polars.
    • NaN values are not counted with the null_count method
      NaN values are filled when you use fill_nan method but are not filled with the fill_null method
    • Calculate the mean / sum of a list , None will be excluded , but NaN will result in a NaN.
  • count : df.null_count()
  • check if it is null : df.select( pl.col("value").is_null() )
  • fill_null function :
    • value: Any | None = None : using a specific value pl.lit():
    • strategy : {None, ‘forward’, ‘backward’, ‘min’, ‘max’, ‘mean’, ‘zero’, ‘one’}
      replace the nulls with the previous non-null value: strategy="forward":
    • with an expression , like use the medium value of that column

Window Function

Goal : group over different columns and perform an aggregation on them . Doing so allows us to use multiple group by operations in parallel in a single select call . A window function will almost always lead to a DataFrame with the same size as the original.

Difference:
groupby + agg : marks that groups are aggregated and we expect a DataFrame of size n_groups.
over : marks computation within a group, and doesn’t modify the original size of the DataFrame.

  • Get statistic value over different group
    使用 .over() 函数 ，在一个 select context 中计算多个 group 的统计信息。

  • Sort within different group

  • 3 strategies to link the values back to df:

    • mapping_strategy = 'group_to_rows' : each value is assigned back to one row.
    • mapping_strategy = 'join' : the values are imploded in a list, and the list is repeated on all rows. This can be memory intensive.
    • mapping_strategy = 'explode' : the values are exploded to new rows. This operation changes the number of rows.
      通过 head 保留了不同 sort 条件的结果，最后爆破 。

Time Series

datetime datatypes:

  1. Date: e.g. 2014-07-08. ; a 32-bit signed integer.
  2. Datetime: e.g. 2014-07-08 07:00:00 ; a 64 bit integer
  3. Duration: A time delta type that is created when subtracting Date/Datetime
  4. Time: Time representation, internally represented as nanoseconds since midnight.

Parse:

Grouping

Grouping by fixed windows
Group rows based into [ days / months / years ] . -> .group_by_dynamic()
In nature, it set a window based on the parameter and do aggregation based on the window.

e.g. Annual average example:

  1. use .group_by_dynamic() func to group Data column on an 1y basis .
  2. take the mean() values of the Close column by agg() func .

Parameter for .group_by_dynamic() :

  • every : set group level ; how often the group starts ; e.g. 2y, 1y6mo …
  • period : the duration of the window ; how long the time period for each group ;
    • e.g. every='1y', period='2y': get groups at one year intervals where each groups spans two years : [2021, 2023], [2022, 2024], ... .
    • If not set, by default period equals to every .
  • offset : be used to offset the start of the windows
  • truncate :
    • truncate=True ; by default , the date for the first year in the annual average is 1st January 1981 .
    • truncate=False : the date for the first year in the annual average is the date of the first data point .
  • closed
