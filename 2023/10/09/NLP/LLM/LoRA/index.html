LoRA

优点：

  1. 一个中心模型服务多个下游任务，节省参数存储量
  2. 推理阶段不引入额外计算量，推理延迟
  3. 和其他高效微调方法正交，可以组合

缺点：

  1. 参与训练的模型参数量不多，也就百万到千万级别的参数量，所以效果比全量微调差很多。
    如果有足够计算资源以及有10k以上数据，建议全参数微调。
  2. 训练时间更久

所以到底为什么能加快训练？

和原本全参数预训练相比，前向传播和后向传播在显存中没有任何节省，只能节省 优化器状态部分 。被冻结住的参数不用更新，自然也就不需要相应的Adam一阶和二阶动量。

因为被冻结住的主干模型参数不用存储相应的优化器状态，显存占用降低，可以用更多显存装输入数据和hidden states，相同硬件下batch size可以开大；于是能导致训练速度更快 。

Aghajanyan 的研究表明：预训练模型拥有极小的内在维度(instrisic dimension)，即存在一个极低维度的参数(low instrisic content)，微调它和在全参数空间中微调能起到相同的效果。

同时发现在预训练后，越大的模型有越小的内在维度，这也解释了为何大模型都拥有很好的few-shot能力。

初始化

B 全 0，A 高斯初始化。

  1. 如果 B，A 全都初始化为0，那么缺点与深度网络全 0 初始化一样，很容易导致梯度消失（因为此时初始所有神经元的功能都是等价的）。
  • 无法破坏对称性，每次反传从后往前打破一层零参的更新（但更新值始终对称）。最终网络表现为多个隐藏层神经元就相当于一个神经元。
  • 如果是 relu 或者 tanh，则甚至会导致无法更新参数。
  1. 如果 B，A 全部高斯初始化，那么在网络训练刚开始就会有概率为得到一个过大的偏移值A W从而引入太多噪声，导致难以收敛。

因此，一部分初始为 0 ，一部分正常初始化是为了在训练开始时维持网络的原有输出（初始偏移为0），但同时也保证在真正开始学习后能够更好的收敛。

超参数

前向传播过程为： $$h= W x+ \dfrac \alpha \gamma B A x$$

超参数 γ\gammaγ 为秩 ， α\alphaα 起到缩放的作用 ；在我们采用Adam做优化器时，调整 ααα 的作用就相当于调整 learning rate 。

如何理解？WWW 表示 checkpoint 中的预训练权重，是在使用 αγBA\dfrac \alpha \gamma B Aγα​BA 近似 ΔW\Delta WΔW ，即新学到的知识 。

当选择的秩 γ\gammaγ 比较小，此时提取的是信息含量最丰富的维度，此时信息精炼，但不全面；如果秩 γ\gammaγ 比较大，此时信息更加全面，但带来的噪声也越多（含有很多冗余无效的信息）。

基于这个猜想，当我们第一次做实验时，我们会尽量把 rrr 调得大些，例如 32、64 ，并假设在这个秩下，低秩权重已经非常近似 ΔWΔWΔW 了，

因此这时我们设置 α=rα=rα=r ，意味着我们假定LoRA低秩微调的效果和全参数微调持平。

那么接下来，我们肯定就要往小的 rrr 进行尝试了。这时我们把 ααα 固定住，意味着随着 rrr 的减小，α/rα/rα/r 会越来越大，我们这样做的原因是：

秩 γ\gammaγ 变小，对应的权重 α/rα/rα/r 变大

推理时：低秩矩阵信息精炼，但不全面。通过调大 α/rα/rα/r，放大新知识对模型的影响。

训练时：低秩矩阵的噪声少，梯度下降的方向也更加确信，通过调大 α/rα/rα/r，增加梯度下降的步伐，也就相当于调整learning rate了。

参数增量 ΔWΔWΔW 和 WWW 的关系？

在训练过程中，低秩的适应矩阵 ΔWΔWΔW 仅仅放大了对下游任务有用的特征，而不是预训练模型中的主要特征。

优点是轻量化，低资源。但缺点很明显，参与训练的模型参数量不多，也就百万到千万级别的参数量，所以效果比全量微调差很多。

推荐Q-LoRA技术，这是一种量化的lora技术。核心在于，将base model的权重量化到4-bit，降低显存占用。在所有linear后面都插入adapter，并且提高adapter中的rank，以更多的训练参数量，弥补量化带来的精度损失。

实验结论

  1. 优化器的选择不是影响结果的主要因素。无论是 AdamW、具有 scheduler 的 SGD ，还是具有 scheduler 的 AdamW，对结果的影响都微乎其微。

  2. 虽然 Adam 经常被认为是需要大量内存的优化器，因为它为每个模型参数引入了两个新参数，但这并不会显著影响 LLM 的峰值内存需求。这是因为大部分内存将被分配用于大型矩阵的乘法，而不是用来保留额外的参数。

  3. 对于静态数据集，像多轮训练中多次迭代可能效果不佳。这通常会导致过拟和，使训练结果恶化。

  4. 如果要结合 LoRA，确保它在所有层上应用，而不仅仅是 Key 和 Value 矩阵中，这样才能最大限度地提升模型的性能。

  5. 调整 LoRA rank 和选择合适的 α 值至关重要。提供一个小技巧，试试把 α 值设置成 rank 值的两倍。

     scaling = alpha /r weight +=( lora _B⋅ lora_A)* scaling \begin{aligned}& \text { scaling }=\text { alpha } / \mathrm{r} \\& \text { weight }+=(\text { lora \_B} \cdot \text { lora\_A)* scaling }\end{aligned}​ scaling = alpha /r weight +=( lora _B⋅ lora_A)* scaling ​

    alpha 的值越大，新增的权重矩阵影响就越大，也就可以看作训练时的学习率。

  6. 一般过大的 r 会造成过拟合 ； 如果模型存在过拟合问题，首先要考虑降低 r 值或增加数据集大小。此外，可以尝试增加 AdamW 或 SGD 优化器的权重衰减率，或者增加 LoRA 层的 dropout 值。

  7. QLoRA 可以作为 LoRA 训练的替代方案，更进一步解决常见 GPU 内存瓶颈问题。

  8. Adam VS SGD ：相比于 Adam，SGD 在峰值内存方面节省内存的作用微乎其微。

    如果 r=8，在 7B 的 Llama 2 模型的所有 6,738,415,616 个参数，只有 4,194,304 个可训练的 LoRA 参数。

    只看数字，4,194,304 个参数可能还是很多，但是其实这么多参数仅占用 4,194,304 × 2 × 16 位 = 134.22 兆位 = 16.78 兆字节。

    2 代表 Adam 存储的额外参数的数量，而 16 位指的是模型权重的默认精度。

  9. 多轮迭代可能不利于指令微调。

QLoRA

以运行时间增长 39% 的代价，节省了 33% 的内存。

https://arxiv.org/pdf/2305.14314.pdf

模型本身用 4bit 加载，训练时把数值反量化到 bf16 后进行训练，利用LoRA可以 锁定原模型参数 不参与训练，只训练少量LoRA参数的特性使得训练所需的显存大大减少。

在前向传播时，运行到哪一层，该层的模型才会被反量化，所以训练期间不会把整个模型反量化，所以显存占用量没有那么大。

训练过程原模型的参数是4bit的，然后转化为 fp16，其实相当于「 使用一个4bit索引列来代表之前的16bit 」，输出的loss是bf16的。

这种全局量化方式存在一个问题，即当输入中存在极大值或者离群值时，一些较小的参数无法被精确的表示，因此量化后的神经网络效果会下降很多。

为了缓解这个问题，作者采用了分块量化，即将输入划分为多个block，每个block分别量化，来降低量化误差。

本文中block的大小被设为 64。

4-bit NormalFloat(NF4) Quantile Quantization

作者提出的4-bit NormalFloat量化是对Quantile Quantization(分位量化)进行了改进，并结合上诉Block-wise Quantization，降低计算复杂度和误差。

  • 4-bit Quantization： 映射后的参数就只能从 24:(0∼15)2^4: (0\sim15)24:(0∼15) 中选一个

  • Quantile Quantization:

    上述传统 4-bit Quantization 使用到缩放因子和round函数计算量化后的值，仅仅使用到了 absmax⁡(XFP32){\operatorname{absmax}\left(X^{F P 32}\right)}absmax(XFP32) 来确定最终量化后的区间。

    尽管使用到了 Block Quantization 方法，可以缓解全局最大最小值带来的误差。但是这样做的话其实没有充分利用我们有的数位，很有可能大部分原始数值都被量化到同一个4bit数上了，原本有的差异或者说信息在这个量化过程中就丢掉了。

    个人认为：量化后的分布和原始分布应该是近似的（为什么差距大呢？），那么可能存在情况：量化后为 7/8的元素数量远远大于为 0/15 的数量。于是就区分不开这里面的值了。

    所以作者采用了分位量化(Quantile Quantization)，那么什么是分位量化呢？类似分箱操作，还是以量化到4-bit为例，一共有16个数字可以选，那么可以先将输入参数从小到大进行排序再等分为16份，每一份映射一个值。

    这样做原始数据在量化后的数字上分布就是均匀的。

特殊值 0 的处理

缺点：

参数矩阵中 0 是一个特殊值（padding/dropout…），我们希望高精度的0被映射到低精度的0，这样反量化时可以直接得到0，保留了这个特殊意义。
由此特殊 0 占据一位（这里没有箱体的概念，而是直接映射，但是占了4bit的一位），剩 15 个箱体，负数部分 [1−offset,0.5][1-\text{offset}, 0.5][1−offset,0.5] 映射其中7个箱体，正数部分 [0.5,offset][0.5, \text{offset}][0.5,offset] 映射8个箱体。

具体做法为：

  1. 根据理论，估计 N(0,1)N(0,1)N(0,1) 中的 2k+12^k + 12k+1 个分位数，对应得到了 2k2^k2k 个分位区间。将这些分位数归一到 [−1,1][-1, 1][−1,1] 的范围中。

    实际代码中，首先正数部分是在 [0.5, 0.9677] 中均匀取 9 个插值表示累计概率值，
    因为概率累计值0.5正好是0，所以这里将其舍弃，得到 8 个插值，然后反转得到随机变量值 x。
    负数部分是在 [-0.9677, -0.5] 中均匀取 8 个插值表示累计概率值，同样删除 0.5 然后反函数得到随机变量值。

量化和反量化：

双重量化(Double Quantization)

分块量化中每个block都会额外产生一个量化常数 ccc ，代码中 quantize_constant 。

以量化32bit参数、block大小64为例，每个block会引入32bit的量化常数，对应每个参数会额外引入 32/64=0.5bit 的额外开销。

对于4bit量化来说额外的0.5bit相当于多12.5%的显存耗用，还是比较可观的。

因此文章中提出了双重量化，对已经通过上面计算方法量化后的 32 bit c 再进行一次量化，block size=256，对应 256*64 参数，量化为 8 bit c’ 。

改进过后每个参数的额外消耗为 8/64 + 32/(64 * 256) = 0.127 bit

分页优化器(Paged Optimizer)

Paged Optimizer在GPU显存不足的时候可以把optimizer转移到内存中，在需要更新optimizer状态时再加载回来，以此来减少GPU显存的峰值占用。

文章称想要达到在24gb上训练33B 参数模型这个机制是必须的。

Qlora有如此效果的原因有三个：

  • Qlora是先量化再微调，效果肯定比微调后再量化好
  • Qlora不仅有量化，另外作者发现lora adapter的数量越多微调效果也就越好，因此相比于常规Lora只把adapter应用在query和value上，Qlora在所有的线性层上都添加了adapter，数量上要比常规lora多很多
  • NF4要比FP4的精度损失更少

LoRA 和 SVD 分解

可以将矩阵 A∈Rm×nA \in \mathbb R^{m\times n}A∈Rm×n 分解为 A=UΣVTA= U\Sigma V^TA=UΣVT ，U∈Rm×r,Σ∈Rr×r,V∈Rr×nU \in \mathbb R ^ {m\times r} , \Sigma \in \mathbb R ^ {r\times r}, V \in \mathbb R ^ {r\times n}U∈Rm×r,Σ∈Rr×r,V∈Rr×n 。

U,VU , VU,V 是左右奇异向量 ，为正交矩阵，即 UTU=VTV=IU^T U = V^T V=IUTU=VTV=I ，Σ\SigmaΣ 为对角矩阵，对角值为 AAA 上的奇异值。

在 LoRA 中，我们是将增量权重矩阵 ΔW\Delta WΔW 分解为两个权重矩阵的乘法，但是存在局限：

  1. 每个增量矩阵的秩 rrr 必须相同，忽略了：不同层不同类型的参数重要性不同，需要不同的秩
  2. 没有办法裁剪预先指定的矩阵， 也就无法找到重要的矩阵，提升模型效果；无法裁剪不重要的矩阵，降低参数计算量，降低模型效果差的风险 。

AdaLoRA 主要解决方案：

  1. 使用原始 SVD 矩阵形式进行分解 Δ=PΛQ\Delta = P\Lambda QΔ=PΛQ ，同时在训练损失中添加额外的惩罚项，以规范奇异矩阵P和Q的正交性 。
  2. 基于重要性动态在训练过程中分配不同权重的秩 rrr
    将关键的增量矩阵分配高秩以捕捉更精细和任务特定的信息，将较不重要的矩阵的秩降低以防止过拟合并节省计算预算 。

矩阵修剪 / 调整秩 rrr

因为奇异值矩阵 Σ\SigmaΣ 是一个对角矩阵，实际情况下使用一个一维张量表示。

所以如果需要修剪时，不会改变左右矩阵 UVU VUV，仅仅将对应的奇异值设置为 0 即可。

这样，在下一次 forward 时，这个三元组里对应的 P向量 和 Q向量 相当于 被mask掉了，对 Loss 没有贡献，也就起到了 变秩 的效果。
但是随后的更新中，这个奇异值可能会被再次更新。这样 在一开始模型认为不重要的三元组，在后续过程中模型可能会慢慢学到它的重要性。

这种方法相当于 mask 了左矩阵的对应列 和 右矩阵的对应行。
因为 左矩阵的列之间相互正交独立，右矩阵的行之间相互正交独立，所以可以直接 mask ，而 LoRA 不是相互独立的，所以不能直接进行裁剪。
