ITERATIVE ZERO-SHOT LLM PROMPTING for KNOWLEDGE GRAPH CONSTRUCTION

Posted on 2023-07-18 In NLP
Symbols count in article: 1.5k Reading time ≈ 7 mins.

Paper：ITERATIVE ZERO-SHOT LLM PROMPTING FOR KNOWLEDGE GRAPH CONSTRUCTION

Proposes an innovative knowledge graph generation pipeline approach:

  • Iteratively prompting large language models to extract relevant components of the final graph
  • Zero Shot Prompt
  • Due to LLM, a scalable solution w/o labeling

技术： Named Entity Recognition, Relation Extraction, and Entity Resolution techniques

影响知识图谱构建工作的因素：
the right tradeoff between providing high data quality, scalability and automation

  • Data availability and acquisition：多个数据源，并且是异构的；数据是动态变化的，所以通过数据构造图谱的方式也要不断更新（对标监督学习）。
  • Data quality：数据质量应该是 accurate, complete, consistent, and reliable ，体现在标注上，但是成本很高。
  • Scalability：由于数据维度与数据源的异质性，需要可拓展的计算资源和成本。
  • Subjectivity and contextual knowledge：schema 需要上下文知识和外部知识。
  • Semantic disambiguation：语义消歧
  • Domain-specific expertise
  • External resources：entity linking，high risk of filtering out many relevant entities or concepts not included in the reference KB，还是泛化性不高。可以使用 Open Information Extraction (OpenIE)，但是会错误标注，precision 很低。
  • Evaluation
  • Pipeline definition

作者：
(i) 使用 LLM 进行抽取工作：Relies on adopting an LLM that is iteratively queried with a sequence of adequately well-formed prompts to perform the main stages of the KG generation
(ii) 不需要外部知识库和标注，自动化进行任务：Can effectively perform an automated entity/predicate resolution without the need of external knowledge-bases or any human effort.

Methodology

LLM Prompting

There are four supported roles, but we are only focused on the following:

  • system: the role which aims to tell the model how to behave in its responses; 提供指令
  • user: the role of the user messages/requests; 提供 input
  • assistant: the role of the model responses. 模型响应

Methodology Overview

  1. Candidate Triplet Extraction
  2. Entity/Predicate Resolution
  3. Schema Inference

Candidate Triplet Extraction

关键：设计合适的 prompt 来抽取三元组 ，作者将三元组的信息进行拓展 ，加入到 prompt 中。拓展内容为：

A proper entity characterization ， 除了需要抽取出 entity mention 外，还需要：

  1. a representative entity label
  2. a proper entity description
  3. a list of types or hypernyms that denote the entity beyond the mention

An appropriate characterization of triplets and predicates ，即关系标签：

  1. a suitable label
  2. a general description of the relationship existing between subject and object

总之，对单一的 label text 进行了拓展，加入了 description 和 hypernyms 。

作者认为 the generated description + the entity type list 能够提供更好的 entity context information，引导我们进行实际的语义化（semantification）。

Entity/Predicate Resolution

抽取出实体和关系后，需要进行消歧工作：

  • 实体消歧：identification and merging of mentions which refers to the same underlying entity 。
  • 关系消歧：identifying and resolving different expressions or textual forms that convey the identical relation between entities

难点：

  • 不借助于外部知识库和人工知识
  • 获取实体或谓词的完整语义知识仍然很复杂，尤其是在处理特定领域或复杂概念时。一词多义

作者使用 clustering + prompt LLM 的方式，构造一个解析模型，将抽取出来的不同的语义概念进行聚合（combines a semantic aggregation of concepts / identify and aggregate similar semantic concepts） 。

Schema Inference

the types of entities，their attributes and relationships ，traditionally requires domain expertise and feedback by human knowledge

作者仍然使用 prompt LLM 的方式自动化构建

Knowledge Graph Construction Pipeline

Candidate Triplet Extraction

在抽取前，首先将文本切块，满足 max seq len 。传统切片方法存在问题：

  1. 上下文语义的缺失
  2. subject 和 object 分布在不同的 text chunk，导致对应的关系无法被抽取出。

所以这里作者进行改进：

  1. 对于第 i 个 chunk，使用 1 - i-1 的chunk 得到其 preceding summary Summaryi−1Summary_{i-1}Summaryi−1​ 。使用 summaryi−1summary_{i-1}summaryi−1​ 和 前面一个 chunk chunki−1chunk_{i-1}chunki−1​ 得到其 summary:

summaryi= summarization_task (summaryi−1,chunki−1)summary_i= \text { summarization\_task } \left( summary_{i-1}, chunk_{i-1} \right) summaryi​= summarization_task (summaryi−1​,chunki−1​)

  1. 增加 chunk size 和 overlap ratio，缓解 subject 和 object 的分布不一致


切分文本之后，作者使用一种不断探索改进的方式来得到最优的 prompt，探索过程分为 the task decomposition 和 prompt definition，两者不断交替进行。

the task decomposition：

  1. step 1: identify the entities in a text chunk (entity extraction)
  2. step 2: check how they relate to each other to find the actual triplets (iterative triplet extraction)

Entity Extraction

Goal: identifying entity mentions in a text chunk

Content of a detailed system prompt:

  • S1) label item: 定义的实体标签和其意义
    不是必须的，但是加上之后召回数量变少，作者发现是 general concept 变少。

  • S2) 需求部分，instruction
    An explicit request to retrieve the entity mentions in the user’s text, specifying the additional requirement of providing a description and a list of types for each;

  • S3) 返回的格式

将抽取出来的所有实体集合计作 EEE 。

Iterative Triplet Extraction

问题：模型会自己生成一些不相干的实体，即 sub/obj not in EEE ，为了缓解这一现象：
作者每次针对一个困难实体 ei∈Ee_i\in Eei​∈E ，抽取潜在关系：

  1. Phrase Selection： 如果提供全部上下文 TTT ，则模型会抽取出不相关实体，所以在这里使用 QA 或者 summary 的方式，让模型首先生成关于实体 eie_iei​ 的关键信息（simple declarative sentences that provide information about the entity ei） TiGT_i^GTiG​ 。
    作者同样尝试使用上面抽取实体时，使用到的 label item ，效果也很好。

  2. Mention Recognition： 使用 prompt ，让模型去判别 E 中哪些其他实体被包含在 TiGT_i^GTiG​ 中。

  3. Predicate Description： tell GPT-3.5 to return the description of each unique predicate

  4. Response validation.

Entity/Predicate Resolution

Semantic aggregation

Semantic aggregation -> a higher-level concept ， 将语义相近的实体进行聚类

computing two specific similarity scores for entities (Se) and relations (Sr),

first consider all the pairs combinations for both entities and relations.

For each pair (i, j) of entities or relations, we compute the individual contributions：

  1. Label similarity：先计算两实体对或者两个关系对的标签相似度
  2. Description similarity：use embedding model

然后加权相加

Cluster disambiguation

在上面的聚类内部进行消歧，即找到指代不同的实体

prompt the GPT-3.

Concept shrinkage

To this end, each group of equal entities or relations returned by the Cluster Disambiguation module is used to compose a further prompt aimed at asking for a unique label for representing the underlying group. Such a label is used as the final unique representation of the given entity/relation.

an iteratively bottom-up LLM-based schema inference.

two main steps at each iteration:

  1. Hypernym Generation
  2. Hierarchical Agglomeration
