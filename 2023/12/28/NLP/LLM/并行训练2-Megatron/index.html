V2 版本：

  1. 作者提出 PTD-P ：将 pipeline model parallelism ， tensor model parallelism ， data parallelism 三者结合起来。

  2. 使用更多的通信量，换取空泡比率降低 ：

    提出了 virtual_pipeline_stage 来减小切分粒度，以 virtaul_pipeline_stage=2 为例，将 0-1 层放 Device1, 2-3 层放在 Device2，…，6-7 层放到 Device4，8-9 层继续放在 Device1，10-11 层放在 Device2，…，14-15 层放在 Device4。
    在 steady 的时候也是 1F1B 的形式，叫做 1F1B-interleaving。
    按照这种方式，Device之间的点对点通信次数(量)直接翻了 virtual_pipeline_stage 倍，但空泡比率降低了。

Pipeline

直接手动分配 Transformer 的层，而不是像 PipeDream 一样启发式搜索

为确保模型前向传播和后向传播使用的参数是一致的，这里引入了 定期管道刷新 periodic pipeline flushes ，而 PipeMare, PipeDream, PipeDream-2BW 则是完全放弃了这种定期刷新机制。

一个 batch 被分为 micro batches，下文讨论了两种在前向传播和后向传播中调度 micro batch 的算法。

Gpipe ：当 minibatch 数 m 远远大于pipeline 级数 p 时，bubble 率很小。但存在问题，每一个 minibatch 都需要保存 activation，导致内存变大。

PipeDream（仍然进行定期管道刷新） ：Forward 之后立即进入 Backwards，这样就不用保存那么多的 activation 了，这样 bubble rate 相同，但是内存利用率增加。

Interleaved Stages

本文提出一种交错的分配 pipeline stage 的方法，每个设备可以用于同一个 pipeline 的多个 stage。

要求：mini-batch 数量 是 pipeline stage 数量的整数倍。

可证明，假设一次前向传播中每一个设备负责 vvv 个 stage（ virtual_pipeline_stage ） ，则空泡比率变成了：
