多模态检索：UniVL-DR

Posted on 2023-08-10 In MultiModal
Symbols count in article: 1.6k Reading time ≈ 8 mins.

ICLR 2023 | UNIVERSAL VISION-LANGUAGE DENSE RETRIEVAL: LEARNING A UNIFIED REPRESENTATION SPACE FOR MULTI-MODAL RETRIEVAL

github: /OpenMatch/UniVL-DR

图 1(a) a divide-and-conquer approach :
为了实现多模态检索过程，当前的多媒体搜索系统通常采用“分而治之”( a divide-and-conquer approach )的方法。这些方法首先在单个模态中进行搜索，包括文本、图像、视频等 ，然后将各个模态的检索结果融合在一起，例如，在这些单/交叉模态检索器之上构建另一个排序模块来进行模态融合。

图 1(b) universal multi-modal retrieval -> a Universal Vision-Language Dense Retrieval (UniVL-DR) :
构建一个 end2end model ，同时检索多模态的字段 。首先将所有模态的字段映射到同一个表征空间（one universal embedding space），然后使用 KNN 来检索多模态的候选字段。

相关性建模（Relevance Modeling） 和 检索结果融合（Retrieval Result Fusion） 二者的建模过程通常交织在一起，以实现更准确的多模态检索结果。但是在多模态检索的场景下，不同的模态之间的建模和结果融合存在 gap 。

最终，本文将相关性建模（Relevance Modeling）、跨模态匹配（Cross-Modality Matching）和检索结果融合（Retrieval Result Fusion）进行统一的建模。

具体地：

  1. 使用 contrastive leaning + hard negatives + balanced-modality sampling，来对齐 vision-language embedding space，并且减轻 the modality preference，即检索时对某一种模态的偏好性。
  2. 同时使用 an image verbalization method ，使用 image caption 来减轻模态之间的 gap 。
    这种方法首先将 caption 和 原始像素块对齐（ aligns the semantics of image captions and figure pixels ），然后对 caption 进行润色，得到最终的 caption。

关于数据集，作者使用 a multi-modal question answering (QA) benchmark Web QA ，并将其转化为：给定一个 query，检索出对应的 candidate text & image collection 。

MULTI-MODAL RETRIEVAL TASK


Single Modality Retrieval
文本-文本检索 & 图像-图像检索，难点在于 「相关性搜索 search relevance」。

Cross Modality Retrieval
「text-image retrieval 文本-图像检索」 & 「image-text retrieval 图像-文本检索」
除了「相关性搜索 search relevance」之外，难点更加侧重于 「模态对齐 align semantics between captions and images」 。

这种跨模态匹配更加注重“浅层”的语义匹配，对于深层的文档理解能力要求不高。

Multi-Modal Retrieval
给定 query qqq ，在不同模态的字段数据集中进行查询，找出候选文本/图片字段，检索结果可能由文本文档、图像文档或文本文档与图像文档的混合序列组成。

多模态检索更加关注查询和文档之间的关联建模，且检索过程中涉及查询与文本文档的单模态匹配、查询与图像文档的跨模态匹配以及不同模态文档与查询的相似度之间的比较。
focuses more on relevance modeling between queries and documents, single/cross modality matching, and modality routing.

同时，我们可以将其他模态的字段转换为某个特定模态的数据（pre-route queries to a single modality），将任务简化为 text-text retrieval 和 text-image retrieval 。

Method

Univ-Search By Learning a Unified Embedding Space

query qqq
multi-modality documents D={dText 1,…,dText n,dImage 1,…,dImage m}\mathcal{D}=\left\{d_{\text {Text }}^1, \ldots, d_{\text {Text }}^n, d_{\text {Image }}^1, \ldots, d_{\text {Image }}^m\right\}D={dText 1​,…,dText n​,dImage 1​,…,dImage m​}

模型将用户查询、文本文档和图像文档编码在一个统一的向量表征空间中，并在该表征空间中进行「相关性建模 relevance modeling 」，「模态转换 modality routing」和「结果汇总融合 result fusion」。

Multi-Modality Dense Retrieval

UniVL-DR 由两个编码器构成：「文本编码器 TextEncoder」和「图像编码器 ImgEncoder」。查询、图像文档和文本文档均通过这两个编码器编码得到稠密向量表示。
