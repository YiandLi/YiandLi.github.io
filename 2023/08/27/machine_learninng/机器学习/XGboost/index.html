精确贪心算法

计算每一个节点的最优划分 – 精确贪心算法：

外层循环，遍历每一个特征 d ；
内层循环，遍历特征 d 的每一个可以取到的特征值 jjj ，用来划分节点。
有小细节：

  1. 选定特征后，在选划分的阀值时，需要对该特征内的特征值进行排序 。
    当特征内的特征值数量很多时，会耗费大量的资源，成为算法最耗时的部分。
  2. 复用更新：$G_L \leftarrow g_j $ : 在计算下一个划分方式时，可以复用上一次的累积结果，只需要加新的样本的 g 就好了 。
  3. 右节点 GR←G−GLG_R \leftarrow G - G_LGR​←G−GL​ : G 为 g 的总和， GL+GR=GG_L + G_R = GGL​+GR​=G 。

总时间复杂度为： O(d∗(nlog(n)+n))O( d *( n log(n) + n ) )O(d∗(nlog(n)+n)) ， ddd 为特征数量， nnn 为样本数量 。
优化思路为：特征筛选，减少 ddd ；样本筛选，减小 nnn -> 近似贪心算法 。

优化第二层遍历
常规操作下，对每一个特征，我们需要遍历内部所有特征，即采样所有的「可能划分点」。
我们要做的是从中减少采样遍历到的特征值的个数，采用分桶的做法。

假设当层样本数量为 D ， 分桶数量为 k ，我们希望每个桶里样本数量是均匀的，即 D/k 个样本。经过排序后，可以逐个累计特征值，直到桶中数量满足条件，然后划分另一个桶，最终得到的「可能划分点」的数量为 k-1 个，然后进行正常计算损失。

但是这有一个弊端，即上面我们是默认样本点是均匀分布的，即可以表示全局样本，这样均分样本点才能表示均分样本空间。
但是真实情况下，他们并不能表示全局样本，所以这里计算他们的权重，然后带权重切分，查找划分点。

加权分位

如何计算每个样本的权重？

然后就可以使用权重来进行加权分位了。首先将样本根据目标特征的对应特征值排序，

真实操作时，首先将所有样本权重进行归一化，然后设置一个超参数 ϵ\epsilonϵ ，同一个桶中的权重和不超过 ϵ\epsilonϵ 。
ϵ\epsilonϵ 取值为0到1之间。由此我们可以看出， ϵ\epsilonϵ 越大buckets数量越少，粒度越粗。


关于具体的策略，同样有两种：全局策略，局部策略。

全局策略

在构建树之前，对所有特征先预先计算进行一次重要性分组，得到对应的划分特征值。

整棵树的所有节点分裂使用某个特征 j 时，选择到的「划分特征值点」都是一样的，即后续都直接复用刚开始计算得到的划分特征值。

缺点：对于某个特征，随着树的生长，其特征值范围肯定是更小的，所以上层得到的「划分特征值点」不匹配，甚至导致没办法划分，因为现在所有样本都在同一个桶里了。

同时也表明，我们选择的 ϵ\epsilonϵ 小一点比较好，这样桶的数量就会多一些，缓解这种缺陷。

局部策略
每次进行节点分割时，我们都重新计算每个样本的二阶梯度并重新构建buckets，即每个节点都重新计算「划分特征值点」。
局部选择的编码复杂度更高，但在实验当中效果极其的好，甚至与Exact Greedy Algorithm一样。

总结分桶加权分位
寻找splitpoint的时候，不会枚举所有的特征值，而是先按照特征值排序，然后根据上述权重的密度分布，构造直方图计算特征值分布的面积，然后划分分布形成若干个bucket(桶)，每个bucket的面积相同，将bucket边界上的特征值作为split point的候选，遍历所有的候选分裂点来找到最佳分裂点。

近似算法首先按照特征取值的统计分布的一些百分位点确定一些候选分裂点，然后算法将连续的值映射到 buckets中，然后汇总统计数据，并根据聚合统计数据在候选节点中找到最佳节点。近似算法有两个变体， global variant和 local variant。

缺失值问题

训练时 ：
情况：选定了根据某特征的「划分特征值」，进行划分，但是存在一些样本在这个特征值上是 NaN 。
措施：将所有的「缺失值节点」看作一个整体，要么全在左子树，要么全在右子树，计算其增益，哪个大就划分到哪边。

推理时 ：
如果训练集没有缺失值而测试集出现缺失值，则要为缺失值（或者指定的值未出现的值）指定分支的默认方向，预测时自动将缺失值的划分到这一分支。

停止分割

(1) 当新引入的一次分裂所带来的增益Gain<0时，放弃当前的分裂。这是训练损失和模型结构复杂度的博弈过程。
(2) 当树达到最大深度时，停止建树，因为树的深度太深容易出现过拟合，这里需要设置一个超参数max_depth。
(3) 当引入一次分裂后，重新计算新生成的左、右两个叶子结点的样本权重和。如果任一个叶子结点的样本权重低于某一个阈值，也会放弃此次分裂。这涉及到一个超参数:最小样本权重和，是指如果一个叶子节点包含的样本数量太少也会放弃分裂，防止树分的太细，这也是过拟合的一种措施。

Shrinkage

加入了步长，即给每棵树加入了一个权重，即收缩率：

y^it=y^it−1+ηft(xi)\hat y_i^t = \hat y_i^{t-1} + \eta f_t(x_i) y^​it​=y^​it−1​+ηft​(xi​)

有助于防止过拟合，通常 $ \eta=0.1 $ 。

系统设计

分块并行

每一个节点分裂时，都需要遍历所有「候选特征」，计算其每一个划分点的增益，然后排序找最优划分点，最后找最优划分特征。

于是考虑并行：

  1. 特征的排序
    在训练之前，预先对每个特征内部进行了排序找出候选切割点，然后保存为block结构。
    xgboost将数据存储在内存单元中，称之为块（block）。每个块中的数据以压缩列（CSC）格式存储，每列按相应的特征值排序。

  2. 结点分裂时增益的计算（用分布式加权直方图确定buckets的分割点）
    各个特征的增益计算就可以开多线程进行，即在不同的特征属性上采用多线程并行方式寻找最佳分割点。
    树结点进行分裂时，需要计算每个特征对每个分割点的增益，分布式加权直方图使得特征的增益可以并行计算。

实现方法：

  • 特征排序：每个特征的值都会被单独排序，得到对应的 sorted key : instance index 序列。
    仅在整个训练过程的开始阶段执行一次离线的排序缓存，而不是每次生成子树时都进行排序。
    相当于牺牲空间，换取时间。

  • 分块存储：排序后的映射以CSC（Compressed Sparse Column）格式存储在内存中，每个特征的映射存储在一个块中。这种分块存储方式允许XGBoost在每个块上进行并行计算，每个块对应一个特征。

  • 并行计算：在节点分裂时，XGBoost可以启动多个并行进程或线程，每个进程/线程负责计算一个特征的分裂增益。
