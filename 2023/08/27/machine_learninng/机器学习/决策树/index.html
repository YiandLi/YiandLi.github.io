决策树

Posted on 2023-08-27 In Machine Learning
Symbols count in article: 4.5k Reading time ≈ 30 mins.

三个步骤：

  1. 特征选择：决定使用哪个特征来划分特征空间
  2. 决策树生成：对应局部最优
  3. 决策树修剪：对应全局最优

每个节点对应着特征的一个取值，叶节点还表示一个类。

特征选择

缺点

  • 普遍的观点：信息增益会偏向于变量取值更多的离散特征。
    当某个特征的经验熵很大时，他的信息增益量会天然地比其他特征要大。举个例子，如果使用 id 这种主键特征，所有不同 id 计算出来的条件熵都是 0，那么信息增益量就很大很大。

但是这种观点是不正确的，反例可以证明：

  • 同时，我们不希望使用这些经验熵很大的特征，因为他们泛化性也不好，容易过拟合。
    从机器学习的角度来看，此时特征a的划分可能更好。原因是b迅速地将样本空间划分的过小了，从而增加了过拟合的风险。

综上所述，当面临差不多的两种划分时，我们应该要避免选择划分分支更多的那一种，因为它会将样本空间划分的更小，从而会导致在其中的统计量的可靠性变差。而信息增益在面临此种选择时，不会偏向任何一方。

ID3 和 C4.5

  • ID3：非二叉树，使用「信息增益」选择特征，只能处理离散特征，只用于分类任务
  • C4.5：非二叉树，使用「信息增益比」选择特征，可以处理离散（set length）和连续特征（二分），只用于分类任务

常用终止条件：

  1. 最小增益：所有特征的「信息增益量/信息增益比」都小于某个阀值
  2. 没有特征可以选择
  3. 当前节点所有样本属于同一个类
  4. 最大深度
  5. 样本数量达到了阈值

从根节点开始，

  1. 对节点计算所有可能特征的 「信息增益」
  2. 选择信息增益最大的特征作为节点特征
  3. 根据特征的所有不同取值建立子节点
    • 离散特征：直接根据每个类别进行划分子节点即可，得到了 len(set(all_value)) 个叶子节点；
    • 连续特征（ id3 无法处理 ）：从小到大排列，然后使用二分法，选择中位数作为区间的划分点，每个子节点表示和一个划分区间。

剪枝 pruning

从已经生成的完全二叉树上裁掉一些子树或者叶节点，样本合并到父节点。

设树 TTT 的叶节点数量为 ∣T∣|T|∣T∣ ，每个叶节点 ttt 有 NtN_tNt​ 个样本点，该叶节点的经验熵为 Ht(T)H_t(T)Ht​(T) ，则代价函数/损失函数为：

Cα(T)=∑t=1∣T∣NtHt(T)+α∣T∣C_\alpha(T) = \sum_{t=1}^{|T|}N_t H_t(T) + \alpha |T| Cα​(T)=t=1∑∣T∣​Nt​Ht​(T)+α∣T∣

$\sum_{t=1}^{|T|}N_t H_t(T) $ 表示每个叶子节点的经验熵 乘以 该节点样本数量 ，表示了预测误差，也可以用总体的错误率来替代，即模型和训练数据的拟合程度；
α∣T∣\alpha |T|α∣T∣ 表示对树的复杂度的惩罚， α\alphaα 为超参数 。

具体流程为：不断计算每一个节点是否可以剪枝，直到无法裁剪，即损失函数无法变小则停止。
在具体运算时，针对某一棵子树，计算是局部进行的，即只需要计算裁剪这一棵子树前后损失函数的变化即可判断是否需要剪裁该子树。

CART

Classification And Regression Tree ，分类与回归树

二叉树，用于分类任务和生成任务，分别对应基尼系数和平方差损失函数

  • 分裂：分裂过程是一个二叉递归划分过程，其输入和预测特征既可以是连续型的也可以是离散型的，CART 没有停止准则，会一直生长下去；
  • 剪枝：采用代价复杂度剪枝，从最大树开始，每次选择训练数据熵对整体性能贡献最小的那个分裂节点作为下一个剪枝对象，直到只剩下根节点。CART 会产生一系列嵌套的剪枝树，需要从中选出一颗最优的决策树；
  • 树选择：用单独的测试集评估每棵剪枝树的预测性能（也可以用交叉验证）。

CART 在 C4.5 的基础上进行了很多提升:
1） C4.5 为多叉树，运算速度慢，CART 为二叉树，运算速度快；
2） C4.5 只能分类，CART 既可以分类也可以回归；
3） CART 使用 Gini 系数作为变量的不纯度量，减少了大量的对数运算；
4） CART 采用代理测试来估计缺失值，而 C4.5 以不同概率划分到不同节点中；
5） CART 采用「基于代价复杂度剪枝」方法进行剪枝，而 C4.5 采用「悲观剪枝」方法。

每次划分时，根据特征 j 和取值 s 将原始空间划分成两个子空间：

R1(j,s)={x∣x(j)⩽s} and R2(j,s)={x∣x(j)>s}R_{1}(j, s)=\left\{x \mid x^{(j)} \leqslant s\right\} \quad \text { and } \quad R_{2}(j, s)=\left\{x \mid x^{(j)}>s\right\} R1​(j,s)={x∣x(j)⩽s} and R2​(j,s)={x∣x(j)>s}

对于离散特征， CART 会遍历每种「两个集合」的可能情况，比如特征A有3个类别：A1,A2,A3，ART会考虑把特征A分成「A1和 A2,A3」，「A2和A1,A3」，「A3和A1,A2」三种情况。

回归树生成

回归树特征选择函数为：平方误差 ∑(yi−f(xi))2\sum(y_i - f(x_i))^2∑(yi​−f(xi​))2 ， yiy_iyi​ 表示当前节点的预测值，即均值。

单步划分步骤：

  1. 遍历所有特征 jjj
  2. 遍历该特征 jjj 下所有可能的切分点 sss ，根据切分点进行切分 。
    1. 离散特征：二分离散特征
    2. 连续特征：取相邻两样本值的平均数做划分点
  3. 分别得到切分后，两叶节点的均值，作为回归标签（期望均值最小）
  4. 计算总平方误差
  5. 保留总平方误差最小的 (j,s)(j,s)(j,s)

这种回归树也被称为 最小二乘回归树（least squares regression tree）

分类树的生成

使用基尼系数加权和选择最优特征，选择令「左右两子树的基尼系数和」最小的特征和特征值，来决定特征的最优二分点。

叶子节点预测值的选择 / unbalanced classes

在 id3 和 c45 中，叶节点的类别是选择 频率最高 的那一类，其实没有考虑类别不均匀的问题。

而 CART 使用了一种先验机制，其作用相当于对类别进行加权。即按照当前节点的 「类别频率/根节点类别频率」 ，即 概率最高 的那一类来分配最终叶节点类别。

对于一个二分类问题，节点 node 被分成类别 1 当且仅当：

N1( node )N1( root )>N0( node )N0( root )\frac{N_{1}(\text { node })}{N_{1}(\text { root })}>\frac{N_{0}(\text { node })}{N_{0}(\text { root })} N1​( root )N1​( node )​>N0​( root )N0​( node )​

比如二分类，根节点属于 1 类和 0 类的分别有 20 和 80 个。在子节点上有 30 个样本，其中属于 1 类和 0 类的分别是 10 和 20 个。因为 10/20>20/80，该节点就属于 1 类。

通过这种计算方式就无需管理数据真实的类别分布。假设有 K 个目标类别，就可以确保根节点中每个类别的概率都是 1/K。这种默认的模式被称为“先验相等”。

缺失值处理方式

在选择特征时，有些样本会缺失某个特征，于是引出了两个问题：

  1. 当某些样本没有该特征值时，如何计算该特征的收益？
  2. 如果确定使用该特征作为切分，那些缺少该特征的样本应该被划分到哪一个子节点？

id3 无法处理缺失特征值的情况
Random Forest，Xgboost，Lightgbm，C4.5 和 CART 是不一样。

C4.5 -> Probability Weights / fractional case

首先为每个样本分配一个权重，初始时候权重全部为 1 。

如何计算带缺失值的特征的收益？

当使用信息增益等计算收益时，需要先删除缺失样本，然后计算原始的收益，然后用收益乘以缩放系数 （ 非缺失值样本权重和总样本权重和\dfrac{\text{非缺失值样本权重和}}{\text{总样本权重和}}总样本权重和非缺失值样本权重和​ ）。
思想就是缺失值多的特征，对应的非缺失值样本数量就少，从而缩放系数就很小。通过这种降低权重的方式来体现信息的缺失 。

如果选中了缺失特征，遇到这个这个特征缺失的样本，如何划分到子节点？

答：将样本同时划分到所有子节点 ，不过将该样本的 权重 按各个子节点中「未缺失特征样本」的数量比例来更新，其实也就是以不同概率划分到不同节点中。
比如选定了特征 A ，有三个特征值(离散)/特征区间(连续) A1, A2, A3A_1,\ A_2,\ A_3A1​, A2​, A3​ ，其中对应无缺失特征元素数量为 $|A1|=2,\ |A2|=3,\ |A3|=4 $ ，那么每个缺失特征值的样本都会被划入这三个节点，对应的权重需要分别乘以 2/9, 3/9, 4/92/9,\ 3/9,\ 4/92/9, 3/9, 4/9 。

详细过程参考西瓜书，这里有节选：决策树之C4.5算法 。

CART -> Surrogate Splits

如何计算带缺失值的特征的收益？

和 C4.5 类似，选择该特征不缺失的样本进行计算。同时乘以非特征缺失样本的比重。

如果选中了缺失特征，遇到这个这个特征缺失的样本，如何划分到子节点？

此时选择的特征无法作为标准去划分缺失特征样本，需要使用其余的特征去代理当前的特征：依次使用其余的特征作为输入，去模拟预测当前的划分，使用误差来模拟划分相似度。

比如当前选择的最优划分条件分别为：“age<40”/“age>=40”，但是 age 这个特征存在一些缺失值，于是需要寻找代理划分特征：

  1. 将 “age<40”/“age>=40” 看作标签，依次遍历剩余的特征，找到其 最优划分点（即构建一个决策树顿），然后计算其误差（可以使用误分类率：#misclassified / n），这个误差其实就能表示这个特征划分方式和原始最优划分方式的相似度。

calculate to what extend alternative splits resemble the best split in terms of the number of cases that they send the same way.

  1. 将这些剩余特征根据误差进行排序，依次表示最优代理到最差代理。
  2. 计算 “blind rule : go with the majority”，即全部放到左边/右边的最小误差 “blind rule error”
  3. 将误差大于 “blind rule error” 的代理节点剔除
  4. 然后就依次划分缺失值样本，即从第一个代理开始进行划分，如果该代理特征缺失，则使用下一个，如果全都缺失，则 “go with the majority” 。

所以是在剩下的其他「非缺失特征」里面，找到和原始分割最接近的划分标准，作为一个代理 surrogate feature，把它选做分裂特征，进行分类。
如果找不到这样的代理特征，或者所有的其他特征都还不如无脑放入到某一个节点中去，那就直接把缺失值样本放到期望误差最小的子树中。
