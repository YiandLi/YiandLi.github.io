暴力搜索

为向量索引进行分区优化 IVF

  • 暴力搜索很慢，但是不需要训练 ；IVF 这里很快，但是需要预先训练，即 K-means 聚类。
  • 暴力搜索在添加 instance 时，直接 add 即可 ； IVF 如果只是少量数据（相对原始训练集）的话，可以直接 add ，但是在面对大量数据时，整个空间会被改变，则需要重新进行训练 。

流程：

  • 构造/训练：将整个样本空间划分为多个小的样本空间：使用 K-means 进行聚类，然后得到每个字空间的中心点，来代表整个空间。
  • 检索时：针对向量数据和每个中心点进行计算，求出它们之间的距离，然后得到最近的几个子空间。然后，将我们的搜索范围限定在这几个子空间内，在这几个子空间内进行暴力搜索。

在构造 IndexIVFFlat 时，需要传入三个参数：
quantizer index container
dimension hidden size
nlist the hyperparameter K of K-means / the number of sub-space

检索时：
topK 为最终返回的数量
index.nprobe 表示选择多少个子空间，数量越多，查询时间越长吗，精度相应变好（存在边际效益）。


为了追求更快的查询性能，IndexIVF 和向量 ID IndexBinaryIVF 都存储在倒排列表中。
我们无法通过向量 ID 来反查索引中的内容，如果我们想要得到某个数据的内容，需要手动重建索引。

乘积量化 Product Quantizer

动机
我们还是需要存储很多向量的，消耗了大量的内存。有没有什么办法，可以降低这个夸张的向量数据的内存资源占用呢？
在海量数据、以及数据快速增长的场景中，业务所使用的服务器资源的内存完全装不下我们的向量数据。无法支撑我们采用分区索引或者平面索引这种相对精确的相似性检索，我们需要想办法大幅降低内存占用。
同时，因为数据量极大，即使采用能够性能提升非常明显的分区索引，也无法满足低延时的计算结果返回。

优势
即能通过分片压缩来降低索引内存，也可以减少计算量，加快推理。

描述
矢量量化方法，即 vector quantization ，其具体定义为: 将向量空间的点用一个有限子集来进行编码的过程。

在 ANN(Approximate Nearest Neighbor,近似最近邻) 搜索问题中，向量量化方法又以乘积量化 (PQ, Product Quantization) 最为典型。

Train

  1. 将原始向量数据拆分为一些子向量 （ 根据超参数切片 ）
  2. 对于每组子向量，分别进行聚类操作 （ 根据超参数得到聚类数量 ），获得每组向量的质心 。
  3. 在向量中的子向量进行替换，使用距离它最近的向量集合的 ID。

训练过程

假设输入为 N (instance_num) 个 128 维度向量 。

设置超参数：「 切片数量 M 」，「 聚类数量 K 」

  1. 切分： 首先将原始向量 128 划分为 M 个片段，这里假设 M=4 ，于是每个向量就由 4 个 32d 的子向量组成，即输入变为 [ N * 4 * 32 ]。

  2. 训练： 转换下维度就是 [ 4 * N * 32 ] ，根据每一段 [ N * 32 ]，进行聚类，假设聚类数量为 K = 256 。
    则得到 4 组不同空间的聚类，每一个空间中有256 个簇心，于是得到了 [ 4 * 256 ] 个簇心，每个簇心为 32 维度向量，则总簇心的向量为 [ 4 * 256 * 32 ] 。

  3. 向量压缩： 对于一个向量 xxx ，可以被切分为 [ 4 * 32 ] 的子向量： [x1,x2,x3,x4],xi∈R32[ x_1, x_2, x_3, x_4 ] , x_i \in \mathbb R ^ {32}[x1​,x2​,x3​,x4​],xi​∈R32 ，将每一个 xix_ixi​ 对应到距离最近的簇心上，使用簇心的 id 进行表示，则为 [k1,k2,k3,k4]∈R4[k_1,k_2,k_3,k_4]\in \mathbb R^4[k1​,k2​,k3​,k4​]∈R4 。

意义：

  1. 通过 vector segmentation 压缩向量维度，然后原本的 float 变成现在的 index int
    这里的id 仅仅是看做index，用于标记簇团，即当前点属于哪一个簇团，不会被用来计算距离。

  2. 可以将理解为：向量被划分为多个子空间，然后子片段的连续向量变成了这个空间内簇团的 id 。

  3. 子空间被唯一一个 id 标识，大大降低内存。

  4. 如果两个向量足够接近，是会存在压缩后向量一致的情况，这个点不知道如何处理呢？有没有人讲一讲

  5. 压缩肯定会降低精度，这部分精度来源于，我们用一个 index id 来代替了原始的 32d float 的向量。

Inference

query 向量的维度还是 128 ， 这里使用相同的思路，切分为 [ 4 * 32 ]

  • Step1: 对于每一个 32d 的片段，去计算对应空间的簇心的距离，得到对应的簇心的距离，这里就可以得到 [ 4 * 256 ] 个距离 。

  • Step2: 然后就是计算当前向量和某一个 candidate 向量的距离，这个 candidate 向量已经在上一步被表示为 [k1,k2,k3,k4][k_1,k_2,k_3,k_4][k1​,k2​,k3​,k4​] ，而这里我们也计算出了当前向量对应这四个簇心的距离 d1,d2,d3,d4d1,d2,d3,d4d1,d2,d3,d4 ，那么直接查表相加即可: ∑(d1,d2,d3,d4)\sum(d1,d2,d3,d4)∑(d1,d2,d3,d4) 。

总体看下来，计算复杂度为：
step 1 : 计算 4×256 次 128/4维向量距离
step 2 : 4×N 次查表，N 次相加操作


其实这里有两种计算方式，分别是 [ SDC / symmetric distance computing ] 以及 [ ADC / asymmetric distance computing ] 。

如图， x 为 128d 查询向量 ，y 为某一个 128d 库内向量； q(y)q(y)q(y) 表示寻找到点 y 所属的聚类中心的过程，即上面训练部分的将其压缩为 4 个簇心 id 的过程 ： [k1,k2,k3,k4][k_1,k_2,k_3,k_4][k1​,k2​,k3​,k4​] 。

先看图片右边 [ ADC / asymmetric distance computing ] ，y 用簇心 id 表示，对应上面我们讲的 " 这个 candidate 向量已经在上一步被表示为 q(y)=[k1,k2,k3,k4]q(y)=[k_1,k_2,k_3,k_4]q(y)=[k1​,k2​,k3​,k4​] " ，然后 query 向量去计算和这个簇心表征向量 q(y)q(y)q(y) 距离，对应上面讲的直接查表 ，即可得到最终的距离。
此所谓非对称，即 query x 不变，还是自己的向量，而 candidate y 是被压缩过的，即 “原始向量 查 压缩向量”。

而相应的，对称版本 [ SDC / symmetric distance computing ] 则是将两个向量都进行压缩 xxx 到 q(x)q(x)q(x) ， yyy 到 q(y)q(y)q(y) ，然后计算 q(x)q(x)q(x) 和 q(y)q(y)q(y) 之间的距离。所以这里 symmetric 就是：都用对应的聚类中心来计算距离，所以是对称的。

SDC有一个很大的优点，两两聚类中心之间的距离，是我们在构建向量库的时候就可以知道的，完全可以离线就计算好，在线要计算q(x)和q(y)的距离的时候，直接查表，提升了在线query的效率。但是，他的误差也比ADC来的大，因为有x和q(x)，y和q(y)两个量化误差。

讨论

本质：使用到聚类的一种计算相似性的方法， 本质上还是去计算相似度 ，还是输入候选样本空间和query，输出最近的几个候选样本点。
缺点：PQ 乘积量化计算距离的时候，距离虽然已经预先算好了，但是对于每个样本到查询样本的距离，还是得老老实实挨个去求和相加计算距离。

Inverted File System

PQ优化了向量距离计算的过程，但是假如库里面的向量特别多，依然逃不了一个遍历整个库的过程，效率依旧还是不够高，所以这时就有了 Faiss 用到的另外一个关键技术 —— Inverted File System / 倒排 。

本质：快速锁定在某一（几）子空间，然后在该（几个）子空间里做遍历。

做法也很简单：

  • 训练：直接对库中向量做聚类（原始维度下），然后每一个聚类内部进行 PQ。
  • 在索引时，给淡定 query 向量，和簇心计算相似度，选择 topK 个簇心，然后将对应簇下所有点看做 candidate set，只计算查询向量与这几个簇底下的向量的距离。

使用了IVF过后，需要「计算距离」和「进行PQ」的向量个数就少了几个数量级，最终向量检索就变成一个很快的操作。至于剩下的如何计算，是直接 L2 相似度，还是选择上面讨论的 DQ 相似度，就自行选择了。

在进行 Product Quantization (PQ) 时，一般是每一个 Inverted File (IVF) 聚类内部单独进行 PQ。

PQ 是一种将高维向量量化为多个低维子向量的技术。在 IVF 中，将数据集划分为多个聚类（或称为倒排文件），然后对每个聚类进行 PQ 编码。这样做的好处是可以将数据集分割为更小的部分，并且在每个部分上应用 PQ 编码，以便更有效地存储和搜索。每个 IVF 聚类内部的数据通常在相同的低维码本中进行编码，这有助于提高搜索的效率。

Residual Vector 残差向量

Faiss 具体实现有一个小细节，就是在计算查询向量和一个簇底下的向量的距离的时候，所有向量都会被转化成与簇心的残差，这应该就是类似于归一化的操作，使得后面用PQ计算距离更准确一点。

在经过第一步聚类后，库内的每一个 yyy 点都有其对应的一个簇，和簇中心点 c(y)c(y)c(y) （非DQ）。我们不会在样本 yyy 上直接做PQ量化，而是对 yyy 和 c(y)c(y)c(y) 的残差向量 y−c(y)y-c(y)y−c(y) 做量化。

笔者这里使用 c(y) 表示粗聚类得到的中心点，并不是 PQ 算法中的 d(y) ，感觉还是不是一个东西的。

合理性

dist⁡(X,Y)=dist⁡(X,K)+dist⁡(X, residual )\operatorname{dist}(X, Y)=\operatorname{dist}(X, K)+\operatorname{dist}(X, \text { residual }) dist(X,Y)=dist(X,K)+dist(X, residual )

最开始，我们是直接对点 y 做量化（就是整个库 set of y ），然后 PQ 计算距离，计算距离最理想的情况是计算 x 到 y 的相似度，这是肯定没有误差的。

但是这里用到了PQ压缩，算的是 x 到 q(y) 的距离，可以说 q(y) 代表了 y ，这部分产生了误差；并且由于数量级的关系，如果原始的向量各个元素比较大，那么可以看作整个空间的分布都比较大，这样一个簇团的方差就很大。

而如果单独看 q(y) 的残差，那么数量级就会小很多。比如说样本向量Y是 [2.4, 2.1] ， 聚类中心K是 [2.3, 2.0] ，那么残差为 [0.1, 0.1 ] 。

这时候就可以先计算 query x 到簇心 K=c(y)K=c(y)K=c(y) 的距离 dist⁡(X,K)\operatorname{dist}(X, K)dist(X,K) （没有误差），再计算残差的部分 dist⁡(X, residual )\operatorname{dist}(X, \text { residual })dist(X, residual ) （误差变小很多）。

计算流程
训练：

  1. 使用 IVF 进行聚类，计算出每个候选 y 对应的中心点 id: qc(y)q_c(y)qc​(y)
  2. 然后计算他们的残差 r(y)=y−qc(y)r(y)=y-q_c(y)r(y)=y−qc​(y)
  3. 然后使用所有的 r(y)r(y)r(y) 去训练一个 PQ 表格，得到对应的 PQ code qp(r(y))q_p(r(y))qp​(r(y))
  4. 这个时候每个 y 就有两个东西，一个是对应的IVF-kernel qc(y)q_c(y)qc​(y) ，一个是 PQ code 表示的残差 qp(r(y))q_p(r(y))qp​(r(y)) 。

每个 y 就被两个东西表示：

  • 对应的 IVF-kernel qc(y)q_c(y)qc​(y)
  • PQ 表示的残差

在检索时：

  1. 还是用类似的表示方式表示query x，得到 对应的 IVF-kernel qc(x)q_c(x)qc​(x) 和残差 r(x)r(x)r(x) 。
    注意这里是残差，而不是残差 code，和 PQ 一样，query 用真实值 r(x)r(x)r(x) ，去和 PQ 中的 qp(r(y))q_p(r(y))qp​(r(y)) 计算相似度。

  2. 然后就是使用 SDC / ADS 来计算点到点的相似度了

    • 图为「同一个簇内 ADC」的情况，使用 query 真实值，所以没有计算 dist⁡(X,K)\operatorname{dist}(X, K)dist(X,K) ，因为簇内部的这一部分全部相同。

    • 记住 PQ 只是计算相似度的一种手段，上面也只是去找，同一个簇内部，谁的残差和自己的比较接近而已。

    • 个人理解：因为我们可以选择多个候选簇，然后遍历内部的样本。在属于不同的簇时，则需要计算的是 (Kx,Ky)+(Rx,Ry)(K_x,K_y) + (R_x, R_y)(Kx​,Ky​)+(Rx​,Ry​) ，前者 (Kx,Ky)(K_x,K_y)(Kx​,Ky​) 可以直接计算精确值 ，后者因为在训练 PQ 时，所有的 course cluster 中的 residue 都是 一起训练的，所以也可以直接去求，其实就是上面的 d(r(x),qp(r(y)))d(r(x), q_p(r(y)))d(r(x),qp​(r(y))) 。
