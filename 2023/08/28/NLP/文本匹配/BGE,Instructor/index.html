Sentence-T5

三种得到句向量的方法 ：

  1. Encoder-only first ：利用T5 最后一层encoder第一个token的特征输出作为句向量。因为不是 cls ，所以不是很靠谱
  2. Encoder-only mean ：利用T5最后一层encoder所有token的特征输出平均值作为句向量。
  3. Encoder-Decoder first ：利用T5最后一层 decoder 的第一个 token 的输出作为句向量。
    为了获得这个 token 的输出，需要将 input 文本输入 encoder，还需要将代表start的 token 喂给 decoder。
    然后用 decoder 生成的 token representation + MLP + Normalization 作为句子的表征。

实验发现 2. 3. 效果比较好。

训练时使用了双塔结构，左右两边使用一个 backbone 。
方法使用 in-batch contrastive learning 。
采用了两阶段的训练方式，第一个阶段现在「通用领域的问答数据训练（Community QA）」,第二阶段才在「人工标注的数据（NLI）」上训练。

实验中训练过程使用了巨大的batch size，第一阶段的batch size是2048，第二阶段的batch size是512，远超simcse实验中的最佳参数配置64。

Generalizable T5-based dense Retrievers (GTR)

  • 双阶段，训练数据更多
  • 增加 pos instance - neg instances 损失

Large Dual Encoders Are Generalizable Retrievers

backbone 和 训练方式还是 Sentence-T5 。
模型使用上面的方法2「Encoder-only mean」，训练数据把第二阶段换成了另外的检索相关的数据集，MS Macro(Bing的搜索数据)跟Natural Questions(常用于检索的问答数据集)。

训练损失在 simcse 基础上增加了 a bi-directional in-batch sampled softmax loss ，首先训练数据由 (query, pos instance, neg instances) 组成，然后：

  1. query 和 pos instance 作为正样本对，query 和 neg instances 作为负样本对，进行对比学习。

L=esim⁡(qi,pi+)/τ∑j∈Besim⁡(qi,pj+)/τ+esim⁡(qi,pj−)/τ\mathcal{L}=\frac{e^{\operatorname{sim}\left(q_i, p_i^{+}\right) / \tau}}{\sum_{j \in \mathcal{B}} e^{\operatorname{sim}\left(q_i, p_j^{+}\right) / \tau}+e^{\operatorname{sim}\left(q_i, p_j^{-}\right) / \tau}} L=∑j∈B​esim(qi​,pj+​)/τ+esim(qi​,pj−​)/τesim(qi​,pi+​)/τ​

  1. pos instance 和 query 作为正样本对，pos instance 和 neg instances 作为负样本对，进行对比学习。

依旧巨大的batch size，两个阶段的训练batch size都是2048。
随着模型规模的增加，召回的文档长度也有增加的趋势。

Instruction-based Omnifarious Representations (INSTRUCTOR)

  • 增加 instruction

对于句向量模型而言，同样一句话，可以通过给定模型不同的instruction，让模型生成适合下游任务的句向量嘛

数据集中的每个instance都是如下格式，将 instruction 和 sentence 进行拼接作为输入。

相似度匹配任务：instruction_1 == instruction_2
类似检索的非对称任务：instruction_1 != instruction_2 ; Sentence_2 表示是跟 sentence_1 相关的文本，而 sentence_3 则是不相关的文本。

构建数据的方式；
首先计算样本间的分数 ：

正样本为 highest spos s_{\text {pos }}spos ​ ； 难负样本为 highest sneg s_{\text {neg }}sneg ​ 。
use one hard negative together with in-batch sampled negatives in the training
通过这种数据集构造，可以保证每个instance都有自身的hard negative，以及同个batch内其他instance充当简单负样本。在训练过程过程，为了提高难度，会保证同个batch里所有instance都来自于同个数据集。

EmbEddings from bidirEctional Encoder rEpresentations (E5)

  • 使用一致性检验，构造训练数据。
  • fine-tune 引入 cross-encoder 蒸馏机制
    Text Embeddings by Weakly-Supervised Contrastive Pre-training

微软的研究人员自建一个庞大且高质量的来自互联网的文本匹配数据集 CCPairs
每个数据 (q,p) 代表一个文本对，q 表示 query，p 表示与之相关的 passage，这里的 p 可以是任意长度，既可以是短文本，也可以是文章的一个段落，或者一个长文档。
过简单的规则例如过滤掉 p 长度超过 4096 个字符或者困惑度太高的数据等自动挖掘的方式后，总共获得了 13亿的文本对数据。

Consistency-based filter

提出了一种一致性过滤（Consistency-based filter）的方法：

  1. 先让模型在13亿的噪声数据上进行训练 。
  2. 利用训练好的模型给每个数据对（q,p）排序，候选池里包括一百万个passage，如果p能落在top_k个候选结果中，就保留该数据对，否则则移除该数据对。
    这种技术的直觉来自神经网络的记忆行为：当在噪声数据集上进行训练时，神经网络倾向于先记忆干净的标签，然后逐渐过拟合噪声标签。

希望模型的预测能跟训练数据保持一致。在文中作者将k设置为2，通过这种方式最终获得包括2.7亿文本对数据的数据集CCPairs。

Method

Model: encoder model, 对输出层进行平均池化

  1. 在没有人工标注的CCPairs数据上通过对比学习训练。
    训练时在双塔模型的两侧q跟p会分别加上前缀“query：”跟“passage:” ，Instructor 弱化版 。

  2. 在一个相对量级较小的高质量人工标注数据集（MS-MARCO, NQ, NLI）上作进一步训练，从而提高生成的句向量质量。

训练方法：

  1. 对比学习，但是这里的负样本不再是同个batch其他样本的数据，而都是hard negative。batch_size 是 32768 ，弥补缺乏hard negative的问题使用了超大的batch size 。

min⁡Lcont =−1n∑ilog⁡esθ(qi,pi)esθ(qi,pi)+∑jesθ(qi,pij−)\min L_{\text {cont }}=-\frac{1}{n} \sum_i \log \frac{\mathrm{e}^{s_{\boldsymbol{\theta}}\left(q_i, p_i\right)}}{\mathrm{e}^{s_{\boldsymbol{\theta}}\left(q_i, p_i\right)}+\sum_j \mathrm{e}^{s_{\boldsymbol{\theta}}\left(q_i, p_{i j}^{-}\right)}} minLcont ​=−n1​i∑​logesθ​(qi​,pi​)+∑j​esθ​(qi​,pij−​)esθ​(qi​,pi​)​

  1. fine-tune 引入一项蒸馏学习损失，训练时引入了cross encoder作为teacher模型，为bi-encoder的优化提供更多信号。batch_size 是 256，使用的都是高质量的hard negative。

min⁡DKL(pce,pstu)+αLcont\min D_{\mathrm{KL}}\left(p_{\mathrm{ce}}, p_{\mathrm{stu}}\right)+\alpha L_{\mathrm{cont}} minDKL​(pce​,pstu​)+αLcont​

前者用于困难标签，后者用于从教师模型中蒸馏软标签。

GTE

注意重点，GTE主要特色是显著增加训练数据，大力出奇迹～
Towards General Text Embeddings with Multi-stage Contrastive Learning

模型：双塔 + last layer mean pooling

损失：进阶版本对比损失 Improved Contrastive Loss
对负样本进行扩充：

Licl=−1n∑i=1nlog⁡es(qi,di)/τZZ=∑jes(qi,dj)/τ+∑j≠ies(qi,qj)/τ+∑jes(qj,di)/τ+∑j≠ies(dj,di)/τ\begin{aligned} L_{\mathrm{icl}} & =-\frac{1}{n} \sum_{i=1}^n \log \frac{e^{s\left(q_i, d_i\right) / \tau}}{Z} \\ Z & =\sum_j e^{s\left(q_i, d_j\right) / \tau}+\sum_{j \neq i} e^{s\left(q_i, q_j\right) / \tau} \\ & +\sum_j e^{s\left(q_j, d_i\right) / \tau}+\sum_{j \neq i} e^{s\left(d_j, d_i\right) / \tau} \end{aligned} Licl​Z​=−n1​i=1∑n​logZes(qi​,di​)/τ​=j∑​es(qi​,dj​)/τ+j=i∑​es(qi​,qj​)/τ+j∑​es(qj​,di​)/τ+j=i∑​es(dj​,di​)/τ​

分母多了 q-q loss 和 p-p loss 。

和 GTR 的（document to question matching loss）的区别：分母部分相当于合并了。

第一阶段：

  • 由于不同源头的数据量显著不同，需要采用特定的抽样策略来维持平衡
  • 要保证同个batch里的数据都是来源于同个任务，防止模型通过学习到不同任务特性来判断这些数据
  • 在这一个阶段由于缺乏hard negative信息，为例保证模型效果，需要使用较大的batch size

第二阶段：

  • 以少量的人工标注的数据集（query，positive document）为基础，利用额外的检索器获得相应的hard negative数据，从而构造出相应的文本相关性三元组（query, positive document, negative document）数据，约300万条 。
  • 由于数据集存在强监督信息（hard negative），batch size就没有必要设置的特别大。

BGE

  • 高效RetroMAE预训练 + 大规模文本对微调（仅仅为 query 添加 instruction）

训练过程分为两阶段，第一阶段是采用RetroMAE方法的预训练过程，第二阶段就是利用对比学习的微调过程 。
微调时会在检索任务中为query单独添加对应的instruction。

RetroMAE预训练

目前检索模型的预训练有两种策略：

  1. self-contrastive learning：受限于数据增强的质量，并且需要采用非常庞大数量的的负样本。
  2. anto-encoding：不受数据增强跟负样本采样策略的影响。基于这种方法的模型性能好坏有两个关键因素，其一是重建任务必须要对编码质量有足够的要求，其二是训练数据需要被充分利用到。

Model

  • encoder ：a full-scale BERT ；用小一点的 mask rate 得到 sentence embedding
    输入句子进行 mask（15%-30%） ，直接使用 cls token 作为句子编码：

hX~←Φenc(X~enc)\mathbf{h}_{\tilde{X}} \leftarrow \Phi_{e n c}\left(\tilde{X}_{e n c}\right) hX~​←Φenc​(X~enc​)

  • decoder ：
    one transformer layer（ 这里应该也是 encoder 结构，而不是decoder结构 ）；
    用大一点的 mask rate 结合 encoder 得到的 sentence embedding 进行重构 。
    输入句子进行 mask（50%-70%），embedding，+ positional embedding 后，和上面的句子表征 $\mathbf{h}_{\tilde{X}} $ 进行拼接，联合输入到 decoder ：

HX~dec←[hX~,ex1+p1,…,exN+pN]\mathbf{H}_{\tilde{X}_{d e c}} \leftarrow\left[\mathbf{h}_{\tilde{X}}, \mathbf{e}_{x_1}+\mathbf{p}_1, \ldots, \mathbf{e}_{x_N}+\mathbf{p}_N\right] HX~dec​​←[hX~​,ex1​​+p1​,…,exN​​+pN​]

损失函数为：Encoder-MLM Loss + decoder MLM Loss

Ldec =∑xi∈ masked CE(xi∣Φdec (HX~dec ))\mathcal{L}_{\text {dec }}=\sum_{x_i \in \text { masked }} \mathrm{CE}\left(x_i \mid \Phi_{\text {dec }}\left(\mathbf{H}_{\tilde{X}_{\text {dec }}}\right)\right) Ldec ​=xi​∈ masked ∑​CE(xi​∣Φdec ​(HX~dec ​​))

解码器结构简单，只有单层transformer，而且输入句子的掩码率很高，会使得解码变得具有挑战性，可以迫使生成高质量的句子嵌入，以便可以以良好的保真度恢复原始输入。

enhanced decoder：要求decoder仅仅通过sentence embedding来生成全部的token
在上述方法基础上，对 decoder 进行了改进。
动机：

  • training signals can only be derived from the masked tokens.
    -> more training signal: 所有 token 并行构建 loss
  • every masked token is always reconstructed based on the same context HX~dec\mathbf{H}_{\tilde{X}_{d e c}}HX~dec​​ .
    -> diversified contexts: sampled context tokens

首先进行双流编码：

query:H1←[hX~+p0,…,hX~+pN]context:H2←[hX~,ex1+p1,…,exN+pN]\begin{aligned} \text{query}: \mathbf{H}_1 &\leftarrow\left[\mathbf{h}_{\tilde{X}}+\mathbf{p}_0, \ldots, \mathbf{h}_{\tilde{X}}+\mathbf{p}_N\right] \\ \text{context}: \mathbf{H}_2 &\leftarrow\left[\mathbf{h}_{\tilde{X}}, \mathbf{e}_{x_1}+\mathbf{p}_1, \ldots, \mathbf{e}_{x_N}+\mathbf{p}_N\right] \end{aligned} query:H1​context:H2​​←[hX~​+p0​,…,hX~​+pN​]←[hX~​,ex1​​+p1​,…,exN​​+pN​]​

hX~\mathbf{h}_{\tilde{X}}hX~​ 为 encoder sentence representation
exie_{x_i}exi​​ 为 decoder token embedding (w/o masking)，和上文一致
pip_ipi​ 为 position embedding

在此基础上进行 position-specific attention：

Q=H1WQ,K=H2WK,V=H2WV;Mij={0, can be attended, −∞, masked ;A=softmax⁡(QTKd+M)V.\begin{gathered} \mathbf{Q}=\mathbf{H}_1 \mathbf{W}^Q, \mathbf{K}=\mathbf{H}_2 \mathbf{W}^K, \mathbf{V}=\mathbf{H}_2 \mathbf{W}^V ; \\ \mathbf{M}_{i j}= \begin{cases}0, & \text { can be attended, } \\ -\infty, & \text { masked } ;\end{cases} \\ \mathbf{A}=\operatorname{softmax}\left(\frac{\mathbf{Q}^T \mathbf{K}}{\sqrt{d}}+\mathbf{M}\right) \mathbf{V} . \end{gathered} Q=H1​WQ,K=H2​WK,V=H2​WV;Mij​={0,−∞,​ can be attended,  masked ;​A=softmax(d​QTK​+M)V.​

  • 使用 encoder sentence representation + position embedding 构建 Q
  • 使用 encoder sentence representation + decoder token embedding + position embedding 构建 K，V
  • 这里 mask 矩阵被改进为 position-specific attention mask M\mathbf{M}M ，从原本的只能看到 prefix token ，变成了现在的随机在整个句子上进行 sample，可以看到被 sample 后的前后 token ；同时保证第一个 token，即 encoder sentence representation 肯定能被看到。
    The sampled tokens, s(X≠i)s(X\ne i)s(X=i), and the 1st position (except for the 1st row) will be visible when reconstructing xix_ixi​ .

each token xix_ixi​ is reconstructed based on the context which are visible to the iii-th row of matrix M\mathbf{M}M .

相当于使用 encoder sentence representation （而不是上一步生成的 token）去从 sampled context token 中查找重要的上下文 token（而不仅仅是前缀）。

这么做，所有的 token 都是并行地进行预测，所以都可以提供 loss （而不是仅仅由 masked token 提供 loss）。

最后使用 residual_connection(A,H1)\text{residual\_connection}(\mathbf A, \mathbf H_1)residual_connection(A,H1​) 来重构每一个 input token:

Ldec =∑xi∈XCE(xi∣A,H1)\mathcal{L}_{\text {dec }}=\sum_{x_i \in X} \mathrm{CE}\left(x_i \mid \mathbf{A}, \mathbf{H}_1\right) Ldec ​=xi​∈X∑​CE(xi​∣A,H1​)

注意这里第一个 token ，也就是对应 encoder sentence representation 的 token，也需要被重构，但是其不会看到自己。

大模型进行文本匹配

如何提高LLMs的文本表征(Text Embedding)能力?

为什么主流方法不用直接用大模型进行文本向量表征？
效果不理想。现在大模型的训练都是基于文本续写形式来训练的，损失函数主要预测下一个Token是否准确，而不是判断整个句子表征的好坏。因此直接用当前的大模型来对文本进行表征，效果并不理想。但如果将大模型以文本表征为目的的训练，也就用point-wise、pair-wise、list-wise等方法训练大模型，相信大模型文本向量表征一定更好。

《Improving Text Embeddings with Large Language Models》:

  1. 构造高质量训练数据: 直接挖掘大模型内部存储的知识内容
    1. 通过GPT4模型进行头脑风暴，生成固定个数的检索任务列表
    2. 通过GPT4模型根据上一步生成的任务定义、Query类型、Query长度、Query清晰度、文档片段长度、语言等生成具体示例。
  2. 文本向量表征时写好提示词
    将数据末尾添加一个[EOS]标记，文本经过大模型后，将[EOS]标记对应的向量作为整个文本的句子向量。并采用InfoNCE作为损失函数，负例采用Batch内负例和一个难负例。
  3. 选对底座大模型
    DeepSpeed ZeRO-3 + Lora 微调 mistral-7b
    基座模型的选择比较重要，选择Llama2-7B要比Mistral-7B低1.6%，说明底座模型越好，预训练数据效果越好
