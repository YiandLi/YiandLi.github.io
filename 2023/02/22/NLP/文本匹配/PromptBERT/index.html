文本匹配：PromptBERT

Posted on 2023-02-22 In NLP , Text Retrieval
Symbols count in article: 1.9k Reading time ≈ 10 mins.

Paper: EMNLP 2022 - PromptBERT: Improving BERT Sentence Embeddings with Prompts

当前使用 BERT 编码句子进行文本匹配效果不好，新方法就来了。本文中

  1. 作者首先通过实验证明两个原因：static token embedding bias 和 ineffective BERT layers，
  2. 然后提出了一个通过 prompt 构造句子向量的方式 ( prompt-based sentence embeddings )，
  3. 然后为了得到更好的 prompt-based sentence embeddings，讨论了两种 prompt 表示方法和三种 prompt 构造的搜索方法。
  4. 最后，作者从 prompt de-noising 出发，提出了一个新的无监督训练目标函数，减小了 unsupervised 和 unsupervised settings 下进行训练的差距。

Rethinking the Sentence Embeddings of the Original BERT

先前的工作已经指出了 bert sentence embedding 做匹配任务的缺点，即词向量的各相异性 (anisotropic token embeddings space) 。
但是作者认为这不是最核心的因素，进而提出了两个影响更大的因素： ineffective BERT layers 和 static token embedding biases 。

Observation 1: Original BERT layers fail to improve the performance.

作者探究 Bert 的 encoder / attention layer 对 sentence embedding 的影响。

这里比较两种 sentence embedding

  1. averaging static token embeddings (input of the BERT layers): the 0-th layer’s output representation
  2. averaging last layer (output of the BERT layers): the 12-th layer’s output representation

作者使用一个直观的「模型编码空间的各向异性」的方法： 在通用的语料中，如果随机抽若干句子，计算它们之间的平均余弦相似度

1n2−n∣∑i∑j≠icos⁡(M(si),M(sj))∣\frac{1}{n^2-n}\left|\sum_i \sum_{j \neq i} \cos \left(M\left(s_i\right), M\left(s_j\right)\right)\right| n2−n1​∣∣∣∣∣∣∣​i∑​j=i∑​cos(M(si​),M(sj​))∣∣∣∣∣∣∣​

MMM 表示按照上述方法计算 sentence embedding 。
如果结果越接近于1，说明异向性越严重，如果接近于0，说明同向性很好 。
换言之，「模型编码空间的各向异性」可以通过一组句子的 average cosine similarity 进行量化。


从 Spearman correlation（越大表示越相关）看出：模型 1，3效果都下降了，而模型 2 也仅仅上升了 0.28 。
从 Sentence anisotropy （越大表示越相关）中，通过对比bert-base-uncased可以看出：Bert layers 不能决定是否会加剧 sentence embedding 的各向异性。
个人认为 Spearman correlation 是衡量空间中的距离，Sentence anisotropy 是衡量角度差（和推理一致）。

从另一种角度，Sentence anisotropy 可以表示 performance，Spearman correlation 可以表示空间上的分布，即是否各向同性。对比可以看到：任务performance和句子层面上的anisotropy之间没关系。

Observation 2: Embedding biases harms the sentence embeddings performance.

普遍认为「各向异性」的原因是token的频率不定，从而导致空间分布不均（词嵌入的偏置），无法比较。

首先作者将「token的频率 影响空间分布」称为一种 bias，除了这一个因素（token频率）会影响空间分布，作者又举例出影响空间分布的另一个因素：subwords case in WordPiece ，即：
在 tokenize 时，token/word piece 被分成的不同情况：

  1. the lowercase beginof-word tokens
  2. the uppercase begin-of-word tokens ( uncased 模型归为第一种情况
  3. the subword tokens.

根据这两种 bias 进行分类，作者用 PCA 进行 token embedding 的可视化：

    我们可以看出词频高的token，分布比较紧凑，词频低的token，分布较分散。

Represent Sentence with the Prompt

模版为： [X] means [MASK]\text{[X] means [MASK]}[X] means [MASK]

  • 第一种方法：使用 [MASK]\text{[MASK]}[MASK] 作为 sentence embedding

  • 第二种方法：选取由 [MASK]\text{[MASK]}[MASK] 输出的top-k tokens，取其 概率加权平均 作为 sentence embedding

第二种是较传统的做法，但是缺点也很明显：
1. 我们利用到了static token embedding，就会受到bias的影响；
2. 加权平均中的离散采样过程会让bert在下游任务上很难去fine-tune。-》 gumble softmax

Prompt Search

难点在于如何搜素模版

  • 人工离散模版
    • relation式，即类似  is a \text{ is a } is a  的方式 。
    • prefix式，即前缀方式， 效果比relation式更好
  • 自动化搜索离散模版
    利用T5去生成prompt templates，效果不一定比「人工构建的离散模版」要好。
  • 自动化可优化连续模版，使用到了 OptiPrompt 模型，效果比「人工构建的离散模版」更好。

Prompt Based Contrastive Learning with Template De-noising

出发点：使用对比学习来进行训练

谈到对比学习，就要考虑如何构造正例。
受到数据增强的启发，作者这里使用不同的模版对相同的输入进行处理，从而得到不同版本的文本编码，可以看成是 positive pair。

尽管很合理，作者认为句向量还是或多或少受到模版的影响，所以提出了一种 De-noising Template 的方法。

个人认为，去掉模版的信息，其实就是仅仅让模版起到了提示作用，而不对编码产生影响，这种影响也是一种 bias 。

首先使用模版进行构造，比如 [X] means [MASK]\text{[X] means [MASK]}[X] means [MASK] ，然后得到句向量 hi\mathbf{h}_ihi​ ，这里的句向量在 attention 中是融入了模版的信息的。
现在要考虑去掉模版的信息，我们单独将这个模版  means [MASK]\text{ means [MASK]} means [MASK] 输入到 bert 中，在输入时候 position id 需要和上面的 position id 相同（这样就完全相同了，不难理解），这样就只得到了 template 对句向量 [MASK]\text{[MASK]}[MASK] 的贡献句向量 h^i′\hat{\mathbf{h}}_i^{\prime}h^i′​ 。
于是， hi−h^i\mathbf{h}_i-\hat{\mathbf{h}}_ihi​−h^i​ 就是 de-noising sentence representation 。

Experiment

对比了无监督（未精调），有监督的结果，均 at-present on-paper SOTA 。

个人总结

  1. embedding 分布空间确实受到词频，token piece type 的 bias 影响，但是这并不绝对地正相关影响「各向异性」，即我们最终推理时的评价逻辑/方式。
    不确定「各向异性」是否和「推理时的评价指标」是一回事，但是本文中评价「各向异性」的指标和「推理时的评价指标」一致。

  2. Bert layer 无法解决「各向异性」问题，同时对比 observation 2，无法优化embedding 分布空间的类聚类分布。

  3. Roberta token embedding 是相对「各向同性」的。

  4. 一种利用 prompt 得到句向量的方法

  5. template 也会带来 bias，并且作者给到了一种降噪方法，个人认为这种方法同样适用于其他任务。
