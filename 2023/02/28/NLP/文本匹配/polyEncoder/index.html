Paper: Poly-encoders: Transformer Architectures and Pre-training Strategies for Fast and Accurate Multi-sentence Scoring

Bi-Encoder: 分开计算 Query 和 key 的向量表示，速度快，并且支持离线计算好 key 的编码。
Cross-encoder: 速度慢，但是精度高。通常用于排序阶段，因为排序阶段的数据量少，对效果要求高。

模型

Poly-encoder主要是希望在bi-encoder (速度) 和cross-encoder (效果) 之间找一个平衡点。

Candidate 端没有变化，仍然是输出一个句向量。
在Query侧中，引入了code向量，融合了「query token embedding」和「candidate text embedding」进行了 two-step attention 后交互：

  1. 首先 query text 过 bert 得到「query token embedding」： (Out1,Out2,...OutN)(Out_1, Out_2, ... Out_N)(Out1​,Out2​,...OutN​) ，维度为 [sequence_len,768][\text{sequence\_len}, 768][sequence_len,768]

  2. 初始化 mmm 个「code embedding」，即一个 [m,768][m, 768][m,768] 的 code矩阵。

    • 这个code矩阵的参数是可以学习的
    • m 和 sequence_len 无关，和时间复杂度线性相关
    • 可以看作是将原始的 sequence_len 个 「query token embedding」的信息融入到 m 个「code embedding」

  3. 然后让每一个「code embedding」去 attend 所有的「query token embedding」，即作 attention 操作，得到 m 个「query embedding」：

Q1=∑isoftmax(C1∗OiT/t,dim=1)∗OiQ_1 = \sum_i softmax(C_1*O_i^T / t, dim=1) * O_i Q1​=i∑​softmax(C1​∗OiT​/t,dim=1)∗Oi​

t 是 softmax 的温度系数
每一个「query embedding」的维度为 [m,768][m, 768][m,768] 。

  1. 最后，「candidate text embedding」去 attend 每一个「query embedding」，得到最终的交互编码，维度为 [1,768][1, 768][1,768] ，再去过线性层即可。

效果

Time：Bi-encoder 优于 Poly-encoder 优于 Cross-encoder
Test performance：Bi-encoder 劣于 Poly-encoder 劣于 Cross-encoder，但是 Poly-encoder 和 Cross-encoder 相差很小，基本在一个点以内。
