Paper: Approximate Nearest Neighbor Negative Contrastive Learning for Dense Text Retrieval, ICLR 2021

Dense Retrieval (DR) 稠密检索可以有效 approximate nearest neighbor (ANN) search 最近邻居搜索的方式得到候选样本。

在做文本匹配任务时，通常是使用 retrieval + re-ranking

  • retrieval：通过 ANN 召回多个候选样本
  • re-ranking：使用二分类继续判断召回的候选样本是否合法

在第二步骤 re-ranking 进行训练时，负样本可以自然地通过上一步召回的错误样本得到。

但是第一步骤 retrieval 训练时，最理想的情况是使用「数据集中所有其他样本 / all irrelevant ones」进行训练，但是在工程中是难以实现的。

通常可以使用 BM25 等统计学习方法进行负采样，但是实际上这些方法采样出来的样本和真实的难负样本是很不同的，见 Fig 1。

一些方法使用「other in-batch samples」作为负样本，但是一些工作指出，在 representation learning 中，这种方法其实还不如 BM25 采样出来的样本。

ANALYSES ON THE CONVERGENCE OF DENSE RETRIEVAL TRAINING

作者分析了稠密检索训练场景下，梯度拟合的决定性条件。

模型的拟合取决于负样本的信息量，直观表现为梯度和损失：梯度大的负样本能能快地降低损失，损失小的负样本无法帮助模型拟合。

通常训练到一定程度时，只有少量样本才能提供足够的信息量指导训练，而传统方法在 batch 维度上随机采集负样本，因为：

  1. batch size bbb 远小于数据集大小 ∣C∣|C|∣C∣
  2. 当前强负样本集大小 ∣D−∗∣|D^{-*}|∣D−∗∣ 远小于数据集大小 ∣C∣|C|∣C∣

导致，一个随机的 batch 内采集到强负样本的概率 p=b∣D−∗∣∣C∣2p=\frac{b\left|D^{-*}\right|}{|C|^2}p=∣C∣2b∣D−∗∣​ 近乎为 0，从而导致训练效率低。

APPROXIMATE NEAREST NEIGHBOR NOISE CONTRASTIVE ESTIMATION

Asynchronous Index Refresh

训练时损失函数为：

qqq 为每个 query， lll 为损失函数，可以为 negative log likelihood (NLL) loss，dot product similarity 。

DANCE−D_{\mathrm{ANCE}}^{-}DANCE−​ 是根据 ANN index 挑选出来的 hardest negatives 集合。

在训练时，负样本的分数是实时更新，导致 DANCE−D_{\mathrm{ANCE}}^{-}DANCE−​ 也会变化。

同时，我们维护一个 ANN index，用来指示样本的困难程度，这需要两个步骤：

  1. inference：每次更新后，使用当前的模型去推理所有的候选样本，得到分数。
  2. index：根据分数更新 ANN index

第一步需要对整个数据集的候选文本进行推理，所以成本极高，作者这里使用了「异步索引刷新 / an asynchronous index refresh」的方法：每 m 个 step/batch 后，进行一次刷新。
这里作者使用一种并行的方法，即在计算负样本分数的同时，模型还是继续进行训练的。等到 index re-encoded 之后，将新的 ANN index 交给 Trainer 进行刷新。

在这种异步的更新方式中，ANN index 的更新速度取决于资源的调度程度。在 GPU 1:1 的情况下（如 Fig 2），模型训练完整个 DANCE−D_{\mathrm{ANCE}}^{-}DANCE−​ 然后更新一次 ANN index，这样就可以弥补异步的 gap 。
