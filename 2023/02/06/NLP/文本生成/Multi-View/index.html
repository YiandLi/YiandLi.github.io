对话摘要：Multi-View Sequence-to-Sequence Models

Posted on 2023-02-06 In NLP , Text Generation
Symbols count in article: 1.4k Reading time ≈ 7 mins.

Paper: EMNLP 2020 - Multi-View Sequence-to-Sequence Models with Conversational Structure for Abstractive Dialogue Summarization
Code: SALT-NLP/Multi-View-Seq2Seq

conversation summarization 的难点：
informal, verbose and repetitive, sprinkled with false-starts, back channeling, reconfifirmations, hesitations, speaker interruptions，the salient information is scattered in the whole chat
不正式、啰嗦和重复、假动作、回话、再确认、犹豫不决、说话中断、突出的信息散布在整个聊天记录中

同时我们可以根据不同的角度拆解对话内容，但是如果仅仅考虑一个角度去分析对话则难以捕捉全部的对话内容。

所以在生成时，需要考虑对话内容的多方面结果 ，本文中作者使用了四个不同的角度 view ：

  1. topic view：当前正在谈论什么话题，和主题相关，针对谈话内容
    e.g. greetings,today’s plan, plan for tomorrow, plan for Saturday, pick up time
  2. stage view：谈话的进度，比较通用，属于一种框架/习惯
    e.g. openings, intention, discussion, and conclusion
  3. global view：将对话看作一个整体
  4. discrete view：将每句话看作是是对话中的离散片段

Model

  1. 首先利用 Sentence-Bert 抽取出不同的 view
  2. 然后使用 BART，对 view 进行编码，传递给 decoder 进行摘要生成。

Conversation View Extraction

if informative structures in the form of small blocks can be explicitly extracted from long conversations, models may be able to understand them better in a more organized way.
e.g. greetings → invitation → party details → rejection

Topic View

使用 Sentence-Bert 将对话中的每个消息进行编码，得到 $ \mathbf C= { u_1, u_2,…, u_m } $ 。

然后使用 C99 算法，根据「句间相似度」将 C\mathbf CC 切分成不同的 block Ctopic={b1,...,bn}\mathbf C_{topic}=\{\mathbf b_1, ..., \mathbf b_n\}Ctopic​={b1​,...,bn​} ，每一个 block 包含多个消息。

Stage View

e.g. introductions → problem exploration → problem solving → wrap up

作者使用 Sentence-Bert 给每个消息进行编码，然后使用 HMM 隐马尔可夫模型进行标注，这里需要事先设定 hidden stages 的数量，作者这里设置为 4 。
其实就是 hmm 的发射标签只是 one-hot stage_id ，没有预先定义并且使用标签文字信息 。

然后又可以得到不同的 block Cstage={b1,...,bn}\mathbf C_{stage}=\{\mathbf b_1, ..., \mathbf b_n\}Cstage​={b1​,...,bn​} 。

Global View and Discrete View

Global View：将所有消息看成一个整体，得到一个总的编码
discrete view：将不同消息分别看成一个 block

Multi-view Sequence-to-Sequence Model

Conversation Encoder

作者提出一种 端到端生成模型，来捕捉上一步所有 view 下抽取出的分区信息。

给定一段对话，通过 view kkk 对其进行分区，得到 Ck={b1k,...,bnk}\mathbf C_k = \{\mathbf b_1^k, ..., \mathbf b_n^k\}Ck​={b1k​,...,bnk​} 。

每一个 block bjk\mathbf b_j^kbjk​ 中有 mmm 个 token， 在每个 block 之前加上一个特殊的 token x0,jkx_{0,j}^kx0,jk​ ，用来标识不同的 block，进而得到 bjk={x0,jk,...,xm,jk}\mathbf b_j^k=\{x_{0,j}^k,...,x_{m,j}^k\}bjk​={x0,jk​,...,xm,jk​} ，

使用 BART encoder 进行编码得到：

{h0,jk,h1,jk,…,hm,jk}=E({x0,jk,x1,jk,…,xm,jk})\left\{h_{0, j}^k, h_{1, j}^k, \ldots, h_{m, j}^k\right\}=\mathbf{E}\left(\left\{x_{0, j}^k, x_{1, j}^k, \ldots, x_{m, j}^k\right\}\right) {h0,jk​,h1,jk​,…,hm,jk​}=E({x0,jk​,x1,jk​,…,xm,jk​})

所以对每个 block bjk\mathbf b_j^kbjk​ 而言， h0,jkh_{0, j}^kh0,jk​ 就是他的编码向量。

然后使用 LSTM 编码这 n 个 block 的 n 个编码向量：

Sjk=LSTM⁡(h0j,k,Sj−1k),j∈[1,n]S_j^k=\operatorname{LSTM}\left(h_0^{j, k}, S_{j-1}^k\right), j \in[1, n] Sjk​=LSTM(h0j,k​,Sj−1k​),j∈[1,n]

我们使用最后一个隐藏层状态 SnkS_n^kSnk​ 来表示这个 view 的总编码，后面计作 VkV_kVk​ 。

Multi-view Decoder

前面我们得到了了不同 view 的总编码，
但是不同的 view，因为其 block token 的插入位置是不一样的，所以其实对应的 encoder representation 也不同，
相当现在有 view_num 个 VkV_kVk​ 和 encoder representation。

作者设计了一个 multi-view decoder，让模型动态地整合不同 view 的信息。具体地，使用不同的 VkV_kVk​ 得到不同 encoder representation 的 权重，然后 线性融合。

{y1,…,yl−1}=D({t1,…,tl−1},E(C))P(t~l∣t<l,C)=Softmax⁡(Wpyl−1)\begin{aligned} & \left\{y_1, \ldots, y_{l-1}\right\}={\mathbf{D}}\left(\left\{t_1, \ldots, t_{l-1}\right\}, \mathbf{E}(C)\right) \\ & P\left(\tilde{t}_l \mid t_{<l}, \mathbf{C}\right)=\operatorname{Softmax}\left(W_p y_{l-1}\right) \end{aligned} ​{y1​,…,yl−1​}=D({t1​,…,tl−1​},E(C))P(t~l​∣t<l​,C)=Softmax(Wp​yl−1​)​

时间步 lll 时，decoder 的输入为前 l-1 时候的 token t1,t2...,tl−1t_1,t_2...,t_{l-1}t1​,t2​...,tl−1​ 。
E(C)\mathbf E(C)E(C) 表示所有 消息 的集合； WpW_pWp​ 是可学习的字典向量 。

得到权重

在 multi-view decoder D\mathbf DD 中，模型会使用 attention 机制，动态地聚合不同的 view 的信息：

uk=tanh⁡(WVk+b)αk=exp⁡(uk⊤v)∑iexp⁡(ui⊤v)\begin{aligned} u_k & =\tanh \left(W V_k+b\right) \\ \alpha_k & =\frac{\exp \left(u_k^{\top} v\right)}{\sum_i \exp \left(u_i^{\top} v\right)} \end{aligned} uk​αk​​=tanh(WVk​+b)=∑i​exp(ui⊤​v)exp(uk⊤​v)​​

W, bW,\ bW, b 是可学习参数 ，
vvv 是一个随机初始化的上下文向量（我觉得这里不应该初始化，应该用cls向量更合理）。


因为不同的 VkV_kVk​ 其实都是通过 encoder 的 attention 机制得到的，底层的上下文都相同，所以他们很大程度上表征也很接近，会弱化这里的 attention 聚合效果。
所以，作者使用一个温度系数 T\mathrm{T}T ，来拉开权重 αk\alpha_kαk​ 之间的差距：

α~k=αk1T/∑iαi1T\tilde{\alpha}_k=\alpha_k^{\frac{1}{T}} / \sum_i \alpha_i^{\frac{1}{T}} α~k​=αkT1​​/i∑​αiT1​​

加权融合

对于每个 view kkk ，他们内部 block 的 token 编码 hi,jkh^k_{i,j}hi,jk​ 都是不一样的，因为 block 划分都不同。
首先不同 view ，在 decoder 第一层使用 self-attention 得到编码 $ A^k$ 。

然后使用上述权重，线性融合得到第二层多头注意力机制的 key, value ：

A~=∑kα~kAk\tilde{A}=\sum_k \tilde{\alpha}_k A^k A~=k∑​α~k​Ak

Challenges in Dialog Summarization

  1. Informal language use
  2. Multiple participants: conversations with more speakers
  3. Multiple turns: conversations with many utterances contain more information to be processed
  4. Referral and coreference
  5. Repetition and interruption
  6. Negations and rhetorical questions: negation related issues are even more frequent
  7. Role and language change:
    the role of a speaker may shift from a questioner to an answerer;
    requiring the summarization model to dynamically deal with speaker roles and the associated language
