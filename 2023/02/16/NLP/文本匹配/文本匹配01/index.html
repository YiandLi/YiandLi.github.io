特定领域知识图谱融合方案：技术知识前置【一】-文本匹配算法
文本去重算法：Minhash/Simhash/Klongsent

场景：

  • 信息检索：可以归结为搜索词和文档资源的匹配
  • 问答系统：可以归结为问题和候选答案的匹配
  • 复述问题：可以归结为两个同义句的匹配
  • 文本数据去重
  • 智能对话

难点：

  • 语言表示不统一：一种意思多种表示方法
  • 歧义：一个单词在不同语境表示不同意思
  • 不规范/错误输入：耗子尾汁 …
  • 依赖知识：需要常识或者外部知识进行指代统一，比如奥运冠军张怡宁绰号“乒乓大魔王”等。

方法：

  • 无监督方式、有监督方式、有监督+无监督方式
  • 传统机器学习方法，深度学习模型方法
    • 传统模型 / 稀疏检索器：Jaccord，BM25，VSM，SimHash，Levenshtein
    • 深度模型 / 稠密检索器：cdssm，arc-ii，match_pyramid，mvlstm，bimpm，drcn，esim，textmatching，bert …
      • bi-encoder ：双塔模型
      • cross encoder ：交互模型
      • late interaction ：后交互模型
        很多基于bi-encoder的稠密检索器在领域内数据优于基于BM25的稀疏检索，但是在领域外的数据的表现就不及稀疏检索。
        而对于cross-encoder或者late interaction的稠密检索方法则在zero-shot场景下表现更佳。
    • 生成模型：直接让模型生成 result doc，然后转化为 doc id。
      优势：
      a) 简化了传统检索+精排的的范式，提供一个更加统一的检索范式
      b) 采用encoder-decoder的模型结构，更容易优化
      c) 不需要构建复杂的索引文件

传统方法（无监督）

Jaccord 杰卡德相似系数

两句子分词后词语的交集大小 / 并集中词语数交集数量

基于 Jaccard 衍生出 K-shingle 文本去重算法，首先设定超参数 k

  1. 对清洗后的文档进行分词处理，指定shingle长度 k，即连续出现的 k 个word。构成了一个字典
  2. 然后 one-hot 编码每个 k-grim 字典
  3. 使用这个 one-hot 字典编码文章，出现则 1，否则为 0
  4. 计算 jaccard 相似度

k 越小，词典越小，对文档语义的捕捉能力就越差。
根据经验，短文本相似度通常取 K=5 ，长文本相似度通常取 K=10 。

在海量文本，如互联网网页数据中，文档个数会远远大于词汇个数。
由于特征向量维数的增加，大大降低了特征向量之间距离的计算效率。

Levenshtein 编辑距离

「一个句子转换为另一个句子需要的编辑次数」除以「最长句子token数量」，这里的编辑次数是对称关系。
原子编辑操作包含：增，删，改/替换

MinHash 相似度

使用 jaccard / L2 等传统方法计算相似度，时间复杂度受制于：

  1. 句向量维度，呈线性增长趋势
  2. 候选文本规模，呈平方增长趋势

对于第一点，可以考虑使用近似算法，即将原本的高维句向量转化低底维句向量，牺牲一些精度来提升计算效率，即这里介绍的 MinHash。

出发点：将不同长度的文本转化成相同长度的 hash 字符串，使用这个字符串去表示文本，进而比较 hash 字符串即可表示文本的相似度。

但是传统hash存在一个问题是，相同内容的文本/word会生成相同的hash，但是相似的文本/word（可能就是一个字的差别）生成的hash会有 很大的不同 。

过程

首先给定 T 个文档， A，B，C … T，均由字典编码过的 one-hot sum 表示，即如果词典大小为 nnn ，则每个文本表示为： [i1,i2,,...,in][i_1, i_2,, ..., i_n][i1​,i2​,,...,in​] ， in∈0,1i_n\in{0,1}in​∈0,1 。
于是可以转化为一张表：

A B …
i1i_1i1​ 1 0 …
i2i_2i2​ 0 1 …
… … … …
ini_nin​ 1 0 …

MinHash 过程为：

  1. row permutation: 对 行 i1,i2,...,ini_1, i_2,...,i_ni1​,i2​,...,in​ 进行重新排序，即打乱行的顺序
  2. first non-zero row index: 对每一列，取值得到首个非零行的 row index

这样每个文档就得到了一个 row index， 然后对这两步重复 m 次，就能够将原始 n 维的向量降维到 m 。即维度从原始的 字典大小 n 降低为 操作次数 m 。

而降维操作，也被称为 Signatures : “hash” each column C to a small signature Sig(C) .

计算相似度：
对应位置相同的元素数量 除以 hash 编码总长度

降维后如果某一维度的值相同，则表示：
在每次 row permutation 后，文档 A B 的 first non-zero row index 是相同的。
这其实就表示了他们有一个共同的 token，即计算 Jaccard(A,B) 时，交集数量增加。

还可以理解为是将原始的多个维度进行合并，有点分桶的概念。
比如四个句子，降维后的某一位置为 1,1,3,3 ，则表示前面两个句子有一个共同的 word，后两个句子有一个共同的 word，并且这两个word不是同一个 word，因为打乱顺序后他们不在同一行。
但是并不能确定 前两个句子就没有 3 这个单词，因为我们取的只是最大的。即我们将两个维度压缩到了一个维度。

在通过随机行打乱后，两个集合的first non-zero row index 相等的几率等于这两个集合的 Jaccard 类似度。

每一维度其实近似为 sample A中的某一个 token，查看该 token 是否也在 B 中，有一种局部对比的效果。
而整体来看，是 sample 不同的 token（permutation操作），来近似全局的一个效果。

近似于一种抽样方法，用少数的哈希值来代替原本稀疏的特征矩阵，至于用第一个1所在的行号作为最小哈希的原因在于行号一样的集合原本的特征值都是1，这样得到哈希值也是一样，那么两个相似的文本得到的哈希在很大程度上就具有相似性

所以两个角度：

  1. 原始 one-hot 表示的话，文档向量是很稀疏的，而这种方式每一个维度上都表示了一个 的确存在的 token，可以起到降维的效果。
  2. 每一步操作相当于是一个 抽样token ，找是否共享的过程。但是抽样还是达不到原始的全局精度，所以这是一种近似，肯定更有一定损伤。

Locality Sensitive Hashing

MinHash 对句向量进行了降维，但是当数据很大时，两两匹配仍然很耗时，复杂度仍然为 O(n2)O(n^2)O(n2) 。

所以这里提出 Locality Sensitive Hashing（LSH，局部敏感哈希） ，来提高效率。

如果我们能先粗略地 将用户分桶 ，将可能相似的用户以较大概率分到同一个桶内，这样每一个用户的“备选相似用户集”就会相对较小，降低寻找其相似用户的计算复杂度，LSH就是这样一个近似算法。

做法：
将 Min Hashing 得到的降维编码进行切分，被称为切成不同的 band，每一个 band 包含 r 个 rows（words）。

分别以 band 维度进行比较，如果两个向量的其中一个或多个band相同，那么这两个向量就可能相似度较高；相同的band数越多，其相似度高的可能性越大。

LSH的做法:

  1. 对各个用户的 signature 向量(降维后的向量)在每一个 band 上分别进行分桶，同一个 band bucket 中的样本被认为是比较相似的 。
    一般一个 band 对应多个 signature id，这里可以直接匹配，或者基于 signature id 再次进行 hash 。

  2. 只需要计算所有candidate用户的相似度即可。同一个 bucket 中的两两对都是 candidate pair 。

  3. 然后把获得的candidate pairs都拿回 signature 矩阵确认它们的相似度，最终再次通过 Jaccard 判断是否真的相似。

bucket的数量越多，两个不同的数被hash到同一个bucket的概率就越小，最 后得到的candidate pairs就越精确，越接近真实相似度。但同样的bucket越多占用 的空间就越大

这样一种基于概率的用户分桶方法当然会有漏网之鱼，我们希望下面两种情况的用户越少越好：

  • False Positives: 相似度很低的两个用户被哈希到同一个桶内
  • False Negatives: 真正相似的用户在每一个band上都没有被哈希到同一个桶内

LSH准确性分析

理想情况下：两个集合的相似度和它们被分到同一个 bucket的概率的函数应该是一个阶跃函数，即 sim(S1,S2)sim(S1,S2)sim(S1,S2) 在阈值s以下时，他 们在LSH中被分到同一个bucket的概率应该是 0%0\%0% ，而 sim(S1,S2)sim(S1,S2)sim(S1,S2) 在阈值s以 上时，他们在LSH中被分到同一个bucket的概率应该是 100%100\%100% 。

「 band 数量 b 」和「每个 band 内部数量 r 」都越大，也就是signatures的长度越长，就越接近于阶跃函数，FN, FP相应就会减小；相应地，占用的空间就越大，minhash 阶段就需要更多工作。。

通俗点就是在 minhash 阶段多弄几个 minhash 函数，就可以使LSH阶段的错误更小一些。

假设两个文档的 signature 矩阵的 Jaccard similarity 为 t ，则每一个长度为 r 的 band 被认为是相似的几率为 trt^rtr （这里使用全部相同进行定义，取决于 hash func）
于是存在不相同 row 的概率为 1−tr1-t^r1−tr
所有band 都不相同的概率为 (1−tr)b(1-t^r)^b(1−tr)b （False Negative），存在最少一个 band 相同的概率为 1−(1−tr)b1-(1-t^r)^b1−(1−tr)b （False Positive）。

一般这里的相似度比较重要：

优化

两文本的 minHash 向量的任意一行相同的概率等于 Jaccard 相似度，设为 s ，一个band中有 r 个bit，一共 b 个 band，则：

  1. 两个向量任意一个 band 内所有行的值都相同的概率为 srs^rsr
  2. 两个向量任意一个 band 内至少一行不同的概率为 1−sr1-s^r1−sr
  3. 两个向量的所有band都不同的概率为 (1−sr)b(1-s^r)^b(1−sr)b
  4. 两个向量至少有一个band相同的概率为 1−(1−sr)b1-(1-s^r)^b1−(1−sr)b ，即是 candidate 的概率

向量点积

即这个点积结果就是，以 OB→\overrightarrow{\mathrm{OB}}OB 为坐标轴， OA→\overrightarrow{\mathrm{OA}}OA 的坐标。
我们称：使用 OB→\overrightarrow{\mathrm{OB}}OB 对 OA→\overrightarrow{\mathrm{OA}}OA 进行了一次映射。

这一操作，就将点 A 的坐标由2维（在新空间中）降到了1维。

Johnson–Lindenstrauss 引理指出，在欧式空间中的若干点，经过相同的映射后进入新的空间，它们仍然会保持原来的相对位置。

那么根据数据找到合适的点 B 呢，随机生成。可以基于高斯分布，生成B点的横坐标和纵坐标。

随机投影的降维——N维到K维

假设有 M 条数据，维度非常高，为 N。

我们通过高斯分布随机生成 K 个 N 维向量，用于降维。然后逐个点乘，得到新的坐标即可。

这种降维方法叫做“随机投影法”。随机投影法将帮我们把这种数据压缩为K维 (K远小于N)。

总的来说，K 越大，降维后的数据，保留的信息越多，数据之间的相对位置越“保真” 。

随机投影降维的简化 —— 基于随机投影的局部敏感哈希

随机投影法可以把高维数据映射到新的空间中。由于特征维度大大降低了，聚类、分类、搜索等任务的计算量也大大降低了。不过呢，人心不足蛇吞象，我还想进一步降低计算量。

新的坐标空间中，还得用 float 类型存储数据，所以还要进行 float*float 的操作，还是比较慢。

操作

于是，在将数据映射到新空间后，

  • 将落在坐标轴负轴的维度(该维度取值为负数)，统一赋值为0，表示数据与对应随机向量夹角大于90度。
  • 将落在坐标轴非负轴的维度，统一赋值为1。这样原始数据就被映射到了一个离散的新空间里。

用途

在原来空间中比较接近的数据点，在新的空间中，在一些维度上可能是相等的，相当于各个维度上进行 分箱/桶 操作。

在搜索文档的时候，我们对查询语句进行hash，然后把 各个维度上、桶的编号相同的文档 召回，最后进行精排，可以实现快速而高质量的搜索。一个基于分桶构建的倒查索引可以帮助我们实现这样的操作。

随机超平面哈希

与随机投影hash的主要区别是，随机超平面hash的编码结果是-1和1组成的串——与随机向量点积为负数，新空间中该维度取值-1；与随机向量点积为非负数，新空间中该维度取值为1。

能更好一些，在文本分类、聚类任务中表现更好。

simhash 优势

（1）simhash是一种将文本数据映射为定长二进制编码的hash算法，用NLP的说法就是一种文本表示模型 ；
（2）基于simhash的文本相似度计算，就是用文本的simhash值作为特征向量，来计算文档之间的相似度或者距离；
（3）基于simhash的文本去重，就是使用基于simhash表示来计算文本相似度，进而发现重复文档并删除。

是一种文本表示模型，即用一个 定长的数值向量 表示文本。

常见的弊端：

  1. TF-IDF：要求首先遍历一遍所有文本来的得到IDF，计算量比较大；同时精度也比较差。
  2. 句向量：更加关注语义，而不是字面表示信息

计算过程

基于hash算法将词语映射为较短的编码
然后使用随机超平面hash的做法得到最终的词语编码。
将每个词语的编码加起来。
采用随机超平面的离散化方法，得到文本的最终表示。

  1. 将词语映射为定长二进制编码
    这里区别于 on-hot，可以使用 [0,1,0,0,1] 这种表示方法。
    二进制向量长度为 n，最理想的情况下则可以表示 2^n 个词。

  2. 将词语的二进制编码再转换一下，将 0 转化为 -1 。
    将 0 转换为 -1 的目的，是将映射后的词语放置在整个空间中，而不是某一个象限，这样可以让数据点分布得更均匀一点。
    转化完仅包含 1/-1 的向量之后，还可以再乘上单词的权重，这个权重可以由 tf-idf 得到，对一个单词，我们可以得到长度为二进制向量长度的列表 [weight，-weight...weight] 。

  3. 计算文档的初级编码：将句子中所有词语的编码加起来，就得到了句子的编码。

  4. 计算文档的最终编码：将句子编码的非正数元素转换为0，正数元素转换为1 ，就得到了句子的最终编码，这就是句子的simhash编码。

双塔模型类

孪生神经网络(Siamese Network)

Paper：《Learning a Similarity Metric Discriminatively, with Application to Face Verification》

传统的分类问题中，类别数目固定，且每类下的样本数也较多。比较适用于有监督学习的方式，即直接学习得到类别的表征。
小样本的分类问题，类别很多（或者说不确定），然而训练样本的类别数较少的分类任务（比如人脸识别等），可以使用基于度量学习的方式。

思路：learn a similar metric from data。核心思想是，寻找一个映射函数/编码器，能够将输入样本转换到一个特征空间，通过一些简单的“距离度量”（比如欧式距离）来表示向量之间的差异，最后通过这个距离来拟合输入样本之间的相似度差异（语义差异）。

双塔结构，有两个结构相同，且共享权值的子网络。分别接收两个输入 X1X_1X1​ 与 X2X_2X2​ ，将其转换为向量 Gw(X1)G_w(X_1)Gw​(X1​) 与 Gw(X2)G_w(X_2)Gw​(X2​) ，再通过某种距离度量的方式计算两个输出向量的距离。

缺点：由于是双塔结构，两个文本在编码时候没有交互，没有充分利用到两个文本 相互影响 的信息。

失效的 Cos

如果使用预测的函数进行训练，即让正样本对的cosine相似度尽可能大，而负样本对的相似度尽可能小，直接优化这种目标的实验结果往往特别差。

原因：使用到的文本匹配训练集中的负样本都是困难样本 hard negative，常见是语义不同但是字面上有较多重合。
而一般训练时我们是让这种样本对的相似度为 0，这和事实不符合。
虽然这两个句子语义不同，但依然是“相似”，相似度不至于0甚至-1那么低，如果强行让它们往0、-1学，那么通常的后果就是造成过度学习，从而 失去了泛化能力，又或者是 优化过于困难，导致根本学不动。
这一点，其实和对比学习也挺像的，如果 负样本是hard negative（字面重叠度非常高但语义却不同），强行通过对比学习将他们拉远，或许也会造成过度学习的后果。

缓解这个问题的一个方法是给负样本对设置更高的阈值，比如：

t⋅(1−cos⁡(u,v))+(1−t)⋅max⁡(cos⁡(u,v),0.7)t \cdot(1-\cos (u, v))+(1-t) \cdot \max (\cos (u, v), 0.7) t⋅(1−cos(u,v))+(1−t)⋅max(cos(u,v),0.7)

即只要负样本对的相似度小于 0.7，就不再优化了，从而不容易过度学习。但是这个阀值很难选择。

mismatch 的 SentenceBERT

尽管 SentenceBERT 的训练和推理存在不一致问题，但是实验证明他还是比上述使用 cosine similarity 作为训练目标的方法效果要好很多。

一般情况下 ，尽管负样本是困难样本，总体而言正样本的相似度还是要大于负样本的相似度的，即 $\mathrm{sim}_p(u-v) > \mathrm{sim}_n(u-v) $ ，即 ∥u−v∥p<∥u−v∥n\|u-v\|_p < \|u-v\|_n∥u−v∥p​<∥u−v∥n​ 。

所以可以假定在距离表征空间中，「正样本对的差值表征」分布在距离原点近的地方，而「负样本对的差值表征」分布在距离原点远的地方。
可以把 ∥u−v∥p\|u-v\|_p∥u−v∥p​ 的分布看成一个小球， ∥u−v∥n\|u-v\|_n∥u−v∥n​ 的分布看成一个大球。
接下来，训练目标就是将这两个聚类之间的差异拉的更远，即使得 ∥u−v∥p\|u-v\|_p∥u−v∥p​ 依然保持更小，负样本对的 ∥u−v∥n\|u-v\|_n∥u−v∥n​ 保持更大，即维持在平面上。

然后在分类时，使用线性分类器做二分类，二分类可以看成是和某一个向量之间的夹角 cosine 大小。
直觉上做不到区分两个球面的功能。而这里文章取 abs，也只是在空间中进行了空间缩小。类比二维空间， 就是全都映射到了第一象限了，可以理解成通过约束原本的表征空间来简化模型学习难度，并不能决绝问题，所以这里提出猜想：

简单起见，我们考虑二维情况，球盖变成圆弧，假如正负样本的 ∣u−v∣|u−v|∣u−v∣ ，分别集中在 r=1r=1r=1 和 r=2r=2r=2 的圆弧上，角度范围都是 [0,π/2][0,π/2][0,π/2]。
这时候Dense依然是不能直接将它们分开，但是经过训练后，正样本可能就集中在 r=1,θ∈[0,π/8]r=1,θ\in[0,\pi/8]r=1,θ∈[0,π/8] 的圆弧上，负样本集中在 r=2,θ∈[π3/8,π/2]r=2,θ\in[\pi3/8,\pi/2]r=2,θ∈[π3/8,π/2] 的圆弧上，这时候Dense就能区分开了。
我们希望的是原本分散开来的点聚为一簇，你觉得原本分散在整个球面上的点聚为一簇容易，还是分散在某个小球盖的点聚为一簇容易？

至于 u,v 的拼接，苏神认为是用来解决各项异性的问题：由于Dense层的类别向量是随机初始化的，所以相当于给了 u,v 一个随机的优化方向，迫使它们各自“散开”，远离当前的各向异性状态。

问题

SentenceBert 有效的原因是初始阶段就有聚类倾向，而标签训练只是强化这个聚类倾向信息，所以“初始阶段就有聚类倾向”就显得相当重要，它意味着其效果比较依赖于初始模型。

同时训练和预测不一致导致，训练时 loss 和 valid 指标不一致。

推理

当使用 Spearman系数 进行评估时，只依赖于预测结果相对顺序，不依赖于阈值，直接输出排序后的样本序列即可。

但是在使用 Acc / f1 进行评估时，必须确定一个阈值，将预测结果大于这个数的预测结果视为正、小于则为负，然后才能计算指标。

和对比学习的区别

样本格式：
- 标准 simcse 的样本格式为 positive pairs ； 或者 (anchor, positive, negative) 的形式。
- CoSENT 的样本格式为 ( sent1, sent2, 1/0 ) 的格式，即标注好的正负样本对。只依赖于 标签相对顺序 的损失函数，不包含随机采样batch内的其他样本来构建负样本的过程。

思想：
- SentenceBERT 强调的是正负样本对的 差异表征 的聚类关系，同时也是一个二分类问题。
- 对比学习强调的是：减少正样本的距离，增大负样本对的距离。盲目增大负样本对的距离容易过度训练。
- CoSENT 强调的是一种顺序关系，即样本对之间相似度的前后位置关系，有点像 lean 2 rank 的感觉。
也可以将它理解为对比学习，但它是“样本对”的对比学习，而不是像SimCSE的“样本”对比学习，也就是说，它的“单位”是一对句子而不是一个句子。

偏移模型空间

transformer 表示模型的问题


未经微调的BERT模型在文本相似度匹配任务上表现不好，原因为「各向异性」：

  1. dot product 度量公式不匹配 ：Transformer 的词向量并不是均匀地在分布空间中的，而是呈锥形。 「词向量分布的各向异性」也导致了「句向量分布的各向异性状态」。
    所以在使用 bert encoder 计算相似度时，使用 cos 或者 dot 无法很好地衡量出两个句子的相似度，因为 空间呈现锥形分布 ，不是基于一个标准的正交基得到的。

  2. 分布在不同的空间 ： 高频词都靠近原点，而低频词训练不充分远离原点，并且分布很稀疏，导致该区域存在语义定义不完整的地方（poorly defined） 。
    相当于这两种词处于了不同的分布空间中，那高频词和低频词之间的相似度就不再适用。

针对低频和高频分布的情况，会存在这样的问题: 假设有一对词“讨厌”、“反感”。如果在训练语料中，反复出现“讨厌”，而“反感”出现的次数极少，则会导致这两个词的向量距离很大。

PromptBERT 中还给出了一个直观的判断各向异性的度量方法：
在通用的语料中，如果随机抽若干句子，计算它们之间的平均余弦相似度

1n2−n∣∑i∑j≠icos⁡(M(si),M(sj))∣\frac{1}{n^2-n}\left|\sum_i \sum_{j \neq i} \cos \left(M\left(s_i\right), M\left(s_j\right)\right)\right| n2−n1​∣∣∣∣∣∣∣​i∑​j=i∑​cos(M(si​),M(sj​))∣∣∣∣∣∣∣​

如果越接近于1，说明异向性越严重，如果接近于0，说明同向性很好。

BERT-Whitening

一个线性变换媲美BERT-flow

苏神发现尽管BERT-flow的思路没有问题，但只要一个线性变换就可以达到相近的效果，flow 模型并不是十分关键。

一般认为向量点积除以模长等于两个向量之间的夹角 cosine ，所以用来表示相似度，但是这仅仅在标准正交基中成立。

而 Bert使用这种方法求相似度效果不好，是因为其句向量分布的坐标系不是标准正交基。

如果一个向量的集合满足各向同性，那么我们可以认为它源于标准正交基。而如果这里不满足各向同性，则我们就重新选择正交基，让空间各向同性一些，然后再用 cosine 相似度。
所以这里想要给集合重新选择基底。在给向量集合选择基底时，会尽量地平均地用好每一个基向量，从统计学的角度看，这就体现为每个分量的使用都是独立的、均匀的。

特征工程

  1. 数据清洗，删除一些停用词，有利于使用无监督学习进行粗召
  2. 数据增强：随机删除，随机替换， 随机交换，注意不要改变语义，同时短句改动语义影响大
  3. 闭包扩充（标签传递），注意要保持原来的分布，否则会过拟合！
    • 上采样：if A=B and B=C, then A=C
    • 下采样：if A=B and B!=C, then A!=C
    • 对偶数据增强：A，B 对调
  4. 负样本下采样，让正负样本比例达到 1:1

可以用到的基于图的特征工程（可以用于粗排，构造负样本，分析数据等）：
将pair 构造成无向图的形式，然后提取：

  1. 频数特征：q1、q2在连通图中出现的频数
  2. 频数特征组合：q1、q2频数特征之间的差、 乘、除、最大、最小等
  3. 交集特征：q1，q2 公共邻居（交集）的数量
  4. 并集特征：q1、q2在连通图中的邻居节点并集中元素个数
  5. 交并集组合特征：q1、q2交集特征和并集特征之间的差、 乘、除、最大、最小等
  6. Max K-core特征：q1、q2在连通图中的最大核数

同时可以基于图来发现潜在正例/负例 ； 或者使用训练集构造图，使用dev集来评估图是否有效：

  1. 首先根据训练集合构造图，会得到多个连通子图
  2. 测试集正例发现：在同一子图的节点必然相似，其实对应上述正采样
  3. 不相似子图对发现：负例 q1，q2 如果在两个不一致的子图中，则这两个子图必然不相似
  4. 测试集负例发现：属于第三步不相似子图对的两个节点，则必然不相似，成为负例对
