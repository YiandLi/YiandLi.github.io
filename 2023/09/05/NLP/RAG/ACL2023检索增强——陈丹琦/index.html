检索 query 的长度： 并不是越长越好，即最好不要将 history 全部作为 query ，可以调整下长度。实验中只用最后一句话进行检索，效果更加好。这个不同模型，不同场景都不一样，自己实验下比较好。

在 when 的问题上，通常做法为检索一次然后拼接在文本前面，但是这样能带来的知识有限 ，所以可以选择在需要的地方进行检索。即 Retrieving more frequently 。
如何判断什么时候该检索文本 ：可以选择困惑度或者论文中 every n tokens。

推理

将 Encoder-Decoder Transformer 中的其中几层 decoder 改成 Retro-block 。

Retro-block 是将原本的 decoder cross-attention 改为 chunked cross-attention layer Retro⁡(H,E)\operatorname{Retro}(H, E)Retro(H,E) ：

  1. Encoding retrieval neighbors：首先使用 Encoder 对 RET⁡D(Cu)\operatorname{RET}_{\mathcal{D}}\left(C_u\right)RETD​(Cu​) 编码得到 EEE 。
    这里拼接了原始 chunk representation HuH_uHu​ 进行 self-encoding：

  1. 然后上一层的输出 HHH 还是进行 masked self-attention ， 得到 ATTN⁡(H)\operatorname{ATTN}(H)ATTN(H)
  2. Chunked cross-attention / CCA：然后使用 H 去 query EEE ：

Retro⁡(H,E)≜FFW⁡(CcA⁡(Atta ⁡(H),E))\operatorname{Retro}(H, E) \triangleq \operatorname{FFW}(\operatorname{CcA}(\operatorname{Atta~}(H), E)) Retro(H,E)≜FFW(CcA(Atta (H),E))

KNN-LM - What:Token


是用前缀作为检索 query 和 key，去检索，然后通过结果，得到 next token set，然后聚合起来。

Pros: More fine-grained; Can be better at rare patterns & out-of-domain Can be very efficient (as long as kNN search is fast)

Cons: Datastore is expensive in space: given the same data, # text chunks vs. # tokens
No cross attention between input and retrieval results

Adaptive retrieval of chunks

FLARE : Forward-Looking Active REtrieval augmented generation

PAST WORK:
retrieve information multiple times while generating outputs .
retrieve documents at fixed intervals using the previous context as queries .

本文推出 主动检索增强机制 active retrieval augmented generation

定义：
short-form generation : single-time retrieval-augmented LMs
long-form generation : retrieve multiple times throughout generation 需要在生成过程中进行多次检索

passively retrieve : utilize the past context (e.g., previous sentences or
tokens) to retrieve additional information -> not accurately reflect what LMs intend to generate in the future
actively retrieve : actively decides when and what to retrieve throughout the generation process

Method

when to retrieve : 当 LLM 生成的 token 概率比较低的时候
what to retrieve : 先让模型生成一个 temporary next sentence ， 然后使用这个 temporary next sentence 来检索内容，最后根据检索内容重新生成 true next sentence 。

formulation:
input : xxx
a document corpus : D={di}i=1∣D∣\mathcal D=\{\mathbb d_i\}_{i=1}^{|\mathcal D|}D={di​}i=1∣D∣​
generated / target answer : $\mathbb y = [w_1,w_2, …, w_m] $
retrieval model function of query qqq : Dq=ret(q)\mathcal D_q = ret(q)Dq​=ret(q)
LM inference based on concatenated input xxx and retrieved documents Dq\mathcal D_qDq​ : y=LM⁡([Dq,x])y = \operatorname{LM}([\mathcal D_q, x])y=LM([Dq​,x])

作者介绍了两种方法

  1. The first method prompts the LM to generate retrieval queries when necessary while generating the answer using retrieval-encouraging instructions, denoted as FLAREinstruct\text {FLARE}_\text{instruct}FLAREinstruct​
  2. The second method directly uses the LM’s generation as search queries,
    denoted as FLAREdirect\text {FLARE}_\text{direct}FLAREdirect​

Method 1. FLARE with Retrieval Instructions

受到 ToolFormer 启发，作者想让模型生成需要检索的内容 [Search(query)]\text{[Search(query)]}[Search(query)] ，比如：
“The colors on the flag of Ghana have the following meanings. Red is for [Search(Ghana flag red meaning)] the blood of martyrs, … “

构造 instruction prompt 时候，the search-related instruction and exemplars 组成了 skill 1 ，the instruction and exemplars of the downstream task 组成了 skill 2 。

模版为：

当模型生成 [Search(query)]\text{[Search(query)]}[Search(query)] ，我们停止生成，然后去检索得到相关文档，将检索文档放置在 input 前面继续进行生成；直到下一次检索才更新 prepended documents 。

实验发现模型生成的 [Search(query)]\text{[Search(query)]}[Search(query)] 还是较少，所以这里将 token [\text{[}[ 的概率自动提高两倍。

还发现如果在 [Search(query)]\text{[Search(query)]}[Search(query)] 后面继续生成 ，会产生 answer disruption ，所以检索完成后，自动将模型生成的 [Search(query)]\text{[Search(query)]}[Search(query)] 删除，然后将 token [\text{[}[ 的概率变成一个负数保证不会再生成该 token（for a few subsequent time step）。

Method 2. Direct FLARE

Confidence-based Active Retrieval

让模型一句一句进行生成，

  1. 先生成一个句子 s^t=LM([x,y<t]\hat{\boldsymbol{s}}_t=\mathrm{LM}\left(\left[\boldsymbol{x}, \boldsymbol{y}_{<t}\right]\right.s^t​=LM([x,y<t​]
  2. 然后判断这一句话是否可信，判断标准为，句子中是否有一个 token 的生成概率低于一个 threshold θ∈[0,1]\theta\in [0,1]θ∈[0,1]
  3. 如果可：则 accept，继续生成下一句；
    如果不可信：则用这句话进行检索 (trigger retrieval and formulate queries)，然后让模型根据检索内容和上文重新生成句子 ( re-generate s^t\hat{\boldsymbol{s}}_ts^t​ )。

Confidence-based Query Formulation
上面存在一个问题 – a risk of perpetuating error，即如果模型生成的 s^t\hat{\boldsymbol{s}}_ts^t​ 中有错误，检索出来的结果也是错的 (retrieve irrelevant information)，那么 RAG(retrieval augmented generation) 的内容肯定也是不正确的 (mislead future generations) ，尤其是错误生成的一些实体导致检索出错误实体相关的文档。

为了解决这个问题，作者提出两个改进方案：

  1. Masked sentences as implicit queries：设置一个阀值 β∈[0,1]\beta \in [0,1]β∈[0,1] ，将 s^t\hat{\boldsymbol{s}}_ts^t​ 中低于阀值的 token 进行 mask 。

  2. Generated questions as explicit queries：对 s^t\hat{\boldsymbol{s}}_ts^t​ 中每一个低于 β\betaβ 的低置信实体 z\mathbb zz 生成一个专属问题，然后逐个问题进行检索。将所有检索到的文本拼接到 input 前面，生成答案。

Adaptive retrieval of tokens / Adaptive kNN-LM

属于 KNN-LM 的后续工作

Efficient Nearest Neighbor Language Models：在模型自信时直接生成next token，在困惑度高时做next token的检索
Neuro-Symbolic Language Modeling with Automaton-augmented Retrieval：认为当前时间步检索到的 neighbor ，在后续时间步上也很有可能被检索到。于是采用无监督的方式在 datastore 的基础上加入了一个带权重的容量有限的自动机 a weighted finite automaton (WFA) ，即加上了 memory 机制，减少检索时间。

Pros: Most effective for entity-centric tasks & space-efficient

Long-range Language Modeling with Self-retrieval

推理过程：在长文本生成过程中，模型进行自举检索

  1. LM 编码当前 a recently generated text chunk ，得到 query representation
  2. 然后检索出早先生成的 earlier chunks in the document
  3. 然后使用 retrieved chunks representation 融入 LM 继续进行生成 next target chunk

目标： retrieve chunks that increase the probability of the next chunk

Problem setting : Retrieval-Pretrained Transformer

a chunk-wise retrieval-augmented LM

输入 $ (x_1, x_2, . . . , x_L)$ 被切分为长度为 mmm 的 l=L/ml=L/ml=L/m 个块 ：$ \mathcal C = (c_1, c_2, . . . , c_l)$ 。

对于每一个 chunk query，模型检索出前文中 C<i\mathcal{C}^{<i}C<i ，最多 K 个 context chunk： R(cq)⊂C<i=(c1,c2,…,ci−w)\mathcal{R}\left(c^{\mathbf{q}}\right) \subset \mathcal{C}^{<i}=\left(c_1, c_2, \ldots, c_{i-w}\right)R(cq)⊂C<i=(c1​,c2​,…,ci−w​) 。

www 为一个窗口，表示在 max seq len 内部的块，就不用检索了，直接作为上文在推理时使用。

Model

模型为 decoder-only model ，主要创新点在两块：

  1. 使用 CCA 将检索到的 chunk 融入自回归的推理过程中 。
  2. 训练自回归模型的同时，也会训练检索起，让其检索到最优的 chunk 。

将 decoder model 分为两部分，下半部分用于编码检索内容，上半部分为 RETRO (Borgeaud et al., 2022) 的 Chunked Cross-Attention (CCA) ，即一个插入 CCA layer 的 decoder layer。

如图，输入 45 token，被分成了 9 个 size=5 的 chunk，max_seq-len=15 / window size www = 3 ，即三个 chunk 为一组，进行编码，得到 token 表征。

然后上层进行 CCA，即使用上述方法得到的表征，query 为长度为 m 的目标 chunk token 表征 ，key，value 为检索出来的 chunk 的 token 表征。
