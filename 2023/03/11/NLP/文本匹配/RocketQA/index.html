RocketQA v1

经典的信息检索模型分为两个部分： 对偶模型 Dual-Encoder（Retriever） 和 交互模型 Cross-Encoder（Re-ranker） 。

传统的 Dual-Encoder（Retriever）使用 in-batch contrastive learning 进行训练，但是存在一些问题：

  1. 训练场景和预测场景中样本数量存在较大差异：
    训练时，每个样本的候选文本数量为 batch size，真正使用时候选文本数量为 dataset size，存在 mismatch 问题。
    同时有其他论文提出，within-batch 中的负样本并不是 hard negative sample，带来的监督信号较低，导致梯度较小，训练不充分，训练效率低。

  2. 数据集中存在大量漏标注的正确答案：
    在 MSMARCO 数据集中，候选段落的总数为 880 万，但每个问题平均只标注了 1.1 个正确答案。
    研究人员发现，在使用对偶模型检索出的首条结果中，70% 的错误结果其实是漏标的正确答案。这种情况下，构造训练数据中的强负例时很容易引入假负例（false negative），给模型训练带来负面影响。

  3. 相对于开放域全集，人工标注训练数据的规模小、成本大：
    和真实场景相比，训练数据还是极为欠缺的，导致模型泛化性差。想要增大标注数据的规模和质量，需要很高的人工成本。
    为了扩充训练集合，需要一些数据增强的技术。

针对上述三个问题，本文提出 RocketQA，提出 跨批次负采样（cross-batch negatives）、去噪的强负例采样（denoised hard negative sampling）与数据增强（data augmentation） ，三个优化策略来优化 Dual-Encoder 的训练过程。

跨批次负采样（cross-batch negatives）

针对多GPU训练场景，在训练时，每个 GPU 计算出对应 batch 的文本向量之后，将文本向量传递给其他 GPU。
于是每个样本可以得到所有 GPU 上的候选文本的向量表示，然后将其作为负样本进行训练。

本质上，是使用 GPU并行技术 达到扩充 batch 的效果。

去噪的强负例采样（denoised hard negative sampling）

传统训练方式将未标注的所有其他候选文本均看作负样本，有很大概率会引入假负例，即将漏标注样本也看成了负例。

本文中，作者先训练一个 cross-encoder 打分模型，然后使用这个打分模型给每个负样本进行打分，如果分数很高，则说明这个样本有可能是漏标注的候选文本，在后续训练时将分数高的伪负样本直接剔除。

这个做法利用了Cross-Encoder对问题和文档的内容更为敏感的特性，能够较好的去除伪负例的影响，达到负样本降噪的效果。

数据增强（data augmentation）

交互模型可以过滤强负例中的噪声，也可以用来选取未标注的正确答案。

当引入大量无标注的问题时，便可以利用交互模型以极低的成本得到大量弱监督数据，进一步扩充训练集合。

这里使用新的无标注的 query集合 在原本的 key集合 中进行匹配，选择 cross-encoder 相似度打分高的 key 作为正样本，选择相似度低于一定阀值的作为负样本。

训练流程

训练过程像一个多级火箭，所以叫RocketQA。

  1. 使用原始数据集 DLD_LDL​ 训练Dual-Encoder MD(0)M_D^{(0)}MD(0)​ 。

  2. 使用原始数据集 DLD_LDL​ 训练Cross-Encoder MCM_CMC​ 。
    训练过程中的正例即 DLD_LDL​ 的正例；每个样本 q∈QLq\in Q_Lq∈QL​ 的负例则是在 MD(0)M_D^{(0)}MD(0)​ 检索结果的 top-k negative 中随机采样。

  3. 使用第二步训练好的 MCM_CMC​ 选择强负样本。使用降噪后的数据训练新一版的Dual-Encoder MD(1)M_D^{(1)}MD(1)​ 。
    具体过程为，让 MCM_CMC​ 给 MD(0)M_D^{(0)}MD(0)​ 召回的 top 结果进行打分，来选择强负例。

  4. 在无监督数据集 DUD_UDU​ 中，让 MD(1)M_D^{(1)}MD(1)​ 去召回，然后让 MCM_CMC​ 打分，得到伪标注数据。
    同时使用 DLD_LDL​ 和 DUD_UDU​ ，训练最终版的Dual-Encoder MD(2)M_D^{(2)}MD(2)​ 。

RocketQA v2

“检索-重排” 的方法需要以 迭代 的方式训练两个模型，即在固定一个模型的情况下优化另一个，这会影响模型的效率。
作者试图通过对两个模块联合训练实现相互促进的效果。

  • 设计了 动态列表式蒸馏方法 Dynamic List-wise Distillation 来统一两个模型的训练方式；
  • 设计 合理的损失函数 来实现联合训练，实现相互促进的效果；
  • 应用 混合数据增强策略 为模型训练构造多样且高质量的训练样本提升模型性能。

List-wise

基于listwise的训练方式，对同一个query下的所有passage样本相似度打分进行优化，其通常包含 1 个正样本和 N-1 个负样本,其中 N 为一个query qqq 对应的 passage 样本总数。

Dynamic List-wise Distillation

联合训练的挑战在于：现有的稠密检索模型和精排模型的训练方式不同，

  • 检索模型采用 批次内负采样（in-batch negative sampling） 的训练方式
  • 精排模型采用基于 pointwise / pairwise 的训练方式

为了能够联合训练，作者使用统一的 list-wise 训练方式；有了统一的训练方式之后，作者使用蒸馏的方式同时动态更新检索模型和精排模型的参数。

Hybrid Data

下一步需要对训练数据构造方式也进行统一，即为每个 query 构造 key passage 的互选列表，这里使用了 动态数据增强 的策略构造多样的训练数据。
这里有两种增强样本类型：random sampling un-denoised instances 和 denoised sampling instances

represent the distribution of all the passages

对每个 query qqq ，首先使用 检索模型 召回 top-n 个「高置信候选样本集合」。

  • un-denoised instances：随机在「高置信候选样本集合」中抽样 hard negatives，这样可以和「原始候选集合」的分布保持一致。
  • denoised instances：使用 精排模型 给「高置信候选样本集合」中的样本打分，首先将分数低的 weak negatives 进行剔除，然后选择 hard negative 和 positive 。

这里我有个问题，剔除后剩余的样本如何划分正样本和强负样本呢？好像只能够用阀值了呢，设定两个阀值，高于 high-thres 的作为 positive，低于 low-negative 的作为负样本。
作者说：取得分最高且大于给定阈值的passage作为去噪正样本，并从前往后取 n−1 个打分小于阈值的文档作为去噪负样本，从而构造出弱监督的去噪样本。
这里 un-denoising 部分是随机抽样 hard negatives，难道不会抽取到 de-noising 部分的 positive 导致相互矛盾吗？这里还是不要去 sample positive 了 …

Train Process

首先，使用训练好的 RocketQA 训练的 retriever 和 re-ranker （可以使用其他版本模型替代）构造数据，得到 Hybrid Train Data，然后使用动态蒸馏的方法继续训练得到 v2 model 。
