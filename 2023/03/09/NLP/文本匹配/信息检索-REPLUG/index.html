信息检索：REPLUG

Paper：REPLUG: Retrieval-Augmented Black-Box Language Models

Retrieval-augmented Models，即使用检索到的外部语料增益 PLM 。
具体地，有两种方式，

  1. 「增益 PLM 输入」，一般需要使用反向传播的方式对 PLM 模型进行训练，即让 PLM 更加适配/适应 retriever 的输出。

    1. 使用 retriever model 根据 query 在知识库中检索出相关的文本
    2. 然后将 query 和检索出来的文本输入到一个 PLM 中，最终得到输出文本

  2. 「修改 PLM 输出分布」，需要知道 PLM 每一个时间步生成的 token representation。

    1. 使用 retriever model 根据 query 在检索出一个 token set
    2. 在 PLM 生成时，使用这些 token 修改生成 token 的分布

上面的方法中，我们都需要知道 PLM 的完整结构，即需要使用“白盒“模型，但是现在模型越来越大，微调起来很困难，并且很多模型都没有开源，无法使用上面的方法。


所以本文作者提出 REPLUG (Retrieve and Plug)，一种新的 Retrieval-Augmented Language Models Generation 方法。

此外，作者还提出一种有监督训练框架 REPLUG LSR (REPLUG with LMSupervised Retrieval) ，将 PLM 看作一种黑盒模型，用来提供监督信号训练 retriever model，让 retriever model 更加适配 PLM。

REPLUG (Retrieve and Plug

  1. 给定一个 query，首先让 retrieval model 从外部语料中检索出相关的文档。
  2. 然后将不同的文档拼接到 query 前面，送入到一个 black-box LM 中。
    由于长度限制，这里分别将不同的相关文档和 query 进行拼接，然后多次送入到 PLM 中得到不同的输出。
  3. 作者还设计了一种新的 ensemble scheme ，使用相同的 black-box LM 编码不同的拼接输入，并且根据不同的的结果得到最终的输出。

Document Retrieval

给定一个 query xxx ，首先让 retrieval model 从外部语料 DDD 中检索出相关的文档 d∈D′d\in D'd∈D′ 。
这里 retrieval model 是一个 dual encoder architecture 的稠密检索器。
即用同一个 encoder 同时对输入 query xxx 和知识库中的候选文档 ddd 同时编码，得到 E(x), E(d)\mathbf E(x),\ \mathbf E(d)E(x), E(d) ，使用 cosine 相似度召回 top-k documents：

s(d,x)=cos⁡(E(d),E(x))s(d, x)=\cos (\mathbf{E}(d), \mathbf{E}(x)) s(d,x)=cos(E(d),E(x))

这里作者首先离线计算了所有 document 的表征，然后使用 FAISS 索引，来提升匹配效率。

Input Reformulation


召回得到的 top-k 个文档中包含了query xxx 的信息，并且能够帮助 PLM 做出更好的生成/预测。

可以选择将所有的文档和query拼接在一起，组成一句话，但是这样句子太长了，超出模型 max seq len。

于是作者分别将每个文档 d∈D′d\in D'd∈D′ 和query xxx 进行拼接 ，得到新的 kkk 个拼接文本 d∘xd \circ xd∘x ，然后依次送入到 PLM 中。
在每个时间步中/生成每个token时，PLM 会生成 k 个不同的 token probability，使用加权集成的方式，将生成的 k 个 probability of next token 集成起来，来决定当前时刻生成的 token probability：

p(y∣x,D′)=∑d∈D′p(y∣d∘x)⋅λ(d,x),p\left(y \mid x, \mathcal{D}^{\prime}\right)=\sum_{d \in \mathcal{D}^{\prime}} p(y \mid d \circ x) \cdot \lambda(d, x), p(y∣x,D′)=d∈D′∑​p(y∣d∘x)⋅λ(d,x),

然后得到一个 token 添加到队尾；每个时间步骤模型的k个输入中，只有最前面的 document d∈D′d\in D'd∈D′ 不一样，之前时间步生成的 token 都是一样的。
尽管需要 k-pass 才能生成答案，但是以前的工作是把所有的 k 个文本和 query 拼接在一起，然后one-pass，其实时间上是差不多的。

REPLUG LSR: (REPLUG with LM-Supervised Retrieval)

冻结 PLM，使用 PLM 提供监督信号，去训练 retriever model，让检索组件适配语言模型。

优化的目标是：
让 PLM 生成「答案文本的困惑度」最小。即检索器检索得到的文档应该可以让模型更加自信地生成答案文本，即让答案文本的困惑度变小。
同时检索器「检索出真正有效文档的概率」应该和「PLM 生成答案文本的概率」成正比，和「PLM 生成答案文本的困惑度」成反比。


具体分为四个步骤：
(1) retrieving documents and computing the retrieval likelihood ：检索器检索文档，并且根据相似度得到文档的「Retrieval Likelihood」
(2) scoring the retrieved documents by the language model ：让 PLM 给每个检索出来的文档打分，得到文档的「LM likelihood」
(3) updating the retrieval model parameters by minimizing the KL divergence between the retrieval likelihood and the LM’s score distribution
最小化「Retrieval Likelihood」和「LM likelihood」之间的 KL 散度，更新检索器。
(4) asynchronous update of the datastore index：更新文档索引

语言模型的困惑度

生成模型可以自然地估计句子的概率，越好的语言模型对于我们人类给出的一句通顺流畅的话，会给出越高的概率，这样的语言模型困惑度也就越小。
可以理解成为目标句子中各个 token 的概率的乘积，乘积越大表示模型生成这一句话的概率越高，即困惑度越小。

检索组件一旦被训练，先前离线计算出来的「文档 embedding」就不是最新的了，所以这里每 T steps，重建一次「文档 embedding」索引表。

实验

retrieval model：Contriever (Izacard et al., 2022a)
supervision LM：GPT-3

Dataset：

  • Training set :
    800K sequences of 256 tokens each, sampled from the Pile training data.
    Each query is split into two parts: the first 128 tokens are used as the input context x, and the last 128 tokens are used as the ground truth continuation y.
  • external corpus :
    sample 36M documents of 128 tokens from the Pile training data (not overlap) .

Task:

  • language modeling：就是文本生成，可以参考上面的 dataset
  • MMLU / Massive Multi-task Language Understanding：a multiple choice QA dataset
  • open-domain QA: Natural Questions (NQ) (Kwiatkowski et al., 2019) and TriviaQA (Joshi et al., 2017).
