GLM

https://aclanthology.org/2022.acl-long.26.pdf

任务：通过 Autoregressive Blank Infilling ， 即自回归空格填充任务，统一实现自然语言的3大任务：

（1）自然语言理解（NLU）；

（2）无条件生成任务（unconditional generation）；

（3）有条件生成任务（conditional generation）；

将文本的某一个序列（ length≥1 ）进行 mask，然后让模型用自回归的方式分别预测被 mask 的序列。

Mask

使用 λ=3\lambda = 3λ=3 的泊松分布在输入句子 x\mathbf xx 上进行随机 mask span，直到总mask 长度到达原长度的 15%

每一个长度 ≥ 1 的被 mask 的 span 都被替换成为 1 个 [MASK] token，得到 xcorrupt\mathbf x_{\text {corrupt}}xcorrupt​ ，模型使用一个自回归的方式预测被 mask 的 span 。

即模型需要通过 xcorrupt\mathbf x_{\text {corrupt}}xcorrupt​ 和已经被预测出来的 span ，来预测后面的 span。

为了建模 span 之间的独立关系，在还原时，随机打乱需要当前需要生成的目标 span 的位置，即随机选择 [MASK] 来还原。
To fully capture the interdependencies between different spans, we randomly permute the order of the spans

训练时候也可以 one-pass 训练；生成的时候，如果生成的不是 [s] ，则position 可以继续，否则需要更新 position。

Input

输入内容

将输入切分称为两部分

Part A 为被处理过的 xcorrupt\mathbf x_{\text {corrupt}}xcorrupt​ 。→ a bidirectional encoder

Part B 为 masked span，即告诉模型当前需要还原的 [MASK] 的位置 。如上图所示，仅仅提供位置信息。 → a unidirectional decoder

为了保证 Part B 中的自回归生成范式，每一个 span 都被 [START] 和 [END] 进行包裹标识。

2D Positional Encoding， 每个 token 使用两个 positional id

  1. 表示在输入文本中的位置信息

    注意被 mask 的 span 中的所有 token 的该位置编码都是其在句子中的首个 token 的位置，如上图的 555 和 33 。→ the model is not aware of the length of the masked span

  2. 用于区分 part A 和 part B （intra-span position）， part A 的 token 全部为 0，part B 的 token 为其在对应 span 中的 1st-index positional id 。

这两个位置编码是可学习的

Attention

Part A 中的词彼此可见

Part B 中的词单向可见，并且可见 Part A

Multi-Task Pre-training

  • Document-level ：只选择一个 span，占到总长度的 50%-100% 。→ long text generation
  • Sentence-level ：限制被 mask 的是一整个句子，占比 15% 。→ seq2seq

Model Architecture

  • rearrange the order of layer normalization and the residual connection → Pre-Norm
  • use a single linear layer for the output token prediction
  • replace ReLU activation functions with GeLUs

f(x)=max⁡(0,x)f(x)=\max (0, x)f(x)=max(0,x) to f(x)=0.5x(1+tanh⁡(2π(x+0.044715x3)))f(x)=0.5 x\left(1+\tanh \left(\sqrt{\frac{2}{\pi}}\left(x+0.044715 x^3\right)\right)\right)f(x)=0.5x(1+tanh(π2​​(x+0.044715x3)))

Unlike ReLU, which gates inputs by their sign, GELU weights inputs by their percentiles.

This means that GELU allows small negative values when the input is less than zero, providing a richer gradient for backpropagation.

Fine tuning

NLU 也被转化为生成任务，套入模版，让模型补全一个 [MASK] ，类似 PET， 比如文本分类：SENTENCE. It’s really [MASK] 。

NLG：输入放在 Part A，concat 一个 [MASK] 在最后，让模型生成。

对比分析

Comparison with Bert

BERT fails to capture the interdependencies of masked tokens due to the independence assumption of MLM.

cannot fill in the blanks of multiple tokens properly → need to enumerate all possible length

Comparison with XLNet

XLNet uses the original position encodings before corruption, During inference, we need to either know or enumerate the length of the answer, as Bert

XLNet uses a two-stream self-attention mechanism, instead of the right-shift → doubles the time cost of pre training.

Comparison with T5

uses independent positional encodings for the encoder and decoder

relies on multiple sentinel tokens to differentiate the masked spans

→ a waste of model capacity and inconsistency between pre-training and fine-tuning

T5 对编码器和解码器使用独立的位置编码，并依赖于多个 sentinel tokens 来区分掩码跨度。在下游任务中，只使用一个 sentinel tokens，导致模型容量的浪费和预训练和微调之间的不一致。此外，T5 总是按照从左到右的固定顺序预测跨度。因此，在参数和数据更少的情况下，GLM在NLU和seq2seq任务上的性能明显优于T5。

GLM-130B

glm-130b：一个开放的双语预训练模型

不是使用175B+参数，因为这样的大小支持在单个A100（8×40G）服务器上进行推理。

在没有后期训练的情况下将GLM-130B量化为INT4精度，而OPT和BLOOM只能达到INT8。由于GLM架构的独特特性，GLM-130B的INT4量化引入了可忽略不计的性能退化。

模型架构和原始 GLM 类似，将原始文本序列 xxx 进行随机覆盖，变成 xcorruptx_{corrupt}xcorrupt​ ，模型被要求自回归地恢复它们。

two corruption objectives：

  1. [MASK]：句子中的短空格，其长度加起来相当于输入的某一部分。→ T5 , BERT
  2. [gMASK]：提供前缀上下文的句子末尾的随机长度长空格。→ GPT

Sequence Length : 2048

Positional Encoding and FFNs

Rotary Positional Encoding (RoPE)，注意在GLM 中使用的还是 a two-dimensional absolute position encoding，这里是不同的。

turn back to conventional one-dimensional positional encoding

simply remove the second dimension used in the original GLM，因为作者认为填充 mask token 的单向过程中的 attention，本身就会提供一些顺序信息。

[MASK] For sequences corrupted by short spans, we discard the second-dimensional position encoding.
[gMASK] For sequences corrupted by a long span at the end, we change the positional ids to one-dimensional 0,1,⋅⋅⋅,s−10, 1, · · · , s − 10,1,⋅⋅⋅,s−1，and generated tokens will just prolong the first-dimensional positional encoding from the last context token s−1s − 1s−1.

PRE-TRAINING SETUP

Self-Supervised Blank Infilling (95% tokens) 自监督空白填充

每一个文本序列被选择应用上述 [mask] 或者[gmask] 处理

  • [MASK] ：30% of training sequences ;
    • The lengths of masked consecutive spans follow a Poisson distribution (λ = 3)
    • add up to 15% of the input.
  • [gMASK] ：the other 70% sequences ;
    • The prefix of each sequence is kept as context and [gMASK] is used to mask the rest of it.
    • The masked length is sampled from the Uniform distribution.

Data: The pre-training data includes 1.2T Pile (train split), English, 1.0T Chinese Wudao Corpora, and 250G Chinese corpora (including online forums, encyclopedia, and QA) we crawl from the web.

Multi-Task Instruction Pre-Training (MIP, 5% tokens) 多任务指令预训练

遵循 T5 的指令预训练，we include a variety of instruction prompted datasets including language understanding, generation, and information extraction.

Compared to recent works that leverage multi-task prompted fine-tuning to improve **zero-shot task transfer，**这里 MIP仅占5%的tokens，并且只用在预训练阶段，以防止破坏LLM的其他一般能力（无条件自由生成）。

包括74个来自（Sanh et al.，2022；Wang et al.，2022a）的提示数据集，列于表12中。

THE TRAINING STABILITY

impacted by the number of tokens it passes

考虑到计算使用限制，必须在浮点（FP）格式的效率和稳定性之间进行权衡：

低精度FP格式（例如，16位精度-FP16）提高了计算效率，但容易出现上溢和下溢错误，导致训练崩溃。

Mixed-Precision

FP16 : forwards and backwards

FP32 : optimizer states and master weights

这种混合精度的选择会出现训练损失尖峰的现象 frequent loss spikes, 与精度相关的尖峰通常没有明确的原因：有的会自行恢复，有的会导致梯度变得很大甚至溢出。

BLOOM 使用 embedding norm technique 技术来缓解。

作者认为导致 损失尖峰 的原因为：

  1. if using Pre-LN, the transformer main branch’s value scale can be extremely large → addressed in GLM130B by using DeepNorm based Post-LN
  2. attention scores exceed FP16’s range , to solve:

    1. In CogView (Ding et al., 2021), PB-Relax is proposed to remove bias terms and deduct extremum value in attention computation → Not work

      • 在注意力计算中删除偏差项并扣除极值以避免该问题

        大规模的文本图像生成预训练任务通常会由于训练数据之间的差异而变得不稳定，通常表现为Loss NaN和梯度弥散，

        CogView分别提出了PB-Relax(Precision Bottleneck Relaxation)和Sandwich-LN(Sandwich LayerNorm)来进行应对。

        预训练过程中的数值溢出主要发生在Transformer的瓶颈层的最后一个LayerNorm或者attention处。

        • 经过进一步探索发现，网络深处的层输出过大会导致后续的LayerNorm溢出，于是利用了LayerNorm的如下性质对输入进行缩放：

          LayerNorm⁡(x)=LayerNorm⁡(x/max⁡(x))\operatorname{LayerNorm}(x)=\operatorname{LayerNorm}(x / \max (x))LayerNorm(x)=LayerNorm(x/max(x))

        • 同时，attention分数的值过大也会导致数值溢出，这里利用了对 softmax的输入加上或者减去一个常数对于结果没有影响:

          softmax⁡(QTK/d)=softmax⁡(QTK/d− constant )\operatorname{softmax}\left(Q^T K / \sqrt{d}\right)=\operatorname{softmax}\left(Q^T K / \sqrt{d}-\text { constant }\right)softmax(QTK/d​)=softmax(QTK/d​− constant )

          通过上述公式，attention层的计算改为如下公式：

          softmax⁡(QTKd)=softmax⁡((QTαdK−max⁡(QTαdK))×α)\operatorname{softmax}\left(\frac{Q^T K}{\sqrt{d}}\right)=\operatorname{softmax}\left(\left(\frac{Q^T}{\alpha \sqrt{d}} K-\max \left(\frac{Q^T}{\alpha \sqrt{d}} K\right)\right) \times \alpha\right)softmax(d​QTK​)=softmax((αd​QT​K−max(αd​QT​K))×α)

    2. In BLOOM-176B：BF16 format is used instead of FP16 → BF16 , But:

      1. consumes ∼15% more run-time GPU memory ;
      2. not supported on other GPU platforms (e.g., NVIDIA Tesla V100).
      3. is in sacrifice of a significant penalty on model performance，因为 embedding norm 会损伤模型的 zero-shot 能力

提出 Embedding Layer Gradient Shrink (EGS) .

经验搜索表明，「梯度范数」可以作为「训练崩溃」的信息指标。具体来说，发现「训练崩溃」通常滞后于「梯度范数的“尖峰”」几个训练步骤。

这种尖峰通常是由「嵌入层的异常梯度」引起的。

论文观察到，在早期训练中，「嵌入层梯度范数」通常比其他层的梯度范数大几个数量级，并且会大幅波动（参见图4（a））。

最后，发现嵌入层上的梯度收缩可以克服损耗尖峰，从而稳定GLM-130B的训练。（Transformer CogView（Ding等人，2021）
