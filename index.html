Analyze-Generate-Refine

Analyze, Generate and Refine: Query Expansion with LLMs for Zero-Shot Open-Domain QA
现有方法：

  1. 有的方法用监督学习训练得到一个 query 拓展模型，成本高，不适合零样本的开放问答（OpenQA）任务。
  2. 有的方法直接用 LLM 拓展query，但是缺乏确保生成QEs质量的机制。

个人思考
总结方法：利用 LLM 生成 query Analysis，然后通过 LLM 生成第一版 QE，通过第一个版本的 QE 检索真实 doc ，然后拿检索到的真实 Doc 生成第二版 QE ，套娃了属于。

本文方法

拆分为三个步骤 ：Analyze, Generate, Refine

输入 original query qqq

LLM 首先对其进行分析，得到 an analytic text A\mathcal AA

然后再通过 LLM 根据分析 进行随机采样，得到 a set of answer-oriented
query expansions： Qe={qe1,qe2,…,qen}\mathcal{Q}_e=\left\{q e_1, q e_2, \ldots, q e_n\right\}Qe​={qe1​,qe2​,…,qen​}

上述生产过程中，随机采样保证了多样性，生成的 QE（Query Expansion）有的是符合最终答案的，是有益的；但是同时也存在一些存在幻觉的 QE，这个时候就需要进行后处理了，旨在过滤存在幻觉的 QE 。

Analysis of Original Query

先用 LLM 提取 qqq 中的关键词 ，然后得到最终分析。

K=LLM⁡(q)A=LLM⁡(q,K)\begin{gathered} \mathcal{K}=\operatorname{LLM}(q) \\ \mathcal{A}=\operatorname{LLM}(q, \mathcal{K}) \end{gathered} K=LLM(q)A=LLM(q,K)​

Generation of Candidate QEs

1st round QE Generation

Qe=LLM⁡(q,A):={qe1,qe2,…,qen}\mathcal{Q}_e=\operatorname{LLM}(q, \mathcal{A}):=\left\{q e_1, q e_2, \ldots, q e_n\right\} Qe​=LLM(q,A):={qe1​,qe2​,…,qen​}

利用 1st-round QE 检索真实 Doc ，这里使用 BM25 DB 中的 doc 进行纠正。
对于每一个 qei∈Qeq e_i \in \mathcal{Q}_eqei​∈Qe​ ，使用 BM25 检索 top-kkk 个 doc ，即 Dqei\mathcal{D}_{q e_i}Dqei​​ 。
最终，全部汇总为一个集合 contextual references CRs\mathcal{C} \mathcal{R}_sCRs​ 。

CRs=∪{Dqei}=∪{BM⁡25(qei)},qei∈Q\mathcal{C} \mathcal{R}_s=\cup\left\{\mathcal{D}_{q e_i}\right\}=\cup\left\{\operatorname{BM} 25\left(q e_i\right)\right\}, q e_i \in \mathcal{Q} CRs​=∪{Dqei​​}=∪{BM25(qei​)},qei​∈Q

2nd round QE generation
然后用检索到的「真实 doc」和「原始q」重新生成 QE ：

Q‾e=LLM⁡(q,CRs)\overline{\mathcal{Q}}_e=\operatorname{LLM}\left(q, \mathcal{C R}{ }_s\right) Q​e​=LLM(q,CRs​)

同样 采用了随机抽样策略，确保LLMs产生一系列潜在扩展，每个扩展都反映了不同的面和初始查询q的不同解释。生成的扩展范围为后续细化步骤提供了全面的候选池。

Refinement for Optimal QE 后处理过滤

虽然这些候选项中的许多与查询q的正确答案高度相关或紧密对齐，但仍然有一些可能不相关或错误，这可能是由于LLMs的幻觉倾向所致。

比较简单，直接用 LLM 进行过滤审查，

qeo=LLM⁡(q,Q‾e)q e_o=\operatorname{LLM}\left(q, \overline{\mathcal{Q}}_e\right) qeo​=LLM(q,Q​e​)

Knowledge-Aware Query Expansion

Knowledge-Aware Query Expansion with Large Language Models for Textual and Relational Retrieval

历史工作的缺点：现有的方法大多关注于增强搜索查询与目标文档之间的文本相似性，而忽略了 文档之间的关系 。

本文如何解决：本文提出了一个知识驱动的查询扩展框架，结合知识图谱（KG）中的结构化文档关系来增强LLMs的效果。为了克服现有KG方法中基于实体评分的局限性，作者将文档文本作为知识图谱节点的丰富表示，并通过文档关系过滤来改进检索效果，称之为“知识感知检索”（KAR）。

个人思考： 就是先检索 KG ，然后用 KG 信息进行 QE，但是有个大问题，KG 怎么建立和维护呢？而且这里仅仅是 QE 就需要这么大的成本吗？

问题定义

给定 a collection of textual documents D\mathcal DD ，其中 di∈Dd_i \in \mathcal{D}di​∈D 表示描述实体 iii 的 a textual document 。
给定 a knowledge graph G=(V,R)\mathcal{G}=(\mathcal{V}, \mathcal{R})G=(V,R) ，其中 vi∈Vv_i \in \mathcal{V}vi​∈V 表示对应实体节点。

给定 query qqq ，目标是检索得到相关的  a set of documents A⊆D\text { a set of documents } \mathcal{A} \subseteq \mathcal{D} a set of documents A⊆D 。

而 QE 需要做的就是根据文档集合和知识图谱生成 Query Expansion

Qe=f(q,D,G),q′=Concat⁡(q,Qe)\begin{gathered} \mathcal{Q}_e=f(q, \mathcal{D}, \mathcal{G}), \\ q^{\prime}=\operatorname{Concat}\left(q, \mathcal{Q}_e\right) \end{gathered} Qe​=f(q,D,G),q′=Concat(q,Qe​)​

本文方法

Entity Parsing by LLM

Eq={q,LLM⁡(q)}\mathcal{E}_q=\{q, \operatorname{LLM}(q)\} Eq​={q,LLM(q)}

Entity Document Retrieval

匹配实体：用一个离线的双塔模型，每一个实体都从 D\mathcal{D}D 检索得到 doc did_idi​ 。

KG Relation Propagation

同步得到节点：每一个 did_idi​ 都对应了 KG 中的一个节点 ，于是直接匹配得到  corresponding entity node vi∈V\text { corresponding entity node } v_i \in \mathcal{V} corresponding entity node vi​∈V 。

同时每个节点会获取其 h-hop neighbors h \text {-hop neighbors }h-hop neighbors  ，即其附近子图 Ni\mathcal N_iNi​ 。

Document-based Relation Filtering

通过多跳关系和过滤得到最终子图

上述检索得到的子图中存在大量和 query 不相关的「冗余节点和冗余关系」。

这里利用 document-based relation filtering，即节点本身的名字和其内部的文本和属性信息来进行过滤 ( 比如节点是一个 paper ，名字是 paper name ，其 abstract，作者，topic 等信息可以被利用 ) 。

其实也是一个双塔模型，只不过 doc tower 的信息比较多，同样也是用相似度分数进行过滤。用 top-k scored nodes 作为 query-focused neighbors 。

Document Triples Construction

拿到了子图，直接抽三元组：

Ti,q={(di,ri,j,dj)∣vj∈Ni,q,ri,j∈Ri,q}\mathcal{T}_{i, q}=\left\{\left(d_i, r_{i, j}, d_j\right) \mid v_j \in \mathcal{N}_{i, q}, r_{i, j} \in \mathcal{R}_{i, q}\right\} Ti,q​={(di​,ri,j​,dj​)∣vj​∈Ni,q​,ri,j​∈Ri,q​}

Knowledge-Aware Expansion

让大模型根据 Ti,q\mathcal{T}_{i, q}Ti,q​ 生成 QE 。
通过一次语言模型（LLM）的推理，生成多个响应，然后把这些响应拼接在一起，作为最终的扩展，附加到原始查询后面。
