2024 - ServiceNow, Facebook - LLM2Vec

https://arxiv.org/pdf/2404.05961

将 Decoder only LLM 改造为 Embedding model ；对 token 进行平均池化得到句向量。

结构改变：

  1. 启动双向注意力：casual-mask attention 改为 full attention

  2. 有监督训练 Masked next token prediction (MNTP)：

    启动双向注意力后，模型的效果变差，所以这个任务的目的是为了让模型快速适应双向编码。

    引入 MLM 任务，输入引入 【mask】，预测目标位置是上一个 token 的输出，而不是【mask】自己。

    use the underscore _ as the mask token ，然后用 LoRA 进行训练，训练结束后 merge LoRA。

  3. 无监督训练 Unsupervised contrastive learning (SimCSE)：

    上面的 task 已经成功得到了一个 encoder-based model，这里继续针对序列编码这个任务来进行针对性地训练。

    应用 SimCSE，两次 dropout 得到正样本，然后对比学习。

    同样也是用 LoRA 进行训练。

2024 - NVIDIA - NV-Embed

NV-Embed: Improved Techniques for Training LLMs as Generalist Embedding Models

本模型直接微调 Mistral 7B，没有使用任何人工数据和合成数据。

  1. 提出 a latent attention layer 用于得到 pooled embedding
  2. 为强化表示能力，移除 Casual Mask
  3. 训练过程：两阶段对比指令调优
    1. 检索数据集上，带有指令的对比训练，利用「批内负例」和「精选的硬负例」。
    2. 将各种非检索数据集混合到指令调优中，继续进行对比训练，但是不使用「批内负例」。

Bidirectional Attention

仅仅是移除了 casual mask 的操作，然后直接用于后续对比学习的训练中。

Latent Attention Layer

现存表示句向量的一些方法：

  1. mean pooling: 弱化一些强token的表现力
  2. < eos >：存在 recent bias，因为注意力的衰减，严重依赖于句子最后的 token 。

本文引入 dictionary learning：

  1. 一个额外的 trainable dictionary K=V∈Rr×dK=V\in\mathbb R^{r\times d}K=V∈Rr×d ，拿模型最后一层输出 Q∈Rl×dQ\in\mathbb R^{l\times d}Q∈Rl×d 去进行 cross attention ，输出为 O∈Rl×dO\in\mathbb R^{l\times d}O∈Rl×d ：O=softmax⁡(QKT)VO=\operatorname{softmax}\left(Q K^T\right) VO=softmax(QKT)V ，这里 r=512r=512r=512 。
  2. GELU FFN
  3. 然后使用 Mean Pooling 得到整个序列最终的 sentence representation

超参数：

512 latents, 4096 hidden dimension size, and 8 multi-head attentions

Two-stage Instruction-Tuning

引入指令，模型可以根据指令定向控制生成的 embedding

「批内负例」 效果很好，但是只适合检索任务，这里为了训练得到的 embedding 模型同时能兼具分类和聚类任务，作者引入二阶段训练方法：

  1. stage 1 ： 在大量检索数据上进行 contrastive training with instructions ，负样本为 in-batch negs + 人工选择的 hard negs

    Instruction 组织格式：

    Query侧 ： qinst +=Instruct : task_definition Query : q+q_{\text {inst }}^{+}=\text {Instruct : task\_definition Query : } q^{+}qinst +​=Instruct : task_definition Query : q+ ，在后续进行 Latent Attention Layer 时，忽略 instruction 处的 token

    Doc 侧：不进行任何改动

  2. stage 2 ：混合检索数据和非检索数据( classification, clustering and semantic similarity (STS ))，继续进行 contrastive instruction-tuning ，但是不使用 in-batch negs 了。

    每条数据包含了：qinst +q_{\text {inst }}^{+}qinst +​ ，d+d^+d+，d0− ... dn−d^-_0 \ ...\ d^-_nd0−​ ... dn−​

训练使用了 LoRA （ rank 16 + alpha 32 ）

将 Mistral 原始的 Casual Attention 换成 Bi-Directional Attention

消融实验

双向 attention 有效，同时LA Layer 也有效。

2024 - 小红书 - NoteLLM

https://zhuanlan.zhihu.com/p/698416915

使用 LLama2 作为 text embedder ，

  1. 使用 Prompt 生成文本 embedding
  2. 传统嵌入将标题或者类别信息融入text，一起进行嵌入。但是作者认为标题和类别应该是代表文本的中心思想，而不是文本的一部份。因此引入一个额外的辅助任务：生成标签和类别。

训练方法 Generative-Contrastive Learning

  1. within-batch 对比学习，不额外补充负样本。

    数据收集：使用了推荐中的协同过滤信号作为标签来进行对比学习，首先会统计所有笔记对的共现分数，然后使用共现分数高的笔记对视为相关性高的笔记，作为正样本。这样可以引入推荐的协同过滤信号，从而使得生成的embedding更适用于下游的推荐任务。

    构造方法如下：

    对于文档 DA 和 DB，统计一段时间窗口（一周）内有点击过他们俩的 user set UUU ，于是 DA 和 DB 的共现分数为：

    snA→nB=∑i=1U1Nis_{n_A \rightarrow n_B}=\sum_{i=1}^U \frac{1}{N_i}snA​→nB​​=∑i=1U​Ni​1​

    这里 UUU 是用户的个数，NiN_iNi​ 代表用户的点击次数, 也就是对活跃用户进行了降权, 防止某些高活用户什么笔记都点导致计算的共现分数不准确 。

    权重取行为数量的倒数是对活跃用户降权，极端情况如果一个用户点击了所有笔记，那么就等于所有笔记都是相关的了，所以进行一个降权操作 。

    然后再卡阈值，将低于或高于某个阈值的笔记（outliers）进行过滤，最后对于每个笔记，都会取 top t 个其他笔记，作为相关笔记，再两两构建笔记对作为输入的正样本。

  2. 用笔记标题和内容来同时做「类别和标签」生成

    这个任务不仅可以生成标签和类别，同时，由于它和生成embedding的过程类似，都是可以提取笔记文本中的关键信息，因此，引入该任务也可以增强第一个任务生成的笔记embedding。
