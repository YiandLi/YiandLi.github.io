目的：使用 LLM 直接生产 doc id，a token generation task

Two Major Contributions：

  1. Item Indexing：
    1. 设计了 a tree-structured vector quantization (VQ) method ，用于 item indexing：一个 item 对应几个可以学习的 id，比如 <a_112><b_32><c_5><d_175>
    2. 设计了 a uniform semantic mapping method，防止 ID Conflicting（ ≥1 个 item 被映射到了同一个 id ）
  2. Alignment Tuning：设计一系列训练目标，用于对齐 「LLM 原本的通用语义」和「推荐系统的协同过滤语义」
    1. explicit index language alignment
    2. implicit recommendation-oriented alignment

Learning Item Indices for Semantic Integration

Vector Quantization Indexing

how to represent an item with index IDs (called item indices)？

作者没有使用原始的 doc one-hot id，而是使用 vector quantization 将 item 映射为 a small number of discrete indices ；同时每个 index 可以被不同的 doc 共享。

  1. 使用 LLM 得到 item embedding eee

  2. 训练一个 RQ-VAE，生成一系列 item indices

RQ-Index 的优点：

  1. 层级 codebook ，允许更小的 code book 去表征更多的 item id
  2. tree based index 能够适配自回归的任务

此外，还需要考虑两点：

  1. no conflicts in item indices
  2. the established semantic spaces of item indices should be aligned with the semantics of LLMs

Conflict Mitigation via Uniform Semantic Mapping

可能会存在现象：lead to index conflicts among items within the same leaf node ，即叶子分布不均匀，存在某些叶子节点被赋予多个 item

[23], [38] 添加了一个补充的 id，比如两个 item 被映射为[ 32,33,1 ] ，那么此时增加一个额外的 id 分别表示两个 item ： [ 32,33,1, ***0*** ] 和 [ 32,33,1, ***1*** ] 。

作者这里用了 conflict mitigation method 来防止叶子节点有很多 item ，作者这里的方法就是让 item 均匀分散在 codebook 最后一层，提出 a uniform distribution constraint

Aligning Language and Collaborative Semantics in LLMs

构建好 index 之后，首先要将 index 加入 LLM 的词表中，但是这些 index id 都是 OOV ，为了对齐 language and collaborative semantics ，作者设计了一序列的 semantic alignment tasks 。

1) Sequential Item Prediction

设计了 personalized recommendation instructions ，LLM 的输入为交互历史和这个指令，预测下一个 item id 。

2) Explicit Index-Language Alignment

目前，相似 item 的共享语义只能体现在相同的前缀上，作者认为这是一种弱约束关系。

为了强化相似 item 的 item id 语义，又提出两个 index-item 对齐任务。

  1. 输入 title/description or a combination of both.，指令生成 item indices 。

  2. 输入 item indices ，指令生成 item information

3) Implicit Recommendation-oriented Alignment

a) Asymmetric item prediction : 真正推理时，模型是根据 lastN item id 预测 next item id，是对称的，这里作者引入非对称的预测结构。

choice 1：用 description / title 替代 next item id

choice 2：用 a text sequence of item titles 替代 lastN item id

这种非对称任务更难，需要模型整合 item indices, language semantics, and collaborative semantics 。

*b) Item prediction based on user intention：*作者认为用户评论也很重要（评论提供了有关用户个人口味和进行特定交互的动机的有价值证据），所以 LLM 应该可以通过 user intention 推荐 item 。

首先用 GPT3.5 根据 item reviews 提取出 user intention ；然后提供两个任务：

  • 输入 instant user intention （directly），输出 item ids

  • 输入 the user’s interaction history 和 user intention （personalized），输出 item ids

c) Personalized preference inference：

用户的 interaction history 可以反映出他的 preference ，这里仍然用 GPT3.5 生成其 preference，然后将结果蒸馏给 LLM

这里，LLM 的输入为 item ID，而 GPT 3.5 输入为 text 。

This task requires the index sequence to act as an effective substitute for the title sequence,
enabling the LLM to understand the joint language and collaborative semantics within the index sequence and accurately extract user preferences.

Training and Inference

Backbone ： Llama

Title + Description Embedding → Avg Pooling

Level of item indices = 4 , each level consisting of 256 codebook vectors, each vector has a dimension of 32.

LLM双塔-1：LLM2Vec, NV-Embed, NoteLLM
