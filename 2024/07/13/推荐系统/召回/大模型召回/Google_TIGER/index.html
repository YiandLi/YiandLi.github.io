Recommender Systems with Generative Retrieval

doc 被编码为 Semantic ID，End2End 直接生成 Semantic ID。

  1. Semantic ID generation using content features
    使用一个编码模型得到 item 编码；然后将每一个 Item 编码 使用 "Semantic ID” 进行表示，这里是训练了一个 RQ-VAE，用 Encoder 部分得到 Item Code

  2. Training a generative recommender system on Semantic IDs
    用得到的 Item Code 作为标签，训练一个 Transformer ，直接根据 item 编码生成 Item Code

使用 Semantic ID 的好处：

  1. 通过 ID，item 之间的语义信息可以传递，也就是相似 item 有相似 ID
  2. ID 是共享的，所以新 doc 能够有效加入系统，直接 VAE encoder 得到 id 即可
  3. ID 能表征很多 doc，扩展性高，因为原本需要为每一个 doc 都维护一个 Embedding，现在 Codebook Embedding 不变，只需要维护 Item ID 即可。

Semantic ID Generation

同样是 RQ-VAE，这里和 LC-Rec 不同的是，使用三层 Codebook，每一层 Codebook 不共享，这里选择 mmm 层，每一层为不共享的 KKK 个 code。

为什么不共享：作者认为随着深度增加，残差的数量级不断减小，所以单独处理比较好。

原理：

RQ-VAE 是一个自编码器，可以同时训练 encoder，decoder 和 codebook embedding 。

有多层，每一层都从 codebook 中找到最近的 code vector 来替代自己，然后将残差作为下一层的输入。

最终用所有层的 code vector 的和来表示自己。

RQ 普遍存在一个问题，即大量的 item 都绑定到少量的 ID 上。为了缓解这个问题，作者选择 k-means clustering-based initialization 。

Collisions

碰撞就是发生语义冲突了，多个item映射到同一个 code id 上了。

为了消除冲突，本文在有序语义码字的末尾添加了一个额外的标记，以使它们具有唯一性。

例如，如果两个项目共享语义ID（12，24，52），附加额外的令牌来区分它们，将这两个项目表示为（12,24,52,0）和（12,24,52,1）。

为了检测冲突，再训练好 RQ-VAE 后，需要维护一个将语义ID映射到相应item的查找表，然后更新 Code ID，再训练。

Experiment

Evaluation Metrics.

We use top-k Recall (Recall@K) and Normalized Discounted Cumulative Gain (NDCG@K) with K = 5*,* 10 to evaluate the recommendation performance.

RQ-VAE Implementation.

Sentence-T5,

Encoder: 512, 256 and 128 with ReLU activation, finally 32 dim

Codebook: size 256 * 3 , 32 dim

为了保证覆盖率和准确度，RQVAE 训练 20 个 epoch

Generation Model.

the vocabulary contains 1024 (256 × 4) tokens to represent items in the corpus

use the Hashing Trick [38] to map the raw user ID to one of the 2000 user ID tokens

construct the input sequence as the user Id token followed by the sequence of Semantic ID tokens corresponding to a given user’s item interaction history
