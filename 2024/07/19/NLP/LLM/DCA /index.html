Dual Chunk Attention (DCA)

Posted on 2024-07-19 In NLP , LLM Model
Symbols count in article: 981 Reading time ≈ 7 mins.

24 - ICML - ALiBaBa. Training-Free Long-Context Scaling of Large Language Models

NTK & YARN 固然好，但是最好还是 post fine tune 下，这里提出一种 train-free 的方法。

Popular techniques for Llama-based models, including Position Interpolation (PI) (Chen et al., 2023b) and NTK-Aware RoPE (NTK) (Lo- calLLaMA, 2023b;a), are adaptations of Rotary Positional Encodings (RoPE) (Su et al., 2022). These scaled positional encodings necessitate fewer finetuning steps compared to the original RoPE, and their training costs can be further reduced via methods such as YaRN (Peng et al., 2023) and CLEX (Chen et al., 2023a). However, in a training-free setting, we find that these approaches usually lead to a notable increase in PPL especially in input lengths that are more than twice the training length.

通过将长序列的「注意力计算」分解为多个小块，引入三个组件：

  1. intra-chunk attention：负责 chunk 内部的 token attention
  2. inter-chunk attention：负责不同 chunk 的 token attention
  3. successive chunk attention：负责连续的 chunk 的连续的 token 的 attention

可以与 Flash Attention 无缝集成。同时DCA 和当前的缩放位置编码（ PI & NTK ）是正交的。

Intra-Chunk Attention

输入总长度 lll ，chunk size 为 sss ，于是可以得到 n=l/sn = l/sn=l/s 个 chunk ，

如图 Fig2.a ，被分成了两个 chunk。在 intra attention 中， Q K 对应的位置 id 是相同的，为：

PqIntra=Pk=[0,1,2,3,4,5⏟chunk 0,0,1,2,3,4,5⏟chunk 1]P_{\mathbf{q}}^{\text {Intra}} = P_{\mathbf{k}}=[\underbrace{0,1,2,3,4,5}_{\text {chunk } 0}, \underbrace{0,1,2,3,4,5}_{\text {chunk } 1}]PqIntra​=Pk​=[chunk 00,1,2,3,4,5​​,chunk 10,1,2,3,4,5​​]

如果满足条件 ⌊i/s⌋=⌊j/s⌋\lfloor i / s\rfloor=\lfloor j / s\rfloor⌊i/s⌋=⌊j/s⌋ ，即两个 token 在同一个 chunk 中，则使用这种方法来定义两个token 之间的相对距离：

M[i][j]=PqIntra [i]−Pk[j]M[i][j]=P_{\mathbf{q}}^{\text {Intra }}[i]-P_{\mathbf{k}}[j]M[i][j]=PqIntra ​[i]−Pk​[j]

于是，从图中可以看到，某个 token 只能看见「相同 chunk」中的「前缀 token」。

具体位置编码方法仍然是类似 RoPE ，直接引入绝对位置信息：

qi⊤kj=f(q,PqIntra⁡[i])⊤f(k,Pk[j])\mathbf{q}_i^{\top} \mathbf{k}_j=f\left(\mathbf{q}, P_{\mathbf{q}}^{\operatorname{Intra}}[i]\right)^{\top} f\left(\mathbf{k}, P_{\mathbf{k}}[j]\right)qi⊤​kj​=f(q,PqIntra​[i])⊤f(k,Pk​[j])

Inter-Chunk Attention

需要保证靠后的 chunk 中的 token 的 Q 更大，即大于 s−1s-1s−1 。

这里比较暴力，会引入一个超参数 「Pre-training Context Length」 ccc 来定义 Q 的位置 id：

PqInter =[c−1,c−1,…c−1⏟l elements ]P_{\mathbf{q}}^{\text {Inter }}=[\underbrace{c-1, c-1, \ldots c-1}_{l \text { elements }}]PqInter ​=[l elements c−1,c−1,…c−1​​]

即预先定义好最长的距离差

如果满足 ⌊i/s⌋≠⌊j/s⌋\lfloor i / s\rfloor\ne \lfloor j / s\rfloor⌊i/s⌋=⌊j/s⌋ ，即两个 token 不在同一个 chunk 中，则使用这种方法来定义两个token 之间的相对距离：

M[i][j]=PqIntra [i]−Pk[j]=c−1−Pk[j]≥c−sM[i][j]=P_{\mathbf{q}}^{\text {Intra }}[i]-P_{\mathbf{k}}[j]=c-1-P_{\mathbf{k}}[j] \geq c-sM[i][j]=PqIntra ​[i]−Pk​[j]=c−1−Pk​[j]≥c−s

比如，图中 ccc 设置为 10，于是所有的 Inter-Chunk Q 都被设为 10−1=910-1=910−1=9 ，然后直接减去 PkP_kPk​ 即可。

Successive-Chunk Attention

Inter 有个问题，也就是对于 K0 来说，Q5和他的相对距离为 5，但是 Q6和他的相对距离突然变成了 9 ，步子太大了。

这里引入 local window length w=c−sw= c - sw=c−s ，只需要调整 PqInter P_{\mathbf{q}}^{\text {Inter }}PqInter ​ 的前 www 个 id 即可：

PqSucc =[ s,s+1,…,s+w−1⏞w elements ,c−1,…,c−1⏟the same for all chunks  ]P_{\mathbf{q}}^{\text {Succ }}=[\ \underbrace{\overbrace{s, s+1, \ldots, s+w-1}^{w \text { elements }}, c-1, \ldots, c-1} _{\text {the same for all chunks }}\ ]PqSucc ​=[ the same for all chunks s,s+1,…,s+w−1​w elements ​,c−1,…,c−1​​ ]
