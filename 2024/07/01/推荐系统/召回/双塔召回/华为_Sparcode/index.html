Beyond Two-Tower Matching: Learning Sparse Retrievable Cross-Interactions for Recommendation

双塔模型效率高，精度低；单塔模型精度高，效率低。本文综合两者的方法与结构，得到一种检索粒度介于两者之间，平衡效率和精度的方法。

十分像 Q2Q2I

借鉴 IVF，使用 VQ (vector quantization) 的方法，构造一个 code book，code book 维护了 embedding segment 2 embedding segment ，通过 bert 得到的 query embedding 去 code book 中找 code，然后通过 code index 得到新的 embedding Tk(q)\mathcal{T}_k(q)Tk​(q)，用这个新的 embedding 去和 doc embedding 计算相似度。

In vector quantization [29], a codebook refers to a series of numbered vectors whose numbers are called codes. The quantification is to input query and return code and the corresponding vector by looking up the codebook, resulting in an arbitrary query sharing the codebook.

对比下相似度：

  • Two-tower model: score=E1(q)∘E2(c)\text {score} =E_1(q) \circ E_2(c)score=E1​(q)∘E2​(c)
  • All-to-all interaction model: score=E(q,c)\text{score} =E(q, c)score=E(q,c)
  • SparCode: score=∑kE(Tk(q),c)\text{score} =\sum_k E\left(\mathcal{T}_k(q), c\right)score=∑k​E(Tk​(q),c)

下面看下流程

1. Embedding

首先 query 和 candiate item 被单独编码

Hu=[h1u,h2u,…,hLu],Hu∈RL×DHv=[h1v,h2v,…,hPv],Hv∈RP×D\begin{aligned}H^u & =\left[h_1^u, h_2^u, \ldots, h_L^u\right], H^u \in \mathbb{R}^{L \times D} \\H^v & =\left[h_1^v, h_2^v, \ldots, h_P^v\right], H^v \in \mathbb{R}^{P \times D}\end{aligned}HuHv​=[h1u​,h2u​,…,hLu​],Hu∈RL×D=[h1v​,h2v​,…,hPv​],Hv∈RP×D​

2. Sparse Tokenizer：global HHH 变 local TTT

然后整体的向量 HHH 通过 tokenizer_i 映射为另一个「 tokenized embedding Ti∈RDTT_i \in \mathbb R^{D^T}Ti​∈RDT 」 ，tokenizer 数量决定了「tokenized embedding」的数量 。

Tiu=Tokenizer⁡(Hu)∈RDTT_i^u=\operatorname{Tokenizer}\left(H^u\right) \in \mathbb{R}^{D^T}Tiu​=Tokenizer(Hu)∈RDT

这个 tokenizer 是一个随意的网络结构，只要输入维度输出维度符合 L×D→DTL\times D \rightarrow D^TL×D→DT 即可，这里好像 DTD^TDT 可以不等于 DDD 。

各个「tokenized embedding」之间相互独立，在不同的应用场景下表示不同的意思，可以表示原始向量 HHH 的某一个方面（类似不同的字段特征，或者多头，或者不同通道）

于是得到所有的「tokenized embedding」，这里 KKK 表示使用了多少独立的 tokenizer

Tu=[T1u,T2u,…,TKuu]∈RKu×DTTi=[T1i,T2i,…,TKci]∈RKc×DT\begin{aligned}T^u & =\left[T_1^u, T_2^u, \ldots, T_{K_u}^u\right] \in \mathbb{R}^{K_u \times D^T} \\T^i & =\left[T_1^i, T_2^i, \ldots, T_{K_c}^i\right] \in \mathbb{R}^{K_c \times D^T}\end{aligned}TuTi​=[T1u​,T2u​,…,TKu​u​]∈RKu​×DT=[T1i​,T2i​,…,TKc​i​]∈RKc​×DT​

3. Quantizer

使用 VQ 将 tokenized embedding TiuT_i^{u}Tiu​ 映射为 codebook embedding ：

切分：每个 token embedding TiuT_i^uTiu​ 被切分为 MMM 个子向量，每个子向量长度为 DT/MD^T / MDT/M

量化：每个子向量对应一个 codebook C∈RN×DTMC \in \mathbb{R}^{N \times \frac{D^T}{M}}C∈RN×MDT​，这里设置 codebook 大小为 NNN 。拿切分后的一段在 codebook 上找最近的 codebook embedding 进行替换。

T~iu,(m)=Quantizer⁡(Tiu,(m),C(m))=Ck(m), where k=arg⁡min⁡j∥Cj(m)−Tiu,(m)∥2\begin{aligned}\widetilde{T}_i^{u,(m)} & =\operatorname{Quantizer}\left(T_i^{u,(m)}, C^{(m)}\right) \\& =C_k^{(m)} \text {, where } k=\arg \min _j\left\|C_j^{(m)}-T_i^{u,(m)}\right\|_2\end{aligned}Tiu,(m)​​=Quantizer(Tiu,(m)​,C(m))=Ck(m)​, where k=argjmin​∥∥∥∥​Cj(m)​−Tiu,(m)​∥∥∥∥​2​​

然后就可以得到新的 「量化后子向量」:

T~iu=Concat⁡(T~iu,(1),T~iu,(2),⋯ ,T~iu,(M))\widetilde{T}_i^u=\operatorname{Concat}\left(\widetilde{T}_i^{u,(1)}, \widetilde{T}_i^{u,(2)}, \cdots, \widetilde{T}_i^{u,(M)}\right)Tiu​=Concat(Tiu,(1)​,Tiu,(2)​,⋯,Tiu,(M)​)

使用元组来表示量化结果，比如 M=2M=2M=2 ，被切成了两个子向量，第一个子向量量化到第二个code，第二个子向量量化为第5个code，则量化结果可以表示为 (2,5)(2,5)(2,5) 。

如何选择 NNN ？如果 NNN 过大，则量化过程耗时比较久；如果 NNN 过小，则量化损失很大。可以表示的 discrete code 的数量最多为 NMN^MNM ，一般 M≥2M\ge 2M≥2 。

4. All2all Interaction based score

如何计算一个 user 和一个 item 的相关性？

原始 user 向量为 Tu=[T1u,T2u,…,TKuu]∈RKu×DTT^u =\left[T_1^u, T_2^u, \ldots, T_{K_u}^u\right] \in \mathbb{R}^{K_u \times D^T}Tu=[T1u​,T2u​,…,TKu​u​]∈RKu​×DT ，量化后每一个「子向量」变为 T~iu=Concat⁡(T~iu,(1),T~iu,(2),⋯ ,T~iu,(M))\widetilde{T}_i^u=\operatorname{Concat}\left(\widetilde{T}_i^{u,(1)}, \widetilde{T}_i^{u,(2)}, \cdots, \widetilde{T}_i^{u,(M)}\right)Tiu​=Concat(Tiu,(1)​,Tiu,(2)​,⋯,Tiu,(M)​) ，
于是一个量化后「tokenized user embedding」和整体的 原始 item 向量 TiT^iTi 的分数为（以 MLP 为例，可以使用别的方法求分数）：

Si=MLPs⁡([sg⁡[T~iu]⊙T1i;⋯ ;sg[T~iu]⊙TKci]),i∈{1,2,⋯ ,Ku}S_i=\operatorname{MLPs}\left(\left[\operatorname{sg}\left[\widetilde{T}_i^u\right] \odot T_1^i ; \cdots ; s g\left[\widetilde{T}_i^u\right] \odot T_{K_c}^i\right]\right), i \in\{1,2, \cdots, K_u\}Si​=MLPs([sg[Tiu​]⊙T1i​;⋯;sg[Tiu​]⊙TKc​i​]),i∈{1,2,⋯,Ku​}

sg[⋅]\text{sg}[\cdot]sg[⋅] 表示 stop gradient，所以 codebook 不会被传递梯度，这里梯度只传递给了 item 侧。

为什么不传递梯度给 codebook，codebook 仅仅起到了一个压缩量化的作用，不对最终效果负责，目的仅仅是希望量化效果不断变好。

原始的 user 是被 KuK_uKu​ 个「tokenized user embedding」表征的，最终也会得到 KuK_uKu​ 个独立的分数，然后用这些独立的分数再聚合一次，求整体分数即可，论文里用简单的累加：

y^i=ReLU⁡(Si+b);   f(q,c)=y^=∑i=1Kuy^i\hat{y}_i=\operatorname{ReLU}\left(S_i+\mathbf{b}\right) ;\ \ \ f(q, c)=\hat{y}=\sum_{i=1}^{K_u} \hat{y}_iy^​i​=ReLU(Si​+b);   f(q,c)=y^​=∑i=1Ku​​y^​i​

b\mathbf bb 是一个可学习向量，作者称其至关重要，起到了一个 threshold 的作用：如果「量化子向量」和 item 过于不相似，就不考虑它了。

在推理过程中，我们直接存储 y^i\hat y_iy^​i​ ，也就是 「量化后子向量」 到每一个 item 的相似度。

其实这里讨论的就是如何用每个 tokenized embedding 求分数，也是类似于 colbert 一样，一个 tokenized user embedding 和所有的 tokenized item embedding 计算分数，然后将所有的分数相加。

只不过进一步将 tokenized user embedding 切分然后量化，直接存储最终的答案，推理时就不用再去计算了。

Inference

  1. 每一个 item 的 embedding 经过 tokenizer，得到多个 「tokenizer 子向量」：

    Tu=[T1u,T2u,…,TKuu]∈RKu×DTTi=[T1i,T2i,…,TKci]∈RKc×DT\begin{aligned}T^u & =\left[T_1^u, T_2^u, \ldots, T_{K_u}^u\right] \in \mathbb{R}^{K_u \times D^T} \\T^i & =\left[T_1^i, T_2^i, \ldots, T_{K_c}^i\right] \in \mathbb{R}^{K_c \times D^T}\end{aligned}TuTi​=[T1u​,T2u​,…,TKu​u​]∈RKu​×DT=[T1i​,T2i​,…,TKc​i​]∈RKc​×DT​

  2. 每个 「tokenizer 子向量」量化为对应的「量化子向量」，推理时不需要保存这个「量化子向量」，只需要保存其 id 即可。

  3. 用 id 得到到每一个 item 的分数

  4. 聚合所有 id 到每一个 item 的分数，得到整体的到每一个 item 的分数

  5. 选择 top-K item 作为召回结果

在第二步中，需要保存 id2itemscore 这个映射，如果保存所有可能的组合情况，有可能内存压力太大，所以这里进行优化。

对于某一个 id，可能和他相关的 item 只有几个，所以没有必要去存储和它不相关的 item & score，这里就需要一个阀值来确定哪些 item 是不相关的。

论文中的做法是，推理中在计算 score 时：y^i=ReLU⁡(Si+b)\hat{y}_i=\operatorname{ReLU}\left(S_i+\mathbf{b}\right)y^​i​=ReLU(Si​+b) ，设置 b=0b = 0b=0 ，这样仅保留正相关的匹配分数。
