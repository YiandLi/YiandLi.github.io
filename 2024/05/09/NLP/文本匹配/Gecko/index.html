Gecko: Versatile Text Embeddings Distilled from Large Language Models

目的： 直接利用 LLM 来改进文本嵌入模型

谷歌 DeepMind 的研究者提出了 Gecko，这是一种从 LLM 中蒸馏出来的多功能文本嵌入模型，其在 LLM 生成的合成 fine-tuning 数据集 FRet（Few-shot Prompted Retrieval dataset）上进行训练，并由 LLM 提供支持。

Gecko 是一个 1.2B 的 pre-trained encoder model M\mathcal MM

训练步骤：

  1. 大规模预训练 pre-fine-tuning ：data is from previous work (Ni et al., 2021; §3.1)
  2. 微调蒸馏 fine-tuning ：create a novel fine-tuning dataset for a diverse set of downstream tasks via a two-step LLM distillation
    数据集名称为 FRet（Few-shot Prompted Retrieval dataset），每一个样本格式为：a task description, input query, positive passage, and negative passage 。

Self-Supervised Pre-finetuning

数据

  1. 使用 Ni 等人提出的大规模社区 QA 数据集，该数据集包括来自在线论坛和 QA 网站的文本对，包含「question-answer pairs」。
  2. 从 Web 上抓取 「title-body text pairs」。

预训练阶段不考虑 hard negative ，并且选择尽量大的 batch size 。

FRet: Two-Step LLM Distillation

一般来讲，训练嵌入模型的传统方法依赖于大型的、手动标记的数据集。然而，创建此类数据集既耗时又昂贵，并且常常会导致不良偏差和缺乏多样性。

LLM-based Diverse Query Generation

首先生成 query
目标：queries cover a diverse set of tasks and linguistic patterns

步骤：通过 few-shot 控制 query 的多样性，输入为 a sampled web passage ，然后让模型生成 task description ttt 和 a relevant query qqq 。

LLM⁡(PQG,pseed )→(t,q)\operatorname{LLM}\left(\mathbb{P}_{\mathrm{QG}}, p_{\text {seed }}\right) \rightarrow(t, q) LLM(PQG​,pseed ​)→(t,q)

  1. Prompt ，包含了 instruction 和 few examples ，example 和 instruction 可以自由组合，来保证多元性。
  2. a passage pseedp_{\text {seed}}pseed​ drawn randomly from the web corpus
  3. task description ttt 表示了 retrieval 的类型，比如 ‘Given a query, find a passage that has the answer to the query’ (question answering) or ‘Given a query, find a passage that allows you to check whether the query is true or not’ (fact checking) 。
  4. query qqq 则是和 ttt 在语意上是匹配对应的。

Unified Fine-tuning Mixture

combine FRet with other academic training datasets in the same format: task description, input query, positive passage (or target), and negative passage (or distractor).

Academic Data

In addition to FRet, we use the following academic training datasets: Natural Questions (Kwiatkowski et al., 2019), HotpotQA (Yang et al., 2018), FEVER (Thorne et al., 2018), MedMCQA (Pal et al., 2022), SNLI (Bowman et al., 2015), MNLI (Williams et al., 2018), and several classification datasets from Huggingface. For the multilingual model, we add training sets from MIRACL (Zhang et al., 2023). All datasets are pre-processed to have a unified encoding format (Appendix B), containing a task description, a query, a positive passage, and a negative passage.

Classification Data for Contrastive Learning

同时整合进一些分类数据，相同 label 作为正例，不同 label 作为负例。

PaRaDe

PaRaDe: Passage Ranking using Demonstrations with LLMs

做法： Query Likelihood ReRank

  1. Instruct LLMs for zero-shot passage re-ranking
  2. Algorithmically selecting few-shot demonstrations
  3. a novel demonstration selection strategy based on difficulty rather than the commonly used semantic similarity

将 query qqq 和 candidate document ddd 的匹配程度定义为：利用 document ddd 生成 qqq 的概率值。

ℓ(q∣d)∝∑i=1..Nlog⁡P(qi∣z1,…,zk,d,q1:i−1)\ell(q \mid d) \propto \sum_{i=1 . . N} \log P\left(q_i \mid z_1, \ldots, z_k, d, q_{1: i-1}\right) ℓ(q∣d)∝i=1..N∑​logP(qi​∣z1​,…,zk​,d,q1:i−1​)

zzz 为 positive query-document pair 。

Demonstrations for Ranking

问题在于如何选择适合的 demonstration zzz 。

这里讨论了三种方案：

  1. random sampling
  2. similarity-based selection (SBS)-> Similar Demonstration：双塔，cosine 相似度
  3. difficulty-based selection (DBS)-> Challenging Demonstration
    • 定义 difficulty
      方法1: 使用 demonstration query likelihood (DQL)

    DQL(z)∝1∣q(z)∣log⁡P(q(z)∣d(z))D Q L(z) \propto \frac{1}{\left|q^{(z)}\right|} \log P\left(q^{(z)} \mid d^{(z)}\right) DQL(z)∝∣∣∣​q(z)∣∣∣​1​logP(q(z)∣d(z))

    分数越低表示越困难
    方法2: 使用 conditional demonstration query likelihood (CDQL) ，用于建模 demonstration 序列的选择（ jointly consider the difficulty of all demonstrations in the prompt）

    CDQL⁡(z1,z2,…,zK)∝∑i=11∣q(zi)∣log⁡P(q(zi)∣z1:i−1,d(zi))\operatorname{CDQL}\left(z_1, z_2, \ldots, z_K\right) \propto \sum_{i=1} \frac{1}{\left|q^{\left(z_i\right)}\right|} \log P\left(q^{\left(z_i\right)} \mid z_{1: i-1}, d^{\left(z_i\right)}\right) CDQL(z1​,z2​,…,zK​)∝i=1∑​∣∣∣​q(zi​)∣∣∣​1​logP(q(zi​)∣z1:i−1​,d(zi​))

先验实验的结论：

  1. Increasing demonstrations does not necessarily help：
    demonstration 数量不是越多越好。随着数量增加，最终的平均效果会轻微上升，然后下降；但整体方差会减小。

  2. Demonstrations influence ranking：
    demonstration 的选择很重要，下图为随机选择100次进行 one-shot demonstration rerank，可以发现方差很大，并且one-shot 并不总是由于 zero-shot 。

  3. similarity-based selection 效果一般， high semantic similarity 和 strong re-ranking 之间没有什么 correlation ，也就是不太相关。
