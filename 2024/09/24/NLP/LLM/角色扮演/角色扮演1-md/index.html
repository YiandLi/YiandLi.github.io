思路

让大模型扮演角色 RRR ，问题为 qqq ，从知识库中抽取所有小说中「角色 R 和其他人的相关对话信息」：

D(q,R)=(u1,v(u1;R)),…,(uM,v(uM;R))D(q, R)=\left(u_1, v\left(u_1 ; R\right)\right), \ldots,\left(u_M, v\left(u_M ; R\right)\right) D(q,R)=(u1​,v(u1​;R)),…,(uM​,v(uM​;R))

umu_mum​ 是其他角色提出的问题
v(uM;R)v\left(u_M ; R\right)v(uM​;R) 为角色 RRR 的回答

当然需要同时提供 system Prompt ，然后可以在问答对基础上额外检索相关context文，如何构造知识库。

SystemPrompt

基础版本：

需要在基础版本上进行 2 点改进：

  1. 问题：经过 RLHF ， LLM 的输出不会重复 context ，但是当前角色扮演场景则是需要重复提供的 context 信息。场景匹配时最好需要做到完全复制。
    解决：显示告诉他可以直接复制原始回复。

  2. 问题：尽管有 RAG ，模型语言习惯还是比较客气僵硬。
    解决：额外补充角色的性格特点，语言习惯等。注意输出结果对这部分的内容十分敏感。

** 部分是新增加的内容。

数据构建

抽取出的数据不是QA的形式，所以要对数据进行合成。

ChatHaruhi主要采取的方法是通过问题来生成对话。将目标角色发言前的内容作为question，让LLM继续完成这个dialogue，基于已有的对话进行合成。
但是有的角色的台词数量较少，所以需要对数据进行增强。论文中采用了问题生成的方式，让LLM参考已有的QA对，生成新的问题，再根据新生成的问题生成对话。
