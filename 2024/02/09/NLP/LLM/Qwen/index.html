Qwen

https://github.com/QwenLM/Qwen

https://huggingface.co/Qwen

Data

  • 数据来源：中英文的web文档、百科全书、书籍、代码等。
  • 获取手段：从HTML中提取文本，并使用语言识别工具确定语言。
  • 去重处理：精确匹配去重，MinHash & LSH 模糊匹配去重。
  • 删除低质量数据：rule based & machine learning - based method，即使用多个模型去打分，包含语言模型，文本质量评分模型，专门用于检测识别潜在冒犯性或不恰当的内容的模型，还有人工审查。
  • 引入高质量指令数据
  • 最终数据量为 3万亿个token

Tokenizer

  • BPE算法：select the vocabulary cl100k base as our starting point
  • 数字作为单独的Token
  • 词表大小：152K

Model

常规：

LayerNorm + RMSNorm

SwiGLU FFN ：4 → 8/3

FlashAttention

RoPE (Rotary Positional Embedding)，计算时候使用 float32 类型

context lengths of 2048

额外的改动：

Embedding与输出层：千问模型没有将Embedding层和输出层的参数共享，通过增加显存以期换取更优的性能

Bias：只在Attention层添加Bias，其他层（dense，norm）去除了Bias，以期有更好的外推性能

长度外推

长度外推性是一个训练和预测的长度不一致的问题:

1、预测的时候用到了没训练过的位置编码（不管绝对还是相对）；

2、预测的时候注意力机制所处理的token数量远超训练时的数量：导致注意力更加分散（或者说注意力的熵变大），从而导致的训练和预测不一致问题

作者的方法 training-free techniques ，也就是不需要去训练 ：

  • Dynamic NTK-aware interpolation → YaRN

  • LogN-Scaling：https://spaces.ac.cn/archives/9444

    根据「推理上下文长度」和「训练上下文长度」，重新缩放 attention 点积的值，确保随着句长的增加，交差熵、点积等值保持稳定。

    softmax⁡(QK⊤d)V ⟹ softmax⁡(log⁡mndQK⊤)V\operatorname{softmax}\left(\dfrac{Q K^{\top}}{\sqrt{d}}\right) V \ \Longrightarrow\ \operatorname{softmax}\left(\dfrac{\log _m n}{\sqrt{d}} Q K^{\top}\right) Vsoftmax(d​QK⊤​)V ⟹ softmax(d​logm​n​QK⊤)V

    mmm 为训练长度，nnn 为预测长度 ；实现时， nnn 为当前 query token 的 id 。

    从下面源码可以看到，这个操作仅推理时使用。用在 Rope 变换之后，key 维度为 [b,s,h] ，query 维度为 [b,1,h] 。

    seq_start:seq_end 等价于 seq_start ，表示 query token 的 id，或者说前缀 token 的长度，正好对应了公式中的 nnn 。

这样做可能仍然有问题，详见 ： leshare : LogN-Scaling 在Alibi下的校正

  • Window Attention：将注意力固定在一个窗口，防止模型关注到句子太远的地方。
    同样观察到：低层 layer 对长度外推更加敏感，因此不同层选择不同的 window size ，底层采用更小的窗口。
    从实验看，好像这个上下文窗口最大为 8192 。
