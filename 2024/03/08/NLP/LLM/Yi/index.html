Data Processing

一个 Pipeline 结构：

  1. Common Crawl 的网络文档
  2. 使用 CCNet 进行语言识别和困惑分数评分
  3. 规则过滤器，过滤低质量文本：
    1. url，域名，单词黑名单和乱码文本过滤。
    2. 文档长度、特殊符号比例以及短、连续或不完整行的比例；
    3. 去重复：重复单词、n-gram 或段落
    4. 识别和匿名化个人可识别信息（PII），如电子邮件地址和电话号码
  4. 可学习过滤器：
    1. 困惑评分器：使用 KenLM 丢弃那些困惑分数明显高于平均水平的文档。
    2. 安全评分器：一个经过训练的分类器，根据是否比较像维基百科数据页面，分配分数，丢弃分数低的。
    3. 文档连贯性评分器：识别由不连贯的句子或段落组成的低质量网络文档，然后分割或者直接删除。
    4. 安全评分器：识别并移除包含有害内容的网络文档，如暴力、色情和政治宣传。
  5. 基于聚类的过滤器：
    使用无监督的语义聚类来对网络文档进行分组，聚类数据随后标注了质量标签，为优化Yi的数据混合策略提供了必要的参考。
  6. 去重：
    1. 受到 Falcon 的 The RefinedWeb Dataset 启发，整合了基于文档的 MinHash 去重和 子文档精确匹配 的去重，有效识别并移除文档内部和跨文档中的重复内容。
    2. 然后使用主题模型进行文档分类。
    3. 在最终的预训练数据集中，我们对不太有帮助的内容，主要是广告，进行了降采样。

Tokenization

SentencePiece BPE ； 词汇量为 64_000
将 numbers 分为单个 digits
允许罕见字符回退到unicode-byte编码
使用 identity tokenizer 防止标点符号退化为半角类型
并没有在英文单词前面使用 dummy prefix (whitespace at the beginning of text)

主要还是 LLAMA2 结构，改动如下：

  1. Attention Mechanism ：LLaMA 2 仅仅在 70B 的模型上使用了 Grouped-Query Attention(GQA)，而 Yi-6B 和 Yi-34B 全都使用了 GQA 。在6B模型上使用GQA后，我们没有观察到性能下降。

  2. Activation Function ：在 post-attention layer 中使用了 SwiGLU，并且中间激活量从 4h4h4h 降低到 8/3h8/3h8/3h 。

  3. Positional Embedding and Long Context ：使用 Rotary Position Embedding (RoPE) ，并且使用了 base frequency (RoPE ABF) ，以支持长上下文窗口长达200K，而基础模型本身是在4K上下文长度上进行训练的。
    为了让基础模型适应更长的上下文，继续在混合预训练数据上对该模型进行微调，其中大部分样本来自上采样过的书籍类别。
    观察到，只用 1-2B token 就足以使模型在4K到200K长度上收敛到极低的损失值，并且轻量级的微调进一步产生了接近完美的长上下文检索性能。
    作者认为，模型能够建模比预训练长度(4K)更长的依赖关系是一种内在的能力，基础模型已经具有建模超过4K依赖的能力，即使模型训练时间较短（100 steps），后训练/微调过程也简单地释放了这种能力。

Fine tuning - Instruction tunning

a sequence length of 4096, alongside a batch size of 64

强调数据质量，而不是数量。方法和 LIMA ，DEITA 保持一致。由于规模不超过10K，团队能够检查并优化每一个数据点。

Data Preprocessing

微调数据集包括不到 10,000 个 multi-turn instruction response dialog ，每条都经过多次构建和修改，并根据用户反馈进行了完善。

  1. prompt distribution selection 复合指令: 作者使用了复合类型的 instruction ，并且在训练过程中不断加大难度。
  2. response formatting 响应格式: 使用 LIMA 的默认样式，即「介绍-主体-结论」的形式结构化 。
  3. CoT data formatting 思维链数据格式: 使用了 “Step-Back”模式
  4. reduce hallucination 减少幻觉: 检验并确保响应中的知识不包含在模型记忆中，删除会导致模型记忆幻觉的响应。
  5. reduce repetition 减少重复: 重写了微调数据中通常存在但可能被忽视的重复轮次的响应。

数据格式：ChatML

训练设置：300 steps，4096 sequence length ，64 batch size。

训练方法：仅仅计算 response 的损失

长上下文窗口支持

包括 Continue Pretraining 和 finetuning

不修改模型架构，仍然是全注意力，完全基于工程：实现和改进计算 - 通信重叠、序列并行计算和通信压缩，实现了最高 200K。

Continue Pretraining

sequence parallelism and distributed attention

数据混合包括：（1）原始预训练数据（2）长度上采样的长上下文数据，其中长文章大多来自书籍；（3）多文档问答合成数据，构造了QA数据，其中答案为相关的上文。

continue pretrain the model on 5B tokens with 4M batch size, which translate to 100 optimization steps.

Supervised Finetuning

mix our short-context SFT data with long-context document question answering data

use model-assisted automated methods (i.e., synthetic data) to construct document QA：随机将多篇文档连接成一个序列，从长序列中随机抽取一个或多个段落，并要求聊天模型基于抽样的段落构建问题和答案对。一个重要的细节：在给出答案之前，我们要求模型陈述原始段落。这种数据格式鼓励模型的检索行为，因此防止产生幻觉行为。

Vision-Language

受开源LLaVA的启发，这里基于 Yi-6B-Chat 和 Yi-34B-Chat 提出了 Yi Vision Language（Yi-VL）的 Yi-VL-6B 和 Yi-VL-34B。

三个模块：

  1. 图像编码：CLIP ViT-H/14
  2. 投影模块：将图像编码映射到文本空间，有层归一化的两层多层感知器（MLP）
  3. 大型语言模型

Depth Upscaling

如何将我们的Yi-6B基础模型16层升级为一个名为Yi-9B基础模型的9B模型48层。

在决定要复制哪些层时，可以通过评估每一层的输入和输出之间的余弦相似度得分来进行决策。

这种对性能的最小影响是由于复制层的输入和输出之间存在接近1的高余弦相似度，复制这些层不会明显改变原模型产生的输出逻辑。

然后再持续训练。
