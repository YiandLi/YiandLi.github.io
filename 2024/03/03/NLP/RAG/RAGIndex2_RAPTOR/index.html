RAPTOR: RECURSIVE ABSTRACTIVE PROCESSING FOR TREE-ORGANIZED RETRIEVALf

现有方法仅从检索语料库中检索短的连续块，限制了对整个文档上下文的整体理解。

RAPTOR 模型引入了递归嵌入、聚类和总结文本块的方法，从下到上构建具有不同摘要级别的递归树。在这棵树中检索，在不同抽象级别集成冗长文档中的信息。

构建树

首先将检索出来的大型语料切分为 chunks of texts of length 100 ，如果一个句子超过了 100 token 的限制，则将整个句子转移到下一个 chunk 。

叶子结点：用 SBERT 进行 sentence embedding ，组成了整个 tree 的 leaf nodes 。

父亲节点：使用聚类算法进行聚类，使用 LLM 对 grouped texts 进行摘要，将摘要再进行 embedding 。

直到无法进一步聚类。

Clustering Algorithm

使用 Soft Clustering，即一个字节点可以属于多个父节点，并且不限制 cluster 的数量。

该聚类算法是基于 Gaussian Mixture Models (GMMs) 的：GMMs assume that data points are generated from a mixture of several Gaussian distributions.

给定 NNN 个 chunk embedding，其中每一个 text vector 为 XXX ，维度为 ddd 。

XXX 属于第 kkk 个高斯分布的概率被表示为：P(x∣k)=N(x;μk,Σk)P(\mathbf{x} \mid k)=\mathcal{N}\left(\mathbf{x} ; \mu_k, \mathbf{\Sigma}_k\right)P(x∣k)=N(x;μk​,Σk​) ，假设第 kkk 个高斯分布的权重为 πk\pi_kπk​ ，则 XXX 的出现概率为 P(x)=∑k=1Kπk N(x;μk,Σk)P(\mathbf x) =\sum_{k=1}^{K} \pi_k \ \mathcal{N}\left(\mathbf{x} ; \mu_k, \mathbf{\Sigma}_k\right)P(x)=∑k=1K​πk​ N(x;μk​,Σk​) 。

首先使用，Uniform Manifold Approximation and Projection (UMAP) 用于降维，方便距离度量。

超参数 n_neighborsn\_neighborsn_neighbors 表示近邻数量，决定了局部和全局平衡。

作者改变 ∗n_neighbors*n\_neighbors∗n_neighbors* 以创建分层聚类结构：它首先识别全局聚类，然后在这些全局聚类内执行局部聚类。因为后续需要进行摘要，如果一个聚类中的 text 组成的 chunk 超过了 LLM 的 最大长度，则在其内部再次使用聚类算法。

为了决定聚类的数量，作者使用了用「 贝叶斯信息准则 Bayesian Information Criterion (BIC) 」 ，在惩罚复杂度的同时奖励优秀的拟合。

树查询

两种方法：tree traversal 树遍历（逐层遍历树，剪枝并选择每一层最相关的节点）&& collapsed tree 树折叠（集体评估所有层的节点，以找到最相关的节点） 。

Tree Traversal 树遍历

不断根据余弦相似度选择前 kkk 个最相关的根节点，直到叶子结点。还有个超参数 ddd 用于控制深度。

最后将每一层索引到的 k 个节点的所有 text 拼接在一起，作为 relevant context 。

通过调整深度 d 和每层选择的节点数 k ，树遍历方法可以控制检索信息的特异性和广度。

Collapsed Tree 树折叠

折叠树方法通过同时考虑树中的所有节点来提供一种更简单的搜索相关信息的方法，该方法不是逐层进行，而是将多层树展平为单层，本质上是将所有节点放在同一级别上进行比较。

首先，将整个 RAPTOR 树折叠成单层。这个新的节点集表示为 ∗C*C∗C* ，包含原始树每一层的节点。

选择与查询具有最高余弦相似度分数的前 ∗k*k∗k* 节点。然后不断从中添加 text，直到满足模型最大输入长度，因为前文设置 chunk length=100，这里检索的段落数量大概为 max length // 100。

通过实验，发现 Collapsed Tree 树折叠 效果更好。通过同时搜索所有节点，没有那么死板。但是缺点是需要 query 所有的 node ，但是可以使用 FAISS 。
