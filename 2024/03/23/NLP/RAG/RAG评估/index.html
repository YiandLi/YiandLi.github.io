Benchmarking Large Language Models in Retrieval-Augmented Generation

主要是用于评估 RAG 系统中的 LLM ，提出了四个方面：

  1. 噪声鲁棒性：从噪声文档中提取有用的信息

    • 测试样本构造：检索上下文存在一些噪声文档，噪声文档被定义为即与问题相关但不包含答案的任何信息的文档。
    • 准确度：用于衡量噪声鲁棒性和信息集成度。 我们采用精确匹配方法，如果生成的文本包含与答案完全匹配的内容，则将其视为正确答案。

  2. 否定拒绝：当任何检索到的文档中不存在所需的知识时，大语言模型应该拒绝回答问题。

    • 测试样本构造：检索上下文仅包含噪声文档
    • 拒绝率：当只提供嘈杂的文档时，大语言模型应该输出具体内容——“由于文档中的信息不足，我无法回答问题。” （我们使用指令来通知模型。）。 如果模型生成此内容，则表明拒绝成功。

  3. 信息集成：是否能够回答需要集成多个文档信息的复杂问题，这时模型需要整合多个检索到的 chunk 。

    • 准确度用于衡量噪声鲁棒性和信息集成度。 我们采用精确匹配方法，如果生成的文本包含与答案完全匹配的内容，则将其视为正确答案。

  4. 反事实鲁棒性：是否能够识别检索文档中已知事实错误的风险。

    • 测试样本构成：包含大语言模型可以直接回答的实例，但外部文档包含事实错误。
    • 错误检测率：当提供的文档包含事实错误时，模型应输出具体内容——“所提供的文档存在事实错误”。 （我们使用指令来通知模型。）。 如果模型生成此内容，则表明模型检测到文档中的错误信息。
    • 纠错率： 在识别事实错误后，要求模型生成正确的答案。 如果模型生成正确的答案，则表明该模型有能力纠正文档中的错误。

Retrieval-Augmented Generation Survey

Retrieval-Augmented Generation for Large Language Models: A Survey

提到两种评估 RAG System 的方法：

  1. 独立评估

    1. 评估RAG检索模块的性能：命中率、MRR、NDCG

    2. 评估生成模块：衡量生成器是否能利用检索到的内容来生成问题的答案 。

  2. 端到端评估，直接根据最终生成的 prediction 和 label 进行任务上的评估 。

    端到端评估可以分为手动评估和使用LLM的自动评估 。
    例如用于问答任务的EM，用于摘要的UniEval和E-F1任务，以及用于机器翻译的BLEU …

通用的评估指标：

  1. 答案的可信度 - generator ：确保答案与 retrieved context 内容一致，防止幻觉
  2. 答案相关性 - generator ：response 和 question 相关
  3. 上下文相关性 - retriever ：检索的上下文信息尽可能准确和有针对性，避免不相关的内容 。

真实工程落地中:

  1. 噪声鲁棒性 - generator ：生成时可以自动过滤一些不相关内容。噪声文档：与问题相关但不包含有用信息的文档。
  2. 否定拒绝 - generator ：检索内容和问题不相关时，模型会拒绝进行生成。测试方法：提供的检索上下文中全部都是噪声不相关文档
  3. 信息整合 - generator ：是否可以集成多个文档中的信息来回答更复杂的问题。
  4. 反事实鲁棒性 - generator ：是否能够识别并处理文档中已知的错误信息。测试方法：问题模型可以直接回复，但是提供的检索上下中包含错误的信息
