RAG评估 2：Info-RAG

Posted on 2024-03-20 In NLP , RAG
Symbols count in article: 1.2k Reading time ≈ 8 mins.

WeChat AI : Unsupervised Information Refinement Training of Large Language Models for Retrieval-Augmented Generation

问题：LLM 在训练时没有考虑 RAG context ，也就是面对质量参差不齐的检索文本，LLM 不知道如何去使用它们。

本文中，作者

  1. 将 LLM 重新定义为 Information Refiner ，不仅仅是根据 context 生成 response，根据检索到的文本，生成「正确/完整/有用」的 response 。
  2. 提出了 INFO-RAG ，用来无监督地训练 LLM，以更加适配 RAG 。
  3. 使用 Info-RAG 训练 zero-shot LLaMA2 ，在 Question Answering, Slot-Filling, Language Modeling, Dialog, and Code Generation 任务上效果都很好 。

Our INFO-RAG

RAG 中，retrieved texts 可以被分成三类：

  1. 检索文本内容很全并且正确：希望模型能filter多余信息，利用正确信息
  2. 检索文本包含了部分相关信息，但是也包含了 incomplete / incorrect 信息：希望模型能自己学会识别，补全，并利用正确信息
  3. 检索文本没有价值，是无效的 ：但是他们的语义和query的语义是近似的，所以我们希望模型能利用到这部分语义信息

所以希望 LLM 具备的能力为：利用有效信息，筛除错误信息，纠错，补全 。

Unsupervised Learning

样本为 input-output pair 的形式，数据源为 English Wikipedia

作者会根据构造的数据，使用 next token prediction 任务，继续预训练 LLM / prefix language modeling 。

训练数据构造

对于每一个 wiki document ，将其截断为 kkk 个文本段 S=[s1,s2,...,sk]S=[s_1,s_2,...,s_k]S=[s1​,s2​,...,sk​] 。

然后随机从 SSS 中选择一个文本段 sls_lsl​ ，在前 1/3 - 2/3 个文本段中 进行随机截断，得到 prefix slps_l^pslp​ ，其余 token 作为 target response slts_l^tslt​ 。

为了模拟 RAG，需要定义 retrieved sentences R(slp)\mathcal R(s_l^p)R(slp​) ，定义方法为：

  1. Select and Copy - extract the correct knowledge from the complex texts - 20% 数据

    直接将全部文本 SSS 作为 Context ，也就是 slp,slts_l^p, s_l^tslp​,slt​ 其实都是包含在 Context 中的

  2. Correct and Complete - verify knowledge, utilize accurate knowledge, rectify incorrect knowledge, and complete missing knowledge - 40% 数据

    给定 SSS 中每一个文本段 sis_isi​ ，先去计算其内部每一个 token 的概率分布，也就是使用 si<as_i^{<a}si<a​ 去计算 sias_i^{a}sia​ 的概率值：

    dj(si[a]∣si<a)=softmax⁡(WHj[a])d_j\left(s_i^{[a]} \mid s_i^{<a}\right)=\operatorname{softmax}\left(\mathbf{W H}_j^{[a]}\right) dj​(si[a]​∣si<a​)=softmax(WHj[a]​)

    $j $ 表示 layer index ， W\mathbf{W}W 是 vocabulary head 的参数 。
    如果某个 token 在相邻层之间的概率值分布不稳定，则认为这个 token 是 informative 的 。

    于是使用 Jensen-Shannon Divergence 去度量层与层之间 token distribution 的差距，由此定义 token stability：

    Oi[a]=arg⁡max⁡j∈JJSD⁡(dN(si[a]∣si<a)∣∣dj(si[a]∣si<a))O_i^{[a]}=\arg \max _{j \in J} \operatorname{JSD}\left(d_N\left(s_i^{[a]} \mid s_i^{<a}\right)|| d_j\left(s_i^{[a]} \mid s_i^{<a}\right)\right) Oi[a]​=argj∈Jmax​JSD(dN​(si[a]​∣si<a​)∣∣dj​(si[a]​∣si<a​))

    这里 JJJ 选择 0-th 到 N/2-th ， dN()d_N()dN​() 表示最后一层的 token distribution 。

    也就是从浅层中和最终输出分布差距最大的分数表示这个差距分数，差距越大，说明 token 信息量越大。

    然后就可以得到这个句子中每一个 token 的 stability：

    Oi={Oi[0],Oi[1],…,Oi[n]}\mathbb{O}_i=\left\{O_i^{[0]}, O_i^{[1]}, \ldots, O_i^{[n]}\right\} Oi​={Oi[0]​,Oi[1]​,…,Oi[n]​}

    从中选择 top 50% 作为 informative token时，然后用 30% 的概率去随机挑选其中的 token（0.5*0.3=0.15），对于这 15% 的token，50%使用 [MASK] 替换， 40% 替换为其他 token，剩余不变。这些不变的 token 就是用于维持原本的信息。

    对 SSS 中所有的句子都进行上述操作，最后得到 S′S'S′ ，作为 RAG 中的 Context 用于训练。

    Jensen-Shannon Divergence 真的有用？没有见到这部分消融实验

  3. Contextual Stimulation - find answers from their knowledge based on relevant texts in context - 40% 数据

    这部分就很简单了，在 SSS 中删除 query slps_l^pslp​ 和 answer slts_l^tslt​

    可以认为 S−sS-sS−s 虽然不包含有用信息，没有办法帮助生成 slts_l^tslt​ ，但是语义上和 slps_l^pslp​ 是相似的，我们希望 LLM 可以根据这些语义激发自己内部的知识。

个人思考

出发点是好的，retrieved context 不可控，可能情况有「完全正确」「部分正确，信息不全」「不正确，会误导模型」，LLM 最好可以有针对性地训一训 。

如果想自己使用，是不是可以有针对性地构造一些数据？上面的构造方法还是太 general 了 。

不期望模型学习到知识，也只是想一个办法去激发模型的能力，适应 RAG 这种范式。或者说稍微削弱下模型过度依赖上下文，这样也许会更加依赖自身的参数记忆，那么又会有幻觉，所以不期望模型学到东西，pre-train 还是最重要的 。
