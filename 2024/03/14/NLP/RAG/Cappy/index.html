Cappy

Posted on 2024-03-14 In NLP , RAG
Symbols count in article: 496 Reading time ≈ 2 mins.

用于给 [ Instruct : response ] 打分，训练时使用回归任务。

预训练数据

  • the same dataset collection that is utilized by T0

    这个数据集由 39 个不同的子数据集组成，涵盖广泛的任务类型，例如问答、情感分析和摘要等。

    每个数据集都和一个或者多个模版有关系，最多 500_000 个样本。

  • 数据构建：预训练期间的每个数据实例必须有一个文本对（指令，响应），并加上与指令相关的响应的正确性标注。

    • Ground Truth (score 1.0): 预训练数据集中的所有 ground Truth 指令-响应对。
    • Incorrect Responses (score 0.0): 将上述数据进行交错，产生不匹配的数据对。
    • Data Augmentation (score within [0, 1]): 对于生成任务中的每个实例，利用现有的多任务大语言模型(BART, T0-3B)，通过根据给定指令进行采样来生成多个响应。分别使用top-k 和 top-p 采样生成两个样本。
      然后利用响应与实例的真实响应之间的相似性 (rouge) ，为指令和每个响应形成的对分配一个标注。

模型和训练
RoBERTa base / medium / large + linear layer regression head
回归任务

应用 Cappy
给定一个 Instruct 和多个 response ，分别为每一对进行打分，选择分数最高的输出即可。

实验
使用不同的生成措施，选择分数最高的，发现效果很好。
同时，样本数量越高，分数越好。

个人思考

  1. 直接辅助生成：让模型通过不同的生成策略生成多次（ 温度，top k/p，beam size ），或者 RAG 中选择的 context 不同，来生成多个response，然后选择分数最高的。
  2. 可以辅助验证：当前 RAG 检索的数据是否可以辅助生成？
  3. 作为 confidence 判别的方式：zero-shot & no RAG 生成，如果打分比较低，则再使用 RAG ；分数高可以直接输出。
