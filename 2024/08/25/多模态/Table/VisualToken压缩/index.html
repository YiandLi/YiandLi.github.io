TokenPacker

TokenPacker: Efficient Visual Projector for Multimodal LLM

作者自己的解读：多模态大模型中高质量视觉token压缩方法-TokenPacker

关于 LLAVA 的 Visual Token ：

  1. 如果比较多，则计算量比较大
  2. 使用 MLP 进行直接降维会带来冗余

输入图像 Iimg∈RH×W×3I_{img}\in \mathbb R^{H\times W\times 3}Iimg​∈RH×W×3 ，对应了 N=HW/P2N=HW/P^2N=HW/P2 个 Patch ，然后通过一个 Projector 得到 MMM 个 visual token。
对于 Qformer， MMM 是自定义的；对于 LLava， M=NM=NM=N ，因为 patch 存在冗余，所以希望满足：

  1. M<NM \lt NM<N ，保证模型计算效率高。
  2. 少量的视觉token下全面地表达出输入图像的特征表示。

本论文引入了 TokenPacker 作为 Projector，

  1. 将视觉特征进行 2D 下采样，得到低分辨率的视觉特征 Iv′∈RM×CI_v^{\prime} \in \mathbb{R}^{M \times C}Iv′​∈RM×C ，其中 M=N/s2M=N / s^2M=N/s2 ， sss 为下采样倍数 。
    从而低分辨率 Iv′∈R1×M×CI_v^{\prime} \in \mathbb{R}^{1\times M \times C}Iv′​∈R1×M×C 中每一个 token 对应原始分辨率 Iv∈Rs2×M×CI_v \in \mathbb{R}^{s^2\times M \times C}Iv​∈Rs2×M×C 中的一个 s2s^2s2 区域。
  2. 然后用低分辨率中的 token 去 cross attend 原始分辨率区域对应的 s2s^2s2 个 token 。
  3. 过 MLP 映射到 text token 的维度。

此外也讨论了高分辨率场景下，如何进行 sub-image 切分，
理想的切分结果：

  1. padding的内容最小，图片本身内容比例最高。
  2. 为了防止图片被裁减，需要划分后图像的整体内容与原始图像大小重合度最大。

DeCo

分析为什么现在不用 Q-Former 了，以及 DeCo 作者本人的论文解读

DeCo: Decoupling Token Compression from Semantic
Abstraction in Multimodal Large Language Models

Deco 主要是分析了 Q-Former 的不足：

  1. 很难学好
  2. Q-former很容易成为MLLM中的一个bottleneck，丢失重要的视觉信息（因为 Query token 是稀疏的重复的，并且存在信息损失）。
  3. 在MLLM中，Q-former结构的视觉语义提取是多余的。
    经过多模态对齐后，LLM本身就是一个很好的视觉语义提取器。本质上，线性层或者MLP层映射后得到的还是patch级别的视觉特征、不是语义级别的，现在LLaVA路线强大的模型表现也证实了LLM能很好地提取视觉语义来生成文本回答。那么，Q-former对视觉语义的预提取其实就是多余的。

所以不如直接 MLP，Qformer 带来了信息损失，而 MLP 不会。
但是 MLP 出来的 token 数量多，所以用 token 压缩算法，变成了无损的低分辨率 token 。

方法很简单：使用 2D Adaptive Average Pooling ( torch 中的 AdaptiveAvgPool ) 。

原始 N 个 visual token 先转换为 2D 形式 ： (N12,N12)(N^{\frac 1 2}, N^{\frac 1 2})(N21​,N21​) 。然后用 2D pooling ，得到 (M12,M12)(M^{\frac 1 2}, M^{\frac 1 2})(M21​,M21​) ，最后 flatten 为 M 个 token 。

LLaVA-PruMerge

LLaVA-PruMerge: Adaptive Token Reduction for Efficient Large Multimodal Models

在 MLP 前面加入了一个 token prune & merge 模块

过程：

  1. Adaptive Important Token Selection (AITS) via Outlier Detection：
    首先，根据未修剪的视觉 token 与 CLS token 的空间相似性来选择合适的视觉 token。
  2. Token Supplement (TS) via Similar Key Clustering：
    然后，根据相似度对修剪后的标记进行聚类，并将聚类标记与未修剪的标记合并以弥补丢失的信息。

AITS

使用最后一层的全局表示 [cls] 去 attend 所有的 patch visual token，得到 attention 分数。

计算 attention score 的四分位数，即 Q3(p75) 和 Q1(p25) 的差距，然后定义区间 [low fence, upper fence]=[Q1−1.5×IOR,Q3+1.5×IOR][\text{low fence, upper fence}] = [Q1 - 1.5 \times IOR, Q3 + 1.5 \times IOR ][low fence, upper fence]=[Q1−1.5×IOR,Q3+1.5×IOR] ，传统异常检测中，在这个区间外的 token 是 Outliers 。

但在本文中，这些异常的 token 被视为在表示图像时起重要作用的标记。因为这些token 的注意力值超出了设定的“上限”，它们被认为是“异常”的，但这里的异常指的是它们的重要性或独特性高于其他标记。所以仅保留 Outlier Tokens 。

TS

修剪之余，也需要利用被 drop 的 token -》 Token merge

使用每个 patch 最后一层的 Key Vector 作为表征，进行聚类，patch-patch 相似度为：

Sim⁡(yi,yj)=ki⋅kjT\operatorname{Sim}\left(\mathbf{y}_i, \mathbf{y}_j\right)=\mathbf{k}_i \cdot \mathbf{k}_j^T Sim(yi​,yj​)=ki​⋅kjT​

每一个 保留的 token，选择最相似的 topk 邻居（包含自己），然后使用 attention layer 进行信息聚合。

PruMerge+: Bridging the Efficiency-Performance Gap

虽然这种压缩技术高效， 但在【TrainFree】 设定下 ，与原始的 LLaVA 模型相比，其 PruMerge 优化版本 LLaVA-PruMerge 在性能上出现了轻微差异。为了解决这一问题，我们引入了 PruMerge+，一个改进版的模型，能够在标记数量减少和模型性能之间实现最佳平衡。

PruMerge+ 改进了我们原有的方法，依然能显著减少视觉标记数量——平均减少四倍——且性能下降极小。这一改进在算法 1 中有详细说明，并基于第 3.2 节中列出的标记选择策略进行优化。PruMerge+ 的一个新特点在于其增强的标记选择过程。除了仅关注之前识别出的重要标记外，PruMerge+ 还扩展其范围，涵盖了最初被认为不太重要的区域中的额外视觉标记。通过按预定比例进行视觉标记的空间均匀采样，这一比例由异常标记的分布情况决定。该方法确保了视觉标记的选择更具全面性和代表性。

LLaVolta

LLaVolta: Efficient Multi-modal Models via Stage-wise Visual Context Compression

是将 token 压缩应用于训练，以提升训练效率，而不是推理时压缩。

  1. 提出 Visual Context Compressor ，用于 token 压缩。
  2. 提出 LLaVolta 训练框架，用于最小化压缩 token 带来的误差。

Compressor

生成 text response 的概率为：

p(Xa∣Xv,Xinstruct )=∏i=1Lpθ(xi∣Xv,Xinstruct ,<i,Xa,<i)p\left(\mathbf{X}_a \mid \mathbf{X}_v, \mathbf{X}_{\text {instruct }}\right)=\prod_{i=1}^L p_\theta\left(x_i \mid \mathbf{X}_v, \mathbf{X}_{\text {instruct },<i}, \mathbf{X}_{a,<i}\right) p(Xa​∣Xv​,Xinstruct ​)=i=1∏L​pθ​(xi​∣Xv​,Xinstruct ,<i​,Xa,<i​)

压缩：将原始 LLL 个 visual token 降采样到 LoutL_{out}Lout​ 个 visual token，降采样率为 SSS 时， Lout=L/SL_{out} = L/SLout​=L/S 。

目的：最小化 Xv\mathbf{X}_vXv​ 的长度，同时最大化 p(Xa∣Xv,Xinstruct )p\left(\mathbf{X}_a \mid \mathbf{X}_v, \mathbf{X}_{\text {instruct }}\right)p(Xa​∣Xv​,Xinstruct ​) 。

操作位置：在 LLM Transformer 中对第 k 层的 visual token 进行压缩。

作者尝试了多种方法：

  1. 随机 drop 1−1/S1-1/S1−1/S 的 token
  2. K-means
  3. Attention-based compression
  4. Attention-based token dropping
  5. Avg pooling with stride （ 本文应该是 1D 的 pooling ）

发现 Avg pooling with stride 效果最好，因为没有额外参数，同时最稳定。

从推理的角度来讲，压缩率可以被定义为 ：

CR=N⋅L(N−K)⋅Lout +K⋅L\mathrm{CR}=\frac{N \cdot L}{(N-K) \cdot L_{\text {out }}+K \cdot L} CR=(N−K)⋅Lout ​+K⋅LN⋅L​

即：「压缩前所有层的 token 总数」 / 「在第 K 层进行压缩后，所有层的 token 总数」

LLaVolta as a Lite Training Scheme

高压缩比情况下，可能会面临信息丢失，效果下降的问题。

这里引入一个三阶段训练框架，

  1. 浅层 + 高压缩率
  2. 深层 + 低压缩率
  3. 不压缩

感觉不太合理，为什么先压缩再不压缩，最后推理时应该是压缩的呀，顺序反过来是不是更好？
因为本文讨论的是高效训练，即训练时压缩，推理时不压缩。

LLaVolta incorporates stage-wise visual context compression to progressively compress the visual tokens from heavily to lightly, and finally no compression at the end of training, yielding no loss of information when testing.
