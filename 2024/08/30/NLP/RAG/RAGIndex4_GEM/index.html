GEM-RAG: Graphical Eigen Memories For Retrieval Augmented Generation

创新点：

  1. 得到 chunk 的 question，作为额外的检索选项。
  2. 使用 谱分析 来得到节点的聚类，同样也是用 LLM 进行 summarize ，得到「高阶 summarized node」（类似 Graph RAG），然后继续得到它的问题。

RAG 的研究难点：

  1. Recall: 如何有效地找到与提示最相关的文本块
  2. Construct: 如何在多个文本块之间建立概念性联系以回答复杂问题

本文提出了 Graphical Eigen Memories For Retrieval Augmented Generation (GEM-RAG)

  1. 为每一段 chunk 生成一系列问题，即实用性描述，来更好地“标记”每一块信息，这些问题有助于表达特定文本块具有哪些信息以及为什么它可能有用。
  2. 每一个 chunk 作为一个 node，用其问题的形似度作为节点之间的边的权重；然后通过谱分解，进行聚类，生成摘要节点。
  3. 检索过程中，先检索得到最接近的问题，然后将相关的子图中的 chunk set 喂给 LLM 。

Memory Graph Construction

Chunking ，固定长度分块

{CHUNK1…CHUNKn}\left\{ \text{CHUNK}_1 \ldots \text{CHUNK}_n\right\} {CHUNK1​…CHUNKn​}

Generating Utility Questions ，每一个 chunk 生成 m 个问题，

Q(CHUNKi,m)={qi1,qi2,…,qim}Q(\text{CHUNK}_i , m) = \left\{q_{i 1}, q_{i 2}, \ldots, q_{i m}\right\} Q(CHUNKi​,m)={qi1​,qi2​,…,qim​}

Text Embeddings ，得到原始chunk和生成的question的编码 。
chunk embedding: vi=E(CHUNKi)v_i = E(\text{CHUNK}_i)vi​=E(CHUNKi​)
question embedding: vij=[E(qij)+vi]/2v_{ij} = \left[E\left(q_{ij}\right)+v_i \right]/ 2vij​=[E(qij​)+vi​]/2

Building the Weighted Complete Graph 构造一个全连接有权图

  • Node：Text Chunk

N={CHUNK1,CHUNK2,…,CHUNKK}\mathcal{N}=\left\{\mathrm{CHUNK}_1, \mathrm{CHUNK}_2, \ldots, \mathrm{CHUNK}_K\right\} N={CHUNK1​,CHUNK2​,…,CHUNKK​}

每一个节点有其对应的问题：

Qt={qt1,qt2,…,qtm}Q_t=\left\{q_{t 1}, q_{t 2}, \ldots, q_{t m}\right\} Qt​={qt1​,qt2​,…,qtm​}

  • Edge：每两个节点之间都有边，边的权重为 ∑i=0mSIM⁡(vti,vv)\sum_{i=0}^m \operatorname{SIM}\left(v_{t i}, v_v\right)∑i=0m​SIM(vti​,vv​) 。

Building the Summary Nodes
对于整个图来说，其每一个「特征值 eigenvalue」 和其对应的「特征向量 corresponding eigenvector」 表示了图的一个 「特殊具体概念维度 conceptual dimension」 。
关键就是如果找到这些特征值和其内部的代表节点。

用上述相似度矩阵来表示边的权重矩阵：

S=(sij)n×nS=\left(s_{i j}\right)_{n \times n} S=(sij​)n×n​

将 S 转化为 a variant of normalized graph Laplacian LLL ：

L=D−1/2(S−I)D−1/2L=D^{-1 / 2}(S-I) D^{-1 / 2} L=D−1/2(S−I)D−1/2

然后进行「谱分解 spectral decomposition」：求解 Lx⃗=λx⃗L \vec{x}=\lambda \vec{x}Lx=λx
得到「特征值 eigenvalues」： λ1,λ2,…,λn\lambda_1, \lambda_2, \ldots, \lambda_nλ1​,λ2​,…,λn​
对应的特征向量 x1⃗,x2⃗,...,xn⃗\vec{x_1}, \vec{x_2}, ..., \vec{x_n}x1​​,x2​​,...,xn​​ 表示主要主题 principal themes 。
然后对于每一个特征向量，选择前 e 个 chunk node {xk1,xk2,…,xke}\{x_{k 1}, x_{k 2}, \ldots, x_{k e}\}{xk1​,xk2​,…,xke​} ，用于代表这个主题 。

然后使用 LLM 来进行摘要，得到 high-level information 。
然后作为新的节点放入图中，并且同时生成 m 个问题。

Retrieval

给定检索输入 p ，首先进行嵌入 vp=E(p)v_p = E(p)vp​=E(p) ，然后先检索 top k 问题，然后通过问题得到原始 chunk 和，并且拓展到其他 chunk 。
