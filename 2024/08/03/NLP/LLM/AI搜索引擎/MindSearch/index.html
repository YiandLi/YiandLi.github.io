Posted on 2024-08-03 In NLP , LLM AI Search

2024 0729 - MindSearch 思·索: Mimicking Human Minds Elicits Deep AI Searcher

作者提出一个 LLM-based multi-agent framework ，来支持 AI搜索 ，其包含：

  1. WebPlanner - dynamic graph construction ：将 query 拆分为 sub-questions，即图中的节点。并且在任务过程中自动增加节点，进行图拓展。
  2. WebSearcher - hierarchical information retrieval ：具体执行搜索工作，返回搜索结果。

WebPlanner - a plan agent

将任务定义为一个 a directed acyclic graph (DAG) 结构 <V,E><V,E><V,E>
每个节点可以看成是一个问题。大问题可以被拆分为子问题，或者一个问题得到答案后引出下一个问题，被表示为「一个节点指向另一个节点」的形式。于是 起始节点为 query，终止节点为 answer 。

在每一轮次中：
prompt 为「整个图结构」和「历史搜索信息 context environment」。
然后 LLM 生成 thoughts 和 新node。作者将图结构相关定义和操作都转化为 code 形式。
最后 Web Searcher 根据 新node 信息进行检索，提供答案作为 context environment 。

WebSearcher - a RAG agent

先重写为多个 query ；然后多源搜索（Google, Bing, and DuckDuckGo）；合并去重；生成答案

同时为了让 agent 关注全局信息：
prefix the response from its father node as well as the root node when executing each search agent

LLM Context Management

Planner 不需要看到搜索内容，只需要看到 node 的问题，和最终的结果即可。
Searcher 只对问题负责，所以只看到问题和搜索内容即可。

思考

默认 node 必搜索。那一个 node 不需要搜索只是要拆解成多个子问题时，怎么办呢？
