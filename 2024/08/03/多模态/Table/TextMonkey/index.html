TextMonkey

TextMonkey: An OCR-Free Large Multi-modal Model for Understanding Document

场景：a large multi-modal model，text-centric tasks

  • Scene Text-Centric tasks
  • Document-Oriented tasks
  • Key Information Extraction

Intro

Related Work

  1. Fig. 1 (a)：之前的工作使用 vit 进行编码，但是他们的分辨率只有 224 / 336 ，原始的图片会被降采样，无法分辨 small texts，只能看到 large texts 。
  2. Fig. 1 (b)：UReaer 和 Monkey 使用采取了一种裁剪策略来扩展输入分辨率。然而，这种裁剪策略可能会无意中将相关单词分割开，导致语义不连贯。
    -> missing cross-window context
  3. Fig. 1 (c)：DocPedia 直接在频域而不是像素空间处理视觉输入。由于频域的特性，它可以在不丢失信息的情况下快速扩展分辨率，但是无法利用 VIT 了。

This Work

  1. 提出 Split Module：为了处理高分图片，作者将原始的大图切分为 non-overlapping 的多个小图。
  2. 提出 Shifted Window Attention：小图内部的 patch 还是使用 vit 自己的 self-attention；同时提出 zero-initialization Shifted Window Attention 来处理 cross-window patches。
  3. 提出 Image Re-Sampler
  4. 提出 Token Re-Sampler：滑动窗口机制导致最终的输出序列长度变长，所以提出一个 re-sampler 剔除冗余 patch，仅仅保留有价值的 patch 。

METHODOLOGY

把原始的大图切分为多个非重叠的 448×448448\times 448448×448 小图，每小图内继续切为 14×1414\times 1414×14 的 patch 。

Attention

  1. Within Window: 每一个 window 内部的 patch 直接用 VIT attention 编码。
  2. Cross Window: 参考 Swin transformer ，对于不同 window 的 patch，设置一个 attention window，来滑动覆盖多个子图，然后进行覆盖范围内的 attention，但是这次每个 patch token 不 attend 原始小图内的 token 。为了更好地初始化训练，这个 Shifted Window Attention 被初始化为 0 ，这里使用了 Lora 的初始化方法。

Image Resampler

参考 Qwen-VL ：
就是一个 Q-Former 结构，随机初始化多个 query token（256 个），去 cross attend VIT 的输出。
为了保证位置信息，还给 Key value 引入 2D 绝对位置编码。

Token Resampler

参考 Token merging: Your vit but faster
使用 image tokens 的 cosine sim 表示相似程度，进而可以表示 importance：和其他 token 越相似，就越不重要。

此外，为了保留全局信息，使用选择出来的 important token 去 attend 原始的 256 token。

resample 的 代码： monkey_model/text_monkeymerge.py#L19

Position-Related Task

希望 LLM 不仅仅能提供答案，还能指出这个答案位于图片的哪一个位置。

  1. 作者人工标注了「位置线索」，加入到原始的答案中。
  2. 增加了额外的训练任务，以提高模型对文本位置的感知，例如文本定位（text spotting）和阅读文本（reading text）。
    为了标准化不同比例的图像，使用范围 (0,1000)(0, 1000)(0,1000) 来表示位置信息。因此，在分辨率为 (Hr×Wr)(H_r \times W_r)(Hr​×Wr​) 的图像中， xxx 将被标准化为 x/Hr×1000x/H_r \times 1000x/Hr​×1000 ， yyy 同理。

基座模型

一共9.7B

视觉编码器：1.9B，来自Qwen-VL的视觉编码器（Vit-BigG，听说基于OpenCLIP）。

图片重采样器：90M，256个可学习的 queries

Token 重采样率：the token resampler’s ratio (r) was set to 512 for images with a resolution of 896 and increased to 1024 for images with a resolution of 1344.

大语言模型：Qwen-7B

移位窗口注意力模块：45M

MiniMonkey

处理高分辨率：recent works have primarily employed the following strategies:

  1. Two vision encoders, one for processing high-resolution images and one for processing low-resolution images [69, 81, 21].
  2. Directly using visual encoders that support high-resolution input [38, 50].
  3. Using a cropping strategy to segment the high-resolution images into several low-resolution images [74, 37].


本文提出:

  1. train-free 的 cropping 方法：multi-scale adaptive cropping strategy (MSAC)
    现有的 cropping 策略存在锯齿现象 (sawtooth effect) 和幻觉问题 (hallucinations)，作者提出 MSAC。
    1) MSAC 首先根据网格的纵横比和分辨率，对预先设置的网格组进行 分层操作 (a stratified operation) 。
    2) 然后，它自适应地从每个分层中选择多个纵横比，确保 同一文本不会被分割到不同的图像中 。
    3) 会生成多个维度下的图片切分方法，一起送入 VIT + MLP，即生成 multi-scale representations 。

  2. training-free & parameter-free 的 token 压缩方法： a Scale Compression Mechanism (SCM)
    based on the input question，使用 LLM 中训练好的 Attention Layer 计算得到 attention weight，然后进行 visual token dropout 。

Multi-Scale Adaptive Cropping Strategy

一共有「三个层/角度」来切图片，每一层都有预定义的网格，每一层选择一个网格来切（ 如 Fig2.b ）：

  1. 详细图层 - The detailed layer AdA_dAd​ : providing detailed information
    因为有预定义的网格，所以也就是变相定义其最大最小 grid 数量，目的是让 sub-image 越大越好，让 object 更清晰。

  2. 自适应图层 - The adaptive layer AaA_aAa​
    根据详细图层自适应生成宽高比，确保详细图层 AdA_dAd​ 上的裁剪线与自适应图层 AaA_aAa​ 上的裁剪线不重叠 CId∩CIa=∅C_{I d} \cap C_{I a}=\emptysetCId​∩CIa​=∅ 。
    具体来说，如果详细图层的长宽比是自适应图层的长宽比的倍数，我们就会将其从自适应图层中移除，然后选择一个新的长宽比。这一过程可确保细节层和自适应层为模型提供不同的语义信息和视觉特征。

  3. 全局层 - a global view - 没有grid，直接 1:1

Multi-Scale Visual Representations Fusion：前文都是直接将不同层的 visual token 直接拼接；这里为了让他们充分交互，使用一个 LLM 进行 Self-Attention 。

Within the LLM, these multi-scale visual representations interact with each other through self-attention.

Scale Compression Mechanism

a parameter-free token compression method : the Scale Compression Mechanism (SCM)

出发点：对于一个训练良好的 MLLM，其内部的 LLM 具有挑选 Visual Token 的能力 。

做法：使用 LLM 的第一层和最后一层来进行 visual token dropout ，这里仅去在详细图层上进行下采样。
input visual token ： Vd∈RL1×C,Va∈RL2×C,Vg∈RL3×CV_d \in \mathbb{R}^{L_1 \times C}, V_a \in \mathbb{R}^{L_2 \times C}, V_g \in \mathbb{R}^{L_3 \times C}Vd​∈RL1​×C,Va​∈RL2​×C,Vg​∈RL3​×C
the textual token ： Tt∈RT×CT_t \in \mathbb{R}^{T \times C}Tt​∈RT×C

  1. 使用「 the adaptive layer, global, and textual token 」去 attend 「 the detailed layer tokens 」：

    Attn⁡w=softmax⁡((Cat⁡(Va,Vg,Tt)+PE⁡(Cat⁡(Va,Vg,Tt)))(Vd+PE⁡(Vd))TD)\operatorname{Attn}_{\mathbf{w}}=\operatorname{softmax}\left(\frac{\left(\operatorname{Cat}\left(V_a, V_g, T_t\right)+\operatorname{PE}\left(\operatorname{Cat}\left(V_a, V_g, T_t\right)\right)\right)\left(V_d+\operatorname{PE}\left(V_d\right)\right)^T}{\sqrt{D}}\right) Attnw​=softmax(D​(Cat(Va​,Vg​,Tt​)+PE(Cat(Va​,Vg​,Tt​)))(Vd​+PE(Vd​))T​)

  2. 然后在第一个维度上进行平均，得到「 the detailed layer tokens 」的权重：

    Attnw∈R(L2+L3+T)×L1⟶Mean PoolingWa∈RL1\mathbf{A t t n}_{\mathbf{w}} \in \mathbb{R}^{\left(L_2+L_3+T\right) \times L_1} \overset{\text{Mean Pooling}}{\longrightarrow} \mathbf{W}_{\mathbf{a}} \in \mathbb{R}^{L_1} Attnw​∈R(L2​+L3​+T)×L1​⟶Mean Pooling​Wa​∈RL1​

    保留 topk 个 「 the detailed layer tokens 」。
