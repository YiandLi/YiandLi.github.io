Abstract and Introduction

层次分类(Hierarchy Classification / HC)：类别存在层级关系，通常可以用「hierarchy label tree」上的「路」表示。
多标签分类(Multi-Label Classification / MLC)：一个样本同时属于多个类别。
层次多标签分类(Hierarchical Multi-Label Classification / HMC)：一条样本的标签同时包含「hierarchy label tree」中的「多条路径」。

有两种方式来解决层次多标签分类问题， local learning 或者 global learning ，对应不同的损失函数。

  • local learning：在每个非叶节点处判别关系，最后将所有预测串联起来。这种方法能够针对每个层级做出判断；但是依赖多个分类器，计算开销很大，容易过拟合。
  • global learning：将样本和其对应的类别组合起来，用一个分类器进行所有预测。这种方式实现简单，且避免了误差传递问题；但是无法利用细粒度的层级信息，容易欠拟合。

本文中，作者提出 HMCN：

  • 同时利用 local loss 和 global loss
  • 实现了两个不同结构的版本：
    • 前馈版本：HMCN-F of a feed-forward architecture
    • 迭代版本：HMCN-R of a recurrent architecture

两种网络结构都是由多个「local output layer」（对应 label tree 的每一层）和一个「global output layer」组成。

  • 每个「local output layer」输出一个层级的标签；对应一个 local loss；能让模型学到对应层级的「局部分类信息」。
  • 而「global output layer」则是从全局出发，捕捉累计的标签关系；对应一个 global loss；能够让模型学到「类别依赖信息」
(直接看下面的引用吧，词穷了。)
  • a multiple-output deep neural network
  • perform both local and global optimization
  • propagates gradients from multiple network outputs
  • a local output per hierarchical level + a local loss function is used for back-propagating the gradients from the classes in the corresponding level
  • a global output capturing the cumulative relationships that were forwarded across the entire network, back-propagating the gradients from all classes of the hierarchy

同时，作者还引入了一个基于层次结构的惩罚项（a hierarchical violation penalty）来鼓励模型输出符合遵循层次结构。

HMCN-F

为了最大化学习标签的层级关系，模型参数随着层次结构增加而增加，当层次结构较深时，模型参数量会变得巨大。

信息以两种方式在模型中传递：

  • global flow：从输入层流出，经过每一个「fully-connected (FC) layer」，最终得到一个「global output」。
  • local flows：从输入层流出，经过对应的「global FC layer」和「local FC layer」，最终得到多个「local output」。

为了得到最终预测结果，所有的「local output」会被拼接在一起，和「global output」做 pooling，得到一个统一的输出（consensual prediction）。

「global flow」能够在各层之间传递语义信息，同时「local flow」也会根据实际情况（具体当前是哪一个层级），通过「反向传播」调整「global flow」。

和上述「global output」类似， PLh∈R∣Ch∣\mathbf P_L^h\in \mathbb R^{|\mathbf C^h|}PLh​∈R∣Ch∣ ，第 i 个元素值表示当前「类别层级」 Ch\mathbf C^hCh 下类别 i 的概率值；而
每个标签层级对应标签类别不同，所以不同层级下权重矩阵的维度不同（a distinct weight matrix per level）；而这也不是分类树中的以节点的维度进行分类，而是在整个层上进行分类。

推理时，设置阀值，取大于阀值位置输出对应标签。

final predictions

使用「global output」和所有「local output」求最终类别概率：

PF=β(PL1⊙PL2⊙,…,PL∣H∣)+(1−β)PG\mathbf{P}_{F}=\beta\left(\mathbf{P}_{L}^{1} \odot \mathbf{P}_{L}^{2} \odot, \ldots, \mathbf{P}_{L}^{|H|}\right)+(1-\beta) \mathbf{P}_{G} PF​=β(PL1​⊙PL2​⊙,…,PL∣H∣​)+(1−β)PG​

⊙\odot⊙ 为 concat 操作；β∈[0,1]\beta\in [0,1]β∈[0,1] 为权重，作者这里取 0.5 。

conclusion

增加「local output layer」可以增加梯度信号，从而：

  1. 避免只用一个「global output」做判断，从而让模型记住局部信息。
  2. 防止一些神经元死亡
  3. 加速拟合（直接的效果）

每次计算「global activation」时，都会复用原始的文本表征，这有助于模型在不同分类层级处学习特定的特征，输出正确的「local output」。

HMCN-R

HMCN-F 的问题在于各层级之间的权重不复用，导致参数增长很快，不适用于层级过高的分类任务。
基于此，作者提出一个类似 LSTM 的递归版本，支持在每次计算「global flow」时共享权重，并且每次迭代的输出对应一个分类层级。

Model

每次迭代需要用到上一层的「global activation」 Ah−1\mathbf{A}^{h-1}Ah−1 和 隐藏层状态 Ch−1\mathbf{C}^{h-1}Ch−1 ；同时计算出遗忘门 Fh\mathbf{F}^{h}Fh ，输入门 Ih\mathbf{I}^{h}Ih 和 输出门 Oh\mathbf{O}^{h}Oh 。

得到「global activation」 Ah\mathbf{A}^{h}Ah 之后，和 HMCN-F 一样，需要使用一个转换矩阵将其转换为「local activation」 ALh\mathbf{A}_{L}^{h}ALh​ ，然后再次变换并且输出「local output」 PLh\mathbf{P}_{L}^{h}PLh​ 。

use a transition weight matrix to project global features Ah\mathbf{A}^{h}Ah into local ones, and individual weight matrices for providing level-specific class predictions.

conclusion

感觉只有global层可以共享参数，local层还是一样的相互独立，这一点是受到任务限制的，所以只能说是缓解了参数线性增长的缺点。

hierarchical violation

通过单独最小化local loss 和 global loss 并不能保证层次分类的一致性，因此文章提出了基于层次的惩罚损失来降低这种风险。这同样是一种后处理的方式，来保证所有的预测都满足层次限制（既然提到后处理，是不是需要）。
作者提出了一个先验知识：因为是先确定父节点再确定子节点，所以分类树父节点 Yin\mathbf{Y}_{in}Yin​ 的预测分数要比其子节点 Yip\mathbf{Y}_{ip}Yip​ 的概率分数高，表示为：

LHi=λmax⁡{0,Yin−Yip}2\mathcal{L}_{H_{i}}=\lambda \max \left\{0, \mathbf{Y}_{i n}-\mathbf{Y}_{i p}\right\}^{2} LHi​​=λmax{0,Yin​−Yip​}2

λ∈R\lambda\in \mathcal Rλ∈R 用来约束这个损失的重要性。

Final loss

min⁡W(LL+LG+LH)\min _{W}\left(\mathcal{L}_{L}+\mathcal{L}_{G}+L_{H}\right) Wmin​(LL​+LG​+LH​)

这里训练和预测存在 gap：预测时使用的是 PF\mathbf{P}_{F}PF​ ，而训练时使用的是 PG\mathbf{P}_{G}PG​ 和所有 PLh\mathbf{P}_{L}^hPLh​ 。不过问题不大，有效果就行。
