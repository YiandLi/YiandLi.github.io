GlobalPointer for NER

Posted on 2022-02-08 In NLP , Named entity recognition
Symbols count in article: 4.5k Reading time ≈ 23 mins.

材料：GlobalPointer：用统一的方式处理嵌套和非嵌套NER by 苏剑林

介绍

GlobalPointer

  • Global即利用全局归一化的思路来进行NER任务，无差别地识别嵌套实体和非嵌套实体。

常规的Pointer Network的设计在做实体识别或者阅读理解时，一般是用两个模块分别识别实体的首和尾，这会带来训练和预测时的不一致。而GlobalPointer就是针对这个不一致而设计的，它将首尾视为一个整体去进行判别，所以它更有“全局观”（更Global）。

  • Pointer即与CRF不同，使用指针的形式（start pointer & end pointer）预测实体。

在理论上，GlobalPointer的设计思想就比CRF更合理；而在实践上，它训练的时候不需要像CRF那样递归计算分母，预测的时候也不需要动态规划，是完全并行的，理想情况下时间复杂度是 O(1)!O(1)!O(1)! 。

苏神：简单来说，就是更漂亮、更快速、更强大！真有那么好的设计吗？不妨继续看看。

思路

NER任务上，输入序列长度为 nnn ，则候选实体数量为 n(n+1)/2n(n+1)/2n(n+1)/2 ，真实实体个数为 n(n+1)/2n(n+1)/2n(n+1)/2 中的 kkk 个。
局部针对不同的实体类型：设有 mmm 个实体类型，则为“ mmm 个 ‘从 n(n+1)/2n(n+1)/2n(n+1)/2 候选实体中选 kkk 个正确实体’” 的多标签分类问题。

实体打分

每个token的编码向量放入两个线性层start_FFN和end_FFN分别得到属于每个实体类别的「start向量」和「end向量」：

α\alphaα 表示一个实体类别，这里相当于针对不同的实体类别识别token的「start向量」和「end向量」。

于是我们得到「实体类别数量 mmm 」对分数序列， 实体类别 α\alphaα 的分数序列为：

[q1,α, q2,α, ..., qn,α][k1,α, k2,α, ..., kn,α]\begin{aligned} &[q_{1,\alpha},\ q_{2,\alpha},\ ...,\ q_{n,\alpha}]\\ &[k_{1,\alpha},\ k_{2,\alpha},\ ...,\ k_{n,\alpha}] \end{aligned} ​[q1,α​, q2,α​, ..., qn,α​][k1,α​, k2,α​, ..., kn,α​]​

相对位置

在前述过程中，没有使用到位置编码（当然可以使用bert作为encoder从而使用bert的编码方式，但是大多数只是使用bert embedding）。

在后面的实验中我们将会看到，加不加相对位置信息，效果可以相差30个百分点以上！

苏神这里使用了其自研的旋转式位置编码（RoPE），其实就是一个变换矩阵 RiR_iRi​ ，满足 RiTRj=Rj−1R_i^T R_j = R_{j-1}RiT​Rj​=Rj−1​ ：

评价指标

使用实体级别的F1，并非标注标签级别的F1。
GlobalPointer本身的预测和输出就是实体级别，达到了训练、评估、预测的一致性。

对比CRF

如果序列标注的标签数为 kkk ，那么逐帧softmax和CRF的区别在于：

  • 前者将序列标注看成是 nnn 个kkk 分类问题
  • 后者将序列标注看成是1个 knk^nkn 分类问题

这也暴露出问题：

  • 逐帧softmax过于宽松了，因为某个位置上的标注标签预测对了，不代表实体就能正确抽取出来了，起码有一个片段的标签都对了才算对。
  • CRF又太过严格，要求所有实体都预测正确才算对，只对部分实体也不给分。虽然实际使用中我们用CRF也能出现部分正确的预测结果，但那只能说明模型本身的泛化能力好，CRF本身的设计确实包含了“全对才给分”的意思。

相比之下，GlobalPointer则更加贴近使用和评测场景：它本身就是以实体为单位的，并且它设计为一个“多标签分类”问题，这样它的损失函数和评价指标都是实体颗粒度的，哪怕只对一部分也得到了合理的打分。因此，哪怕在非嵌套NER场景，GlobalPointer能取得比CRF好也是“情理之中”的。

Efficient GlobalPointer

GlobalPointer的一个改进版——Efficient GlobalPointer，它主要针对原GlobalPointer参数利用率不高的问题进行改进，明显降低了GlobalPointer的参数量，并且效果更好。

问题

根据前文：

qi,α=Wq,αhi+bq,αki,α=Wk,αhi+bk,αq_{i,\alpha} = W_{q,\alpha}h_i+b_{q,\alpha}\\ k_{i,\alpha} = W_{k,\alpha}h_i+b_{k,\alpha} qi,α​=Wq,α​hi​+bq,α​ki,α​=Wk,α​hi​+bk,α​

有多少种实体类型，就有多少个不同的 Wq,αW_{q,\alpha}Wq,α​ 和 Wk,αW_{k,\alpha}Wk,α​ 的向量。
设 Wq,α,Wk,α∈RD×dW_{q,\alpha}, W_{k,\alpha}\in \mathbb{R}^{D\times d}Wq,α​,Wk,α​∈RD×d ，则一共有 2nDd2nDd2nDd 个参数， DDD 为encoder输出向量维度(768)， ddd 为自定义隐藏层向量维度(64)， nnn 为实体类型数量。

解决思路

对于任意类型 α\alphaα ，打分矩阵内大多数实体应该都是non-entity，所以些非实体的正确打分都是负的。
NER任务可以拆分为 “实体抽取” 和 “实体分类” ，则求打分矩阵 sα(i,j)s_{\alpha}(i, j)sα​(i,j) 的方式对应转换为：

sα(i,j)=(Wqhi)⊤(Wkhj)+wα⊤[hi;hj]s_{\alpha}(i, j)=\left(\boldsymbol{W}_{q} \boldsymbol{h}_{i}\right)^{\top}\left(\boldsymbol{W}_{k} \boldsymbol{h}_{j}\right)+\boldsymbol{w}_{\alpha}^{\top}\left[\boldsymbol{h}_{i} ; \boldsymbol{h}_{j}\right] sα​(i,j)=(Wq​hi​)⊤(Wk​hj​)+wα⊤​[hi​;hj​]

第一项对应实体抽取部分，所有的实体类型的参数 Wq,  Wk\boldsymbol{W}_{q},\;\boldsymbol{W}_{k}Wq​,Wk​ 都相同。
第二项对应分类不同的实体， hih_ihi​ 和 hjh_jhj​ 分别表示span的起始和结束位置的向量维度(768)。
wα∈R2D\boldsymbol{w}_{\alpha}\in \mathbb{R}^{2D}wα​∈R2D ，则参数量一共为 2D(d+n)2D(d+n)2D(d+n) 。

进一步地，记 qi=Wqhiq_i = \boldsymbol{W}_{q} \boldsymbol{h}_{i}qi​=Wq​hi​ ， kj=Wkhjk_j=\boldsymbol{W}_{k} \boldsymbol{h}_{j}kj​=Wk​hj​ ，则 [qi,ki][q_i,k_i][qi​,ki​] 可以用来代替 hih_ihi​ ：

sα(i,j)=qi⊤kj+wα⊤[qi;ki;qj;kj]s_{\alpha}(i, j)=\boldsymbol{q}_{i}^{\top} \boldsymbol{k}_{j}+\boldsymbol{w}_{\alpha}^{\top}\left[\boldsymbol{q}_{i} ; \boldsymbol{k}_{i} ; \boldsymbol{q}_{j} ; \boldsymbol{k}_{j}\right] sα​(i,j)=qi⊤​kj​+wα⊤​[qi​;ki​;qj​;kj​]

wα∈R4d\boldsymbol{w}_{\alpha}\in \mathbb{R}^{4d}wα​∈R4d ，则参数量一共为 2d(D+2n)2d(D+2n)2d(D+2n) ，通常 d≪Dd\ll Dd≪D ，所以参数量更小。

分析与评述

  • 原版GlobalPointer参数过大，那么平均起来每个参数更新越稀疏，相对来说也越容易过拟合；
  • 而Efficient GlobalPointer共享了“抽取”这一部分参数，仅通过“分类”参数区分不同的实体类型，那么实体抽取这一步的学习就会比较充分，而实体分类这一步由于参数比较少，学起来也比较容易。
  • 个人疑问：原版GP是根据头token和尾token共同判别实体类型的。但是Efficient GP则是 对 [qi;ki;qj;kj]\left[\boldsymbol{q}_{i} ; \boldsymbol{k}_{i} ; \boldsymbol{q}_{j} ; \boldsymbol{k}_{j}\right][qi​;ki​;qj​;kj​] 做分类判别的，拆分/代码实现 下来其实就是头token的分数+尾token的分数，这样有点反直觉，因为判断实体的话肯定是需要同时利用两个token，而不是简单地分别判断 “头token作为类型a的分数 + 尾token作为类型a的分数” 。
