CASREL for RTE

Posted on 2022-02-27 In NLP , Relationship Extraction
Symbols count in article: 3.4k Reading time ≈ 17 mins.

论文：A Novel Cascade Binary Tagging Framework for Relational Triple Extraction
代码：Github: weizhepei/CasRel

Abstract

关系三元组抽取(Relational Triple Extraction, RTE)，也叫实体-关系联合抽取，是信息抽取领域中的一个经典任务，旨在从文本中抽取出结构化的关系三元组(Subject, Relation, Object)用以构建知识图谱。有时也与关系分类(Relation Classification, RC)统称为关系抽取(Relation Extraction, RE)。

RC与RTE的主要区别在于：RC是在给定实体对和输入文本的情况下，抽取出实体对在句子中所表达的关系；而RTE则是在仅给定输入文本的情况下，抽取出包含在文本中的所有可能的关系三元组。

重叠三元组问题（the overlapping triple problem）：不同的三元组之间共享相同实体。
作者以一种新的视角审视重叠三元组问题，并且提出一种端到端的级联二元标记框架 end-to-end cascade binary tagging framework (CASREL)。

Introduction


上面列举了三元组共享一个或者多个实体的情况，即重叠三元组问题。

关系分类的缺点：

  • 类别分布高度不平衡（highly imbalanced），很多实体对之间不存在关系，两两进行匹配会产生大量的负样本。
  • 当同一对实体之间有多个关系时（重叠三元组问题），使用关系分类会让问题变成一个不平衡多分类问题，严重混淆分类器。
  • 如果没有足够的训练实例，分类器就很难判断实体参与的关系。

EPO：同一对实体，传统的关系分类方法只能分类出一种关系。
SEO：数据分布不均衡，比如有的关系标签很多，有的很少，有的正例很多，有的负例很多，这样导致传统模型不能很好的进行学习。

思想

该框架并没有使用传统的关系分类来判别实体间关系，而是将关系看作一个将 subject 映射到 object 的函数。

传统的关系分类器 relation classifier： f(s,o)→rf(s,o) \rightarrow rf(s,o)→r
本文的关系特定的尾实体标注器 relation-specific taggers： fr(s)→of_r(s)\rightarrow ofr​(s)→o

  • 每一个关系特定的尾实体标注器都能在特定关系下将subject映射到所有正确的object
  • 或者不返回任何object，而是返回null，表示与给定subject和relation时，不存在三元组。

采用这种思想，关系抽取可以被视为两部：

  1. 识别所有可能的 subject
  2. 将每一个 subject 放入关系特定的尾实体标注器中，来同时识别所有可能的 relation 和相应的 object 。

Contributions

  1. 引入一个新的视角重新审视关系抽取问题，设计一个通用算法框架来解决重叠三元组问题。
  2. 将上述框架实例化为一个基于Transformer编码器的级联二元标记模型，这使得该模型能够将新的标记框架的能力与预训练模型的先验知识结合起来。
  3. 该框架的性能明显优于最新的方法，在两个数据集上分别获得了17.5和30.2的F1明显提高。

优点：

  1. 直接从三元组出发，，整体框架是在元组级别上进行的，和最终的评判标准一致。
  2. 没有假设元组应该怎么样共享实体，所以解决了重叠三元组问题。
  3. 引出一种全新的元组抽取框架，可以学习出一个subject标注器 p(s∣xj)p\left(s \mid x_{j}\right)p(s∣xj​) ，和不同关系的多个object标注器 pr(o∣s,xj)p_{r}\left(o \mid s, x_{j}\right)pr​(o∣s,xj​) 。

使用框架步骤：

  1. 使用 subject标注器 得到所有的 candidate subject
  2. 对每一个 candidate subject，使用所有关系的 object标注器 来判断是否可以将 subject 与 object 关联起来。

本论文中的subject 和 object 只需找出边界即可，无需确定类型。

Model

两个大模块 Bert Encoder + Cascade Decoder
同时 Cascade Decoder 由两个小模块组成：a subject tagger + a set of relation-specific object taggers 。
这两个小模块，也是 CASREL 被称为 「层叠式指针网络」 的原因：两个表示层叠，同时两者都用了指针的形式。

Bert Encoder

h0=SWs+Wphα=Trans⁡(hα−1),α∈[1,N]\begin{aligned} \mathbf{h}_{0} &=\mathbf{S W}_{s}+\mathbf{W}_{p} \\ \mathbf{h}_{\alpha} &=\operatorname{Trans}\left(\mathbf{h}_{\alpha-1}\right), \alpha \in[1, N] \end{aligned} h0​hα​​=SWs​+Wp​=Trans(hα−1​),α∈[1,N]​

S\mathbf{S}S ：the matrix of one-hot vectors of sub-words indices
Ws\mathbf{W}_{s}Ws​ ：sub-words embedding matrix
Wp\mathbf{W}_{p}Wp​ ： the positional embedding matrix where p represents the position index in the input sequence
因为输入的是一个单句，所以 segment id 被忽略。

Subject Tagger/ low level tagging module

指针的打分过程

对每个token，使用两个独立的二分类器预测其是否为 「subject start token」 和 「subject end token」，设定某个阈值，大于则标记为1，否则标记为0。

Relation-specific Object Taggers/ high level tagging module

可以根据前面标注出来的 subject 同时找出关系和 Object

每一个 Object Tagger 将同时考虑 subject feature vsub\mathbf{v}_{sub}vsub​ 和 object feature x\mathbf{x}x 。

看作三项：

  • 第一项：每一个token都参与损失计算，start_sub/end_sub 被看作正例，其余被看作负例，使用序列标注问题的计算方式。
  • 第二项：跟任何subject有关的关系，都会参与损失计算，start_obj/end_obj被看作正例，其余被看作负例，使用序列标注问题的计算方式。但是这里没有说 s 是哪一个s，是不是加上一个 ∑s\sum s∑s 更好呢；还是说任意一个 s 都可以，和博客一样，每个句子一个 s，多个 s 相当于加了 batch size，也不好操作。
  • 第三项：跟任何subject无关的关系，都参与损失计算，预测结果应该为 null。

Main Result

  • Even without taking advantage of the pre-trained BERT, CASREL-random and CASREL-LSTM are still competitive to existing state-of-the-art models.
  • Encoder中的先验知识很重要，所以使用bert。
  • 两个训练集横向对比：NYT数据集三元组类型多为Normal类型，而WebNLG数据集的三元组却多为EPO和SEO类型，EPO和SEO问题会使模型效果下降，同时CASREL解决重叠三元组问题确实有效。

Detailed Results on Different Types of Sentences


作者分别增加了三个针对Normal，EPO和SEO情况的专门试验，最终效果呈现递减，可证明难度依次增加。
同时，对一个句子中多个三元组的情况，作者也增加了试验，

从实验看出，CASREL效果都很好，而且在三元组数量 N≥5N\ge 5N≥5 的情况时，效果提升最大。作者认为这是因为 relation-specific object tagger 可以同时考虑多种关系。

Error Analysis

为了探究影响 抽取得到三元组 (E1,R,E2)(E1,R,E2)(E1,R,E2) 的因素，作者单独分析了抽取三元组中的不同元素。

  • 在NYT数据集上，E1，E2 和 (E1，R)，(R，E2)效果差不多；但是 (E1，E2) 和 (E1，R，E2) 指标之间的gap很小，但是 (E1，R)/(R，E2) 和 (E1，R，E2) 之间的gap很大。
    所以作者指出大部分关系能够被抽取出，但是抽取的实体对不正确；即模型判别关系的能力比判别实体的能力要强。
  • 在WebNLG数据集中，(E1，E2) 和 (E1，R，E2) 之间的gap 大于 (E1，R)/(R，E2) 和 (E1，R，E2) 之间的gap。
    所以作者指出错分关系带来的影响更大。这可能是WebNLG数据集的关系类别更多导致的。

思考：对比多头选择机制

多头选择机制：
Joint entity recognition and relation extraction as a multi-head selection problem
BERT-Based Multi-Head Selection for Joint Entity-Relation Extraction

  • CASREL是subject抽取，不需要判别subject的类别
    而 Multi-Head Selection 是 命名实体抽取，需要判别subject的类别。同时 Multi-Head Selection 也使用到了命名实体的向量表示构造subject的向量表示，属于使用了ner任务的信息。

  • CASREL 和 Multi-Head Selection 本质上都是将concat(subject,object/token)放入二分类器中，判别object是否合理，不同的二分类器表示不同的关系。
    但是 CASREL 中，每个关系对应两个二分类器，即两个指针；Multi-Head Selection 中，每个关系对应一个二分类器，即尾巴指针。
    所以从定义上说，CASREL 也能叫 Multi-Head Selection 。因为 Multi-Head Selection 指的是多种关系，而不是 biaffine 中的最后一维为 softmax。

  • CASREL 中的concat(subject,object/token)，二者分别为subject pooling和object token。
    Multi-Head Selection 中的concat(subject,object/token)，二者均为BiLSTM encodding+entity embedding。

  • CASREL是指针网络，每个句子层级的维度为 [relation_num, 2, seq_len]
    Multi-Head Selection 类似 Biaffine ，每个句子层级的输出维度为 [relation_num, seq_len, seq_len]
    具体的维度顺序可能不正确，不过具体的意思就是说CASREL是多个并行的指针序列，而 Multi-Head Selection 是多个二维的概率矩阵。

问题

  1. 关注下损失的实现，似然概率，start和end相当于是排在一起做的，如果end被预测成了start，似然值是一样的，这一点怎么实现。
  2. 是不是真的每一个句子在训练后一个tagger时，只用了一个subject，取第一个还是随机取。
  3. 如何实现模型推理出 null object ？是不是指针全都预测为 0 。
