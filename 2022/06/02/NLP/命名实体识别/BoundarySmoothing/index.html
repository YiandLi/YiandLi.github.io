Boundary Smoothing

Posted on 2022-06-02 In NLP , Named entity recognition
Symbols count in article: 2.2k Reading time ≈ 11 mins.

论文：Boundary Smoothing for Named Entity Recognition
代码：syuoni/eznlp

NER 任务存在的问题：

  • over-confidence：模型过于自信了，比如原本只是可能性稍微高一点的样本，模型却打出概率值为0.9（这是使用ce loss训练的结果）。
  • over-sharpness：边界的标注过于绝对，概率矩阵中标注实体附近的实体 target=0，有点过于绝对了。
    eg. Model(‘莲花清瘟’) = None ；Model(‘莲花清瘟胶囊’) = 药品。个人觉得不是很符合人类进行这项任务的思维方式。
  • inconsistent annotation：不同场景下，边界标注的规则不同，比如人名到底要不要加 Mr.。

基于边界存在的问题，参考 Label Smoothing，作者提出了 boundary smoothing 的训练方法，即使用 biaffine 模型得到 span 分数，在求出的概率矩阵上进行 boundary smoothing，重新分配 span 标注的的概率。
注意论文提出的是一种即插即用的方法，而非模型。个人感觉，其实所有使用概率矩阵进行推理/训练的模型都能够使用这种方式。

优点：

  • 缓解 over-confidence / 过拟合
  • 增强模型校准能力（calibration
  • 缓解错误标注样本带来的误导效果
  • 缓解正负样本不均问题

训练

回到开头的例子，「莲花清瘟」和「莲花清瘟胶囊」的区分其实没那么绝对。标注准则不同，标注结果往往天差地别，但是其实二者都可以看作药品实体。
着眼于一个特定的标注准则，「莲花清瘟胶囊」的标签就是 1，而「莲花清瘟」的标签却是 0，但是他们的语义很接近。作者将这种标注方式称为「hard boundary」。

这引出一个猜想：概率矩阵中，ground-truth span 附近的 span 也值得分到一些标注值，比如「莲花清瘟」值得 0.1 的标注值，「莲花清瘟胶囊」的标注值相应地变为 0.9。

具体地，每个 ground-truth span 保留 1−ϵ1-\epsilon1−ϵ 的概率值，其余的概率值 ϵ\epsilonϵ 被分配给其周围的其他实体：
使用曼哈顿距离（Manhattan distance）d 衡量「周围 span」和原始 ground-truth span 之间的距离。其中，距离 d (d < D) 的所有其他「周围span」平分 ϵ/D\epsilon/Dϵ/D ， DDD 为 smoothing size，如图使用颜色的深浅表示。作者称这种方式为「smoothed boundary」。

消融实验

Boundary Smoothing Parameters


作者实验了smoothing parameter ϵ∈{0.1,0.2,0.3}\epsilon \in \{0.1,0.2,0.3\}ϵ∈{0.1,0.2,0.3} , smoothing size D∈{1,2}D\in\{1,2\}D∈{1,2} 的不同搭配效果，多数都比 baseline 的效果要好，证明了该方法的鲁棒性，不过真对不同语料，还是要根据实验选择最适合的超参数。

Label Smoothing

作者想探究这种 boundary smoothing 带来的增益到底是不是只是因为用了 smoothing，所以使用 label smoothing 代替 boundary smoothing 进行训练。

table4 中 LS 的结果表明，label smoothing 不会带来增益，甚至让模型效果变差。
原因：实体标签之间的差距已经很明显了，所以没必要进行 smoothing 操作。

Pretrained Language Models


作者想要探究针对不同的 baseline，这种 boundary smoothing 的方式是否依然有效。
于是使用了预训练模型的不同版本（base/large）进行试验，发现都能带来增益。

BiLSTM Layer（凑数的）

作者移除了 biaffine 中的 Bi-LSTM 进行试验对比，发现也会带来增益，实验结果见上一个 section。
同时，移除 Bi-LSTM 会导致模型效果降低。

Further In-Depth Analysis

Over-Confidence and Entity Calibration

模型的两个特点/属性：

  • over-confident：模型过于自信
  • calibration（校准能力）：即预测分数能够正确反应模型的认知判断（概率值和模型真正的判断正确分数一致）（ the extent to which the prediction confidences produced by a model can represent the true correctness probability）

作者发现，使用 ce loss 进行训练时，训练后期开发集 loss 和 f1 都在增长，有点违背了客观认知。

于是作者猜测是因为模型 over-confident ，即一些预测错误的样本的概率越来越高，导致损失越来越大。

为了进一步研究这种 over-confident 的问题，作者将所有被预测出来的实体根据概率值划分为10组，然后计算 precision/confidence ，即 expected calibration error（ECE）。如果模型的 calibration（校准能力）比较强，这两者应该是正相关的，即相除为1。

可以看到，baseline（即 ϵ=0\epsilon=0ϵ=0 ）的 准确率 p 远远小于对应的置信度 c，说明模型过于自信了。
而引入 boundary smoothing 后，模型校准能力大大降低，甚至在 ϵ=0.3\epsilon=0.3ϵ=0.3 时，准确略大于置信度。

同时，因为解码时，低置信度样本要给高置信度样本让路，所以模型的校准能力很重要，这也是为什么 f1 可以增长的原因。

Loss Landscape Visualization

作者随机选择一些可训练参数，记录下取不同值时，模型的损失来模拟取到最优参数时，参数更新的路径。

  • 使用 boundary smooth 后，参数最优值附近更加平滑，即泛化性更强。
  • 传统 ce loss 更新时，容易出现局部最优点。

思考

  • 万物皆分类，所以其他任务也可以借鉴这种 smoothing 方式，但是差距大的（比如entity label），或者说不可能过拟合的，就没必要smoothing了，不过还是基于实验说话。
  • smoothing 的本质到底是什么，以前一直以为就是防止过拟合，读完论文发现没那么简单，后面自己再多想想吧。
