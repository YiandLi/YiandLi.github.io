DEBERTA: DECODING-ENHANCED BERT WITH DISENTANGLED ATTENTION

解耦attention - DISENTANGLED ATTENTION

原始 bert 是直接在第一层将 token embedding 和 position embedding 相加，后面就不管了，这里解耦合。

每个 token hidden state 变成了两个部分：token 部分和 position 部分，然后计算分数变成了：

Ai,j={Hi,Pi∣j}×{Hj,Pj∣}}⊤=HiHj⊤+HiPj∣i⊤+Pi∣jHj⊤+Pi∣jPj∣i⊤\begin{aligned} A_{i, j} & =\left\{\boldsymbol{H}_i, \boldsymbol{P}_{i \mid j}\right\} \times\left\{\boldsymbol{H}_j, \boldsymbol{P}_{j \mid\}}\right\}^{\top} \\ & =\boldsymbol{H}_{\boldsymbol{i}} \boldsymbol{H}_j^{\top}+\boldsymbol{H}_{\boldsymbol{i}} \boldsymbol{P}_{j \mid \boldsymbol{i}}^{\top}+\boldsymbol{P}_{\boldsymbol{i} \mid j} \boldsymbol{H}_j^{\top}+\boldsymbol{P}_{\boldsymbol{i} \mid j} \boldsymbol{P}_{j \mid i}^{\top} \end{aligned} Ai,j​​={Hi​,Pi∣j​}×{Hj​,Pj∣}​}⊤=Hi​Hj⊤​+Hi​Pj∣i⊤​+Pi∣j​Hj⊤​+Pi∣j​Pj∣i⊤​​

上图两两点积，实际应用中删除了最后一项：

做了阶段，i-k 之前的 token 位置编码都是 0；i+k 之后的 token 位置编码都是 k 。

根据试验，每一层的 positional mapper 直接和 content mapper 进行共享，i.e. Wq,r=Wq,c,Wk,r=Wk,cW_{q, r}=W_{q, c}, W_{k, r}=W_{k, c}Wq,r​=Wq,c​,Wk,r​=Wk,c​ ，效果不会下降。

增强版掩码解码器 - Enhanced Mask Decoder (EMD)

attention 时引入了相对位置编码，这里引入绝对位置编码。

会在 encoder 后加入 n 层 Enhanced Mask Decoder (EMD) 。
每一层 EMD 中，DeBERTa都会把绝对位置信息 III 作为输入，参与到 attention score 的计算中来。

虚拟对抗学习算法 - Scale-invariant fine-tuning (SiFT)

applying the perturbations to the normalized word embeddings。

具体为：

  1. SiFT先把word embedding vectors标准化为随机向量；

  2. 在标准化之后的word embedding vectors上增加随机扰动。

需要注意的一点是，SiFT只被应用到了DeBERTa1.5B这个模型上面，用于SuperGLUE任务的测试。

DeBERTa v2

(1) Vocabulary In v2 the tokenizer is changed to use a new vocabulary of size 128K built from the training data. Instead of a GPT2-based tokenizer, the tokenizer is now sentencepiece-based tokenizer.

(2) nGiE(nGram Induced Input Encoding) The DeBERTa-v2 model uses an additional convolution layer aside with the first transformer layer to better learn the local dependency of input tokens.

(3) Sharing position projection matrix with content projection matrix in attention layer Based on previous experiments, this can save parameters without affecting the performance.

(4) Apply bucket to encode relative positions The DeBERTa-v2 model uses log bucket to encode relative positions similar to T5.

(5) 900M model & 1.5B model Two additional model sizes are available: 900M and 1.5B, which significantly improves the performance of downstream tasks.

损失函数

Encoder 损失：

LMLM=E(−∑i∈Clog⁡pθG(x~i,G=xi∣X~G))L_{M L M}=\mathbb{E}\left(-\sum_{i \in \mathcal{C}} \log p_{\theta_G}\left(\tilde{x}_{i, G}=x_i \mid \tilde{\boldsymbol{X}}_G\right)\right) LMLM​=E(−i∈C∑​logpθG​​(x~i,G​=xi​∣X~G​))

Decoder 损失：
对于 encoder 重构的完整序列的每一个 token 进行二分类，假设 token 没有被 mask，则保留原始 token，如果是被 mask 的，则采样一个新的 token：

x~i,D={x~i∼pθG(x~i,G=xi∣X~G),i∈Cxi,i∉C\tilde{x}_{i, D}=\left\{\begin{array}{cc} \tilde{x}_i \sim p_{\theta_G}\left(\tilde{x}_{i, G}=x_i \mid \tilde{\boldsymbol{X}}_G\right), & i \in \mathcal{C} \\ x_i, & i \notin \mathcal{C} \end{array}\right. x~i,D​={x~i​∼pθG​​(x~i,G​=xi​∣X~G​),xi​,​i∈Ci∈/C​

然后二分类，将 generator 伪造成功的错误 tokens 看作负样本，真实 tokens 看作正样本进行二分类。

训练方法

不同的训练方法实验尝试：

  1. [ES]生成模型和判别模型共享embedding层训练；
    在 RTD 预训练时，ELECTRA 的 generator 和 discriminator 是共享 token embeddings EEE 的，因此 EEE 的梯度为

    gE=∂LMLM∂E+λ∂LRTD∂Eg_{\boldsymbol{E}}=\frac{\partial L_{M L M}}{\partial \boldsymbol{E}}+\lambda \frac{\partial L_{R T D}}{\partial \boldsymbol{E}} gE​=∂E∂LMLM​​+λ∂E∂LRTD​​

    相当于是在进行 multitask learning，但 MLM 倾向于使得语义相近的 tokens 对应的 embed 也比较接近，而 RTD 则倾向于使得语义相近的 tokens 对应的 embed 相互远离 (方便进行区分)，目标冲突，这会使得训练的收敛速度很慢。

  2. [NES]生成模型和判别模型不共享embedding层，也不共享参数层，交替训练；
    在对 generator 和 discriminator 的 token embed 解耦后可以看到， EGE_GEG​ 的 token embed 之间比较接近，而 EDE_DED​ 的 token embed 之间彼此远离。
    模型收敛速度加快了，但是模型性能降低了。

  3. [GDES]生成模型和判别模型部分共享，共享了 token embeddings，但只使用 LMLML_{MLM}LMLM​ ，而不使用 λLRTD\lambda L_{RTD}λLRTD​ 去更新 EGE_GEG​ 。
    此外，为了同时训练 discriminator ，作者还引入了一个初始化为 zero matrix 的 EΔE_\DeltaEΔ​ 去适配 EGE_GEG​ ：

ED=sg⁡(EG)+EΔ\boldsymbol{E}_D=\operatorname{sg}\left(\boldsymbol{E}_G\right)+\boldsymbol{E}_{\Delta} ED​=sg(EG​)+EΔ​
