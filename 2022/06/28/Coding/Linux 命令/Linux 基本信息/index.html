文件与目录结构

单颗树的结构，层级式目录结构。

目录结构只是逻辑上的表现形式，和物理关系没有对应关系，而是和分区，挂载点有关。

所以又称 虚拟目录 。

可以将分区 挂载 到单独的文件夹，用于专门存放这个特定文件夹的内容，像上面我们将 单独一个分区挂载到 了boot 文件夹上，其他的分区是没挂载的 ，即存储 根目录下下其他文件。

usr ：包含用户所有应用程序，文件，数据

bin binary：存放二进制命令文件，即直接可以使用的命令。指向 usr/bin

sbin ：存放系统级别 二进制命令文件，仅管理员使用的。指向 usr/sbin

lib library：库目录，存放系统和一些应用程序需要的动态链接库

lib64 ：64位特殊的动态链接库

usr/local ：user 本地下的应用文件，也有一个 usr/local/bin 目录。

boot ：引导分区所需要文件的挂载文件

dev device：设备管理目录，管理每个硬件

etc ：系统需要的配置文件，比如 数据库/java 配置文件。

media / mnt：可移动 媒体设备（u盘，光驱）/存储设备 的挂载点

proc ：process：进程映射

sys ：系统硬件信息

root ：超级管理员 root 的主目录文件夹

home ：每个普通用户的主目录文件夹

opt optional ：可选目录，给第三方软件包留的

tmp temporary ：临时目录

var ：一般用来存放可变文件，比如 log

一些环境变量

PATH

存储了一些常用命令所存放的目录路径。
当要求系统运行一个程序而没有告诉它程序所在的完整路径时, 系统除了在当前目录下面寻找此程序外, 还会在 PATH 列出的目录下去寻找。
比如在输入命令ls时，当前目录下其实是没有ls程序的脚本的，此时系统就会在 $PATH 地址中寻找这个程序。

PATH 的值是一系列目录，使用:间隔开，比如PATH=$PATH:<PATH 1>:<PATH 2>:<PATH 3>:------:<PATH N>。

添加方法：

  • 暂时添加：所以一般添加环境变量都是 export PATH=[/opt/....]:$PATH，属于叠加操作。
    这种添加方法，在终端关闭后就会消失。
  • 永久添加：通过编辑 /etc/profile或者~/.bashrc 来更改 PATH 变量，就可以永久添加了。
    这种方法，更改环境变量后，不是立即生效的，而是在用户下次登陆时生效；
    如果想立刻生效，则可执行source .bash_profile ; ~/.bashrc ; etc/profile。

shell环境依赖于多个文件的设置。当shell被调用时，它从两个初始文件读取命令。/etc/profile包含了系统变量，它由系统管理员维护，由系统管理员设置本地系统变量和特殊命令，对所有用户都生效，不安全。

普通用户的启动信息文件($HOME/.bash_profile)由各用户自己维护，该文件可以被修改以实现任何特定的系统初始化。
当用户登录时，该文件被 仅仅执行一次! 默认情况下，可以设置一些环境变量，并且执行用户的~/.bashrc文件。
注意是仅登陆时执行一次，打开 shell 时候不执行 ！for login shells

~/.bashrc：当登录时以及每次打开新的shell(开机)时 for interactive non-login shells ，该文件被读取。
通常二者设置大致相同，因为通常$HOME/.bash_profile会调用~/.bashrc。 所以一般优先把变量设置在~/.bashrc里面。

  • 无效！！： 需要注意的一点是，与Linux系统不同的是，在Mac OS中环境变量的默认配置文件为~/.zshrc而不是~/.bashrc或~/.bash_profile
  • 有效：Linux 里面是 ~/.bashrc 而 Mac 是 ~/.bash_profile

磁盘维护命令 DSCI 551

man (manual pager)

查看某命令的说明书，比如 man ls 。

参数：

  • 数字：指定从哪本man手册中搜索帮助；
    1：用户在shell环境可操作的命令或执行文件；
    2：系统内核可调用的函数与工具等
    3：一些常用的函数(function)与函数库(library)，大部分为C的函数库(libc)
    4：设备文件说明，通常在/dev下的文件
    5：配置文件或某些文件格式
    6：游戏(games)
    7：惯例与协议等，如Linux文件系统，网络协议，ASCII code等说明
    8：系统管理员可用的管理命令
    9：跟kernel有关的文件

  • 关键字：指定要搜索帮助的关键字。

wget

是一个下载文件的工具。

支持自动下载：即wget可以在用户退出系统的之后在后台执行。这意味这你可以登录系统，启动一个wget下载任务，然后退出系统，wget将在后台执行直到任务完成。

稳定，在带宽很窄的情况下和不稳定网络中有很强的适应性。

  • 如果是由于网络的原因下载失败，wget会不断的尝试，直到整个文件下载完毕。
  • 如果是服务器打断下载过程，它会再次联到服务器上从停止的地方继续下载。这对从那些限定了链接时间的服务器上下载大文件非常有用。

命令格式：wget [参数] [URL地址]
如果不加参数，则下载 url 指向的文件到当前目录。

  • -0 以不同的名称保存下载的文件
  • -P 以指定不同目录保存下载的文件
  • -b 允许你在后台下载文件。这在下载大文件时非常有用。
    要查看wget命令的下载进度等信息，可以运行tail命令 tail -f wget-log 查看wget命令的下载状态。

apt (Advanced Packaging Tool)

apt 是一个在 Debian 和 Ubuntu 中的 Shell 前端软件包管理器。命令执行需要超级管理员权限(root)，即加前缀 sudo。

语法：apt [options] [command] [package ...]

  • 安装指定的软件命令：sudo apt install <package_name>
    安装多个软件包：sudo apt install <package_1> <package_2> <package_3>
  • 更新：sudo apt update <package_name>
  • 卸载：sudo apt remove <package_name>
  • 安装 mplayer 如果存在则不要升级：sudo apt install mplayer --no-upgrade
  • 查看 pinta 包的相关信息：apt show pinta

更多详见 Linux apt 命令

vim

Vim 是从 vi 发展出来的一个文本编辑器。

如果 vim 后面跟的文件名称不存在的话，会自动创建 。

共分为三种模式，分别是

  • 命令模式（Command mode）
  • 输入模式（Insert mode）
  • 底线命令模式（Last line mode）

普通模式

用户刚刚启动 vim，便进入了命令/普通模式。
此状态下敲击键盘动作会被Vim识别为命令，而非输入字符。

  • i/o/a 切换到输入模式，以输入字符。
  • x 删除当前光标所在处的字符。
  • : 切换到底线命令模式，以在最底一行输入命令。

移动光标快捷键：

  • [Ctrl] + [f] 屏幕『向下』移动一页，相当于 [Page Down]按键 (常用)
  • [Ctrl] + [b] 屏幕『向上』移动一页，相当于 [Page Up] 按键 (常用)
  • [Ctrl] + [d] 屏幕『向下』移动半页
  • [Ctrl] + [u] 屏幕『向上』移动半页
  • + 光标移动到非空格符的下一行
  • - 光标移动到非空格符的上一行
  • $ 或功能键[End]：移动到这一行的最后面字符处(常用)
  • 0 / ^ 这是数字『 0 』：移动到这一行的最前面字符处 (常用)
  • w：跳到下一个单词的开头 ，e：跳到当前单词单词的词尾，b：跳到上一个单词的开头 ，
  • gg移动到这个档案的开始(常用)
  • G：移动到这个档案的最后一行(常用)
  • [n] gg 移动到第n行
  • nG：n 为数字。移动到这个档案的第 n 行。例如 20G 则会移动到这个档案的第 20 行。

复制粘贴：

  • yy：复制当前一行
  • y + $ ，复制光标到该行结尾的内容（$ 表示结尾）。
  • y + ^ ，复制光标到该行结尾的内容（$ 表示开头）。
  • [n] yy：复制
  • [n] dd：删除/剪切游标所在的那 一/n 整行(常用)
  • p/P ：粘贴一次，[n] p：粘贴 n 次。

撤销：

  • u是撤销你刚才做的动作
  • ctrl+r 是恢复你刚才撤销的动作

搜索替换快捷键：

  • /word：切换到底线命令模式，向光标之下寻找一个名称为 word 的字符串。例如要在档案内搜寻 vbird 这个字符串，就输入 /vbird 即可！ (常用)

    • 在命令模式下敲斜杆( / )这时在状态栏（也就是屏幕左下脚）就出现了 “/” 然后输入你要查找的关键字敲回车就可以了。
    • 如果你要继续查找此关键字，敲字符 n 就可以继续查找了。
    • 敲字符 N（大写N）就会向前查询；

  • 替换，先进入底线命令模式：

    • :s/word1/word2：替换光标所在行第一个匹配到的单词
    • :%s/word1/word2/g：从第一行到最后一行寻找 word1 字符串，并将该字符串取代为 word2 ！(常用)
    • :%s/word1/word2/gc：从第一行到最后一行寻找 word1 字符串，并将该字符串取代为 word2 ！且在取代前显示提示字符给用户确认 (confirm) 是否需要取代！(常用)

输入模式

想要在档案里面输入字符时， 一定要在左下角处看到 INSERT 或 REPLACE 才能输入喔！

在命令模式下按下i就进入了输入模式，用于编辑文本。
没有快捷键，所有的按键都是 input text。

ESC 退出输入模式，切换到命令模式

底线命令模式

在命令模式下按下:（英文冒号）就进入了底线命令模式。

set nu/set nonu：开启/关闭行号
q 退出程序
w 保存文件
ESC 退出输入模式，切换到命令模式

nano (Nano’s ANOther editor)

也是字符终端文本编辑器，但是比 vim 简单很多，有点像DOS下的editor程序。

默认情况下它会自动断行，即在一行中输入过长的内容时自动拆分成几行，但用这种方式来处理某些文件可能会带来问题，如果你想避免这种情况出现，就加上 -w 选项吧。

语法：nano [选项] [[+行,列] 文件名]...

-w：禁用自动换行
+行,列：从所指列数与行数开始，在打开文件使光标在指定行和字符上。
-S：按行滚动而不是半屏
-t：离开时自动储存，不要提示
-v：查看(只读)模式

在进入编辑模式后：
Ctrl+_，在line number, column number:字段中输入数字，再按Enter：将光标移动到指定的行和字符上
Ctrl+O：保存修改
Ctrl+X：退出
Ctrl+K：剪切一整行
Ctrl+U：粘贴一整行
要移动多行，只需多按几次Ctrl+K把需要移动内容都剪切下来，然后按一次Ctrl+U就可以把刚剪切的内容全部粘贴上来。

我做不到：精确地剪切/复制：

  1. 给文本做标记 ：移动光标到需要剪切文本的开头，然后 Ctrl+6（或者Alt+A），被做了标记的文本便会反白。
  2. 撤消文本标记：Ctrl+6
  3. 复制标记文本：Alt+6
