Event Centered Sentiment Analysis

Posted on 2022-07-17 In NLP , Event Extraction
Symbols count in article: 1.8k Reading time ≈ 9 mins.

论文：Implicit Sentiment Analysis with Event-Centered Text Representation

研究领域为「Implicit sentiment analysis」，即没有「显示情感词」出现的场景下的「情感分析/分类」；否则直接字典匹配了。
本文中，作者探究了「event-centric implicit sentiment analysis」，即如何利用已有的「sentiment-aware event」进行「情感分析/分类」。
作者将事件信息定义为一个三元组，「事件类型」+ <subject, predicate, object>（类似关系抽取，没有论元分类），目标是「情感分析/分类」。

主要内容：

  • 作者认为当前的方法对于事件信息的利用都过于简单或者过于间接，所以提出了一种更加直接的编码方式，对事件进行编码，来更有效地帮助「情感分析/分类」。
    提出的新模型使用了 hierarchical tensor-based composition 机制（个人感觉就是 hierarchical bi-affine）。
  • 此外，作者提出了一个 给定事件信息的隐式情感分类数据集：EveSA：Impli-Sentiment-Analysis。

比如：
Sentence: ‘You abandon me for a week to go off on holiday with daddy, come back and barely 2 days later you go off out with him again.’
Event triplet: <you, abandon, me>
Event type: abandonment
Sentiment: negative

Dataset: EveSA

在 FrameNet 和 Tweet数据 上进行过滤，人工筛选，详见论文。

Model

主要分为三部分

  1. 使用「hierarchical tensor-based composition」将事件元组信息 < subject, predicate, object > 和句子进行整合，得到新的表征。
  2. 除了「情感分类损失 LsL_sLs​ 」外，使用「事件类型分类损失 LeL_eLe​ 」进行联合训练，让模型保留事件知识。
  3. 考虑到相同类型的事件在不同的句子中有不同的三元组表达形式，作者认为「句子表征 hsenh_{\mathrm{sen}}hsen​」应该服从「事件表征 eee」的高斯分布，即 hsen∼N(e,σ2)h_{\mathrm{sen}} \sim \mathcal{N}\left(e, \sigma^{2}\right)hsen​∼N(e,σ2) ，让「句子表征空间」和「事件表征空间」更加接近。
    为此，使用一个「高斯相似度损失 LsimL_{sim}Lsim​ 」进行限制，和上述两个损失函数同时做联合训练。


事件三元组和句子是同一个 bert 编码，三元组经过 Hierarchical Tensor-based Compositions 机制得到一个事件三元组表征 eee ；句子则直接得到句子表征 hsenh_{sen}hsen​ 。

Constraint of Sentence-Event Similarity

相同的事件在不同的句中有不同的表示方法，于是作者认为「句子表征」应该服从其内部事件的「事件表征」的高斯分布，即 hsen∼N(e,σ2)h_{\mathrm{sen}} \sim \mathcal{N}\left(e, \sigma^{2}\right)hsen​∼N(e,σ2) 。

其中，平均值即 eee ，而方差则是通过 MLP「事件表征和句子表征的交叉编码 rfinalr_{final}rfinal​ 」得到的：

Baseline model

  • Sentence as only input

    • BiLSTM：use GloVe Embedding
    • BERT：BERT-base
    • BiLSTM + orthogonalAtt(Wei et al., 2020)： bidirectional LSTM neural network with BERT encoder and orthogonal attention

  • Event triplet as only input
    输入为：< subject, predicate, object >\text{< subject, predicate, object >}< subject, predicate, object >

    • BiLSTM
    • BERT
    • NTN：use tensor-based compositions，GloVe Embedding
    • BERTNTN：NTN with BERT encoder

  • Sentence and event triplet as input

    • BERTNTN + Att (Vaswani et al., 2017)：The results of BERTNTN serves as queries, and hidden states of the sentence are keys and values to get the final attention output.
    • BERTNTN ⊕\oplus⊕ sentence：the BERT-encoded hidden state of the subject, predicate and object is concatenated with the sentence separately. The three concatenations are then fed into NTN.
    • BERTNTN ⊕\oplus⊕ sentence + Att：similar to BERTNTN + Att
    • BERTNTN + Event Embedding (Tang et al.,2015b)：query Event Embedding

Result

MTL：是否同时使用 Event Type Classification 进行 Multi-task learning
Sen-Event：Sentence-Event Similarity 损失

  1. 提出的 model 效果最好
  2. 事件信息可以增益情感分类，但是单独用句子比单独用事件三元组效果好。
  3. Event type 的信息也有增益效果（HTC + MTL+ Sen-Event > HTC + Sen-Event）。

个人想法

  1. 这个任务不算主流
  2. 如何融入其他信息：
    • 编码注入：提供一种新的思路融入事件信息，其实也就是关系（三元组）信息了，进行后续的分类之外的下游任务。
    • 联合训练：除融入编码之外，使用联合训练也能增益模型，保留损失。
    • 相似度限制：个体的句向量表征需要服从通用的类别表征。
      好多论文也都在高斯分布上做文章，底层机制还是要学习，不过这里确实提出了新的相似度损失。
