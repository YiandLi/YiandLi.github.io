Query and Extract for Event Extraction

Posted on 2022-07-19 In NLP , Event Extraction
Symbols count in article: 2.9k Reading time ≈ 19 mins.

论文：Query and Extract: Refining Event Extraction as Type-oriented Binary Decoding
代码：Event_Query_Extract

Abstract and Introduction

将「事件标签」，「触发词」和「论元角色」转化为「自然语言 Query」的形式检索句子中的每个token，来进行事件抽取。

关键点：

  • 提出解决 事件抽取 的一种新范式：Query and Extract，比 pipeline 和 MRC 方式都要好。
  • 将 事件信息 和 论元信息 转化为自然语言形式参与任务，可以让 token 表征捕捉到了事件类型和论元的 标签信息 。

by leveraging our proposed attention mechanism to capture the interactions of event triggers and argument roles with input texts.

  • 提出了新的 attention 机制，用来捕捉「实体」和「触发词」之间，「事件」和「论元」之间，「论元」和「论元」之间的信息，即 global features（这也是一个很多人探讨的难题）。
  • 可以应用在 zero-shot 的场景，因为引入了 标签信息，而且多次 编码 ，充分利用了标签信息。

Model

Trigger Detection

Event Type Representation

为了融入事件类型信息，一个很自然的做法就是将「事件类型标签」拼接到句子后面一同做编码。

但是作者认为，单独一个「事件类型标签」不能完全表达「事件类型」的信息，可能还是会有一些歧义，比如 meet 可以指会议或者碰头见面。
所以作者使用训练集中对应的「标注触发词」一同拼接到输入句子后，送入 Bert 进行编码。

use a short list of prototype triggers to enrich the semantics of each event type

一个问题：数据集很大时，「触发词」也很多，全部拼接到后面肯定不现实。
所以作者使用一个类似TF-IDF 对指标来选择 top K 个「触发词」进行拼接。后续实验表明 K=4 效果最好。
具体地，对于具体的「事件类型 t」和某个「触发词」

  1. 首先统计该「触发词」在数据集中一共出现的频率 fof_ofo​
  2. 然后统计其作为「事件类型 t」的「触发词」的概率 ftf_tft​
  3. 最后计算其被选择的概率 ftfo\dfrac{f_t}{f_o}fo​ft​​

最终得到「事件类型文本」为

T={t,τ1t,τ2t,…,τKt}T=\left\{t, \tau_{1}^{t}, \tau_{2}^{t}, \ldots, \tau_{K}^{t}\right\} T={t,τ1t​,τ2t​,…,τKt​}

Context Encoding

输入句子为 W={w1,w2,...,wN}W=\{w_1,w_2,...,w_N\}W={w1​,w2​,...,wN​} ，「事件类型文本表征」为 T={t,τ1t,τ2t,…,τKt}T=\left\{t, \tau_{1}^{t}, \tau_{2}^{t}, \ldots, \tau_{K}^{t}\right\}T={t,τ1t​,τ2t​,…,τKt​} 。
除了「事件类型文本表征」外，作者在句子前面加入了一个特殊 token [EVENT]\text{[EVENT]}[EVENT]，来给强调模型当前在做 事件类型识别任务，输入句子为：

[CLS] [EVENT] [SEP] w1 … wN [SEP] t τ1t τ2t … τKt\text{[CLS]}\ \text{[EVENT]}\ \text{[SEP]}\ w_{1}\ \dots\ w_{N}\ \text{[SEP]}\ t\ \tau_{1}^{t}\ \tau_{2}^{t}\ \dots\ \tau_{K}^{t} [CLS] [EVENT] [SEP] w1​ … wN​ [SEP] t τ1t​ τ2t​ … τKt​

放入 Bert 中，得到「句子编码」和「事件类型编码」：

W={w0,w2,…,wN}T={t,τ0t,τ2t,…,τNt}\boldsymbol{W} = \left\{\boldsymbol{w}_{0}, \boldsymbol{w}_{2}, \ldots, \boldsymbol{w}_{N}\right\}\\ \boldsymbol{T} = \left\{\boldsymbol t ,\boldsymbol{\tau}_{0}^t, \boldsymbol{\tau}_{2}^t, \ldots, \boldsymbol{\tau}_{N}^t\right\} W={w0​,w2​,…,wN​}T={t,τ0t​,τ2t​,…,τNt​}

将事件类型信息融入句中 token 表征：每个文本 token 对「事件类型文本」做 cosine attention
使用「event-type oriented attention」，得到每个 token wiw_{i}wi​的 「event-type aware contextual representation」：

Event Argument Extraction

Context Encoding

对上一步已经抽取出的事件类型 ttt 和触发词 rrr ，首先根据模版得到对应论元类型 Gt={g1t,g2t,...,gDt}G^t=\{g_1^t,g_2^t,...,g_D^t\}Gt={g1t​,g2t​,...,gDt​} 。

Multiway Attention

给定事件类型 ttt 和其触发词 rrr ，和一个实体 eie_iei​ 。
需要判定触发词 rrr 和实体 eie_iei​ 之间是否对应一个论元类型 gjtg_j^tgjt​ ，于是作者提出 Multiway Attention 来更有效地建模其关系：

Experiments

Dataset

标准监督学习：ACE and ERE

Zero-shot：ACE，选择十个最频繁的事件作为训练集，将剩余的23个事件作为0样本测试集合 。

迁移学习：

  • cross-ontology direct transfer / domain adaptation setting：在一个数据集上训练，在另一个数据集上测试。
  • joint ontology enhancement / multi-domain learning setting：在 混合数据集上训练，在其中一个单一数据集上测试。

Supervised Event Extraction

本文提出的模型效果很好，并取得了三个 best score。

同时，作者对比了 BER_QA_Arg 模型，因为和本文模型的机制很像：

  • trigger detection：baseline全都将事件类型看作是one-hot形式（seen as symbols），没有融入标签信息，所以效果不好。
    因此，baseline 在小样本场景下的效果也不好，比如数据集中 End_Org 只有 31 个标签，BER_QA_Arg 精度为 0.35，而本模型精度为 0.667。
  • argument extraction：multiway attention 机制中 entity 之间的 self-attention 可以捕捉潜在的语义联系。

Zero-Shot Event Extraction


现有的模型无法融入标签信息，将 label 看作 one-hot 编码，不支持 zero-shot 场景。
虽然 BER_QA_Arg 也使用了标签信息，但是仅仅依赖 Bert 内部的 self-attention，融合标签信息的能力还是不足。

作者同时横向对比不同标签的结果性能，可以看到性能差别还是比较多的。

作者提出两点原因：

  1. 事件的类型语义有差异，一些事件标签本身就更加直白（Marry，Divorce），模型更能理解其语义；而有的就比较模糊（Charge-Indict …）。
  2. 事件触发词的多样性有差异，一些事件的触发词来来回回就那么几个（Divorce 2/9），而有的事件的触发词就很多（Demonstrate 6/7）。

Ablation Study

作者探究模型中每一个 attention机制 是否有效。结论就是缺少哪一个，效果都会降低。

Pros and Cons of Type-oriented Decoding

优点还是上述讲过的

缺点：binary decoding 算法复杂度比较高，作者提出两点改进。

  1. 随机选择那些不含 触发词 的样本进行训练（ 20% 负采样
  2. 相比于 MRC抽取方式，本模型一次性抽取出所有的论元。

个人觉得缺点同时还有模型需要多次计算 attention，而且要过两次 Bert，所以复杂度更高。

With these strategies, for trigger detection, our approach takes 80% more time for training and 19% less for inference compared with BERT_QA_Arg which relies on multi-class classification, while for argument extraction, our approach takes 36% less time for training and inference than BERT_QA_Arg.

Personal thinking

  • 使用一种新的基于注意力机制的方法融入 trigger 标签信息，而不是在 label 上融合，比较巧妙。但是这样计算量比较大。
  • 在 multiway attention 处，一个直觉的想法是为什么要更新 Arg Role 的表征，而且还是用 entity 来更新。
    但是消融实验 w/o Arg Role Self-attention 后效果下降，所以说还是有一定道理的。
    看过一些多任务信息抽取任务解读后，发现一般任务语料比较复杂，但是 Arg Role 标签信息或者描述表较简单。所以使用上下文或者task信息更新 Arg Role 表征可以让 Arg Role 表征更加贴合语境。
