MAML

Model-Agnostic Meta-Learning for Fast Adaptation of Deep Networks

元学习的目标：是在大量不同的任务上预先训练出一个模型，用这个模型作为初始化模型，在新任务的support set上进行微调，然后在新任务的query set上验证效果。

快速学习是人类的一个特点，即在多个（source）任务上提升自己的学习能力，然后面对新任务（target）时，将之前学习的经验与少量的新信息（target support）集成在一起，然后做出良好的预测（target query），同时也需要避免过拟合。

MAML 是一个通用的「模型无关」（任何模型都能用）的元学习算法，构建适用于多任务的模型初始化表示(an internal representation that is broadly suitable for many tasks)，这样在新任务上只需要微调模型就可以产生很好的效果。

If the internal representation is suitable to many tasks, simply fine-tuning the parameters slightly (e.g. by primarily modifying the top layer weights in a feed forward model) can produce good results.

即让模型拥有快速学习的能力，能够在不同任务上高效微调。从 dynamical systems 的角度来说，模型训练过程就是在 最大化新任务的损失函数对参数的敏感度，即参数的微小变动就可以给损失函数带来较大的波动。

From a dynamical systems standpoint, our learning process can be viewed as maximizing the sensitivity of the loss functions of new tasks with respect to the parameters: when the sensitivity is high, small local changes to the parameters can lead to large improvements in the task loss.

（这里有点抽象了，我理解中的高效微调就是：只要参数没达到十分理想，损失就会很大，所以带来的更新反馈/梯度也就多，这样可以高效更新参数。）


既然我们可以使用梯度下降法来优化模型，我们想学习到一个模型，该模型能够根据梯度下降算法快速适应新任务。这里的快速是指模型参数对任务很敏感(任务变化可以使用 loss 来体现)，参数的微小变化会对任务的损失函数产生很大的影响，进而导致不同的梯度方向。

同时 MAML 也是 optimization-based meta-learning 的开山之作。 optimization-based meta-learning 是通过之前大量的相似任务的学习，给网络模型学习到一组不错的/有潜力的/比较万金油的参数，使用这组参数作为初始值，在特定任务上进行训练，只需要微调几次就可以在当前的新任务上收敛了，

作者通过cv的分类实验发现，使用一阶导代替之前的多阶导后，模型效果没有改变很多，但是时间降低 33% ，说明模型的主要效果都来自于 post-update parameter values，即在 query set上的导数。

Reptile

Paper：On First-Order Meta-Learning Algorithms
Code：gabrielhuang/reptile-pytorch
OpenAI 对 MAML 做了简化，提出 Reptile 算法，但效果却优于 MAML。
从标题也可以看出，该方法只使用了一阶导。

注意和 MAML 不同，MAML 中每一个 task 都会被分成用于训练的 support set 和用于提供梯度的 query set；而 Reptile 并不需要划分 task，而是将数据全部用于训练。

Algorithm

顺序版本，每次只采样一个任务：

不断地在 task 中采样，进行 k 次 inner loop update，不断更新 fast weight。
当从该 task 中采样的 k 个子集全部训练结束后，网络初始化的参数 ϕ\phiϕ 就更新为了 ϕ~\widetilde{\phi}ϕ​ (fast weight)，然后我们根据 ϕ−ϕ~\phi-\widetilde{\phi}ϕ−ϕ​ 得到 slow weight 更新的方向。


图中，不同颜色就对应不同的 task。有点像传统 SGD 了。

从几何角度上讲，绿色代表第一个 task，三个绿色箭头代表三次更新时的梯度方向，可以看到，Reptile的模型就是朝着每个任务的梯度和的方向上不断地进行更新。


task batch 的版本：每次采样多个任务得到不同的损失，根据不同任务的损失一次性更新模型：

每次起点都是原始模型 θ\thetaθ ，在不同任务上更新 k 轮参数。
在 k=1 时，其实和传统迁移学习差不多，但是 k>1 是就不一样了：

  • 传统迁移学习：计算loss+得到导数+更新，模型不断顺序地更新。
  • Reptile：使用同一个起点更新 k 次，使用这 k 次的终点一起（avg）更新模型。
