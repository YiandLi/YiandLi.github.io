Adaptive Meta-learner via Gradient Similarity for Few-shot Text Classification

Posted on 2022-11-13 In Machine Learning , NLP , Meta Learning
Symbols count in article: 2.4k Reading time ≈ 12 mins.

论文：Adaptive Meta-learner via Gradient Similarity for Few-shot Text Classification

Abstract and Introduction

很多人尝试使用 optimization-based meta-learning 来解决 few-shot text classification 任务，因为可以得到一个好的初始化模型，相当于变相得到 task distribution 。
但是他们忽略了一些问题，作者提出 Adaptive Meta-learner via Gradient Similarity (AMGS) 方法对其进行改进：

  1. 数据量很少，但是模型参数很大，容易引起过拟合的问题。
    针对此问题，有文章提出 “让模型学习更多 task distribution” 来缓解过拟合问题。
    在 inner-loop update 中，作者额外引入一个 self-supervised Mask Token Prediction (MTP) 任务，从无标注文本数据中强化模型对 task distribution 的感知，同时也可以对梯度约束，增强鲁棒性缓解过拟合。
  2. 在训练模型时，没有区分 useful and useless task features。
    可以根据 deep model 选择 positive 和 negative features，进而缓解过拟合问题。
    考虑到梯度包含了所有的信息，这里作者使用 gradient similarity 来区分 useful and useless task features。

我们将在不同的 source support tasks 上 inner update 后的模型称作 base-leaner，将根据不同的 source query tasks 上 outer update 的过程称为 meta training/learning。

在 meta training 时，一些传统模型（First Order MAML， REPTILE）直接将 base-leaner 在 source query set 上loss的梯度作为拟合对象，不作区分。

如图 [Fig a,b]，MAML 直接复制 base-leaner 在 source query set 上loss的梯度方向作为 meta learner 的梯度方向，而 Reptile 直接用 (θ^0t−ψ0)(\hat{\theta}^t_0 - \psi_0)(θ^0t​−ψ0​) 的方向作为 meta learner 的梯度方向。这些都是不区分梯度的，即直接使用 base-leaner 的梯度指导 meta learner 的更新。



作者提出 adaptive meta-learner，在 outer-loop update / meta-training 的过程中更好地利用 inner-loop task features。
具体地，首先得到「当前 base-leaner 在 source query tasks 上的梯度」和「当前 meta-model 在 source query set 上的梯度」之间的相似性。然后调整梯度，指导模型更快速地拟合到最优空间。

Related Work

Among meta-learning frameworks, there are two popular strategy categories:

  • metric-based meta-learning: construct a metric space between classes and samples.
    • Prototypical Network (Snell et al., 2017)
    • Induction Network (Geng et al., 2019)
    • Relation Network (Sung et al., 2018)
  • optimization-based meta-learning: most of them consist of an inner (or base) algorithm and an outer (or meta) algorithm.

Method

Problem setup

Few-shot classification 包含 training episode 和 testing episode ，我们首先需要使用 training episode 的有标注数据训练模型，然后对 testing episode 上的无标注数据作出预测，进行验证。
training episode 中标签类别集合为 ytrainy_{train}ytrain​ ，testing episode 标签类别集合为 ytesty_{test}ytest​ ，两个标签集合的交集为空。
training episode 中包含了 N-way K-shot task 的集合。对于每个 task，我们采样出 N 个类别，每个类别采样出 K+q 个样本。其中 N*K 个样本组成了 support set，N*q 个样本组成了 query set。我们使用同样的方法构造test episode。

Model architecture

模型可以表示为：

θ={θb,θcpri ,θpaux }\theta=\left\{\theta_{b}, \theta_{c}^{\text {pri }}, \theta_{p}^{\text {aux }}\right\} θ={θb​,θcpri ​,θpaux ​}

θb\theta_{b}θb​ 表示「编码器 BERT」， θcpri \theta_{c}^{\text {pri }}θcpri ​ 表示用于主任务的「文本分类器」， θpaux \theta_{p}^{\text {aux }}θpaux ​ 表示用于辅助任务的「token预测器」。

Self-supervised Mask Token Prediction(MTP) task

其实就是 MLM 任务，这里沿用原始论文中 mask 的策略。
但是原始论文中的概率（15%；80%；10%；10%）会导致问题：如果输入句子太短，则不会有 token 被 mask。
所以作者将选择 token 的概率从 15% 提升到 30%，同时将替换 token 和 不变 token 的概率直接设置为 0%。

作者认为这个任务可以知道模型学习到更多的 task 分布，同时增强模型鲁棒性。

Adaptive Meta-learner via Gradient Similarity (AMGS)

思想：用 梯度方向 区分 positive 和 negative gradient


AMGS 的训练过程分为两个部分：

  1. base-leaner 首先在 support set 上，使用两个任务进行 inner update。
    然后在 query set 上只进行 supervised primary task，并且得到对应 loss 的梯度。
  2. adaptive meta-learner 使用 gradient similarity 作为标准，区分出上一步梯度中的 positive 和 negative gradient，根据这个结果进行 outer update。

Collecting gradient for adaptive meta-learner

在 support set 上训练时，使用损失函数：

Ltotal =(1−ρ)Lpri +ρLaux\mathcal{L}_{\text {total }}=(1-\rho) \mathcal{L}_{\text {pri }}+\rho \mathcal{L}_{a u x} Ltotal ​=(1−ρ)Lpri ​+ρLaux​

这里作者设置 ρ=10−3\rho=10^{-3}ρ=10−3


base-leaner 更新方式为下降法：

θ^=θ−α∇θLTitotal (xs,ys;θ)\hat{\theta}=\theta-\alpha \nabla_{\theta} \mathcal{L}_{T_{i}}^{\text {total }}\left(x_{s}, y_{s} ; \theta\right) θ^=θ−α∇θ​LTi​total ​(xs​,ys​;θ)

where xsx_sxs​ and ysy_sys​ are texts and corresponding labels in the support set. α\alphaα is the adaptation learning rate.
θ^\hat{\theta}θ^ 即在 support set 上更新后的 base-leaner.

然后使用 query set 训练主任务 {θb,θcpri }\{\theta_{b}, \theta_{c}^{\text {pri }}\}{θb​,θcpri ​} ：

arg⁡min⁡θb,θcpriLTipri(xq,yq;θ^b,θ^pri)\underset{\theta_{b}, \theta_{c}^{p r i}}{\arg \min } \mathcal{L}_{T_{i}}^{p r i}\left(x_{q}, y_{q} ; \hat{\theta}_b, \hat{\theta}^{p r i}\right) θb​,θcpri​argmin​LTi​pri​(xq​,yq​;θ^b​,θ^pri)

注意，这里只计算主任务的 loss，并且更新的是 meta-model。

Upgrade meta-learner with AMGS

使用 gradient cosine similarity 来判别 query set 上的样本是 positive 还是 negative。

根据上面的两个损失函数 ∇θLTitotal \nabla_{\theta} \mathcal{L}_{T_{i}}^{\text {total }}∇θ​LTi​total ​ 和 ∇θLTipri\nabla_{\theta} \mathcal{L}_{T_{i}}^{p r i}∇θ​LTi​pri​ ，这里cosine 相似度为：

cos⁡(∇θLTitotal (xs,ys;θ),∇θLTipri(xq,yq;θ^b,θ^cpri))\cos \left(\nabla_{\theta} \mathcal{L}_{T_{i}}^{\text {total }}\left(x_{s}, y_{s} ; \theta\right), \nabla_{\theta} \mathcal{L}_{T_{i}}^{p r i}\left(x_{q}, y_{q} ; \hat{\theta}_{b}, \hat{\theta}_{c}^{p r i}\right)\right) cos(∇θ​LTi​total ​(xs​,ys​;θ),∇θ​LTi​pri​(xq​,yq​;θ^b​,θ^cpri​))

第一项为 原始 meta-learner 在 support set 上的更新方向，即 base-leaner 的更新方向。
第二项为 base-leaner 在 query set 上的 loss, 对 meta-learner 的梯度。

如果这个相似度值为非负数，即两个梯度方向夹脚为锐角，则表示 query set 可以增强模型的泛化性。meta-learning 的目标为：

arg⁡min⁡θb,θcpri ,θpaux ∑Ti∼p(T)(LTitotal (xs,ys;θ)+LTipri (xq,yq;θ^pri ))\begin{array}{l} \underset{\theta_{b}, \theta_{c}^{\text {pri }}, \theta_{p}^{\text {aux }}}{\arg \min } \sum_{T_{i} \sim p(T)}\left(\mathcal{L}_{T_{i}}^{\text {total }}\left(x_{s}, y_{s} ; \theta\right)+\mathcal{L}_{T_{i}}^{\text {pri }}(x_{q}, y_{q} ; \hat{\theta}^{\text {pri }})\right) \end{array} θb​,θcpri ​,θpaux ​argmin​∑Ti​∼p(T)​(LTi​total ​(xs​,ys​;θ)+LTi​pri ​(xq​,yq​;θ^pri ))​

反之，如果角度为钝角，则直接抛弃 query set 上的损失，则 meta-learning 的目标为：

arg⁡min⁡θb,θcpri ,θpaux ∑Ti∼p(T)(LTitotal (xs,ys;θ))\underset{\theta_{b}, \theta_{c}^{\text {pri }}, \theta_{p}^{\text {aux }}}{\arg \min } \sum_{T_{i} \sim p(T)}\left(\mathcal{L}_{T_{i}}^{\text {total }}\left(x_{s}, y_{s} ; \theta\right)\right) θb​,θcpri ​,θpaux ​argmin​Ti​∼p(T)∑​(LTi​total ​(xs​,ys​;θ))

Algorithm

拿到 task batch 之后，

  • 第5行：对每个 task，先在 support set 上使用两个任务更新模型得到 base-leaner θ^\hat{\theta}θ^ 。
  • 第6行：在 query set 上使用主任务，计算base-leaner loss 对 meta-leaner 的梯度。总感觉这一行写的怪怪的，应该是没有更新的，只得到导数。
  • 第8-11行，计算两个梯度相似度，选择对应梯度更新meta-learner。

不过这里有个疑问，传统MAML方法，拟合的是base-leaner在query set上的梯度 LTipri (xq,yq;θ^pri )\mathcal{L}_{T_{i}}^{\text {pri }}(x_{q}, y_{q} ; \hat{\theta}^{\text {pri }})LTi​pri ​(xq​,yq​;θ^pri ) ，而这里却选择丢弃这个梯度，主要拟合 LTitotal (xs,ys;θ)\mathcal{L}_{T_{i}}^{\text {total }}\left(x_{s}, y_{s} ; \theta\right)LTi​total ​(xs​,ys​;θ) ，感觉有点像 reptile 了，只是加了个 query set 进行筛选性增益？
