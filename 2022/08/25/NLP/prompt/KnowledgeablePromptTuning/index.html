Knowledgeable Prompt Tuning

Posted on 2022-08-25 In NLP , Prompt

Abstract and Introduction

For Text Classification task, Prompt Tunning has been a promising approach.
Usually, the verbalizer is handcrafted or searched by gradient descent, which is:

  1. in lack of coverage and bring considerable bias and high variances to the results
  2. need large amount of labeled data for searching

This work focuses on incorporating external knowledge into the verbalizer, to expand the label word space of the verbalizer using external knowledge bases (KBs), forming a knowledgeable prompt-tuning (KPT).


KPT contains three steps:

  1. use external KBs to generate a set of label words for each label, to cover different granularities and perspectives of the label.
  2. propose four refinement methods to cope with the noise in the unsupervised expansion of label words:
    1. frequency refifinement
    2. relevance refifinement
    3. contextualized calibration
    4. learnable refifinemen
  3. apply either a vanilla average loss function or a weighted average loss function for the utilization of expanded verbalizers

Overview of Prompt Tunning

  • input sequence: x=(x0,x1,…,xn)\mathbf{x}=\left(x_{0}, x_{1}, \ldots, x_{n}\right)x=(x0​,x1​,…,xn​)
  • class label set: Y={y1,...,yn}\mathcal{Y} = \{y_1,...,y_n\}Y={y1​,...,yn​}
  • mask language model: M\mathcal{M}M
  • the label word set: V\mathcal{V}V
  • prompt template: xp=[CLS] A [MASK] question : x\mathbf{x}_{\mathrm{p}}=[\mathrm{CLS}]\ \mathrm{A}\ [\mathrm{MASK}] \text { question }:\ \mathbf{x}xp​=[CLS] A [MASK] question : x
  • the probability of label yyy: P(y∣xp)=g(PM([MASK⁡]=v∣xp)∣v∈Vy)P\left(y \mid \mathbf{x}_{\mathrm{p}}\right)=g\left(P_{\mathcal{M}}\left([\operatorname{MASK}]=v \mid \mathbf{x}_{\mathrm{p}}\right) \mid v \in \mathcal{V}_{y}\right)P(y∣xp​)=g(PM​([MASK]=v∣xp​)∣v∈Vy​)
    ggg is a function transforming the probability of label words into the probability of the label.

Usually

  1. most method only use one label word for each class label.
  2. ggg is only an identity function

Knowledgeable Prompt-tuning

  1. use KBs to generate multiple label words related to each class yyy
  2. propose four refinement methods to eliminate the noise in the expanded V\mathcal{V}V
  3. explore the vanilla average and weighted average approaches for the utilization of the expanded V\mathcal{V}V

Verbalizer Construction

For prompt tunning, the label words should be of two attributes: wide coverage and little subjective bias.
External structured knowledge could simultaneously meet both requirements.

  • For topic classification task, the core issue is to extract label words related to the topic.
    A knowledge graph Related Words G\mathcal{G}G is used to get topic-related label words.

    1. The name of each class v0v_0v0​ is seen as the anchor node.
    2. Its neighbors whose edge between the anchor node is larger than a threshold η\etaη are picked to form a neighbor node set NG(v0)N_{\mathcal{G}}\left(v_{0}\right)NG​(v0​). In experiments, η=0\eta=0η=0.
    3. Each class is mapped into a set of label words Vy=NG(v0)∪{v0}\mathcal{V}_{y}=N_{\mathcal{G}}\left(v_{0}\right) \cup\left\{v_{0}\right\}Vy​=NG​(v0​)∪{v0​}.

  • For binary sentiment classification, the core issue is to extend the binary sentiment.
    The author use the sentiment dictionary summarized by previous researchers.

    • positive words
    • negative words

Verbalizer Refinement

The first step has picked many extra label words.
But the collected label words can be very noisy since the vocabulary of the KB is not tailored for the PLM.
Thus it is necessary to refine such verbalizer by retaining high-quality words.

In this section, the author proposes four refinement methods for different problems.

Frequency Refinement

Problem: several words in the KB are rare to the PLM

Solution: use label words’ contextualized prior (instead of a word-frequency dictionary) as an evaluation indicator to filter words.

Specifically, given a text classification task,

  1. define the distribution of the example sentences x\mathbf{x}x in the corpus as D\mathcal{D}D
  2. wrap each sentence in D\mathcal{D}D into the template, then calculate the predicted probability for each label word vvv in the masked position

PM([MASK]=v∣xp)P_{\mathcal{M}}\left([\mathrm{MASK}]=v \mid \mathbf{x}_{\mathrm{p}}\right) PM​([MASK]=v∣xp​)

  1. then the prior distribution of the label words in the masked position is

PD(v)=Ex∼DPM([MASK⁡]=v∣xp)P_{\mathcal{D}}(v)=\mathbb{E}_{\mathbf{x} \sim \mathcal{D}} P_{\mathcal{M}}\left([\operatorname{MASK}]=v \mid \mathbf{x}_{\mathrm{p}}\right) PD​(v)=Ex∼D​PM​([MASK]=v∣xp​)

  1. Back to step 1, how to get or to simulate the distribution of the corpus?
    Empirically, an unlabeled small-sized support set C~\tilde{\mathcal{C}}C~ sampled from training set works well.
    So here the author uses this support set C~\tilde{\mathcal{C}}C~ to represent the distribution D\mathcal{D}D.
    Then, the contextualized prior is approximated by

PD(v)≈1∣C~∣∑x∈C~PM([MASK⁡]=v∣xp)P_{\mathcal{D}}(v) \approx \frac{1}{|\tilde{\mathcal{C}}|} \sum_{\mathbf{x} \in \tilde{\mathcal{C}}} P_{\mathcal{M}}\left([\operatorname{MASK}]=v \mid \mathbf{x}_{\mathrm{p}}\right) PD​(v)≈∣C~∣1​x∈C~∑​PM​([MASK]=v∣xp​)

Then remove the label words whose prior probabilities are less than a threshold.
In different tasks, it is difficult to get exact thresholds, so the author choose to filter the label words that appear in the lower half of the contextualized prior probability.

This method is only used in zero-shot settings, the reasons will be discussed later.

But in zero-shot settings, there is no training set.
So in real situations, maybe we should find the support set from the domain/task of the testing set.

Contextualized Calibration

Problem:

  • When do mask language predicting, the prior probabilities of label words if of drastic difference, which means some label words are less likely to be predicted than the others.
  • In prompt learning, the PLM has a natural tendency to predict one word over another words regardless of the real sentence input.

Solution: Use the contextualized prior (See Frequency Refinement) of label words to calibrate the predicted distribution, namely, contextualized calibration (CC).

P~M([MASK]=v∣xp)∝PM([MASK]=v∣xp)PD(v)\tilde{P}_{\mathcal{M}}\left([\mathrm{MASK}]=v \mid \mathbf{x}_{\mathrm{p}}\right) \propto \frac{P_{\mathcal{M}}\left([\mathrm{MASK}]=v \mid \mathbf{x}_{\mathrm{p}}\right)}{P_{\mathcal{D}}(v)} P~M​([MASK]=v∣xp​)∝PD​(v)PM​([MASK]=v∣xp​)​

PD(v)P_{\mathcal{D}}(v)PD​(v) is contextualized prior. The final probability is normalized to 1.

In few-shot setting, calibration may not be necessary, because the probability of a label word can be trained to the desired magnitude.

Relevance Refinement

Problem:

  1. some label words may be relevant to different classes.
  2. some label words may be more relevant to their belonging class than the others.

To measure the relevance of a label word to each class, two kinds of representations over the support set are calculated:

  1. label word’s representation
    The author uses prediction probability of the label word vvv on the support set C~\tilde{\mathcal{C}}C~ as the label word’s representation $\mathbf q^v \in \mathbb{R}^{\left |\tilde{\mathcal{C}} \right | } $.
    For the label word vvv filled in the i-th sentence xi\mathbf{x}_{i}xi​:

qiv=PM([MASK⁡]=v∣xip),xi∈C~\mathbf{q}_{i}^{v}=P_{\mathcal{M}}\left([\operatorname{MASK}]=v \mid \mathbf{x}_{i \mathrm{p}}\right), \mathbf{x}_{i} \in \tilde{\mathcal{C}} qiv​=PM​([MASK]=v∣xip​),xi​∈C~

  1. class’s representation
    Similar to the label word’s representation, the author uses prediction probability qv0\mathbf q^{v_0}qv0​ of the class label v0v_0v0​ on the support set C~\tilde{\mathcal{C}}C~ as the class’s representation qy∈R∣C~∣\mathbf q^{y} \in \mathbb{R}^{\left |\tilde{\mathcal{C}} \right | }qy∈R∣C~∣.

Therefore, the relevance score between a label word vvv and a class yyy is calculated using the cosine similarity:

r(v,y)=cos⁡(qv,qy)=cos⁡(qv,qv0)r(v, y)=\cos \left(\mathbf{q}^{v}, \mathbf{q}^{y}\right)=\cos \left(\mathbf{q}^{v}, \mathbf{q}^{v_{0}}\right) r(v,y)=cos(qv,qy)=cos(qv,qv0​)


Apart from that, some label words may contribute positively to multiple classes, resulting in confusion between classes.
e.g., label word “physiology” --> class SCIENCE, class SPORTS

The author design a metric that favors the label word with high relevance merely to its belonging class and low relevance to other classes:

R(v)=r(v,f(v)) : ∑y∈Y,y≠f(v)(r(v,y))∣Y∣−1=r(v,f(v))∣Y∣−1∑y∈Y,y≠f(v)(r(v,y))\begin{aligned} R(v)&= {r(v, f(v))}\ :\ {\frac{\sum_{y \in \mathcal{Y}, y \neq f(v)}(r(v, y))}{|\mathcal{Y}|-1}}\\ &=r(v, f(v)) \frac{|\mathcal{Y}|-1}{\sum_{y \in \mathcal{Y}, y \neq f(v)}(r(v, y))} \end{aligned} R(v)​=r(v,f(v)) : ∣Y∣−1∑y∈Y,y=f(v)​(r(v,y))​=r(v,f(v))∑y∈Y,y=f(v)​(r(v,y))∣Y∣−1​​

f(v)f(v)f(v) is the corresponding class of label word vvv.

Similar to TF-IDF:

  • r(v,f(v)){r(v, f(v))}r(v,f(v)) is the word’s relevance score to its corresponding class f(v)f(v)f(v)
    If the label word is highly positively to its corresponding class, it will be very big.
  • ∑y∈Y,y≠f(v)(r(v,y))∣Y∣−1{\dfrac{\sum_{y \in \mathcal{Y}, y \neq f(v)}(r(v, y))}{|\mathcal{Y}|-1}}∣Y∣−1∑y∈Y,y=f(v)​(r(v,y))​ is the word’s average relevance score to other corresponding classes.
    If the label word is contribute highly positively to other classes, it will be very big and thus R(v)R(v)R(v) will be small.

A good label word should at least has a higher relevance score for its belonging class than the average relevance score for the other classes. So, the label words with R(v)<1R(v) < 1R(v)<1 are removed.

Learnable Refinement

In few-shot learning, the refinement can be strengthened by a learning process,
which is to assign a learnable weight wvw_vwv​ each label word vvv.

The weights of all the label words form a vector w∈R∣V∣w\in \mathbb{R}^{|V|}w∈R∣V∣, which is initialized to be a zero vector.
The weights are normalized within each Vy\mathcal{V}_yVy​:

αv=exp⁡(wv)∑u∈Vyexp⁡(wu)\alpha_{v}=\frac{\exp \left(w_{v}\right)}{\sum_{u \in \mathcal{V}_{y}} \exp \left(w_{u}\right)} αv​=∑u∈Vy​​exp(wu​)exp(wv​)​

If a label word is noisy, it will be trained with a small weight in the training process, to minimize its influence on the prediction.

e.g., class SPORT is of two label words { milk, ballgame }, label word milk is noisy.
The class probability is PSPORT=αmilkpmilk+αballgamepballgameP_{SPORT}=\alpha_{milk}p_{milk}+\alpha_{ballgame}p_{ballgame}PSPORT​=αmilk​pmilk​+αballgame​pballgame​.
Since label word milk is noisy, pmilkp_{milk}pmilk​ will be very small, and thus when update parameter, ▽αmilk=pmilk\bigtriangledown \alpha_{milk}=p_{milk}▽αmilk​=pmilk​ will be very small.
Since αmilk\alpha_{milk}αmilk​ and αballgame\alpha_{ballgame}αballgame​ are initialized from 0, finally αballgame\alpha_{ballgame}αballgame​ is much larger than αmilk\alpha_{milk}αmilk​.

Many label words are out-of-vocabulary for the PLM and are split into multiple tokens by the tokenizer.
The author use the average prediction score of each token as the prediction score for the word, which is to average the prediction probability of each token in the single [MASK] position.

Verbalizer Utilization

Final problem: how to map the predicted probability on each refined label word to the class label.

There are two methods: Average and Weighted Average.

Average

After refinement, we can assume that each label word of a class contributes equally.

Therefore, the average of the predicted scores on Vy\mathcal{V}_{y}Vy​ is used as the predicted score for label yyy:

y^=argmax⁡y∈Y∑v∈VyP~M([ MASK ]=v∣xp)∣Vy∣\hat{y}=\underset{y \in \mathcal{Y}}{\operatorname{argmax}} \frac{\sum_{v \in \mathcal{V}_{y}} \tilde{P}_{\mathcal{M}}\left([\text { MASK }]=v \mid \mathbf{x}_{\mathrm{p}}\right)}{\left|\mathcal{V}_{y}\right|} y^​=y∈Yargmax​∣Vy​∣∑v∈Vy​​P~M​([ MASK ]=v∣xp​)​

This method is used in zero-shot learning since there is no parameter to be trained.

Weighted Average

Supported by the Learnable Refinement, a weighted average of label words’ score is used as the prediction score.

y^=argmax⁡y∈Yexp⁡(s(y∣xp))∑y′exp⁡(s(y′∣xp))s(y∣xp)=∑v∈Vyαvlog⁡PM([MASK]=v∣xp)\begin{aligned} &\hat{y}=\operatorname{argmax}_{y \in \mathcal{Y}} \frac{\exp \left(s\left(y \mid \mathbf{x}_{\mathrm{p}}\right)\right)}{\sum_{y^{\prime}} \exp \left(s\left(y^{\prime} \mid \mathbf{x}_{\mathrm{p}}\right)\right)} \\ &s\left(y \mid \mathbf{x}_{\mathrm{p}}\right)=\sum_{v \in \mathcal{V}_{y}} \alpha_{v} \log P_{\mathcal{M}}\left([\mathrm{MASK}]=v \mid \mathbf{x}_{\mathrm{p}}\right) \end{aligned} ​y^​=argmaxy∈Y​∑y′​exp(s(y′∣xp​))exp(s(y∣xp​))​s(y∣xp​)=v∈Vy​∑​αv​logPM​([MASK]=v∣xp​)​

This method is used in few-shot learning.

Be Noticed that here we use PMP_{\mathcal{M}}PM​, which means Contextualized Calibration is not used.

This objective function is suitable for continuous optimization by applying a cross-entropy loss.

Experiments

  • Dataset:

    • three topic classification datasets
      • AG’s News (Zhang et al., 2015)
      • DBPedia (Lehmann et al., 2015)
      • Yahoo (Zhang et al., 2015)
    • two sentiment classification datasets
      • IMDB (Maas et al., 2011)
      • Amazon (McAuley and Leskovec, 2013)

  • four manual templates for each dataset

  • report both the average results (with standard error) and the results of the best template

  • Baseline：

    • Fine-tuning (FT)
    • Prompt-tuning (PT): PET w/o tricks like self-training and prompt ensemble
    • Automatic Verbalizer (AUTO): uses labeled data to select the most informative label words inside a PLM’s vocabulary.
    • Soft Verbalizer (SOFT)
    • PT+CC: For zero-shot setting, PT is introduced combined with contextualized calibration. To see how much
      improvement is made by contextualized calibration instead of knowledgeable verbalizer.

In few shot settings, the supervised training data can train the output probability of each label word to the desired magnitude,
thus CC and FR are not used in the KPT.

  • Zero-shot(Table 2):
    • KPT > PT / PT+CC
    • these 4 refinement method work
    • Contextualized Calibration is very effective in the zero-shot setting.
    • KPT-FR-RR-CC: label noise is severe in the automatically constructed knowledgeable label words.
    • (KPT-FR-RR - KPT-FR-RR-CC) > (PT+CC - PT): the drastic difference in the prior probabilities of the knowledgeable
      label words
  • Few-shot(Table 3):
    • in sentiment classification dataset, KPT works well without the refinements,
      because the sentiment dictionary has little noise.
    • KPT enjoys smaller variances than baseline methods in most cases,
      demonstrating that the better coverage of label words stabilizes the training.
    • with more training data(20-shots), LR does not become more powerful as we may hypothesize,
      because all label words, even with some noise, can serve as training objectives for prompt tuning.

Pilot Experiments

To further illustrate the advantage of manual verbalizer, we conduct pilot experiments in soft verbalizer, which is to test the performance of Soft Verbalizer with and without the manually defined class name in 5 and 10 shot experiments.

further improving the verbalizer with manually defined class name is a promising direction than the learned-from-scratch verbalizer without any human prior.

A Theoretical Illustration of KPT

If multiple label words are used to contribute to a single label, the predicted probability of the label is defined by marginalizing(!!) the probability of predicting all the label words:

p(Y=y∣x)=∑v∈VYp(Y=y,[M]=v∣x)p(\mathrm{Y}=y \mid \mathbf{x})=\sum_{v \in V_{\mathcal{Y}}} p(\mathrm{Y}=y,[\mathrm{M}]=v \mid \mathbf{x}) p(Y=y∣x)=v∈VY​∑​p(Y=y,[M]=v∣x)

Since the prediction of YYY is independent of xxx given vvv , the probability is:

∑v∈VYp(Y=y∣[M]=v)p([M]=v∣x)=∑v∈VYp(Y=y∣[M]=v)PM([M]=v∣xp)\begin{aligned} & \sum_{v \in V_{\mathcal{Y}}} p(\mathrm{Y}=y \mid[\mathrm{M}]=v) p([\mathrm{M}]=v \mid \mathbf{x}) \\ =& \sum_{v \in V_{\mathcal{Y}}} p(\mathrm{Y}=y \mid[\mathrm{M}]=v) P_{\mathcal{M}}\left([\mathrm{M}]=v \mid \mathbf{x}_{\mathrm{p}}\right) \end{aligned} =​v∈VY​∑​p(Y=y∣[M]=v)p([M]=v∣x)v∈VY​∑​p(Y=y∣[M]=v)PM​([M]=v∣xp​)​

Using the Bayes Theorem:

∑v∈VYp([M]=v∣Y=y)p(Y=y)p([M]=v)PM([M]=v∣xp)∝∑v∈VYp([M]=v∣Y=y)p([M]=v)PM([M]=v∣xp)\begin{array}{l} \sum_{v \in V_{\mathcal{Y}}} \dfrac{p([\mathrm{M}]=v \mid \mathrm{Y}=y) p(\mathrm{Y}=y)}{p([\mathrm{M}]=v)} P_{\mathcal{M}}\left([\mathrm{M}]=v \mid \mathbf{x}_{\mathrm{p}}\right) \\ \\ \propto \sum_{v \in V_{\mathcal{Y}}} \dfrac{p([\mathrm{M}]=v \mid \mathrm{Y}=y)}{p([\mathrm{M}]=v)} P_{\mathcal{M}}\left([\mathrm{M}]=v \mid \mathbf{x}_{\mathrm{p}}\right) \end{array} ∑v∈VY​​p([M]=v)p([M]=v∣Y=y)p(Y=y)​PM​([M]=v∣xp​)∝∑v∈VY​​p([M]=v)p([M]=v∣Y=y)​PM​([M]=v∣xp​)​

Now, the prediction probability of the label is composed of three parts:

  • p(v∣Y=y)p(v \mid \mathrm{Y}=y)p(v∣Y=y): the probability of predicting the specific label word vvv given the class label yyy.

    • If a label word vvv is relevant to the label yyy, this term will be big.
    • The Relevance Refinement and Learnable Refinement is used to weaken this prejudice.

  • p([M]=v)p([\mathrm{M}]=v)p([M]=v): the part of the denominator. This term is actually the prior probability of label words vvv.

    • It is estimated by our Contextualized Calibration.

  • PM([M])=v∣xpP_{\mathcal{M}}\left([\mathrm{M}]\right)=v \mid \mathbf{x}_{\mathrm{p}}PM​([M])=v∣xp​: the probability of the label words vvv predicted by the PLM.
    which is the only component in most works.
