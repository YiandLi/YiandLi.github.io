Transferability of PromptTuning

Posted on 2022-08-05 In NLP , Prompt
Symbols count in article: 3k Reading time ≈ 15 mins.

论文：On Transferability of Prompt Tuning for Natural Language Processing
代码：thunlp/Prompt-Transferability
分享会议：NAACL专场分享会 - Language Modeling & Summarization

Abstract and Introduction


Prompt Tunning 的问题：和微调相比，需要更多的训练时间来拟合。所以如何提升 Prompt Tunning 的效率很重要。
作为唯一的训练参数，可以直接使用训练好的 soft prompt 在别的任务/数据集上进行推理（zero-shot）；或者将其作为初始化 soft prompt 继续训练，来提升训练效率。这种方法被作者称之为 prompt transfer 。
以此为出发点，作者探究了 trained soft prompt 在多个「下游任务」和多个「预训练语言模型」上的迁移能力。



prompt transfer 的场景：

  • cross-task transfer：不同任务之间（17 NLP tasks of 6 types
  • cross-model transfer：不同预训练模型之间（RoBERTa and T5

结论：

  • 在 zero-shot 场景下直接复用（directly reusing）：
    • same PLM：trained soft prompt 在相似任务上迁移能力很好。
    • different PLMs：
      • 直接复用效果并不好，因为不同的 PLM 有不同的语义空间
      • 使用 trained cross-model projector 对 soft prompt 进行跨模型映射得到的 projected prompts of other PLMs ，在相似任务上迁移能力也很好。
  • Transfer with Initialization（将 trained soft prompt 作为另一个 task 的「初始化 prompt」进行训练），可以有效减少训练时间，并且表现更好。
  • 提出了一系列衡量「prompt 相似性」的指标，用来作为「prompt 迁移效果」的评价指标，最终发现激活神经元的重叠率是最准确的一项指标（the overlapping rate of activated neurons）。
    这个指标表明 prompt 本质上是在刺激 PLM 的内在能力，这种能力分布在神经元之间，以完成特定的NLP任务。

This suggests the prompts are essentially stimulating PLM’s inner ability distributing among neurons to do specific NLP tasks

Cross-Task Transfer

首先是迁移复用的效果，然后是迁移初始化的效果。

Zero-shot Transfer Performance

都使用同一个 PLM，在一个 source task/dataset 上训练得到 trained soft prompt，然后直接在另一个 target task/dataset 上复用，进行预测。

  • 当 source task 和 target task 属于同一个类型时：

    • 直接迁移复用的效果很好
    • 甚至超过直接在 target task 上进行 prompt tunning 的效果（尤其是在 source task 数据量更多的场景下），比如 IMDB -> Movie , restaurant -> laptop 。

  • 当 source task 和 target task 不属于同一个类型时，复用 trained soft prompt 的效果比较差，和 randomly initialized prompts 效果差不多。

  • 尽管不属于同一个类型，有些人物之间效果还不错，比如 QA/SUM -> SA 。

Transfer with Initialization

对于跨任务 prompt 迁移场景，作者定义一个方法 cross-task transferable prompt tuning (TPTTASK\text{TPT}_\text{TASK}TPTTASK​) method ：先在最相关的 source task 上训练得到 trained soft prompt，然后在 target task 上继续 prompt tunning。

在实验时，面对众多的 source task，作者选择 zero-shot transfer performance 最好的 trained soft prompt 继续训练。

  • TPTTASK\text{TPT}_\text{TASK}TPTTASK​ 方法效果很好，甚至可以超过传统随机初始化后 Prompt tunning 的效果。
  • TPTTASK\text{TPT}_\text{TASK}TPTTASK​ 方法训练时间更短。

Cross-Model Transfer

如果 cross-model 可行，那么在小模型上训练，然后迁移到大模型上可以有效减少大量训练成本。
直接复用是不可行的，因为不同 PLM 语义空间分布不同。
作者在此小节探究如何进行 Cross-Model Transfer ，同时尝试提升 cross-model transfer initialization 的效果。

Cross-Model Prompt Projection

方法：使用一个 cross-model projector 将 trained soft prompt 映射为适用于 target task 的 initialization soft prompt。

直觉来讲，对不同的 source task 和 target task 对，我们都应该单独训练一个 projector。但是这样效率较低，作者想要训练出一个所有任务通用的，但是 model-targeted 的 projector。这么做也合理，因为区别只在于两个 PLM 的语义空间，而我们只需要将两者进行语义对齐，而不关心具体的下游任务。

给定 source PLM 的 prompt：Ps={p1s, ..., pls}P^s = \{p_1^s,\ ...,\ p_l^s\}Ps={p1s​, ..., pls​}

  1. 在每个 token embedding 后面随机拼接一个向量，得到拼接后向量 Ps∈Rlds\mathbf{P}^{s} \in \mathbb{R}^{l d_{s}}Ps∈Rlds​ ， dsd_sds​ 为 source PLM 的 input embedding 。
  2. 然后使用一个 Projector Proj⁡(⋅){\operatorname{Proj}(\cdot)}Proj(⋅) 将 Ps∈Rlds\mathbf{P}^{s} \in \mathbb{R}^{l d_{s}}Ps∈Rlds​ 投影为 P~s∈Rldt\tilde{\mathbf{P}}^{s} \in \mathbb{R}^{l d_{t}}P~s∈Rldt​ ， dtd_tdt​ 为 target PLM 的 input embedding 。
    其中，project 结构为 a two-layer perceptron：

P~s=Proj⁡(Ps)=W2(σ(PsW1+b1))+b2\tilde{\mathbf{P}}^{s}=\operatorname{Proj}\left(\mathbf{P}^{s}\right)=\mathbf{W}_{2}\left(\sigma\left(\mathbf{P}^{s} \mathbf{W}_{1}+\mathbf{b}_{1}\right)\right)+\mathbf{b}_{2} P~s=Proj(Ps)=W2​(σ(PsW1​+b1​))+b2​

W1∈Rdh×lds,W2∈Rldt×dh,b1∈Rdh,b2∈Rldt\mathbf{W}_{1} \in \mathbb{R}^{d_{h} \times l d_{s}}, \mathbf{W}_{2} \in \mathbb{R}^{l d_{t} \times d_{h}}, \mathbf{b}_{1} \in \mathbb{R}^{d_h}, \mathbf{b}_{2} \in \mathbb{R}^{ld_t} W1​∈Rdh​×lds​,W2​∈Rldt​×dh​,b1​∈Rdh​,b2​∈Rldt​


训练时有两个可以选择的训练目标：

  1. Distance Minimizing：
    最小化 projected prompt 和 parallel trained prompt Pt\mathbf{P}^{t}Pt （ trained on the target PLM
    with the same task）之间的距离 eg. ∥Proj⁡(Ps)−Pt∥2\left\|\operatorname{Proj}\left(\mathbf{P}^{s}\right)-\mathbf{P}^{t} \right\|_{2}∥Proj(Ps)−Pt∥2​ 。

  2. Task Tuning：
    使用 target PLM 上的 task-specific supervision signals 进行训练，即将 projected prompt 在一些下游任务上进行 prompt tunning 。
    注意是单独的 target PLM，但是不是单独的下游任务。作者称这样可以 stimulate target PLM 以提高对 task 的泛化性。
    这种方法的效果取决于具体的 training tasks。

实验时，作者选择的 source task 为 laptop(Sentiment Analysis) 和 MNLI(Natural Language Inference) 。

Zero-shot Transfer Performance

将 projected soft prompt 直接在 target task 上进行预测，结果如 图2。

  • Distance Minimizing：
    仅对 project-training task 有效 ，其余的 task 上的表现和随机初始化 prompt 效果差不多。
    因为目标是想找到对 task 鲁棒性强的训练方法，所以作者提出 embedding distance 和 prompt transferability 没有关系。

  • Task Tuning：
    效果很好，并且只要是和 project-training task 相同类型的 target task，即便没有参与训练，也全都 work；但是对于 unseen different-type task，效果仍然差。

Transfer with Initialization

作者提出 cross-model transferable prompt tuning TPTMODEL\text{TPT}_\text{MODEL}TPTMODEL​ ，使用 Task Tuning projector 得到 projected soft prompt，然后再 target task 上继续 prompt tunning。

同样可以看到，对于 seen same-type task 效果很好。

Exploring Transferability Indicator

作者想探究为什么 soft prompts 能够在不同任务之间迁移，并且是什么决定了这种 cross-task 的迁移能力。
作者提出了两种 prompt similarity matrices：Embedding Similarity 和 Model Stimulation Similarity，并且检验他们是否和 zero-shot transfer 的效果匹配。

Embedding Similarity

给定两个任务 t1t_1t1​ 和 t2t_2t2​ 的 trained soft prompt

Pt1={p1t1,…,plt1}Pt2={p2t1,…,plt2}P^{t_{1}}=\left\{\mathbf{p}_{1}^{t_{1}}, \ldots, \mathbf{p}_{l}^{t_{1}}\right\} \\ P^{t_{2}}=\left\{\mathbf{p}_{2}^{t_{1}}, \ldots, \mathbf{p}_{l}^{t_{2}}\right\} Pt1​={p1t1​​,…,plt1​​}Pt2​={p2t1​​,…,plt2​​}

然后将其全部一维拼接，得到 Pt1,Pt2∈Rld\mathbf{P}^{t_{1}}, \mathbf{P}^{t_{2}} \in \mathbb{R}^{l d}Pt1​,Pt2​∈Rld

然后使用两个指标 Euclidean similarity 和 cosine similarity 计算相似度：

Econcat (Pt1,Pt2)=11+∥Pt1−Pt2∥Cconcat (Pt1,Pt2)=Pt1⋅Pt2∥Pt1∥∥Pt2∥\begin{aligned} &\mathrm{E}_{\text {concat }}\left(P^{t_{1}}, P^{t_{2}}\right)=\frac{1}{1+\left\|\mathbf{P}^{t_{1}}-\mathbf{P} t_{2}\right\|} \\ &\mathrm{C}_{\text {concat }}\left(P^{t_{1}}, P^{t_{2}}\right)=\frac{\mathbf{P}^{t_{1}} \cdot \mathbf{P}^{t_{2}}}{\left\|\mathbf{P}^{t_{1}}\right\|\left\|\mathbf{P}^{t_{2}}\right\|} \end{aligned} ​Econcat ​(Pt1​,Pt2​)=1+∥Pt1​−Pt2​∥1​Cconcat ​(Pt1​,Pt2​)=∥Pt1​∥∥Pt2​∥Pt1​⋅Pt2​​​

这里的计算方式对 soft token 的位置很敏感，所以作者使用一种位置不敏感的方法计算相似度：

Eaverage (Pt1,Pt2)=11+1l2∑i=1l∑j=1l∥pit1−pjt2∥,Caverage (Pt1,Pt2)=1l2∑i=1l∑j=1lpit1⋅pjt2∥pit1∥∥pjt2∥\begin{aligned} &\mathrm{E}_{\text {average }}\left(P^{t_{1}}, P^{t_{2}}\right)=\frac{1}{1+\frac{1}{l^{2}} \sum_{i=1}^{l} \sum_{j=1}^{l}\left\|\mathbf{p}_{i}^{t_{1}}-\mathbf{p}_{j}^{t_{2}}\right\|}, \\ &\mathrm{C}_{\text {average }}\left(P^{t_{1}}, P^{t_{2}}\right)=\frac{1}{l^{2}} \sum_{i=1}^{l} \sum_{j=1}^{l} \frac{\mathbf{p}_{i}^{t_{1}} \cdot \mathbf{p}_{j}^{t_{2}}}{\left\|\mathbf{p}_{i}^{t_{1}}\right\|\left\|\mathbf{p}_{j}^{t_{2}}\right\|} \end{aligned} ​Eaverage ​(Pt1​,Pt2​)=1+l21​∑i=1l​∑j=1l​∥∥∥​pit1​​−pjt2​​∥∥∥​1​,Caverage ​(Pt1​,Pt2​)=l21​i=1∑l​j=1∑l​∥∥∥​pit1​​∥∥∥​∥∥∥​pjt2​​∥∥∥​pit1​​⋅pjt2​​​​

Model Stimulation Similarity

之前有研究表明，模型的表现能力和前向传播中神经元的激活分布有关。
这里，作者认为 prompt similarity 应该和 how theystimulate the PLMs 有关系，i.e. the similarities between the responses of PLMs 。
于是作者提出使用 激活神经元的重叠率（overlapping rate of activated neurons） 来表示 prompt similarity 。

首先 Transformer 中的 FFN 层的公示为：

FFN⁡(x)=max⁡(xW1⊤+b1,0)W2+b2\operatorname{FFN}(\mathbf{x})=\max \left(\mathbf{x} \mathbf{W}_{1}^{\top}+\mathbf{b}_{1}, \mathbf{0}\right) \mathbf{W}_{2}+\mathbf{b}_{2} FFN(x)=max(xW1⊤​+b1​,0)W2​+b2​

W1,W2∈Rdm×d\mathbf{W_1},\mathbf{W_2} \in \mathbb{R}^{d_m\times d}W1​,W2​∈Rdm​×d

这里 max⁡(xW1⊤+b1,0)\max \left(\mathbf{x} \mathbf{W}_{1}^{\top}+\mathbf{b}_{1}, \mathbf{0}\right)max(xW1⊤​+b1​,0) 就是一个 non-negative activation values ，如果该结果大于0，则表示神经元被激活，于是该 token 的 one-hot 激活值为 1，否则为 0。然后得到了整句的 one-hot activation state vector s\mathbf{s}s 。

将上述激活标准应用在 PLM 的 output 上：

  • RoBERTa：输入为 { Prompt, <s>, [mask] }\text{\{ Prompt, <s>, [mask] \}}{ Prompt, <s>, [mask] } ， <s>\text{<s>}<s> 表示 start of output，选择 [mask]\text{[mask]}[mask] 的激活状态作为 output 的激活状态。
  • T5：输入为 { Prompt, <s> }\text{\{ Prompt, <s> \}}{ Prompt, <s> } ，选择生成的第一个 token 的激活状态作为 output 的激活状态。

然后拼接 PLM 的所有中间层激活状态得到 overall activation states：

AS⁡(P)=[s1;s2;…;sL]\operatorname{AS}(P)=\left[\mathbf{s}_{1} ; \mathbf{s}_{2} ; \ldots ; \mathbf{s}_{L}\right] AS(P)=[s1​;s2​;…;sL​]

同时，因为层数越高，神经元越 task-specific，所以实验时作者只选择最上面 3 层的激活状态。

于是就可以计算 the overlapping rate of activated neurons ON⁡(Pt1,Pt2)\operatorname{ON}(P^{t_1}, P^{t_2})ON(Pt1​,Pt2​) ：

ON(Pt1,Pt2)=AS⁡(Pt1)⋅AS⁡(Pt2)∥AS⁡(Pt1)∥∥AS⁡(Pt2)∥\mathrm{ON}\left(P^{t_{1}}, P^{t_{2}}\right)=\frac{\operatorname{AS}\left(P^{t_{1}}\right) \cdot \operatorname{AS}\left(P^{t_{2}}\right)}{\left\|\operatorname{AS}\left(P^{t_{1}}\right)\right\|\left\|\operatorname{AS}\left(P^{t_{2}}\right)\right\|} ON(Pt1​,Pt2​)=∥AS(Pt1​)∥∥AS(Pt2​)∥AS(Pt1​)⋅AS(Pt2​)​

Experimental Results

为了验证上述计算两种 prompt similarity 的方法，作者检验

  1. same task trained prompt 是否相似，different task trained prompt 是否不同。

  2. 是否和 zero-shot transfer performance 匹配。


实验结果

  • same task 时，使用不同的 random seed 得到两个 trained prompt。

    上述两个指标区分效果都很好，这表明不同任务上训练得到的 soft prompt 的确分布不同（但是并不代表 same-type tasks 分布类似，这里只是改了随机数，感觉没什么参考价值）。
    同时激活了 PLM 的不同神经元，即激活了 PLM 的不同能力。

  • 对于每个 target task，按照相似度分数对 source task 进行排序，同时对 zero-shot transfer performance 进行排序，然后计算两个序列的 Spearman’s rank correlation 。

    • overlapping rate of activated neurons (ON) 指标比其他所有指标效果都好
    • ON 在 T5 XXL 上的表现比在 RoBERTa Large 上的差，表明大模型参数更加冗余（redundant），所以更大的模型可以激活不同的神经元来进行不同的任务。

      作者针对不同的模型尺寸进行实验，的确验证了猜想。
    • 同时为了验证是否在 cross-model transfer setting 场景上有效，作者检验了相同任务上的 the projected prompts by Task Tuning projectors 和 original turned prompts 之间的相似度。

      发现 Embedding Similarity 指标与迁移 prompt 的表现不一致；
      而 ON 在 same-type projector-training tasks 上表现好，在 different-type projector-training tasks 上表现差，与迁移 prompt 的表现一致。
