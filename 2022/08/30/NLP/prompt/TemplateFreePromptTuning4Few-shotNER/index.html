Template free Prompt Tuning for Few-shot NER

Posted on 2022-08-30 In NLP , Prompt
Symbols count in article: 1.3k Reading time ≈ 6 mins.

Paper: Template-free Prompt Tuning for Few-shot NER
Code: EntLM

Abstract and Introduction

For prompt tunning, the template-based prompt methods are designed for sentence-level tasks.
But NER is based on token-level recognition. In this case, some prompt-based method for NER enumerate every possible spans for a classifications:

However, it is very time-consuming !!

In this paper, the author propose to reformulate NER as an LM task with an Entity-oriented LM(EntLM) object w/o any templates.
More specifically, the pre-trained LMs are fine-tuned to

  • at the entity positions: predict class-related pivot words (or label words) instead of the original words
  • at none-entity positions: predict the original word

To find better class-related pivot words (or label words), some methods are proposed:

  • select discrete label words based on the word distribution in lexicon-annotated corpus or LM predictions
  • obtain the prototypes as virtual label words

Benefits:

  • No new parameters are introduced.
  • EntLM only introduces a slight adjustment of the output distribution for NER tasks, which suits few-shot learning.
  • Well-selected label words accelerate the adaptation of LM distribution, which promotes few-shot performance.
  • EntLM requires only one-pass decoding, saving lots of time.

Model

A template-free prompt tuning method: Entity-oriented LM (EntLM) fine-tuning .

E.g., when getting an input sequence, “Obama was born in America",
the LM is trained to predict a label word “John" at the position of the entity “Obama" as an indication of the label “PER".

Formally, given an input sequence X={x1,...,xn}X=\{x_1,...,x_n\}X={x1​,...,xn​} and corresponding label sequence Y={y1,...,yn}Y=\{y_1,...,y_n\}Y={y1​,...,yn​}.
Then, the token at the entity position iii (its entity label is yiy_iyi​) will be replaced with corresponding label word M(yi)\mathcal M(y_i)M(yi​).
After replacing all the entity tokens, we will get a target sentence XEnt={x1,...,M(yi)...,xn}X^{Ent}=\{x_1,...,M(y_i)...,x_n\}XEnt={x1​,...,M(yi​)...,xn​}.

Label Word Engineering

How to automatically selecting proper label words ?
Specifically, the purpose of label word engineering is to find a pivot word that can mostly represent the words in each class.

Low-resource Label word selection

Since it is in a few-shot setting, the randomness sampling will largely affect the selection randomness.

So the author use unlabeled data and lexicon-based annotation as a resource for label word searching.

In NER, the entity lexicon of target classes are usually easy to access.

The author adopt the KB-matching approach proposed by Liang et al. (2020), which leverages an external KBs, wikidata, as the source of lexicon annotation.

These lexicon-based annotation is inevitably noisy, but the authors says this approach do not suffers a lot from the noise since we only regarded it as an indication of the data distribution and do not train the model directly with the noisy annotation.

Label word searching

We have gotten a lexicon-annotated data Dlexicon={(Xi,Yi∗)}i=1N\mathcal D_{lexicon}=\left\{\left(X_{i}, Y_{i}^{*}\right)\right\}_{i=1}^{N}Dlexicon​={(Xi​,Yi∗​)}i=1N​.

There are three methods for label word searching:

Data search

Searching with data distribution (Data search): Select the most frequent word of the given class in the corpus.

Formally, calculate the frequency of each candidate label word w∈Vw\in \mathcal Vw∈V labeled as class label CCC, and select the most frequent words by ranking:

M(C)=arg⁡max⁡wϕ(x=w,y∗=C)\mathcal{M}(C)=\arg \max _{w} \phi\left(x=w, y^{*}=C\right) M(C)=argwmax​ϕ(x=w,y∗=C)

LM search

Searching with LM output distribution (LM search): Leverage the pre-trained language model for label word searching.

Formally, feed each sample (X,Y∗)\left(X, Y^{*}\right)(X,Y∗) into the LM, and get the probability distribution p(x^i=w∣X)p\left(\hat{x}_{i}=w \mid X\right)p(x^i​=w∣X). From the distribution, an indictor representing whether label word www belongs to the top-k predictions of xix_ixi​ in sample (X,Y∗)\left(X, Y^{*}\right)(X,Y∗) is calculated as:

Virtual label words

To search for virtual label words, the mean vectors of the high-frequency words of each class is calculated as prototypes.
