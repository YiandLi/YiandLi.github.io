EPR：Efficient Prompt Retriever

Posted on 2022-08-12 In NLP , Text Retrieval
Symbols count in article: 3.1k Reading time ≈ 16 mins.

论文：Learning To Retrieve Prompts for In-Context Learning

Abstract and Introduction

In-Context Learning：将训练集中的一些/个 training example(s) 看作 prompt，和 test instance 拼接在一起输入到模型中，在模型不更新参数的情况下，让模型直接输出答案。即 PLM(xtest,(xtrain,ytrain))→ytest\text{PLM}\left(x_{test}, \left(x_{train}, y_{train}\right)\right) \to y_{test}PLM(xtest​,(xtrain​,ytrain​))→ytest​ 。

上下文学习（in-context learning），检索和OOD外推：

  • 这也是一种隐式的 meta-learning，模型如果能有这个能力说明他可能完成了某种从 training examples 中了解到自己该是怎么通过x预测y这件事，然后完成对真正我们的输入进行预测。
  • 优势：在面对新 task 时，模型能够不需要额外的训练，自己 “外推” 出解决方案 PLM(xtest,(xtrain,ytrain))→ytest\text{PLM}\left(x_{test}, \left(x_{train}, y_{train}\right)\right) \to y_{test}PLM(xtest​,(xtrain​,ytrain​))→ytest​ .这种期待是我们之前在对待模型时候没有的，在这一点上实际上已经和传统的 few-shot（tunning）不一样了。
  • in-context learning 使得模型有着天然的泛化能力和实际部署的潜质，在很多领域，通过合理的构建 prompt 和选取 training examples，in-context learning 的水平已经接近于比他自己小但是本身不小的模型的能力了（比如在 GPT3 175B 上做 in-context learning 的性能基本和 T5-large 770M 全数据训练finetune 持平.

In-Context Learning 已经成为 NLU 任务的一种范式（ paradigm ）。
但是这种方法很依赖 in-context examples ，进而带来一个新的方向，即如何更好地选择 in-context examples，作者称其为 prompt retrieval 。

现存工作的做法为：

  • 无监督：离线计算相似度
  • 有监督：训练一个 prompt retriever，来根据相似度选择 in-context examples。

本文中，作者提出了一种新方法，名为 EPR：

  1. 使用一个 scoring PLM 挑选最合适的 in-context examples / prompt examples，组成 training data
  2. 使用对比学习训练出一个 prompt retriever。

作者认为这种 scoring PLM 构造训练数据的方法，比之前提出的方法都要有效，因为这种方式和推理时的过程一致，并且可以引入 scoring PLM 的先验信息。

  1. 不用得到 PLM 的参数，适合离线先训练 retriever，或者直接调用 API。
  2. 当 scoring LM 比 inference LM 小时，可以作为一种 cheap and efficient 的训练样本生成方式。（不过这一点我有一些疑问，soft prompt 迁移不适用于不同模型，为什么这个可以？）

Efficient Prompt Retriever

Setup

训练集合： D={(xi,yi)}i=1n\mathcal{D} = \{(x_i, y_i)\}^n_{i=1}D={(xi​,yi​)}i=1n​
测试样本： xtestx_{test}xtest​
目标为训练得到一个 prompt 检索器： R(xtest,D)\mathcal R(x_{test}, \mathcal D)R(xtest​,D) ，得到适合 xtestx_{test}xtest​ 的 Prompt P={(xj,yj)}j=1m⊂D\mathcal{P}=\left\{\left(x_{j}, y_{j}\right)\right\}_{j=1}^{m} \subset \mathcal{D}P={(xj​,yj​)}j=1m​⊂D 。
In-Context Learning 过程： LM g([P;xtest])→ytest\text{LM}\ g([\mathcal P ; x_{test}])\to y_{test}LM g([P;xtest​])→ytest​

Generating the Training Data

对每个训练样本，都需要找好的 prompt training examples 来得到 prompt training examples。
但是使用枚举法计算消耗太大，作者的做法为先使用一个非监督 retriever 得到一个候选子集合 E‾=Ru((x,y),D)\overline{\mathcal{E}}=R_{u}((x, y), \mathcal{D})E=Ru​((x,y),D) 。

  • 对于这个 非监督 retriever ，作者尝试了 BM25 (使用表层语义相似度) 和 SBERT (使用深层文本表征)
  • 对于 非监督 retriever 的输入，作者尝试使用 example pair (x,y)(x, y)(x,y) 和只使用 target output sequence yyy ，发现后者效果更好。

Scoring the candidate set

粗排得到训练样本 (x,y)(x,y)(x,y) 的候选 prompt training examples E‾={eˉ1,⋯ ,eˉL}\overline{\mathcal{E}}=\left\{\bar{e}_{1}, \cdots, \bar{e}_{L}\right\}E={eˉ1​,⋯,eˉL​} 后，使用一个 scoring LM g^\hat{g}g^​ 独立为每个 eˉl\bar{e}_leˉl​ 进行独立地打分：

s(eˉl)=Prob⁡g^(y∣eˉl,x)s\left(\bar{e}_{l}\right)=\operatorname{Prob}_{\hat{g}}\left(y \mid \bar{e}_{l}, x\right) s(eˉl​)=Probg^​​(y∣eˉl​,x)

然后根据 s(eˉl)s\left(\bar{e}_{l}\right)s(eˉl​) ，选择其中 top-k 个样本作为 正样本集合 E(pos){\mathcal{E}}_{\text(pos)}E(pos)​ ； bottom-k 个样本作为 负样本集合 E(neg){\mathcal{E}}_{\text(neg)}E(neg)​ 。

关于数据质量：粗排已经保证了样本相似度都类似，所以再使用 Top-K 的做法选择 正负样本集合能够有效地得到 positive 和 hard negative 集合。

Training and Inference

两个 encoder：

  • EX(⋅)E_{X}(\cdot)EX​(⋅) ：编码输入样本的输入序列（input） xxx
  • EP(⋅)E_{P}(\cdot)EP​(⋅) ：编码某个候选 prompt 样本的输入输出对（input-output pair） (x,y)(x,y)(x,y)

上述两个encoder 全都使用 BERT-base ，并且使用 CLS 做为输出表征。

Training

目标是学习一个 similarity metric，进行两者的相似度打分，进而在推理时选择分数高的作为 prompt。

使用对比学习，batch size 为 BBB ，
其中每个训练样本为 ⟨xi,ei+,ei,1−,…ei,2B−1−⟩\left\langle x_{i}, e_{i}^{+}, e_{i, 1}^{-}, \ldots e_{i, 2 B-1}^{-}\right\rangle⟨xi​,ei+​,ei,1−​,…ei,2B−1−​⟩ ，其中包含：

  • xix_ixi​ ：anchor train instance
  • 一个 ei+e_{i}^{+}ei+​ ：anchor positive example sampled from the set E(pos){\mathcal{E}}_{\text(pos)}E(pos)​
  • 一个 anchor hard negative example sampled from the set E(neg){\mathcal{E}}_{\text(neg)}E(neg)​
  • B-1 个 other instances’ positive example
  • B-1 个 other instances’ hard negative example

应该是任务比较特殊，所以可以使用其他样本的 positive examples 和 hard negative examples 作为 anchor example 的负样本。

使用「输入样本」和「输入输出对」的内积结果表示相似度：

sim⁡(x,e)=EX(x)⊤EP(e)\operatorname{sim}(x, e)=E_{X}(x)^{\top} E_{P}(e) sim(x,e)=EX​(x)⊤EP​(e)

使用负对数似然函数表示对比学习的损失：

L(xi,ei+,ei,1−,…ei,2B−1−)=−log⁡esim⁡(xi,ei+)esim⁡(xi,ei+)+∑j=12B−1esim⁡(xi,ei,j−)\begin{aligned} & L\left(x_{i}, e_{i}^{+}, e_{i, 1}^{-}, \ldots e_{i, 2 B-1}^{-}\right) \\ =&-\log \frac{e^{\operatorname{sim}\left(x_{i}, e_{i}^{+}\right)}}{e^{\operatorname{sim}\left(x_{i}, e_{i}^{+}\right)}+\sum_{j=1}^{2 B-1} e^{\operatorname{sim}\left(x_{i}, e_{i, j}^{-}\right)}} \end{aligned} =​L(xi​,ei+​,ei,1−​,…ei,2B−1−​)−logesim(xi​,ei+​)+∑j=12B−1​esim(xi​,ei,j−​)esim(xi​,ei+​)​​

使用这种 in-batch negatives trick 后，计算复杂度为 O(B2)O(B^2)O(B2) 。

Inference

给定 xtestx_{test}xtest​ ，使用 EX(⋅)E_{X}(\cdot)EX​(⋅) 编码输入序列；使用 EP(⋅)E_{P}(\cdot)EP​(⋅) 将训练集输入输出对全部编码。
使用内积相似度，筛选并且按照相似度从高到低得到 top-L 个训练样本作为 prompt P=(e1,…,eL)\mathcal{P}=\left(e_{1}, \ldots, e_{L}\right)P=(e1​,…,eL​) 。
这里我们可以设置一个超参数 CCC ，通常为 inference model 支持的 maximal context size ，来得到最终的 prompt p′\mathcal{p}'p′ ，比如：

∑i=1L′∣ei∣+∣xtest ∣+∣y′∣≤C\sum_{i=1}^{L^{\prime}}\left|e_{i}\right|+\left|x_{\text {test }}\right|+\left|y^{\prime}\right| \leq C i=1∑L′​∣ei​∣+∣xtest ​∣+∣y′∣≤C

L′L^{\prime}L′ 表示 prompt examples 的最大数量，这其实也是一种 贪心算法。

最终得到输入序列为：

[eL′;eL′−1;…;e1;xtest ]\left[e_{L^{\prime}} ; e_{L^{\prime}-1} ; \ldots ; e_{1} ; x_{\text {test }}\right] [eL′​;eL′−1​;…;e1​;xtest ​]

注意 Training 和 Inference 其实存在一个 gap，即训练时独立筛选 example ，而推理时将多个 examples 一起输入到模型。

Experiment

  • Baseline：

    • Unsupervised Baselines：directly used when testing

      • RANDOM：随机抽取 prompt examples
      • SBERT：根据输入序列相似度抽取 prompt examples
      • BM25
      • BRUTEFORC：根据 Prob⁡g(xtest ∣xi)\operatorname{Prob}_{g}\left(x_{\text {test }} \mid x_{i}\right)Probg​(xtest ​∣xi​) 抽取 prompt examples

    • Supervised Baseline：
      firstly BM25 for 50 candidate examples, top-5 and bottom-5 are picked with scoring function：

      • DR-BM25：仍然选择 BM25 挑选 contrastive learning training examples，并且训练一个 dense retriever
      • CASE-BASED REASONING (CBR)： Das et al. (2021)
      • EFFICIENT PROMPT RETRIEVAL (EPR)：本文的方法
      • Oracle models 预言模型：
        • BM25-ORACLE：将 gold test output 放入 DR-BM25。
          这应该能够表示 DR-BM25 的上限：相当于直接按照标准答案用 BM25 进行检索，
        • LM-ORACLE：将 gold test output 放入 DR-BM25 中，然后使用 scoring LM 进行 sort。
          这应该是 EPR 的上界：相当于直接按照标准答案用 BM25 进行检索，但是按照 scoring LM 的效果进行排序。

  • Models：

    • a dense retriever（这里说的有点模糊了，本文的方法是两个 bert 才对，不过组合起来确实是一个双塔的 dense retriever）
    • scoring LM： GPT-NEO（2.7B）
    • inference LMs：
      • GPT-J（6B）：scoring LM > inference LM;
      • GPT-3（175B）：scoring LM < inference LM;
      • CODEX（175B）：scoring LM < inference LM;

  • maximum context size：2048 tokens


  • Datasets：强调模型的表示能力（ map utterances to meaning representations ）
    • BREAK (Wolfson et al., 2020)：将复杂的问题转化为原子操作的序列
      development set 上的评价指标为 LF-EM (Hasson and Berant, 2021)，test set 上评价指标为 NEM。
    • MTOP (Li et al., 2021)：A semantic parsing dataset, focused on task-oriented dialogue。将 command 映射到 多个query 。
      使用 Exact Match (EM) 作为评价指标。
    • SMCALFLOW (Andreas et al., 2020)：将 command 映射到 dataflow program 。
      使用 Exact Match (EM) 作为评价指标。

Result

检测方法：

  • LM-as-a-service：
    • scoring model 和 inference model 都是 GPT-NEO
    • 在 full development sets 上检测效果
  • LM-as-a-proxy：
    • scoring model 为 GPT-NEO
    • 因为 inference model 为 GPT-3 和 CODEX ，需要自费调用 API，所以只选择 1000 个 dev instances 进行测试。

LM-as-a-service


dev集合 和 test集合上效果都最好

  • unsupervised BM25 > unsupervised SBERT：所以在 supervised 粗排时，使用 BM25。
  • EPR > BM25-ORACLE：score LM 给出的信号 比 surface text similarity 要好。
  • LM-ORACLE > EPR：score LM 给出的信号 很有效。


前文提到训练和测试时候使用的 prompt 数量存在 gap，这里 one-shot 即只选择一个最优 prompt example。
可以看到尽管存在 gap，但是只有一个 prompt example 效果还是没那么好。

LM-as-a-proxy

scoring model 为 GPT-NEO，inference model 为其他大模型。

作者称小模型向大模型迁移是可行的，但是可以看到，有时候表现还是会变差，可以研究下如何进行对齐工作。

Analysis

Prompt copying：作者想看 in-context prompts 是如何被模型利用的。

作者定义两种 copy 形式：

  1. Exact copying：target output 是否和某一个 prompt example 完全一致
  2. Abstract copying：target output 的结构是否和 prompt examples 的结构一致

实验我有点读不太懂，结论是：

  1. 模型可以学习到 prompt 中的一些结构化的知识
  2. 模型的 copying 现象最常出现在 highly-ranked prompts

Personal Idea

  • 反推本文的思路：
    • 问题：从 training set 中找到最合适的 prompt examples
    • 上述问题可以定义检索问题（按照概率排序取 top），评价对象为 source（test sequence）和 candidate（training examples）的相似度（dot product）。
    • 那么如何检索呢，当然可以用 unsupervised 的方式，但是效果不好啊。那么我们既然有标注数据，那就考虑训练一个检索器吧。
    • 那么对每一个 training example，我们应该从训练集中，找到最合适的 prompt example。
    • 怎么找呢？就定义一个 score LM 吧，让模型自己去评价好坏。
    • OK，现在有了真正的训练检索器的训练数据了，我们就要这个检索器使用什么模型呢 ？那就对比学习模型吧。
    • 谈到对比学习，肯定需要一个靠谱的 编码器 来编码评价对象，这时我们就想到了 bert cls 表示。
    • 但是直接用 cls 效果估计不太好，因为两者表示的语义都不一样，一个是输入样本，一个是 样本pair。那就用双塔结构吧！

可以看到，其实本文的创新点就在于两点：

  1. 让 score LM 模型自己去评价好坏。这一点其实很重要，我们在自动挑选一些训练指标或者其他element时候，一定要保证和 inference 保持一致，这里体现的还不明显。但是训练时的 score LM 最好和推理时的 inference LM 一致。
  2. 使用对比学习的思想构造 检索器，这里不用对比学习行不行，当然也可以，只要能保证可以计算(x,y) 和 x 之间的相似度（或者其他指标），从而根据这个指标选择合适的prompt example 就可以了。 eg. 我直接使用语义匹配做行不行，当然也可以！

  • in-context learning 是不是只能使用在生成模型上呢，或者说用在自编码模型上效果不佳。因为
    • 填空法对比 prefix 补全，感觉模型学到的 x 到 y 到知识没那么好。
    • 同时，对于信息抽取任务，或者其他非生成的方式，我们还需要一个 decoder ，而这个 decoder 的参数 in-context learning 肯定是学不到的。
      prompt 倒是提供了一种思路，使用 verbalizer 进行映射，但是抽取式怎么办呢，使用指针网络吗？
      这样看来，为什么统一模型都适用 生成式 倒是有原因的，不依赖 decoder，肯定能 generalize。
  • 虽然本文提出的方法比 baseline 要强，但是实际效果还是不忍直视。回头再看看 “in-context learning 的水平已经接近于比他自己小但是本身不小的模型的能力了”。
  • 本论文还是有一个迁移问题，scoring model 选择出来的 prompt，迁移到 inference model ，估计还是有语义 gap，但是为什么 有些情况下效果反而更好呢？这应该是一个可以继续讨论的点。
  • 既然都需要训练，是否可以尝试 构造连续 prompt 呢？
