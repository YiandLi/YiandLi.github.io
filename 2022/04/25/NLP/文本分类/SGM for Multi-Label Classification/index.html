SGM for Multi-Label Classification

Posted on 2022-04-25 In NLP , Text Classification
Symbols count in article: 2.2k Reading time ≈ 14 mins.

论文：SGM: Sequence Generation Model for Multi-label Classification (COLING 2018)
代码：lancopku/SGM

Abstract and Introduction

多标签分类问题（multi-label classification / MLC）：一个样本同时属于多个类别。
和单标签分类问题相比，多标签问分类题的难点在于：

  1. 标签之间存在联系
  2. 文本的「不同部分」属于「不同的标签」
    「different textual contents」 do 「different contributions」 for 「different labels」

针对以上问题，作者提出 一个序列生成模型 和 一种全新的解码策略。
使模型在预测时能够 考虑不同 label 之间的关系 ，并且 自动选择最相关的单词。

Proposed Method

标签集合： L={l1,l2,...,lL}\mathcal L=\{l_1, l_2,..., l_L \}L={l1​,l2​,...,lL​}
输入序列 xxx ，长度为 mmm 。
预测标签集合 yyy ，大小为 nnn 。
则使用序列生成的方式，训练目标是最大化 标签序列 的概率：

p(y∣x)=∏i=1np(yi∣y1,y2,⋯ ,yi−1,x)p(\boldsymbol{y} \mid \boldsymbol{x})=\prod_{i=1}^{n} p\left(y_{i} \mid y_{1}, y_{2}, \cdots, y_{i-1}, \boldsymbol{x}\right) p(y∣x)=i=1∏n​p(yi​∣y1​,y2​,⋯,yi−1​,x)

在标签集中，样本标签是已知，那么如何得到每个样本的 标签序列 ？即如何判定不同标签的先后位？
作者首先根据「频率」将标签排序，即「大类标签」放前面，「小类标签」放后面。然后，在序列头尾插入[bos]和[eos]标签（begin/end of sequence）。

出现次数更多的标签在标签相关性训练中具有更强的作用，即先被预测出的「大类标签」可以更好地指导「小类标签」/序列的输出。

标签是以序列的方式生成的，即每个时间步生成一个 label 。

预测步骤为：

  • 先将每个输入 token 编码为对应隐藏层向量
  • 在每个时间步 ttt 上：
    • decoder：使用上一个时间步的上下文向量 ct−1c_{t-1}ct−1​ ，隐状态 st−1s_{t-1}st−1​ 和预测标签的 embedding g(y(t−1))g(y(t-1))g(y(t−1)) ，得到当前时间步的隐状态 sts_{t}st​ 。
    • encoder：使用attention机制将这些隐藏层向量聚合为当前的上下文向量 ctc_tct​ 。
    • decoder：将 ctc_tct​ 和 sts_tst​ 放入 masked softmax layer ，得到标签概率分布 yty_tyt​ ，并输出标签。

总结：每一个时间步有自己的上下文编码 ctc_tct​ ，隐状态 sts_tst​ 和标签的概率分布 yty_tyt​ ，同时得到的先后顺序为 st,ct,yts_t, c_t, y_tst​,ct​,yt​ 。

Train and Inference

训练时使用交叉熵损失函数进行训练
推理时，使用 Beam-search ，保留 top K 条路径，在模型输出 [eos] 时停止生成标签。
beam search 参考：https://baijiahao.baidu.com/s?id=1676193446313598976&wfr=spider&for=pc

Global Embedding

上面在计算隐状态时，使用的是 g(yt−1)g\left(\boldsymbol{y}_{t-1}\right)g(yt−1​) ，即最大概率标签的 embedding（贪心算法，hard label）。
这会带来问题： exposure bias ，即一步错，步步错。
推理时，Beam-search 在一定程度上缓解了这个问题，因为有候选序列。但是这并不能从根本上解决这个问题，因为存在情况：所有的候选序列都是错的。

受到 adaptive gate in highway network (Srivastava et al., 2015) 的启发，作者提出 Global Embedding，用来代替上述 hard label embedding：

soft label embedding

首先使用预测概率值求得 soft label embedding / weighted average embedding：

e‾=∑i=1Lyt−1iei\overline{\boldsymbol{e}}=\sum_{i=1}^{L} y_{t-1}^{i} \boldsymbol{e}_{i} e=i=1∑L​yt−1i​ei​

LLL 为标签数量； eie_iei​ 为第 i 个标签的 embedding ； yty_tyt​ 为第 t 个标签的概率值。

transform gate

计算 soft label embedding 的权重 H\boldsymbol{H}H （ transform gate ）：

H=σ(W1e+W2e‾)\boldsymbol{H}=\sigma\left(\boldsymbol{W}_{1} \boldsymbol{e}+\boldsymbol{W}_{2} \overline{\boldsymbol{e}}\right) H=σ(W1​e+W2​e)

论文中没有加 σ\sigmaσ 激活函数，并指明 W1,W2∈RL×LW_1, W_2\in \mathbb{R}^{L\times L}W1​,W2​∈RL×L ，但是 LLL 是 label 数量，这应该是写错了。

get global embedding

最后得到加权平均，得到 the optimized combination of the original embedding and the weighted average embedding ：

g(yt−1)=(1−H)⊙e+H⊙e‾g\left(\boldsymbol{y}_{t-1}\right)=(\mathbf{1}-\boldsymbol{H}) \odot \boldsymbol{e}+\boldsymbol{H} \odot \overline{\boldsymbol{e}} g(yt−1​)=(1−H)⊙e+H⊙e

e\boldsymbol{e}e 仅仅包含当前 label 的信息 (hard embedding)；而 e‾\overline{\boldsymbol{e}}e 包含了所有 label 的信息 (soft embedding)。
这样模型能够同时考虑所有的信息，减少先前时间步带来的错误。

Experiment

Dataset

  • Reuters Corpus Volume I (RCV1-V2)
  • Arxiv Academic Paper Dataset (AAPD)：作者自定义的，数据为 cs 领域的 paper，一共 54 个类别。

Evaluation Metrics

  • Hamming loss：计算错误分类的 instance-label pairs 的分数，错误分为：
    • 预测出错误 label
    • 没有预测出目标 label
  • Micro F1

Detail

hidden size: 256 for encoder, 512 for decoder
max seq len: 500
beam size: 5 / 9
opt: Adam

Result

Baseline:

  • Binary Relevance (BR)：将多标签任务转换为多个单独的二分类任务
  • Classifier Chains (CC)：将多标签任转换为多个二分类任务，但是在预测时，考虑 high-order 标签的结果。
  • Label Powerset (LP)：将多标签任务转换为多类别任务（multi-class problem），with one multi-class classifier trained on all unique label combinations.

Analysis and Discussion

Exploration of Global Embedding

原始 Global Embedding 公式为：

g(yt−1)=(1−H)⊙e+H⊙e‾g\left(\boldsymbol{y}_{t-1}\right)=(\mathbf{1}-\boldsymbol{H}) \odot \boldsymbol{e}+\boldsymbol{H} \odot \overline{\boldsymbol{e}} g(yt−1​)=(1−H)⊙e+H⊙e

作者想探究 e‾\overline{\boldsymbol{e}}e 可以带来多大提升。于是将 H\boldsymbol{H}H 设置为一个常量：

g(yt−1)=(1−λ)⊙e+λ⊙e‾g\left(\boldsymbol{y}_{t-1}\right)=(\mathbf{1}-{\lambda}) \odot \boldsymbol{e}+{\lambda} \odot \overline{\boldsymbol{e}} g(yt−1​)=(1−λ)⊙e+λ⊙e

下图横坐标为 λ\lambdaλ 系数的值，纵坐标为模型的分数：

λ=0\lambda=0λ=0 时，效果最差，即引入 e‾\overline{\boldsymbol{e}}e 一定可以带来效果增益。
不同的 λ\lambdaλ 取值下，模型表现相差很多。这也是 adaptive gate H\boldsymbol{H}H 有效的原因：模型可以动态调节权重，达到最优效果。

The Impact of Mask and Sorting

mask 机制：防止模型重复预测标签
sorting 机制：训练时将标签按照频率排列

sorting 带来更多增益
mask 机制带来的增益不多，说明模型本身重复预测的情况就不严重。
（ the mask module has little impact on the models’ performance

Error Analysis

随着标签序列长度增加，模型效果变差。
作者将测试集按照 LLS(the length of the label sequence) 分成了多个子集合，并且实验了多个模型。

随着序列增长，所有模型的效果都会变差，但是 SGM 仍然是最好的。

Visualization of Attention

预测标签时需要用到隐状态 s\boldsymbol{s}s ，计算隐状态时需要用到上下文表征 c\boldsymbol{c}c ，此时每个 token 有对应的权重 α\alphaα 。
作者进行了可视化：

预测 CV 标签时，image, visual, captioning 权重较高。
预测 CL 标签时，sentence, memory, recurrent 权重较高。
表明 模型在预测时可以有效参考信息量最高，最相关的单词。

Case Study

探究模型是否能够 label 之间的信息

红色标签表示两两强相关
BR algorithm 不能捕捉 label 之间的信息；SGM 可以捕捉交互信息，加上 GLobal Embedding 之后模型可以预测更多正确标签。
