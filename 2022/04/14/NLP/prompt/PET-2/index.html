Abstract and Introduction

GPT-3 在 few-shot 场景下效果很好，但是：

  1. 需要一个巨大的 LM 模型，但是真实情况下我们无法支持这么大的模型
  2. It does not scale to more than a few examples as the context window of most LMs is limited to a few hundred tokens.

上篇 PET 存在一个缺点：套用模版后，输入序列只有一个[mask]，即 one-hot label 只能对应一个 token ，这在很多任务上是不适用的。
所以本文中，作者：

  1. 对上篇的PET模型做出 改进 ，支持标签对应 多个tokens （predict multiple tokens）。
  2. 证明该模型在 不使用未标注数据 的情况，仍然能得到相似的表现能力。
  3. 分析了导致 PET模型 强大效果的 因素 。

从实验结果上看：

  1. 同样仅使用 32 个样本，效果超越 GPT-3 。
  2. 参数量为 GPT-3 的 0.1% ，训练成本和时间也短得多。

Training

使用上面递归的方式计算训练样本token的得到成本很高，为了在一个 forward-pass 中得到整体标签分数，作者选择：

  1. 总是选择最大的 mask size 进行模版转换。
  2. 并行预测所有的token，对于那些没有到最大长度 l(x)l(x)l(x) ，我们则直接忽略 l(x)−kl(x)-kl(x)−k 位置上的 token 分数。

Analysis

Pattern

三种设置 PVP 的方式：

  1. 使用 GPT-3 的模型
  2. 使用本论文中的方式
  3. 结合 1 2 中的方式

结论：

  1. 针对任务选择 PVP 很重要
  2. 组合 PVP 效果好很多

Unlabeled Data Usage

  1. 直接使用 pvp 集成模型（不使用未标注数据）
  2. 使用最终的 PET 蒸馏模型

从表2中看出，集成模型甚至比蒸馏模型效果更好；但是蒸馏模型在后续预测时候，占用的内存和算力更小。

Labeled Data Usage

  1. 小样本场景下，PET 优于 微调范式。
  2. 比较了 32-examples 和 single-example learning 场景下模型表现的差异，GPT priming 提升能力很小，甚至出现负增涨，而 PET 下的效果则大幅度提升（最后一行）。

PET with Multiple Masks

在本文中，在预测多个 [mask] 时，作者选择依次选择最大分数的token得到最终分数(max-first)。
此外，作者还尝试了几种其他的预测方式：

  1. 从左到右依次预测（ltr）
  2. 同时预测所有的token（parallel），就像做训练时一样。

同时，取一个未经过训练的 ALBERT 作为对照组。
