论文：Adversarial Transfer for Named Entity Boundary Detection with Pointer Networks

Abstract and Introduction

针对 命名实体边界识别问题/named entity boundary detection （不对判别实体类别），本文首先提出一种全新的实体边界标注方式（ptr network），然后提出 AT-BDRY 模型，应用于 无标注场景下的 ner 模型迁移。

AT-BDRY could achieve state-of-the-art transferring performance against recent baselines.

研究命名实体边界识别问题/named entity boundary detection （不对判别实体类别）的意义：

  1. 将「命名实体识别问题」转化为「实体边界预测问题」+「实体分类问题」，精确的「实体边界预测模型」可以避免「实体边界预测错误」带给「命名实体识别问题」的误差。
  2. 「实体链接任务」更重视「实体边界」。

AT-BDRY：

  1. 将 Pointer 机制 整合到序列标注架构中，解决实体边界中的不定词表问题。
  2. 引入 对抗学习 提高模型迁移的鲁棒性：使用 源鉴别器 a domain discriminator (adversary) （参考GAN）。

如何训练 模型迁移：让鉴别器分辨不出来编码句来自「source语料」还是「target语料」。
即加入对抗学习的目的是：

  1. 使 context encoder 的输出不包含输入源的提示信息
  2. 从而误导 domain discriminator ，并且引导 tag decoder。
  3. 最终使得模型保留 sequence labeling 的特征，忽略区别句子来自 source 还是 target 的特征（encourage domain invariant features）。

In particular, we incorporate adversarial training in the shared space to ensure that the intermediate representations by the context encoder contain no discriminative information about the origin of input (source or target). That is, we hope that the intermediate representations can mislead the domain discriminator and correctly guide the tag decoder prediction.

Model

  1. The input representation layer: Embedding
  2. The context encoder
  3. The tag decoder (ptr network)
  4. The domain discriminator

Input Representation

character-level representations + word-level representations

数据集：

  • annotated data DS:S1,S2,…,SN\mathcal{D}_{S}: S_{1}, S_{2}, \ldots, S_{N}DS​:S1​,S2​,…,SN​ in the source domain
  • unannotated data： DT:T1,T2,…,TN\mathcal{D}_{T}: T_{1}, T_{2}, \ldots, T_{N}DT​:T1​,T2​,…,TN​ in the target domain

输入句子：

W∈{DS,DT}W=(W1,W2,…,WL)\begin{aligned} &\boldsymbol{W} \in \{\mathcal{D}_{S},\mathcal{D}_{T}\} \\ &\boldsymbol{W}=\left(W_{1}, W_{2}, \ldots, W_{L}\right) \end{aligned} ​W∈{DS​,DT​}W=(W1​,W2​,…,WL​)​

对于单词 WlW_lWl​ ，使用双层 CNN 得到「字粒度向量」，然后和「词向量」拼接，得到最终的词向量 xl∈RK\boldsymbol{x}_l\in\mathbb{R}^Kxl​∈RK 。

Context Encoder

X=(x1,x2,…,xL)\boldsymbol{X}=\left(\boldsymbol{x}_{1}, \boldsymbol{x}_{2}, \ldots, \boldsymbol{x}_{L}\right) X=(x1​,x2​,…,xL​)

使用 BiGRU (因为比 BiLSTM 更有效率)，得到隐藏层向量：

hl=h→l⊕h←l\boldsymbol{h}_{l}=\overrightarrow{\boldsymbol{h}}_{l} \oplus \overleftarrow{\boldsymbol{h}}_{l} hl​=hl​⊕hl​

⊕\oplus⊕ 表示 concatenation 。于是，序列的向量表示为 h∈RL∗2H\boldsymbol{h}\in\mathbb{R}^{L*2H}h∈RL∗2H。

单层隐藏层维度为 HHH，拼接后单词维度为 2H2H2H ，序列维度为 L∗2HL*2HL∗2H 。

Tag Decoder

分两步，先是seq2seq后一步的RNN，然后是 ptr network。

Unidirectional GRU

使用 单层 GRU 进行解码，输出 dl\boldsymbol{d}_{l}dl​ 。
注意不是所有的单词都要输入到 decoder，而是只需要输入实体的 起始单词。比如 “Michael Jeffrey Jordan”，只需要输入 “Michael” 。
与encoder不同，这里的 单层隐藏层维度为 2H2H2H 。于是，序列的向量表示为 d∈RJ∗2H\boldsymbol{d}\in\mathbb{R}^{J*2H}d∈RJ∗2H 。

Ptr networking

在指针阶段 pointing phase，作者实现了一种机制：

  • 对于 实体起始单词，预测其对应的 实体结束单词。
  • 对于其他单词，将其设置为inactive 状态，不做预测。

记上面两个序列分别为 h∈RL∗2H\boldsymbol{h}\in\mathbb{R}^{L*2H}h∈RL∗2H 和 d∈RJ∗2H\boldsymbol{d}\in\mathbb{R}^{J*2H}d∈RJ∗2H 。
作者在 h\boldsymbol{h}h 后增加一个 pad word ，让非实体起始单词指向它表示 inactive。于是： h=[h,0]∈R(L+1)∗2H\boldsymbol{h}=[\boldsymbol{h},0]\in\mathbb{R}^{(L+1)*2H}h=[h,0]∈R(L+1)∗2H 。

然后，对于单词 WjW_jWj​ ，使用attention机制得到其 实体结束单词。其实就是 拼接+非线性+softmax打分：

uij=vTtanh⁡(G1hi+G2dj), for i∈(j,…,J)p(yj∣xj)=softmax⁡(uj)\begin{gathered} u_{i}^{j}=\boldsymbol{v}^{T} \tanh \left(\boldsymbol{G}_{1} \boldsymbol{h}_{i}+\boldsymbol{G}_{2} \boldsymbol{d}_{j}\right), \text { for } i \in(j, \ldots, J) \\ p\left(y_{j} \mid \boldsymbol{x}_{j}\right)=\operatorname{softmax}\left(\boldsymbol{u}^{j}\right) \end{gathered} uij​=vTtanh(G1​hi​+G2​dj​), for i∈(j,…,J)p(yj​∣xj​)=softmax(uj)​

Domain Discriminator

输入： h=[h,0]∈R(L+1)∗2H\boldsymbol{h}=[\boldsymbol{h},0]\in\mathbb{R}^{(L+1)*2H}h=[h,0]∈R(L+1)∗2H
使用 MLP 做一个二分类，得到输入句子的 domain label，即判断具体来自哪一个 domain。

ω=softmax⁡(tanh⁡(h⋅P+p))c=hωp(yd∣c)=MLP⁡(c)\begin{aligned} \boldsymbol{\omega} &=\operatorname{softmax}(\tanh (\boldsymbol{h} \cdot P+p)) \\ \boldsymbol{c} &=\boldsymbol{h} \boldsymbol{\omega} \\ p\left(y_{d} \mid \boldsymbol{c}\right) &=\operatorname{MLP}(\boldsymbol{c}) \end{aligned} ωcp(yd​∣c)​=softmax(tanh(h⋅P+p))=hω=MLP(c)​

Adversarial Training

Tag prediction loss 和 Domain prediction loss 分别为：

L(θf,θs)=∑DS∑j=1J−log⁡p(yj∣xj;θf,θs)L(θf,θt)=∑DS,DT−log⁡p(yd∣c;θf,θt)\begin{aligned} \mathcal{L}\left(\boldsymbol{\theta}_{f}, \boldsymbol{\theta}_{s}\right) &=\sum_{\mathcal{D}_{S}} \sum_{j=1}^{J}-\log p\left(y_{j} \mid \boldsymbol{x}_{j} ; \boldsymbol{\theta}_{f}, \boldsymbol{\theta}_{s}\right) \\ \mathcal{L}\left(\boldsymbol{\theta}_{f}, \boldsymbol{\theta}_{t}\right) &=\sum_{\mathcal{D}_{S}, \mathcal{D}_{T}}-\log p\left(y_{d} \mid \boldsymbol{c} ; \boldsymbol{\theta}_{f}, \boldsymbol{\theta}_{t}\right) \end{aligned} L(θf​,θs​)L(θf​,θt​)​=DS​∑​j=1∑J​−logp(yj​∣xj​;θf​,θs​)=DS​,DT​∑​−logp(yd​∣c;θf​,θt​)​

θf\boldsymbol{\theta}_{f}θf​ 为共享层（Input Representation+Context Encoder）的参数。
θs\boldsymbol{\theta}_{s}θs​ 为 tag decoder 的参数。
θt\boldsymbol{\theta}_{t}θt​ 为 Domain Discriminator 的参数。

我们需要最大化 Ldomain\mathcal{L}_{domain}Ldomain​ ，最小化 Ltag\mathcal{L}_{tag}Ltag​ ，于是目标函数变为：

E(θf,θs,θt)=L(θf,θs)−λL(θf,θt)E\left(\boldsymbol{\theta}_{f}, \boldsymbol{\theta}_{s}, \boldsymbol{\theta}_{t}\right)=\mathcal{L}\left(\boldsymbol{\theta}_{f}, \boldsymbol{\theta}_{s}\right)-\lambda \mathcal{L}\left(\boldsymbol{\theta}_{f}, \boldsymbol{\theta}_{t}\right) E(θf​,θs​,θt​)=L(θf​,θs​)−λL(θf​,θt​)

训练目标为：

(θ^f,θ^s)=arg⁡min⁡θf,θsE(θf,θs,θ^t)θ^t=arg⁡max⁡θtE(θf^,θ^s,θt)\begin{aligned} \left(\hat{\boldsymbol{\theta}}_{f}, \hat{\boldsymbol{\theta}}_{s}\right) &=\arg \min _{\boldsymbol{\theta}_{f}, \boldsymbol{\theta}_{s}} E\left(\boldsymbol{\theta}_{f}, \boldsymbol{\theta}_{s}, \hat{\boldsymbol{\theta}}_{t}\right) \\ \hat{\boldsymbol{\theta}}_{t} &=\arg \max _{\boldsymbol{\theta}_{t}} E\left(\hat{\boldsymbol{\theta}_{f}}, \hat{\boldsymbol{\theta}}_{s}, \boldsymbol{\theta}_{t}\right) \end{aligned} (θ^f​,θ^s​)θ^t​​=argθf​,θs​min​E(θf​,θs​,θ^t​)=argθt​max​E(θf​^​,θ^s​,θt​)​

Experiments

Experimental Setup

使用五个数据集，全都忽略实体类型判别。
使用六个 baseline model，其中 INIT, TranT-B, CDMA-NER 需要有标注的 target-domain 数据完成模型迁移。所以作者随机将 20% 的训练集语料整合到开发集语料中作为 target-domain 数据。

Model setup

  • All neural network models use GloVe 300-dimensional pre-trained word embeddings
  • The dimension of character-level representation is 100
  • The CNN sliding windows of filters are [2, 3, 4, 5]
  • The total number of CNN filters is 100
  • Each bidirectional encoder GRU has a depth of 3 and hidden size of 128.
  • Each decoder GRU has a depth of 3 and hidden size of 256.
  • Adam optimizer was adopted with a learning rate of 0.001, selected from {0.01, 0.001, 0.0001}.
  • a dropout of 0.5 after the convolution or recurrent layers
  • Decay rate is 0.09
  • The gradient clip is 5.0.
  • For neural baselines, we use exactly the same hyper-parameter grid and training procedure as our proposed model above.
  • Implemented with PyTorch framework and evaluated on NVIDIA Tesla V100
    GPU.

Result

In-domain Entity Boundary Detection Performance

数据集：OntoNotes5.0

  • AT-BDRY 超越所有其他模型，作者认为原因是使用了指针网络，使得在数据稀疏的情况下，有效捕捉输入句子的依赖关系。
  • StanfordNER 效果不好，作者给出原因：off-the-shelf NER systems 不是专门针对实体边界判别问题。同时得出结论：off-the-shelf NER systems 不适合 cross-domain datasets 场景。

Formal Text → Formal Text

对比从 formal text data(e.g., CoNLL03) 迁移到另一个 formal text (e.g., Wikigold) 的效果，同时设置 in-Domain 作为在目标数据集训练的模型的效果。

  1. AT-BDRY 效果最好
  2. 本文提出的 adversarial transfer mechanism 是有效的
  3. 在 CoNLL03 → Wikigold 里，和 in-Domain 很接近，但是 AT-BDRY 是没有使用目标语料的标注数据的。

Formal Text → Informal Text

Informal Text 最典型的就是 Twitter
很多ner迁移模型在目标为 Informal Text 的情况下表现并不好。
因此，作者分析了当 data distribution gap 增大时，AT-BDRY 的效果。

  1. 多数情况下，Recall 明显大于 Precision。
    作者提出原因为 非正式文本不规范，有很多噪音（noisy），并且有很多新实体（novel surface）。
  2. OntoNote5.0 → WNUT17 的效果很不错，作者认为是因为 AT-BDRY 可以有效学习 domain-invariant representations ，有效减小 distance in data distributions / data distribution gap。
