W2NER

论文：Unified Named Entity Recognition as Word-Word Relation Classification
代码：ljynlp/W2NER

Abstract and Introduction

NER 有不同的类型，包括 flat，overlapped (aka.nested)，discontinuous NER 。
当前的工作主要使用四种框架解决不同的 ner 问题：

  1. sequence labeling methods
  2. hypergraph-based methods
  3. sequence-to-sequence methods
  4. span-based methods

本文中作者推出一个统一的框架来解决这些不同的类型，命名为 W2NER\text{W}^2\text{NER}W2NER。

主要思想为将 NER 视为单词的关系分类问题 (word-word relationship classification)，用于区分单词的两个类型：

  1. the Next-Neighboring-Word (NNW)：判断两个token是否是相临 (adjacent) 的（有点crf的意思）。同时，天然地可以区分非实体类型。
  2. the Tail-Head-Word-* (THW-*)：判断实体边界和实体分类 (entity boundary and type detection)。

NER as Word-Word Relation Classification

输入句子： X={x1,x2,...,xN}X=\{x_1,x_2,...,x_N\}X={x1​,x2​,...,xN​}
模型需要抽取出各个实体对 (xi,xj)(x_i,x_j)(xi​,xj​) 之间的关系 R={None,NNW,THW-*}\mathcal{R}=\{\text{None},\text{NNW},\text{THW-*}\}R={None,NNW,THW-*} ：

  1. None：没有关系
  2. Next-Neighboring-Word：两个单词同属于一个实体，并且「列单词」是「行单词」的下一个单词。
  3. Tail-Head-Word-*：「行单词」为实体结尾字符，「列单词」为实体起始字符，“*” 表示实体类型。

Model

  1. BERT + LSTM for Word Representation
  2. A Multi-Granularity 2D Convolutions for Word-Pair Representations
  3. A Co-Predictor for Word-Word Relations Reasoning and Entity Type

Encoder Layer

  1. 输入单词放入 Bert ，得到 sub-word representation 。
  2. 使用 max pooling 得到 word representations 。
  3. 送入 LSTM 得到最终 word representations H={h1,h2,...,hN}∈RN×dhH=\{h_1,h_2,...,h_N\}\in \mathbb{R}^{N\times d_h}H={h1​,h2​,...,hN​}∈RN×dh​

Convolution Layer

包含三个子模块：

  1. a condition layer with normalization：generate word-pair grip representation
  2. a BERT-style grid representation build-up：enrich the representation of the word-pair grid
  3. a multi-granularity dilated convolution 多粒度膨胀卷积：capturing the interactions between close and distant words

BERT-Style Grid Representation Build-Up

BERT输入包含三部分： token embeddings + position embeddings + segment embeddings ，作者使用类似的方法构建 单词对表征矩阵：

  1. V∈RN×N×dh\mathbf{V} \in \mathbb{R}^{N \times N \times d_{h}}V∈RN×N×dh​ 表示单词对表征信息
  2. Ed∈RN×N×dEd\mathbf{E^d} \in \mathbb{R}^{N \times N \times d_{E^d}}Ed∈RN×N×dEd​ 表示单词对的位置信息（距离信息）
  3. Et∈RN×N×dEt\mathbf{E^t} \in \mathbb{R}^{N \times N \times d_{E^t}}Et∈RN×N×dEt​ 用于区分上下三角矩阵信息（方向信息）

然后拼接三个向量，然后放入 MLP 中降维，得到 position-region-aware representation C∈RN×N×dc\mathbf{C} \in \mathbb{R}^{N \times N \times d_{c}}C∈RN×N×dc​ ：

C=MLP⁡1([V;Ed;Et])\mathbf{C}=\operatorname{MLP}_{1}\left(\left[\mathbf{V} ; \mathbf{E}^{d} ; \mathbf{E}^{t}\right]\right) C=MLP1​([V;Ed;Et])

Multi-Granularity Dilated Convolution

受到 TextCNN 启发，作者使用多重二维相关卷积 (multiple 2-dimensional dilated convolutions / DConv) ，并且使用不同的膨胀率 lll （e.g.， l∈[1,2,3]l\in[1,2,3]l∈[1,2,3] ），来得到不同距离下单词之间的交互信息：

Ql=σ(DConvl(C))\mathbf{Q^l} = \sigma(\text{DConv}_l(C)) Ql=σ(DConvl​(C))

Ql∈RN×N×dc\mathbf{Q^l}\in \mathbb{R}^{N\times N\times d_c}Ql∈RN×N×dc​ ， σ\sigmaσ 为GLUE激活函数。
于是拼接不同的 Ql\mathbf{Q^l}Ql 得到最终的 单词对表征向量：

[Q1,Q2,Q3]∈RN×N×3dc\left[\mathbf{Q^1},\mathbf{Q^2},\mathbf{Q^3} \right] \in \mathbb{R}^{N\times N\times 3d_c} [Q1,Q2,Q3]∈RN×N×3dc​

Co-Predictor Layer

使用 MLP 和 Biaffine 结果的加和得到最终的分数，然后接 softmax 得到概率。

Decoding

我们已经得到了单词和单词间的关系，是一个有向图模型。
解码的目的是要找到「起始单词」到「结束单词」的特定路线，每条路对应一个实体。
论文中列举了四种情况，都可以通过两个关系找出正确的实体。

Results




Model Ablation Studies

  1. 不使用「距离编码向量」和「上下三角位置向量」，效果下降。
  2. 不使用膨胀卷积操作，效果下降，其中膨胀率为 2 的 DCNN 最关键。
  3. Biaffine 可以带来提升。
  4. 不使用 NNW 关系，即只预测 头尾实体，效果下降。同时不符合离散实体任务（CADEC）。

思考

个人觉得这一篇文章最有亮点的地方就是使用两个关系来统一命名实体问题。
至于模型，亮点在于使用了三个矩阵来表示三种信息（挺符合 TENER 中相对位置编码的出发点），其余的更像缝合怪，就是在打比赛时将所有能涨点的 module 全用上一样。
随便写点感想，没有喷的意思，还是比我这菜狗牛了不知道多少倍。

W2NER 的模型架构中，别的不说，的确最后的 W2W 矩阵也是有一些道理的，计算 token 之间的关系，用于最后的解码，从结构上看，的确似乎可以解决这个问题，当然，实际能否收集到那么多的产业数据，以及最终效率上的表现则是另一个故事了。
那篇工作也有自己的槽点，比如看论文里面消融分析，你会发现，那个复杂出天际的模型结构里面，有的模块凑数的味道非常明显。
GitHub 中 issue 也有人指出，某些地方的随机过程没有锁死，意味着该结果无法复现，作者也承认了这一事实。
