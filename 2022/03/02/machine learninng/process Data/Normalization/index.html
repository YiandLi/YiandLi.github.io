Normalization

Posted on 2022-03-02 Edited on 2022-05-22

协变量转移 / ICS

  • 定义：网络层的输出经过线性变换作为下层网络的输入，网络输出直接影响下层网络输入分布。
  • 解决：固定网络层的输入分布（固定输入的均值和方差）。

BN 宣称能缓解ICS（internal covariance shift）。不过目前这个观点也是饱受质疑，仅仅靠减均值除以方差的操作，是不足以来维持input和output的分布一致的。
准确的讲，BN缓解的是Variance Shift，主要是加性操作产生的（CNN，Attention 都有大量加性操作）。
由于Variance Shift，对深层网络而言，归一化几乎是必选项。哪一个维度进行大量加性操作，哪一个维度就需要标准化，比如instance norm对CNN而言，以及layer norm对transformer而言，是在归一化时在特征维度上做选择的最小集。\

只要能在某个维度做独立同分布假设，那么把归一化拓展到这个维度就不会带来信息损失，那就可以进行拓展。

因为：归一化操作一般会带来信息损失，但损失信息不一定就是坏的，损失掉一些无用的信息和自由度，对网络的优化和收敛是有帮助的。也可以理解成增加了额外的先验，帮助网络更好的学习，那么归一化增加的先验就是最开始的观点“特征沿着这个维度做切分，得到的随机变量是独立同分布的。

对transformer而言，同样的，拓展到batch之间是完全OK的，也就是说，在transformer中使用BN是完全OK的。之所以效果没有layernorm好，那更多的是工程上的原因，例如句子的长短造成的影响，BS的大小。
那能否拓展到token这个维度呢？答案是否定的，一个句子中不同的token之间相关性极强，彼此独立的假设显然不成立。
总结下，transformer中的embeddings（B，N， C），可以在（B，C）这两个维度做归一化，其中（C）这个维度是必须有的。

为什么做 Normalization

归一化尽可能的让输入的数据/特征变得独立同分布。这是一个很好的性质，可以让后续模型拟合的难度降低。

数学公式

y=γ(x−μ(x)σ(x))+βy=\gamma\left(\frac{x-\mu(x)}{\sigma(x)}\right)+\beta y=γ(σ(x)x−μ(x)​)+β

Batch Normalization

论文：Batch Normalization: Accelerating Deep Network Training by Reducing Internal Covariate Shift

纵向规范化：每一个特征都有自己的分布
使得每一层神经网络的输入保持相同分布的，在输入到激活函数之前，对每个mini-batch输入，做如下处理：
针对的是一个 batch 中的所有样本数据，按照样本数计算归一化统计量的。
强行将同一个 batch 中的数据拉回到均值为0，方差为1的正太分布上。
推理时存储的是训练时，同一位置的 batch 的设置：均值，方差，线性权重。

  • 过程：

    最后一层引入了可训练参数的 β\betaβ 和 γ\gammaγ 。

  • 优点：

    • 不断调整输入分布，让数据分布一致。
    • 解决梯度爆炸/消失问题：使得 sigmoid 输出在 0 附近。
    • BN 后，可以适当加大 lr，否则需要尽量减少 lr。

  • 缺陷：

    • 对 batch size 的大小比较敏感。由于每次计算均值和方差是在一个 batch 上，所以如果 batch size 太小，则计算的均值、方差不足以代表整个数据分布；
    • BN 实际使用时需要计算并且保存某一层神经网络 batch 的均值和方差等统计信息，对于对一个固定深度的前向神经网络（DNN，CNN）使用BN，很方便；但对于 RNN 来说，sequence 的长度是不一致的，换句话说 RNN 的深度不是固定的，不同的 time-step 需要保存不同的 statics 特征，可能存在一个特殊 sequence 比其他 sequence 长很多，这样 training 时，统计信息不能反映全局分布。
    • 另外如果在测试时我们遇到了长度大于任何一个训练样本的测试样本，我们无法找到保存的归一化统计量，所以BN无法运行。

  • 整个流程：

    • inference 的过程为第7-11行，第十行表示均值和方差是训练时决定的，同时两个线性变换的参数也是训练时决定的。
    • 这里固定的均值和方差，通过使用对应位置的（某个激活函数之前）所有mini-batch的均值的期望作为均值，所有mini-batch的方差的无偏估计作为方差（第10行）

  • torch内部有两种BatchNorm，分别是BatchNorm1D和BatchNorm2D，BatchNorm1D作用于2维张量（BD，存储D个统计量），BatchNorm2D作用于4维张量（BCWH，存储C个统计量）
    NLP任务中的张量多为3维，BND，需存储序列长度上的统计量，而不同batch的序列长度不一致，不好统计，而且推理阶段的序列长度不能大于训练时所用的最大长度。

Layer Normalization

横向标准化：每一个时间步都有自己的分布，从而可以处理单一样本、变长序列，而且 训练和测试处理方式一致。
针对深度网络的某一层的 所有神经元 的输出进行 normalize 操作，同层网络的所有隐藏单元共享均值和方差。
在 RNN 中，每个时刻加权后的输入通过标准化被重新调整在合适的范围，很大程度避免了梯度消失、梯度爆炸问题，隐藏状态的传递更加稳定。
