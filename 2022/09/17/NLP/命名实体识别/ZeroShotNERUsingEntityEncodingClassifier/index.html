Zero-Shot NER Using Entity-Encoding Classifier

Posted on 2022-09-17 In NLP , Named entity recognition
Symbols count in article: 331 Reading time ≈ 2 mins.

Paper: Zero-Shot Learning in Named-Entity Recognition with External Knowledge

当前 NER 模型的弊端：缺乏泛化性，无法迁移，即在 unseen dataset 上表现很差。

作者提出 ZERO 模型，应用于 zero-shot 和 few-shot 场景下。
该模型 融合了已经存在的外部知识，通常为 target domain/dataset 的无监督知识；将目标 entity 进行编码，当作 label anchor embedding 使用。


具体步骤为：

  1. 使用 LUKE (Language Understanding with Knowledge-based Embeddings) 作为 sentence 的编码器，编码所有的 token 和 entity span，得到：

hi=ϕ(xi), where i∈1,…,K\mathbf{h}_i=\phi\left(\mathbf{x}_i\right), \text { where } i \in 1, \ldots, K hi​=ϕ(xi​), where i∈1,…,K

LUKE 是一个 transformer 架构的，在 entity-annotated corpora from Wikipedia 上训练的模型。训练时将 words 和 entity 全部看作 independent token。

  1. 从通用常识型知识图谱 ConceptNet 中拿到 label entity 的相关知识 text。使用 ConceptNet Numberbatch 进行编码（ an ensemble of the word embeddings Word2Vec, GloVe, and FastText ），得到 label entity feature ej\mathbf{e}_jej​ 。

  2. 将所有的 hi\mathbf{h}_ihi​ 放入一个线性层 FCN 中，进行降维，和 ConceptNet Numberbatch 的编码对齐。

  3. 使用降维后的 hi\mathbf{h}_ihi​ 和每个 ej\mathbf{e}_jej​ 进行点乘，得到最终的预测标签：

yi=argmax⁡j(softmax⁡(hi⋅ej))y_i=\underset{j}{\operatorname{argmax}}\left(\operatorname{softmax}\left(\mathbf{h}_i \cdot \mathbf{e}_j\right)\right) yi​=jargmax​(softmax(hi​⋅ej​))

这其实就是把 label entity feature ej\mathbf{e}_jej​ 看作 label anchor，将其组合成一个 classifier。
