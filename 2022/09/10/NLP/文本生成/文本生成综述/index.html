Pretrained Language Models for Text Generation: A Survey

Text generation is also be referred as natural language generation (NLG).
Base model differs, but share the same point: enable end-to-end learning of semantic mappings from input to output in text generation.

Standard Architectures

When pre-training, a large amount of data could enhance the understanding of language and improve the generation quality.
In this way, PLMs can successfully perform on new tasks with their old experience and knowledge.

Model architecture:

  • Encoder-Decoder Transformer: Transformer, MASS [Song et al., 2019], T5 [Raffel et al., 2020], and BART [Lewis et al., 2020] …
  • Decoder-only Transformer: GPT [Radford et al., 2019; Brown et al., 2020] and CTRL [Keskar et al., 2019] …
    These models do not have an independent module to encode input sequence. So
    • a special token (e.g., “[SEP]”) is concatenated between the input sequence and output sequence.
    • a novel seq2seq masking is employed to
      • let the input sentence attend to each other
      • let generated tokens attend to all input tokens and previous generate ones.

The research of Raffel et al. [2020] shows that the addition of an explicit encoder-decoder attention is beneficial.

Input and Output techniques

The core of text generation tasks is to learn the semantic mappings from input to output.

On one hand, different tasks will correspond to a variety of input data, and we need to develop special techniques to model different data types.
On the other hand, the generated text should satisfy important properties in order to cope with different task requirements.

Modeling Different Data Types from Input

Unstructured Input, Structured Input, and Multimedia Input

Unstructured Input

text input (e.g., sentence, paragraph, and document).

  • Shot Sentence: Liu and Lapata [2019] and Zheng and Lapata [2019] employed PLMs (e.g., BERT) as text encoder.
  • Long Sentence: Zhang et al. [2019b] and Xu et al. [2020b] proposed hierarchical BERT to learn interactions between sentences with self-attention for document encoding.
  • To Capture Inter-Sentential Relations: DiscoBERT [Xu et al., 2020a] stacked graph convolutional network (GCN) on top of BERT.
    By directly operating on the discourse units, DiscoBERT retains capacities to include more concepts or contexts, leading to more concise and informative output text.
  • Multilingual Generation Tasks (e.g., machine translation): Conneau and Lample [2019] propose cross-lingual language models (XLMs)

Structured Input

e.g., graph and table

It is difficult to collect a large amount of labelled structured data with ground-truth text for training.
Since PLMs show strong few-shot ability, Chen et al. [2020b] and Gong et al. [2020] explored incorporating PLMs for data-to-text generation, especially in few-shot settings.

  1. When applying PLMs to structured data, a major challenge is how to feed structured data into PLMs, which are originally designed for sequential text.
  • Ribeiro et al. [2020] and Mager et al. [2020] linearized input knowledge graph (KG) and abstract meaning representation (AMR) graph into a sequence of triples,
  • Li et al. [2021b] introduced an additional graph encoder to encode the input KG,
  • Gong et al. [2020] employed a template-based method to serialize input table into text sequence.

  1. However, direct linearization will lose the structural information of original data, Gong et al. [2020] proposed an auxiliary reconstruction task for recovering the structural information of input data.

  2. The output text should retain as much as important information from structured data.
    Therefore, to generate high-fidelity text adhereing to input, the pointer generator mechanism is adopted to copy words from input knowledge data [Chen et al., 2020b].
    Besides, Gong et al. [2020] proposed a content matching loss for measuring the distance between the information in input data and the output tex.

Satisfying Special Properties for Output Text

In different text generation tasks, the generated text should satisfy several key properties, i.e., relevance, faithfulness, and order-preservation …

Relevance

Relevance refers that the topics in output text is highly related to the input text.

A representative example is dialogue systems:

  • the generated response to be relevant to the input dialogue history.
  • condition of the type of response might be also provided as an external input such as the topic of response and the persona of speaker.

Due to the absence of long-term memory, RNN-based models still tend to generate irrelevant output text and lack consistency with input.

Recommendation:

  • TransferTransfo [Wolf et al., 2019]
  • DialoGPT [Zhang et al., 2020]
  • Zeng and Nie [2020] utilized elaborated condition blocks to incorporate external conditions.
    They used BERT for both encoder and decoder by utilizing different input representations and self-attention masks to distinguish the source and target sides of dialogue.
    On the target (generation) side, a new attention routing mechanism is adopted to generate context-related words.
    Similar approaches have been used in non-conditioned dialogue [Bao et al., 2020].

Faithfulness

Faithfulness means the content in generated text should not contradict the facts in input text.
Sometimes, it further means the generated text is in accord with the world facts.

A representative example is text summarization.

Pre-trained on large collections of text, PLMs are potentially beneficial to generate faithful text by utilizing background knowledge.

Recommendation:

  • To improve summary faithfulness, Kryscinski et al. [2018] proposed to decompose the decoder into a contextual network that retrieves relevant parts of the source document and a PLM that incorporates prior knowledge about language generation.
  • To generate faithful text in different target domains, Yang et al. [2020b] fine-tuned PLMs on target domains through theme modeling loss.
    The role of the theme modeling module is to make the generated summary semantically close to the original article.

Order-preservation

order-preservation denotes that the order of semantic units (word, phrase, etc.) in both input and output text is consistent.

The most representative example is machine translation.
When translating from source language to target language, keeping the order of phrases consistent in source language and target language will ensure the accuracy of the translation results to some extent.

Recommendation:

  • One line of research to achieve the order-preservation property is to perform semantic alignment in machine translation.
    Yang et al. [2020a] proposed Code-Switching Pre-training (CSP) for machine translation. They use the word-pair alignment information of the source and target language to enhance order preserving.
  • For multilingual machine translation, Lin et al. [2020] proposed mRASP, an approach to pretraining a universal multilingual machine translation model.
    Wada and Iwata [2018] focused on aligning word representations of each language to preserve the word order consistent cross multiple languages.

Fine-tuning Strategies for Text Generation

Data View

Key: how to design suitable and effective fine-tuning strategies adapting to the characteristics of new domain

Few-shot Learning

E.g., in multilingual translation, some low-resource languages lack sufficient parallel corpus.
[Conneau and Lample, 2019] leverage the knowledge learned in high-resource languages to low-resource languages.

…

Domain Transfer

PLMs have powerful generalization capability.
However, they still cannot directly adapt to new domains with large distribution discrepancy from pre-training domain.

An effective solution is to continue training PLMs on specific data with pre-training objectives before fine-tuning on target tasks.
For Mask Prediction task, there exist several variants of masking ways in domain transfer:

  • [Zeng and Nie, 2020]proposed TF-IDF based masking to select more condition-related tokens to mask, in order
    to focus on domain features
  • Document masking is usually utilized in summarization task to capture document-level features of long documents [Zhang et al., 2019b].

Task View

E.g., language coherence and text fidelity in specific generation tasks when fine-tuning PLMs.

Enhancing Coherence

Models fine-tuned by contrastive learning are good at distinguishing whether a sentence pair is similar or not.
PLMs are forced to understand the positional or semantic relationship between two sentences, so that they can derive better representations.

solutions:

  • Next sentence prediction (NSP)
  • Denoising autoencoding (DAE) takes the corrupted text as input and aims to recover the original text.
    The model fine tuned with DAE has a strong ability to understand the overall sentences and capture longer-range correlations.
    • TED [Yang et al., 2020b] utilized DAE to refine essential semantic information for abstract summarization.

Preserving Fidelity

Text fidelity refers that how the generated text adheres to the original input information

Future Outlooks

  • Model Extension:
    • for model architecture: how to figure out the gap between downstream generation tasks and pre-training
    • for data: how to incorporating external knowledge into PLMs during pre-training
  • Controllable Generation: to control some attributes of the generated text
    • e.g., generating positive response to patients with depression in dialogue systems
  • Model Compression: how to deploy model in resource constrained environments
  • Language-agnostic PLMs: to capture universal linguistic and semantic features across different languages
    • An interesting direction is how to reuse existing English-based PLMs for text generation in non-English languages.
  • Ethical Concern
