CONTaiNER for few-shot NER via Contrastive Learning on Gaussian Embeddings

Posted on 2022-09-27 In NLP , Named entity recognition
Symbols count in article: 1.9k Reading time ≈ 9 mins.

Paper: CONTaiNER: Few-Shot Named Entity Recognition via Contrastive Learning
Code: psunlpgroup/CONTaiNER


Traditional Few-shot NER face 3 main challenges:

  • Difficulties/Requirements of task:
    NER tasks, the sequence labeling tasks, need to consider concordant context and the dependencies within the labels.
  • Low generalization:
    No-entity tokens are labeled with Outside (O), but a token labeled as O in training set may correspond to a valid target entity in test set.
    Therefore, the trained weights will be closely tied to the source classes and the model will save the biased knowledge of training set O-tokens.
  • Difficulty of few-shot setting:
    There are not enough samples for a validation set, which reduces the capability of hyperparameter tuning and template-based method.
    It is also a dismissed point for earlier few-shot works.

The author present CONTAINER, a novel contrastive learning technique that optimizes the inter-token distribution distance for Few-Shot NER.

  • This model tries to decrease the distance of token embeddings of similar entities while increasing it for dissimilar ones.
  • This model is trained with a generalized objective, it can effectively avoid the pitfalls of O-tokens.
  • This model does not require any dataset specific prompt or hyperparameter tuning.

Specifically,

  • Unlike traditional contrastive learners, this model optimizes a generalized objective of differentiating between token categories based on their Gaussian-distributed embeddings.
  • Gaussian Embedding faces an additional constraint of maintaining class distribution through the variance estimation.
  • Thus Gaussian Embedding explicitly models entity class distributions
    • which not only promotes generalized feature representation,
    • but also helps in few-sample target domain adaptation.

Previous works in Gaussian Embedding have shown that

  1. mapping to a density captures representation uncertainties (Vilnis and McCallum, 2014)
  2. and expresses natural asymmetries (Qian et al., 2021)
  3. while showing better generalization

requiring less data to achieve optimal performance.

Fine-tuning to Target Domain using Support Set

Fine-tune the model using a small number of target domain support samples. The loss function is still:

d(p,q)=12(DKL[Nq∥Np]+DKL[Np∥Nq])d(p, q)=\frac{1}{2}\left(D_{\mathrm{KL}}\left[\mathcal{N}_{q} \| \mathcal{N}_{p}\right]+D_{\mathrm{KL}}\left[\mathcal{N}_{p} \| \mathcal{N}_{q}\right]\right) d(p,q)=21​(DKL​[Nq​∥Np​]+DKL​[Np​∥Nq​])

For the small number of support set, all the support samples are combined into a single batch(one batch per one epoch).

Advanced loss function

When with enough few-shot samples (e.g. k=5), model could adapt to the new domain after fine tuning.
In contrast, with few few-shot samples (e.g. k=1), model fail to adapt to the new domain.

Because, when the model has no prior knowledge(from direct training or indirectly from source domain training where the target class entities are marked as O-type\text{O-type}O-type), a single example might not be sufficient to deduce the variance of the target class distribution.
Thus, for 1-shot scenario when training set is shows no prior knowledge, we optimize Euclidean Distance: d′(p,q)=∥μp−μq∥22d^{\prime}(p, q)=\left\|\boldsymbol{\mu}_{p}-\boldsymbol{\mu}_{q}\right\|_{2}^{2}d′(p,q)=∥∥∥​μp​−μq​∥∥∥​22​ .

Early Stop

In lack of supporting data, traditional method to get the best parameter / training steps is not suitable.
Hence, we rely on the calculated contrastive loss and use it as our early stopping criteria with a patience of 1.

Algorithm

Instance Level Nearest Neighbor Inference

Similar to SimCLR (Chen et al., 2020), we found that representations before the projection layers actually contain more information than the final output representation, and thus contributes to better performance.

So, when doing inference, we drop the two projection heads and only use the pre-trained language model encoder PLM.

To assign each token a tag, we calculate the representations of the test data from PLM and find nearest neighbor support set representation:

  1. For each of the support token (xjsup ,yjsup )∈Xsup \left(x_{j}^{\text {sup }}, y_{j}^{\text {sup }}\right) \in \mathcal{X}_{\text {sup }}(xjsup ​,yjsup ​)∈Xsup ​ , we get each’s PLM representations hjsup\boldsymbol{h}_{j}^{\text {sup}}hjsup​.
  2. For each of the test token xitest ∈Xtest x_{i}^{\text {test }} \in \mathcal{X}_{\text {test }}xitest ​∈Xtest ​, we get each’s PLM representations hitest \boldsymbol{h}_{i}^{\text {test }}hitest ​.
  3. Then, assign each xitestx_{i}^{\text {test}}xitest​ the same label as the support token that is nearest in the PLM representation space:

Viterbi Decoding

Previous works noticed a performance improvement by using CRFs, so here we also employ Viterbi decoding in the inference stage.

  • The emission probabilities are calculated from Nearest Neighbor Inference stage.
  • For the transition probabilities, the transition between three abstract tags O\text OO, I\text II, and I-other\text{I-other}I-other is estimated by counting their occurrences in the training set.
    Then for the target domain tag-set, these transition probabilities are evenly distributed into corresponding target distributions.

Interestingly, from the experiment results, if there is no significant domain shift involved in the test data, contrastive learning allows CONTAINER to automatically extract label dependencies, obviating the requirement of extra Viterbi decoding stage.

Experiments

Tag-set Extension Setting

Experiment tag-set extension capability using OntoNotes dataset.
The 18 existing entity classes are split in three groups: A, B, and C, each having 6 classes.

Model is firstly trained on two data sets, then it is test on the left data set with a small support set.
During training, all test group entities are replaced with O-tag.


CONTaiNER outperforms present SOTAs by upto 12.75 absolute F1 points

Domain Transfer Setting

we take OntoNotes (General) as source text domain, and evaluate the Few-Shot performance in I2B2 (Medical), CoNLL (News), WNUT (Social) domains.


All the other domains have almost no intersection with OntoNotes.
Target entities in CoNLL are fully contained within OntoNotes entities, that makes it comparable to supervised learning.

Compare Training Objective

Traditional contrastive learners usually optimize cosine similarity of point embeddings.
We compare the performance of point embeddings with euclidean distance and cosine similarity to that of CONTAINER using Gaussian Embedding and KLdivergence in OntoNotes tag-set extension.

And for a better analysis, we use t-SNE visualization of support set and test set of a sample few-shot scenario.


According to the picture,

  1. Figure 3 (c): Euclidean distance tries to force the same class support examples to collapse into essentially a single point representation, leading to over-fitting on the small supporting set.
  2. Figure 3 (b): Prior to fine-tuning, in both support and test sets, we observe different class samples mostly clustered together. This indicates that even before fine-tuning it shows good generalization to unseen classes.
  3. Figure 3 (d): After fine-tuning, the KL-divergence optimization objective maintains the class distribution; and the separation boundary of clusters
