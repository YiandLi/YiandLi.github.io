Prompt-based Metric Learning Few-Shot NER

Posted on 2022-12-29 In NLP , Named entity recognition
Symbols count in article: 1.2k Reading time ≈ 6 mins.

Paper: PROMPT-BASED METRIC LEARNING FOR FEW-SHOT NER

针对 Few-Shot NER，当前 metric learning 的方法都

  1. 只针对 token-level similarity
  2. 无法利用 标签信息，即 agnostic to the labels in the support set。
    或者仅仅利用标签信息得到更好的 label representations，而不是得到更好的 text representations。

本文中，作者提出 ProML - Prompt-based Metric Learning：

  1. 设计不同的 prompt 模版来利用标签信息
  2. 提出新的模型架构来组合不同的 prompt-based text representation。
    即我们可以得到带有标签信息的 text/token/span representation，这其实是有利于生成更健壮的 prototype representation，有利于 metric learning 中的 similarity scores。

PROMPT SCHEMAS

作者提出 mask-reducible prompts，用于 token-level similarity metric learning。
主要包含两种 Prompt 类型：the option-prefix prompt and the label-aware prompt。

MASK-REDUCIBLE PROMPTS

这样模型可以知道我们需要预测哪些实体标签，然后得到 label-dependent representations。

PROMPT B: LABEL-AWARE PROMPTS

仅仅适用于有标签信息的情况，即 support 集合。

当 token label sequence y\mathbf{y}y 已知时，可以直接将 entity label E\mathbf EE 拼接在 entity e\mathbf ee 出现的位置后面： [e∣E][\mathbf e|\mathbf E][e∣E] 。
我们使用 x′=fB(x,y)\mathbf{x}^{\prime}=f_{B}(\mathbf{x}, \mathbf{y})x′=fB​(x,y) 来表示 这种 prompt 方法。

NEAREST NEIGHBOR INFERENCE

推理时，和 CONTaiNER 一样，就不再得到 token 的高斯分布了，直接使用 token representation，使用 Euclidean distance 找到距离每个 query token 最近的 support token，然后赋予其对应的 tag。

可以看到这里还是 token-level，或者说是 token-tag level，没有单独考虑 O-tag 的情况，是混在一起的。
