贪心搜索

是 beam search 中 beam size=1 的版本。
首先是最传统的贪心搜索算法，完全不考虑上一步，每一步直接生成概率最大的那一个token。
运行效率很高，但是每一步得到的都是局部最优解，基本不能得到全局最优解。

Beam search

Beam Search 对贪心搜索进行了改进，扩大了搜索空间，更容易得到全局最优解。
参数 beam size=k，即维护当前分数最高的k条路径，下一时刻继续使用这k个序列得到下一个token。
当k=3，词典大小为v时：t时刻得到了3个分数最高的序列，在t+1时刻每个序列都会生成v个候选，所以总候选序列为3v个，然后从继续找到3个分数最高的序列，作为下个时间步骤的前缀。
所以算法复杂度为 O(beam_size * vocab_size * max_seq_len)。

  • 可以看到B这个数组存储了 top-k 个前缀序列。
  • 每个序列存储形式为 <s,y>，s为分数，y为前缀序列。
  • 每次迭代时，会使用x（transformer中的input）和y（前缀序列）得到当前时间步的每个候选token的分数，使用score函数来得到 s。

score⁡(x,y)=∑t=1Nyscore⁡(x,y<t∘yt)=∑t=1Nylog⁡p(yt∣x,y<t)\operatorname{score}(x, y)=\sum_{t=1}^{N_{y}} \operatorname{score}\left(x, y_{<t} \circ y_{t}\right)=\sum_{t=1}^{N_{y}} \log p\left(y_{t} \mid x, y_{<t}\right) score(x,y)=t=1∑Ny​​score(x,y<t​∘yt​)=t=1∑Ny​​logp(yt​∣x,y<t​)

因为前面的s已经有了，这里只需要简单地加「log分数」就好了，其实也就是「每个时间步概率累乘」的对数形式，取 log 是为了防止太小导致边界溢出。

  • 当结尾为EOS时，就不生成了，但是会继续存在候选数组中和其他序列进行比分。
  • 结束条件为，top-k个序列全都是EOS结尾（这里伪代码没有break）或者到达预先设置的最大长度n_{max}。
  • 训练时不需要 beam search，只在生成时需要。

coverage penalty

比较Coverage Model (Huawei)和Coverage Penalty (Google)
华为诺亚 Coverage-based Neural Machine Translation 发现 over translation or under-translation due to attention coverage 问题。该方法的主要思想是为每个源端词维护一个coverage vector以表示该词被翻译（或覆盖）的程度。在解码过程中该覆盖率信息会传入attention model，以使它更关注于未被翻译的源端词。

受此启发，《Google’s Neural Machine Translation System: Bridging the Gap between Human and Machine Translation》 显式地对译文候选的覆盖率进行评价，从而选择覆盖率更高的译文。

Best-First Beam Search

香侬读 | Beam Search还能更快？结合优先队列的最佳优先化Beam Search

最基本的 Beam Search的计算过程中维护了一个「k-最大堆」：

  • 每一次迭代中，堆中序列的长度都是相同的。
  • 每一次迭代中，每个句子都要计算 ∣V∣|V|∣V∣ 次分数。
    而正是计算分数的次数决定了总体的时间复杂度。

这里引入 A∗A*A∗ 搜索算法，是一种启发式搜索，即通过预定义的得分函数，每次都首先探索得分最高的那一条路径（相当于得分优先搜索）。
显然，只要得分函数定义得好，就能比DFS和BFS更快地找到正确答案。

和基本的 Beam Search 相比吗，多了颜色标出的来的四项：

  1. 比较运算符：用于比较两个序列优先级，即判定出哪一个序列优先级更高，在堆最上面最先被pop。
  2. 停止策略
  3. k ( 基本的 Beam Search 也有
  4. 经验函数

算法思想就是维护一个堆，每一次取出最顶层的序列（line5），可以看作是当前分数最高的序列，对其进行拓展（line12），然后使用概率分数和启发式函数得到最终分数（line13，14），然后将结果推到堆中（line15）。
这里其实就没有 beam search 的剪枝的策略了，但是同样是有 beam_size 的限制的。绿色区域其实代替的就是 Bt←B.top(k)B_t \leftarrow B.\text{top}(k)Bt​←B.top(k) ，保证每个序列长度只拓展堆中分数最高的前 k 个序列。
当然还有会出现上层高分序列都是短句的情况，所以这里如果超过了 beam_size，我们直接 pop() 后 continue 就好了，相当于直接丢弃。
普通 beam search 堆中只有 beam_size 个序列，而这里序列有很多，每个时间步增加 V−1V-1V−1 个序列。
值得注意的就是 比较运算符 如何排序两个序列 还有 经验函数 。
如果最顶层的序列最后一项为 EOS\text{EOS}EOS ，则我们在其后面再次追加一个 EOS\text{EOS}EOS ，这样在后面进行排序时因为 比较运算符r 内部规则，它就能往下移了。
