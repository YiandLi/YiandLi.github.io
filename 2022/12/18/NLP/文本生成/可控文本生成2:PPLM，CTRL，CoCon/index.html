Delete, Retrieve, Generate: A Simple Approach to Sentiment and Style Transfer

属于修改 生成器的输出空间 + token 离散空间。
斯坦福 – NAACL 2018

假设：通常文本特征迁移可以通过改变一些标志词或短语（attribute markers） 来完成。正是这些 attribute markers 决定了该文本的特征。

所以我们只需要找到这些 attribute markers，将他们看作是可插拔的，然后使用目标 attribute markers 进行替换就可以得到目标风格的文本了。
本篇介绍了如何寻找离散的 attribute markers，然后根据 attribute markers 进行文本迁移的工作。

寻找 attribute markers

某个n-gram uuu 在「文本风格特征 vvv 」的所有语料 Dv\mathcal{D}_{v}Dv​ 中的「显著性频率」为：

s(u,v)=count⁡(u,Dv)+λ(∑v′∈V,v′≠vcount⁡(u,Dv′))+λs(u, v)=\frac{\operatorname{count}\left(u, \mathcal{D}_{v}\right)+\lambda}{\left(\sum_{v^{\prime} \in \mathcal{V}, v^{\prime} \neq v} \operatorname{count}\left(u, \mathcal{D}_{v^{\prime}}\right)\right)+\lambda} s(u,v)=(∑v′∈V,v′=v​count(u,Dv′​))+λcount(u,Dv​)+λ​

有点像 TF-IDF ， λ\lambdaλ 是平滑系数.
计算出数值后根据一个 阈值 最终确定所有的 attribute marker。

风格迁移操作

a(x,vsrc)a(x, v^{src})a(x,vsrc) 表示 句子 xxx 的所有 风格为 srcsrcsrc 的 attribute marker。
首先找到每个句子对应的 a(x,vsrc)a(x, v^{src})a(x,vsrc)

Delete

删除所有的 a(x,vsrc)a(x, v^{src})a(x,vsrc) ，删除后的句子表示为 c(x,vsrc)c(x, v^{src})c(x,vsrc) 。

Retrieve

使用 c(x,vsrc)c(x, v^{src})c(x,vsrc) 在目标特性 vtgtv^{tgt}vtgt 句子集合中检索一个内容最接近的句子 c(x′,vtgt)c(x', v^{tgt})c(x′,vtgt) 。这里可以使用不同的策略去找语意距离。

现在得到了：原始句子 c(x,vsrc)c(x, v^{src})c(x,vsrc) ，目标风格的 attribute marker a(x′,vtgt)a(x', v^{tgt})a(x′,vtgt) 。

Generate

这是最后一步，即获得最终的结果。文章里有四种策略

  1. Retrieve Only：直接返回第二步的索引结果 x′x'x′ 。这么做生成的句子在语言角度应该是正确的且带有目标特性，但可能在内容上和源句有出入。
  2. Template Based：直接替换原始 attribute marker a(x,vsrc)a(x, v^{src})a(x,vsrc) 为 a(x′,vtgt)a(x', v^{tgt})a(x′,vtgt) 。
    简单粗暴，可能产生不通顺的句子。
  3. Delete Only：使用一个 RNN encoder 编码 c(x,vsrc)c(x, v^{src})c(x,vsrc) ，然后拼接 vtgtv^{tgt}vtgt 的 embedding，再交由一个解码器解码。
    没有使用 a(x′,vtgt)a(x', v^{tgt})a(x′,vtgt) ，所以相当于直接 delete 了。
  4. Delete And Retrieve：和上一个类似，只不过这里用另一个RNN编码 a(x′,vtgt)a(x', v^{tgt})a(x′,vtgt) ，拼接 encoder 编码进行解码输出。

方法 1,2 不需要训练，方法 3,4 需要训练。
对于Delete Only，使用**重建句子任务（即训练一个自编码器）**来训练。
对于Delete And Retrieve则复杂一些，为了防止「特征迁移」的能力退化成「句子拼接」（c+a）的能力，作者在这里训练一个「降噪自编码器」，具体地说就是「随机替换a里的AM」，其实就是给任务提升难度。

PPLM

Uber – ICLR 2020 : Plug and Play Language Models: A Simple Approach to Controlled Text Generation

从标题也可以看出来，是引入了一个可插拔的组件对语言模型进行操作，从而修改生成文本。该方法可以直接用于文本生成，不需要微调。

首先假设文本风格/属性为变量 aaa ，模型生成的文本为 xxx ，则「模型基于某个属性aaa ，生成文本 xxx 」可以表示为：

p(x∣a)∝p(a∣x)p(x)p(x|a) \propto p(a|x)p(x) p(x∣a)∝p(a∣x)p(x)

p(x)p(x)p(x) 为文本的总体分布，即一个形式为 p(X)=∏i=1np(xi∣x0,⋯ ,xi−1)p(X)=\prod_{i=1}^{n} p\left(x_{i} \mid x_{0}, \cdots, x_{i-1}\right)p(X)=∏i=1n​p(xi​∣x0​,⋯,xi−1​) 的 unconditional LM ，比如 GPT2。
p(x∣a)p(x|a)p(x∣a) 为我们想得到的 conditional LM，即给定文本特征 aaa ，该模型可以生成具有该特征的文本。
p(a∣x)p(a|x)p(a∣x) 为额外的一个「特征分类器/attribute model」，输入文本，输出对应的风格。这个模型和语言模型相比，要小很多。

模型在生成文本时，每个时间步上都会进行三个步骤：

  1. 前向过程：当模型生成一个 token 时，通过已经生成的部分文本 xxx 得到 p(a∣x)p(a|x)p(a∣x) ，和其对语言模型的梯度。
  2. 反向过程：使用该梯度去更新语言模型的 隐状态 (internal latent representations of the LM)，让 p(a∣x)p(a|x)p(a∣x) 变大。
  3. 重采样：重新生成这一个 token

简而言之就是在生成的每个时间步，将历史隐向量往更接近目标风格的方向拉，从而使得新生成的token更符合目标的风格。
每个时间步都会让模型/隐向量进行微小地更新（lr比较小），并且用两个「梯度/目标函数」指导模型，使其生成具有先验文本特征的，流畅的文本，分别对应 p(a∣x)p(a|x)p(a∣x) 和 p(x)p(x)p(x) 。

该方法有三个优点：

  1. 属性控制强度可控；
  2. 多个属性可随意结合；
  3. 生成模型、属性模型可选。

这里就牵扯到如何更新 HHH 了。

首先 HHH 是什么 ？
论文中因为用的是 GPT2 语言模型，所以模型基本组件是 Transformer，所以历史隐状态 HtH_tHt​ 为 K-V pair 序列：

Ht.=[(Kt(1),Vt(1)),⋯ ,(Kt(l),Vt(l))]H_{t .}=\left[\left(K_{t}^{(1)}, V_{t}^{(1)}\right), \cdots,\left(K_{t}^{(l)}, V_{t}^{(l)}\right)\right] Ht.​=[(Kt(1)​,Vt(1)​),⋯,(Kt(l)​,Vt(l)​)]

是一个序列形式， (Kt(i),Vt(i))(K_{t}^{(i)}, V_{t}^{(i)})(Kt(i)​,Vt(i)​) 表示 第 iii 层的所有时间步 [0,t][0,t][0,t] /所有token 的K-V pair。
同时也只有这个变量，控制了下一个 token 的概率分布。
为了提高计算效率，可以选择仅修改最近过去的某个窗口内的隐变量，如红色虚线区域所示。

而如果要用其他模型，也可以根据模型不同改变具体 H 实现。

怎么求 ΔHt\Delta H_{t}ΔHt​ ？
ΔHt\Delta H_{t}ΔHt​ 为这一轮「文本隐状态 HHH 」的改变量，则改变后「文本隐状态」变为 ΔHt+Ht\Delta H_{t}+H_{t}ΔHt​+Ht​ ，那么我们将判别模型 p(a∣x)p(a|x)p(a∣x) 重写为 p(a∣H)p(a|H)p(a∣H) （其实有点不严谨，但是也还好， HtH_tHt​ 可以代表 xtx_txt​ ）。
然后 ΔHt\Delta H_{t}ΔHt​ 梯度更新的公式就是：

ΔHt←ΔHt+α∇ΔHtlog⁡p(a∣Ht+ΔHt)∥∇ΔHtlog⁡p(a∣Ht+ΔHt)∥γ\Delta H_{t} \leftarrow \Delta H_{t}+\alpha \frac{\nabla_{\Delta H_{t}} \log p\left(a \mid H_{t}+\Delta H_{t}\right)}{\left\|\nabla_{\Delta H_{t}} \log p\left(a \mid H_{t}+\Delta H_{t}\right)\right\|^{\gamma}} ΔHt​←ΔHt​+α∥∇ΔHt​​logp(a∣Ht​+ΔHt​)∥γ∇ΔHt​​logp(a∣Ht​+ΔHt​)​

α\alphaα 为学习率， γ\gammaγ 为标准化的缩放系数， ΔHt\Delta H_{t}ΔHt​ 初始化为0。

同时，作者还提到可以通过 p(a∣x)p(a|x)p(a∣x) 分数的大小，对结果进行 Re-ranking。

p(x)

如果不使用 p(x)p(x)p(x) 这个目标的话，模型容易退化，比如生成 “great great great great great”。所以我们期望PLM保持其原先的分布。
有两种方法：

  1. 最小化「更新后模型预测分布」和「原始模型预测分布」之间的KL散度，该项超参数设置为 0.01 效果不错。
  2. 融合修改前后的单词的输出概率（post-norm fusion），即

xt+1∼1β(p~t+1γgmpt+11−γgm)x_{t+1} \sim \frac{1}{\beta}\left(\tilde{p}_{t+1}^{\gamma_{g m}} p_{t+1}^{1-\gamma_{g m}}\right) xt+1​∼β1​(p~​t+1γgm​​pt+11−γgm​​)

其中 pt+1{p}_{t+1}pt+1​ 和 p~t+1\tilde{p}_{t+1}p~​t+1​ 是修改前后的词库概率分布。
γgm→1\gamma_{g m} \rightarrow 1γgm​→1 表示收敛于更新后的分布，γgm→0\gamma_{g m} \rightarrow 0γgm​→0 表示收敛于无条件的语言模型分布。实验发现 γgm∈[0.8,0.95]\gamma_{g m} \in [0.8, 0.95]γgm​∈[0.8,0.95] 效果不错。

接下来看主模型 PPLM 每一个时间步如何修改隐状态，
length为我们想让模型生成文本的长度，这里有一个for i in trange(length, ascii=True)，即每个时间步都进行修改：

  1. 更新变量
  2. 调用generate_text_pplm()方法得到根据 discriminator 更新后的隐状态pert_past
  3. 根据新的隐状态pert_past和last token重新得到模型输出pert_logits
  4. 根据输出，得到字典上的概率分布 pert_probs，这里有个小 trick
  5. 融合修改前后的单词的输出概率（post-norm fusion）
  6. 得到 token last，更新到 output_so_far 中

重要的变量就是：

  • past: 最后一个token之前序列的 past_key_values
  • last: 最后一个 token
  • unpert_past: 全序列的 past_key_values
  • unpert_logits: 全序列的 logits(vocab_size)

然后使用perturb_past()方法修改隐状态得到pert_past，使用pert_past和last得到输出 token。过程详见 section:perturb_past。
这里有个小 trick，用来放缩相似度，希望预测出来重复的 token：得到 pert_logits 之后，如果这个token 和序列中已经出现的 token 相似（相似度 > 0），则让分数减少，反之分数变大。

然后就是融合修改前后的生成token的输出概率，修改前的为 unpert_probs，修改后的为pert_probs，这里使用超参数 gm_scale。

然后 top-k-sample，见下面的 section：top-k-sample，得到token，更新 output_so_far。

CTRL

CTRL: A CONDITIONAL TRANSFORMER LANGUAGE MODEL FOR CONTROLLABLE GENERATION
在生成文本时可以通过 control codes 指定文章的领域、风格、主题、时间、实体，实体间的关系，以及任务相关的行为等等。
模型使用 Control Code 和 sub-reddit data 对生成的文本内容进行约束。

Model

使用 temperature-controlled stochastic nucleus sampling 方法：

  • 在有多个非零的高概率候选 token 时，不采用模型，而是 “贪婪” 地选择下一个 token（k=1）。

CoCon

CoCon: A Self-Supervised Approachfor Controlled Text Generation from ICLR2021

PPLM，CTRL 都是只针对全局的情感特征，忽略了更加细粒度的内容（单词/短语级别）。
以CTRL为代表的方法只能从一个总体的层面对生成文本增加限制，比如某种风格主题，但无法对具体内容进行精细控制，CoCon则是直接以原文中的部分内容作为condition。

作者设计了一个叫做CoCon的模块，插入transformer的encoder和decoder中，修改隐状态。
在具体实现中，使用 GPT2-medium 模型，其前7层作为encoder，后17层作为decoder。
GPT2 模型参数是冻结的，只训练 CoCon。

ccc 为风格描述/目标内容，即control部分，长度为 lcl_clc​ 。
在进行 CTG 时，每个token的隐状态都会根据 ccc 进行修改，从而达到 CTG 的目的。

生成的句子为 xxx ，长度为 lll 。在时间步 ttt 时， xxx 被划分成了 1:t−11:t-11:t−1 和 t:lt:lt:l 两部分 。
我们是需要使用 ccc 和 1:t−11:t-11:t−1 去预测第 ttt 个 token。

首先使用 encoder 单独编码 ccc 和 x1:t−1x_{1:t-1}x1:t−1​ ，得到 hch_chc​ 和 h1:t−1h_{1:t-1}h1:t−1​ ，和对应的 QKV 矩阵：

h1:t−1→Q,K,V∈R(t−1)×dhc→Kc,Vc∈Rc×d\begin{aligned} h_{1:t-1} &\rightarrow \mathbf{Q,K,V}\in \mathbb R^{(t-1)\times d}\\ h_{c} &\rightarrow \mathbf{K^c,V^c}\in \mathbb R^{c\times d} \end{aligned} h1:t−1​hc​​→Q,K,V∈R(t−1)×d→Kc,Vc∈Rc×d​

训练内容

作者使用了四个 Loss 进行训练。
现在考虑两个文本样本 x, y，分别被切分为 x1, x2 和 y1, y2 。

  1. Self Reconstruction Loss
    x1 作为源文本，后半段 x2 作为控制文本，希望模型能还原 x2。
    是为了让模型能够捕捉并利用控制条件中的信息。

  2. Null Content Loss
    将控制条件替换为 Null ，模型退化为非可控文本生成模型，训练出 CoCon 作为单向语言模型的性能。

  3. Cycle Reconstruction Loss

zy1,x2=CoCon⁡(p=y1,c=x2)Lcycle =−log⁡P(x2∣p=x1,c=zy1,x2)\begin{gathered} z_{y_1, x_2}=\operatorname{CoCon}\left(\mathbf{p}=y_1, \mathbf{c}=x_2\right) \\ \mathcal{L}_{\text {cycle }}=-\log \mathrm{P}\left(x_2 \mid \mathbf{p}=x_1, \mathbf{c}=z_{y_1, x_2}\right) \end{gathered} zy1​,x2​​=CoCon(p=y1​,c=x2​)Lcycle ​=−logP(x2​∣p=x1​,c=zy1​,x2​​)​

首先将y1作为源文本，将x2作为条件，希望模型生成文本 zy1,x2z_{y_{1},x_{2}}zy1​,x2​​ 。
然后使用x1作为源文本 zy1,x2z_{y_{1},x_{2}}zy1​,x2​​ 作为条件，希望模型能够生成x2。
这就要求模型
- 在生成的结果 zy1,x2z_{y_{1},x_{2}}zy1​,x2​​ 中尽可能保留控制条件 x2 中的信息，并且和 y1 衔接流畅；
- 在生成过程中尽可能利用上控制条件中保留的信息，即依据 zy1,x2z_{y_{1},x_{2}}zy1​,x2​​ 中保留的x2的信息复原x2，同时和 x1 衔接顺畅。

  1. Adversarial Loss
    最后是过去的工作都会用的对抗损失，即让生成的文本接近真实的文本：

Ladv =Ex[log⁡fdisc (enc⁡(x))]+Ey[log⁡(1−fdisc (enc⁡(y))]\mathcal{L}_{\text {adv }}=\mathbb{E}_{\mathbf{x}}\left[\log f_{\text {disc }}(\operatorname{enc}(\mathbf{x}))\right]+\mathbb{E}_{\mathbf{y}}\left[\log \left(1-f_{\text {disc }}(\operatorname{enc}(\mathbf{y}))\right]\right.
