文本生成评价指标

Posted on 2022-12-11 In NLP , Text Generation
Symbols count in article: 1.6k Reading time ≈ 8 mins.

词重叠率

在词汇级别上，计算模型的生成文本和人工的参考文本之间的相似性。
BLEU 和 METEOR 常用于机器翻译任务，ROUGE 常用于自动文本摘要。

BLEU

背景：在 SMT（统计机器翻译）时代，机器翻译效果稀烂，需要同时评价翻译的准确度和流畅度。

BLEU （Bilingual Evaluation Understudy，双语评估辅助工具）。
是所有评价指标的鼻祖，它的核心思想是比较 候选译文 和 参考译文 里的 n-gram 的重合程度，重合程度越高就认为译文质量越高。
uni-gram 用于衡量单词翻译的准确性，n-gram用于衡量句子翻译的流畅性。

BLEU=BP⋅exp⁡(∑n=1Nwnlog⁡Pn)BP={1 if c>re1−r/c if c≤r\begin{gathered} B L E U=B P \cdot \exp \left(\sum_{n=1}^N w_n \log P_n\right) \\ B P= \begin{cases}1 & \text { if } c>r \\ e^{1-r / c} & \text { if } c \leq r\end{cases} \end{gathered} BLEU=BP⋅exp(n=1∑N​wn​logPn​)BP={1e1−r/c​ if c>r if c≤r​​

  • 需要计算译文 1-gram，2-gram，…，N-gram 的精确率，一般 N 设置为 4 即可。
    充当了精确率指标。1-gram 反映充分性，而较长的 n-gram 反映翻译的流畅性。
  • PnP_nPn​ 为 n-gram 的精确率， wnw_nwn​ 为其对应的权重，一般设置为等权重。
    注意有时存在 Pn=0P_n=0Pn​=0 的情况，所以要进行 smoothing。
  • BP (Brevity Penalty) 是惩罚因子，ccc 为机器输出的翻译译文的长度， rrr 为长度最接近输出译文的参考译文的长度。弥补了 BLEU 得分没有召回率项的缺陷。

原理和代码实现

P_n 的计算

  1. 统计「候选译文」里每个 n-gram词 出现的次数，得到 counts。
  2. 统计 counts 中的每个词在所有「参考译文」中出现的最大次数 ，得到 max_counts。
  3. 统计 counts 和 max_counts 每个元素的最小值，得到 clipped_counts。 其实就是交集，判断输出译文中对了多少。
  4. 最后计算得到 PnP_nPn​ ，即clipped_counts的「value和」除以 counts的「value和」。

BP 的计算

BP={1 if c>re1−r/c if c≤rB P= \begin{cases}1 & \text { if } c>r \\ e^{1-r / c} & \text { if } c \leq r\end{cases} BP={1e1−r/c​ if c>r if c≤r​

选择时，使用min()方法，选择 ( 1.长度最接近 2.长度最短 ) 的候选句子。
只要输出的翻译长度等于任意的候选译文的长度，BP就等于1。

优缺点

  • 优点：
    计算很快；应用范围广泛，这可以让你很轻松将模型与相同任务的基准作对比。
  • 缺点：
    • 不考虑语义，句子结构，只考虑长度，和n-gram信息
    • 偏向于较短的翻译结果（brevity penalty 没有想象中那么强）
    • 不能很好地处理形态丰富的语句（BLEU原文建议大家配备4条翻译参考译文）

ROUGE

背景：等到 NMT （神经网络机器翻译）出来以后，神经网络脑补能力极强，翻译出的结果都是通顺的，但是有时候容易瞎翻译，所以需要强调 recall。ROUGE的出现很大程度上是为了解决 NMT 的漏翻问题（低召回率）。

全称 Recall-Oriented Understudy for Gisting Evaluation，可以看做是BLEU 的改进版，专注于召回率而非精度。
即 BLEU 是将「输出译文」看作为基准，而 ROUGE 是将「参考译句」看作为基准。

大致分为四种（常用的是前两种）：

  • ROUGE-N （将BLEU的精确率优化为召回率）
  • ROUGE-L （将BLEU的n-gram优化为公共子序列）
  • ROUGE-W （将ROUGE-L的连续匹配给予更高的奖励）
  • ROUGE-S （允许n-gram出现跳词(skip)）

下面都是讲解对应单个译文的计算方法。
如果给定多个参考译文 Si，Chin-Yew Lin 也给出了一种计算方法，假设有 M 个译文 S1, …, SM。ROUGE-N 会分别计算机器译文和这些参考译文的 ROUGE-N 分数，并取其最大值，公式如下。这个方法也可以用于 ROUGE-L，ROUGE-W 和 ROUGE-S。

ROUGE-N

“N” 指的是N-gram，主要统计 N-gram 上的召回率。
给定『输出译文』和一个「参考译文」，公式的分母是统计在参考译文中 N-gram 的个数，分子是统计参考译文与机器译文共有的 N-gram 个数。

ROUGE-L

“L” 指最长公共子序列 (longest common subsequence, LCS)。
使用了机器译文 C 和参考译文 S 的最长公共子序列进行计算：

RLCS=LCS(C,S)len⁡(S)PLCS=LCS(C,S)len⁡(C)FLCS=(1+β2)RLCSPLCSRLCS+β2PLCS\begin{aligned} & R_{L C S}=\frac{L C S(C, S)}{\operatorname{len}(S)} \\ & P_{L C S}=\frac{L C S(C, S)}{\operatorname{len}(C)} \\ & F_{L C S}=\frac{\left(1+\beta^2\right) R_{L C S} P_{L C S}}{R_{L C S}+\beta^2 P_{L C S}} \end{aligned} ​RLCS​=len(S)LCS(C,S)​PLCS​=len(C)LCS(C,S)​FLCS​=RLCS​+β2PLCS​(1+β2)RLCS​PLCS​​​

RLCSR_{L C S}RLCS​ 表示召回率，而 PLCSP_{L C S}PLCS​ 表示精确率，FLCSF_{L C S}FLCS​ 就是 ROUGE-L。
通常 β\betaβ 很大，如果 β\betaβ 很大，则 PLCS 那一项可以忽略不计。
