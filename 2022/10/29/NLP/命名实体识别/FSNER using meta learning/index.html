FSNER using meta learning

Posted on 2022-10-29 In NLP , Named entity recognition
Symbols count in article: 3.1k Reading time ≈ 16 mins.

最近看了一些 Few-Shot NER 的工作，发现主要有两种思路，

  • 第一种就是和原始 full-shot 场景一样，只不过是使用迁移学习的方法；
  • 第二种就是使用度量学习和元学习，构造 Prototype Network ，先表示 support set 上 labeled entity/token 的原型表征，然后用欧式距离作为度量，将距离最近的「原型点」的 label 作为「目标点」的结果，本文就介绍一些这方面的工作吧。

关于度量学习，推荐： 10个机器学习中常用的距离度量方法。
关于元学习，推荐：Meta-learning核心思想及近年顶会3个优化方向。
Meta-learning 直接评测在 trainset 训练几轮后的模型在 testset 上的效果(loss)，再使用这个效果作为信号计算(loss)并回传梯度，指导模型更新：

ω∗=arg⁡min⁡ω∑i=1MLmeta (θ∗(i)(ω),ω,Dsource val (i)) s.t. θ∗(i)(ω)=arg⁡min⁡θLtask (θ,ω,Dsource train (i) ) \begin{aligned} \omega^{*} &=\underset{\omega}{\arg \min } \sum_{i=1}^{M} \mathcal{L}^{\text {meta }}\left(\theta^{*(i)}(\omega), \omega, \mathcal{D}_{\text {source }}^{\text {val }(i)}\right) \\ \text { s.t. } \theta^{*(i)}(\omega) &=\underset{\theta}{\arg \min } \mathcal{L}^{\text {task }}\left(\theta, \omega, \mathcal{D}_{\text {source }}^{\text {train }(i)}\right. \text { ) } \end{aligned} ω∗ s.t. θ∗(i)(ω)​=ωargmin​i=1∑M​Lmeta (θ∗(i)(ω),ω,Dsource val (i)​)=θargmin​Ltask (θ,ω,Dsource train (i)​ ) ​

第二行公式表示在 support集 上进行训练，得到更新后的模型 $\theta^{*(i)}(\omega) $ 。
第一行表示使用这个更新后的模型在 query集 上进行检测，得到loss，使用这个loss对原始模型 ω\omegaω 进行更新，即训练目标就是最小化在 query 集合上的表现能力。

通常 Few-shot NER 任务满足一种 meta learning 的设定：

  1. 从训练集中采样出多个 Training episodes: Etrain ={(Strain ,Qtrain ,Ytrain )}\mathcal{E}_{\text {train }}=\left\{\left(\mathcal{S}_{\text {train }}, \mathcal{Q}_{\text {train }}, \mathcal{Y}_{\text {train }}\right)\right\}Etrain ​={(Strain ​,Qtrain ​,Ytrain ​)}
    • Strain \mathcal{S}_{\text {train }}Strain ​ 表示 support set
    • Qtrain \mathcal{Q}_{\text {train }}Qtrain ​ 表示 query set
    • Ytrain \mathcal{Y}_{\text {train }}Ytrain ​ 表示 实体/tag 标签集合
    • 有些方法也不会分这么细，就是直接在 source dataset 上训练。
  2. 从测试集中采样出多个 novel episodes: Enew ={(Snew ,Qnew ,Ynew )}\mathcal{E}_{\text {new }}=\left\{\left(\mathcal{S}_{\text {new }}, \mathcal{Q}_{\text {new }}, \mathcal{Y}_{\text {new }}\right)\right\}Enew ​={(Snew ​,Qnew ​,Ynew ​)}

具体地，模型从 Etrain \mathcal{E}_{\text {train }}Etrain ​ 采样多个 episode，进行训练。
训练之后在每个 novel episode 上进行验证，即先使用 Snew \mathcal{S}_{\text {new }}Snew ​ 进行训练，然后在 Qnew \mathcal{Q}_{\text {new }}Qnew ​ 上进行验证。

特点

  1. 使用一个降维线性层将高纬度的bert特征投影为低纬原型特征
  2. 原型点是随机初始化的，在后续训练中进行更新的
  3. 同时引入分布稀疏损失，让两两原型点的距离的 平均值 为超参数 τ\tauτ。
  4. 概率为： softmax(-1 * 欧氏距离)

ESD

论文: An Enhanced Span-based Decomposition Method for Few-Shot Sequence Labeling

本文中，作者想强调 span 之间的关系，并且将few-shot ner任务定义为一种 span matching 任务。

ESD 模型包含三种个模块：

  1. Span Representation: 考虑 intra-span and inter-span interactions，即 span representation 可以通过如下两个信息进行强化：

    • 句内其他 span 的信息
    • query 和 support 集合中的不同 span 交互信息

  2. Span Prototype Aggregation：将相同 class 的 span 聚合成一个原型表征

  3. Span Conflicts Resolution：预测的 span 可能存在冲突，针对此提出 Beam Soft-NMS。

主要是使用 Attention 机制交互不同的原型，挺复杂的。个人不太喜欢这种方式，不多介绍了。

SpanProto

论文：SpanProto: A Two-stage Span-based Prototypical Network for Few-shot Named Entity Recognition

前面一般都是将 O 标签也看作一个 span 类型，而这里是单独将其拎出来做处理。目的是为了防止模型将所有的 O span 都看成一类，实际情况下有些 O span 在别的体系中是属于一种实体类别的。

本文中作者将 FS-NER 任务拆解成两步任务：

  1. span extraction：抽取出所有 entity
  2. mention classification：判别抽取出的 entity 的实体类型

Margin-based Learning

Span Extraction 有可能抽取出 非实体span，所以这里针对这些span 进行训练。

因为没有为 O-span 设置样本点，所以这里拉远 O-span 表征和其他「实体原型表征」之间的距离，直到距离达到一个超参数值，所以使用了一个 margin 作为约束：

后续推理时，如果某个 span 距离所有的「实体原型表征」之间的距离都大于超参数 rrr
，那么该 span 为 O-span，即不是实体。

特点

  1. 拆解任务为span分类和entity分类
  2. 使用support set上同类别span表征的平均值作为实体的原型表征
  3. 概率计算方式为： Softmax(-1*欧氏距离)；训练直接令golden概率最大。
  4. 没有O-span 原型点。但是考虑了第一步抽取实体时出现 false positive 的情况，使用 Margin-loss 单独优化 O-span。

MAML-ProtoNet

Paper: Decomposed Meta-Learning for Few-Shot Named Entity Recognition

同样也是两步走：

  1. Entity Span Detection：先预测span
  2. Entity Typing：给每个 span 判别实体类型

Entity Span Detection

这个模块仅仅判别span 是不是实体，不考虑实体类别。所以作者称其为 type-agnostic，即「实体类型不可知」的。

模型部分，首先使用一个 BERT 对序列进行编码，得到 h={h1,...,hL}\mathbf h=\{h_1, ...,h_L\}h={h1​,...,hL​} 。

Inference

因为上一步骤已经抽取出所有 span，所以这里就不需要考虑 O-tag 的情况了。
推理时，使用概率最大的标签类型作为 span 的类别。

特点

  1. 拆解任务为span分类和entity分类
  2. 使用span平均表征作为类别原型表征
  3. 没有 O-span 原型表征，默认抽出来的span都有标签。
  4. 概率计算方式 softmax(-1 * 欧氏距离 )

self one
