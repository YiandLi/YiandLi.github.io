Full-Shot Generative NER

Paper: A Unified Generative Framework for Various NER Subtasks
Code: github: yhcc/BARTNER

三种 NER 任务：

  • flat NER
  • nested NER：「北京大学」和内部的「北京」都是实体。
  • discontinuous NER：“用于治疗胃部和毛细血管细胞的炎症” 中的「胃部炎症」。

本文提出模型，使用 Bart + Pointer Network 。

  • 输入长度为 nnn 的序列 X=[x1,x2,…,xn]X=\left[x_{1}, x_{2}, \ldots, x_{n}\right]X=[x1​,x2​,…,xn​] ，Entity Tag Token 序列为 G=[g1,…,gl]G=\left[g_{1}, \ldots, g_{l}\right]G=[g1​,…,gl​] ，比如 “Person”，“Location” …
    这两者的索引序列组成了 Pointer Network 的索引序列，总序列长度为 n+ln+ln+l 。
  • 输出序列为 Y=[s11,e11,…,s1j,e1j,t1,…,si1,ei1,…,sik,eik,ti]Y=\left[s_{11}, e_{11}, \ldots, s_{1 j}, e_{1 j}, t_{1}, \ldots, s_{i 1}, e_{i 1}, \ldots, s_{i k}, e_{i k}, t_{i}\right]Y=[s11​,e11​,…,s1j​,e1j​,t1​,…,si1​,ei1​,…,sik​,eik​,ti​] 。
    sss 和 eee 为某 span 的 start and end index， ttt 为 entity tag index，对应 GGG。

Detailed Entity Representation with BART

BART 使用的是 BPE 编码，所以会将单独的 word 拆分成多个 token。

所以这里作者使用三种 pointer-based entity representations 方法来定位 entity：

  1. Span：使用输入序列中的 start index 和 end index 表示不同的 span。
  2. BPE：直接输出所有 token 的 index
  3. Word：只输出每个 word 的首个 token 的 index

如果句子中没有 entity，则输出序列为 <s>,</s><\mathrm{s}>, </ \mathrm{s}><s>,</s> 。

同时作者还在后面讨论了这三种表示方法的效果和原因，主要围绕序列长度和是否符合pre-training编码两个方面，这两个点确实很不错，但是作者讨论出的结果还是有点牵强的。

Few-Shot Light NER

Paper: LightNER: A Lightweight Tuning Paradigm for Low-resource NER via Pluggable Prompting
Code: LightNER: A Lightweight Tuning Paradigm for Low-resource NER via Pluggable Prompting

低资源场景下 NER 任务有两个难点：

  1. Class Transfer：迁移学习时，source domain 和 target domain 的标签不一致。
  2. Domain Transfer：source domain 和 target domain 的语法习惯（grammar style）和语义（allegorical
    theme）不一致，

针对这两个难点，作者：

  1. 在原始的生成模型架构中加入一个 unified learnable verbalizer，实现 Class Transfer。这个 verbalizer 和模型是非耦合的，所以不用重新训练模型，对模型迁移很友好。
  2. 将可学习参数加入到 self-attention 层中，作为 pluggable guidance；并使用一个轻量化微调框架/方法（a lightweight tuning paradigm）让模型学到不同 domain 的语义知识，实现 Domain Transfer。

Preliminaries

Low-resource NER

a rich-resource source NER dataset: H={(X1H,Y1H), ..., (XRH,YRH)}\mathbb{H}=\left\{(\mathbf X_1^H,\mathbf Y_1^H ),\ ...,\ (\mathbf X_R^H,\mathbf Y_R^H ) \right\}H={(X1H​,Y1H​), ..., (XRH​,YRH​)}
每个样本的输入序列为 XH={x1H,...,xnH}\mathbf X^H = \{x_1^H, ..., x_n^H\}XH={x1H​,...,xnH​} ，对应的 token标签/tag 序列为 YH={y1H,...,ynH}\mathbf Y^H = \{y_1^H, ..., y_n^H\}YH={y1H​,...,ynH​} ，其中标签集合为 CH\mathcal C^HCH 。

a low-resource target NER dataset: L={(X1H,Y1L), ..., (XRL,YRL)}\mathbb{L}=\left\{(\mathbf X_1^H,\mathbf Y_1^L ),\ ...,\ (\mathbf X_R^L,\mathbf Y_R^L ) \right\}L={(X1H​,Y1L​), ..., (XRL​,YRL​)} ，其中标签集合为 CL\mathcal C^LCL 。

通常 CL\mathcal C^LCL 和 CH\mathcal C^HCH 交集标签数量很小，甚至交集为空集。

Label-specific Classifier for NER

Encoder：通常序列标注任务需要一个 encoder 将 token 动态编码为对应的隐藏层表征，这里尝试 BERT 和 BART的Encoder 作为编码器。

Label-Specific Classifier：还需要用到一个分类器对 token 的隐藏层表征进行分类，

整个模型表示为：

h1:n=Encoder⁡(x1:n)q(y∣x)=softmax(hiWC+bC)(i∈[1,…,n])\begin{aligned} \mathbf{h}_{1: n} &=\operatorname{Encoder}\left(x_{1: n}\right) \\ q(\boldsymbol{y} \mid \boldsymbol{x}) &=softmax \left(\mathbf{h}_i \mathbf{W}_{\mathcal{C}}+\mathbf{b}_{\mathcal{C}}\right)(i \in[1, \ldots, n]) \end{aligned} h1:n​q(y∣x)​=Encoder(x1:n​)=softmax(hi​WC​+bC​)(i∈[1,…,n])​

根据 encoder 的不同，作者称其为 LC-BERT 和 LC-BART。

Methodology

输入序列（tokenized）： X={x1,x2,...,xn}X=\{x_1,x_2,...,x_n\}X={x1​,x2​,...,xn​}
输出序列：X={p1,p2,...,pl}X=\{p_1,p_2,...,p_l\}X={p1​,p2​,...,pl​} ，其中 p={eistart,eiend,ti}p=\{e_i^{start}, e_i^{end}, t_i\}p={eistart​,eiend​,ti​} ， eistart,eiende_i^{start}, e_i^{end}eistart​,eiend​ 分别表示 entity token 的 start index 和 end index， ttt 表示 tag index。

输入序列和输出序列前后都有占位符 <s>,</s><\mathrm{s}>, </ \mathrm{s}><s>,</s> 。

Unified Learnable Verbalizer

传统预测 entity tag 时，因为原始的 classifier 中没有对应的 tag，所以需要从头训练。
如果不这样重新初始化训练，也很难从 embedding layer 中找到其对应的合适的 token。
同时，有些 tag 包含了多个字符，也给初始化带来了困难。

为了解决上述问题，作者提出使用一个可以学习的 Verbalizer mapping M\mathcal MM， 将实体类型标签 C\mathcal CC 映射到对应的 label words V\mathcal VV ：

M:C⟶V\mathcal M: \mathcal C \longrightarrow \mathcal V M:C⟶V

比如 c="return_date.month_name"c = \text{"return\_date.month\_name"}c="return_date.month_name" ，其对应的 label words 为 Vc={"return", "date", "month", "name"}\mathcal V_c = \{\text{"return",\ "date",\ "month",\ "name"}\}Vc​={"return", "date", "month", "name"} 。

同时，直接对 V\mathcal VV 内的所有 label word 进行 average 效果可能不好，作者这里采用了可学习的权重 β\betaβ 对每个 label word 进行加权求和，得到最终的 pointer logistic：

Etag=WordEmbed⁡(M(C))ptag = Concat [∑v∈Vβvc∗Etag c⊗ht]\begin{aligned} E_{t a g} &=\operatorname{WordEmbed}(\mathcal{M}(\mathcal{C})) \\ p_{\text {tag }} &=\text { Concat }\left[\sum_{v \in \mathcal{V}} \beta_v^c * E_{\text {tag }}^c \otimes h_t\right] \end{aligned} Etag​ptag ​​=WordEmbed(M(C))= Concat [v∈V∑​βvc​∗Etag c​⊗ht​]​

βvc\beta_v^cβvc​ 表示 label word vvv 对应的权重， ∑βvc=1\sum \beta_v^c=1∑βvc​=1 。

所以时间步 nnn 时得到 index 的过程为：

Eseq= WordEmbed (X)H~en=α⋅Hen+(1−α)⋅Eseqpseq=H~en⊗htEtag=WordEmbed⁡(M(C))ptag = Concat [∑v∈Vβvc∗Etag c⊗ht]pt=Softmax⁡([pseq;ptag])\begin{aligned} & E_{s e q} =\text { WordEmbed }(X) \\ & \tilde{H}_{e n} =\alpha \cdot H_{e n}+(1-\alpha) \cdot E_{s e q} \\ & p_{s e q} =\tilde{H}_{e n} \otimes h_t \\ & E_{t a g} =\operatorname{WordEmbed}(\mathcal{M}(\mathcal{C})) \\ & p_{\text {tag }} =\text { Concat }\left[\sum_{v \in \mathcal{V}} \beta_v^c * E_{\text {tag }}^c \otimes h_t\right] \\ & p_t =\operatorname{Softmax}\left(\left[p_{s e q} ; p_{t a g}\right]\right) \end{aligned} ​Eseq​= WordEmbed (X)H~en​=α⋅Hen​+(1−α)⋅Eseq​pseq​=H~en​⊗ht​Etag​=WordEmbed(M(C))ptag ​= Concat [v∈V∑​βvc​∗Etag c​⊗ht​]pt​=Softmax([pseq​;ptag​])​

和上一个模型相比，少了一个对 H~en\tilde{H}_{e n}H~en​ 进行 MLP 的操作，更新了 ptag p_{\text {tag }}ptag ​ 的求法。

Pluggable Guidance Module

Parameterized Setting

为 decoder 和 encoder 分别设置两套 embedding 参数 {ϕ1,...,ϕN}\{\phi^1, ..., \phi^N \}{ϕ1,...,ϕN}

NNN 为 transformer layer 数量
ϕθ∈R2×∣P∣×d\phi_\theta \in \mathbb R^{2\times |P|\times d}ϕθ​∈R2×∣P∣×d ， ∣P∣|P|∣P∣ 为 prompt 长度， ddd 为 dim(ht)dim(h_t)dim(ht​) ， 222 表示分别应用于 key 和 value。

在模型中，只有 {ϕ1,...,ϕN}\{\phi^1, ..., \phi^N \}{ϕ1,...,ϕN} 和 上述label word 的 β\betaβ 参数需要更新，其他参数都是冻结的。

Pluggable Guidance Layer

给定输入序列 X={x1, ..., xn}X=\{x_1,\ ...,\ x_n\}X={x1​, ..., xn​} ，对应的隐状态为 X∈Rnd\mathbf X\in \mathbb R^{nd}X∈Rnd 。

首先使用 X\mathbf XX 得到对应的 QKV

Ql=XlWQ,Kl=XlWK,Vl=XlWV\boldsymbol{Q}^l=\boldsymbol{X}^l \boldsymbol{W}^Q, \boldsymbol{K}^l=\boldsymbol{X}^l \boldsymbol{W}^K, \boldsymbol{V}^l=\boldsymbol{X}^l \boldsymbol{W}^V Ql=XlWQ,Kl=XlWK,Vl=XlWV

在这个基础上，加入上述的 θ\thetaθ （其实就是 prefix tunning）：

 Attention l=softmax⁡(Ql[ϕkl;Kl]Td)[ϕvl;Vl]\text { Attention }^l=\operatorname{softmax}\left(\frac{\boldsymbol{Q}^l\left[\boldsymbol{\phi}_k^l ; \boldsymbol{K}^l\right]^T}{\sqrt{d}}\right)\left[\boldsymbol{\phi}_v^l ; \boldsymbol{V}^l\right]  Attention l=softmax⎝⎜⎜⎛​d​Ql[ϕkl​;Kl]T​⎠⎟⎟⎞​[ϕvl​;Vl]

这一步操作可以 re-modulate attention 的分布。

Transfer Knowledge from a General Domain to Specific Domains

作者发现随着 target domain 数据量增多，传统的 prototype-based 方法效果增长缓慢，但是 Light NER 效果增长显著，可以表示 LightNER 迁移知识的能力很好。
同时对比 LC-BERT 和 LC-BART，可以说明 Pluggable Guidance Module 可以有效地迁移知识。
