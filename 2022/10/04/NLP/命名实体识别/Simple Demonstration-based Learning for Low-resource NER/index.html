Simple Demonstration-based Learning for Low-resource NER

Paper: Good Examples Make A Faster Learner Simple Demonstration-based Learning for Low-resource NER
Coding: INK-USC/fewNER

Abstract and Introduction

本篇文章介绍利用 Demonstration-based Learning 解决 Few-Shot NER 的问题
提起 In-Context Learning，大家都会认为是属于 GPT 一族的；
而本文的「Demonstration-based Learning 变体」是针对 BERT 一族的 In-Context Learning，
即将 (xtrain,ytrain)(x_{\text{train}},y_{\text{train}})(xtrain​,ytrain​) 和 xpredictx_{\text{predict}}xpredict​ 一同输入给模型，期待使用 task demonstrations，即 (xtrain,ytrain)(x_{\text{train}},y_{\text{train}})(xtrain​,ytrain​) 提示模型。

我们拼接的 examples 后续就类似 prefix-tunning 中的 soft prefix 一样，
被当成 sequence 放入 BERT 中做编码，但是在预测 tag 时，只有原始序列的 token 被预测。

Demonstration Strategy: How to do Demonstration-based Learning ：

  • what to include (only use entity span or add original context)
  • how to select the prompt examples
  • what templates to use to concatenate test sequence and prompt examples

针对上述三个角度，作者分别介绍了不同的策略，同时针对两种场景（in-domain learning and domain adaptation），分别做了实验。
结果证明，合适的 demonstration strategy 可以大大减少对标注数据的需求量，实现 Few-Shot NER。

low-resource setting

  1. In-Domain Setting：training set 和 test set 来自于同一个语料
  2. Domain-Adaptation Setting：training set 和 test set 来自于不同语料

In-domain Low-resource Learning

Training set Dtrain\mathcal D_{train}Dtrain​ 和 Test set Dtest\mathcal D_{test}Dtest​ 来自于同一个语料，但是 Dtrain\mathcal D_{train}Dtrain​ 中样本数量很少 (e.g. N<50)。

Low-resource Domain Adaption

这里又有两种 setting：

  • label-sharing setting：两个集合标签一致。集合一致不代表 grammar habit 和 corpus 一致，所以还是要考虑迁移的问题。
  • label-different setting：两个集合标签不同

具体实现步骤为：

  • 先在 high-resource source set S\mathcal SS 上微调得到模型 Ms\mathcal M_sMs​
  • 然后在 low-resource support target set T\mathcal TT 上微调得到模型 Mt\mathcal M_tMt​
    • 其实就是先复用 Ms\mathcal M_sMs​ 参数，然后再继续训练。
    • 实验表明，只复用 contextualized embedder（PLM）参数效果最好；即不复用 crf 或者 tag classifier 层。
  • 最后在 test set 上验证模型效果

Strategy

Automatic task Demonstration construction

Entity-oriented Demonstration

Entity type 集合: L={l1,...,l∣L∣}\mathbf L=\{l_1, ..., l_{|L|}\}L={l1​,...,l∣L∣​}
在训练集 Dtrain\mathcal D_{train}Dtrain​ 中为每个 entity type lll 选择一个 entity span eee ，组成 demonstration，拼接到 sequence 中。

  • random：随机选择 entity span eee
  • popular：选择频率最高的 entity span eee
  • search：
    • 首先得到每个 entity type lll 频率最高的 k 个 entity span eee
    • 使用网格搜索，在测试集 Ddev\mathcal D_{dev}Ddev​ 中，选择分数最高的 entity span eee ，遍历次数为 k∣L∣k^{|L|}k∣L∣

Instance-oriented Demonstration

在训练集中按照一定标准找到和 input sentence 最相关的 instance，作为 demonstration 拼接到 sequence 中。

  • SBERT (Reimers and Gurevych, 2019)：输入到 pre-trained bi-encoder 中，找到 [cls] cosine similarity 最大的样本。
  • BERTScore (Zhang et al., 2020)：按照 token-level，找到 sum cosine similarity 最大的样本。

Fixed vs Variable Demonstration

  • Variable demonstration: random, SBERT, BERTScore
  • Fixed demonstration: popular, search

Demonstration template

有了目标 Entity-oriente Demonstration 或 Instance-oriented Demonstration 后，就需要选择合适的模版 T\mathcal TT 来组合 Demonstration 内部的信息了。
因为还是 hard token 模式，当然后续自己用的时候可以考虑换成 soft token。


从图中可以清晰地看到模版的定义：

  • no-context：只用于 entity-oriented demonstration
    每个 (e,l)(e,l)(e,l) 的模版格式为 e is l. ，中间使用[SEP]间隔开。

  • context

    • entity-oriented demonstration：同时找到 (e,l)(e,l)(e,l) 对应的 context sss ，组成 (e,l,s)(e,l,s)(e,l,s) 。模版格式为 s. e is l. ，中间使用[SEP]间隔开。
    • instance-oriented demonstration：找到所有提及的 (e,l)(e,l)(e,l) 对应的 context sss 。
      模版为s1. e1 is l1. e2 is l2. ... en is ln. [SEP] s2. en+1 is ln+1. ...。

  • lexical

    • entity-oriented demonstration：找到每个 entity span eee 对应的context sss ，使用标签 lll 替换 entity span eee ，将替换过的 context sss 使用[SEP]隔开。
    • instance-oriented demonstration：同理，只不过替换所有提及的 eee 。

Model Training with Demonstration

如 Fig 1，拼接后送入编码器中，得到表征：

[h;h~]=[h(1),…h(n),h~(1),…h~(n)]=embed⁡([x;x~])[\mathbf{h} ; \tilde{\mathbf{h}}]=\left[h^{(1)}, \ldots h^{(n)}, \tilde{h}^{(1)}, \ldots \tilde{h}^{(n)}\right]=\operatorname{embed}([\mathbf{x} ; \tilde{\mathbf{x}}]) [h;h~]=[h(1),…h(n),h~(1),…h~(n)]=embed([x;x~])

然后将 h\mathbf hh 送入 CRF 层中。

损失函数为：

L=−∑i=1nlog⁡pθ(y∣h)\mathcal{L}=-\sum_{i=1}^{n} \log p_{\theta}(\mathbf{y} \mid \mathbf{h}) L=−i=1∑n​logpθ​(y∣h)

对于 domain adaptation 设定：

  1. 首先不加模版，在 target set 上微调模型得到 Ms\mathcal M_sMs​
  2. 然后迁移 embedder 权重初始化 Mt\mathcal M_tMt​
  3. 然后使用本文的方法在 test support set 上继续训练 Mt\mathcal M_tMt​
